How to use Image server API

 

Image Server provides a cClient iInterface for Java applications as well as for Visual C++ or Visual Basic applications.

From Java Applications

To use Iimage sServer from Java-based applications, a package named ISPACK.jar is available. The following steps are need to be performed donein order to use Image Server it from java applications:

 

1. The client needs to have following three jars in classpath for making calls to Image Server.

 

§      ispack.jar    

 Provides the exposed API for Image Server calls.

§      jdts.jar   

 It contains supporting objects for Image Server and encapsulates the logic for communication with JTS.

§      nsms.jar      

The jar encapsulates the logic for communication with SMS.

 

ispack.jar     provides the exposed API for Image Server calls.

jdts.jar    it contains supporting objects for Image Server and encapsulates the logic for communication with JTS.

nsms.jar       the jar encapsulates the logic for communication with SMS.

 

Image Server API exposes functions that communicate both with both iImage sServer cCabinet as well as with SMS through a class CImageServer. These functions take parameters both by value and reference. Parameters passed by value are iInput parameters for the call and those passed by reference are oOutput parameters for the call. For passing parameters by reference a wrapper class is available above the primitive types available in java.  For example,  for e.g. iin case of int, wrapper class is JPDBInteger, which is available in JDTS package. The function will update these parameters with the results. The client can extract values from these objects using the exposed API.

 

Steps for call

 

     To describe the steps for call, a sample call For describing the steps a sample call for JPISGetsiteList is takenconsidered.

 

1.       The client essentially imports package Package ISPack. *, as the Package contains all the functions. Because every function is contained in it.

 

import ISPack. *;

 

2.       Import Jdts.DataObject. * Package to create objects for parameters NoOfSitesFnd of type JPDBInteger and SiteList a vector of type JPDBISAddress.This step is required if the call that User is using contains parameters of wrapper class like JPDBInteger, JPDBChar etc.

 

import Jdts.DataObject.*;

 

3.       Create an object of CImageserver class with required parameters.

 

CImageServer  cImageServer  = new CImageServer(null, IpAddress, PortId);

 

 

4.       If the User has to communicate with SMS, then User has to explicitly create a connection for it using the call JPISRegisterConnection. After making the call, User has to disconnect from SMS using the call JPISUnRegisterConnection.

 

5.       Call the function JPISGetSitesList.

 

int status = cImageServer .JPISGetSitesList (cabinetName ,  NoOfSitesFnd,   SiteList);

 

6.       Read number of sites Sites found from NoOfSitesFnd

 

int onOfSitesFount  =  NoOfSitesFnd. Value;

 

7.       Parse the vector SiteList for list of sitesSites.

 

for(int i=0;i<noOfSitesFnd.value;i++)

{

                            site = (JPDBISAddress)siteList.elementAt(i);

                            String sitePath = site. m_strPath;

From Visual C++ / Visual Basic Applications

 

In order to use Image Server on Windows platform, or from Visual C++ / Visual Basic programs, a DLL is available that exposes all the functions.

 

Steps for call

 

For describing the steps, a sample call for NGISGetSitesList is takenconsidered.

 

1.       1. The cClient should include the hHeader file "isrvrapi.h”, because every function exposed by the DLL has its declaration contained in the header Header file.

 

#include "isrvrapi.h"

 

2.       2.If the User has to communicate with both JTS and SMS, then invoke the function NGISRegisterConnection() has to be invoked with the required parameters. If no valid Socket connection to JTS is passed to it, then Tthis function builds connection to the SMS and also to JTS, if no valid socket connection to JTS is passed to it. In that the case, the connection is closed internally closed. But if a valid JTS connection is passed, then it uses that JTS Connection and does n'ot close the connectionit.

 

NGISRegisterConnection(SOCKET * phSock, LPTSTR IPAddress, short PortId, LPTSTR cabinetName,short SiteId)

 

If NGISRegisterConnection is used, then after making the calls , User has   to call NGISUnregisterConnection to disconnect from SMS.

 

3.       3.If interaction is to be done with the Image Cabinet aloneonly and not with SMS, then invoke the function NGISConnectToTS() with the required parameters. This function makes a JTS connection , if a valid JTS connection not passed to it and closes it internally.

 

NGISConnectToTS (SOCKET * phSock, LPTSTR SiteAddress, short PortId).

 

4.       4. Call the function NGISGetSitesList.

 

int stat = NGISGetSitesList ((LPTSTR)(LPCTSTR)m_CabinetName,&NoOfSites,&mylist);

 

5.       5.Get the number of sites Sites and site Site information by traversing the site Site lList.
