Delete Document

 

Deletion of a document from the System is achieved using two distinct API calls.

1.    Delete document property and information about the document from Document Cabinet.

2.       Delete Document data from the Image Server.

 

Step 1:  Delete document from document Cabinet

 

Function Name: getDeleteDocumentExt

 

Class: com.newgen.dmsapi.DMSInputXml

 

Description: This function is used for deleting documents of a Folder. We can delete single as well as multiple documents of a Folder.

 

 

Input Parameters

Sample InputXML

Output Parameters

Sample OutputXML

View xml API::NGOGetDeleteDocumentExt

  

 

Prototype:

public String getDeleteDocumentExt( String cabinetName, String userDBId,

String documentList, String referenceFlag)

 

Input Parameters:

Name

Description

Valid Values

Default Behaviour

cabinetName

Name of the Database Cabinet to which the User wants to connect.

 

N/A

UserDBId

UserDatabase ID, which is returned after the execution of getConnectCabinetXml call.

 

N/A

documentList

Contains information about documents to be deleted from a Folder. This List can be generated using function GetDeleteDocumentIndexList

 

N/A

referenceFlag

Its a flag that indicates whether the documents are to be deleted even if Reference exists. (Currently not in use)

Y-Delete documents even if Reference exists.

N-Do not delete documents if Reference exists.

No check for References of the document is made.

 

Output Parameters:

Input Xml of call NGOGetDeleteDocumentExt.

 

Step 2:  Delete document data from Image Server

 

Function Name: DeleteDoc_MT

 

Class: ISPack. CPISDocumentTxn

 

Description: This function deletes an Active Document from Image Server.

 

Prototype:

public static void DeleteDoc_MT ( JtsConnection jtsCon,String jtsAddress,

short jtsPortId, String cabinetName,

JPISIsIndex   IsIndex)

throws JPISException

 

 

Input Parameters:

Name

Description

Valid Values

Default Behaviour

JtsCon

This Object holds a connection to the JTS. However, it can be sent as Null. If it is set to Null, then a new socket connection to JTS will be created by Image Server Client using jtsAddress and portId, passed as a parameter to the function.

 

N/A

JtsAddress

IP Address on which JTS Server is running.

 

N/A

PortId

The port on which JTS is running.

short>0

N/A

cabinetName

Image Server Cabinet Name.

 

N/A

IsIndex

Object of type JPISIsIndex   which contains volumeid and docindex at ImageServer.
