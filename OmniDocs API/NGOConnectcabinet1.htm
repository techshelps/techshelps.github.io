About the OmniDocs API Manual

 

The OmniDocs API Manuals enables programmers to quickly develop a Document Management System (DMS) application using the OmniDocs Engine, and its features. The API manual describes the OmniDocs Engine and its features and provides an overview of the APIs that are available for developing a DMS application. The OmniDocs API manual not only mentions the advantages of using the OmniDocs but also lists the problems that arise during the development of a DMS Application with appropriate solutions.

 

The OmniDocs API manuals consists of the following sections:  

 

Introduction to OmniDocs – Introduces the programmer to the OmniDocs Server and its components.

 

Features of OmniDocs – Lists all the features that are available with the OmniDocs system.

 

Inside OmniDocs Server – Describes the architecture of the OmniDocs system and the process of communication between various components of the OmniDocs. This section also provides an overview of all the building blocks of OmniDocs .

 

OmniDocs Customization – Enables easy integration of OmniDocs with third party business applications.

 

Terminology – Describes all the keywords  and features used in OmniDocs that helps the programmer to understanding the basic concepts of a DMS application.

 

Application Development using OmniDocs –Describes a step-by-step process for developing a DMS application using the OmniDocs Engine. This section also describes XML calls exposed by the Transaction Server API and Image Server client and provides an overview of workflow and how it can be implemented using the OmniDocs Engine.

 

Installation and Configuration for OmniDocs Component –Explains the steps for installing and configuring components of the OmniDocs Engine.

 

Sample Calls – Helps implementation of some complex calls that enables the programmer to develop a DMS application.

 

Advantages of Using the OmniDocs Engine –Mentions the advantages of using the OmniDocs Engine for rapid development of a DMS application.

 

Troubleshooting and Debugging – Explains general problems that may arise during development of a DMS application with solutions to such problems.

 

Glossary –  Displays a list of terms and their definitions used in the OmniDocs in alphabetical order.

 

Appendix – Lists JTS calls, Image Server calls, errors, warnings, and supporting structure and classes used by Image Server client. This section also contains an XML Library that mentions the sample Inputxml and Outputxml for all xml calls exposed by JTS.
