Basic Setup

The OmniDocs Engine helps a User to build DMS applications. The Installation and Configuration of the OmniDocs components are the pre-requisites of starting the OmniDocs Engine. The steps to configure and deploy the OmniDocs Engine are:

 

1.       Cabinet Creation

The OmniDocs cabinet creation is the first step of using the OmniDocs. A cabinet is a place where most of the business logic and data structure of the OmniDocs System resides. The OmniDocs provides the “OmniDocs Cabinet Manager” utility for creating a cabinet. To know more about the steps of creating a Cabinet click here.

              

              

2.       Image Server Configuration

Image Server stores actual document data and its offset information and is installed as a part of the OmniDocs Engine. Image Server can be configured for storing data on any online or offline devices such as Optical Media, Jukebox, or the Hard Disk. Label is a term that defines the path of the actual data of the OmniDocs System. Any number of Labels can be created and configured with Image Server. At least one Label should be created for using the OmniDocs System. Image Server receives a request made by a client and transfers the request to the JTS for processing. Image Server also receives the document information from the JTS and then transfers the information to SMS for document image retrieval. To know more about the steps of Installation and Configuration of Image Server click here.

 

3.       Java Transaction Server Configuration

Java Transaction Server (JTS) manages the consistency and atomicity of OmniDocs operations. It uses XML standards to provide consistent and optimized interface with Cabinet. For using any Cabinet, it should be first registered or associated with JTS. JTS processes the request made by a client and obtains the document image and information specific to the client’s request. JTS also enables transfer of documents and data between the Shared Cabinets and the OmniDocs Desktop. To know more about the steps for installing and associating a Cabinet with JTS, click here

 

Quick overview of Application development using the OmniDocs Engine

 

The steps for developing any application using the OmniDocs can be summarized in the following steps:

 

§         Install and Configure the OmniDocs Engine.

§         Create InputXml (Java Bean API or COM API may be used).

§         Execute InputXml (Java Bean API or COM API may be used).

§         Use OutputXml to design User Interface (Java Bean API or COM API may be used).

§         Use Image Server Interface to perform storage operations.
