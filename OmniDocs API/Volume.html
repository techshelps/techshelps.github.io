Volume


Each Image Archive is logically divided into one or more Image Volumes, which that can replicate across various Sites. This replication can be either automatic Automatic or manualManual.

 

Volume Block - One or more Image Volume Blocks are created for each Image Volume, which provide the actual physical storage for the documents stored in an Image Volume. Image Server for an Image Volume Block allocates no actual physical space at the time of creation; the space is allocated as and when required. The size of Image Volume Block is specified at the time of volumeVolume creation. This size refers to the maximum size of the document data file that will be created corresponding to each VolBlock in the volumeVolume. Documents are added serially in the document data file until the maximum size is reached. A new VolBlock is created for the next document to be added after the maximum size is reached. This VolBlock size is flexible and can be modified after volumeVolume creation also.

 

If a document, larger in size than the max VolBlock size, is added it is dumped completely in the current vol blockBlock and a new one is created for next document. There is no limit on the physical storage occupied by an Image VolumeVolume.

 

The concatenated string of Site Id , Volume ID and VolBlock ID in Base 36 notation and prefixed by PN forms the name of a document data file in which data of the volblock is stored. These files contain actual document together with iImage cCabinet recovery information as headers.

 

Name of data file created corresponding to a volblock : PN00V0VB.SITE

Volume Replication - A volumeVolume can be replicated over multiple sites .ThisSites. This feature has twofoldhas twofold advantage

1)       Instant Backup - Replication generates immediate backup. In case of loss of data on one siteSite it can be easily recovered from another siteSite.

2)       Faster retrieval - The idea is to Replicate document data to all concerned siteSites so that a client can access data from its local cluster thereby reducing the need for Remote Access.

 

A list of siteSites can be associated with a volume . Volume. With VolumeVolume creation, Image Server gives the option to replicate a volumeVolume on different siteSites. This option can be either Replicate Always or Replicate Later.

Replicate Always specifies Specifies Automatic Replication i.e. Aa newly added document in the volumeVolume will be replicated in these siteSites.

Replicate Later specifies ManualSpecifies Manual Replication i.e. User will send a call explicitly to replicate data in volumeVolume over siteSites.

Image Server API exposes methods to retrieve and modify replica list as well as replication type

over volumeVolume. There also a call to explicitly invoke replication of a volumeVolume on specified siteSite.

 

Volume Compaction Volume Compaction is release of unused disk space by occupied by the Volume. When a document is deleted by SMS fragmentation may happen as the documents added after it areis not shifted. This entire unused space can be revoked by compact volumeVolume calls.

 

Compaction can be done at both the

1)       VolBlock level - At VolBlock level compaction is done only on corresponding document data file.

2)       Volume level. - At Volume level compaction is done in all VolBlocks in the volumeVolume.

 

 

Volume Recovery This functionality is used to recover volumeVolume blockBlock from a PN File. It is used when the PN file exists but its information from the Image Cabinet is lost. It recovers all the information from the header Header information i.e. that is stored in PN file. User has to specify the full path name of PN file on SMS on the format SMS.
