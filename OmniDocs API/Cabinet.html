Cabinet

Cabinet is a starting point for accessing the Document Management System. After connecting to the Cabinet, Users can perform operations on the Cabinet. User can obtain objects of Folders, Documents, Users, and Groups. By default, OmniDocs Cabinet has a User named "Supervisor", and gGroups "Supervisor", "Public", "Guest" and "Everybody". User can create a folder hierarchy; create iImage and nNon-iImage documents into the folder. The User can create categories and User- defined iIndexes and dData cClass for easier and faster retrieval / search for one or more documents. The information pertaining to Cabinet can be stored in Database. Following calls are exposed for cCabinet operations.

 

 

NGOConnectCabinet

For cConnectsing to a Cabinet.

NGODisConnectCabinet

For dDisconnectsing from the Cabinet.

NGOGetCabinetReport

To gGets a report on the Cabinet.

NGOGetCabinetProperty

For rRetrievesing the Cabinet Properties.

NGOChangeCabinetProperty

To mModifies odify the Cabinet Properties.

NGORegisterServices

To rRegisters a sService on the Cabinet.

NGOGetServices

To gGets the lList of Services registered on the Cabinet.

NGOModifyServices

To mModifies y the registered Services on the Cabinet.

NGODeleteServices

To dDeletes the registered Services from the Cabinet.
