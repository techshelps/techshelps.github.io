OmniDocs Transaction Server

 

OmniDocs Transaction Server is middle tier framework for enterprise business document solution. The main responsibilities assumed by the OmniDocs Transaction Server (JTS) are as follows:

 

§OmniDocs Transaction Server provides service implementation whereby; Clients can transparently invoke transactions on Local Area Network or Internet.

 

§It maintains a persistent connection for data exchange with every client. The connection between a client and the transaction server is established only once and remains persistent i.e. there is no need to connect-process-disconnect on a transaction-to-transaction basis.

 

§It also provides a Database Session Manager that manages multiple client requests with limited Database connections.

 

§OmniDocs Transaction Server provides a portable implementation of Server through Java database connectivity (JDBC) with the Database and Socket Layer Interface with the Client.

 

Installation procedure

 

Follow the instructions below to Install OmniDocs Transaction Server.

 

1.       Double Click on OmnidocsServer3.0.exe. This will unpack the exe on the Users machine.

 

 

 

2.       The following screen appears. Click Next to start the installation.

 

 

 

3.       To continue with installation accept the License agreement by clicking on Yes Button.

 

4.       Provide User Information for the machine on which you are installing Server.

 

 

5.       Choose the directory where you want to install server by clicking on browse button. By default it will install in directory C:\Program files\Omni Docs Transaction Server.

 

 

6.       Select the name of program folder that will be added to your programs icon. Default value for it is Newgen OmniDocs. It will copy all the required files on your system.

 

 

7.       Click Finish Button to finish the OmniDocs Transaction Server Installation.
