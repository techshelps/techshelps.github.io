User and Group

 

A person who uses the system System is referred to as User in the Systemsystem. Every User in the System system is a member of some groupGroup. Depending upon the Privileges that the User have, the User can Powers of a User in the system like whether he can create a User or he can add Data Class to the System. system depend upon the privileges that User has.

 

Group is a logical eEntity used for collection of Users. Those Users who have pPrivileges to Add / Delete Group in the System system can create gGroups in the Systemsystem. Rights on any oObject can be specified for gGroups. All members of that gGroup will inherit the same rRights.

 

This section defines the calls related to gGroup and user User operations.

 

User

To Aadd a User to the Systemsystem.

NGOGetUserListExt

To Gget the User lList.

NGODeleteUser

NGOIsAdmin

To Ccheck whether the User is aAdmin.

NGOSearchUser

To Ssearch for Users in the Systemsystem.

NGOGetLoggedInUserLIst

To Gget lList of currently logged in User in the cCabinet.

NGOGetShortUserProperty

To gGet short pProperties for multiple User.

NGOCreateUserTable

To cCreates a User-defined tTable in the database.

NGODropUserTable

To Drops drop the User Table created by using NGOCreateTable call.

NGOUserSQL

NGOUserSQLEx

To perform select operation on User-defined Tables.

NGOValidateUserForSession

Validates a User for a specified Session.
