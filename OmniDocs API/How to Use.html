How to Use

 

 

Java Bean API provides a flexible wrapper over XML API to ease the development of Document Management System. Following steps are used to interface with Java Transaction Server

·          Generate InputXml

·          Execute InputXml (either through JTS or through JNDI)

·          Using OutputXML

API for XML generation

InputXml can be generated using class DMSInputXml in package com.newgen.dmsapi. Functions are exposed for each call. These functions take values for tags that are to be appended in InputXml as input parameters. All input parameters are of String type and the output of all these functions is always a String, which is in standard XML format.

                               

§          Class Name: DMSInputXml

§          Package:  com.newgen.dmsapi

 

Generating Simple InputXml 

                Simple inputxml has tags at root level only.

 

Example

import com.newgen.dmsapi.*;

DMSInputXml inputXml = new DMSInputXml();

                String strInputXml = inputXml.getConnectCabinetXml

("OmniDocs","supervisor","supervisor","","Y", "0");

 

Here variable strInputXml will have the XML to be sent over to JTS for execution.

 

If value for password (3rd parameter) for instance is not to be sent the function invocation will look like.

String strInputXML = inputXml.getConnectCabinetXml

("OmniDocs","supervisor", null,"","Y", "0");

 

Generating List type InputXml

Some calls take Xml list as input. For example set alarms call. This call is for setting a series of alarms. The exposed API call getSetAlarmsXml call takes alarmlist as input parameter. This alarmlist can be generated using a supporting function getAlarmXml().

 

       Example

Suppose a list of 5 alarms is to be generated. GetAlarmXml function will have to be called in loop taking data from array or StringTokenizer as the case may be. This list will then be sent as parameter in getSetAlarmsXml function to get the final executable XML.

 while(i< 5)

 {

alarmList+=inputXml.getAlarmXml(alarmType[i],objectType[i],objectid[i],userindex[i],   actiontype[i],informmode[i],comment[i]);

i++;

 }

 String strInputXML = inputXml.getSetAlarmsXml("OmniDocs","516010592", str);

 

Points to remember

·         For parameters that are not to be appended in inputXml NULL can be sent

 

API for Execution

Execution of OmniDocs API through JTS

 

Class DMSCallBroker communicates with JTS over the socket and returns the OutputXml.

 

·         Class Name: DMSCallBroker

·         Package:  com.newgen.dmsapi

               

The API provides static overloaded execute function in class DMSCallBroker to communicate with JTS.  Invocation of this function is as follows.

 

First prototype takes 4 parameters viz. InputXml, Java Transaction Server Ip and OmniDocs Transaction Server Port and debug level.

Debug level signifies the generation of log. For value 0 of debug level no log is generated and for value 1 of debug level all input xmls are dumped in a file named inputxml.log

 

 

Example

import com.newgen.dmsapi.*;

 

String strOutputXML = DMSCallBroker.execute(strInputXML,

"192.168.5.87",2222,0);

 

Second prototype of this function takes 3 parameters viz. InputXml , Socket connection to Java Transaction Server and debug level. Applications can develop Connection pooling logic of their own and use this function.

 

Example

                import com.newgen.dmsapi.*;

Socket sock = new Socket("192.168.5.87",2222);

String strOutputXml = DMSCallBroker.execute(strInputXml, sock,0);

 

Execution of OmniDocs API through JNDI

Class DMSCallBroker have overloaded method that communicates with JTS over the RMI-IIOP and returns the Output XML as String Object.

 

·         Class Name: DMSCallBroker

·         Package:  com.newgen.dmsapi

 

The API provides static overloaded execute function in class DMSCallBroker to communicate with application server through JNDI.  Invocation of this function is as follows.

 

Method takes only one parameter viz. InputXML, which is of type String.

 

Example

import com.newgen.dmsapi.*;

 

String strOutputXML = DMSCallBroker.execute(strInputXML);

 

API for Using OutputXml

The outputXML received from OmniDocs will have to be parsed to retrieve information.

Parsing is simplified by DMSXmlResponse and DMSXmlList.

 

·         Class Name :DMSOutpuXml

·         Package:  com.newgen.dmsapi

 

Parsing Simple OutputXml

getVal function of DMSXmlResponse is used to retrieve values of tags at root level.

 

<NGOConnectCabinet_Output>

  <Option>NGOConnectCabinet</Option>

  <Status>0</Status>

  <UserDBId>516010592</UserDBId>

…

</NGOConnectCabinet_Output>

 

Example

import com.newgen.dmsapi.*;

Object of DMSXmlResponse is created and then getVal function is invoked for values.

 

DMSXmlResponse connectCabinetResponse = new DMSXmlResponse(strOutputXML);

String userDbId = connectCabinetResponse.getVal(“UserDBId”);

Parsing List type OutputXml

List type OutputXML has repeated tags. A list object is created specifying the parent tag and the child tag name.using createList function of DMSXmlResponse to generate an object EWXmlList type. GetVal function of DMSXmlList can then be called to parse this list.

 

<?xml version="1.0"?>

<NGOGetUserListExt_Output>

  <Option>NGOGetUserListExt</Option>

  <Status>0</Status>

  <NoOfRecordsFetched>12</NoOfRecordsFetched>

  <TotalNoOfRecords>12</TotalNoOfRecords>

<Users>

  <User>

    <UserIndex>212</UserIndex>

    <Name>Anand</Name>

    <PersonalName>xyz</PersonalName>

  </User>

  <User>

    <UserIndex>203</UserIndex>

    <Name>abc</Name>

  </User>

</Users>

 

Following is a brief introduction to the functions used in example.

createList                                - To create an object of DMSXmlList

hasMoreElements                - To check if any more child exists or not.

skip                                           - To jump to next child position.

getVal                                       - To retrieve the value of a tag.

 

import com.newgen.dmsapi.*;

DMSXmlResponse userListResp =

new DMSXmlResponse(retXml);

 for (

DMSXmlList userList = userListXmlResponse.createList("Users","User");

userList.hasMoreElements();               

userList.skip())

{

out.println(userList.getVal("Name"));

}

 

 

Points to remember

·          For a tag for which value does not exist in outputxml empty String is sent as default value.

The parser is case sensitive. To retrieve the value of tag “Status” getVal(“Status”) must be used and not getVal(“status”)
