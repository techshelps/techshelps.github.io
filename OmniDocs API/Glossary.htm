Glossary

It provides a quick view of terms used in OmniDocs system in alphabetical order.

 

A

 

Annotation

A tool for marking information on Documents in order to draw User's attention on it.

Annotation Groups

Annotations applied to a Document can be clubbed together as a Group. User can perform annotations collectively.

Alarms

Alerts that are used to notify Users that specified operation has been performed

Attachments

Any document attached with a document is an attachment.

Audit Log

Log generated for the actions performed by the Users.

 

B

 

Batch

The User specified number for retrieving documents, folders, keywords, and search results in a batch.

 

C

 

Copy

Copy them object like folder or document to some other place.

Connecting to cabinet

Establishes the connection to the Cabinet so further calls can be made to the cabinet.

CheckOut

If the User wants to do some changes in a document then he can check out that document and after that nobody else can do any changes in that document till the time it is checked out.

CheckIn

After doing the required changes in a document a User can CheckIn the document checked out by him and a new Version of the document will be created.

 D

 

DataBase Cabinet

It is the path to the database where folders and document are stored

Data Class

The User defined information associated with document or folders. It contains field or indexes.

Document

An Image or Non Image file that is stored in the folders.

DataClass (Data Definition)

Data Definitions are set of indexes that can be associated with the documents or folders thereby providing a unique entity to them.

 

F

 

Folder

Repository for storing the documents. User can create the folders under a cabinet or other folders.

 

G

 

Global Indexes

Global Indexes are User-defined fields that could be associated to any document across different Data Classes. They facilitates in search across the cabinet.

Group

The Users are clubbed together as a Group.

H

 

Home Site

Parent Site in which a volume is added.

 

I

 

Image Server

Image server compromise of storage management server and its interface to communicate with it.

 

K

 

Keyword

A keyword is a word that User would like to associate with a document, so that User can

Later perform search for the documents with which that keyword is attached.

 

L

 

Linked Documents

A document can be linked with other documents such that when User sees the document next time list of linked documents is also displayed.

Label

A Label basically represents the physical path where the data files corresponding to volblocks will be stored.

P

 

Privileges

Power of a User to do some operations in a system is given by Privileges.

PickList

It is a list of values that are already associated with document or folder for a particular field of Data Class.

Preferred Site

Image Server clients while requesting for a connection specifies a site, to which it wants to get connected. This site is called the Preferred Site.

 

 R

 

Rights

Specifies the rights of a User on a given object thereby helping the User in getting his objects shared with other Users in the manner he wants.

Reminders

Reminders are alerts set on Documents, which are based on Date and Time.

References

Quick reference to an object like folder or document is termed as reference of the object

Remote Site

All the sites in a system other than the preferred site are the remote sites in which the Image Server looks for data if it is not found on the Preferred Site.

 

S

 

Sharing

Providing access to other Users in the system on objects is referred as sharing of object

Search

Quick access to some objects, which satisfy a certain criteria. The search can be made for Folders or Documents.

SMS

SMS is the Storage Management Server, which listens for requests from Image server client and consequently performs document addition or retrieval or any other operation on request.

Site

A Site represents a SMS and is a collection of volumes.

 

U

 

User

A person who uses the System is referred as User in the System.

      

V

 

Versioning

The process of creating a new Version of the document by checking in the document.

Volume

A Volume is a logical unit, which can have multiple Volume Blocks.

Volume Block

Volume Blocks provide the actual physical storage for the documents stored in an Image Volume.

VolBlockPath

It represents the path of the actual data file on the SMS, corresponding to a volblock.

Volume Replication

The process of replication of volume over multiple sites.

Volume Compaction

It is the release of unused disk space occupied by the Volume.

Volume Recovery

Image Server provides a mechanism for recovery of Image Cabinet data that is used when the data file corresponding to a volblock exists but its information from the Image Cabinet is lost.
