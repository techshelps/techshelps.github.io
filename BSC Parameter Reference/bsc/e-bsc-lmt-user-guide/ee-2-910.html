Collecting Statistics on DTRU Configuration By BM

This describes how to measure the number of DTRUs using one TRU, maximum number of TRXs that can be added, and number of TRUs that can be added through license application by a BM subrack.

Prerequisites

  • The LMT runs normally.
  • The server is functional.
  • The communication between the LMT and the BSC is normal.

Context

On the LMT, the GMPS and GEPS are called BM.

The DTRU configuration files exported by BM are in *.xls format. The default save path is LMT installation directory\BSC6000\version number\BackupCfgData.

  • TRXs that can be added (reference value) = Maximum number of TRXs of BM - Number of currently configured TRXs of BM The calculated value is a theoretical data on the basis that the number of TRXs is given in full-rate state.

    When the BM and TC are located separately, the maximum number of TRXs supported by the BM is 512. When the BM and TC are located in the same subrack, the maximum number of TRXs supported by the BM is 786.

  • TRUs that can be added through license application uses a smaller value of Number of DTRUs using one TRU and TRXs that can be added (reference value).

Procedure

  1. Choose Configuration > DTRU Configuration Statistics > DTRU Configuration Statistics By BM. A dialog box is displayed, as shown in Figure 1.

    Figure 1 DTRU Configuration Statistics by BM dialog box

  2. Right-click the table list area, and then choose Export. The Save As dialog box is displayed.
  3. Type the file name, and then click Save.
