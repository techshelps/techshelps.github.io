Tracing IP Messages

In the IP transmission mode, this describes how to trace IP messages between the BSC and BTS, the MSC and BSC, or between the interface boards in the BSC on the LMT.

Prerequisites

  • The LMT runs normally.
  • The communication between the LMT and the BSC is normal.

Context

  • For details of Message Type, refer to Message Types.
  • For details of Tracing Mode, refer to Tracing Modes.

In the Trace IP Message dialog box, specify the location parameters. Table 1 lists the parameters shown in the Trace IP Message dialog box.

Table 1 Parameters shown in the Trace IP Message dialog box

Parameter

Meaning

IP Information

Source IP Address

Source IP address by which the datagram packet is sent, for example, the port IP address at the BSC side is 172.16.5.156. This parameter is set according to the actual requirements.

Destination IP Address

Destination IP address to which the datagram packet is sent. The port IP address at the BTS side, for example, is 172.20.7.40. This parameter is set according to the actual requirements.

Source Port No./Destination Port No.

Source port number/destination port number of the OML, RSL or EML link. For example, the source port number of an RSL link is 12288, and the destination port number of the RSL link is 12288. This parameter is set according to the actual requirements.

Value range: 0-65535.

Protocol Type

Consisting of the UDP and SCTP protocols. The protocol type used depends on the services.

Position Information

Subrack No./Slot No.

Subrack number and slot number of the IP-based interface board or transmission optimization interface board.

The following description is based on the configuration using the GOMU.

Procedure

  1. On the Trace & Monitor tab page, choose Trace > Trace CS Message > Trace IP Message. A dialog box is displayed, as shown in Figure 1.

    Figure 1 Trace IP Message dialog box

  2. In the IP Information area, enter the corresponding information. Then, set the parameters in the Position Information and Tracing Mode areas.
  3. Click OK.

Result

If the operation is successful:

  • If the Tracing Mode is set to Save on GOMU, the LMT does not display the traced messages. You can view the output file of the traced messages. For details, refer to Reviewing Trace Message Files.
  • If the Tracing Mode is set to Report, a dialog box is displayed, prompting the information of the message tracing task.

If the operation fails, a dialog box is displayed, prompting the possible causes.

Postrequisite

You can double-click a message to view the detailed information.

Right-click the message to be analyzed. The shortcut menu provides the following options:
  • Clear Message: Clear the displayed messages.
  • Stop Scroll: Stop scrolling the displayed messages.
  • Stop Trace: Stop the current tracing task.
  • Restart Trace: Restart the tracing task.
  • View Task: View the established tracing task.
  • Search: Search for the specified character string in the displayed messages.
  • Save Selection: Save the selected messages.
  • Save All: Save all the traced messages.
