Unlocking the User Account of BSC6000 LMT

When logging in to the BSC6000 LMT, if a non-ADMIN user enters an incorrect password over a certain times, the non-ADMIN user account is automatically locked. The ADMIN users, administrators, and custom users can unlock these accounts.

Prerequisite

You have logged in to the LMT in online mode.

Context

For details on the user classification, refer to User Authority.

The following procedure describes how ADMIN users, administrators, and custom users log in to the BSC6000 again, so as to unlock the non-ADMIN user account.

Procedure

  • Through GUI
    1. Choose System > Unlock User Account. A dialog box is displayed, as shown in Figure 1.

      Figure 1 Unlock User Account dialog box

    2. Enter User Account, and then click Unlock.

      NOTE:
      The BSC6000 Local Maintenance Terminal provides the auto unlock function. When the Auto Unlock Time expires, the user account is unlocked automatically. Then the user can enter the user name and password again. For details on setting Auto Unlock Time, refer to Setting the Password Policy on the BSC6000 Local Maintenance Terminal.

  • Through MML
    1. On the LMT, run the ULK USER command to unlock the user account.
