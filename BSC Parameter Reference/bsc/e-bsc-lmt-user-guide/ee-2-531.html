Monitoring BTS Resources

This describes how to monitor the resource usage of the TRX and of the main control board on the LMT.

Prerequisites

  • The LMT runs normally.
  • The communication between the LMT and the BSC is normal.
  • The communication between the BSC and the BTS is normal.

Context

The type and status of resources are shown in Table 1.

Table 1 Parameters of BTS resources

Resource Name

Description

CPU usage

CPU usage of the TRX and of the main control board of a BTS.

RAM usage

Memory usage of the main control board of a BTS.

Mother-tier voltage

DC outputs of rectifying unit.

When the voltage is too high, the power monitoring module reports an alarm about the over-high voltage to the BSC through the main control board.

Temperature inside the cabinet

Line obtained based on the real-time monitor of the temperature inside the cabinet.

Humidity inside the cabinet

Line obtained based on the real-time monitor of the humidity inside the cabinet.

Current of storage batteries

Line obtained based on the real-time monitor of the batteries current.

Temperature of storage batteries

Line obtained based on the real-time monitor of the batteries temperature.

Load current

Line obtained based on the real-time monitor of the internal current of the BTS.

View AC voltage

Line obtained based on the real-time monitor of the AC voltage provided for the BTS.

Quantity of state

Charging and discharging status of the BTS batteries. This attribute is realized by the power monitoring module.

Board temperature

Line obtained based on the real-time monitor of the board temperature of the BTS.

The procedures for monitoring different resources are similar. This topic takes monitoring the CPU usage as an example.

Procedure

  • Through GUI
    1. Choose BTS Maintenance > Maintain Site > Monitor BTS Resource. A dialog box is displayed, as shown in Figure 1.

      Figure 1 Monitor BTS Resource dialog box

    2. Set Select Site and Resource Type.
    3. Click OK. The results are displayed, as shown in Figure 2.

      Figure 2 Monitoring the CPU usage

  • Through MML
    1. Run the DSP BTSRES command on the LMT to query the BTS resources.
