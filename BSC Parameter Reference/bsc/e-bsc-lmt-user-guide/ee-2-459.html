Querying Status of MTP2 Links

This describes how to query the status of MTP2 links.

Prerequisites

  • The LMT runs normally.
  • The communication between the LMT and the BSC is normal.
  • The GEIUT and the GOIUT are functional.

Context

Table 1 describes the seven states of MTP2 links.
Table 1 Seven states of MTP2 links

State

Description

No Link Configuration

The MTP2 link is not configured or the MTP3 initial message is not received.

Out of Service

The MTP2 link is disrupted and the links cannot carry services.

Start Locating

The MTP3 start command is received and the locating starts.

Locating Not Ready

The locating is not ready in either of the following conditions:

  • In Start Locating state, a local processor fault occurs before the periodic check starts.
  • In Locating Ready state, a local processor fault occurs.

Locating Ready

The locating is ready in either of the following conditions: ÔÅ¨

  • In the Start Locating state, no local processor fault occurs before the periodic check starts.
  • In the Locating Not Ready state, the local processor restores.

Service Ready

The link can carry services.

Procedure

  1. On the LMT, choose BSC Maintenance > Maintain Transmission and Signaling > Maintain MTP2 Link.
  2. Set parameters in the Location area, and then click Query. The query result is displayed, as shown in Figure 1.
