Querying the Active Alarms

This describes how to query active alarms and receive the reported alarms in real time. In addition, you can delete unimportant alarms by performing the Manually Clear Alarm operation and obtain the ways of clearing the alarms by viewing Alarm Detail.

Prerequisites

  • The LMT runs normally.
  • The communication between the LMT and the BSC is normal.

Context

If the alarms displayed on the LMT are inconsistent with the alarms on the server, the is displayed on the upper right of the Browse Alarm window, requesting you to update the existing alarm data.

Procedure

  • Through GUI
    1. Choose Alarm Maintenance > Browse Alarm, or click . A window is displayed, as shown in Figure 1.

      Figure 1 Browse Alarm window

    2. Optional: If you choose Alarm Maintenance > Display Event Alarm, all the active alarms and event alarms are displayed, as shown in Figure 2.

      Figure 2 Window of browsing alarms including the event alarm

    3. Optional: In the output list, click Alarm Serial Number, Alarm Name, Alarm Raised Time, Alarm ID, Alarm Severity, Event Type, or Location info. The alarms are displayed in a certain order. There are multiple orders to display alarms.
    4. Right-click an alarm shown in Figure 2 to perform the following operations:

      • Manually Clear Alarm: Manually clear an alarm. This operation can only clear the alarms displayed on the LMT.
      • Save All Alarms: Save the query results of all the alarms.
      • Save Selected Alarms: Save the query result of selected alarms.
      • Set Show Columns: Set the alarm information to be displayed.
      • Alarm Detail: Copy alarm information and query alarm details.
      • Filter Alarms: Set alarm filtering conditions.
      • Refresh: Refresh all the results of querying the alarms.
      • Auto Scroll: Automatically scroll all the results of querying the alarms.

      NOTE:

      The manual removal of alarms can only clear the alarm items displayed on the LMT, it cannot rectify the fault corresponding to the alarm. The fault needs to be rectified by engineers.

      Right-click an alarm, and then choose Alarm Detail. The Alarm Detail Information dialog box is displayed. Click the Alarm Detail button. The information relevant to the alarm is displayed, such as description, attribute, parameters, impact on system, system actions, possible causes, and handling procedure.

  • Through MML
    1. You can run the following commands on the Local Maintenance Terminal to view the alarm information.

      • Run the LST ALMAF command to query the active alarms.
      • Run the RMV ALMFLT command to manually clear alarms.
