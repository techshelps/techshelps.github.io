Alarm Location Information

This describes the location information that is used to locate alarms. The location information consists of the alarm ID, alarm serial number, alarm synchronization serial number, location information, and alarm raised time.

Alarm ID

The alarm ID is the unique index of an alarm and the main keyword of the alarm configuration table. For details, refer to Alarm ID.

Alarm Serial Number

The alarm serial number is arranged based on the time when the alarm is raised. The alarm serial number is the only index in the alarm logs. The alarm generated and the alarm cleared use the same alarm serial number.

Synchronization Serial Number

The alarm serial number is arranged on the basis of the time when the alarm is generated. The synchronization serial number ranges from 1 to 2147483646 in a cyclical order.

Location Information

Location information is the information used to locate alarms. The location information in the alarm recovery report is the same as that in the corresponding fault alarm report.

Alarm Raised Time

Alarm raised time is the time when a fault alarm or an event alarm is generated.

Recovery Type

Four recovery types are available for a fault alarm.

  • Normal recovery: A recovery alarm is received and the fault alarm is cleared.
  • Reset recovery: After the BSC6000 is restarted, the original fault alarm is automatically cleared. Check whether the fault persists.
  • Configuration recovery: After the alarm object is deleted, the alarm related to this object is cleared.
  • Correlation recovery: When processing parent-child correlation, the fault management system reports the parent alarm and clears the child alarm if it receives a parent alarm whose child alarm is not cleared. If the parent correlation function is disabled, the correlation recovery type is unavailable.
