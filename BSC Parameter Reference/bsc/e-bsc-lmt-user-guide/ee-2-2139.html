Rolling Back the Version Software

This describes how to roll back the BSC software. Rollback is the reverse process to the activation. After the rollback, the software returns to the status before the activation is performed.

Prerequisites

  • The LMT runs normally.
  • The communication between the LMT and the BSC is normal.
  • The communication within the BSC is normal.

Context

  • The rollback is performed when errors occur during the activation or when the activated software needs to return to the former status.
  • Before the rollback, ensure that the software stored in the standby area of the server is of the version that needs to be rolled back.
  • If the software of the first version is running on the board, or the software of a new version is downloaded but not yet loaded to the board, then the rollback fails.

Procedure

  1. Run RBK SOFTWARE on the LMT to roll back the version software.
