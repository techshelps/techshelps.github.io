Tracing PS Messages

This describes how to trace messages on the Gb interface, Um interface and User messages.

Context

During PS message tracing, five tracing tasks with different location information can be performed simultaneously.
  • Tracing the Messages on the Gb Interface
    This describes how to trace the PTP messages and SIG messages on the Gb interface. Checking the information about the interaction between the PCU and each type of SGSN signaling, which can help identifying the PS problem.
  • Tracing PS Domain Messages on the Um Interface
    This describes how to trace the PS domain messages on the Um interface based on the specified TRX or cell on the LMT when the PCU is installed inside the BSC. The messages consist of DUMMY message, system message, null paging message, and data block message. It can be used to check the information about the signaling interaction process of the PS service between the BTS and the MS, which helps identifying the PS problem.
  • Tracing the PS User Messages
    This describes how to trace the Gb/Abis/Um signaling messages or the Um data blocks of a specified user on the LMT. The user is specified by the IMSI or TLLI. It is applicable to the process of identifying problems, such as specifying the dialing test, which helps identifying the PS problem.
