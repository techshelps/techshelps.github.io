Managing the Version Software

This describes how to manage version software and ensure the compatibility of hardware version and software version by downloading, loading, activating, and rolling back version software.

  • Downloading and Decompressing the Version Software
    This describes how to download and decompress the software package from the FTP server to the BSC server. When the downloading is complete, the BSC server starts the decompression of the software package automatically.
  • Loading the Version Software
    This describes how to load the version software from the BSC server to the flash memory of the boards where the software runs. The version software is forcibly loaded.
  • Activating the Version Software
    This describes how to activate the version software that is loaded on each service board.
  • Rolling Back the Version Software
    This describes how to roll back the BSC software. Rollback is the reverse process to the activation. After the rollback, the software returns to the status before the activation is performed.
  • Loading Configuration Files
    This describes how to load the data configuration files from the server to each board.
  • Querying the Status of the Version Software
    This describes how to query the status of the version software running or stored on the BSC server.
