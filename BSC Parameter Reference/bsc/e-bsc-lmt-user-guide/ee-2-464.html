Querying the Status of the Abis Interface Timeslot

This describes how to query the status of the Abis interface timeslot, such as timeslot number, timeslot type, and timeslot state of the Abis interface circuits.

Prerequisites

  • The LMT runs normally.
  • The communication between the LMT and the BSC is normal.
  • The GEIUB/GOIUB, GFGUB/GOGUB,or the GEHUB is functional.
  • The GXPUM works normally.

Context

One GEIUB has 32 ports. Each of them corresponds to a site. The BSC6000 can provide 256 timeslots of 8 kbit/s or 128 timeslots of 16 kbit/s on the Abis interface.

Table 1 lists the parameters about the status of the timeslots on the Abis interface.

Table 1 Parameters about the status of the timeslots on the Abis interface

Parameter

Description

Site Name

Name of the site corresponding to the selected port

TRX No.

Number of the TRX configured in a cell

Channel No.

When the channel type is half-rate, all the channel numbers are same. When the channel type is full-rate or PS, the channel numbers are different.

Sub-channel No.

The Sub-channel No. of half-rate channels is 1 or 2. The Sub-channel No. of full-rate channels or PS domain channels is 0.

Timeslot No.

Numbered 0-256. When the channel type is full-rate, the timeslot on each Abis interface is 8 kbit/s and each timeslot number matches one timeslot. When the channel type is half-rate, the timeslot on each Abis interface is 16 kbit/s and every two timeslot numbers match one timeslot.

Type

Half Rate, Full Rate

State

Occupied, Failure, Uninstalled, and Idle:

  • Idle: The current timeslot does not carry services.
  • Occupied: Services are transmitted on the current timeslot.
  • Failure: The current timeslot is not functional.
  • Uninstalled: The current timeslot is not assigned to corresponding channel.

Free Timeslot Index

The value is N/A on the LMT.

Procedure

  • Through GUI
    1. On the LMT, choose BSC Maintenance > Maintain User Resource > Query Abis Interface Timeslot State. The Query Abis Interface Timeslot State dialog box is displayed, as shown in Figure 1.
    2. Set the parameters in the Abis Interface Location area, and then click Query. The query result is displayed, as shown in Figure 1.

      Figure 1 Querying the Abis interface timeslot state

      NOTE:
      • In the Abis Interface Location area, if State is selected, the result will be displayed based on the filtering condition selected.
      • If you do not select State, all the timeslots on the Abis interface is queried.

  • Through MML
    1. Run the DSP ABISTS command on the Local Maintenance Terminal to query status of timeslots on the Abis interface.
