Tracing the PS User Messages

This describes how to trace the Gb/Abis/Um signaling messages or the Um data blocks of a specified user on the LMT. The user is specified by the IMSI or TLLI. It is applicable to the process of identifying problems, such as specifying the dialing test, which helps identifying the PS problem.

Prerequisites

  • The LMT runs normally.
  • The communication between the BSC and the BTS is normal.
  • The communication between the BSC and the SGSN is normal.

Context

  • Each BSC6000 LMT supports a maximum of five tracing tasks of a PS user.
  • For details of Tracing Mode, refer to Tracing Modes.
  • The following description is based on the configuration using the GOMU.

Procedure

  1. On the Trace & Monitor tab page, choose Trace > Trace PS Message > Trace PS User Message. A dialog box is displayed, as shown in Figure 1.

    Figure 1 Trace Ps User Message dialog box

  2. Select Interface Type and Tracing Mode, and set the IMSI or TLLI of the user to be traced.

    NOTE:

    To select Trace Um Data Block, you need to set Interface Type to Um Interface first.

    • If you select IMSI, you must determine the reassignment strategy on the MSC side. If the MS initiates an IMSI call, you can start tracing by IMSI.
    • If you select TLLI, you must query the TLLI of the MS. You can run the DSP MSCONTEXT command on the LMT to query the TLLI of the MS.

  3. Click OK to start tracing.

Result

  • If the operation is successful:
    • If the tracing mode is Save on GOMU, the LMT does not display the traced messages. You can review the output file of the traced message. For details, refer to Reviewing Trace Message Files.
    • If the tracing mode is Report, a dialog box is displayed, prompting the information of the message tracing task, as shown in Figure 2.
      Figure 2 Tracing user messages

  • If the operation fails, a dialog box is displayed, prompting the possible causes.

Postrequisite

You can double-click a message to view the detailed information.

Right-click the message to be analyzed. The shortcut menu provides the following options:
  • Clear Message: Clear the displayed messages.
  • Stop Scroll: Stop scrolling the displayed messages.
  • Stop Trace: Stop the current tracing task.
  • Restart Trace: Restart the tracing task.
  • View Task: View the established tracing task.
  • Search: Search for the specified character string in the displayed messages.
  • Save Selection: Save the selected messages.
  • Save All: Save all the traced messages.
