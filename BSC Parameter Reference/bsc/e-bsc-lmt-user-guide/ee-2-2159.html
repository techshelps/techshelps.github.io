Tracing CS Messages

This describes how to trace the CS messages such as the messages on the A interface, Abis interface, Pb interface (when the external PCU is configured), Um interface, and BSC-CBC interface, user messages, group call messages, and IP messages.

  • Tracing the Messages on the A Interface
    This describes how to trace the BSSAP, SCCP, MTP3, MTP2, SCTP,and M3UA messages on the A interface.
  • Tracing the Messages on the Abis Interface
    This describes how to trace the RSL, OML, ESL,EML and LAPD messages on the Abis interface.
  • Tracing the Messages on the Pb Interface
    When the PCU is in external mode, you can trace the messages on the Pb interface on the BSC6000 Local Maintenance Terminal. This describes how to trace the application messages and LAPD messages on the Pb interface.
  • Tracing CS Domain Messages on the Um Interface
    This describes how to trace the CS domain layer 3 signaling messages on the Um interface by TRX or by cell on the LMT. The messages can either be paging messages or other types of messages.
  • Tracing CS Domain User Messages
    This describes how to trace signaling messages on the A/Abis/Um interface on the LMT. Using the IMSI, IMEI, TMSI, MSISDN, or channel to specify a user to be traced
  • Tracing Messages on the BSC-CBC Interface
    This describes how to trace signaling messages of the interfaces between the BSC and CBC through the LMT and identify the CBC problem.
  • Tracing Group Call Messages
    This describes how to trace the VGCS message and VBS message on the A/Abis/Um interface on the LMT.
  • Tracing IP Messages
    In the IP transmission mode, this describes how to trace IP messages between the BSC and BTS, the MSC and BSC, or between the interface boards in the BSC on the LMT.
