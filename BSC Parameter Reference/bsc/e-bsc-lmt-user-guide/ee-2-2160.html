Tracing SCTP Messages on the A Interface

This describes how to trace SCTP messages based on specified subrack, slot, CPU number, and link number on the LMT. The messages consist of control message, data message, and heartbeat message.

Prerequisites

  • The LMT runs normally.
  • The communication between the LMT and the BSC is normal.
  • An SCTP link is configured.

Context

For details of Message Type, refer to Message Types.

For details of Tracing Mode, refer to Tracing Modes.

The following description is based on the configuration using the GOMU.

Procedure

  1. On the Trace & Monitor tab page, choose Trace > Trace CS Message > Trace A Interface Message > SCTP Message. A dialog box is displayed, as shown in Figure 1.

    Figure 1 Trace SCTP Message on the A Interface dialog box

  2. Set the parameters in the Message Type, Location, and Tracing Mode areas, and select Save File.
  3. Click OK.

Result

  • If the operation is successful:
    • If the tracing mode is Save on GOMU/GBAM, the LMT does not display the traced messages. You can review the output file of the traced message. For details, refer to Reviewing Trace Message Files.
    • If the tracing mode is Report, a dialog box is displayed, prompting the information of the message tracing task, as shown in Figure 2.
      Figure 2 Tracing SCTP messages on the A interface

  • If the operation fails, a dialog box is displayed, prompting the possible causes.

Postrequisite

You can double-click a message to view the detailed information.

Right-click the message to be analyzed. The shortcut menu provides the following options:
  • Clear Message: Clear the displayed messages.
  • Stop Scroll: Stop scrolling the displayed messages.
  • Stop Trace: Stop the current tracing task.
  • Restart Trace: Restart the tracing task.
  • View Task: View the established tracing task.
  • Search: Search for the specified character string in the displayed messages.
  • Save Selection: Save the selected messages.
  • Save All: Save all the traced messages.
