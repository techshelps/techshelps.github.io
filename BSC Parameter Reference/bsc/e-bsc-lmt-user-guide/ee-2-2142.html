Managing Patches

Management of patches is performed through operations such as downloading, loading, activating, confirming, deleting, and rollback.

Context

CAUTION:

You should activate/deactivate the patches and confirm the operations in low traffic hours (generally early on the morning).

When activating/deactivating the patches and confirming the operations, pay attention to the following:
  • Ensure that the GBAM/GOMU operates normally. If the GBAM/GOMU does not operate normally, do not perform any operation associated with the patches.
  • Confirm the load mode of the GXPUM/GXPUT, AUTO or SERVER.
  • Downloading and Decompressing the Patch
    This describes how to download the patch package from the FTP server to the BSC server and decompress the package.
  • Loading the Patch
    This describes how to load the patch from the patch area of the server to the patch area in the memory of a board. The patch file is also saved in the flash memory of a board.
  • Activating the Patch
    This describes how to activate a patch in the patch area of a board.
  • Confirming the Status of the Patch
    This describes how to confirm the status of a patch that is activated and effective.
  • Deactivating the Patch
    This describes how to deactivate a valid patch in the patch area of a board.
  • Rolling Back the Patch
    This describes how to roll back a patch to the status before last loading of the patch.
  • Deleting the Patch
    This describes how to delete patch files. All patch files of a BSC, that is, all patch files on the server and boards, are deleted together.
