Importing Site Data

This describes how to import the site parameters in batches to improve the configuration efficiency.

Prerequisite

  • The LMT runs normally.

Context

When you start the LMT in offline mode, you can import the site data from the local PC to the LMT. When you log in to the LMT in online mode, you can import the site data to the server directly.

CAUTION:
  • You can import the site data only as *.sxd files. You can import only the site data files that match the version of the current system. During the import:
  • In an operation of the basic site import/export unit, the data must be both imported and exported.
  • The manually allocated timeslot of the imported data cannot be identified by the LMT.
  • The adjacent relation is not imported. The adjacent relation can be imported through the Import Network Plan Parameter function.
  • If the CGI in the imported data is the same with the CGI of an external cell in the same BSC, delete the external cell. If the CGI in the imported data is the same with the CGI of an internal cell in the same BSC, the import fails.
  • If the cell type is Enhance Dual Band cell, the import fails.
NOTE:
The BTSs and sub BTSs connected to a BSC form a complete unit, this unit is called the basic BTS import/export unit.

Procedure

  1. Choose Configuration > Import or Export Data > Import Site Data.
  2. Select the data file that needs to be imported and click Open. A confirm dialog box is displayed, prompting whether the data to be imported is correct.
  3. Click OK.

Postrequisite

If the import fails, the failure information is displayed. You can check the error information in ImportErr_Freq.log file in LMT installation directory\version number\Cfg\SysData\DataImportLog, modify the parameters, and import them again.
