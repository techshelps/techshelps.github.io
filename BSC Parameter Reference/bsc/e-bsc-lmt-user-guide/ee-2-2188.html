Backing Up the GOMU Data Files

This describes how to manually back up the files in the active GOMU to the standby GOMU.

Prerequisites

  • The LMT runs normally.
  • The communication between the LMT and the BSC is normal.
  • The standby GOMU is operational.
  • The DAEMON process on the standby GOMU is started.
  • The DAEMON versions of the active and standby GOMUs are the same.

Context

  • The directories that you can manually back up are as follows:

    \BSC6000\bscswm\area1\software\

    \BSC6000\bscswm\area2\software\

    \BSC6000\bscswm\area1\extbios\

    \BSC6000\bscswm\area2\extbios\

    \BSC6000\data\datafile\cdm\

    \BSC6000\data\mtndata\pfm\tsk\

    \BSC6000\bscswm\area1\license\

    \BSC6000\data\datafile\alm\

    \BSC6000\data\mtndata\log\opr\

  • Do not change the names and directories of the files after they are backed up to the standby board.
  • If the backup function is disabled, you cannot perform backup.

Procedure

  1. Check the status of the swapping switch on the standby GOMU.

    Status

    Operation

    		 
    backup switch = enable backup

    On the standby GOMU, swapping is enabled. Go to 4.

  2. On the LMT, run the DSP OMUBAKSTAT command to query the status of the standby GOMU.
  3. View the status of the standby GOMU.

    Status of the Standby Board

    Operation

    		 
    Backup stat of Daemon = No Backup

    You cannot back up the GOMU data files because the GOMU does not have a standby board. End the operation.

    		 
    Backup stat of Daemon = Initial Backup

    The GOMU is being started. After the GOMU is started, go to 4.

    		 
    Backup stat of Daemon = Daily Backup State

    The standby GOMU is operational. Go to 4.

  4. Run BAK OMUALL on the LMT to manually back up data files to the standby GOMU.
