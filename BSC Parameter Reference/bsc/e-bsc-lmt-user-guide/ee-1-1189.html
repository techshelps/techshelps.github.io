Process of Running MML Commands

This describes the process of running MML commands.

NOTE:
When the Local Maintenance Terminal is running commands, no data is verified. After all the commands are issued to the server, the server verifies the data.

The process of running MML commands is as follows:

  1. MML commands stand in a queue to wait for the server to run the commands.
  2. The server has an Application (APP) unit that lists the commands to be run, and each service board has their own APP unit.
  3. The server issues a command to all the service boards.
  4. The service boards check whether the command is listed in their own APP unit. If the command is listed in a service board, the service board instead of other service boards runs the command.
