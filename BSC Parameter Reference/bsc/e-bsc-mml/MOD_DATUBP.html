Modify DATU|GATM Board Parameters(MOD DATUBP)

Function

A configuration command that modifies the parameters of the DATU/GATM. In this command, the following parameters uniquely identify a board:

BTS Index or BTS Name

Subrack No.

Slot No.

Note

1. BTS3012, BTS3012AE, and BTS3012_II support configuring the DATU.

2. DBS3900 GSM, BTS3900 GSM, BTS3900A GSM, DBS3036, BTS3036, and BTS3036A support configuring the GATM.

3. BTS3006C and BTS3002E support configuring the DATM.

4. When Board Parameter Configuration Enabled is set to NO(NO), other board parameters are reset with the default values.

Parameters

ID Name Description
IDXTYPE Index Type

Meaning:Index Type

Value range:BYNAME(By Name),BYIDX(By Index)

Explanation:Index type.

Default value:None
BTSIDX BTS Index

Meaning:BTS Index

Value range:0~2047

Explanation:This parameter specifies the index number of a BTS. Each BTS is numbered uniquely in a BSC.

Default value:None
BTSNAME BTS Name

Meaning:BTS Name

Value range:1-32 characters

Explanation:This parameter specifies the name of a BTS. The name of each BTS is unique in a BSC. Excluding the invalid characters ';',',','"','''','=','\','%%','+++','&','#'.

Default value:None
SRN Subrack No.

Meaning:Subrack No.

Value range:0~53

Explanation:None.

Default value:None
SN Slot No.

Meaning:Slot No.

Value range:0~23

Explanation:None.

Default value:None
CFGFLAG Board Parameter Configuration Enabled

Meaning:Board Parameter Configuration Enabled

Value range:NO(NO),YES(YES)

Explanation:None.

Default value:None
AMPC0 Feed Tri. 0 TMA Configuration

Meaning:Feed Tri. 0 TMA Configuration

Value range:DISABLED(Disabled),ENABLED(Enabled)

Explanation:This parameter specifies whether the DATU or GATM supplies power to the TMA on feed tributary 0.

Default value:None
MODE0 Feed Tri. 0 Alarm Mode

Meaning:Feed Tri. 0 Alarm Mode

Value range:MODE1(Mode1),MODE2(Mode2),MODE3(Mode3)

Explanation:This parameter specifies the mode used by the DATU or GATM to report an alarm about feed tributary 0. The three modes are as follows:

Mode 1: In this mode, there is only one alarm severity. The current for alarming is constant and higher than the normal operating current.

Mode 2: In this mode, there are two alarm severities: warning alarm and critical alarm. The current for alarming is constant and higher than the normal operating current.

Mode 3: In this mode, there are two alarm severalties: warning alarm and critical alarm. The current for reporting a warning alarm is a periodic pulse. The current for reporting a critical alarm is a constant current.

Default value:None
MAJORALMUP0 Feed Tri. 0 TMA Over Critical Alm Thred.

Meaning:Feed Tri. 0 TMA Over Critical Alm Thred.

Value range:0~2000

Explanation:This parameter specifies the threshold for reporting a TMA Feed Current Critical Alarm when the feed current of the TMA on feed tributary 0 is too high.

Default value:None
MINORALMUP0 Feed Tri. 0 TMA Over Warning Alm Thred.

Meaning:Feed Tri. 0 TMA Over Warning Alm Thred.

Default value:None

Example

1. Modify the parameters of the DATU (Index Type = BYIDX(By Index); BTS Index = 4; Subrack No. = 0; Slot No. = 3; Board Parameter Configuration Enabled = NO(NO)):
MOD DATUBP: IDXTYPE=BYIDX, BTSIDX=4, SRN=0, SN=3, CFGFLAG=NO;
2. Modify the parameters of the DATU (Index Type = BYIDX(By Index); BTS Index = 4; Subrack No. = 0; Slot No. = 3; Board Parameter Configuration Enabled = YES(YES); Feed Tri 0 TMA Configuration = ENABLED(Enabled); Feed Tri. 0 TMA Over Critical Alm Thred. = 1000; Feed Tri. 0 TMA Low Current Alm Thred. = 500):
MOD DATUBP: IDXTYPE=BYIDX, BTSIDX=4, SRN=0, SN=3, CFGFLAG=YES, AMPC0=ENABLED, MAJORALMUP0=1000, ALMD0=500;
