Modify Cell Quick Handover Parameters(MOD CELLHOFAST)

Function

A configuration command that modifies the quick handover parameters of the cell. In this command, Cell Index or Cell Name uniquely identifies a cell.

Note

None

Parameters

ID Name Description
IDXTYPE Index Type

Meaning:Index Type

Value range:BYNAME(By Name),BYIDX(By Index)

Explanation:Index Type.

Default value:None
CELLIDX Cell Index

Meaning:Cell Index

Value range:0~2047

Explanation: This parameter indicates the cell number, which is unique in a BSC.

Default value:None
CELLNAME Cell Name

Meaning:Cell Name

Value range:1-32 characters

Explanation: This parameter indicates the name of a cell. The value ranges from 1 to 32 characters.

Default value:None
HOUPTRIGE Quick handover up trigger level[db]

Meaning:Quick handover up trigger level[db]

Value range:0~63

Explanation:During a measurement period, if the filtered uplink level of the serving cell is smaller than the value of this parameter, a quick handover is triggered.

The level values 0 through 63 map to -110 dBm to -47 dBm.

Default value:None
HODOWNTRIGE Quick handover down trigger level[db]

Meaning:Quick handover down trigger level[db]

Value range:0~63

Explanation:During a measurement period, if the compensated downlink level of the serving cell is smaller than the value of this parameter, a quick handover is triggered.

The level values 0 through 63 map to -110 dBm to -47 dBm.

Default value:None
MOVESPEEDTHRES Quick move speed threshold[m/s]

Meaning:Quick move speed threshold[m/s]

Value range:0~600

Explanation:During a measurement period, if the MS moves at a speed greater than the value of this parameter, a quick handover is triggered.

Default value:None
HOSTATICTIME Quick handover static time[0.5s]

Meaning:Quick handover static time[0.5s]

Value range:1~32

Explanation:The quick handover must comply with the P/N criterion. That is, the triggering conditions of quick handover must be met for N consecutive seconds within P seconds.

This parameter corresponds to P of the P/N criterion. That is, if the triggering conditions of quick handover is met for a period longer than or equal to the value of this parameter, a quick handover is triggered.

Default value:None
HOLASTTIME Quick handover last time[0.5s]

Meaning:Quick handover last time[0.5s]

Value range:1~32

Explanation:The quick handover must comply with the P/N criterion. That is, the triggering conditions of quick handover must be met for N consecutive seconds within P seconds.

This parameter corresponds to N of the P/N criterion, that is, the period during which the triggering conditions of quick handover are met. Such a period is within the value defined by this parameter.

Default value:None
SCELLFILTER Serving cell filter length mr number

Meaning:Serving cell filter length mr number

Value range:1~20

Explanation:This parameter specifies the number of measurement reports used for filtering after the BSC receives the measurement reports of the serving cell from the BTS. This helps to avoid improper handover decision based on a single inaccurate measurement report.

Default value:None
NCELLFILTER Neighbor cell filter length mr number

Meaning:Neighbor cell filter length mr number

Value range:1~20

Explanation:This parameter specifies the number of measurement reports used for filtering after the BSC receives the measurement reports of the adjacent cell from the BTS. This helps to avoid improper handover decision based on a single inaccurate measurement report.

Default value:None
IGNOREMRNUM Ignore measurement report number

Meaning:Ignore measurement report number

Value range:0~32

Explanation:This parameter specifies the allowed number of invalid measurement reports when the BSC uses the measurement reports for filtering. If the number of received measurement reports is smaller than or equal to the value of this parameter, no filtering is performed and no quick handover decision is made.

Default value:None
TIMEPUNISH Quick handover punish time[s]

Meaning:Quick handover punish time[s]

Value range:0~255

Explanation:This parameter specifies the duration of penalty that is imposed on the originating cell after a successful quick handover.

Default value:None
HOPUNISHVALUE Quick handover punish value[db]

Meaning:Quick handover punish value[db]

Value range:0~63

Explanation:This parameter specifies the penalty that is imposed on the downlink level of the originating cell after a successful quick handover.

Default value:None
HOOFFSET Quick handover offset[db]

Meaning:Quick handover offset[db]

Value range:0~127

Explanation:During a measurement period, a quick handover can be triggered only if the difference of path loss between a chain adjacent cell and the serving cell is greater than or equal to the value of this parameter.

The level values 0 to 127 map to -64 dB to 63 dB.

Default value:None
MSLEVSTRQPBGT Enable Quick PBGT HO Algorithm When MS Leaves BTS

Meaning:Enable Quick PBGT HO Algorithm When MS Leaves BTS

Value range:NO(NO),YES(YES)

Explanation: This parameter specifies whether to enable the quick PBGT algorithm when an MS leaves the BTS.

Default value:None

Example

Modify the quick handover parameters of the cell (Index Type = BYIDX(By Index); Cell Index = 0; Quick handover up trigger level[db] = 62):
MOD CELLHOFAST: IDXTYPE=BYIDX, CELLIDX=0, HOUPTRIGE=62;
