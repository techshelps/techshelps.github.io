Forcibly Back Up OMU Files(BAK OMUALL)

Function

An OM command that backs up the important files from the active GOMU to the standby GOMU.

Note

1. If the Daemon on the local GOMU or the peer GOMU is not running, execution of this command fails.

2. When the backup switch is set to OFF(Disable), execution of this command fails. You can run the DSP OMUDAEMON command to query the status of the backup switch.

3. This command cannot be run on the GBAM.

4. The paths and names of the files backed up by this command are the same as those of the files backed up in the initial backup.

Parameters

None.

Example

When the active and standby GOMUs run normally, backed up files:
BAK OMUALL:;
