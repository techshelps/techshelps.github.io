Display Clock Status(DSP CLKSTAT)

Function

An OM command that queries the working status of the current system clock.

Note

The GGCU in the GMPS must be functional.

Parameters

None.

Example

Query the status of the system clock:
DSP CLKSTAT:;

Output Description

Domain Name Description
Subrack No.

Number of the subrack where the board is located.
Slot No.

Number of the slot where the board is located.
Current Reference Clock

Type of the reference source in use, including GPS, BITS0, BITS1, Line 0, Line 1, or 8K. If it is not configured, Unconfigured is displayed.
Reference Clock Sub Type

Subtype of the reference source in use, including 2 MHz, 2 MHz BITS, or 8K_Hz. If it is not configured, Unconfigured is displayed.
Current Priority

Priority of the clock in use. The value ranges from 1 to 4. If it is not configured, no information is returned for this item.
Work Mode

Current working mode of the system clock. The mode can be Automatic, Manual, or Free Running.
Pll Status

Status of the phase-lock loop on the specified GGCU. The status can be Unknown, Free Running, Holdover, Fast Tracking, or Locked.
Pll Last Change Time

Time when the PLL is last changed.
Clock Type (1-4)

Type of reference clock (1-4). The type can be GPS, BITS0, BITS1, Line 0, Line 1, 8 kHz, or Unconfigured.
Clock Sub Type (1-4)

Subtype of reference clock (1-4). The subtype can be 2M_Hz, 2M_Bps, 8K_Hz, or Unconfigured.
Item Reference Clock Status (1-4)

Status of reference source (1-4). The status can be Unknown, Available, or Unavailable.
Clock Lost Reason (1-4)

Cause of the loss of clock (1-4). The cause can be Normal, Frequence Check Error, Measure Fail, 1PPS Signal Error, Line Source Lost, Phase Lost, Phase Value Error, Frequence Over Range, or E1 LOS Alarm.
