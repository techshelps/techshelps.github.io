Add Bearer Channel(ADD BC)

Function

A configuration command that adds a bearing channel (BC) at the FR layer.

Bound with a certain number of timeslots on the E1/T1, the BC provides the physical channel in FR mode. The following key parameter must be negotiated with the peer SGSN:

Protocol Type

Note

The BC can be established on the GEPUG only.

Parameters

ID Name Description
SRN Subrack No.

Meaning:Subrack No.

Value range:0~11

Explanation: This parameter specifies the number of the subrack that holds the GEPUG.

Default value:None
SN Slot No.

Meaning:Slot No.

Value range:0~27

Explanation: This parameter specifies the number of the slot in which the GEPUG is located.

Default value:None
PN Port No.

Meaning:Port No.

Value range:0~31

Explanation: This parameter specifies the port number of the BC on the GEPUG.

Default value:None
BCID BC Identifier

Meaning:BC Identifier

Value range:0~255

Explanation: Each bearer channel (BC) on an E1 is assigned a number to facilitate local management. This number is called bearer channel ID.

The local BC number and the peer BC number can be different. For one BC, the timeslot distribution at one end of the E1 must be the same as that at the other end of the E1 link. Therefore, the BC is available for communication.

Default value:None
TS Bearing Timeslot

Meaning:Bearing Timeslot

Value range:TS0(0),TS1(1),TS2(2),TS3(3),TS4(4),TS5(5),TS6(6),TS7(7),TS8(8),TS9(9),TS10(10),TS11(11),TS12(12),TS13(13),TS14(14),TS15(15),TS16(16),TS17(17),TS18(18),TS19(19),TS20(20),TS21(21),TS22(22),TS23(23),TS24(24),TS25(25),TS26(26),TS27(27),TS28(28),TS29(29),TS30(30),TS31(31)

Explanation: This parameter specifies the timeslot occupied by the BC.

Default value:None
PROT Protocol Type

Meaning:Protocol Type

Value range:Q933,ANSI

Explanation: This parameter specifies the type of the frame relay protocol. The setting of this parameter must be consistent with the setting on the SGSN side.

Default value:None
N391 Parameter of DTE [N391]

Meaning:Parameter of DTE [N391]

Value range:1~255

Explanation: This parameter specifies the counter on the DTE side.

After receiving the link integrity acknowledgment message for N391 times, the DTE sends a full state query request.

Default value:6
N392 Parameter of DTE [N392]

Meaning:Parameter of DTE [N392]

Value range:1~10

Explanation: This parameter specifies the counter on the DTE side.

Among N393 checks, if the link integrity acknowledgment event occurs for N393 times, the status is changed.

Default value:3
N393 Parameter of DTE [N393]

Meaning:Parameter of DTE [N393]

Value range:1~10

Explanation: This parameter specifies the counter on the DTE side.

Among N393 checks, if the link integrity acknowledgment event occurs for N393 times, the status is changed.

Default value:4
T391 Parameter of DTE [T391]

Meaning:Parameter of DTE [T391]

Value range:5~30

Explanation: This parameter specifies the timer set to wait for a link integrity acknowledgment message.

The timer is started when the link is initialized. When the timer expires, the link integrity acknowledgment message is resent.

Default value:10

Example

Add a BC on the FR layer (Subrack No. = 0; Slot No. = 16; Port No, = 0; BC Identifier = 0; Bearing Timeslot = TS1(Timeslot 1); Protocol Type = Q933(Q933)):
ADD BC: SRN=0, SN=16, PN=0, BCID=0, TS=TS1-1, PROT=Q933;
