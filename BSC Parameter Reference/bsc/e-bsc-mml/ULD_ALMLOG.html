Upload Alarm Log(ULD ALMLOG)

Function

A configuration command that uploads the alarm log files.

The alarm log files are generated through the LST ALMLOG command.

Note

1. The parameters must be correctly set.

2. Before running this command, run the LST ALMLOG command to set "The type of return result" to "file(queried file)".

Parameters

ID Name Description
FN File name

Meaning: file name

Value range: 1-63 characters

Explanation: The file name is in fixed format, where AlarmLog_ is followed by the creation time of the log, that is, AlarmLog_YYYYMMDD_HHMMSS.log.

Default value: None
IP IP address of FTP server

Meaning: IP address of the FTP server

Value range: 0.0.0.0-255.255.255.255

Explanation: This parameter specifies the IPv4 address for logging in to the FTP server.

Default value: None
USR Username of FTP server

Meaning: user name for logging in to the FTP server

Value range: 1-256 characters

Explanation: This parameter specifies the user name for logging in to the FTP server.

Default value: None
PWD User password of FTP server

Meaning: password for logging in to the FTP server

Value range: 1-256 characters

Explanation: This parameter specifies the password for logging in to the FTP server.

Default value: None
ZIPFLG Compress flag

Meaning: whether to compress the file

Value range: yes(YES), no(NO)

Explanation: This parameter specifies whether to compress the uploaded file.

Default value: no(NO)

Example

ULD ALMLOG:FN="*.almlog",IP="10.10.10.10",USR="user",PWD="password";
