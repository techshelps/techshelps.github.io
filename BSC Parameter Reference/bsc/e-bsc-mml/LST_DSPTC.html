List DSP TC Attributes(LST DSPTC)

Function

An OM command that queries the TC data of a destination signaling point.

To query the TC attributes of all the destination signaling points on a specific GDPUC/GDPUX, set Subrack No. and Slot No..

To query the TC attributes of a destination signaling point on a specific GDPUC/GDPUX, set Subrack No., Slot No., and DSP No..

Note

None

Parameters

ID Name Description
SRN Subrack No.

Meaning:Subrack No.

Value range:0~11

Explanation:This parameter specifies the number of a subrack.

The BSC6000 can be configured with up to four remote subracks or eight local subracks (four GTCSs and four GMPSs/GEPSs).

Only one local main subrack and one remote main subrack can be configured.

Subrack 0 is the local main subrack and cannot be deleted.

If the BSC should be configured with the GBAM, the GBAM is installed in subrack 12. Deleting subrack 12 is prohibited.

Default value:None
SN Slot No.

Meaning:Slot No.

Value range:0~27

Explanation:This parameter specifies the number of the slot in which the sub-board is located.

Default value:None
DSPNO DSP No.

Meaning:DSP No.

Value range:0~21

Explanation:This parameter specifies the number of the DSP in the DPU.

There is a one-to-one mapping between the DSP TC object and the DSP. The DSP object manages the communication between the DSP chip and the CPU, and the DSP TC object processes services, such as communication status query and DSP log recording. If the DSP TC object is not configured, the DSP chip can run normally but cannot process services.

Default value:None
