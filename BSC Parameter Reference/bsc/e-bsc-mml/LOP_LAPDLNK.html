Loop Back LAPD Link(LOP LAPDLNK)

Function

An OM command that runs a loopback test on a specified LAPD link.

Note

When loopbacking a specified LAPD link, "LINE LOOP" requires manual disconnection of E1 before interconnecting TX and RX ports, which disrupts the link communication. "LVDS LOOP" and "MCC LOOP" are run automatically, and they are released after the test. However, this may still cause temporary service interruption.

Parameters

ID Name Description
CESNO CES No.

Meaning: logical link number

Value range: 0-12287

Explanation: A key field of an LAPD link that indicates the LAPD link to be looped back. You can obtain the field through the LST LAPDLNK command.

Default value: None
LOOPTYPE Loop type

Meaning: loopback mode

Value range: LOOPTYPE_TDM(Line Loop), LOOP_TYPE_LVDS(LVDS Loop),LOOPTYPE_MCC(MCC Loop)

Explanation: This parameter specifies the loopback mode in a loopback test. "Line Loop" requires manual interconnection of E1 TX and RX ports that carry the LAPD link, whereas "LVDS Loop" and "MCC Loop" are performed by the system.

Default value: None

Example

Loop back an LAPD link (CES No. = 0; Loop type = TYPE_TDM(Line Loop)):
LOP LAPDLNK: CESNO=0, LOOPTYPE=LOOPTYPE_TDM;
