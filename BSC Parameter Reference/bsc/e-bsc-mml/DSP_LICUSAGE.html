Display License Usage Information(DSP LICUSAGE)

Function

An OM command that queries the actual usage of the activated license in terms of permissible traffic volume and occupied traffic resources

Note

1. The MML client should communicate with the BSC properly.

2. The GXPUM in the GMPS should run normally.

3. When NE License Administration State is set to Local, the number of NE licenses depends on the license file. In this case, the NE is controlled by its license file.

4. When NE License Administration State is set to Share, the number of NE licenses depends on the license file. In this case, the NE is controlled by its license file.

5. If you query the usage of the license whose file is not activated, NULL is displayed for each item.

6. If a function item in an NE license file is not applied for, Not Activated is displayed for this function item after the license file is activated; Otherwise, Activated is displayed for this function item.

7. If the parameter "Configuration Value" is set to 65535 (FFFF in hexadecimal), the value is invalid.

Parameters

ID Name Description
TYPE Type of Usage

Meaning: type of the usage

Value range: Peak(Peak Usage in 24 Hours), Current(Current Usage)

Explanation: None

Default value: Peak(Peak Usage in 24 Hours)

Example

1. Query the peak usage of the license in the latest 24 hours:
DSP LICUSAGE:;
The result is as follows:
