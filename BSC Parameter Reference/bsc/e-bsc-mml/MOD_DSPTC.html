Modify DSP TC Attributes(MOD DSPTC)

Function

A configuration command that modifies some parameters related to the DSP TC in the configuration database. In this command, the system uniquely identifies the DSP TC data record according to the combination of the following key parameters.

Subrack No.

Slot No.

DSP number

This command supports the batch modification function: If you enter the subrack No., slot No., DSP number, and the attributes to be modified, you can modify the TC attributes of a DSP on the specified GDPUC board. If you just enter the subrack No., slot No., and the attributes to be modified, you can modify the TC attributes of all the DSPs on the specified GDPUC/GDPUX board.

Note

None

Parameters

ID Name Description
SRN Subrack No.

Meaning:Subrack No.

Value range:0~11

Explanation:This parameter specifies the number of a subrack.

The BSC6000 can be configured with up to four remote subracks or eight local subracks (four GTCSs and four GMPSs/GEPSs).

Only one local main subrack and one remote main subrack can be configured.

Subrack 0 is the local main subrack and cannot be deleted.

If the BSC should be configured with the GBAM, the GBAM is installed in subrack 12. Deleting subrack 12 is prohibited.

Default value:None
SN Slot No.

Meaning:Slot No.

Value range:0~27

Explanation:This parameter specifies the number of the slot in which the sub-board is located.

Default value:None
DSPNO DSP No.

Meaning:DSP No.

Value range:0~21

Explanation:This parameter specifies the number of the DSP in the DPU.

There is a one-to-one mapping between the DSP TC object and the DSP. The DSP object manages the communication between the DSP chip and the CPU, and the DSP TC object processes services, such as communication status query and DSP log recording. If the DSP TC object is not configured, the DSP chip can run normally but cannot process services.

Default value:None
G711_MODE G711 Mode

Meaning:G711 Mode

Value range:A0,U1

Explanation:This parameter specifies the coding compression mode. Multiple transmission modes such as E1/T1 can be used on the A interface. The code stream in different transmission modes uses different coding compression modes. The value of this parameter indicates the coding compression mode.

Default value:None
IDLE_CODE Idle Code

Meaning:Idle Code

Explanation:This parameter specifies the idle code on the A interface.

Default value:None
MUTE_DETECT Mute Detect

MUTE_TIMER Mute Timer

Meaning:Mute Timer

Value range:0~255

Explanation:This parameter specifies the duration of the one-way audio check.

If this parameter is set to Open, the BSC instructs the GDPUC to check whether one-way audio occurs after a call is set up. The value of this parameter indicates the maximum duration for the GDPUC to perform one-way audio check.

Default value:None

TFO_SWITCH TFO Switch

Meaning:TFO Switch

Value range:CLOSE(Close),OPEN(Open)

Explanation:This parameter specifies whether to enable the Tandem Free Operation (TFO) function.

If the voice quality of an ongoing MS-to-MS call is bad, the TFO function can be enabled to improve end-to-end voice quality.

The TFO function cannot be used to improve the voice quality of an MS-to-PSTN call.

Default value:None

Explanation: This parameter specifies whether the AEC function is enabled.

Default value:None
AECNLPGATE AEC Nlp Gate[dBm0]

Meaning:AEC Nlp Gate[dBm0]

Value range:-60~-35,0

Explanation: This parameter specifies the threshold to enable the NLP. To reduce the echo, if the input level on the near end is lower than the value of this parameter when the far end is speaking, comfortable noise is generated to substitute the voice of the near end. In this way, low voices may be cut.

Example

1. Set the A/U configuration of DSP 0 of GDPUC board at slot 0 in subrack 1.
MOD DSPTC: SRN=1, SN=0, DspNo=0, G711_Mode=A0;
2. Set the A/U configuration for all the DSPs of the GDPUC board that is on slot 0 of subrack 1 to U1:
MOD DSPTC: SRN=1, SN=0, G711_Mode=U1;
