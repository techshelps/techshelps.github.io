Add SS7 Signaling Link(ADD N7LNK)

Function

A configuration command that adds an SS7 signaling link.

Note

1. The SS7 signaling link cannot be configured in A over IP mode. In BM/TC separated configuration mode, you need to configure the Ater interface data, such as Ater Connection Path Index and Ater Interface Timeslot Mask; in BM/TC combined configuration mode, you need to set SubRack No. to specify the BM subrack where the A interface board is located. The preceding parameters are determined by the parameter Service Mode.

2. The EIU/OIU board is already configured with other links of different type. You should reset the EIU/OIU if you need to configure this link.

In SEPARATE(BM/TC Separate) mode:

1. You must configure an Ater connection path between the BM subrack and the TC subrack before you configure an SS7 signaling link, and this Ater connection path must have idle timeslots.

2. Configure an E1/T1 on the A interface, and the E1/T1 on the A interface must have idle timeslots.

3. The GMPS/GEPS must have been configured with the GXPUM.

4. The number of timeslots on the A interface must be the same as the number of timeslots on the Ater interface.

5. The selected destination signaling point must be in the DPC group to which the E1/T1 of the A interface belongs.

6. If the selected destination signaling point is not directly connected to the originating signaling point, you must select a signaling transfer point.

7. Before configuring the Ater connection path, ensure that the BM subrack is bound with the originating signaling point.

8. In a GMPS/GEPS, each signaling link must work at the same data rate, and the link of each GMPS/GEPS connected to the same originating signaling point must work at the same data rate.

9. Each pair of signaling points (each link set) can be configured with up to 16 links.

10. The satellite link cannot work at 2 Mbit/s.

11. The SLC must be different from the link set.

12. The SLCSend must be different from the link set.

13. Congestion Start Threshold must be greater than Congestion End Threshold.

14. In E1 transmission mode, if the SS7 signaling link works at 2 Mbit/s, a GMPS/GEPS supports up to eight SS7 signaling links with the total bandwidth of 4 Mbit/s; if the SS7 signaling link works at 64 kbit/s, a GMPS/GEPS supports up to 32 SS7 signaling links.

15. In T1 transmission mode, if the SS7 signaling link works at 2 Mbit/s, a GMPS/GEPS supports up to eight SS7 signaling links with the total bandwidth of 3 Mbit/s, if the SS7 signaling link works at 64 kbit/s, a GMPS/GEPS supports up to 32 SS7 signaling links.

In TOGETHER(BM/TC Together) mode:

1. The GMPS/GEPS must have been configured with the GXPUM.

2. Configure an E1/T1 on the A interface, and the E1/T1 on the A interface must have idle timeslots.

3. The selected destination signaling point must be in the DPC group to which the E1/T1 of the A interface belongs.

4. If the selected destination signaling point is not directly connected, you must select a signaling transfer point.

5. Each pair of signaling points (each link set) can be configured with up to 16 links.

6. The satellite link cannot work at 2 Mbit/s.

7. The SLC must be different from the link set.

8. The SLCSend must be different from the link set.

9. Congestion Start Threshold must be greater than Congestion End Threshold.

10. All the SS7 signaling links on an A interface board must work at the same data rate. which can be 64 kbit/s or 2 Mbit/s.

11. In E1 transmission mode, if the SS7 signaling link works at 2 Mbit/s, each A interface board supports up to eight SS7 signaling links with the total bandwidth of 4 Mbit/s; if the SS7 signaling link works at 64 kbit/s, each A interface board supports up to 32 SS7 signaling links.

12. In E1 transmission mode, if the SS7 signaling link works at 2 Mbit/s, each A interface board supports up to eight SS7 signaling links with the total bandwidth of 3 Mbit/s; if the SS7 signaling link works at 64 kbit/s, each A interface board supports up to 32 SS7 signaling links.

Parameters

SERVICEMODE Service Mode

Meaning:Service Mode

Value range:SEPARATE(BM/TC Separate),TOGETHER(BM/TC Together)

Explanation: Index Type.

Default value:None
ATERPIDX Ater Connection Path Index

Meaning:Ater Connection Path Index

Value range:0~254

Explanation: This parameter specifies the index of an Ater connection path.

Default value:None
SRN SubRack No.

Meaning:SubRack No.

Value range:0~11

Explanation: Mtp2FramNo.

Default value:None
ASN A Interface Slot No.

Meaning:A Interface Slot No.

Value range:0~27

Explanation: This parameter specifies the number of the slot where the A interface board (configured with MTP2LINK) is located.

Default value:None
APN A Interface Port No.

Meaning:A Interface Port No.

Value range:0~83

Explanation: This parameter specifies the number of the port on the A interface board (configured with MTP2LINK).

Default value:None
ATSMASK A Interface Timeslot Mask

Meaning:A Interface Timeslot Mask

Value range:TS0(0),TS1(1),TS2(2),TS3(3),TS4(4),TS5(5),TS6(6),TS7(7),TS8(8),TS9(9),TS10(10),TS11(11),TS12(12),TS13(13),TS14(14),TS15(15),TS16(16),TS17(17),TS18(18),TS19(19),TS20(20),TS21(21),TS22(22),TS23(23),TS24(24),TS25(25),TS26(26),TS27(27),TS28(28),TS29(29),TS30(30),TS31(31)

Explanation: This parameter specifies the timeslot mask of the port on the A interface board (configured with MTP2LINK).

Default value:None
ATERTSMASK Ater Interface Timeslot Mask

Meaning:Ater Interface Timeslot Mask

Value range:TS0(0),TS1(1),TS2(2),TS3(3),TS4(4),TS5(5),TS6(6),TS7(7),TS8(8),TS9(9),TS10(10),TS11(11),TS12(12),TS13(13),TS14(14),TS15(15),TS16(16),TS17(17),TS18(18),TS19(19),TS20(20),TS21(21),TS22(22),TS23(23),TS24(24),TS25(25),TS26(26),TS27(27),TS28(28),TS29(29),TS30(30),TS31(31)

Explanation: TsIdOrBits.

Default value:None
SLC SLC

Meaning:SLC

Value range:0~15

Explanation: This parameter is used to identify the signaling link between two signaling points that can be directly connected. In the signaling network, a signaling point communicates with another one by the signaling link. The signaling link between two signaling points that can be directly connected is identified by the SLC.

Default value:None
SLCSEND SLC Send

Meaning:SLC Send

Value range:0~15

Explanation: When the SS7 signaling link is faulty, messages are sent and received in form of dummy message during the fault location and analysis process. In this case, an SLC and a signaling link are specified to identify the signaling link for sending and receiving messages.

Default value:None
DPC DPC

Meaning:DPC

Value range:1~16777215

Explanation: 1.This parameter specifies the code of the signaling network where the DPC is located. In the signaling network, each signaling point has a unique signaling point code.

2.This parameter can be displayed in decimal, hexadecimal, and binary mode. By default, this parameter is displayed in decimal mode. The hexadecimal format is H' and the binary format is B'.

Default value:None
STPC STP Code

Meaning:STP Code

Value range:1~16777215

Explanation: After the BSC receives the paging response message, the BSC obtains the DPC according to the NRI of the TMSI, and sends the paging response message to the MSC.

Default value:None
LKRATE Link Rate Type

Meaning:Link Rate Type

Value range:64K(64Kbit/s),2M(2Mbit/s)

Explanation: LinkType.

Default value:64K
STFLG Satellite Flag

Meaning:Satellite Flag

Value range:NO(No),YES(Yes)

Explanation: This parameter is used to identify whether the signaling link is a satellite link.

The signaling link is categorized into satellite link and non-satellite link. The difference is the error correction mode.

The value YES specifies that the SS7 signaling link uses the satellite link. The preventive cyclic error correction mode is used.

The value NO specifies that the SS7 signaling link uses the non-satellite link. The basic error correction mode is used.

Default value:NO

Example

1. Add an SS7 signaling link (Service Mode = SEPARATE(BM/TC Separate); Ater Connection Path Index = 0; A Interface Slot No. = 20; A Interface Port No. = 0; A Interface Timeslot Mask = 4; Ater Interface Timeslot Mask = 4; SLC = 0; STP Code = 0; DPC = 2; Link Rate Type = 64K(64Kbit/s); Satellite Flag = NO(No)):
ADD N7LNK: SERVICEMODE=SEPARATE, ATERPIDX=0, ASN=20, APN=0, ATSMASK=TS4-1, ATERTSMASK=TS4-1, SLC=0, SLCSEND=0, DPC=2, LKRATE=64K, STFLG=NO;
2. Add an SS7 signaling link (Service Mode = TOGETHER(BM/TC Together); Subrack No. = 20; A Interface Slot No. = 20; A Interface Port No. = 0; A Interface Timeslot Mask = 4; SLC = 0; STP Code = 0; DPC = 2; Link Rate Type = 64K(64Kbit/s); Satellite Flag = NO(No)):
ADD N7LNK: SERVICEMODE=TOGETHER, SRN=0, ASN=20, APN=0, ATSMASK=TS4-1, SLC=0, SLCSEND=0, DPC=2, LKRATE=64K, STFLG=NO;
