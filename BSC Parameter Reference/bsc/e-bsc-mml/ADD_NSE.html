Add NSE(ADD NSE)

Function

A configuration command that adds a Network Service Entity (NSE).

Note

1. The agent subrack must be a BM subrack.

2. One BSC can be configured with up to 128 NSEs.

3.If the protocol type of the subnet is set to GB_OVER_FR,please configure the GEPUG board.

4.If the protocol type of the subnet is set to GB_OVER_IP,please configure the GFGUG board.

Parameters

ID Name Description
NSEI NSE Identifier

Meaning:NSE Identifier

Value range:0~65534

Explanation: This parameter specifies the identifier of a network service entity (NSE).

An NSE manages a group of NSVCs.

Default value:None
SRN Subrack No.

Meaning:Subrack No.

Value range:0~11

Explanation: This parameter specifies the number of the subrack that holds the NSE managing the point-to-point communication.

Default value:None
PT Protocol Type

Meaning:Protocol Type

Value range:GB_OVER_FR(Gb over FR),GB_OVER_IP(Gb over IP)

Explanation: This parameter specifies the type of the subnet protocol. Two types are Gb over FR and Gb Over IP.

Gb Over FR: The Gb interface uses the Fr transmission mode. The data link layer on the Gb interface supports frame relay.

The SGSN and the BSS are connected through the virtual circuits of frame relay. The LLC PDUs from different users are multiplexed onto these virtual circuits.

The user data and signaling on the Gb interface can be transmitted through frame relay.

Gb Over IP: The Gb interface uses the IP transmission mode.

The FR connection is replaced by the IP network. The user data and signaling are transmitted through UDP packets.

The protocol stacks and functions of the NS layer and upper layers in Gb Over IP mode are the same as those in Gb Over FR mode.

Default value:None
ISNCMODE Subnetwork Configure Mode

Meaning:Subnetwork Configure Mode

Value range:STATIC(Static),DYNAMIC(dynamic)

Explanation: This parameter specifies the configuration mode of the IP subnet. Two configuration modes are static configuration and dynamic configuration.

Static Configuration: The remote NSVL can be configured on the LMT.

Dynamic Configuration: The configuration of the remote NSVL can be obtained from the SGSN through the procedures defined in the protocol.

Default value:None
SVRIP Server IP

Meaning:Server IP

Value range:0.0.0.0-255.255.255.255

Explanation: This parameter specifies the IP address of the SGSN.

Default value:None
SVRPORT Server UDP Port

Meaning:Server UDP Port

Value range:0~65535

Explanation: This parameter specifies the UDP port number of the SGSN.

Default value:None
CNOPNAME Service Provider Name

Meaning:Service Provider Name

Value range:1-32 characters

Explanation: This parameter uniquely identifies an Service Provider entity in the system.

Default value:None
CNID SGSN Node ID

Meaning:SGSN Node ID

Value range:0~4095

Explanation: The SGSN Node ID and the Service Provider Name uniquely identify an SGSNNODE.

Default value:None
RIMSUP RIM Support

Meaning:RIM Support

Value range:NO(No Support),YES(Support)

Explanation:This parameter specifies whether this NSE supports the RIM function. After this switch is enabled, the SIGBVC should be reset. The RIM function can be supported only when the SGSN also supports the RIM function.

Default value:None
PFCSUP PFC Support

Meaning:PFC Support

Value range:NO(No Support),YES(Support)

Explanation:This parameter specifies whether this NSE supports the PFC function. After this switch is enabled, the SIGBVC should be reset. The PFC function can be supported only when the SGSN also supports the PFC function.

Default value:None

Example

Add an NSE in subrack 0 with the NSE ID=1, Protocol Type=Gb over FR, Service Provider Name="46000" and SGSN Node ID=101:
ADD NSE: NSEI=1, SRN=0, PT=GB_OVER_FR, CnOpName="46000", CNId=101;
