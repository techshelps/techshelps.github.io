Modify AMR Call Control Data of Cell Call Control Parameters(MOD CELLCCAMR)

Function

A configuration command that modifies the AMR call control parameters of a cell. In this command, the following parameters uniquely identify the cell:

Cell Index or Cell Name

Note

When the parameter Auto Adjust UL Threshold and Hysteresis [F] is set to YES(YES), the following parameters are invalid: AMR UL Coding Rate adj.th1[F], AMR UL Coding Rate adj.th2[F], AMR UL Coding Rate adj.th3[F], AMR UL Coding Rate adj.hyst1[F], AMR UL Coding Rate adj.hyst2[F], AMR UL Coding Rate adj.hyst3[F]. When the parameter Auto Adjust UL Threshold and Hysteresis [H] is set to YES(YES), the following parameters are invalid: AMR UL Coding Rate adj.th1[H], AMR UL Coding Rate adj.th2[H], AMR UL Coding Rate adj.th3[H], AMR UL Coding Rate adj.hyst1[H], AMR UL Coding Rate adj.hyst2[H], AMR UL Coding Rate adj.hyst3[H].

Parameters

ID Name Description
IDXTYPE Index Type

Meaning:Index Type

Value range:BYNAME(By Name),BYIDX(By Index)

Explanation:Index Type.

Default value:None
CELLIDX Cell Index

Meaning:Cell Index

Value range:0~2047

Explanation: This parameter indicates the source cell index. The source cell index must be unique.

Default value:None
CELLNAME Cell Name

Meaning:Cell Name

Value range:1-32 characters

Explanation: This parameter indicates the name of the source cell. The name of each cell under one BSC must be unique.

Default value:None
RATECTRLSW AMR rate control switch

Meaning:AMR rate control switch

Value range:ALG1(Algorithm I),ALG2(Algorithm II),NONE(None)

Explanation:This parameter specifies whether AMR rate adaptation algorithm is enabled and which algorithm should be used for codec mode adjustment.

Default value:None
ACTCDSETF AMR ACS[F]

Meaning:AMR ACS[F]

Value range:4_75Kbps(0),5_15Kbps(1),5_90Kbps(2),6_70Kbps(3),7_40Kbps(4),7_95Kbps(5),10_2Kbps(6),12_2Kbps(7)

Explanation:This parameter specifies the set of active coding rates. The active coding set (ACS) is a set of coding rates currently available for calls. Use a BIT map to present the speech coding rates contained in the ACS, wherein a BIT corresponds to a coding rate. If a bit is 1, the coding rate is included in the ACS. Otherwise, the ACS does not include the coding rate. The value of this parameter has eight bits.

Default value:None
AUTOADJULTHHYSTF Auto Adjust UL Threshold and Hysteresis [F]

Meaning:Auto Adjust UL Threshold and Hysteresis [F]

Value range:NO(NO),YES(YES)

Explanation:This parameter specifies whether a full-rate AMR call can automatically adjust the uplink rate threshold and hysteresis.

Default value:None
ULTHF1 AMR UL Coding Rate adj.th1[F]

Meaning:AMR UL Coding Rate adj.th1[F]

Value range:0~63

Explanation:Based on the RQI in the call measurement report, the BTS and MS automatically adjust the current speech coding rate according to the related algorithm. The coding rate adjustment threshold is the threshold of RQI. The RQI indicates the carrier-to-interference ratio (CIR) of the call. If RQI equals 1, the CIR is 0.5 dB; if RQI equals 2, the CIR is 1 dB; and so forth. Since there are multiple coding rates in the ACS, there is an adjustment threshold and an adjustment hysteresis between the neighboring coding rates.

Default value:None

MOD CELLCCAMR: IDXTYPE=BYIDX, CELLIDX=0, ACTCDSETF=4_75Kbps-1&5_15Kbps-1&5_90Kbps-1&6_70Kbps-1, ACTCDSETH=5_15Kbps-1&5_90Kbps-1&6_70Kbps-1&7_40Kbps-1;
