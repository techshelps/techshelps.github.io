Display DSP Service Status(DSP DSPSVRSTAT)

Function

An OM command that queries the service status of the DSP. DSP service status information includes the service type, rate, loop type, loop direction, and TFO status of all 70 channels.

Note

1. The DSP should be configured and functional.

2. The DSP to be queried refers to the DSP running the TC service, that is, the DSP on the GDPUC/GDPUX.

3. If the optional parameter DSP No. is not specified, the service status of all the DSPs in the specified slot is queried.

Parameters

ID Name Description
SRN Subrack No.

Meaning: subrack number

Value range: 0-11

Explanation: The subrack number uniquely identifies a subrack.

Default value: None
SN Slot No.

Meaning: slot number

Value range: 0-27

Explanation: The subrack number and slot number uniquely identify a board.

Default value: None
DSPN DSP No.

Meaning: destination signaling point number

Value range: 0-21

Explanation:

1. The subrack number, slot number, and DSP number uniquely identify a destination signaling point.

2. The DSP number is an optional parameter. If not specified, the service status of all destination signaling points of the specified board is queried.

Default value: None

Example

Query the service status of DSP 1 in slot 0 of subrack 0:
DSP DSPSVRSTAT: SRN=0, SN=0, DSPN=1;
The result is as follows:
MML% DSP DSPSVRSTAT: SRN=0, SN=0, DSPN=1;
+++    HUAWEI        2008-01-28 09:55:26
O&M    #31382
%%DSP DSPSVRSTAT: SRN=0, SN=0, DSPN=1;%%
RETCODE = 0  Operation succeeded

Output Description

Domain Name Description
Subrack No.

Number of the subrack where the board is located.
Slot No.

Number of the slot where the board is located.
DSP No.

Number of the DSP where the board is located.
TFO

Tandem Free Operation. The TFO type can be Disconnect; Connect.
Service type

Type of the speech services. The type can be Idle(IDLE); FR; EFR; HR; FAMR; HAMR; FRD; HRD; or Data Service.
Rate Type

Type of voice rate. The type can be No Rate; 4.75K; 5.15K; 5.6K; 5.9K; 6.7K; 7.4K; 7.95K; 8K; 10.2K; 12.2K; 13K; 16K.
Scene

Conversational scenario. The scenario can be MODE0(PTRAU-RTP); MODE01(PTRAU-PCM); MODE2(TRAU-RTP); MODE3(TRAU-PCM); MODE4(PTRAU-TRAU); or MODE5(IDLE).
Loopback Type

The loopback type can be No Loop; A Interface Loopback; Abis Interface Loopback.
A Sid

Number of the channel that carries the traffic of the DSP when A over IP mode is used.
ABIS Sid

Number of the channel that carries the traffic of the DSP when Abis over IP mode is used.
