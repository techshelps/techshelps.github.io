Modify SS7 Signaling Link(MOD N7LNK)

Function

A configuration command that modifies the parameters of an SS7 signaling link. In this command, when the Service Mode is set to SEPERATE(BM/TC Seperate), BM Subrack No. and MTP3 Link No. uniquely identify the SS7 signaling link data record; when the Service Mode is set to TOGETHER(BM/TC Together), BM Subrack No., A Interface Slot No., and MTP3 Link No. uniquely identify an SS7 signaling link.

Note

The value of Congestion Start Threshold must be greater than that of Congestion End Threshold.

Parameters

ID Name Description
BMSRN BM Subrack No.

Meaning:BM Subrack No.

Value range:0~11

Explanation: Mtp2FramNo.

Default value:None
SERVICEMODE Service Mode

Meaning:Service Mode

Value range:SEPARATE(BM/TC Separate),TOGETHER(BM/TC Together)

Explanation: Index Type.

Default value:None
ASN A Interface Slot No.

Meaning:A Interface Slot No.

Value range:0~27

Explanation: This parameter specifies the number of the slot where the A interface board (configured with MTP2LINK) is located.

Default value:None
LINKNO MTP3 Link No.

Meaning:MTP3 Link No.

Value range:0~31

Explanation: LinkNo.

Default value:None
PRI Priority

Meaning:Priority

Value range:0~254

Explanation: This parameter specifies the priority of a signaling link. The link with high priority is preferred during the link selection.

Default value:None
CGSTHD Congestion Start Threshold

Meaning:Congestion Start Threshold

Value range:1~100

Explanation: This parameter is used control the congestion of the SS7 signaling link. When the SS7 signaling link carries many services and the usage of cache memory on the MTP3 layer exceeds the value of this parameter, congestion occurs on the MTP3 layer of the SS7 signaling link. In this case, the upper link is notified of MTP3 layer congestion.

Default value:None
CGETHD Congestion End Threshold

Meaning:Congestion End Threshold

Value range:1~100

Explanation: This parameter is used to control the congestion of the SS7 signaling link. When the SS7 signaling link carries many services and the usage of cache memory on the MTP3 layer is lower than the value of this parameter, the congestion on the MTP3 layer of the SS7 signaling link is cleared. In this case, the upper link is notified of congestion end.

Default value:None
TESTC Test Code

Meaning:Test Code

Value range:0~255

Explanation: After the A interface starts to work, a signaling link test message is transmitted on the SS7 signaling link at an interval of one minute to check whether the signaling link is available. The message contains a test code. This parameter specifies the value of each byte in the test code.

Default value:None
TESTCLEN Test Code length

Meaning:Test Code length

Value range:1~15

Explanation: After the A interface starts to work, a signaling link test message is transmitted on the SS7 signaling link at an interval of one minute to check whether the signaling link is active. The message contains a test code. This parameter specifies the value of each byte in the test code.

Default value:None
CT1 Timer Alignment Ready

Meaning:Timer Alignment Ready

Value range:40000~50000

Explanation: This parameter specifies the length of the timer for the MTP2 link to control the reporting of link status in the initial alignment procedure.

This timer is triggered when the MTP2 link is in the initial alignment state. When the timer expires, the link is set to the service disrupted state and the initial alignment starts again. This parameter only applies to the 64 kbit/s links. For the 2 Mbit/s links, the value of this timer is always 150000.

Default value:None
CT2 Timer Not Aligned

Meaning:Timer Not Aligned

Value range:5000~150000

Explanation: This parameter specifies the length of the timer for the MTP2 link to report link not aligned status in the initial alignment procedure.

This timer is triggered when the initial alignment procedure is initiated. When the timer expires, the alignment failure message is sent to the MTP3 layer and the MTP2 link is set to the not aligned state. This parameter only applies to the 64 kbit/s links. For the 2 Mbit/s links, the value of this timer is always 130000.

Default value:None
CT3 Timer Aligned

Meaning:Timer Aligned

Value range:1000~2000

Explanation: This parameter specifies the length of the timer for the MTP2 link to report link aligned status in the initial alignment procedure.

When the MTP2 link is set to the not aligned state, this timer is triggered to wait for the response from the MSC. When the timer expires, the link is set to the service disrupted state and the initial alignment starts again. This parameter only applies to the 64 kbit/s links. For the 2 Mbit/s links, the value of this timer is always 1000.

Default value:None
CT4N Timer Normal Proving

Meaning:Timer Normal Proving

Value range:7500~9500

Explanation: This parameter specifies the length of the timer for the MTP2 link to control the start of the normal alignment to wait for the response from the MSC.

In the initial alignment procedure, this timer is triggered when the MTP2 link is in the aligned state to wait for the normal alignment response from the MSC.When the timer expires, the link is set to the service disrupted state and the initial alignment starts again. This parameter only applies to the 64 kbit/s links. For the 2 Mbit/s links, the value of this timer is always 30000.

Default value:None
CT4E Timer Emergency Proving

Meaning:Timer Emergency Proving

Value range:400~600

Explanation: This parameter specifies the length of the timer for the MTP2 link to control the start of the emergency alignment to wait for the response from the MSC.

When the MTP2 link is in the aligned state, this timer is triggered to wait for the emergency alignment response from the MSC. When the timer expires, the link is set to the service disrupted state and the initial alignment starts again. This parameter only applies to the 64 kbit/s links. For the 2 Mbit/s links, the value of this timer is always 500.

Default value:None
CT5 Timer Sending SIB

Meaning:Timer Sending SIB

Value range:80~120

Explanation: This parameter specifies the length of the timer for the MTP2 link to control local congestion.

This timer is triggered when services are available on the MTP2 link.When the timer expires, the MTP2 congestion information is reported to the MTP3 layer and a system information block (SIB) is sent to the MSC. This parameter only applies to the 64 kbit/s links. For the 2 Mbit/s links, the value of this timer is always 100.

Default value:None
CT6 Timer Remote Congestion

Meaning:Timer Remote Congestion

Value range:3000~6000

Explanation: This parameter specifies the length of the timer for the MTP2 link to control peer congestion.

This timer is triggered when the BSC receives an SIB from the MSC.When the timer expires, the BSC stores the information and sends it after receiving a Fill-In Signal Unit (FISU). This parameter only applies to the 64 kbit/s links. For the 2 Mbit/s links, the value of this timer is always 5000.

Default value:None
CT7 Timer Excessive Delay of Acknowledgement

Meaning:Timer Excessive Delay of Acknowledgement

Value range:500~2000

Explanation: This parameter specifies the length of the timer for the MTP2 link (service available) to wait for the acknowledgement message from the peer end.

After the MTP2 link sends a service message to the MSC, this timer is triggered to wait for the response from the MSC. When the timer expires, the message is sent again. This parameter only applies to the 64 kbit/s links. For the 2 Mbit/s links, the value of this timer is always 2000.

Default value:None
CT9 Timer Interval of Resending FISU

Meaning:Timer Interval of Resending FISU

Value range:1~100

Explanation: This parameter specifies the length of the timer for the MTP2 link (no service message) to send an FISU message.

This timer is triggered when services are available on the MTP2 link. When the timer expires, the BSC sends an FISU message to the MSC if there is no service message to be sent on the MTP2 link. This parameter only applies to the 64 kbit/s links. For the 2 Mbit/s links, the value of this timer is always 10.

Default value:None

Example

1. Modify the parameters of an SS7 signaling link (BM Subrack No. = 0; Service Mode = SEPERATE(BM/TC Seperate); MTP3 Link No. = 0; Priority = 6; Congestion Start Threshold = 90; Congestion End Threshold = 60; Timer Alignment Ready = 40000):
MOD N7LNK: BMSRN=0, SERVICEMODE=SEPARATE, LINKNO=0, PRI=6, CGSTHD=90, CGETHD=60, CT1=40000;
2. Modify the parameters of an SS7 signaling link (BM Subrack No. = 0; Service Mode = TOGETHER(BM/TC Together); A Interface Slot No. = 14; MTP3 Link No. = 0; Priority = 6; Congestion Start Threshold = 90; Congestion End Threshold = 60; Timer Alignment Ready = 40000):
MOD N7LNK: BMSRN=0, SERVICEMODE=TOGETHER, ASN=14, LINKNO=0, PRI=6, CGSTHD=90, CGETHD=60, CT1=40000;
