Download and Decompress Patch Package(DLD PATCH)

Function

An OM command that downloads the hot patches from the FTP server to the GOMU/GBAM.The FTP server is the PC that is installed with LMT or M2000 server software.

Note

1. When a hot patch is downloaded, its status is Idle.

2. During the command running process, the GOMU/GBAM downloads the hot patch to the temporary storage location, and check the integrity of the hot patch through CRC check. If CRC check is successful, the hot patch is generated in the storage directory of the GOMU/GBAM. A successful or failed message is reported.

Parameters

ID Name Description
IP FTPServer IP

Meaning: IP address of the FTP server

Value range: 0.0.0.0-255.255.255.255

Explanation: This parameter specifies the IPv4 type as xxx.xxx.xxx.xxx.

Default value: None
USR FTPServer user

Meaning: user name used to log in to the FTP server

Value range: 0-24 characters

Explanation: 1.This parameter is a character string *******.

2.The default user name for logging in to the FTP server to which the LMT is connected is swmtest.

Default value: None
PWD User password

Meaning: user password used to log in to the FTP server

Value range: 0-24 characters

Explanation: 1.This parameter is a character string *******.

2.The default password for logging in to the FTP server to which the LMT is connected is swmtest.

Default value: None
DIR Source directory

Meaning: the FTP server directory where the files to be downloaded are stored

Value range: 1-127 characters

Explanation: This parameter is a character string *:\***\***.

Default value: None

Example

Download and decompress patch package:
DLD PATCH:IP="80.1.102.111",USR="swmtest",PWD="swmtest",DIR="Common\SWM\patch";
