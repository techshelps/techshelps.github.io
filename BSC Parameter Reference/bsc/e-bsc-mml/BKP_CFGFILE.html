Backup Configuration Data(BKP CFGFILE)

Function

An OM command that backups the active configuration data for the future restoration.

Note

Check the backup configuration data file through the command LST CFGFILE or upload the backup configuration data file through the command ULD CFGFILE.

Parameters

ID Name Description
FN File name

Meaning: file name

Value range: 1-32 characters

Explanation:

1. For a network management terminal, this parameter is optional. If you want to specify a file name parameter, then it must be named M2000.dat.

2. For other terminals, this parameter is mandatory and cannot be named M2000.dat.

3. File name cannot contain the space or characters such as "/, \, /, \, ', :, *, ?, ", <, >, |," , and it cannot contain two successive ".". File name cannot end with the character "." dots.Default value: None
FT File type

Meaning: File type

Value range: BIN(BIN format)

Explanation: This parameter specifies the format of the backup configuration data file.

Default value: BIN(BIN format)

Example

1. Back up the active configuration data to the NEs through the M2000. The backup file name is M2000.dat:
BKP CFGFILE:; or BKP CFGFILE:FN="M2000.dat";
2. Back up the active configuration data to the NEs through the LMT. The backup file name must be bak_2007_09_30.dat:
BKP CFGFILE:FN="bak_2007_09_30.dat";
