Set Software Parameters of PS Domain(SET PSSOFTPARA)

Function

A configuration command that modifies the soft parameters of the PS domain.

Note

1. The PS-domain soft parameters are preset by default when the system is initialized.

Parameters

ID Name Description
ALLOCATETWOULTS Allocate Two Timeslots

Meaning:Allocate Two Timeslots

Value range:NOTALLOCATE(Not Allocate Two Timeslots),ALLOCATE(Allocate Two Timeslots)

Explanation: This parameter specifies whether two timeslots can be assigned to the uplink of the MS.

When two timeslots are assigned to the uplink, two timeslots are assigned to the MS;

When two timeslots are assigned to the uplink, only one timeslot can be assigned to the uplink of the MS.

Default value:None
FORBIDEDGU Allow E Down G Up Switch

Meaning:Allow E Down G Up Switch

Value range:OPEN(Open),CLOSE(Close)

Explanation: This parameter specifies whether to enable the multiplexing of EDGE download and GPRS upload onto the same channel.

If this parameter is set to Open, the EDGE download and GPRS upload can use the same channel;

If this parameter is set to Close, the EDGE download and GPRS upload cannot use the same channel.

In dynamic allocation or extended dynamic mode, the downlink block must use the GMSK coding scheme (including CS1-4, MCS1-4) to detect the USF assigned for the uplink by the GPRS MS. Then, the downlink cannot use the high-rate coding scheme, thus decreasing the EGPRS rate.

If the Allow E Down G Up Switch is set to Close, this can prevent the EGPRS downlink and GPRS uplink from multiplexing the same channel to ensure the EGPRS rate. However, the channel needs to be properly allocated, the GPRS channel configured is to prevent decreasing the access of the GPRS MS due to no GPRS channel.

If the Allow E Down G Up Switch is set to Open, the downlink uses the GMSK coding scheme, thus decreasing the EGPRS rate.

At present, this parameter is usually set to Open.

Default value:None
ACOOPERATPAGINGSWITCH A Interface Collaboration Paging Switch

Meaning:A Interface Collaboration Paging Switch

Value range:CLOSE(Close),OPEN(Open)

Explanation: This parameter specifies whether the MS can receive CS paging on the A interface when handling the PS service.

If this parameter is set to Close, the MS performing PS services cannot be called;

If this parameter is set to Open, the MS performing PS services can be called.

Default value:None
DSPKIFFOFFTIMER DSP Kickoff Timer[s]

Meaning:DSP Kickoff Timer[s]

Value range:1~3600

Explanation: This parameter specifies the time waiting for the recovery of all the DSPs.

This timer is started when the first DSP recovers.

When all the DSPs in the subrack recover or the timer expires, the timer is stopped and cells are distributed on the DSPs recovered.

For example, value 1 specifies that the time length is one second, and value 3600 specifies that the time length is 3600 seconds.

Default value:None
DSPRECOVERTIMER DSP Recover Timer[s]

Meaning:DSP Recover Timer[s]

Value range:0~30

Explanation: This parameter specifies the time waiting for the recovery of the DSP before cells are redistributed.

This timer is started when a DSP fault is detected.

When a DSP fault recovers before the timer expires, if the DSP Recovers or the timer expires, the timer is stopped and cells re redistributed.

For example, value 0 specifies that the recovery time length is zero seconds and value 30 specifies that the recovery time length is 30 seconds.

Default value:None
PVCUSERATETIMER PVC Utilization Report Timer[ms]

Meaning:PVC Utilization Report Timer[ms]

Value range:1~65535

Explanation: This parameter specifies the periodic timer for the PVC utilization.

This timer is started when the board is initiated.

The PVC utilization is reported once when the timer expires.

Default value:None
FLUSHTRANSFERSWITCH PDU Transfer Configuration

Meaning:PDU Transfer Configuration

Value range:CLOSE(Close),OPEN(Open)

Explanation: When detecting that an MS initiates the cell reselection, cell update, or routing area update, the SGSN initiates a FLUSH-LL procedure to request the PCU to discard or transfer the MS data buffered.

Open: transfer;

Close: no transfer.

Default value:None
BCCONGESTALMBOUND BC Congestion Alarm Threshold

Meaning:BC Congestion Alarm Threshold

Value range:0~100

Explanation: This parameter specifies the threshold for the BC congestion alarm. If the BS congestion exceeds this threshold, the BC congestion alarm is reported.

Default value:None
BCCONGESTRESALMBOUND BC Congestion Restore Alarm Threshold

Meaning:BC Congestion Restore Alarm Threshold

Value range:0~100

Explanation: This parameter specifies the threshold for the BC congestion restore alarm. If the BS congestion is below this threshold, the alarm is cleared.

Default value:None
ALLOCDIFFTRX Reassignment TBF for Different Trx

Meaning:Reassignment TBF for Different Trx

Value range:NOTALLOW(Not Allow),ALLOW(Allow)

Explanation:This parameter specifies whether the TBF can be allocated to another TRX.

Default value:None
CLEARBITRATEASSINTBF Allow Clear BitRate In TBF

Meaning:Allow Clear BitRate In TBF

Value range:NOTALLOW(Not Allow),ALLOW(Allow)

Explanation:This parameter specifies whether the BITRATE in the TBF can be cleared.

Default value:None
SUPPORTDL5TS Support High Multislot Class

Meaning:Support High Multislot Class

Value range:NOTSUPPORT(Not Support),SUPPORT(Support)

Explanation:This parameter specifies whether the high multislot class is supported.High multislot class function enables a single MS to be allocated with a maximum of five timeslots on the uplink or downlink, thus improving the uplink or downlink throughput of a single MS. The BSS supports the MS with high multislot classes from 30 to 34. A maximum of five timeslots on the downlink can be assigned to an MS. If an MS is of the high multislot class 34, a maximum of five timeslots on the uplink can be allocated to the MS. Thus, the uplink and downlink throughput is increased by 25%.The total number of timeslots on the uplink and downlink cannot exceed six. That is, if five timeslots on the downlink are allocated to the MS, then only one timeslot on the uplink can be allocated to the MS.

The high multislot class function requires the support from the MS.For an MS of a high multislot class 32, 33, or34, the extended dynamic allocation (EDA) function must be used if more than three timeslots are required on the uplink. The parameter used for supporting the EDA is Support EDA.

Default value:None
USFGRAN4BLK Support USF Granularity 4 Switch

Meaning:Support USF Granularity 4 Switch

Value range:NOTSUPPORT(Not Support),SUPPORT(Support)

Explanation:This parameter specifies whether to support USF granularity 4.

Default value:None
FREQHOPSWITCH Support Frequency Hop Switch

Meaning:Support Frequency Hop Switch

Value range:NOTSUPPORT(Not Support),SUPPORT(Support)

Explanation:This parameter specifies whether the frequency hopping is supported.

Default value:None
SUPPORTEDA Support EDA

Meaning:Support EDA

Value range:NOTSUPPORT(Not Support),SUPPORT(Support)

Explanation:Extended dynamic allocation (EDA) enables the MS to be allocated with more timeslots on the uplink, thus improving the uplink rate and helping to transmit large amounts of data on the uplink.

Default value:None
RESREASSWHILEBLKNONCTRLPDCH Allow Resource Reassignment

Meaning:Allow Resource Reassignment

Value range:NOTALLOW(Not Allow),ALLOW(Allow)

Explanation:This parameter specifies whether the resource reallocation initiated by busy channels can be blocked.

Default value:None
ALLOWEDADAMULTIPLEX Allow EDA Multiplex

Meaning:Allow EDA Multiplex

Value range:NOTALLOW(Not Allow),ALLOW(Allow)

Explanation:This parameter specifies whether the EDA multiplexing is allowed.

Default value:None
NC2LOADRESEL NC2 Load Reset Switch

Meaning:NC2 Load Reset Switch

Value range:NotSupport(Not Support),Support(Support)

Explanation:NC2 load reselect switch.

Default value:None
LOADBROADCASTPERIOD NC2 Load Broadcast Period

Meaning:NC2 Load Broadcast Period

Value range:1~10

Explanation:NC2load broadcast period.

Default value:None
AQMMINTH Aqm MINth

Meaning:Aqm MINth

Value range:0~65535

Explanation: This parameter specifies the congestion threshold of the TBF buffer queue.

Default value:None
AQMTARTH Aqm TarTh

Meaning:Aqm TarTh

Value range:0~65535

Explanation: This parameter specifies the target length threshold of the TBF buffer queue.

Default value:None
AQMMAXTH Aqm MaxTh

Meaning:Aqm MaxTh

Value range:0~65535

Explanation: This parameter specifies the maximum length threshold of the TBF buffer queue.

Default value:None
AQMSWITCH Aqm Switch

Meaning:Aqm Switch

Value range:CLOSE(Close),OPEN(Open)

Explanation: This parameter specifies whether to enable the AQM algorithm.

Default value:None
AQMNINIT Aqm Ninit

Meaning:Aqm Ninit

Value range:0~65535

Explanation: This parameter specifies the initial value of the packet discarding interval N of the TBF buffer queue.

Default value:None
AQMNLOWERBOUND Aqm NLowerBound

Meaning:Aqm NLowerBound

Value range:0~65535

Explanation: This parameter specifies the lower threshold of the packet discarding interval of the TBF buffer queue.

Default value:None
AQMNUPBOUND Aqm NUpBound

Meaning:Aqm NUpBound

Value range:0~65535

Explanation: This parameter specifies the upper threshold of the packet discarding interval of the TBF buffer queue.

Default value:None
AQMM Aqm M

Meaning:Aqm M

Value range:0~65535

Explanation: This parameter specifies the update frequency of the packet discarding interval N, that is, number of the packets that are transmitted after which the packet discarding interval N is updated.

Default value:None
DISTRIBLIMIT Channel Distribution Upper Threshold on DSP

Meaning:Channel Distribution Upper Threshold on DSP

Value range:20~255

Explanation:Channel Distribution Upper Threshold on DSP.

Default value:None

Example

Modify the soft parameters of the PS domain (BC Congestion Alarm Threshold = 100):
SET PSSOFTPARA: BCCONGESTRESALMBOUND=100;
