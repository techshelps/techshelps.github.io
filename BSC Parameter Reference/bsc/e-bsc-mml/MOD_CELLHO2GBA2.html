Modify 2G BA2 of Cell Handover Parameters(MOD CELLHO2GBA2)

Function

A configuration command that modifies the 2G BA2 of the handover parameters of the cell. In this command, Cell Index or Cell Name uniquely identifies a cell.

Note

When "Support High Frequency Band" is set to "PCS1900", the frequency range in the BA table cannot be 811-885.

To ensure that the frequencies in the BA table are correctly encoded, some frequencies in the table should be restricted by rules. The set of these frequencies are called the rule-restricted frequency set.

The way of obtaining the rule-restricted frequency set varies with the types of frequency bands in the cell.

1. When "Support High Frequency Band" is set to "DCS1800":

If the frequency range of the BCCH is 1-124, the range of the frequencies in the rule-restricted frequency set is 0, 128-251, 512-885, and 955-1023.

If the frequency range of the BCCH is 0 or 975-1023, the range of the frequencies in the rule-restricted frequency set is 128-251, 512-885, and 955-974.

If the frequency range of the BCCH is 955-974, the range of the frequencies in the rule-restricted frequency set is 128-251 and 512-885.

In other cases, the rule-restricted frequency set is null.

2. When "Support High Frequency Band" is set to "PCS1900":

If the frequency range of the BCCH is 1-124, the range of the frequencies in the rule-restricted frequency set is 0, 128-251, 512-810, and 955-1023.

If the frequency range of the BCCH is 0, 975-1023, the range of the frequencies in the rule-restricted frequency set is 128-251, 512-810, and 955-974.

If the frequency range of the BCCH is 955-974, the range of the frequencies in the rule-restricted frequency set is 128-251 and 512-810.

In other cases, the rule-restricted frequency set is null.

In the case of a co-BCCH cell, the rule-restricted frequency set is divided into four sets according to the types of the frequency bands (GSM900, DCS1800, GSM850, and PCS1900). Each set is verified based on the following rules. All frequencies in the BA table can be correctly encoded only when all the four sets comply with the rules.

In the case of a non co-BCCH cell, all frequencies in the BA table can be correctly encoded when the rule-restricted frequency set complies with the following rules.

The rules are as follows:

Sort the frequencies in the frequency set in an ascending order. If the set complies with one of the following rules, the frequencies can be correctly encoded.

1. Any ARFCN is between 0 and 124.

2. The difference between the maximum ARFCN and the minimum ARFCN is smaller than or equal to 111.

3. In the case of the sorted frequency set, the difference between any adjacent frequencies is greater than or equal to 913.

4. The number of frequencies in a set is smaller than or equal to 16.

5. The number of frequencies in a set is 17 and the frequency [0] is contained in the set.

6. The number of frequencies in a set is smaller than or equal to 18 and the difference between any adjacent frequencies is greater than 512. (Note: The maximum ARFCN and the minimum ARFCN are also adjacent. The formula for calculating their difference is: 1024 - maximum ARFCN + minimum ARFCN. The formulas for calculating the difference between the maximum ARFCN and the minimum ARFCN in rules 7 and 8 are the same.)

7. The number of frequencies in a set should be smaller than or equal to 22. In addition, the difference between two adjacent frequencies should be greater than 768.

8. The number of frequencies in a set should be smaller than or equal to 29. In addition, the difference between two adjacent frequencies should be greater than 896.
