Roll Back Patch(RBK PATCH)

Function

An OM command that rolls the patch of the NE back to the state before the latest loading of the patch.

Note

1. Hot patch rollback is at the level of the entire NE. This command restores the version and status of the hot patch running on the board to the version and status before the latest loading. The hot patch rollback removes the loaded, activated, or running hot patch that is the latest from the board, and restores the hot patch to the status before the loading.

2. The patch are rollback at the level of the entire NE. The patch cannot be rolled back at the board level.

3. Normally, patches are applied in the SPH001 -> SPH002 -> SPH003 sequence. This means that the current deactivated or activated/confirmed SPH003 is rolled back to the running state of the SPH002.

4. The loading of hot patches allows skip of hot patch operations, after a latest hot patch is loaded. The command rolls back the hot patch to the previous version and status, no matter whether it is deactivated, activated or running or how long it runs.

5. A patch file can be rolled back only to the status before the patch installation but cannot be rolled back any further.

Parameters

None.

Example

Roll back patch:
RBK PATCH:;
