Mod Device IP Address(MOD DEVIP)

Function

A configuration command that modifies the parameters of the IP address of a device. In this command, the following parameters uniquely identify the IP address of a device:

Subrack No.

Slot No.

SubSystem No.

Note

1. The modified IP address of the device must be unique.

2. The modified IP addresses of the devices must not be in the same subnet as those of the ports.

3. If the IP address of a device is used by the M3UALNK or NSVLLOCAL, the IP address in the M3UALNK and NSVLLOCAl change with that of the device.

4. If the IP address of a device is used by the PTUPARA, the IP address and mask in the PTUPARA change with those of the device.

5. Modifying the IP address of the device disrupts the communication. You should perform the call re-establishment after running this command.

Parameters

ID Name Description
SRN Subrack No.

Meaning:Subrack No.

Value range:0~11

Explanation: This parameter specifies the subrack number of an IP-based interface board.

Default value:None
SN Slot No.

Meaning:Slot No.

Value range:0~27

Explanation: This parameter specifies the slot number of an IP-based interface board.

Default value:None
SSN SubSystem No.

Meaning:SubSystem No.

Value range:0~4

Explanation: The GFGUG board value is 0~4, the other IP-based interface board value is 0.

Default value:None
IPADDR IP address

Meaning:IP address

Value range:0.0.0.0-255.255.255.255

Explanation: This parameter specifies the equipment IP address of an IP-based interface board.

Default value:None
MASK Subnet mask

Meaning:Subnet mask

Value range:0.0.0.0-255.255.255.255

Explanation: This parameter specifies the network mask of equipment IP address on the IP-based interface board.

Default value:None

Example

Modify the IP address of a device (Subrack No. = 3; Slot No. = 20; SubSystem No. = 0; IP address = 10.171.35.123; Subnet mask = 255.255.255.0): 
MOD DEVIP: SRN=3, SN=20, SSN=0, IPADDR="10.171.35.123", MASK="255.255.255.0";
