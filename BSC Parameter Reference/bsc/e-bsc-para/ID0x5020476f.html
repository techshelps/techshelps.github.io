IBCA Allowed

Parameter Name

IBCA Allowed

Description

This parameter determines whether to enable the IBCA algorithm. The IBCA algorithm is used for assigning the channel with the interferences between channels taken into account. The assigned channel must be the channel with the minimum interference to other calls on the same timeslot and with the minimum interference from other calls on the same timeslot. The IBCA algorithm is compatible with the existing channel assignment mechanism, which also assigns the optimal channel according to the channel priority.

GUI Value Range

[No,Yes]

Default Value

No

Unit

None

Step

-

Modification Rules

None

Configuration Policy

The IBCA algorithm is generally applicable to the tight frequency reuse scenarios. In addition, the IBCA algorithm must be enabled on the synchronization network, which can be implemented through the technique such as GPS network synchronization and Um interface software synchronization.

Recommended Value

None

None

Parameters About Radio Resources

Caution

1. The IBCA algorithm applies to only the network where the Um interface is synchronized.

2. The IBCA algorithm applies to the TRXs participating in FH only. For the TRXs not participating in FH, the IBCA algorithm does not take effect.

3. The IBCA algorithm and the FlexMAIO function use the same mechanism, and thus they are mutually exclusive. That is, if the IBCA algorithm is enabled in a cell, the FlexMAIO function cannot be enabled in the same cell.

4. The IBCA algorithm does not support the external PCU because the MAIO cannot be modified dynamically with the PCU externally configured.

5. The IBCA algorithm does not support the DTM function.
