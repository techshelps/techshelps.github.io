CicPool Send Flag In Ho Req Ack

Parameter Name

CicPool Send Flag In Ho Req Ack

Description

This parameter specifies whether the HO_REQ_ACK message carries the circuit pool information.

This parameter should be set on the basis of whether the MSC supports the HO_REQ_ACK message carrying the circuit pool information.

GUI Value Range

[No,Yes]

Default Value

Yes

Unit

None

Step

-

Modification Rules

None

Configuration Policy

This parameter should be set on the basis of the negotiation with the MSC at the peer end.

If the MSC does not support the assignment complete message carrying the circuit pool information, this parameter is invalid even when this parameter is set to Yes.

Recommended Value

Yes

Parameter Relationship

None

BSC Feature

None

Parameter Class

Parameters About Equipment

Caution

Circuits are classified according to the bearing capability and service type.

The configuration of the circuit pool on the BSC side should be the same as that of the circuit pool on the MSC side. There are totally 22 types of circuit pools in Phase 2 or later-version protocol.

Where, circuit pools 1 and 3 support only the Full Rate Speech Service of type 1.

Circuit pools 4 and 6 support only the Enhanced Full Rate Speech Service of type 2.

Circuit pools 5, 7, 10, 13, 20, 21, and 22 support both the Full Rate Speech Service and the Enhanced Full Rate Speech Service.

Circuit pools 2, 8, 9, 11, 12, 14, 15, 16, 17, 18, and 19 support neither the Full Rate Speech Service nor the Enhanced Full Rate Speech Service.

According to the information about circuit pools, the BSC can check the circuits assigned and services supported by the MSC. If the circuit pool where the MSC-assigned circuits are located does not match the assigned services, the assignment fails. For example, if the circuit assigned by the MSC is in circuit pool 1 and the speech code algorithm is EFR, the assignment fails.
