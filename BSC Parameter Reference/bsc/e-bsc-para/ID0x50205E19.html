CicPool Send Flag In Ass Cmp

Parameter Name

CicPool Send Flag In Ass Cmp

Description

This parameter specifies whether the assignment complete message carries the circuit pool information.

This parameter should be set on the basis of whether the MSC supports the assignment complete message carrying the circuit pool information. Therefore, the MSC can process the assignment complete message properly.

GUI Value Range

[No,Yes]

Default Value

Yes

Unit

None

Step

-

Modification Rules

None

Configuration Policy

This parameter should be set on the basis of the negotiation with the MSC at the peer end.

If the MSC does not support the assignment complete message carrying the circuit pool information, this parameter is invalid even when this parameter is set to Yes.

Recommended Value

Yes

Parameter Relationship

None

BSC Feature

None

Parameter Class

Parameters About Equipment

Caution

Circuits are classified according to the bearing capability and service type.

The configuration of the circuit pool on the BSC side should be the same as that of the circuit pool on the MSC side. There are totally 50 types of circuit pools in Phase 2 or later-version protocol.

According to the information about circuit pools, the BSC can check the circuits assigned and services supported by the MSC. If the circuit pool where the MSC-assigned circuits are located does not match the assigned services, the assignment fails. For example, if the circuit assigned by the MSC is in circuit pool 1 and the speech code algorithm is EFR, the assignment fails. For details, refer to the 3GPP 48.008.
