Second User Type

Parameter Name

Second User Type

Description

This parameter specifies the type of the second user to be bound. The type of the second user can be TMSI, IMSI, or MSISDN.

GUI Value Range

[TMSI,IMSI,MSISDN,NULL]

Default Value

NULL

Unit

None

Step

-

Modification Rules

None

Configuration Policy

If you set the Second User Type to MSISDN (you are advised to bind the called number to the resource of the calling party):

1. If the resource of the calling party is bound, the MSISDN is the called number, that is, the number that the MS calls. For example, if the MS calls 12345, then the Second User Identity is 12345.

2. If the resource of the called party is bound, the MSISDN is the calling number. In this case, the caller ID display function must be enabled. For example, if 54321 is displayed on the MS, enter 54321 in the Second User Identity; if 8654321 is displayed on the MS, enter 8654321 in the Second User Identity.

If you set the Second User Type to TMSI or IMSI, you must determine the reassignment strategy at the MSC side.

1. If the MS initiates a TMSI call, the Second User Type is set to TMSI.

2. If the MS initiates an IMSI call, the Second User Type is set to IMSI.

When the Second User Type is set to NULL, the Second User Identity is invalid. In other words, the user to be bound is not specified.
