Jitter Buffer init time

Parameter Name

Jitter Buffer init time

Description

This parameter specifies the initial depth of the Jitter Buffer.

The purpose of setting this parameter is to remove jitters in the IP transmission.

Because a certain amount of jitters exist in IP transmission, the Jitter Buffer is used to buffer IP packets and remove some of the jitters. The larger the parameter is set, the stronger the Jitter Buffer's jitter clearing capability is. But this can also increase the voice transmission delay. On the other hand, the smaller the parameter is set, the weaker the Jitter Buffer's jitter clearing capability is. But this can reduce the voice transmission delay. Generally, this parameter can be set to 40 ms, that is, buffer 2 IP packets. This increases the delay by 40 ms.

GUI Value Range

[20ms,40ms,60ms,80ms,100ms]

Default Value

40ms

Unit

Milliseconds

Step

-

Modification Rules

None

Configuration Policy

This parameter is the initial value of the Jitter Buffer depth. If the condition of the network transmission becomes better, the Jitter Buffer depth decreases accordingly. On the other hand, if the condition of the network transmission becomes worse, the Jitter Buffer depth increases accordingly.

Recommended Value

40 ms
