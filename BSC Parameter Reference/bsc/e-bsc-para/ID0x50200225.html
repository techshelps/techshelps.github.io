Pg Max Message Number In Period

Parameter Name

Pg Max Message Number In Period

Description

This parameter specifies the maximum number of paging messages that the BSC can process over the A interface within a Pg State Period(ms).

When a Pg State Period(ms) begins, the BSC starts a timer. Each time a paging message is received from the A interface, the counter increases by 1. In a Pg State Period, if the number of paging messages received from the A interface exceeds this parameter, all the excessive paging messages are discarded until a new Pg State Period starts.

GUI Value Range

[10,400]

Default Value

160

Unit

None

Step

-

Modification Rules

None

Configuration Policy

If this parameter is set to a too great value, more paging messages are processed. When the BSC is overloaded, the control of the arrival rate of paging messages does not take effect.

If this parameter is set to a too small value, some paging messages that can be processed by the BSC are discarded, thus affecting the call success rate.

Recommended Value

160

Parameter Relationship

If the value of Pg State Period(ms) is changed, you need to change the value of this parameter accordingly.
