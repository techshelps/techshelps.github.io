Minimum Timeslots between Two Successive Channel Requests

Parameter Name

Minimum Timeslots between Two Successive Channel Requests

Description

This parameter specifies the minimum number of timeslots between two successive channel requests.

The MS sends an access request and waits for a response. If no response is received after the minimum number of timeslots, the MS resends the access request.

GUI Value Range

[12,15,20,30,41,55,76,109,163,217]

Default Value

20

Unit

None

Step

-

Modification Rules

None

Configuration Policy

If this parameter is set to an excessive value, the MS needs to wait for a long time before sending the next request. This may affect MS services.

If this parameter is set to a modest value, it is possible that a response is sent, but the MS has not received it because of transmission delay. In this case, the MS also resends the access request.

Recommended Value

20

Parameter Relationship

None

BSC Feature

GPRS&EGPRS
