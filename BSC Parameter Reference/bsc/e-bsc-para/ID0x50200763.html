CM Update Optimize Type

Parameter Name

CM Update Optimize Type

Description

This parameter specifies whether to optimize the classmark update procedure after the BSC receives the classmark update request message.

0: No optimization, namely, the standard classmark update procedure

1: Medium optimization

2: Optimal optimization

GUI Value Range

[0,2]

Default Value

0

Unit

None

Step

-

Modification Rules

None

Configuration Policy

If the value of ECSC is set to Yes and the value of this parameter is set to 2, the BSC neither sends a classmark update message to the MSC nor sends a classmark query request message to the MS. Instead, the BSC waits for the classmark update message from the MS.

When the value of ECSC is set to No and the value of this parameter is set to 1, the BSC sends a classmark update message to the MSC if the BSC has received from the MS a classmark update message with CM2 and CM3.

The BSC sends a classmark query request message to the MS in the following three conditions:

1. The value of ECSC is set to No and the BSC does not receive from the MS a classmark update message with CM2 and CM3.

2. The value of ECSC is set to Yes and the value of this parameter is set to 1 and the BSC does not receive from the MS a classmark update message with CM2 and CM3.

3. The value of ECSC is set to Yes and the value of this parameter is set to 0.
