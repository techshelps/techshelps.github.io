Compress Rate

Parameter Name

Compress Rate

Description

This parameter specifies the compress condition of the current BTS transmission. Only BTSs such as HDLC BTS, HDLC HubBTS, and IP BTS need to be configured with such parameter.When the value is converted to the actual one, it should be multiplied by 0.01. The default value is 100, which indicates no compression is required.

GUI Value Range

[1,250]

Default Value

100

Unit

None

Step

-

Modification Rules

None

Configuration Policy

According to the actual conditions of the current network, static bearing service quantity on the single E1 cable can be controlled.

If the value of the compression rate increases, you can infer that the allocated bandwidth of the Abis HDLC is larger than the bandwidth in non-compression mode. Thus, the number of TRXs configured on one E1 cable reduces.

If the value of the compression rate decreases, you can infer that the allocated bandwidth of the Abis HDLC is smaller than the bandwidth in non-compression mode. Thus, the number of TRXs configured on one E1 cable increases.

Recommended Value

100

Parameter Relationship

None

BSC Feature

None

Parameter Class

Parameters About Equipment

Caution

This parameter is also available in transmission optimization and the IP BTS.
