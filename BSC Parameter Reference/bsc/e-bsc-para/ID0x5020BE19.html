Maximum Value of N3105

Parameter Name

Maximum Value of N3105

Description

This parameter specifies the maximum value of N3105.

After a downlink TBF is established, the network initiates the N3105.

Upon setting the RRBP field in the downlink RLC data block, the network resets the N3105 when it receives the packet acknowledgment message from the MS on the uplink RLC data block corresponding to the RRBP field; otherwise, the network increases N3105 by one and resends the downlink data block in which the RRBP field is set.

When N3105 overflows, the network initiates the T3195. When the timer T3195 expires, the current TBF abnormally releases.

GUI Value Range

[3,10]

Default Value

10

Unit

None

Step

-

Modification Rules

None

Configuration Policy

If this parameter is set to a lower value, the tolerance of the network to downlink errors decreases and the probability of the frequent TBF release increases.

If this parameter is set to a higher value, the abnormal TBF may occur (such as the MS does not receive the message of current cell in the network caused by the MS activities, and the network still assigns the radio resources to the MS), the network cannot release this TBF, thus wasting the network resources.

Based on the actual condition of existing network (for example, the N3105 overflow caused by the bad Um interface quality, the unstable transmission link quality, and the MS activities), you should properly adjust this parameter to ensure the downlink TBF does not abnormally release due to the frequent overflow of the N3105.

Recommended Value

10
