Data to Be Negotiated and Planned for DBS3900 Initial Configuration

This describes the data to be negotiated and planned for the initial configuration of a DBS3900. Before configuring a DBS3900, you need to obtain the data of the BTS, which consists of site configuration, site attributes, RXU chain attributes, board attributes, cell attributes, cell broadcast parameters, and TRX attributes.

Before configuring a BTS, the following data must be obtained.

  • For the configuration data of the DBS3900 in TDM transmission mode, see Table 1.
  • For the configuration data of the DBS3900 in IP transmission mode, see Table 2.
  • For the configuration data of the DBS3900 in HDLC/HUB transmission mode, see Table 3.
NOTE:
  • CGI is comprised of MCC, MNC, LAC, and CI and is displayed in the format of "MCC-MNC-LAC-CI". In the following example, CGI is set to 460-01-8240-2.
  • If FH is applied, you need to obtain the FH serial number of each cell. FH serial numbers vary with different cells under the same BTS.
  • When the chain type is ring, the chain tail port number is required.
  • The parameter "Ring Mode" in "BTS attributes" indicates the network topology between the BSC and the BTS. The Chain Type in the RXU Chain Attribute specifies the cascading methods of the RF modules.
  • The RF modules are classified into the DRRU, DRFU, GRRU, GRFU, MRRU, and MRFU.
