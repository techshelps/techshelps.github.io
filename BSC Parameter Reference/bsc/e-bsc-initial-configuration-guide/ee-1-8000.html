Principles of Channel Configuration

The principles of channel configuration are as follows:

  • A cell must be and can be configured with only one Main BCCH, Combined BCCH, or BCCH+CBCH.
  • If the main BCCH channel type of a cell is Main BCCH, then the cell must be configured with one SDCCH (SDCCH or SDCCH+CBCH). The channels for the two-timeslot TRX of the two-timeslot cell cannot be included.
  • The Main BCCH, Combined BCCH, or BCCH+CBCH of the main BCCH TRX must be configured on channel 0.
  • If the main BCCH TRX is configured the BCH, then Channel Type for channel 0 must be Main BCCH. In addition, the BCH can be configured only for channel 2, channel 4, or channel 6.
  • The non-main BCCH TRX cannot be configured with Main BCCH, Combined BCCH, BCCH+CBCH, or BCH.
  • Based on the channel allocation algorithm and the dynamic channel conversion algorithm, it is recommended that when a static PDCH needs to be configured, configure the required channel in the sequence of 6, 5, 7, 4, 3, 2, 1, and 0 to maximize the use of the configured static PDCH. Otherwise, the configured static channel may not be fully utilized, thus resulting in the waste of resources, or the PS performance cannot be optimal. For example, if a TRX needs to be configured with one PDCH, then configure channel 6 as PDCH. If two PDCHs need to be configured, configure channel 6 and channel 5 as PDCHs. You can configure other channels by analogy.
  • When the parameter Multi-Density Trx Power Sharing is set to Dynamic Power Sharing, the following conditions should be met:
    • SD and BCCH must not configured at the same timeslot.
    • BCCH and SDCCH must configure staggered when there have space.
