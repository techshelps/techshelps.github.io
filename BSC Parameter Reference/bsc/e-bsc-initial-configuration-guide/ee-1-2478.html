Principles of Configuring the Built-in PCU Devices

This describes the principles of configuring the built-in PCU devices. When the built-in PCU is used, the BSC must be configured with the related boards to process the PS services, and the configuration must follow the related principles.

Principles of Configuring the Devices

The BSC should be configured with the GDPUP to process the PS services and configured with the GEPUG/GFGUG to provide the Gb interface. To determine the numbers of boards and Gb interface links, you need to obtain the following information:

  • Total number of PDCHs
  • Data transmission rate on the Um interface

The data transmission rate on the Um interface varies with the coding scheme. See Table 1.

The principles of configuring the GDPUP/GEPUG/GFGUG are as follows:

  • Each GDPUP can process services on 1,024 PDCHs. One redundant GDPUP is configured to improve the system reliability.
  • Each pair of GEPUPs/GFGUGs can process the services on eight GDPUPs. One BSC is usually configured with one pair of Gb interface boards. If more than one pair of Gb interface boards are required, configure them evenly in subracks.
  • Each pair of GEPUGs provide 32 E1 links, which support the traffic flow of 64 Mbit/s.
  • Each pair of GFGUGs provide eight FE electrical ports or two GE electrical ports, which support the traffic flow of 128 Mbit/s.
  • The number of E1 links to be configured on the Gb interface depends on the factors such as the number of PDCHs.
