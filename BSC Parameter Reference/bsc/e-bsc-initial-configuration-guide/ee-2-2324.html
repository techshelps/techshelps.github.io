Configuring the NSVC

This describes how to configure the Network Service Virtual Connection (NSVC) for service transmission on the Gb interface in Gb over FR mode. The NSVC and the BC are bound together to implement the data transmission between the BSC and the SGSN.

ScenarioBSC initial configuration and BSC capacity expansion
Mandatory/OptionalOptional. The NSVC is configured only when the Gb interface uses the FR transmission mode.
NOTE:
  • NSVC refers to the virtual connection on the NS layer. An NSVC is located on the BC of an E1/T1 and belongs to only one BC/NSE. A BC/NSE can be configured with multiple NSVCs.
  • One NSVC corresponds to one Permanent Virtual Connection (PVC). When configuring NSVC, you need to specify the mapping PVC.
  • The NSVC identifier and data link connection identifier on both ends must be consistent. The BC and NSE that the NSVC belongs to must be also consistent.

Prerequisites

  • The GEPUG is configured. For details, see Configuring the GEPUG (Gb over FR).
  • The NSE is configured. For details, see Configuring the NSE.
  • The BC is configured. For details, see Configuring the BC.

Procedure

  1. On the BSC6000 Local Maintenance Terminal, right-click GEPUG.

  2. Choose Configure NSVC from the shortcut menu. A dialog box is displayed, as shown in Figure 1.

    Figure 1 Configure NSVC dialog box

  3. Click Add. A dialog box is displayed, as shown in Figure 2.

    Figure 2 Add NSVC dialog box

  4. Set the parameters by referring to Table 1.
  5. Click OK to return to the dialog box, as shown in Figure 1.
  6. Click Finish. The configuration of the NSVC on the Gb interface is complete.
