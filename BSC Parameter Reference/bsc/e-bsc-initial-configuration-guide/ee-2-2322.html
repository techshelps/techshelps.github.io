Configuring the NSE

This describes how to configure the Network Service Entity (NSE) on the Gb interface and determine the subnetwork protocol type.

ScenarioBSC initial configuration and BSC capacity expansion
Mandatory/OptionalOptional. The NSE is configured only when the BSC is configured with the built-in PCU.
NOTE:
  • When adding an NSE, you need to set the NSE identifier, subnetwork protocol type, and agent subrack number.
  • The NSE identifier on the BSC side and that on the SGSN side must be consistent.
  • One BSC can be configured with up to 128 NSEs. Multiple NSEs can be assigned on the same agent subrack.

Prerequisites

  • The SGSN node is configured. For details, see Configuring the SGSN Node.
  • The built-in PCU is configured.

Procedure

  1. On the BSC6000 Local Maintenance Terminal, right-click the BSC6000 root node.

  2. Choose Configure PCU Data > Configure NSE from the shortcut menu. A dialog box is displayed, as shown in Figure 1.

    Figure 1 Configure NSE dialog box

  3. Click Add. A dialog box is displayed, as shown in Figure 2.

    Figure 2 Add NSE dialog box

  4. Set the parameters by referring to Table 1. Click OK to return to the dialog box, as shown in Figure 1.
  5. Click Finish. The configuration of the NSE is complete.
