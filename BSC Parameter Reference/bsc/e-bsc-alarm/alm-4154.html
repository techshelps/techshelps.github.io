4154 TRX main clock alarm

Description

This alarm is reported when A Carrier Module detects that main clock signal input to it is not available.

Attribute

Alarm ID Alarm Severity Alarm Type
4154 Major Fault

Parameters

ID Name Meaning

Number of the site where the faulty board or module is located

Value range: 0-2047

101:GRFU

102:GRRU

151:QTRU

Number of the fault board

Number of the passage where the faulty board is located.

Alarm sub-category number

Alarm Character

Impact on the System

  • The services on the DTRU will be interrupted if
  • The standby main control module is not configured
  • If the standby main control module is configured and the DTRU Slave Clock Alarm occurs.
  • A DTRU Slave Clock Alarm is also reported when this alarm occurs.

System Actions

Check if the alarm is cleared every certain period of time.

Possible Causes

  • The DCCU is faulty.
  • The main control module is faulty.
  • The DTRU hardware is faulty.
  • The cable distribution of the rack is faulty.

Procedure

  1. Replace the DCCU.

    Replace the DCCU, and check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm is not cleared. Go to procedure 2.

  2. Replace the main control module.

    Replace the main control module, and check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm is not cleared. Go to procedure 3.

  3. Replace the TRX.

    Replace the TRX, and check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm is not cleared. Go to procedure 4.

  4. Replace the rack.

    Replace the rack, and check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm is not cleared. Contact Huawei Customer Service Center.
