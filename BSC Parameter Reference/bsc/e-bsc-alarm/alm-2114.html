ALM-2114 TRX LAPD Link Interrupt Alarm

Description

A TRX reports this alarm upon detecting that an LAPD link is not available.

Attribute

Alarm ID Alarm Severity Alarm Type
2114 Major Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

135:TRX
3 Board No.

Number of the fault board

Value range: 0-71
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

255:None
6 Cell Index

Index of the cell where the board or module is located

0-2047: Cell index of the faulty cell in the data configuration

65535: None
7 Site Type

Type of the site where the faulty board or module is located

3: BTS30

5: BTS312

10: BTS3012A

11: BTS3006A
8 RackNo

Number of the rack where the faulty board is located.
9 BoxNo

Number of the subrack where the faulty board is located.
10 SlotNo

Number of the slot where the faulty board is located.
11 Site Name

Site Name
12 Alarm Attribute

Alarm Character

0:Normal

1:Transient Count

Impact on the System

All the services of the TRX will be interrupted upon occurrence of this alarm.

System Actions

  • When the alarm occurs, the BSC or TRX will automatically attempt to re-establish a link. If the attempt fails in 20 seconds, the BSC or TRX will turn off the corresponding power amplifier. When the alarm is cleared, the BSC or TRX will automatically turn.
  • If the TRX carries a BCCH, the system will start BCCH mutual-assistance.

Possible Causes

  • TRX hardware fault;
  • Data bus fault in the BTS racks;
  • Abis interface board fault.

Procedure

  1. Replace the TRX Board

    Replace the TRX and check whether the alarm is cleared.

    Y=> The alarm is cleared. The alarm handling is complete.

    N=> The alarm is not cleared. Go to step 2.

  2. Replace the Data Bus in the Racks

    Replace the data bus in the racks and check whether the alarm is cleared.

    Y=> The alarm is cleared. The alarm handling is complete.

    N=> The alarm is not cleared. Go to step 3.

  3. Check the Abis Interface Board. For details, see the BSC help system. If the alarm is not cleared, contact Huawei Customer Service Center.
