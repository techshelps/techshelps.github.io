4104 TRX Config Mismatch Alarm

Description

The alarm is reported when any of the following conditions occurs: (1) The frequency of the TRX is not consistent with the type of the TRX. (2) The static power of the TRX is lowered by the main control module to protect the DAFU. (3) The type of a board actually installed in the rack slot is different from that of the board configured on the BSC side. (4) The BTS detects that the channel type in the system message is different from that specified in the channel property, or the TRX is configured with incorrect frequencies.

Impact on the System

The TRX services where the alarm is generated are disrupted.

System Actions

The system automatically shuts down the PA when this alarm is generated. When the alarm is cleared, the TRX automatically turns on the PA.

Possible Causes

  • Tributary No. 1-7: The type of the board in the slot is different from the configured board type on the BSC side.
  • Tributary No. 8: The RF front end power is small.
  • Tributary No. 9 or 28: The configured ARFCNs and TRX types are inconsistent. The ARFCN range of P-GSM900 is from 1 to 124. The ARFCN range of E-GSM900 is from 0 to 124 and 975 to 1023. The ARFCN range of R-GSM900 is from 0 to 124 and 955 to 1023. The ARFCN range of DCS1800 is from 512 to 885. The ARFCN range of PCS1900 is from 512 to 810. The ARFCN range of GSM 850 is from 128 to 251.
  • Tributary No. 21: The attribute configurations of the RAN sharing mode are abnormal.
  • Tributary No. 22: The frequency configuration exceeds the operating range of the board.
  • Tributary No. 23: The configuration of the frequency band exceeds the operating range of the board.
  • Tributary No. 24: The configuration of frequency spacing of boards is abnormal.
  • Tributary No. 25: The power configuration of the board exceeds the TX capability of the board.
  • Tributary No. 26: The check of UMTS configuration of the board fails.
  • Tributary No. 27: The value of data configuration attributes is invalid.
  • Tributary No. 29: The alarm of abnormal co-cell configuration is reported.
  • Tributary No. 30: The configuration of TRX attributes is abnormal.
  • Tributary No. 31: Tx combine mode configuration and jumper physical connection mismatch.
  • Tributary No. 32: Jumper physical failure.
  • Tributary No. 255: The data configuration is incorrect. Since the LMT automatically checks all the data, no configuration alarm is reported in normal situation. Therefore, you should check that someone uses the LMT to configure the data.

Procedure

  1. Tributary No. 0 and 255:
    1. Check the data configuration.

      Log in to the LMT, select the browse mode, and then check whether the following data configurations are correct:

      (1) Check the channel attributes.

      (2) Check the CCCH parameters in the system information.

      (3) Check the channel type configuration.

      Y => The data configuration is correct. Go to procedure 2.

      N => The data configuration is incorrect. Go to procedure 2.

      (4) Modify the data configuration.

    2. Reset the BTS.

      Reset the site in level 4 on the LMT or the SMT. Check whether the alarm is cleared.

      Y => The alarm is cleared. No further action is required.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.

  2. Tributary No. 1, 2, 3, 4, 5, 6, and 7:
    1. Replace the TRX board.

      Replace the board in the slot according to the data configuration. Check whether the alarm is cleared.

      Y => The alarm is cleared. No further action is required.

      N => The alarm is not cleared. Go to step b.

    2. Set up the cell again.

      Set up the cell again. Ensure that the system types of the cell are consistent with the actual board types. Check whether the alarm is cleared.

      Y => The alarm is cleared. No further action is required.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.

  3. Tributary No. 8:
    1. Check the data configuration.

      Check whether the power level of the TRX in the data configuration is within the operating range of the RF front end. The range is determined by the power type of the DDPU, for example, supporting 60 W.

      Y => The alarm is cleared. No further action is required.

      N => The alarm is not cleared. Go to step b.

    2. Check the RF front end.

      Check the RF front end.. Check whether the alarm is cleared.

      Y => The alarm is cleared. No further action is required.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.

  4. Tributary No. 9 or 28:
    1. Check whether the ARFCNs are consistent with the TRX types.

      Log in to the LMT, and check whether the configured ARFCNs are consistent with the actual TRX types.

      Y => The configured ARFCNs are consistent with the actual TRX types, yet the alarm is not cleared. Contact Huawei Customer Service Center.

      N => The configured ARFCNs are inconsistent with the TRX types. Change the ARFCNs to be consistent with the TRX band. No further action is required.

  5. Tributary No. 29:
    1. Check whether there are carriers configured in only a part of the location groups in the co-cell configuration.

      Log in to the LMT, and check whether the configured ARFCNs are consistent with the actual TRX types.

      Y => The configured ARFCNs are consistent with the actual TRX types, yet the alarm is not cleared. Contact Huawei Customer Service Center.

      N => The configured ARFCNs are inconsistent with the TRX types. Change the ARFCNs to be consistent with the TRX band. No further action is required.

  6. Tributary No.30：

    Check the Trx configuration.

    Log in to the LMT，andCheck the configuration information on the LMTto check the config data is right or not.

    Y => Configuration is right，can not find the error，Contact Huawei Customer Service Center；

    N => Find out the error，adjust the config data. No further action is required.

  7. Tributary No. 21, 24, 26, 27, and 28:
    1. Check the data configuration.

      Log in to the LMT, select the browse mode, and then check whether the following data configurations are correct:

      (1) Check the channel attributes.

      (2) Check the CCCH parameters in the system information.

      (3) Check the channel type configuration.

      (4) Check frequency configuration.

      (5) Check the configuration information on the LMT.

      (6) Check the assigned frequency.

      Y => The data configuration is correct. Go to procedure 2.

      N => The data configuration is incorrect. Go to procedure 5.

      (7) Modify the data configuration.

    2. Reset the BTS.

      Reset the site in level 4 on the LMT or the SMT. Check whether the alarm is cleared.

      Y => The alarm is cleared. No further action is required.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.

  8. Tributary No. 26:
    1. Check whether the ARFCNs are consistent with the TRX types.

      Log in to the LMT, and check whether the configured ARFCNs are consistent with the actual TRX types.

      Y => The configured ARFCNs are consistent with the actual TRX types, yet the alarm is not cleared. Contact Huawei Customer Service Center.

      N => The configured ARFCNs are inconsistent with the TRX types. Change the ARFCNs to be consistent with the TRX band. No further action is required.

  9. Tributary No. 23:

    The current BTS version is not compatible. Contact Huawei Customer Service Center.

  10. Tributary No. 25:
    1. Check the data configuration.

      Log in to the LMT, and select the browse mode.

      Check TRX power mode.

      Check that Current Power Mode of the TRX is smaller than its Maximum Power Mode.

      Y => The data configuration is correct. Go to procedure 5.

      N => The data configuration is incorrect. Modify the data configuration. Lower the configured power.

  11. Tributary No. 22 and 27:
    1. Replace the TRX board.

      Replace the board in the slot according to the data configuration. Check whether the alarm is cleared.

      Y => The alarm is cleared. No further action is required.

      N => The alarm is not cleared. Go to step b.

    2. Set up the cell again.

      Set up the cell again. Ensure that the system types of the cell are consistent with the actual board types. Check whether the alarm is cleared.

      Y => The alarm is cleared. No further action is required.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.

  12. Tributary No. 31:
    1. Query the attributes of the RF unit, such as power type and transmission mode, through which the combination mode of the board can be determined. Check whether the combination mode in data configuration is consistent with the connections of jumpers for combination on the panel of the module.

      Y => The combination mode in data configuration is consistent with the connections of jumpers. Contact Huawei Customer Service Center.

      N => The combination mode in data configuration is inconsistent with the connections of jumpers. Go to sub-step b.

    2. If the combination mode in data configuration is incorrect, modify the data configuration, including the board TX/RX mode, power type, and transmission mode.

      If the connections of jumpers for combination on the panel of the module are incorrect, change the connections of jumpers in the equipment room.

      Check whether the alarm is cleared.

      Y => The alarm is cleared. No further action is required.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.

  13. Tributary No. 32:
    1. Check the connections of jumpers for combination on the panel of the module.

      In the equipment room of the base station, check whether the jumpers for combination on the panel of the module are faulty, such as disconnection, improper contact, and damage.

      Y => The jumpers are functional. Contact Huawei Customer Service Center.

      N => The jumpers are faulty. Go to sub-step b.

    2. Replace or reconnect the jumpers.

      Check whether the alarm is cleared.

      Y => The alarm is cleared. No further action is required.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.
