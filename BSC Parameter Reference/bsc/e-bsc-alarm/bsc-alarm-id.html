Changing the Environment Alarm Name of the BTS

This describes how to change the name of the environment alarm on the relevant BTS. The operation is performed after the related information of the environment alarm is collected on the site.

ScenarioNetwork planning
ImpactFor the active alarms, after this operation is performed, the existing alarms are manually cleared.
NEs InvolvedBSC, BTS, and M2000.

Prerequisites

  • This is a formal method to change the name of the environment alarm in BSC6000 V900R008, not a temporary method.
  • This method does not apply to the reported active alarms. To change the name of the active alarms, you must manually clear the alarms first.

Preparation

The operation is performed after the related information of the environment alarm is collected on the site.

Procedure

  1. Collect the related information of the environment alarm on site by referring to Table 1.

    Table 1 Environment alarm information

    Item

    Content

    Alarm ID

    Alarm ID to be changed.

    BTS Name

    Name of the BTS where the alarm is reported. If the alarm is reported on multiple BTSs, the information related to each BTS must be collected.

    Subrack Number

    Number of the subrack to which the environment alarm device is connected.

    Slot Number

    Number of the slot to which the environment alarm device is connected.

    Port Number

    Number of the port to which the environment alarm device is connected.

    Port Type

    BOOL-digital type, VALUE-analog type (rarely used at present).

  2. Bind the environment alarm ID. On the MML client, run the SET ALMPORT command to customize the alarm ID. For example: SET ALMPORT: SITENAME="BTS3012", SRN=0, SN=1, PN=1, AID=65384 parameters Site Name, Subrack Number, Slot Number, Port Number, and Port Type in the command specify the environment alarm, alarm ID, and the new alarm ID.

    NOTE:

    One alarm ID represents one type of environment alarm. One alarm ID cannot be bound with different types of environment alarm ports on different BTSs. For each BTS that reports the alarm, the environment alarm port must be bound with the alarm ID through MML commands. Before upgrading the data, you must uniformly plan the environment alarms first, write MML command scripts based on the rules, and then bind the environment alarm port on the relevant BTS with an alarm ID.

  3. Manually clear the reported environment alarm. If the active alarms contain the environment alarms, you must manually clear them on the BSC Local Maintenance Terminal.
  4. Change the name of the environment alarm. The name of the environment alarm is saved on the NE side and M2000 side, thus, you must change the name of the environment alarm on the M2000 client.
    1. On the M2000 client, choose Monitor > NE Alarm Settings > User Defined Alarm > Alarm Definition, and then click Query. On the displayed dialog box, click BSC6000 to query the user defined alarm.
    2. If the query results contain the relevant alarm information, select the record, and then click Modify. On the displayed dialog box, select the alarm name, severity, and the relevant NE. Click OK, and then click Apply to load the alarm information to the NE.
    3. If the query results do not contain the relevant alarm information, click Add. On the displayed dialog box, select the alarm name, severity, and the relevant NE. Click OK, and then click Apply to load the alarm information to the NE.
