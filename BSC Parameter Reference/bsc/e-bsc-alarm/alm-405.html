405 Cell Out of Service

Description

The BSC periodically checks the BCCH TRX and the RSL of the BCCH TRX. This alarm is generated when the RSL of the BCCH TRX is faulty or when the BCCH TRX is still faulty even after the TRX mutual aid is performed.

Attribute

Alarm ID Alarm Severity Alarm Type
405 Major Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site that is out of service
2 Cell Index.

Index of the cell that is out of service
3 Site Type

0=BTS22C,2=BTS20,3=BTS30,5=BTS312,7=BTS3001C-108 or BTS3001C-116,8=BTS3001C-204,9=BTS3002C,10=BTS3012A,11=BTS3006A,12=BTS3012,13=BTS3006C,14=BTS3002E,15=BTS3012AE,16=BTS3012 II,17=DBS3900 GSM,18=BTS3900 GSM,19=BTS3900A GSM,20=BTSDBS3036,21=BTS3036,22=BTS3036A,29=BTS3900B,30=BTS3900E
4 OutServiceCause

0=Manual blocking,1=Other OM operation,2=BTS power failure,3=Other causes,255=Null
5 Site Name

Name of the site where the cell is out of service
6 Cell Name

Name of the cell that is out of service
7 Cell CGI

Cell Global Identifier of the cell that is out of service
8 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN?

Impact on the System

The CS services and PS services of the out-of-service are disrupted.

System Actions

None.

Possible Causes

  • Manual block: The cell or BCCH TRX is manually blocked.
  • Other OM operations: This includes manual reset of the BTS or cell, and automatic BTS reset because of dynamic data configuration (for example, cell deactivation).
  • BTS power failure: There is power input for the BTS.
  • Other causes: The BTS is unavailable for use for a short period because the LAPD links or TRX attributes are adjusted; the BTS is reset because of the BSC reset; the BTS is automatically reset; the board is faulty; transmission failure occurs; the data configuration is improper; the BTS boards are reset manually.

Procedure

  1. Check the value of the parameter "out-of-service" in the alarm location information.
    1. If the parameter value is "1=other OM operations", check whether operations associated with the reset of the BSC or BTS is performed on the LMT by choosing BSC Maintenance > Maintain Logs > Query Operation Logs Online in the thirty minutes. You can also consult the maintenance personnel to check whether dynamic data configuration is performed.

      Y=> If the BSC or BTS is reset or if dynamic data configuration is performed, Go to e.

      N=> If the BSC or BTS is not reset or no dynamic data configuration is performed,Go to 2.

    2. If the parameter value is "0=manual block", Go to 3.
    3. If the parameter value is "2=BTS power failure", Go to 4.
    4. If the parameter value is "3=other causes", Go to 4.
    5. check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Go to 4.

  2. Check the loop status of the port and timeslot that carry the RSL of the main BCCH TRX.
    1. Check whether the port on the Abis interface board to which the faulty BTS is connected is looped.

      Y=> The port is looped, Go to b.

      N=> The port is not looped, Go to c.

    2. Stop the port loopback. Then, check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Go to d.

    3. Check whether the timeslot that carries the RSL of the main BCCH TRX is looped..

      Y=> The timeslot is looped, Go to d.

      N => The timeslot is not looped. Contact Huawei Customer Service Center.

    4. Stop the timeslot loopback. Then, check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.

  3. Check the cell and the main BCCH TRX of the cell.
    1. Check whether the cell is blocked manually.

      Y=> The cell is blocked manually. Go to b.

      N=> The cell is not blocked. Go to c.

    2. Unblock the cell. Then, check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Go to c.

    3. Check whether the main BCCH TRX of the cell is blocked manually.

      Y=> The main BCCH TRX is blocked manually. Go to d.

      N => The main BCCH TRX is not blocked. Contact Huawei Customer Service Center.

    4. Unblock the TRX. Then, check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.

  4. Check the BTS power supply.
    1. Contact the maintenance engineer of the BTS equipment and check whether the BTS is equipped with batteries.

      Y=> The BTS is equipped with batteries. Go to b.

      N=> The BTS is not equipped with batteries. Go to d.

    2. Log in to the LMT, and choose Alarm Maintenance > Browse Alarm. Based on the Site No. in the alarm location information, check whether the BTS reports the 5886 No Mains Supply Alarm or other associated alarms.

      Y => Any of the previous alarms is reported. Go to c.

      N => The alarm is not reported. Go to 5.

    3. Rectify the fault according to the alarm help. Then, check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Go to 5.

    4. Check whether the external power cable, the power cable between the DC arrester and the power input wiring terminal block, or the power cable from the cabinet top to the busbar is loose or burned out.

      Y=> The power cable is loose or burned out. Go to e.

      N=> The power cable is normal. Go to f.

    5. Secure or replace the power cables.
    6. Power the BTS, and then check whether this operation is successful.

      Y=> The BTS is powered successfully. Go to g.

      N=> The BTS is not powered successfully. The power supply system in the equipment room may be faulty. Please contact the maintenance personnel of the equipment room for troubleshooting.

    7. Check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.

  5. Check the parent alarm.
    1. Query the location information of the Abis interface board to which the faulty BTS is connected.
    2. Log in to the LMT, and choose Alarm Maintenance > Browse Alarm. Check whether the corresponding port on the Abis interface board reports any of the following parent alarms:

      20085 Loss of E1/T1 Multiframes (LOM)

      20086 E1/T1 Link Loopback

      20087 High BER over the E1/T1 Link

      20221 Loss of Clock on the Optical Port (RLOC)

      20222 Loss of Signals on the Optical Port (RLOS)

      20223 Loss of Received Signal Frames on the Optical Port (RLOF)

      20224 Out-of-Synchronization of Received Signal Frames on the Optical Port (ROOF)

      20226 Regenerator Section Signal Failure (RSF)

      20227 Regenerator Section Signal Deteriorated (RSD)

      20228 Multiplex Section Alarm Indication Signal (MAIS)

      20229 Multiplex Section Remote Defect Indication (MRDI)

      20230 Loopback on the Optical Port

      20236 High-Order Path Signal Failure (HPSF)

      20237 High-Order Path Signal Deteriorated (HPSD)

      20239 Loss of Tributary Unit Multi-frames (TULOM)

      20241 Loss of Tributary Unit Pointer (TULOP)

      20255 Optical Module Fault

      20969 Ethernet Link failure

      20978 Software Abnormal

      20981 Ethernet Port Duplex Mode Change

      Y => Any of the previous alarms is reported. Go to c.

      N => The alarm is not reported. Go to d.

    3. Rectify the fault according to the alarm help. Then, check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Go to 6.

    4. Log in to the LMT, and choose Alarm Maintenance > Browse Alarm. Check whether BTS corresponding to the Site No. in the alarm location information reports any of the following alarms.

      The trunk parent alarms are:

      4102 TRX LAPD Link Interrupt Alarm or Other Alarms Related to Transmission on the Abis Interface on the BTS Side.

      The hardware parent alarms are:

      4192 TRX communication alarm or other associated alarms

      4108 Radio link critical Alarm or other associated alarms

      4116 TRX Voltage Alarm

      4770 Site Reset Alarm or other associated alarms

      4106 TRX Software Running Abnormal Alarm or other associated alarms

      Y => Any of the previous alarms is reported. Go to e.

      N => The alarm is not reported. Go to 6.

    5. Rectify the fault according to the alarm help. Then, check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Go to 6.

  6. Check the Abis interface board to which the faulty BTS is connected and the GXPUM of the same subrack as the Abis interface board.
    1. Check whether the BSC board is faulty.

      Y=> The board is faulty. Go to b.

      N => The board runs normally. Contact Huawei Customer Service Center.

    2. Reset the board. Then, check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Go to c.

    3. Reinsert the faulty Abis interface board and GXPUM. Then, check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Go to d.

    4. Replace the board. Then, check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.
