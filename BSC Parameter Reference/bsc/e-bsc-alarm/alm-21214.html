21214 Low space disk

Description

This alarm is generated when the space of the OMU hard disk is insufficient and 10% less than the total space of the disk.

Attribute

Alarm ID Alarm Severity Alarm Type
21214 Minor Fault

Parameters

ID Name Meaning
1 Subrack No.

Number of the subrack where the alarm is generated
2 Slot No.

Number of the slot where the alarm is generated
3 Partition ID.

0=System partition, 1=Software partition, 2=Data partition
4 Used Space

Used Space of the Hard Disk
5 Remaining Space

Remaining Space of the Hard Disk
6 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

  • The traffic statistical files or other files may not be saved successfully if the space of the hard disk is insufficient.
  • The data backup between the active OMU and standby OMU fails.

System Actions

When the size of log files is larger than 15 GB, the system automatically deletes the file saved at the earliest time so that the size of log files does not exceed 15 GB.

Possible Causes

  • A large number of log files or other files not related to the OMU software are saved on the hard disk.
  • The application installed on the OMU hard disk is excessive. Thus, the available space is 10% less than the total space.

Procedure

  1. Log in to the GOMU by the PuTTY and then run the df -h command to query the disk partition information. Check whether the remaining disk space of the /BSC6000 directory is smaller than 3 GB and whether other data files exist in the partition with a usage above 80%.

    Y=> Go to 2.

    N=> The alarm is not cleared. Contact Huawei Customer Service Center.

  2. Move the .zip files in the directories /BSC6000/coredump/ and /BSC6000/data/mtndata/log/dbg/ to other memory devices. Do not delete the original directories and their subdirectories.

    Y=> The alarm is cleared. The alarm handling is complete.

    N=> The alarm is not cleared. Go to 3.

  3. Move the large-sized files in the /home directory to other memory devices.

    Y=> The alarm is cleared. The alarm handling is complete.

    N=> The alarm is not cleared. Contact Huawei Customer Service Center.
