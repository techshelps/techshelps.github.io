342 PTP BVC Faulty

Description

This alarm is generated when the PTP BVC protocol procedure is abnormal, or the PTP BVC is blocked.

Attribute

Alarm ID Alarm Severity Alarm Type
342 Major Fault

Parameters

ID Name Meaning
1 NSEI

ID of the NSE where the alarm is generated
2 BVCI

ID of the BVC where the alarm is generated
3 CellId

ID of the Cell where the alarm is generated
4 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

This PTP BVC cannot carry services.

System Actions

None.

Possible Causes

  • The PTP BVC protocol procedure is abnormal.
  • The PTP BVC is blocked.

Procedure

  1. Check the parent alarms.
    1. Log in to the LMT. Choose Alarm Maintenance > Browse Alarm to check whether any of the following parent alarms is reported.

      293 GB BC Faulty

      333 NSE Faulty

      Y => Go to b.

      N => Go to 2.

    2. If any of these alarms reported, rectify the fault according to the alarm help. Then, check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N => Go to 2.

  2. Log in to the LMT to check whether the Gb interface board is the GEPUG.

    Y => Go to 3.

    N => Go to 4.

  3. If the Gb interface board is the GEPUG, check whether any of the following parent alarms is reported.
    1. Log in to the LMT. Choose Alarm Maintenance > Browse Alarm to check whether any of the following parent alarms is reported.

      331 NSVC Faulty

      Y => Go to b.

      N => Go to 5.

    2. If any of these alarms reported, rectify the fault according to the alarm help. Then, check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N => The alarm is not cleared. Go to 5.

  4. If the Gb interface board is the GFGUG, check whether any of the following parent alarms is reported.
    1. Log in to the LMT. Choose Alarm Maintenance > Browse Alarm to check whether any of the following parent alarms is reported.

      332 NSVL Faulty

      Y => Go to b.

      N => Go to 5.

    2. If any of these alarms reported, rectify the fault according to the alarm help. Then, check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N => The alarm is not cleared. Go to 5.

  5. Run the MML command DSP PTPBVC to check whether the PTP BVC is blocked.

    Y => Go to 6.

    N => Go to 7.

  6. Run the MML command UBL PTPBVC to unblock the PTP BVC. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm is not cleared. Go to 7.

  7. Log in to the LMT, Trace the SIG BVC on the Gb Interface.Run the MML command RST PTPBVC to reset the PTP BVC. Then, check whether the alarm is cleared.

    Y=> The alarm handling is complete.

    N => Contact Huawei Customer Service Center."
