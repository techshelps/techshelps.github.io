ALM-3704 UPS Communication alarm

Description

This alarm is triggered when the communication between the UPS and BTS3002C fails.

Attribute

Alarm ID Alarm Severity Alarm Type
3704 Minor Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

129:IOMU
3 Board No.

Number of the fault board

0:IOMU
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

255:None
6 Cell Index

Index of the cell where the board or module is located

0-2047: Cell index of the faulty cell in the data configuration

65535: None
7 Site Type

Type of the site where the faulty board or module is located

9: BTS3002C
8 RackNo

Number of the rack where the faulty board is located.
9 BoxNo

Number of the subrack where the faulty board is located.
10 SlotNo

Number of the slot where the faulty board is located.
11 Site Name

Site Name
12 Alarm Attribute

Alarm Character

0:Normal

1:Transient Count

Impact on the System

When this alarm occurs, the alarms related to the UPS cannot be monitored and reported.

System Actions

None.

Possible Causes

  • The communication cable between the UPS and the BTS3002C is damaged.
  • The communication cable between the UPS and the BTS3002C is improperly connected.

Procedure

  1. Check whether the UPS has been installed.

    Check whether the UPS has been installed in the BTS3002C.

    Y=> Go to Step 3.

    N=> Modify the data configuration, that is, the UPS is not configured. Go to step 2.

  2. Modify the BSC data configuration

    Log in to the LMT online, modify the BTS attributes. You can see the parameter Alarm Shield Flag 2.

    Set Alarm Shield Flag 2 to Unequipped UPS. Check whether the alarm is cleared.

    NOTE:

    This parameter is hexadecimal. It uses bit 5 in the BTS3002C.bit5 = 0 The UPS is equipped.

    bit5 = 1 The UPS is equipped and the alarm can be reported.

    Y=> The alarm is cleared. The alarm handling is complete.

    N=> The alarm persists. The software system is abnormal. Please contact Huawei Customer Service Center.

  3. Check the communication cable between the UPS and the BTS3002C
    1. Check whether the communication cable between the UPS and the BTS3002C is properly connected. If no, power down the BTS3002C and re-connect the cable. Power up the BTS3002C again upon completion of the re-connection. Check whether the alarm is cleared.

      Y=> The alarm handling ends.

      N=> Go to Step b.

    2. Power down the BTS3002C. Remove the connector of the UPS alarm cable at the BTS side. Set the multimeter to a proper resistance range. Measure the resistance between the core and the outer layer of the BTS3002C output alarm cable. Check whether the cable is short circuited.

      Y=>Please contact Huawei Customer Service Center.

      N=> Go to step 4.

  4. Check whether the UPS works normally

    Ask the UPS manufacturer to check whether the UPS works normally.

    Y=> Go to Step 5.

    N=> Ask the UPS manufacturer to handle the alarm.

  5. Reset the IOMU

    Reset the IOMU and check whether the alarm is cleared.

    Y=> The alarm handling ends.

    N=> Some hardware in the BTS is faulty.Please contact Huawei Customer Service Center.
