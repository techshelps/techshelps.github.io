20960 2M Clock reference from line fault alarm

Description

Abnormality in 2M Signal(No.0) From SubBoardUp

Attribute

Alarm ID Alarm Severity Alarm Type
20960 Major Fault

Parameters

ID Name Meaning
1 Subrack No.

The number of the subrack
2 Slot No.

The number of the slot
3 CPU No.

The number of the CPU
4 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".
5 Cause Code

1=2M Clock reference source 0 from up subboard,2=2M Clock reference source 1 from up subboard,3=2M Clock reference from down subboard

Impact on the System

The recovered receive data clock can't be offered

System Actions

None.

Possible Causes

  • (1) The input signal from up subboard is lost.
  • (2) The frequency offset of input signal from up subboard goes beyond the pull-in range.

Procedure

  1. (1) Check whether the physical connection of the lines is proper.

    Y => Go to Step (2).

    N => Go to Step (3).

    (2) The service board may be damaged.

    1)Replace the service board.

    2)Check whether the alarm is cleared.

    Y => The alarm handling is complete.

    N => Go to Step(3).

    (3) Contact Huawei technical support.
