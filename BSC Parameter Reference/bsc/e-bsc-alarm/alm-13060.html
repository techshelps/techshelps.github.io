ALM-13060 Monitoring Unit Maintenance Link Broken

Description

This alarm is reported when the communication between the EMU/PMU/TCU/FMU/HPMU and the main control board is lost for a long time.

Attribute

Alarm ID Alarm Severity Alarm Type
13060 Major Fault

Parameters

ID Name Meaning

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

162:GMU

Number of the fault board

0:GMU
4 PassNo

Number of the passage where the faulty board is located.

4: The FMU maintenance link is broken.

Impact on the System

The BTS cannot monitor the working status of the monitoring units.

System Actions

None.

Possible Causes

  • The monitoring unit is not installed or is not powered on.
  • The communication between the monitoring unit and the main control board is faulty.
  • The BTS is reset. The alarm is cleared when BTS reset is complete.

Procedure

  1. Check whether the monitoring unit is installed and powered on.

    After the monitoring unit is properly installed and powered on, check whether the alarm is cleared:

    Y => The alarm is cleared. No further action is required.

    N => The alarm is not cleared. Go to procedure 2.

  2. Check the communication cable connections of the monitoring unit.

    Reinstall a cable that is not properly installed, or replace a cable that is damaged. Then, check whether the alarm is cleared:

    Y => The alarm is cleared. No further action is required.

    N => The alarm is not cleared. Go to procedure 3.

  3. Replace the monitoring unit.

    Replace the monitoring unit, and then check whether the alarm is cleared:

    Y => The alarm is cleared. No further action is required.

    N => The alarm is not cleared. Go to procedure 4.

  4. Check other units connected to the RS485 bus.

    The monitoring unit is connected to the RS485 bus of the BTS. The fault in a unit on the bus may affect other units on the bus. Therefore, disconnect the other units (such as the CMU, EMU, PMU, and FMU) from the RS485 bus one after another, and check whether the alarm is cleared:

    Y => The alarm is cleared. Replace the faulty unit and end the alarm handling.

    N => The alarm is not cleared. Go to procedure 5.

  5. Replace the main control board.

    Replace the main control board, and then check whether the alarm is cleared:

    Y => The alarm is cleared. No further action is required.

    N => The alarm is not cleared. Contact Huawei Customer Service Center.
