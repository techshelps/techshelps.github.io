Replace the BSC board

1) Check whether the ACT LED on the panel of the faulty board is on.

Y => Log in to the LMT and then choose BSC Maintenance > Maintain Device > Switch Over BSC Board to switch the board to the standby state.

N => Go to 2).

2) Remove the cable from the faulty board.

a. If the cable is an E1/T1 cable, use a flat-head screwdriver to loosen the screws from the connector of the E1/T1 cable. Then, remove the E1/T1 cable.

b. If the cable is an optical cable, remove the optical cable directly and then apply a dustproof cap on the optical connector. If the optical connector is dirty, clean it with a dustfree cloth or a fiber cleaner.

c. If the cable is an Ethernet cable, remove the Ethernet cable directly.

d. If no cable is connected to the board, skip this step.

3) Remove the faulty board from the subrack.

4) Insert the spare board into the subrack. The standby board automatically loads the data configuration file and the RUN LED blinks (on for 0.125s and off for 0.125s).

5) Install each cable on the new board.

a. If the cable is an E1/T1 cable, insert the connector of the E1/T1 cable into the socket on the board and then use a flat-head screwdriver to tighten the screws on the connector of the E1/T1 cable.

b. If the cable is an optical cable, remove the dustproof caps from the optical port on the new board and from the optical connector and then keep them for future use. Connect the optical cables according to the labels.

c. If the cable is an Ethernet cable, install the Ethernet cable directly.

d. If no cable is connected to the board, skip this step.

6) If the RUN LED on the board blinks (on for 1s and off for 1s), log in to the LMT and then choose BSC Maintenance > Maintain Device > Query BSC Board Information. In the displayed dialog box, set the parameters in the Board Location area and click Query. Then, click the Board Status tab and check whether the Usage state is Normal.

Y => The Usage state is Normal. Then, go to 7).

N => The Usage state is not Normal. Then, go to 8).

7) On the LMT, choose BSC Maintenance > Query Software > Query Board Version Information. Then, set the parameters in the Query Mode and Board Location areas. Click the Query button. Then, compare the query result on the Software and Hardware Information tab page with the Version Configuration Information form to check whether the result meets the version matching relations.

Y => The result meets the version matching relations. Then, go to 8).

N => The result does not meet the version matching relations. Ensure that the program files of the board in the bsc6000\bscswm\area1\software directory on the OMU are correct. If the files are incorrect, contact Huawei engineers for the correct files. Upgrade the host file. Load and activate the level 2 BIOS file. Then, reset the board through the LMT. Go to 6).

8) Check whether a board fault alarm is reported on the LMT.

Y => Find out the cause and clear the alarm according to the alarm information. Then, go to 7).

N => Go to 9).

9) Replacing the BSC interface board is complete.
