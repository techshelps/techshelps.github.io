20113 Faulty GE Link on the Sub-board of GXPUx

Description

The GE link of the pinch board is disconnected.

Attribute

Alarm ID Alarm Severity Alarm Type
20113 Major Fault

Parameters

ID Name Meaning
1 Subrack No.

Subrack No.
2 Slot No.

Slot No.
3 Cpu No.

CPU No.
4 CPU No. that corresponds to the GE port

The CPU No. corresponding GE port
5 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

The service on the faulty link CPU is interrupted.

System Actions

  • Handling of the active board in the active/standby operation mode: Perform the switchover if the standby is ready for the operation.
  • Handling of the standby board in the active/standby operation mode: None.

Possible Causes

  • The CPU which corresponds to the faulty link is reset.
  • The sub-board where the faulty link CPU resides is not securely inserted.
  • The sub-boar where the faulty link CPU resides is out-of-position.

Procedure

  1. Check whether a correlated parent alarm occurs.

    Y=> Clear the parent alarm according to the corresponding handling suggestions. Check whether the alarm is cleared.

    N=> Go to 2.

    Correlated parent alarm

    20030 Faulty Sub-Board

    The parent alarm and the child alarm have the same Subrack No. and Slot No. in the alarm location information.

  2. Check whether the alarm is cleared within 10 minutes.

    Y => The alarm is cleared. The alarm is caused by the restoration of the CPU. The alarm handling is complete.

    N => The alarm persists. Go to 3.

  3. Remove the GXPUx and insert tightly the sub-board where the faulty link CPU resides. Then reinstall the GXPUx and wait until the new board is successfully restarted. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 4.

  4. Remove the GXPUx and replace the sub-board where the faulty link CPU resides. Then reinstall the GXPUx and wait until the new board is successfully restarted. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 5.

  5. Replace the GXPUx and wait until the new board is successfully restarted. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Please contact Huawei Customer Service Center.
