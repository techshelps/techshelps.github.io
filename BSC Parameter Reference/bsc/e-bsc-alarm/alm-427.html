427 Configuration Data Not Supported by Current BTS Version

Description

The support capability provided by the BTS is inconsistent with data configuration; thus, the BSC fails to start up the BTS.

Attribute

Alarm ID Alarm Severity Alarm Type
427 Major Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site whose capability is inconsistent with data configuration.
2 Cause

0=BTS does not support FlexAbis.,1=BTS does not support physical/logical separation.,2=BTS does not support Abis transmission optimization.,3=BTS does not support automatic planning.
3 Site Name

Site Name
4 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

The initialization of the BTS fails.

System Actions

The initialization of the BTS fails when the alarm is generated.

Possible Causes

  • In configuration data, the BTS is set to support Abis transmission optimization but actually the BTS does not support Abis transmission optimization.
  • In configuration data, the BTS is set to support physical/logical separation but actually the BTS does not support physical/logical separation.
  • The BTS actually does not support the FlexAbis attribute but in configuration data the FlexAbis attribute is supported by the BTS.
  • The BTS is configured with automatic planning mode but actually it does not support this feature.

Procedure

  1. Check whether the cause code in the alarm location information is "BTS not supporting FlexAbis".

    Y=> Change the FlexAbis Mode of the BTS to Fix or upgrade the BTS software so that the BTS supports FlexAbis.

    N=> Go to 2.

  2. Check whether the cause code in the alarm location information is "BTS not supporting physical/logical separation".

    Y=> Upgrade the BTS version so that the BTS supports physical/logical separation.

    N=> Go to 3.

  3. Check whether the cause code in the alarm location information is "BTS not supporting Abis transmission optimization".

    Y=> Upgrade the BTS version so that the BTS supports Abis transmission optimization.

    N=> Go to 4.

  4. Check whether the cause code in the alarm location information is "BTS not supporting automatic planning".

    Y=> Upgrade the BTS version so that the BTS supports automatic planning.

    N=> Contact Huawei Customer Service Center .
