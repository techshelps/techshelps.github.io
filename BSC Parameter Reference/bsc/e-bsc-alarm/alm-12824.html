ALM-12824 LAPD Link Abnormal Alarm

Description

This alarm is reported when the communication between the BTS3900E and the BSC is abnormal due to broken LAPD links.

Impact on the System

When the alarm is generated, the services carried by the base station are disrupted.

System Actions

The BSC and the BTS try to perform the link setup automatically when the alarm is generated.

Possible Causes

  • The connection of the transmission links between the BSC and the BTS is abnormal.
  • The data configuration on the BSC side is faulty.
  • The interface board corresponding to the BSC is faulty.
  • The BTS3900E module hardware is faulty.

Procedure

  1. In IP mode,
    1. Check for the related alarms.

      Check whether the LMT also reports the ALM-12842 Ethernet Port Status Abnormal Alarm. If this alarm is reported, handle the alarm according to handling suggestions in ALM-12842 Ethernet Port Status Abnormal Alarm, and then check whether the alarm is cleared.

      If the alarm is not reported, go to step b.

    2. Check the data configuration.

      Check whether the IP information of the BSC and BTS3900E module are correctly configured.

      Y=> The configuration is correct. Go to procedure 3.

      N => The configuration is incorrect. Modify the configuration, and then go to procedure [3].

    3. Check the transmission link.

      Check whether the transport network between the BTS and the BSC is normal.

      Y => Remove and then install the cables to ensure that the connections are proper. Go to step d.

      N => Rectify the connection fault, and then go to step d.

    4. Check whether the alarm is cleared.

      Y=> The alarm is cleared. End the alarm handling.

      N => The alarm is not cleared. Go to procedure 3.

  2. In E1 mode,
    1. Check whether the LMT reports ALM-12846 E1/T1 Local Failure Alarm and ALM-12848 E1/T1 Remote Failure Alarm.

      Y=> The alarms exist. Handle the alarms according to the related handling suggestions to check whether the alarms are cleared. If the alarms are cleared, End the alarm handling. Otherwise, Go to step b.

      N => The alarms are not reported. Go to step b.

    2. Check whether the data on the timeslot cross-connection device is consistent with the on-site data, if the 64 k timeslot cross-connection or compression device is used.

      Y => Go to step c.

      N => Configure the cross-connection device again, and then check whether the alarm is cleared. If the alarm is cleared, end the alarm handling. Otherwise, go to step c.

    3. Check whether the interface board corresponding to the BSC is faulty.

      Y => The interface board is faulty. Replace the faulty board and check whether the alarm is cleared. If the alarm is cleared, end the alarm handling. Otherwise, go to step d.

      N => The alarm is not cleared. Go to step d.

    4. Check whether the data configuration of links such as OML and RSL between the BSC and the BTS3900E reporting the alarm is correct.

      Y => The data configuration is correct. Go to procedure 3.

      N => The data configuration is incorrect. Correct the data configuration and check whether the alarm is cleared.

  3. Replace the BTS3900E Module. Check whether the alarm is cleared.

    Y=> The alarm is cleared. End the alarm handling.

    N=> The alarm is not cleared. Contact Huawei Customer Service Center.
