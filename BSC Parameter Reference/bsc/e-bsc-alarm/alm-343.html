343 NSVL Dynamic Configuration Process Failure

Description

This alarm is reported when the communication link between the BSC and SGSN is faulty, or the configuration of SERVER IP/UDP interface between the BSC and SGSN is incorrect.

Attribute

Alarm ID Alarm Severity Alarm Type
343 Critical Fault

Parameters

ID Name Meaning
1 NSEI

ID of the NSE where the alarm is generated
2 Cause

0=Unknow cause,

1=SIZING:Send SNS-SIZE PDU failed,

2=SIZING:Wait for SNS-ACK timeout,

3=SIZING:"Invalid number of NS-VCs",

4=SIZING:"Invalid number of IP4 Endpoints",

5=SIZING:"Invalid number of IP6 Endpoints",

6=CONFIGING:Send SNS-CONFIG PDU failed,

7=CONFIGING:Wait for CONFIG-ACK timeout,

8=CONFIGING:"Invalid number of IP4 Endpoints",

9=CONFIGING:"Invalid number of IP6 Endpoints",

10=CONFIGING:"Invalid weights",

11=CONFIGING:Wait for Remote NSVL information timeout
3 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

All the PTP BVC in this NSE cannot carry services.

System Actions

None

Possible Causes

  • The communication link between the BSC and SGSN is faulty;
  • The configuration of SERVER IP/UDP interface between the BSC and SGSN is incorrect.

Procedure

  1. Check whether the IP connection between the IP-Interface-Boards of BSC and the SGSN is normal;

    Y => Go to 2;

    N => Connect BSC and SGSN again.

  2. Check whether the server IP address of BSC and SGSN are consistent;

    Y => Go to 3;

    N => Reconfigure the server IP address of BSC.

  3. Check whether the server UDP port number of BSC and SGSN are consistent;

    Y => The alarm is cleared.

    N => Reconfigure the server UDP port number of BSC.
