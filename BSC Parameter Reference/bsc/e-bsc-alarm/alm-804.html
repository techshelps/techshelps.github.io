804 Automatic Planning Optimization Failure Alarm

Description

This alarm is generated when the base station is faulty during the automatic planning and optimization process.

Attribute

Alarm ID Alarm Severity Alarm Type
804 Minor Event

Parameters

ID Name Meaning
1 Site No.

Number of the Automatic Planned and Optimized site
2 Malfunction Cause

1=The M2000 has no information about the frequencies that the base station requests,2=The M2000 has no information about the network operators that the base station requests,3=The algorithms supported by the M2000 do not match those requested by the NE,4=The automatic planning function is not enabled at the M2000,5=The configuration of the automatic planning parameters at the M2000 is incomplete,6=The frequency band reported by the base station does not match that configured in the cell,7=The M2000 fails to assign frequencies during automatic frequency planning,8=The M2000 is performing the automatic planning and thus the automatic planning cannot be restarted
3 Site Name

Site Name
4 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

The automatically planned base station cannot work.

System Actions

None.

Possible Causes

  • The M2000 has no information about the frequencies that the base station requests.
  • The M2000 has no information about the network operators that the base station requests.
  • The algorithms supported by the M2000 do not match those requested by the NE.
  • The automatic planning function is not enabled at the M2000.
  • The configuration of the automatic planning parameters at the M2000 is incomplete.
  • The frequency band reported by the base station does not match that configured in the cell.
  • The M2000 fails to assign frequencies during automatic frequency planning.

Procedure

  1. Check the value of the parameter Malfunction Cause in the alarm location information.
    1. The value of the parameter Malfunction Cause is 1 or 7. Check whether the M2000 configures the frequency band information of the automatically planned BTS.

      Y => Contact Huawei Customer Service Center.

      N => Configure the frequency band information of the automatically planned BTS at the M2000. Go to g.

    2. The value of the parameter Malfunction Cause is 2. Check whether the M2000 configures the operator information of the automatically planned BTS.

      Y => Contact Huawei Customer Service Center.

      N => Configure the operator information of the automatically planned BTS at the M2000. Go to g.

    3. The value of the parameter Malfunction Cause is 3. Check whether the automatic planning and optimization algorithms of the automatically planned BTS configured at the M2000 are consistent with those requested by the NE.

      Y => Contact Huawei Customer Service Center.

      N => Modify the automatic planning and optimization algorithms of the NE or those configured at the M2000. Go to g.

    4. The value of the parameter Malfunction Cause is 4. Check whether the automatic planning function is enabled at the M2000.

      Y => Contact Huawei Customer Service Center.

      N => Enable the automatic planning function. Go to g.

    5. The value of the parameter Malfunction Cause is 5. Check whether the configuration of the automatic planning parameters sent to the NE from the M2000 is complete.

      Y => Contact Huawei Customer Service Center.

      N => Modify the automatic planning and optimization algorithms of the NE or those configured at the M2000. Go to g.

    6. The value of the parameter Malfunction Cause is 6. Check whether the frequency band of the automatically planned BTS configured at the M2000 is consistent with the actual frequency band configured for the BTS.

      Y => Contact Huawei Customer Service Center.

      N => Modify the frequency band of the automatically planned BTS configured at the M2000 or change the hardware of the BTS. Go to g.

    7. Restart the automatic planning process and check whether the alarm is generated.

      Y => Contact Huawei Customer Service Center.

      N => The alarm is cleared. The alarm handling is complete.
