21019 Failed Loading of Board Software

Description

This alarm is reported when the loading of software files fails and the board cannot work normally during the board startup.

Attribute

Alarm ID Alarm Severity Alarm Type
21019 Critical Fault

Parameters

ID Name Meaning
1 Subrack No.

Number of the subrack where the board on which the loading fails is located
2 Slot No.

Number of the slot where the board on which the loading fails is located
3 CPU No.

Number of the CPU on which the loading fails (GXPUx has four CPUs numbered from 0 to 3; other boards have only one CPU numbered 0)
4 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

The corresponding board or CPU is not available. The board cannot work normally.

System Actions

The board is reset automatically. Make an attempt to load software from the GBAM/GOMU. If the loading fails again, the board will be reset repeatedly with an internal of about 10-30 seconds.

Possible Causes

  • The GSCU in the GMPS or where the GBAM/GOMU is located is faulty.
  • The communication among the GBAM/GOMU, the GSCU, and the faulty board is abnormal.
  • The files under the directory of the version area on the GBAM/GOMU are damaged.
  • The board hardware (such as flash device) is faulty.

Procedure

  1. Wait one minute, and then check whether the alarm is cleared.

    Y=> The alarm handling is complete.

    N=> Go to 2.

  2. Observe the RUN indicators on the board panels. If they blink slowly, the boards are operating at normal levels. Check whether the GBAM/GOMU or the GSCU is faulty.

    Y =>Please contact Huawei Customer Service Center.

    N => Go to 3.

  3. Check whether there are alarms 21210 CPU Overloadand21205 Board Faultin the Alarm Browse window on the LMT.

    Y=> Handle the alarm according to the handling suggestions of alarms 21210 CPU Overload and 21205 Board Fault.

    N=> Go to 4.

  4. Check whether the network connection between the GBAM/GOMU and the board is normal. Check whether the network connection between the GSCU and the board is normal.

    Reinsert or change the Ethernet cable. Then check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 5.

  5. Check whether there is the alarm21024 GBAM File Check Failurein the Alarm Browse window on the LMT.

    Y => Download the software version again. The alarm handling is complete.

    N => Go to 6.

  6. Change the board, and then check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Please contact the Huawei Customer Service Center.
