ALM-5084 Board Clock Abnormal

Description

This alarm is reported when system detects that the board clock is abnormal, for example, the oscillator does not output clock, the secondary phase-lock cannot lock the main clock, the working clock is abnormal, and so on.

Attribute

Alarm ID Alarm Severity Alarm Type
5084 Major Fault

Alarm Character

0:Normal

Impact on the System

  • Critical The board key oscillator clock or secondary phase-lock is abnormal. This may cause the board to fail to work.
  • Major Part of the board fails to work or the whole board cannot work normally.
  • Minor The board secondary oscillator clock is abnormal. The system can run normally but the reliability decreases.

System Actions

  • When the faulty code is 0, system automatically switches over to the standby crystal oscillator clock.
  • When the faulty code is 1, system doest not carry out any operation.
  • When the faulty code is 2, system sets the board as unavailable.
  • When the faulty code is 3, system sets the related ASIC as unavailable or the board as unavailable.

Possible Causes

The BBU hardware is faulty

Procedure

  1. Reset the board.

    Reset the board. Check whether the alarm is cleared:

    Y => The alarm is cleared. End alarm handling.

    N => The alarm is not cleared. Go to Step 2.

  2. Replace the board.

    Replace the board.Check whether the alarm is cleared

    Y => The alarm is cleared. End alarm handling.

    N => The alarm is not cleared. Please contact Huawei Customer Service Center.
