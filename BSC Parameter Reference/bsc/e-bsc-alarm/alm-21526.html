21526 SCCP Subsystem Prohibited

Description

The alarm is reported when the SCCP subsystem is prohibited.

Attribute

Alarm ID Alarm Severity Alarm Type
21526 Critical Fault

Parameters

ID Name Meaning
1 SCCP Sub System Index

Record of a Dest Signaling Point, which is automatically generated during the configuration
2 Net Indication

0=International,1=International Standby,2=National,3=National Standby
3 SCCP Dest Signaling Point Code

It may be the local signaling point or the DSP.
4 SCCP Subystem No.

It indicates the SCCP user
5 Local Signaling Point Code

Code of the signaling point in the signaling network, which is specified during data configuration of the signaling point
6 Subrack No.

Logical subrack number of the GXPUM CPU where the alarm is reported
7 Slot No.

Logical slot number of the GXPUM CPU where the alarm is reported
8 CPU No.

Logical CPU number of the GXPUM CPU where the alarm is reported
9 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

The services of the SCCP subsystem are stopped.

System Actions

The services of the SCCP subsystem are stopped.

Possible Causes

  • The DSP is inaccessible or the SCCP users prohibit the subsystem.
  • The SSP message is received from the remote end.
  • The state of the SCCP subsystem is disabled.

Procedure

  1. In the Brow Alarm window on the LMT, check whether 21525 SCCP DSP Unreachable is reported.

    Y => The SCCP DSP is inaccessible. Handle the alarm according to the suggestions in 21525 SCCP DSP Unreachable.

    N => Go to 2.

  2. Check the state of the SCCP subsystem.. Check whether the alarm is cleared.

    If the SCCP subsystem is in the OPERATION ALLOW state, the prohibit message of the destination point is received. You have to wait for the allow message of the destination point without any handling.

    If the SCCP subsystem is in the OPERATION PROHIBIT state, uninhibit the subsystem operation.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Please contact Huawei Customer Service Center.
