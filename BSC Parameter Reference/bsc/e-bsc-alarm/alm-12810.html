ALM-12810 BTS Configuration Conflict Alarm

Description

This alarm is reported when the configuration of the BSC is inconsistent with the actual installation, for example, the frequency set is wrongly configured, or the configured frequency is inconsistent with the band type of the BTS3900E.

Impact on the System

When the alarm is generated, the BTS3900E module related to such configuration may fail to work properly.

System Actions

None.

Possible Causes

  • The transmission between the BSC and the BTS is faulty.
  • The BSC Abis interface board is faulty.
  • The BTS3900E module hardware is faulty.
  • The configuration of combined cabinets is inconsistent with the actual connection.
  • The connections between combined cabinets are broken.
  • The frequency configured is inconsistent with the band type of the BTS3900E module.

Procedure

  1. Check the data configuration.

    Log in to the BSC and check the data configuration as follows:

    1. Check Channel Attributes.
    2. Check CCCH Parameters in the System Information [/#2003].
    3. Check Channel Type Configuration.
    4. Check Frequency Configuration.
    5. Check the Configuration Information on the LMT .
    6. Check the Assigned Frequency [/#2120]

      Y => The data configuration is correct. Go to procedure 2.

      N => The data configuration is incorrect. Go to procedure 2.

    7. Check whether the configured frequencies are consistent with the actual band type of the BTS3900E module.

      Log in to the BSC, and check whether the configured frequencies are consistent with the actual band type of the BTS3900E.

      Y=> The configured frequencies are consistent with the actual frequency band type of the BTS3900E module, yet the alarm is not cleared. Contact Huawei Customer Service Center.

      N=> Go to step h.

    8. Check whether the number of networking levels is consistent with the actual configuration.

      Y=> Check whether the ALM-12812 SFP Link Fault exists. If the alarm exists, check the handling suggestions of the ALM-12812 SFP Link Fault for troubleshooting.

      N => Send the configuration to the BTS again and then go to procedure [2].

    9. Modify Data Configuration.
  2. Reset the BTS.

    Reset the Site in Level 4 on the BSC or the LMT. Check whether the alarm is cleared.

    Y=> The alarm is cleared. End the alarm handling.

    N=> The alarm is not cleared. Go to procedure 3.

  3. Replace the BTS3900E module.

    Replace the BTS3900E Module that is faulty. Check whether the alarm is cleared.

    Y=> The alarm is cleared. End the alarm handling.

    N=> The alarm is not cleared. Contact Huawei Customer Service Center.
