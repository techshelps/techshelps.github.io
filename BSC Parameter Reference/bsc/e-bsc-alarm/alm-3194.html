ALM-3194 ASU E1 Port alarm

Description

This alarm is triggered when T_ALOS (2M analog signal loss), E1_LOS (2M line signal loss), T_LOXC (2M interface external clock loss) or T_DLOS (2M interface digital signal loss) occurs to the ASU.

Alarm ID Alarm Severity Alarm Type
3194 Major Fault

Impact on the System

If the E1 line between the BTS and its upper layer fails, all services supported by this BTS may be interrupted.

System Actions

None.

Possible Causes

  • The ASU branch interface circuit fails.
  • The configuration of the build-in optical transmission services are incorrect, as a result, E1 signals cannot be transmitted properly.
  • The build-in optical transmission branch cable is not in proper connection.
  • The E1 chip of the MMU fails.

Procedure

  1. Check the service configuration data of the ASU

    On the optical transmission network tube, select an icon of an network element, click the <Payload> button on the toolbar. Check the service configuration data: whether the service data of a timeslot is configured over a branch not in use, or whether the timeslot configured for the branch 1 does not support the service data required by the local site. Modify the service configuration if any error. Then check whether the alarm is cleared.

    Y=> The alarm handling ends.

    N=> Go to Step 2.

  2. Check the board configuration of the ASU branch

    On the optical transmission network tube, check whether the ASU branch is configured with "Loopback" or "Protect". If it is configured with "Loopback", then change the configuration into "Normal". If the network element is in a ring network channel protection, the branch should be configured with "Protect". If otherwise, modify the configuration as "No Protect". After the modification, check whether the alarm is cleared.

    Y=> The alarm handling ends.

    N=> Please contact Huawei Customer Service Center to replace the faulty inner container hardware.
