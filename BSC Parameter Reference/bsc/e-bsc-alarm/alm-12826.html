ALM-12826 Radio Link Abnormal Alarm

Description

This alarm is reported when the BTS detects continuous abnormal radio links. The following cases may lead to abnormal radio links: 1. Abnormal channel activation: The channel type or status of the BSC is inconsistent with that of the BTS. 2. Abnormal channel release: The ratio of abnormal radio channel release exceeds the alarm threshold. 3. Warning alarm of no traffic on the main BCCH: No traffic is detected on the main BCCH within the continuous detection period. The period is four hours by default can be set as required. 4. Imbalanced uplink and downlink levels: The uplink RX level measured by the BTS and the downlink RX level reported by the MS differ greatly.

Impact on the System

  • Warning for abnormal channel activation (sub-category 0)
  • The BTS fails to respond or has no respond to channel activation. In this case, call failures occur and traffic measurements are affected.
  • Warning for abnormal channel release (sub-category 1)
  • When the uplink or downlink link is faulty, proper interaction cannot be implemented on the radio interface. After channel activation, this channel fails to perform normal communication on the Um interface, which leads to release of the channel by the BSC.
  • Warning for no traffic on the main BCCH (sub-category 2)
  • Access or handover to this cell does not occur.
  • Warning for power level imbalance between uplink and downlink (sub-category 3)
  • When the uplink RX channel or downlink TX channel of the BTS is faulty, the traffic measurements are directly affected.
  • If the downlink TX channel of the main BCCH is faulty, the coverage and traffic of the cell are reduced.
  • If the downlink TX channel of the non main BCCH is faulty, the coverage of the BTS is reduced. If the channel is assigned to this BTS, the assignment failure may occur and handover success rate is lowered.
  • If the uplink RX channel of the main BCCH is faulty, within the coverage of this cell, MSs can have signals but fail to make calls. Therefore, traffic is reduced and handover success rate is lowered.
  • If the uplink RX channel of the non main BCCH is faulty and the channel is assigned to this BTS, the assignment failure may occur and handover success rate is lowered.

System Actions

The system starts auto detection after the alarm is generated. The alarm will disappear when the conditions are met.

Possible Causes

  • The channel types or channel states between BTS and BSC are not consistent.
  • The proportion of abnormal channel release exceeds the relevant alarm threshold.
  • There is no traffic detected on the main BCCH within the set period.
  • The BTS detects that the UL power level and the DL power level deviate from each other greatly.

Procedure

  1. Check the channel configuration of the main BCCH.

    Check whether the main BCCH is configured with the TCH channels.

    Y => Go to procedure 2.

    N => Check whether the TCHs are converted to the PDCHs. If yes, the alarm handling is complete. If not, go to procedure [2].

  2. Originate a call through the main BCCH.

    Lock the frequency of the test MS to the frequency of the main BCCH and originate a call. Check whether the alarm is cleared:

    Y => The alarm is cleared. End the alarm handling.

    N => The alarm is not cleared. Go to procedure 3.

  3. Reset the BTS.

    Reset the Site in Level 4 on the BSC or the LMT. Check whether the alarm is cleared.

    Y => The alarm is cleared. No further action is required.

    N => The alarm is not cleared. Go to procedure 4.

  4. Replace the BTS3900E module.

    Replace the 3900E Module that is faulty. Check whether the alarm is cleared.

    Y => The alarm is cleared. End the alarm handling.

    N => The alarm is not cleared. Contact Huawei Customer Service Center.
