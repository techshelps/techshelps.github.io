ALM-3192 Optical Transmit Channel alarm

Description

This alarm is triggered when the ASU receives an Rx Optical Path Alarm Indicator from the peer end.

Attribute

Alarm ID Alarm Severity Alarm Type
3192 Major Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

129:MMU
3 Board No.

Number of the fault board

0:MMU
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

255:None
6 Cell Index

Index of the cell where the board or module is located

0-2047: Cell index of the faulty cell in the data configuration

65535: None
7 Site Type

Type of the site where the faulty board or module is located

7: BTS3001C-108/BTS3001C-116

8: BTS3001C-204
8 RackNo

Number of the rack where the faulty board is located.
9 BoxNo

Number of the subrack where the faulty board is located.
10 SlotNo

Number of the slot where the faulty board is located.
11 Site Name

Site Name
12 Alarm Attribute

Alarm Character

0:Normal

1:Transient Count

Impact on the System

When this alarm occurs, if the fiber between the BTS3001C and the upper-level system fails, the services on the BTS3001C might be interrupted.

System Actions

None.

Possible Causes

  • Too much attentuation of the the optical path.
  • The fiber is broken or in bad connection or the fiber connector is damaged.
  • The Tx port of the optical module in the ASU is damaged.

Procedure

  1. Loop Westward and Eastward Fibers

    At tail fibers, loop westward and eastward fibers. Check whether the alarm is cleared.

    Y=>The alarm handling ends.

    N=>Go to step 2.

  2. Measure the power of optical signals at the tail fiber

    Measure the power of optical signals at the tail fiber and check whether the measured power is between -15dBm ~ -8 dBm.

    Y=>The optical channel is normal. Go to step 3.

    N=>The possible cause is that the hardware in the BTS3001C is faulty. Contact Huawei Customer Service Center.

  3. Check BTS3001C tail fiber connector and ring flange

    Check whether the BTS3001C tail fiber connector is clean and whether the ring flange connection is correct. If not, clean the tail fiber and connect the ring flange correctly. After that, check whether the alarm is cleared.

    Y=>The alarm handling ends.

    N=>Go to step 4.

  4. Check optical transmission network

    Measure the power of the optical signals from the near end at the NE involved in the Rx Optical Path Alarm Indicator. If the power is too low, the alarm might be cause by too much attentuation or bad connection of the fibers. Loop back the optical path segment by segment and measure the power of the optical signals on the optical path segment by segment to locate the faults. Remove the faults and check whether the alarm is cleared.

    Y=>The alarm handling ends.

    N=>The possible cause is that the hardware in the BTS3001C is faulty.Please contact Huawei Customer Service Center.
