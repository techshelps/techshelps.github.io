ALM-3572 LAPD alarm

Description

This alarm is triggered upon disconnection of the LAPD link.

Attribute

Alarm ID Alarm Severity Alarm Type
3572 Major Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

135:DRU
3 Board No.

Number of the fault board

0:DRU0

1:DRU1
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

255:None
6 Cell Index

Index of the cell where the board or module is located

0-2047: Cell index of the faulty cell in the data configuration

65535: None
7 Site Type

Type of the site where the faulty board or module is located

9: BTS3002C
8 RackNo

Number of the rack where the faulty board is located.
9 BoxNo

Number of the subrack where the faulty board is located.
10 SlotNo

Number of the slot where the faulty board is located.
11 Site Name

Site Name
12 Alarm Attribute

Alarm Character

0:Normal

1:Transient Count

Impact on the System

All services supported by the carrier are interrupted upon occurence of this alarm.

System Actions

When this alarm occurs, the system shuts off the power amplifier automatically.

Possible Causes

  • The transmission line is not in proper connection. For example, the transmission intermittence occurs.
  • The data configuration is incorrect.
  • The BTS hardware is faulty.

Procedure

  1. Check whether there are any correlated alarms

    Check whether ALM-3672 E1 Local alarm is reported on the LMT.

    Y=>Handle the ALM-3672 E1 Local alarm according to the corresponding handling suggestions to check whether the alarm is cleared .The alarm is cleared, Alarm handling is completed. The alarm is not cleared, Go to Step 2.

    N=>Go to Step 2.

  2. BER test

    Loopback transmission: If the BTS3001C adopts E1 transmission mode, then perform loopback test for the E1 cable from the BSC or the senior BTS. If the BTS3001C adopts optical transmission mode, perform loopback test for the optical fiber from the BSC or the senior BTS and test the timeslot corresponding to the LAPD to see whether there are any error codes.

    Y=> There is a fault on the peer transmission line. Repair the transmission line. The alarm handling ends.

    N=>Go to Step 3.

  3. Check the data configuration

    The operation procedure is as follows:

    1. Log in to the BSC6000 Local Maintenance Terminal.
    2. Right-click the faulty TRX and choose Configure TRX Prop on the shortcut menu.

      The Configure TRX Property Wizard is displayed.

    3. Click Set TRX Attributes.

      The Configure TRX Property dialog box is displayed.

    4. Click the Frequency Attributes tab.
    5. Check whether the settings of Out BSC Port No. and RSL In Site Port No. are consistent with their actual connections.
  4. Reset the BTS

    Perform fourth-level reset on the BTS3002C through the LMT. Check whether the alarm is cleared.

    Y=> The alarm handling ends.

    N=> Go to Step 5.

  5. Remove and reinsert the MCU and DRU

    Power down the BTS3002C. Remove and reinsert the modules on the backplane to ensure they are in well connection. Power on the BTS3002C and check whether the alarm is cleared.

    Y=> The alarm handling ends.

    N=> Go to Step 6.

  6. Replace the MCU and DRU and see if the alarm is cleared

    Y=>The alarm is cleared. Alarm handling is completed.

    N=>The alarm is not cleared. Please contact Huawei Customer Service Center.
