20194 Disconnected Link Between the Ports on the Master and Slave GSCU

Description

The link of network port between master and slave GSCU is disconnected.

Attribute

Alarm ID Alarm Severity Alarm Type
20194 Minor Fault

Parameters

ID Name Meaning
1 Subrack No.

Subrack No.
2 Slot No.

Slot No.
3 Cpu No.

CPU No.
4 Ethernet Port No.

1=network port between master and slave GSCU
5 PHY Fault Type

0=LINK UP,1=LINK DOWN
6 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

The communication between the active and the standby GSCUs is interrupted.

System Actions

None.

Possible Causes

  • The peer GSCU is reset.
  • The peer GSCU slot is empty.
  • One or both GSCUs are not securely inserted.
  • A GSCU is faulty.

Procedure

  1. Check whether the peer GSCU is in position.

    Y => The peer GSCU is in position. Go to 2.

    N => The peer GSCU is not in position. The fault is caused by the empty slot and the alarm handling is complete.

  2. Check whether the peer GSCU is reset.

    Y => The peer GSCU is reset. The fault is caused by the resetting, and the alarm handling is complete.

    N => The peer GSCU is not reset. Go to 3.

  3. Insert securely the peer GSCU and check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 4.

  4. Insert securely the local GSCU and check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 5.

  5. Reset the standby GSCU and wait until the board is successfully restarted. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 6.

  6. Reset the active GSCU and wait until the new board is successfully restarted. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 7.

  7. Replace the standby GSCU and wait until the new board is successfully restarted. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 8.

  8. Replace the active GSCU and wait until the new board is successfully restarted. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Please contact Huawei Customer Service Center.
