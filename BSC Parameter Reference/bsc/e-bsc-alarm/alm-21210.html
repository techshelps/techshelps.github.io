21210 CPU Overload

Description

This alarm is generated when the CPU usage exceeds 85% and this situation lasts for more than one minute.

Attribute

Alarm ID Alarm Severity Alarm Type
21210 Major Fault

Parameters

ID Name Meaning
1 Subrack No.

Number of the subrack where the board is generated
2 Slot No.

Number of the slot where the faulty CPU is generated
3 CPU No.

Number of the CPU that reports the alarm
4 CPU Usage

Actual utilization of the CPU
5 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

When the CPU usage is overhigh, the flow control occurs. As a result, the capability of processing services reduces and even the board is reset.

System Actions

None.

Possible Causes

  • The heavy traffic volume leads to overhigh CPU usage. Thus, this alarm is reported by the CPU.
  • The hardware or software fault of the board leads to overhigh CPU usage. Thus, this alarm is reported by the CPU.

Procedure

  1. Check whether the current traffic is increasing.

    Y => The alarm is cleared after the traffic reduces. The alarm handling is complete.

    N => The alarm is not cleared. Go to 2.

  2. Check whether any hardware or software fault of the board occurs.
    1. If the boards work in active/standby mode, log in to the LMT to perform the board switchover. If the boards do not work in active/standby mode, reset the board (the services will be interrupted). Then, check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N => Go to b.

    2. Remove and then reinsert the board when the traffic volume is low. Then, check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N => Go to c.

    3. Replace the board. Then, check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N=> Contact Huawei Customer Service Center.
