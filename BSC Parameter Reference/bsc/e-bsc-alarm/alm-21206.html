21206 CPU Fault

Description

This alarm is generated if the active CPU fails to detect the status of the standby CPU on a board that runs multiple CPUs, indicating that the CPU is not available.

Attribute

Alarm ID Alarm Severity Alarm Type
21206 Major Fault

Parameters

ID Name Meaning
1 Subrack No.

Number of the subrack where the faulty CPU is generated
2 Slot No.

Number of the slot where the faulty CPU is generated
3 CPU No.

Number of the faulty CPU in a board
4 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

In active/standby mode:

  • The generation of this alarm indicates that a CPU is faulty. Then, the switchover will be performed. If the switchover is successful, the services are not affected. If the switchover fails, the established calls drop, while the new calls will not be affected.

In separated mode:

  • The established calls on the CPU for which the alarm is generated drop, while the new calls will not be affected.

System Actions

  • Active board in active/standby mode: The switchover is performed if the switchover is allowed.
  • Standby board in active/standby mode: The alarm is not handled.
  • Board in separate mode: The alarm is not handled.

Possible Causes

  • The communication between the CPUs of the service boards is abnormal.
  • The CPU is faulty.

Procedure

  1. On the LMT, check whether the CPU is being loaded. After the data loading is completed successfully, check whether the alarm is cleared.

    Y => The alarm handling is complete.

    N => Go to 2.

  2. On the LMT, reset the board. In the case of non-active/standby configuration, the resetting leads to service interruption. Then, check whether the alarm is cleared.

    Y => The alarm handling is complete.

    N => Go to 3.

  3. Remove and then reinsert the board when the traffic volume is low. Then, check whether the alarm is cleared.

    Y => The alarm handling is complete.

    N => Go to 4.

  4. Replace the board. Then, check whether the alarm is cleared.

    Y => The alarm handling is complete.

    N => Contact Huawei Customer Service Center.
