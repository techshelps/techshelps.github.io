160 HDLC Link Fault

Description

This alarm is generated when the HDLC transmission is faulty.

Attribute

Alarm ID Alarm Severity Alarm Type
160 Minor Fault

Parameters

ID Name Meaning
1 HDLC No.

HDLC No.
2 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

The channel resources on the faulty HDLC are unavailable.

System Actions

None.

Possible Causes

  • The E1 port where the HDLC is configured is faulty.
  • The board where the HDLC is configured is faulty.
  • The interface board for BTS transmission is faulty.

Procedure

  1. Check the parent alarms.
    1. Log in to the LMT. Choose Alarm Maintenance > Browse Alarm to check whether the following parent alarm is reported.

      21205 Board Fault

    2. If the alarm is reported, rectify the fault according to the alarm help. Then, check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N => Go to 2.

  2. The Abis interface board where the HDLC is configured is faulty.
    1. Confirm the slot where the HDLC is configured.Check whether the board where the HDLC is configured is faulty.

      Y => Go to b.

      N => Go to 3.

    2. Remove and reinsert the board. Check whether the board runs normally.

      Y => The alarm handling is complete.

      N => Go to c.

    3. Replace the board. Then, check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N => Go to 3.

  3. The E1 port to which the HDLC is configured is faulty.
    1. Confirm the E1 port to which the HDLC is configured. Check whether the E1 port is faulty.

      Y => Go to b.

      N=>Go to 4.

    2. Reconfigure the E1 port. Then, check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N => Go to 4.

  4. The interface board for BTS transmission is faulty.
    1. Check whether any alarm related to the interface board for BTS transmission is reported.

      Y => Clear the alarm according to the corresponding handling suggestions. If the alarm is cleared, the alarm handling is complete.

      N => Go to b.

    2. Reconnect the interface board cables, and remove and reinsert the interface board. Then, check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N => Contact Huawei Customer Service Center.
