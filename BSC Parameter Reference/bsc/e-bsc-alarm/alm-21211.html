21211 Board Switchover

Description

This alarm is generated when the active board is replaced and then reset during the active/standby switchover.

Attribute

Alarm ID Alarm Severity Alarm Type
21211 Warning Event

Parameters

ID Name Meaning
1 Subrack No.

Number of the subrack where the board is switched over
2 Slot No.

Number of the slot where the active board is inserted
3 Swap Result

0=successful swap, 1=failed swap
4 Swap Reason

0=manual swap,1=fault swap
5 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

The services may be interrupted for a short period.

System Actions

None.

Possible Causes

  • When the active board receives a major alarm while the standby board is in good condition, the system automatically performs the active/standby switchover.
  • Manual operations lead to the active/standby switchover.

Procedure

  1. None.
