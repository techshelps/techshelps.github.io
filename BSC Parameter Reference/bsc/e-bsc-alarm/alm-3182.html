ALM-3182 Version Rollback alarm

Description

This alarm is triggered with concomitant auto version fallback when the version security record of the stack top is found to be 0 in the version stack check during MMU reboot.

Attribute

Alarm ID Alarm Severity Alarm Type
3182 Warning Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

129:MMU
3 Board No.

Number of the fault board

0:MMU
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

1:The version in the first Flash memory falls back.

2:The version in the second Flash memory falls back.
6 Cell Index

Index of the cell where the board or module is located

0-2047: Cell index of the faulty cell in the data configuration

65535: None
7 Site Type

Type of the site where the faulty board or module is located

7: BTS3001C-108/BTS3001C-116

8: BTS3001C-204
8 RackNo

Number of the rack where the faulty board is located.
9 BoxNo

Number of the subrack where the faulty board is located.
10 SlotNo

Number of the slot where the faulty board is located.
11 Site Name

Site Name
12 Alarm Attribute

Alarm Character

0:Normal

1:Transient Count

Impact on the System

When this alarm occurs, the problems caused by version inconsistency may occur concomitantly.

System Actions

None.

Possible Causes

  • The MMU is reset in no more than five minutes after it starts running normally. In this case, the MMU activating is not successful.
  • The version stack content in the FLASH memory of the MMU is erased by mistake.

Procedure

  1. Load the updated MMU software and activate it. After that, check whether the alarm is cleared.

    Y=>The alarm handling ends.

    N=>The possible cause is that the hardware in the BTS3001C is faulty.Please contact Huawei Customer Service Center.
