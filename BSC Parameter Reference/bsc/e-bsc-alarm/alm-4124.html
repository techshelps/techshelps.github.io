4124 TRX Hardware Prompt Alarm

Description

This alarm is reported when the TRX detects that the transmitting/receiving phase-locked loop or the test phase-locked loop in the TRX cannot output the required frequency.

Attribute

Alarm ID Alarm Severity Alarm Type
4124 Warning Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

65:DTRU/GTRU/DDRM

88:DRRU

92:DRFU

96:MRRU

100:MRFU

101:GRFU

102:GRRU

151:QTRU
3 Board No.

Number of the fault board
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

0: Receiving phase-lock loop 2 alarm(For the BTS3012 series)/Faulty hardware(For the BTS3900 series)

1: Receiving phase-lock loop 1 alarm

2: Transmitting phase-lock loop 2 alarm

3: Transmitting phase-lock loop 1 alarm

255: Test phase-lock loop alarm
6 Cell Index

Index of the cell where the board or module is located

0-2047: Cell index of the faulty cell in the data configuration

65535: None
7 Site Type

Type of the site where the faulty board is located.

12:BTS3012;

13:BTS3006C;

15:BTS3012AE;

16:BTS3012 II;

17:DBS3900 GSM;

18:BTS3900 GSM;

19:BTS3900A GSM;
8 RackNo

Number of the rack where the faulty board is located.
9 BoxNo

Number of the subrack where the faulty board is located.
10 SlotNo

Number of the slot where the faulty board is located.
11 Site Name

Site Name
12 Alarm Attribute

Alarm Character

0:Normal

1:Transient Count

Impact on the System

  • For the BTS3012 series
  • When the Sub No. is 0, 1, 2, or 3, the services may not be influenced for the send phase-lock loop and receive phase-lock loop both have backups.
  • When the Sub No. is 255, the fault will not affect the service but the RF loopback test function of the DTRU.
  • For the BTS3900 series
  • The fault will affect the service.

System Actions

  • Check if the alarm is cleared every certain period of time.

Possible Causes

  • The DTRU hardware is running abnormally.
  • The DTRU hardware is faulty.

Procedure

  1. When the Sub No. is 0, 1, 2, or 3 for the BTS3012 series, do as follows:
    1. Check for other relevant alarms.

      Check whether the LMT reports 4154 TRX main clock alarm.

      Y => The alarm is reported. Rectify the fault according to the handling suggestions of 4154 TRX main clock alarm. Check whether the alarm is cleared. If the alarm is cleared, the alarm handling is complete. If the alarm is not cleared, go to step b.

      N => The alarm is not reported. Go to step b.

    2. Check for 4184 TRX Clock Critical Alarm.

      Check whether the LMT reports 4184 TRX Clock Critical Alarm.

      Y => The alarm is reported. Rectify the fault according to the handling suggestions of 4184 TRX Clock Critical Alarm. Check whether the alarm is cleared. If the alarm is cleared, the alarm handling is complete. If the alarm persists, go to step c.

      N => The alarm is not reported. Go to step c.

    3. Reset the BTS.

      Perform a level-4 reset or power-off reset of the BTS on the LMT or SMT.

      You are advised to perform a power-off reset of the BTS, and check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Go to step d.

    4. Reinstall the main control module and TRX board.

      Power off the BTS. Reinstall each module. Ensure that each module is in good contact with the backplane. Power on the BTS, and check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. The hardware of the TRX or the main control module may be faulty. Contact Huawei Customer Service Center.

  2. When the Sub No. is 0 for the BTS3900 series, do as follows:

    Replace the TRU.

    Replace the DTRU, and check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm is not cleared. Contact Huawei Customer Service Center.

  3. When the Sub No. is 255, do as follows:
    1. Reset the DTRU.

      Reset the DTRU, and check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Go to step b.

    2. Replace the DTRU.

      Replace the DTRU, and check whether the alarm is cleared.

      Y => The alarm is cleared. The alarm handling is complete.

      N => The alarm is not cleared. Contact Huawei Customer Service Center.
