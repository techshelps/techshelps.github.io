21514 MTP2 Link Service Disrupted

Description

The alarm is reported by the MTP3 when the signaling link MTP2 reports the indication message of quitting the services.

Attribute

Alarm ID Alarm Severity Alarm Type
21514 Minor Event

Parameters

ID Name Meaning
1 PHY Link No.

Number of the link

Each link has only one number on the GXPUM CPU
2 Subrack No.

Number of the subrack where the board is located
3 Slot No.

Number of the slot where the GXPUM is located
4 CPU No.

Number of the CPU where the GXPUM is located
5 Network Indicator

0=International,1=International Standby,2=National,3=National Standby
6 Dest Signaling Point Code

Code of the DSP in the signaling network, which is specified when configuring data of the signaling point
7 Source Signaling Point Code

Code of the signaling point in the signaling network, which is specified when configuring data of the signaling point
8 Signaling Link Code

Only code of the signaling link in the link set that is a set of links
9 Cause

The values of alarm causes are as follows:

0x02: MTP3 sends the stop command.

0x08: The MTP2 reports that the location is impossible during the IAC.

0x09: MTP2 receives the abnormal BSN.

0x0A: MTP2 receives the abnormal FIB.

0x0C: MTP2 receives SIO from the destination point.

0x0D: MTP2 receives SIN from the destination point.

0x0E: MTP2 receives the critical SIE from the destination point.

0x0F: MTP2 receives SIOS from the destination point.

0x12: The error rate of the signal rate is high.

0x13: T1 (45 seconds) expires.

0x14: T6 (5 seconds) expires. The signaling links are congested for a long time.

0x15: T7 (1.5 seconds) expires. The messages are delayed for a long time.

0x16: T2 (30 seconds) expires.

0x17: T3 (1.3 seconds) expires.

0x19: The error rate is high.

0x1B: MTP3 sends a stop message during the LSC.

0x30: Receive the COO signal from the destination point.

0x31: The test of the signaling links fails.

0x32: The signaling links are deactivated.

0x33: Software on layer 2 is unavailable.

0x34: The communication with the management module stops.

0x35: Commands on layer 2 and layer 3 have no responses.

0x36: The physical links are faulty for a long time.

0x37: The link switchover expires.

0x38: Software data on layer2 are reconfigured.

0x40: The local management is prohibited, and the links are unavailable.

0x41: The remote management is prohibited, and the links are unavailable.

0x42: The remote processor is faulty.

0x44: Set data in the table again. The system restarts and reports alarms.

0xFF: Other reasons.
10 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

  • The links do not transfer the service. If no other links in the link set are available, the route of the links is unavailable.
  • If the DSP of a route does not have other routes, the DSP cannot be visited. The "21501 MTP3 DSP Inaccessible" alarm is reported.

System Actions

  • MTP2 stops sending the SLTM messages on the links. The switchover procedure of the link is initiated..
  • If no other links in the link set are available, the route of the links is unavailable.
  • If the DSP of a route does not have other routes, the DSP cannot be visited. The "destination point inaccessible" alarm is reported and received by the SCCP on the user plane. The user plane stops the services to the DSP. The MTP3 will periodically initiates signaling link re-establishment.

Possible Causes

  • The interruption of link is detected on the bottom MTP2.
  • The signaling links are deactivated.
  • The MSC is faulty.
  • The physical link over the channel is faulty. For example, the E1 cable is removed or the board is faulty.

Procedure

  1. In the Brow Alarm window on the LMT, check whether the21503 MTP3 Link Unavailable alarm is reported.

    Y => Handle the alarms according to the suggestions in the21503 MTP3 Link Unavailable. The alarm handling is complete.

    N => Go to 2.

  2. Check whether the signaling links are deactivated.

    Y => Activate the signaling links manually on the LMT.

    N => Go to 3.

  3. Handle the alarm according to the returned cause value, and check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => Please contact Huawei Customer Service Center.
