ALM-12310 Radio Link Abnormal Alarm

Description

This alarm is reported when the BTS detects continuous RL abnormalities. RL abnormalities include the following situations: a. Warning for abnormal channel activation. It indicates that the channel types or states of the BTS is inconsistent with those of the BSC. b. Warning for abnormal channel release. It indicates that the proportion of abnormal channel release exceeds the relevant alarm threshold. c. The traffic volume of the BCCH TRX is zero. The traffic volume of the BCCH TRX is zero in a continuous testing period. The default period is four hours. You can define the period. d. Warning for power level imbalance between UL and DL. It indicates that the UL power level and the DL power level deviate from each other greatly.

Impact on the System

The traffic statistics counters and cell traffic load where the alarm is generated are directly impacted, the handover success ratio is lowered.

System Actions

The system starts auto detection after the alarm is generated. The alarm will disappear when the conditions are met.

Possible Causes

  • The channel types or states of the BTS is inconsistent with those of the BSC.
  • The proportion of abnormal channel release exceeds the relevant alarm threshold.
  • There is no traffic detected on the main BCCH within the set period.
  • The BTS detects that the UL power level and the DL power level deviate from each other greatly.

Procedure

  1. Check the channel configuration of the main BCCH.

    Check whether the main BCCH is configured with the TCH.

    Y => Go to procedure 2.

    N => Check whether the TCHs are converted to the PDCHs. If yes, the alarm handling is complete. If not, go to procedure [2].

  2. Originate a call through the main BCCH.

    Lock the frequency of the test MS to the frequency of the main BCCH and originates a call. Check whether the alarm is cleared:

    Y => The alarm is cleared. End the alarm handling.

    N => The alarm is not cleared. Go to procedure 3.

  3. Reset the BTS.

    Reset the Site in Level 4 on the SMT or the LMT. Check whether the alarm is cleared.

    Y => The alarm is cleared. End the alarm handling.

    N => The alarm is not cleared. Go to procedure 4.

  4. Replace the BTS3900B module.

    Replace the BTS3900B Module that is faulty. Check whether the alarm is cleared.

    Y => The alarm is cleared. End the alarm handling.

    N => The alarm is not cleared. Contact Huawei Customer Service Center.
