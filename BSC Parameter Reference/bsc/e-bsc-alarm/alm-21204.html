21204 Unavailable Subrack

Description

Both active SCU and standby SCU are unavailable. Services related to the subrack are unavailable.

Attribute

Alarm ID Alarm Severity Alarm Type
21204 Critical Fault

Parameters

ID Name Meaning
1 Subrack No.

Number of the faulty subrack
2 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

Both active SCU and standby SCU are unavailable. Services related to the subrack are unavailable.

System Actions

None.

Possible Causes

  • The inter-subrack communication is faulty and the inter-subrack heartbeat is lost.
  • The logical object of the GSCU in the subrack is reset or abnormal.

Procedure

  1. Check the location information of the alarm.If the extension subrack reports the alarm, Go to [2].If remote subrack reports the alarm, Go to [4].

  2. Insert the Ethernet cable securely between the main subrack and extension subrack. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 3.

  3. Replace the Ethernet cable between the main subrack and extension subrack. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 6.

  4. Insert the E1 cable securely between the local subrack and remote subrack. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 5.

  5. Replace the E1 cable between the local subrack and remote subrack. Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Go to 6.

  6. Request the maintenance personnel to check whether the status of the GSCU board in the subrack is available.

    Y => The status of the GSCU is available. Contact Huawei Customer Service Center.

    N => The status of the GSCU is unavailable. Identify the cause for unavailability and reset the GSCU. The alarm handling is complete.
