20329 Unavailable Input Clock Source for GGCU

Description

All the clock module input resources of the GGCU board are unusable. The phase-lock loop is in non-tracing non-hold status.

Attribute

Alarm ID Alarm Severity Alarm Type
20329 Major Fault

Parameters

ID Name Meaning
1 Subrack No.

Subrack No.
2 Slot No.

Slot No.
3 Cpu No.

CPU No.
4 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

  • If the alarm is reported by the active board, the service will not be affected once the switchover between the active and standby boards succeeds.
  • If the alarm is reported by the active board, the service might be affected once the switchover between the active and standby boards fails.
  • If the alarm is reported by the standby board, the service will not be affected.

System Actions

  • Handling of the active board in the active/standby operation mode: Perform the switchover if the standby is ready for the operation.
  • Handling of the standby board in the active/standby operation mode: None.

Possible Causes

  • The clock reference is not configured.
  • The clock reference chosen in manual mode is unavailable.
  • All the clock references chosen in automatic mode are unavailable.
  • The clock reference with highest priority is in frequent switching between availability and unavailability in the automatic mode.

Procedure

  1. Check whether the clock reference is not configured.

    Y => The clock reference is not configured. Go to 2.

    N => The clock reference is configured. Go to 3.

  2. Configure the clock reference and access to the available clock reference. Wait for 10 minutes, and then check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Please contact Huawei Customer Service Center.

  3. Check whether the work mode is of manual mode.

    Y => The work mode is of manual mode. Go to 4.

    N => The work mode is of automatic mode. Go to 5.

  4. Check whether the current clock reference which is of manual mode is unavailable.

    Y => The current clock reference is unavailable. Go to 6.

    N => The current clock reference is available. Please contact Huawei Customer Service Center.

  5. Check whether all the clock references being configured are unavailable.

    Y => All the clock references being configured are unavailable. Go to 6.

    N => Some clock references are available. Please contact Huawei Customer Service Center.

  6. Clear the fault by adhering to the handling of 20322 Unavailable Current Clock Reference for GGCU and wait for 10 minutes, and then check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => The alarm persists. Please contact Huawei Customer Service Center.
