ALM-3794 Optical Receive Channel Alarm

Description

This alarm is triggered when the optical receiver in the IASU detects receive loss of signal (RLOS) or receive loss of frame (RLOF).

Attribute

Alarm ID Alarm Severity Alarm Type
3794 Major Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

55:IASU

Number of the fault board

0:IASU
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

0:westward optical path (on optical port 1)

1:eastward optical path (on optical port 2)

Impact on the System

When the alarm occurs, the transmission services on the optical path corresponding to the "Sub No." will be interrupted in a non-self-healing network. As a result, the services of all the lower-level sites will be interrupted and remote maintenance cannot be performed on these sites.

System Actions

In a self-healing network the system can automatically enable the service protection switching function and normally provide services.

Possible Causes

  • The power supply to the peer site fails or no peer site is installed.
  • The receiving optical power of the local site is low.
  • The fiber connector connected with the local or peer site is loose or damaged.
  • The pigtail fiber connected with the local or peer site is severely bent or broken.
  • The optical module in the local or peer site is faulty.
  • The cables between the local and peer sites are broken.

Procedure

  1. Check whether power supply to the peer site fails or no peer site is installed

    Check whether the power supply to the peer site fails or no peer site is installed based on the optical path corresponding to the "Sub No.".

    If the power supply to the peer site fails, repair the power supply devices.

    If the peer site is uninstalled, install a site or re-configure the logical system on the IASU in the local site as terminal multiplexer (TM).

    Check if the alarm is cleared.

    Y=> The alarm handling ends.

    N=> Go to Step 2.

  2. Check the fiber connectors and pigtail fibers

    Check whether the fiber connector is loose or blurred. If yes, clear it with alcohol and securely connect it to the port.

    Check whether the pigtail fiber connected with the local or peer site is severely bent or broken. If yes, bundle it well or replace it with a normal one.

    Check if the alarm is cleared.

    Y=> The alarm handling ends.

    N=> Go to Step 3.

  3. Check the transmit and receive optical power of the optical module

    Measure the receive optical power of the local site and the transmit optical power of the peer one with an optical power meter based on the optical path corresponding to the "Sub No.".

    Check whether the receive optical power is within -35 dBm to -28 dBm and Check whether the transmit optical power is within -15 dBm to -8 dBm.

    Y=> The receive and transmit optical power is normal.Please contact Huawei Customer Service Center

    N=> The measured transmit optical power is low.Please contact Huawei Customer Service Centerto check whether the fiber is in bad contact with the port and whether it is damaged. Verify that the fiber is normal, replace the MCU in the local site. The alarm handling ends.

    N=> The receive optical power is low.Please contact Huawei Customer Service Centerto check whether the fiber is in bad contact with the port and whether it is damaged. If yes, securely connect the fiber or replace it with a normal one. The alarm handling ends.
