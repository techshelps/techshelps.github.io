ALM-2116 TRX Config Mismatch Alarm

Description

This alarm is reported when any of the following conditions occur: [1]The TRX detects that the channel type in the system information issued by the BSC is inconsistent with the channel type in channel attributes. [2]The TRX is connected to the cavity combiner, but the TRX is configured with RF frequency hopping. [3]When the TRX, PBU, and CDU work together, the static power of the TRX exceeds the limited power. [4]The 16 kbit/s signaling switching function is configured, but the TRX does not support the 16 kbit/s signaling switching function. [5]The TRX frequency is inconsistent with the TRX type. [6]The actual board type in the slot is inconsistent with the configured board type on the BSC.

Impact on the System

  • The TRX cannot work when the alarm is generated on tributary No. 0-8, 10, 11, and 255.
  • If the alarm is generated on tributary No. 9, the TMU sends a TPC command "down" to set the static power level to a reasonable range. The coverage may be shrinked.

System Actions

  • If the alarm is generated on tributary NO. 8, the TRX shuts down the power amplifier (PA). After the alarm is cleared, the TRX automatically turns on the PA.
  • If the alarm is generated on tributary NO. 9, the TMU sets the TRX static power level to a lower range to protect the combiner and divider.
  • No automatic actions of the system in other cases.

Possible Causes

  • The type of the board installed in the slot is different from that configured on the BSC (tributary No. 0-7, 255).
  • Abnormal data configuration (tributary No. 8);
  • Abnormal data configuration (tributary No. 9) or low power of the combiner or divider;
  • 16 kbit/s signaling exchange is not supported by the TRX hardware (tributary No. 10);
  • The configured frequency is not in compliance with the TRX (tributary No.11)

Procedure

  1. Alarm on Tributary No. 0-7 or 255

    Replace the board according to the data configuration or modify the data configuration according to the installed board. Check whether the alarm is cleared.

    Y=> The alarm is cleared. The alarm handling is complete.

    N=> The alarm is not cleared. Contact the Huawei Customer Service Center.

  2. Alarm on Tributary No. 8
    1. Check the CCCH Configuration

      Query the system information to check whether the CCCH is in compliance with the channel configuration of the TRX attributes.

      Y=> Correct. Go to step b.

      N=> Incorrect. Modify the data configuration. The alarm handling is complete.

    2. Check the Connection Between the TRX and the Cavity Combiner

      Check whether the TRX and the cavity combiner are correctly connected and whether the FH mode is enabled. The TRX that is connected to the cavity combiner cannot be configured with the RF FH function.

      Y=> The connection and configuration are all correct. Contact the Huawei Customer Service Center.

      N=> Incorrect. Reset the configuration. The alarm handling is complete.

  3. Alarm on Tributary No. 9
    1. Check Whether the TRX is Working Properly with the PBU

      If the TRX is connected to the PBU (the PBU is on the left to the TRX and is connected to the antenna output port of the TRX), the combiner should be the ECDU. Query the CDU type on the BTS side. If the combiner is the CDU, change the CDU with the ECDU; if the combiner is ECDU, go to step b.

    2. Change the ECDU

      Replace the ECDU and check whether the alarm is cleared.

      Y=> The alarm is cleared. The alarm handling is complete.

      N=> The alarm is not cleared. Contact the Huawei Customer Service Center.

  4. Alarm on Tributary No. 10

    Replace the installed TRX board with a TRX board that can support 16 kbit/s signaling exchange. Check whether the alarm is cleared.

    Note: To obtain the hardware version, you can query the board information on the BSC side. If the last double-digit number of the hardware version is greater than or equal to 16, such as 22.10.16, the TRX board can support the 16 kbit/s signaling exchange.

    Y=> The alarm is cleared. Alarm handling is complete.

    N=> The alarm is not cleared. Contact the Huawei Customer Service Center.

  5. Alarm on Tributary No. 11

    Check Whether the Frequency Is in Compliance with the TRX Type. See the following steps:

    1. Enter the System Dynamic Configuration Work Mode;
    2. Choose Dynamically Configure > Changing TRX Frequencies
    3. Select the target cell in the Dynamically Configure TRX Frequencies dialog box. Click Next;
    4. Select the target frequency. Click Changing TRX Frequencies;
    5. Check whether the assigned frequency of the TRX is a valid frequency.

      Frequency range of P-GSM900: 1-124;

      Frequency range of E-GSM900: 0-124, 975-1023;

      Frequency range of R-GSM900: 0-124, 955-1023;

      Frequency range of DCS1800: 512-885.

      Y=> The assigned frequency of the TRX is in compliance with the TRX type, the alarm is not cleared. Contact the Huawei Customer Service Center.

      N=> The assigned TRX frequency is not in compliance with the TRX type. Configure the TRX frequency. The alarm handling is complete.
