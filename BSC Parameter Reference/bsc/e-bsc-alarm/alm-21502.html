21502 MTP3 Layer 3 Congestion

Description

This alarm is reported when the usage of the cache memory of the layer 3 message exceeds that of the start threshold in the buffer area of the layer 3 message.

Attribute

Alarm ID Alarm Severity Alarm Type
21502 Major Fault

Parameters

ID Name Meaning
1 PHY Link No.

Each link has only one number on the GXPUM CPU
2 Subrack No.

Logical subrack number of the XPUM CPU to which the signaling link belongs
3 Slot No.

Logical slot number of the GPUM CPU to which the signaling link belongs
4 CPU No.

Number of the CPU where the GXPUM is located
5 Network Indicator

0=International,1=International Standby,2=National,3=National Standby
6 Dest Signaling Point Code

Code of the DSP in the signaling network, which is specified when configuring data of the signaling point
7 Source Signaling Point Code

Code of the signaling point in the signaling network, which is specified when configuring data of the signaling point?
8 Signaling Link Code

Only code of the signaling link in the link set that is a set of links
9 Buffer Usage

The usage of the cache memory occupied by the Layer 3 message over this signaling link.
10 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

When the layer 3 congestion alarm over the signaling link is reported, the user information will be discarded on the MTP3 layer.

System Actions

The MTP3 reports the congestion primitive to the SCCP, requiring that the SCCP and the SCCP users lessen the MSUs to eliminate the congestion.

Possible Causes

Too many messages are sent to layer 3.

Procedure

  1. In the Brows Alarm window on the LMT, check whether the alarm is cleared within 10 seconds.

    Y => The alarm handling is complete.

    N => Go to 2.

  2. Add a signaling link.

    For details about how to add a signaling link, refer to the HUAWEI BSC6000 Base Station Controller Data Configuration Guide.

    Check whether the alarm is cleared.

    Y => The alarm is cleared. The alarm handling is complete.

    N => Please contact Huawei Customer Service Center.
