21257 DPU Service Reduced

Description

This alarm is generated when the DPU capability of processing services reduces if at least two DSPs of the DPU are not available for use.

Attribute

Alarm ID Alarm Severity Alarm Type
21257 Major Fault

Parameters

ID Name Meaning
1 Subrack No.

Number of the subrack where the alarm is generated
2 Slot No.

Number of the slot where the board is generated
3 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

  • The system capability of processing the traffic is affected.
  • All calls carried on the occupied speech channels being processed on the DSP drop. There are a maximum of 44 speech channels.

System Actions

  • The system automatically resets the DSP and the DSP then attempts to reload the file every 10 seconds until the loading is successful. The reloading lasts up to 20 minutes. If the DSP is manually reset, the DSP can attempt to reload the file for up to five times.
  • If 11 DSPs are faulty, 15 minutes later, the system resets the DPU and reloads the DPU. The number of reloadings is not limited.

Possible Causes

  • The GXPUM is faulty.
  • The DPU is initially started.
  • The file loading to the DSP fails.
  • The DSP chip is faulty.

Procedure

  1. On the LMT, check whether at least one GXPUM in the BM subrack is normal.
    1. Log in to the LMT, choose Alarm Maintenance > Browse Alarm to check whether the following alarm is reported.

      21205 Board Fault

    2. Rectify the fault according to the alarm help. Then, check whether the alarm is cleared.

      Y=> The alarm handling is complete.

      N=> Go to 2.

  2. Reset the TRX on the LMT. Then, check whether the alarm is cleared.

    Y=> The alarm handling is complete.

    N=> Go to 3.

  3. Reset the DPU on the LMT.

    Y=> The alarm is cleared. The alarm handling is complete.

    N=> The alarm is not cleared. Go to 4.

  4. Replace the DPU.

    Y=> The alarm is cleared. The alarm handling is complete.

    N=> The alarm is not cleared. Contact Huawei Customer Service Center.
