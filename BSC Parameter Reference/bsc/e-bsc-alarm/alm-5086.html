ALM-5086 BBU Power Alarm

Description

The power supply of each board is unavailable

Attribute

Alarm ID Alarm Severity Alarm Type
5086 Critical Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

87:GTMU
3 Board No.

Number of the fault board

0:GTMU
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

0：The 1.0 V power supply to the board Over Voltage.

1：The 1.0 V power supply to the board under Voltag.

2：The sensor of 1.0 V power is faulty

3：The 1.2 V power supply to the board Over Voltage.

4：The 1.2 V power supply to the board under Voltag.

6：The 2.5 V power supply to the board Over Voltage.

7：The 2.5 V power supply to the board under Voltag.

9：The 3.3 V power supply to the board Over Voltage.

10：The 3.3 V power supply to the board under Voltag.

15：The 12.0 V power supply to the board Over Voltage.

16：The 12.0 V power supply to the board under Voltag.

17：The sensor of 12.0 V power is faulty

Alarm Character

Impact on the System

When the alarm is generated, the main control module of the BTS fails to work properly.

System Actions

None.

Possible Causes

  • The hardware of the board is faulty.
  • The BTS ambient temperature is too high. Thus, overtemperature protection is triggered for the BBU.

Procedure

  1. Check the BTS ambient temperature.
    1. Run the LST ALMAF command to view the current active alarms, and then check whether 9746 BBU Temperature Alarm and 9744 BFU Fault Alarm are generated.

      Y => If either of the previous alarms is generated, perform "Clear the alarms".

      N => If neither of the previous alarms is generated, go to Step [2].

    2. Clear the alarms.
    3. Check whether the alarms are cleared.

      Y => If the alarms are cleared, the alarm handling is complete.

      N => If the alarms are not cleared, go to step [2].

  2. Reset the board.
    1. Reset the board.
    2. Check whether the alarms are cleared.

      Y => If the alarms are cleared, the alarm handling is complete.

      N => If the alarms are not cleared, go to step [3].

  3. Replace the board.
    1. Replace the board.
    2. Check whether the fault in the board is rectified.

      Y => If the fault is rectified, the alarm handling is complete.

      N => If the fault persists, contact Huawei Customer Service Center.
