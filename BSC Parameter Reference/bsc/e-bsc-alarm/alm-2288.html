ALM-2288 FCU Staggering Failure Alarm

Description

This alarm is reported when the staggering of a certain channel in the Frequency Combination Unit (FCU) fails, and the staggering of this channel does not go to the expected position. This alarm is reported when some cavity of the FCU staggers successfully but the TRX actually linked to it is inconsistent with the BSC data configuration.

Attribute

Alarm ID Alarm Severity Alarm Type
2288 Major Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

130:FCU（FCU uses the ID of the CDU)
3 Board No.

Number of the fault board

0:FCU 0

1:FCU 1
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

0-3:This alarm with the Sub No. 0 to 3 is reported if some cavity of the FCU fails to stagger, where the Sub No. is equal to the corresponding cavity number.

04-7:This alarm with the Sub No. 4 to 7 is reported when some cavity of the FCU staggers

successfully but the TRX actually linked to it is inconsistent with the BSC data configuration, where the Sub No. is equal to the corresponding cavity number plus 4.

255:The TRXs that connect to the cavity do not stagger successfully.
6 Cell Index

Index of the cell where the board or module is located

0-2047: Cell index of the faulty cell in the data configuration

65535: None
7 Site Type

Type of the site where the faulty board or module is located

3: BTS30

5: BTS312

10: BTS3012A

11: BTS3006A
8 RackNo

Number of the rack where the faulty board is located.
9 BoxNo

Number of the subrack where the faulty board is located.
10 SlotNo

Number of the slot where the faulty board is located.
11 Site Name

Site Name
12 Alarm Attribute

Alarm Character

0:Normal

1:Transient Count

Impact on the System

When the alarm generates, the basic services proceed, but the downlink coverage goes down.

System Actions

None.

Possible Causes

  • A hardware fault occurs to the FCU.
  • The data configuration fails.

Procedure

  1. Check whether the alarm Sub No. is 4–7.
    1. Check whether the alarm Sub No. is 4–7.

      Y=> Go to b.

      N=> Go to step 2.

    2. Add TRXs in the FCU connection configuration. See whether the alarm disappears.

      Y=> The alarm handling is complete.

      N=> Go to step 4.

  2. Check whether the Sub No. is 255
    1. Check whether the Sub No. is 255.

      Y=> Go to b.

      N=> Go to step 3.

    2. Check whether the TRXs software supports the cavity staggering function.

      Y=> Go to step 3.

      N=> Go to step c.

    3. Upgrade the TRX software. See whether the alarm disappears.

      Y=> The alarm handling is complete.

      N=> Go to step 3.

  3. Check whether redundant TRXs are configured in the data configuration.
    1. Check whether redundant TRXs are configured in the data configuration.

      Y=> Go to b.

      N=> Go to step 4.

    2. Delete the redundant TRXs in the data configuration. See whether the alarm disappears.

      Y=> The alarm handling is complete.

      N=> Go to step 4.

  4. Reset the FCU. See whether the alarm disappears.

    Y=> The alarm handling is complete.

    N=> Go to step 5.

  5. Replace the FCU. See whether the alarm disappears.

    Y=> The alarm handling is complete.

    N=> Please contact Huawei Customer Service Center.
