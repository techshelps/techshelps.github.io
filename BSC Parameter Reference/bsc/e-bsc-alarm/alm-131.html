131 BSC Unable to Reset a Circuit

Description

This alarm is generated when the circuit reset fails because the number of timeouts and no responses after the BSC sends the reset circuit message to the MSC exceeds the value of "MAX Retrans Reset Circle Message on A-Interface".

Attribute

Alarm ID Alarm Severity Alarm Type
131 Minor Event

Parameters

ID Name Meaning
1 DPCGROUP

DPCGROUP of the reset circuit that is not configured
2 CicNum

Numbers of Circuit identification code (CIC) of the reset circuits those are not configured
3 CIC

Circuit identification code (CIC) of the reset circuit that is not configured
4 OSP

Original signaling point (OSP) of the subrack where the reset circuit is located
5 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

The circuit status of the BSC and that of the MSC are inconsistent and thus the services cannot be processed.

System Actions

None.

Possible Causes

The E1 transmission of the SS7 signaling link is intermittent or disrupted.

Procedure

  1. Check the parent alarms.
    1. Log in to the LMT. Choose Alarm Maintenance > Browse Alarm to check whether any of the following parent alarms is reported.

      20081 Loss of E1/T1 Signals (LOS)

      20082 Loss of E1/T1 Frames (LOF)

      20087 High BER over the E1/T1 Link

      20092 E1/T1 Excessive Bit Error Rate Per Hour

    2. If any of these alarms is reported, rectify the fault according to the alarm help. Then, check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N => Go to 2.

  2. Check the E1 transmission of the SS7 signaling link.
    1. Check whether the number of the E1 port connected to the BTS is consistent with the number that is configured.

      Y => Go to b.

      N => Reconfigure the data.

    2. Check whether the GEIUA/GOIUA is secure.

      Y => Reinsert the GEIUA/GOIUA.

      N => Go to c.

    3. Check whether the transmission over the A interface is faulty.

      Y => Contact the maintenance engineers.

      N => Go to d.

    4. Check whether the alarm is cleared.

      Y => The alarm handling is complete.

      N => Contact Huawei Customer Service Center.
