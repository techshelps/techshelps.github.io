ALM-3672 E1 Local alarm

Description

This alarm is triggered when the IOMU has not received correct E1 input signals.

Attribute

Alarm ID Alarm Severity Alarm Type
3672 Major Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

129:IOMU
3 Board No.

Number of the fault board

0:IOMU
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

0:local alarm on E1 port 0

1:local alarm on E1 port 1
6 Cell Index

Index of the cell where the board or module is located

0-2047: Cell index of the faulty cell in the data configuration

65535: None
7 Site Type

Type of the site where the faulty board or module is located

9: BTS3002C
8 RackNo

Number of the rack where the faulty board is located.
9 BoxNo

Number of the subrack where the faulty board is located.
10 SlotNo

Number of the slot where the faulty board is located.
11 Site Name

Site Name
12 Alarm Attribute

Alarm Character

0:Normal

1:Transient Count

Impact on the System

  • If the "Sub No." is 0, the BTS3002C cannot normally communicate with the BSC or the site connected with the BTS3002C through E1 port 0 and the BTS3002C services will be interrupted.
  • If the "Sub No." is 1, the BTS2002C cannot normally communicate with the BSC or the site connected with the BTS3002C through E1 port 1 and the BTS3002C services will be interrupted.

System Actions

None.

Possible Causes

  • The transport link is abnormal.
  • The DIP switches of the BTS3002C is wrong.
  • The hardware of BTS3002C is faulty.

Procedure

  1. Check DIP switches of BTS3002C

    Check whether the DIP switches of BTS3002C is correct.

    Y=>Go to Step 2.

    N=>Power down the BTS3002C and set the DIP switches correctly. Power up the BTS3002C and check whether the alarm is cleared.The alarm is cleared, Alarm handling is completed. The alarm is not cleared, Go to Step 2.

  2. Check the transmission at the near-end BTS3002C
    1. Check whether the transmissions at the near-end BTS3002C are normal.After that, check whether the LIU indicator on the lower-level BTS3002C is off.

      Y=> The LIU indicator is off or the alarm is cleared. Go to step b.

      N=> The LIU indicator is on or the alarm still exists.Please contact Huawei Customer Service Center.

    2. Resume the transmission and check whether the alarm is cleared.

      Y=>The alarm handling ends.

      N=>Go to Step 3.

  3. Check the transmissions at the peer end

    Check whether the transmission link at the peer end is normal.

    Y=> The transmission links at the peer end is normal.Please contact Huawei Customer Service Center.

    N=>The possible cause is that the hardware in the BTS3002C is faulty.Repair the link and the alarm handling ends.
