ALM-12816 BTS Clock Synchronization Abnormal Alarm

Description

This alarm is reported when the upper-level clock fails to be locked in the case of the tracing BSC clock or external synchronization clock configured as the BTS clock.

Attribute

Alarm ID Alarm Severity Alarm Type
12816 Major Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

107:BTS3900E
3 Board No.

Number of the fault board

0-3: BTS3900E
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

255:None
6 Cell Index

Index of the cell where the board or module is located

0-2047: Cell index of the faulty cell in the data configuration

65535: None

Type of the site where the faulty board is located.

30:BTS3900E;
8 RackNo

Number of the rack where the faulty board is located.

Number of the subrack where the faulty board is located.
10 SlotNo

Number of the slot where the faulty board is located.

Site Name
12 Alarm Attribute

Alarm Character

1:Transient Count

Impact on the System

When this alarm is generated, the clock of this BTS may differ from the clock of other BTSs. Thus, exceptions may occur in the case of handover, network selection, or cell reselection.

System Actions

No handling is required.

Possible Causes

  • The upper level clock reference is abnormal.
  • The BTS3900E software fails to run normally.

Procedure

  1. Check for related alarms.
    1. Log in to the BSC. Choose "Alarm Maintenance" > "Browse Alarm". Check whether ALM-12846 E1/T1 Local Failure Alarm and ALM-12848 E1/T1 Remote Failure Alarm is reported on the LMT.

      Y => The alarm is reported on the LMT. Go to step b.

      N => ALM-12846 E1/T1 Local Failure Alarm and ALM-12848 E1/T1 Remote Failure Alarm is not reported on the LMT. Go to procedure 2.

    2. Rectify the fault according to handling suggestions in ALM-12846 E1/T1 Local Failure Alarm and ALM-12848 E1/T1 Remote Failure Alarm . Check whether the alarm is cleared.

      Y => The alarm is cleared. End the alarm handling.

      N => The alarm is not cleared. Go to procedure 2.

  2. Check whether the BTS works in IP mode.

    Y => Go to procedure 4.

    N => Go to procedure 3.

  3. Check the clock signals in the transmission link of the BTS.

    Rectify the Transmission Faults from the BTS to the BSC. Check whether the alarm is cleared.

    Y => The alarm is cleared. End the alarm handling.

    N => The alarm is not cleared. Go to procedure 5.

  4. Reset the Site in Level 4. Check whether the alarm is cleared.

    Y => The alarm is cleared. End the alarm handling.

    N => The alarm is not cleared. Go to procedure 5.

  5. Replace the BTS3900E Module. Check whether the alarm is cleared.

    Y => The alarm is cleared. End the alarm handling.

    N => The alarm is not cleared. Contact Huawei Customer Service Center.
