ALM-11270 Transmission LAPD Link Interrupt Alarm

Description

LAPD link is broken including ESL,EML-DPTU and EML-DTMU.

Attribute

Alarm ID Alarm Severity Alarm Type
11270 Major Fault

Parameters

ID Name Meaning
1 Site No.

Number of the site where the faulty board or module is located

Value range: 0-2047
2 Board Type

Number of the type which the faulty board belongs to.

86:DPTU
3 Board No.

Number of the fault board

1: DPTU on COMM1

2: DPTU on COMM2
4 PassNo

Number of the passage where the faulty board is located.
5 SubLink No.

Alarm sub-category number

0:ESL Link Alarm

1:DTMU-EML Link Alarm

2:DPTU-EML Link Alarm
6 Cell Index

Index of the cell where the board or module is located

0-2047: Cell index of the faulty cell in the data configuration

65535: None
7 Site Type

Type of the site where the faulty board is located.

12:BTS3012;

15:BTS3012AE;

16:BTS3012 II;
8 RackNo

Number of the rack where the faulty board is located.
9 BoxNo

Number of the subrack where the faulty board is located.
10 SlotNo

Number of the slot where the faulty board is located.
11 Site Name

Site Name
12 Alarm Attribute

Alarm Character

0:Normal

1:Transient Count

Impact on the System

  • ESL Link Alarm(Sub No.00)
  • All the services carried on the local Hub BTS and its lower level BTSs are interrupted when this alarm is generated.
  • DTMU-EML Link Alarm(Sub No.01)
  • None services carried on the local Hub BTS and its lower level BTSs are interrupted when this alarm is generated.
  • DPTU-EML Link Alarm(Sub No.02)
  • None services carried on the local Hub BTS and its lower level BTSs are interrupted when this alarm is generated.

System Actions

  • When this alarm is generated, the BSC or the DPTU attempts to re-establish links automatically.
  • The DPTU will reset if no alarm recovery is reported in ten minutes.

Possible Causes

  • The connection of the transmission line between the BSC and the DPTU is faulty.
  • The data configuration is incorrect.
  • The DPTU software probe failed.

Procedure

  1. Check for the related alarms.
    1. For IP over Fe mode,

      check whether the LMT also reports the FE link alarm. If this alarm is reported, handle the alarm according to handling suggestions in ALM-11282 ETH link alarm, and then check whether the alarm is cleared.

    2. For HDLC over E1 mode,

      check whether the LMT also reports the E1 local alarm. If this alarm is reported, handle the alarm according to handling suggestions in ALM-11278 E1/T1 Local Alarm, and then check whether the alarm is cleared.

      Y=> The alarm handling ends.

      N => Go to step 2.

      If the previous alarm is not reported, go to step [2].

  2. Check the data configurations.

    Log in to the LMT, check whether the site is configured as HubBTS and whether the configurations of GFGUB, DPTU, and IP are correct.

    Y=> The alarm handling ends.

    N => Go to step 3.

  3. Reset the DPTU.

    On the LMT or local maintenance terminal, reset the DPTU. Check whether the alarm is cleared.

    Y=> The alarm handling ends.

    N => Go to step 4.

  4. Check the transmission equipment.

    Check whether the routing equipment is used between the DPTU and the BSC through consulting the customer.

    Y=> Check whether the routing equipment runs normally. If not, handle the fault. Go to step 5.

    N=> Go to step 5.

  5. Check whether the alarm is cleared.

    Y=> The alarm handling ends.

    N => Please contact Huawei Customer Service Center.
