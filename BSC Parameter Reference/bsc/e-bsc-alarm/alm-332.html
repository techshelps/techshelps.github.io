332 NSVL Faulty

Description

The NSVL refers to the virtual link on the IP NS layer of the Gb interface. This alarm is generated when the GFGUG is faulty, the network between the BSC and the SGSN is disconnected, or the configuration of the BSC and SGSN is faulty.

Attribute

Alarm ID Alarm Severity Alarm Type
332 Major Fault

Parameters

ID Name Meaning
1 NSEI

ID of the NSE where the alarm is generated
2 NSVLI

ID of the NSVL where the alarm is generated
3 Alarm Attribute

0=Normal, Alarms period are longer than the Transient Threshold.

1=Transient Count, The times and the period of the alarms whose periods are shorter than the Transient Threshold are summed up. If the sum result is longer than the Alarm Occurrence Period Threshold or Alarm Occurrence Times Threshold, an alarm so called Transient Count Alarm is triggered. Because Transient Count Alarm is based on accumulative result, the recovery of the alarm may occur at least one Summing Cycle later. The Summing Cycle can be query by MML "LST STATSLDWIN".

Impact on the System

  • If the backup NSVL is not configured, the data services in the relevant cells cannot be performed;
  • If a backup NSVL is configured, the data services in the relevant cells can be performed, but the service capacity decreases.

System Actions

None.

Possible Causes

  • The GFGUG is faulty;
  • The IP communication between the BSC and the SGSN is disconnected;
  • The configuration of the BSC and SGSN is faulty.

Procedure

  1. On the LMT of the BSC and the SGSN, separately run the MML commands LST NSVLLOCAL and LST NSVLREMOTE. Check whether the configuration of the NSE and NSVC on the BSC and that on the SGSN are consistent.

    Y => Go to 2.

    N => Reconfigure the NSE and the NSVC of the BSC to ensure that they are consistent with those of the SGSN.

  2. On the LMT, check whether the GFGUG runs normally.

    N => Remove and reinsert or replace the GFGUG until the board works normally.

  3. Check whether the IP connection between the BSC and the SGSN is normal.

    Y => Go to 4.

    N => Reconnect the network.

  4. Check whether the alarm is cleared.

    Y=> The alarm handling is complete.

    N => Contact Huawei Customer Service Center.
