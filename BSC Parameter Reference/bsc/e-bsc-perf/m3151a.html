Call Drops on TCHF Due to Failures to Return to Normal Call from BTS Loopback (Territorial Resource Request Failure)

Measurement Counter

M3151a:CELL_CALL_DROPS_LOOP_RESTORE_APPLY_TRSL_FAIL_TIMES_BTS_LOOP_TCHF

Description

During the process where the BTS loopback is returned to a normal call, the TC and Abis resources should be requested and the transmission channel between the Ater interface and the Abis interface should be reconnected at the same time. If the request for TC resources, the reconnection, or the request for Abis resources fails, the cause value in the loopback acknowledgement message is Territorial Resource Request Failure. Then, the BSC regards it as a call drop and then releases the call.

This measurement provides the number of call drops on TCHF due to failures to return to normal call from BTS loopback (territorial resource request failure).

Unit

Integer number or integer.

Measurement Point

During the process where the BTS loopback is returned to a normal call, the TC and Abis resources should be requested and the transmission channel between the Ater interface and the Abis interface should be reconnected at the same time. If the request for TC resources, the reconnection, or the request for Abis resources fails, the cause value in the loopback acknowledgement message is Territorial Resource Request Failure. Then, the BSC regards it as a call drop and then releases the call. If the channel that carries the call during the call drop is a TCHF, this counter is measured.

Formula

This is an original counter without involving any formula.
