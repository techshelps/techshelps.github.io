Description

During the incoming internal inter-cell handover procedure, if the handover fails because of various causes before the BSC receives HO CMP message from the MS, the specific counter is measured based on the target channel types, handover failure causes, handover causes. Together with Incoming Internal Inter-Cell Handover Requests and Incoming Internal Inter-Cell Handover Responses, this counter indicates the performance of incoming internal inter-cell handovers. Together with the counters related to the outgoing internal inter-Cell handovers, this counter indicates the performance of internal inter-cell handovers.

The following counters provides the number of failed incoming internal inter-cell handovers based on the target channel types and the frequency bands between which the handover is performed. The channel types include the SDCCH and TCH, and the handover occurs between the frequency bands as listed in Table 1.

The main causes for a failed incoming internal inter-cell handover include:
  • 1) Causes related to resource allocation:
    • No traffic channel is available.
    • No speech version is available.
    • No Abis resource is available.
    • The BSC fails to obtain the terrestrial resources.
  • 2) Causes related to access on the Um interface
    • The MS fails to access the new channel and then reconnects to the old channel.
    • The timer for the BSC to wait for an HO CMP message expires.
  • 3) Abnormal causes
    • The BSC fails to activate the allocated channel.
    • A fault occurs on the A interface.
The following counters are measured based on the specific handover failure causes.
  • 1) The following counters are measured when the incoming internal inter-cell handover fails because of failed allocation of relevant resources.
    • H3220A:CELL_INTRABSC_INCELL_HO_FAIL_CONG_SD provides the number of incoming internal inter-cell handover failures because no channel resource is available when the target channel is an SDCCH. H3229A:CELL_INTRABSC_INCELL_HO_FAIL_CONG_TCH provides the number of failed incoming internal inter-cell handovers because no channel resource is available when the target channel is a TCH.
    • H322H:CELL_INTRABSC_INCELL_HO_FAIL_NO_SPEECH_VER provides the number of incoming internal inter-cell handover failures because no proper speech version is available based on the speech versions carried in the ASS REQ message from the MSC, the speech versions supported by the target cell, and the speech versions permitted by the A interface phase tag. This counter is measured when the BSC fails to obtain the speech version.
    • H322M:CELL_INTRABSC_INCELL_HO_FAIL_NO_IDLE_ABIS provides the number of incoming internal inter-cell handover failures because no Abis CS resource is available in the target cell when the Abis dynamic allocation is enabled.
    • H322B:ELL_INTRABSC_INCELL_HO_FAIL_APPLY_TRSL_FAIL provides the number of incoming internal inter-cell handover failures because the BSC fails establish the terrestrial link.
  • 2) The following counters are measured when the incoming internal inter-cell handover fails because the MS fails to access the new channel on the Um interface and also fails to return to the old channel.

  • H3220C:CELL_INTRABSC_INCELL_HO_FAIL_EXP_SD

    H3227Cb:CELL_INTRABSC_INCELL_HO_FAIL_EXP_TCHF_SIG

    H3228Cb:CELL_INTRABSC_INCELL_HO_FAIL_EXP_TCHH_SIG

    H3227Ca:CELL_INTRABSC_INCELL_HO_FAIL_EXP_TCHF_TRAF

    H3228Ca:CELL_INTRABSC_INCELL_HO_FAIL_EXP_TCHH_TRAF

    After successfully allocates and activates the channel in the target cell, if the BSC releases the call because T3103 expires before the BSC receives the HO CMP message from the MS, the specific counter is measured in the target cell based on the target channel, that is, signaling channel (SDCCH/TCHF/TCCH) or traffic channel (TCHF/TCHH).
  • H322D:CELL_INTRABSC_INCELL_HO_FAIL_RECONN_SUCCESS: After the BSC successfully allocates and activates the channel in the target cell, it sends an HO CMD message to the MS, and starts the T3103 to wait for the HO CMP message. This counter is measured in the target cell if the BSC receives an HO FAIL message on the old channel after the MS returns to the old channel before timer T3103 expires.
  • 3) The following counters are measured when the incoming internal inter-cell handover fails because of equipment faults.
    • H322I:CELL_INTRABSC_INCELL_HO_FAIL_CHACT_FAIL: After the BSC successfully performs channel allocation and speech version confirmation in the target cell, it sends a CH ACT message to the BTS for activating the channel, and starts the corresponding timer to wait for the response. This counter is measured if the BSC receives a CH ACT NACK or no response from the BTS before the timer expires.
    • H322G:CELL_INTRABSC_INCELL_HO_FAIL_A_INTERF_FAIL provides the number of failed outgoing internal inter-cell handovers because the BSC detects an SS7 link failure on the A interface and releases the call.
The following counters indicate the numbers of failed incoming internal inter-cell handovers based on the handover causes.
  • H322a:CELL_INTRABSC_INCELL_HO_FAIL_BSC_LOOP
  • H322:CELL_INTRABSC_INCELL_HO_FAIL_BTS_LOOP

These counters are measured based on the loop types. The loop types include the BSC loop and BTS loop.

Unit

Integer number or integer.
