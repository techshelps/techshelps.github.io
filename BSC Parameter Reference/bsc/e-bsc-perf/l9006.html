Number of Valid Uplink RLC Data Blocks Using CS1-CS4

Measurement Counter

L9006: UP_CS1_RLC_BLK_VALID_TOTAL

L9007: UP_CS2_RLC_BLK_VALID_TOTAL

L9008: UP_CS3_RLC_BLK_VALID_TOTAL

L9009: UP_CS4_RLC_BLK_VALID_TOTAL

Description

This measurement provides the number of valid uplink RLC data blocks using CS1, CS2, CS3, and CS4 coding scheme in a cell in a granularity period. High proportion of valid RLC data blocks indicates few retransmissions of RLC data blocks.

Unit

Integer number or integer.

Measurement Point

In a granularity period, when the BSC receives a GPRS RLC data block using CS1 coding scheme from the MS for the first time and the data block in the V(N) on the BSC side is in NACKED, UNRECEIVED, or RECEIVED state, you can infer that there is a valid data block. In this case, the counter is incremented by one. The measurement point of Number of Valid Uplink RLC Data Blocks Using CS2, CS3, or CS4 is the same as that of Number of Valid Uplink RLC Data Blocks Using CS1.

Formula

This is an original counter without involving any formula.
