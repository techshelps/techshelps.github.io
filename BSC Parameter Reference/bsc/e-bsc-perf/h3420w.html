Description

If the incoming external inter-cell handover fails during the handover procedure, the specific counter is measured in the target cell under control of the target BSC, based on the type of the target channel and the cause of the failure. Together with Incoming External Inter-Cell Handover Requests and Incoming External Inter-Cell Handover Responses, this counter indicates the performance of the incoming external inter-cell handover.

The following counters provide the numbers of failed incoming internal inter-cell handovers when the handover occurs between the frequency bands as listed in Table 1. The target channel for handover can be SDCCH or TCH. They are measured when the incoming internal inter-cell handover is terminated.
  • H3420W: CELL_INTERBSC_INCELL_HO_FAIL_SD_900_900
  • H3420X: CELL_INTERBSC_INCELL_HO_FAIL_SD_1800_1800
  • H3420Y: CELL_INTERBSC_INCELL_HO_FAIL_SD_900_1800
  • H3420Z: CELL_INTERBSC_INCELL_HO_FAIL_SD_1800_900
  • H3429W: CELL_INTERBSC_INCELL_HO_FAIL_TCH_900_900
  • H3429X: CELL_INTERBSC_INCELL_HO_FAIL_TCH_1800_1800
  • H3429Y: CELL_INTERBSC_INCELL_HO_FAIL_TCH_900_1800

The following counters provide the numbers of failed incoming external inter-cell handovers based on the causes. The specific counter is measured when the incoming external inter-cell handover is terminated.
  • (1) The following counter provides the number of failed incoming external inter-cell handovers because the BSC fails to decode an HO REQ message from the MSC:
    • H342I: CELL_INTERBSC_INCELL_HO_FAIL_CONTENTS_INVALID
  • (2) The following counters provide the numbers of failed incoming external inter-cell handover based on the causes related to resource allocations.
    • H3420A:CELL_INTERBSC_INCELL_HO_FAIL_CONG_SD and H3429A:CELL_INTERBSC_INCELL_HO_FAIL_CONG_TCH provides the number of failed incoming external inter-cell handovers because the SDCCH or TCH is unavailable in the target cell. It is measured when the channel allocation fails.
    • H342L:CELL_INTERBSC_INCELL_HO_FAIL_NO_IDLE_ABIS provides the number of failed incoming external inter-cell handovers because no Abis circuit resource is available in the target cell when the Abis dynamic allocation is enabled.
    • H342J:CELL_INTERBSC_INCELL_HO_FAIL_NO_SPEECH_VER provides the number of failed incoming external inter-cell handovers because no proper speech version is available based on the speech versions carried in the HO REQ message from the MSC, the speech versions supported by the target cell, and the speech versions permitted by the A interface phase tag. It is measured when no speech version is available.
    • H342B:CELL_INTERBSC_INCELL_HO_FAIL_APPLY_TRSL_FAIL provides the number of failed incoming external inter-cell handovers because the BSC fails to obtain the circuit resource when establishing the terrestrial connection.
    • H342E:CELL_INTERBSC_INCELL_HO_FAIL_NO_CIC provides the number of failed incoming external inter-cell handovers because the specified CIC on the A interface is unavailable, after the BSC receives the HO REQ from the MSC.
    • H342F:CELL_INTERBSC_INCELL_HO_FAIL_CIC_AVAIL provides the number of failed incoming external inter-cell handovers because the specified CIC on the A interface has already been allocated to another MS, after the BSC receives the HO REQ from the MSC.
  • (3) The following counters provide the numbers of failed incoming external inter-cell handovers because the MS fails to access the new channel on the Um interface.
    • During incoming external inter-cell handover procedure, after successfully allocates and activates the channel in the target cell, the BSC sends an HO REQ ACK message to the MSC and starts the timer to wait for the HO CMP message. If no HO COMP message is received before the timer expires, the BSC releases the call. Thus the incoming external inter-cell handover fails. Then the specific one of the following counters is measured, based on the channel type and service type of the target channel (SDCCH, TCH as signaling channel, or TCH as traffic channel):
      • H3420C: CELL_INTERBSC_INCELL_HO_FAIL_EXP_SD
      • H3429Cb: CELL_INTERBSC_INCELL_HO_FAIL_EXP_TCH_SIG
      • H3429Ca: CELL_INTERBSC_INCELL_HO_FAIL_EXP_TCH_TRAF
    • If the incoming external inter-cell handover fails because the BSC receives from the CLEAR CMD message with the Cause IE being 0x01 (Radio interface failure) and because the timer for HO CMP message expires, the following counter is measured:
      • CH342D: CELL_INTERBSC_INCELL_HO_FAIL_RADIO_INT_FAIL
  • (4) The following counter provides the number of failed incoming external inter-cell handovers because the BSC releases the call after receiving the CLEAR CMD message from the MSC.
    • If the incoming external inter-cell handover fails because the BSC releases the call after receiving the CLEAR CMD message from the MSC, the following counter is measured:
      • H342H: CELL_INTERBSC_INCELL_HO_FAIL_MSC_CLR_CMD
NOTE:
The main causes for a failed incoming external inter-cell handover include:
  • (1) Causes related to message decoding
    • The BSC fails to decode the HO REQ message.
  • (2) Causes related to resource allocation
    • No traffic channel is available.
    • No Abis resource is available.
    • No speech version is available.
    • The BSC fails to obtain the terrestrial resources.
    • No CIC is available.
    • The CIC is already allocated.
  • 3) Causes related to access on the Um interface
    • MCS releases the call because of failed access on the radio interface
    • The timer for the BSC to wait for an HO CMP message expires.
  • (4) Other causes
    • The BSC releases the call after receiving the CLR CMD from the MSC.

Unit

Integer number or integer.

Measurement Point

Figure 1 shows the measurement of counters related to failed incoming external inter-cell handovers.

Figure 1 Measurement of counters related to failed incoming external inter-cell handovers

As shown in Figure 1, if the incoming external inter-cell handover fails, the specific one of the following counters is measured at A, respectively because the message has an invalid content, no suitable channel (SDCCH/TCH) is available in the target cell, or no Abis interface resource is available:
  • H342I: CELL_INTERBSC_INCELL_HO_FAIL_CONTENTS_INVALID
  • H3420A: CELL_INTERBSC_INCELL_HO_FAIL_CONG_SD
  • H3429A: CELL_INTERBSC_INCELL_HO_FAIL_CONG_TCH
  • H342L: CELL_INTERBSC_INCELL_HO_FAIL_NO_IDLE_ABIS
  • H342J: CELL_INTERBSC_INCELL_HO_FAIL_NO_SPEECH_VER
As shown in Figure 1, the specific counter is measured at B if the incoming external inter-cell handover fails because the BSC fails to apply for the terrestrial link or no circuit resources is available:
  • H342B: CELL_INTERBSC_INCELL_HO_FAIL_APPLY_TRSL_FAIL
  • H342E: CELL_INTERBSC_INCELL_HO_FAIL_NO_CIC
  • H342F: CELL_INTERBSC_INCELL_HO_FAIL_CIC_AVAIL
As shown in Figure 1, during the incoming external inter-cell handover procedure, after the BSC sends the HO CMD to the MS, the specific counter is measured at C if the timer for the HO CMP message expires:
  • H3420C: CELL_INTERBSC_INCELL_HO_FAIL_EXP_SD
  • H3429Cb: CELL_INTERBSC_INCELL_HO_FAIL_EXP_TCH_SIG
  • H3429Ca: CELL_INTERBSC_INCELL_HO_FAIL_EXP_TCH_TRAF
As shown in Figure 1, during the incoming external inter-cell handover procedure, if the BSC releases the call after receiving the CLEAR CMD from the MSC, the following counter is measured at D:
  • H342H: CELL_INTERBSC_INCELL_HO_FAIL_MSC_CLR_CMD
    • The counter H342HA:CELL_INTERBSC_INCELL_HO_FAIL_MSC_CLR_CMD_UM_MSG_FAIL is measured when the cause value carried in the CLEAR CMD message is Radio Interface Message Failure.
    • The counter H342HB:CELL_INTERBSC_INCELL_HO_FAIL_MSC_CLR_CMD_UM_FAIL is measured when the cause value carried in the CLEAR CMD message is Radio Interface Failure.
    • The counter H342HC:CELL_INTERBSC_INCELL_HO_FAIL_MSC_CLR_CMD_OM is measured when the cause value carried in the CLEAR CMD message is OM Intervention.
    • The counter H342HD:CELL_INTERBSC_INCELL_HO_FAIL_MSC_CLR_CMD_EQP_FAIL is measured when the cause value carried in the CLEAR CMD message is Equipment Failure.
    • The counter H342HE:CELL_INTERBSC_INCELL_HO_FAIL_MSC_CLR_CMD_PREEMPTION is measured when the cause value carried in the CLEAR CMD message is Preemption.
    • The counter H342HF:CELL_INTERBSC_INCELL_HO_FAIL_MSC_CLR_CMD_INVAILD_MSG is measured when the cause value carried in the CLEAR CMD message is Invalid Message.
    • The counter H342HG:CELL_INTERBSC_INCELL_HO_FAIL_MSC_CLR_CMD_PROTOCOL_ERR is measured when the cause value carried in the CLEAR CMD message is Protocol Error between BSS and MSC.
    • The counter H342HH:CELL_INTERBSC_INCELL_HO_FAIL_MSC_CLR_CMD_OTHER_FAIL is measured when the cause value carried in the CLEAR CMD message is Other Causes.
As shown in Figure 1, during the incoming external inter-cell handover procedure, if the handover fails because the timer for the HO CMP message expires or the BSC receives the CLEAR CMD from the MSC with the cause value 0x01 (Radio interface failure), the following counter is measured at C and D:
  • CH342D: CELL_INTERBSC_INCELL_HO_FAIL_RADIO_INT_FAIL
As shown in Figure 1, if the incoming external inter-cell handover (TDD) fails, the specific one of the following counters is measured at A, B, C, and D irrespective of the cause:
  • H3420W: CELL_INTERBSC_INCELL_HO_FAIL_SD_900_900
  • H3420X: CELL_INTERBSC_INCELL_HO_FAIL_SD_1800_1800
  • H3420Y: CELL_INTERBSC_INCELL_HO_FAIL_SD_900_1800
  • H3420Z: CELL_INTERBSC_INCELL_HO_FAIL_SD_1800_900
  • H3429W: CELL_INTERBSC_INCELL_HO_FAIL_TCH_900_900
  • H3429X: CELL_INTERBSC_INCELL_HO_FAIL_TCH_1800_1800
  • H3429Y: CELL_INTERBSC_INCELL_HO_FAIL_TCH_900_1800
  • H3429Z: CELL_INTERBSC_INCELL_HO_FAIL_TCH_1800_900
