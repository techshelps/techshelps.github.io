Number of Unsuccessful Application Attempts of Abis Timeslot for Connecting Network Failure

Measurement Counter

R9110: CIC_APPLY_FAIL_CONNECT_NET_FAILC

Description

This measurement provides the number of failed requests for Abis timeslots because of network connection failure. The request for Abis timeslots is initiated when the coding scheme of the MS is changed to the high-rate coding scheme. The coding scheme of the MS can be changed only when the quality of the Um interface is good enough and the flexible coding scheme can be used on the network side. If the request for Abis timeslots fails, the coding scheme of the MS cannot be changed to the high-rate coding scheme. In addition, the request for Abis timeslots can also be initiated when the dynamic channel conversion occurs. If the request for Abis timeslots fails, the dynamic channel conversion fails.

Unit

Integer number or integer.

Measurement Point

During the coding scheme adjustment of the PDCH or the dynamic channel conversion, the bandwidth request is initiated. When the coding scheme of the PDCH is adjusted, the request for Abis timeslots fails because of the fault of the BTS port or repeated requests for Abis timeslots, then the number of failed requests for Abis timeslots is accumulated to the value of the counter.

Formula

This is an original counter without involving any formula.
