Number of Uplink EGPRS TBF Establishment Attempts

Measurement Counter

A9201: UP_EGPRS_TBF_ESTB_ATMP

Description

This measurement provides the number of uplink EGPRS TBF establishment attempts in a granularity period. Together with Number of Failed Uplink EGPRS TBF Establishments due to No Channel and Number of Uplink EGPRS TBF Abnormal Releases due to No Channel, this measurement indicates the congestion of the uplink EGPRS TBF establishment due to no channel in a granularity period. Together with Number of Successful Uplink EGPRS TBF Establishments, this measurement indicates the uplink assignment success rate in a granularity period. Together with other measurements, this measurement indicates the PDCH packet assignment success rate in a granularity period.

Unit

Integer number or integer.

Measurement Point

The uplink EGPRS TBF establishment attempts involve the following aspects:

  • Uplink EGPRS TBF establishment using one phase access

    Figure 1 shows the procedure in which the MS initiates one phase access by sending a CHANNEL REQUEST message on the RACH. Each time the BSC receives a CHANNEL REQUEST message indicating one phase EDGE access from the MS (see measurement point A), the counter is incremented by one.

    Figure 1 One phase packet access request received on the RACH

  • Uplink EGPRS TBF establishment using single block access

    The MS generally establishes the uplink EGPRS TBF using one phase access until the BSC sends an IMMEDIATE ASSIGNMENT message instructing the MS to use the single block access. The message contains the packet assignment construction of the single block or that of the multiblock. Figure 2 shows the procedure in which the MS establishes the uplink EGPRS TBF using single block access. When sending the IMMEDIATE ASSIGNMENT message, the BSC reserves the corresponding radio resources on the PS service channel for the MS to respond with a PACKET RESOURCE REQUEST message. Each time the BSC receives a PACKET RESOURCE REQUEST message from the MS indicating the use of single block access for uplink EGPRS TBF establishment (see measurement point C), the counter is incremented by one.

    Figure 2 Single block packet access request received on the RACH

  • Uplink EGPRS TBF establishment in a downlink EGPRS TBF (successful establishment of the uplink EGPRS TBF on the PACCH)

    The MS can initiate an uplink EGPRS TBF establishment request in a downlink EGPRS TBF by attaching the Channel Request information element to the PACKET DOWNLINK ACK/NACK message. The PACKET DOWNLINK ACK/NACK message is triggered by the transmission request from the LLC PDU. Figure 3 shows the procedure in which the MS sends a PACKET DOWNLINK ACK/NACK message that contains the Channel Request information element. Each time the BSC receives an uplink EGPRS TBF establishment request message from the MS (see measurement point A), the counter is incremented by one.

    Figure 3 Uplink TBF establishment in the downlink TBF

Formula

This is an original counter without involving any formula.
