Call Drops on TCH per BSC (Connection Failure) (Service Channel)

Measurement Counter

ZTM348: BSCRPT_TOTAL_CELL_CALL_DROPS_CONN_FAIL_TRAF_TCH

Description

After the MS is in the stable communication state on the TCH carrying traffic, the BTS sends the BSC a CONN FAIL IND message, indicating that call drop occurs on the Um interface because of the abnormal connection at the radio link layer or hardware failure. This measurement is triggered each time a call drops due to the abnormal connection at the radio link layer.

Unit

Integer number or integer.

Measurement Point

The value of this counter is calculated according to the following formula.

Formula

Call Drops on TCH per BSC (Connection Failure) (Service Channel) =

[Call Drops Due to the CONN FAIL Received from TCHF in Stable State (Radio Link Failure) (Traffic Channel) per BSC] +

[Call Drops Due to the CONN FAIL Received from TCHF in Stable State (HO Access Failure) (Traffic Channel) per BSC] +

[Call Drops Due to the CONN FAIL Received from TCHF in Stable State (OM Intervention) (Traffic Channel) per BSC] +

[Call Drops Due to the CONN FAIL Received from TCHF in Stable State (Radio Resource Not Available) (Traffic Channel) per BSC] +

[Call Drops Due to the CONN FAIL Received from TCHF in Stable State (Other Causes) (Traffic Channel) per BSC] +

[Call Drops Due to the CONN FAIL Received from TCHH in Stable State (Radio Link Failure) (Traffic Channel) per BSC] +

[Call Drops Due to the CONN FAIL Received from TCHH in Stable State (HO Access Failure) (Traffic Channel) per BSC] +

[Call Drops Due to the CONN FAIL Received from TCHH in Stable State (OM Intervention) (Traffic Channel) per BSC] +

[Call Drops Due to the CONN FAIL Received from TCHH in Stable State (Radio Resource Not Available) (Traffic Channel) per BSC] +

[Call Drops Due to the CONN FAIL Received from TCHH in Stable State (Other Causes) (Traffic Channel) per BSC]
