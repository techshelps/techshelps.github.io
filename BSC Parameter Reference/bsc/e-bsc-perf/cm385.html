Call Drops due to Failures to Return to Normal Call from Loopback

Measurement Counter

CM385:CELL_LOOP_RESTORE_FAIL_DROPS

Description

After a call is in the BSC/BTS local switch state, it incurs a handover. The local switch, however, cannot be continued because the target cell of the handover may not support local switch, the outgoing BSC handover fails, the TRX that carries the target channel of the handover may not support local switch, or the specified handover fails. The BSC attempts to restore the call to a normal one. The restoration may fail due to various reasons. If the restoration fails, the MS incurs call drops. This measurement provides the number of call drops in different local switching states (intra-BSC or intra-BTS).

Unit

Integer number or integer.

Measurement Point

At the end of the granularity period, this counter is calculated by using the following formula.

Formula

Call Drops due to Failures to Return to Normal Call from Loopback = Call Drops due to Failures to Return to Normal Call from BSC Loopback (Terrestrial Resource Request Failure) + Call Drops due to Failures to Return to Normal Call from BTS Loopback (Terrestrial Resource Request Failure) + Call Drops due to Failures to Return to Normal Call from BSC Loopback (Others) + Call Drops due to Failures to Return to Normal Call from BTS Loopback (Others)
