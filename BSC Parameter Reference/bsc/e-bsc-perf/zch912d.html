Successful Reconnections to Old Channels upon Internal Outgoing Cell Handover Failure per BSC

Measurement Counter

ZCH912D:BSCRPT_INTRABSC_OUTCELL_HO_FAIL_RECONN_SUCC

Description

This measurement provides the number of successful reconnections to old channels upon internal outgoing cell handover failure in all the cells controlled by a BSC.

In the outgoing internal inter-cell handover procedure, the BSC sends an HO CMD message to the MS through the originating cell and initiates a timer to wait for an HO CMP message. If the MS reconnects to the old channel and sends an HO FAIL message on the old channel before the timer expires, the counter is measured based on the cause. The measurement object is the target cell.

The handover can be performed between GSM900/850 and GSM900/850, between GSM1800/1900 and GSM1800/1900, between GSM900/850 and GSM1800/1900, or between GSM1800/1900 and GSM900/850.

This counter contains the following cause values:

Abnormal release, unspecified

Abnormal release, channel unacceptable

Abnormal release, timer expired

Abnormal release, no activity on the radio path

Preemptive release

Handover impossible, timing advance out of range

Channel mode unavailable

Frequency not implemented

Call already cleared

Semantically incorrect message

Invalid mandatory information

Message type non-existent or not implemented

Message type not compatible with protocol state

Conditional IE error

No cell allocation available

Protocol error unspecified

Other

Unit

Integer number or integer.
