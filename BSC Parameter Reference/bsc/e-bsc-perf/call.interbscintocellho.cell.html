Incoming External Inter-Cell Handover Measurement per Cell

Description

The external inter-cell handover per cell indicates that after receiving the measurement report from the MS, the BSC uses the handover algorithm to decide whether a handover is required or not according to the factors such as the cell load, the signal interference, and the information in the measurement report, including the signal quality, the signal strength, and the receive level. If the originating (serving cell) and the target cell are under the control of different BSCs, an external inter-cell handover is initiated.

The incoming external inter-cell handover per cell is part of the external inter-cell handover procedure, which takes the target cell as the measurement object. The following describes the incoming external inter-cell handover procedure. (Note: The BSC in which the originating cell is located is referred to as BSC1, and the BSC in which the target cell is located is referred to as BSC2 hereinafter.)

BSC1 initiates the incoming external inter-cell handover after receiving the measurement report sent by the MS. The handover algorithm of BSC1 analyzes the received measurement report, load of the current cell, and its neighbor cell, and then notifies BSC1 of the incoming external inter-cell handover.

After the handover algorithm sends out the HDM, BSC1 sends an HO RQD message to notify the MSC of the target cell the hangover is aimed at. The MSC searches for the corresponding BSC2 according to the information about the target cell in the data configuration, and then sends an HO REQ message to BSC2. After receiving the HO REQ message from the MSC, BSC2 applies to the target cell under its control for the required traffic channel.

After the channel is successfully allocated, BSC2 sends a CHAN ACTIV message to the target cell, and requests resources for speech services after receiving the CHAN ACTIV ACK message from the target cell. When the resources for speech services are allocated, the target cell is ready for handover. Then BSC2 sends an HO REQ ACK message to the MSC to notify the MSC that the allocated channel in the target cell is ready for MS to access.

After receiving the message, the MSC sends an HO CMD message to BSC1. BSC1 sends an HO CMD message of the Um interface to instruct the MS to access the target cell in BSC2. The MS sends an HO ACC message to the target cell to attempt to access the target cell. BTS2 sends an HO DETECT message to BSC2 and at the same time sends a PHY INFO message to the MS. The MS sends the FIRST SABM frame to access the network. After receiving the FIRST SABM frame, BTS2 sends an EST IND message to BSC2 and sends the UA frame to the MS. After the MS accesses the target cell, it sends an HO CMP message to BSC2, and BSC2 also sends an HO CMP message to the MSC, indicating that the handover is successful. The MSC sends a CLEAR CMD message to BSC1 to instruct BSC1 to release terrestrial resources in the originating cell. After the resources are released, BSC1 sends a CLR CMP message to the MSC. The measurement of the incoming external inter-cell handover provides the number of incoming external inter-cell handovers based on the target cell in BSC2.
