Number of MSs Supporting TDD

Measurement Counter

CH355A:CELL_SUPPORT_TDD_MS_NUM

Description

After the MS initiates a random access and seizes a signaling channel successfully, the MSC sends a CLASSMARK REQUEST message to the MS, requesting the MS to report the classmark. The BSC sends a CLASSMARK ENQUIRY message to the MS, and the MS returns a CLASSMARK CHANGE message to the BSC. Then, the BSC sends a CLASSMARK UPDATE message to the MSC. If Early Classmark Sending Control (ECSC) is enabled, the BSC uses the system message to instruct the MS to send the classmark. In this case, the MS sends a CLASSMARK CHANGE message to the BSC. Then, upon receiving the CLASSMARK CHANGE message from the MS, the BSC sends a CLASSMARK UPDATE message to the MSC. If the MSC has obtained the MS classmark and needs to notify the BSC, it sends a CLASSMARK UPDATE message to the BSC.

After obtaining the MS classmark, the BSC decodes the classmark to check whether the MS supports another network such as the UMTS or TD-SCDMA.

This measurement provides the number of MSs supporting the TD-SCDMA network in the serving cell.

Unit

Integer number or integer.

Measurement Point

When the UMTS TDD Radio Access Technology Capability field in classmark 3 is set to 0x01, this measurement is triggered if any of the following situations occurs: the MS sends a CLASSMARK CHANGE message; the MSC sends a CLASSMARK UPDATE message; the incoming BSC handover is complete.

Formula

This is an original counter without involving any formula.
