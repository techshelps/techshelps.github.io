Total Number of Valid RLC Downlink Data Blocks Using CS1-CS4

Measurement Counter

L9106: DOWN_CS1_RLC_BLK_VALID_TOTAL

L9107: DOWN_CS2_RLC_BLK_VALID_TOTAL

L9108: DOWN_CS3_RLC_BLK_VALID_TOTAL

L9109: DOWN_CS4_RLC_BLK_VALID_TOTAL

Description

This measurement provides the total number of valid downlink RLC data blocks using CS1, CS2, CS3, and CS4 coding schemes in a cell in a granularity period. High proportion of valid downlink RLC data blocks indicates few retransmissions of downlink RLC data blocks.

Unit

Integer number or integer.

Measurement Point

Figure 1 shows the procedure in which the BSC sends a downlink GPRS RLC data block using CS1 coding scheme and receives a PACKET DOWNLINK ACK/NACK message. After receiving a PACKET DOWNLINK ACK/NACK message from the MS (see measurement point A), the BSC determines the number of downlink GPRS RLC data blocks using CS1 coding scheme received by the MS and updates the V(A) based on the bitmap in the PACKET DOWNLINK ACK/NACK message. Every time the V(A) is incremented by one, the counter is incremented by one.

Figure 1 Transmission of downlink GPRS RLC data blocks using CS1 coding scheme

Formula

This is an original counter without involving any formula.
