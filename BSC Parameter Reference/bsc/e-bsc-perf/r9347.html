Number of Dynamic PDCH Requests Without Application Attempts Because No Abis Timeslot is Available for Use

Measurement Counter

R9347: NEED_DYN_PDCH_FAIL_WITHOUT_USEABLE_ABIS

Description

This measurement provides the number of failed dynamic PDCH requests in a cell in a granularity period because no Abis timeslot is available for use.

If multiple requests fail or the rate of successful dynamic PDCH requests is lower than 80%, you can infer that the traffic of the BSC is high or the channel state of the BSC is abnormal. One of the following methods can be implemented:
  • Perform the capacity expansion to increase Abis timeslots.
  • Set the uplink and downlink PDCH multiplexing thresholds to higher values.
  • Set Maximum Ratio Threshold of PDCHs to a lower value.

Unit

Integer number or integer.

Measurement Point

In dynamic channel request process, if no Abis timeslot is available for use, then the counter is incremented by one.

Formula

This is an original counter without involving any formula.
