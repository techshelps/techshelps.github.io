Number of Uplink GPRS TBF Abnormal Releases due to N3103 Overflow (MS No Response)

Measurement Counter

A9007:UP_GPRS_TBF_ABNORM_REL_N3103_OVERFLOW

Description

This measurement provides the number of uplink GPRS TBF abnormal releases due to N3103 overflow in a granularity period. A high measurement value is yielded because the radio quality of the cell is bad.

Unit

Integer number or integer.

Measurement Point

During the countdown procedure of uplink GPRS TBF, the BSC sends a PACKET UPLINK ACK/NACK (FAI = 1) message upon receiving the uplink RLC data blocks (CV = 0). If the BSC receives the PACKET CONTROL ACKOWNLEDGEMENT message in the uplink radio block assigned to the MS within a certain period predefined by the BSC timer, then the uplink GPRS TBF is normally released and the N3103 is reset. Otherwise, the N3103 is incremented by one, and the BSC shall re-initiate the uplink GPRS TBF release procedure by resending the PACKET UPLINK ACK/NACK (FAI = 1) message. If N3103 overflows, the BSC releases the uplink GPRS TBF. Figure 1 shows the procedure of abnormal release of the TBF due to no response from the MS during the uplink release process. Each time N3103 overflows (see measurement point A), the counter is incremented by one.
