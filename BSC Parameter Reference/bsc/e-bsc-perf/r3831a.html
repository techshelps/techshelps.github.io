Number of Successful BTS Loopback Attempts after the Handover during the BTS Loopback

Measurement Counter

R3831a:CELL_LOCALSWITCH_BTSLOOP_HO_BTSLOOP_SUCCESS

Description

A handover is initiated in the stable BTS loopback state (BTS loopback is successful and the calling and called MSs are in loopback state). After the handover, the BTS loopback procedure is restarted. The BSC determines the loopback start conditions, performs BSC local switch call detection, sends a BTS loopback start request message, adjusts the speech versions of the calling and called MSs, and performs loopback on the BTS side. If all these actions are successful, the BTS loopback is successful.

This measurement provides the number of successful BTS loopback attempts after the handover during the BTS loopback.

Unit

Integer number or integer.

Measurement Point

A handover is performed in the stable BTS loopback state. After the handover, the BSC selects the BSS local switch policy, determines the local switch start conditions, and performs local switch call detection. If all the local switch conditions are met, the BTS local loopback procedure is started. Within the BSC, the call processing module initiating loopback start at one end sends a loopback request message to the call processing module at the peer end. The call processing module at the peer end sends a BTS loopback request message to the BTS. If the BTS loopback is successful, the BTS responds to the call processing module with a BTS loopback acknowledgement message (loopback success). Then, the call processing module at the peer end returns the call processing module at the TX end a loopback acknowledgement message (loopback success). In this case, this counter is measured in the cell where the loopback start is initiated.

Formula

This is an original counter without involving any formula.
