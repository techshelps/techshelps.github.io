Peak PCH Paging Queue Usage

Measurement Counter

A343: CELL_QUE_OCCUPY_PK_VALUE

Description

When the new paging messages are successfully added to the PCH queue after scheduling, the number of current PCH messages in the PCH queue is calculated, and then divided by the valid PCH queue length (calculated based on the current queuing ratio). Thus, the average paging queue usage is obtained. Then, record the peak paging queue usage within the granularity period of one minute. If the average paging queue usage is 80%, then the field value is 80. The BTS measures the value of the CELL_QUE_OCCUPY_PK_VALUE and reports it in the BTS_STATIS_REPORT message to the BSC.

Unit

%

Measurement Point

As shown in Figure 1, this counter is counted at C1 when the information element CELL_QUE_OCCUPY_PK_VALUE decoded from the BTS_STATIS_REPORT message by the BSC is greater than the history value.
