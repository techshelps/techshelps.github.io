Call Drops on TCHH Due to Failures to Return to Normal Call from BSC Loopback (Others)

Measurement Counter

M3252:CELL_CALL_DROPS_LOOP_RESTORE_OTHER_FAIL_TIMES_BSC_LOOP_TCHH

Description

After the call is put through or the incoming BSC handover for the speech service is complete, the BSC selects the BSS local switch policy, determines the local switch start conditions, and performs local switch call detection in the local loopback procedure. If all the local switch conditions are met, the BTS loopback procedure is started. The loopback, however, fails because of other reasons (rather than Territorial Resource Request Failure). Then, the BSC regards it as a call drop and releases the call directly.

This measurement provides the number of call drops on TCHH due to failures to return to normal call from BSC loopback (others).

Unit

Integer number or integer.

Measurement Point

After the BSC receives the CONNECT ACK message from the MSC in the MS terminated call procedure, as shown in Figure 1(A1), or after the BSC receives the HANDOVER CMP message from the MS in the incoming BSC handover procedure for the speech service, as shown in Figure 2 (B1), the BSC selects the BSS local switch policy, determines the local switch start conditions, and performs local switch call detection. If all the local switch conditions are met, the BSC local loopback start procedure is started. Within the BSC, the call processing module initiating loopback start at one end sends a loopback request message to the call processing module at the peer end. The call processing module at the peer end, however, responds with a loopback reject message because of other reasons (rather than Territorial Resource Request Failure). The loopback start fails. Then, the BSC regards it as a call drop and releases the call directly. If the channel that carries the call during the call drop is a TCHH, this counter is measured.
