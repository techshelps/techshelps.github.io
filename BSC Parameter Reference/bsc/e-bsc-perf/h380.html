Incoming Inter-Cell Handover Requests

Measurement Counter

H380:CELLCELL_INCELL_HO_REQ

Description

This measurement provides the number of incoming inter-cell handover requests in the target cell. Along with Failed Cell Incoming Handovers, this counter indicates the performance of incoming inter-cell handovers. Along with the counters related to the incoming inter-cell handover, this counter indicates the performance of inter-cell handovers.

Unit

Integer number or integer.

Measurement Point

If the originating cell and the target cell are GSM cells, this counter is measured in the following situations:

1. In the internal intra-cell handover procedure, the directed retry procedure, or the internal intra-cell handover (BSC loop) procedure, this counter is incremented by one when the originating sends an Innercell Handover Request message over the SDCCH or TCH to the target cell (also the originating cell) after handover algorithm decision. This counter is incremented by one when the target cell (serving cell) receives an Innercell Handover Request message over the SDCCH or TCH.

2. In the internal inter-cell handover procedure, the directed retry procedure, or the internal inter-cell handover (BSC loop) procedure, the originating cell sends an Intercell Handover Request message to the target cell after handover decision at C1, as shown in the following figure. This counter is incremented by one when the target cell (serving cell) receives the Intercell Handover Request message over the SDCCH or TCH.



3. In the inter-BSC handover procedure, after the target BSC receives a HANDOVER REQUEST message from the MSC, the BSC checks whether the incoming handover meets the conditions. If the handover is required, the BSC requests a channel, generates a HANDOVER COMMAND message and saves it in the buffer. Then, the BSC sends a CHANNEL ACTIVATION message to the BTS. After receiving a CHANNEL ACTIVATION ACKNOWLEDGE message from the BTS, the BSC sends a HANDOVER REQUEST ACKNOWLEDGE message encapsulated with the HANDOVER COMMAND message to the MSC and this counter is incremented by one. After that, the MSC forwards the HANDOVER COMMAND message to the originating BSC, and the originating BSC sends the HANDOVER COMMAND message to the MS.



Formula

This is an original counter without involving any formula.
