PS Measurement(PSALL)

Overview

PS Measurement provides the performance of the PS domain. The measurement object is BSC. The measurement result is used to analyze the traffic load of the BSC. According to the measurement results of several days or weeks, the traffic load model can be obtained and offers guidance on the engineering design and network planning.

For example, the counter Average Number of PDCHs Occupied per BSC indicates the PS status such as the busy status and idle status. The configuration type of channels in the cell can be adjusted according to the PS status. More fixed PDCHs can be configured for the cell that the PS is busy.

If the number of seized PDCHs is few, it indicates that the actual traffic is not heavy. Many fixed PDCHs are allocated and thus the channel utilization is low. Under this condition, according to the cell-level counters, the cell with low PDCH utilization to reduce the number of fixed PDCHs. If the number of seized PDCHs is great, it indicates that the actual traffic volume is high. The network expansion is recommended to be performed.

If the PDCH allocation fails because of the congestion of the BSC, the capacity expansion is recommended to be performed.

If the number of online EGPRS subscribers is greater than the number of online GPRS subscribers, it indicates that current subscribers mainly perform the EGPRS service, EGPRS channels can be added. Otherwise, if the number of online GPRS subscribers is greater than the number of online EGPRS subscribers, the GPRS channels can be added.
