Trouble Shooting & Debugging

1. In case you are not able to connect to the OmniFlow Server, please ensure that the JTS is running on the machine through which you wish to connect.

2. If JTS is running and still you are not able to connect, please ensure that Database server is running.

3. If you are not able to run the call. Check the syntax of the call and then check the API name. Being case-sensitive, the API name has to be specified exactly in the same way.

4. If you are using the filter criterion specified for any call, you need to pass correct filter parameters. You cannot leave them blank.

for example:

In WMFetchProcessDefinitions call you can pass the filter criterion to fetch the list of selected process definitions.

<? Xml Version="1.0"? >

<WMFetchProcessDefinitions_Input>

<Option> WMFetchProcessDefinitions </Option>

<EngineName></EngineName>

<SessionID></SessionID >

<CountFlag></CountFlag>

<Filter>

<Type></Type>

<AttributeName></AttributeName>

<Comparison></Comparison>

<FilterString></FilterString>

</Filter>

</WMFetchProcessDefinitions_Input>

Type:

Type of the filter criteria specified. Filter type can be SQL string or an attribute. If you are filtering on attributes, then filter string contains the value of the attribute. 0-255 are reserved , if it is 256 than its SQL type filter. Then you Filter string will contains an SQL query.

Attribute Name:

It is the name of attribute for filtering.

Comparison:

It is operator for the filter.

1 for > ; 2 for >= ;3 for =; 4 <> ; 5 for <; 6 for <=; 7 for like, 8 for not like.

Filter String:

It is string specified for the filtering.

These parameters are to be specified properly.

 

5. If you are fetching an information batch-wise, please ensure that the Batch Info has to be properly mentioned.

For example:

In WMFetchProcessDefinitions call you can pass the filter criterion to fetch the list of selected process definitions.

<? Xml Version="1.0"? >

< WMFetchWorkItems _Input>

<Option>WMFetchWorkItems</Option>

<EngineName></EngineName>

<SessionID></SessionID >

<BatchInfo>

<NoOfRecordsToFetch></NoOfRecordsToFetch>

<LastWorkItem ></LastWorkItem>

<LastProcessInstance></LastProcessInstance>

</BatchInfo>

</ WMFetchWorkItems _Input>

NoOfRecordsToFetch:

It is the batch size specified for the workitems to be fetched.

LastWorkItem:

It is the id of the last workitem which is fetched. In case you wish to fetch the information from the 1st workitem, then you do not require this tag.

LastProcessInstance:

It is the last process instance id which is fetched through this call. In case you wish to fetch the information from the 1st workitem, then you do not require this tag.

6. If routing can be set on one process and not on the other, please ensure on which Process Server the failed process is configured. Then check your connection settings for that process.
