Sample Code for External Interface Implementation

System integrators can write customize programs and can hook them for specialized processing. If any window or external interface is associated or defined in the process and that external interfaces wishes to interact with the Workflow engine for its definition and status.

Then we need to provide the JAVA implementation of the external interface which provides methods for the following operations:

  • GetExternalData : Fetching the definition and the status of the external interface. This method is used by WMGetExternalData, to return the definition of all the registered external interfaces for the given process and activity.

View the Function Prototype

 

  • SetExternalData: Modifying the data and the status of the external interface. This method is used by WMSetExternalData, to modify the data or status of all the registered external interfaces for the given workitem.

View the Function Prototype

 

  • GetList: List of external elements associated with the given process or its versions. This method is used by WMGetExternalList, to return the list of all the elements of any  registered external interfaces for the given process and activity.

View the Function Prototype

 

  • SearchExternalData: Searches workitems on the basics of external interface element's status. This method is used by WFSearchWorkitemList, to return the list of all workitems with the specified interface element associated and its status.

View the Function Prototype

 

 

Your implementation needs to extend the WFExternalInterface abstract class.

Deployment

Step1: The implementation class should be in class path of the OmniFlow Server.

The implementation classes should be included in the installation of the OmniFlow Server

 

Step2: Adding the implementation to the OmniFlow Server Transaction pool.

You need to modify the transaction pool of the OmniFlow Server by adding the following entries.

1. Go to the OmniDocs Server Installation directory, where you can find the- Object pool named file. This file has following XML tag structure.

 

<Category List>

<Category>

<Transaction List>

<Transaction>.

.<Name>.</Name>

<Path> </Path>

</Transaction>

.......

<Transaction>....

</Transaction>

 

</Transaction List>

</Category>

<Category>

<Transaction List>

..............

</Transaction List>

</Category>

</Category List>

 

2. Go to the Category --<WRKFLW> (where the workflow entries are provided) and add the following tag:

 

<Transaction>

<Name>[Interface Name]</Name>

<Path>mssql#[Interface Implementtaion Class for SQL Databases]#oracle#[Interface Implementtaion Class for Oracle Databases]#</Path>

</Transaction>

 

 

Step3: Restart the JTS server.
