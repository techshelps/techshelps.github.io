Connection Functions

For connection with the OmniFlow Server, you need to implement the following WAPI calls.

WMConnect

Informs the WFM Engine that other commands will be originating from this source. It allows the connection of external application with the WFM Engine for this series of interactions.

WMDisconnect

Allows ending the logged in user session on the WFM Engine.

 

After the successful connection, the login user session id has to be maintained for his series of interaction with the WFM Engine. For setting the session, you need to implement the following call.

WMSetUserSession

Sets the session id for the logged in user.
