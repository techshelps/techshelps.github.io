Installations and Configurations

Following Installations are required for getting started, if you have the existing OmniDocs database.

Installating OmniDocs Server

1. Click on the OmniDocs Setup icon. The installation screen is displayed.

 

 

 

2. Click Next to continue.

The License Agreement screen is displayed.

 

 

3. Click Yes to proceed with the installation.

 

 

4. Specify the User Name, Company Name and Serial Number for installing the OmniDocs Server.

5. Click Next to proceed.

 

6. Select the Destination Folder for installation. Click Browse for selection.

7. After selection click Next to proceed.

 

 

8. Select the Mail Server Name /IP Address and EMail Address option for mail configurations.

9. Specify the IP Address of the mail server in Mail Server Address.

10. Specify the Email Address.

11. Click Next to proceed the installation.

 

 

12. Select the Program Icon Name. You can also specify the Program Icon name according to your preference.

13. Click Next to proceed the installation,

 

The installation process starts.

 

14. There is an option to create a cabinet at the time of installation or you can create the cabinet later on through OmniDocs Service Administration.

Select Yes to create the cabinet..

 

 

15. Click Next to proceed with the selected option.

 

16. Specify the name for the new cabinet in Cabinet Name.

17. The list of SQL Server running on the network is available. Select the Server on which the cabinet has to be connected.

18. Specify Login ID and Password for the server connection.

19. Click Next for creating cabinet.

 

The cabinet is created with the specified name.

20. Click Next to proceed.

 

21. It is recommended to select the option for Restarting the computer on which the installation is done and click Finish.

 

The OmniDocs Server installation is completed.

Cabinet Creation

For creating a cabinet:

1. Click Start Menu, point to Programs, then point to Newgen OmniDocs Professional Server and then click OmniDocs Service Administration.

Login dialog box is invoked.

2. Select the user name from the User list and specify the Password for login.

3. Click OK to login.

The OmniDocs Service Administration screen is invoked.

 

4. Click on Servers node from the tree view. It will display the list of JTS running on the network. The list of all the Cabinets registered on the Server are displayed.

 

5. Click Create for cabinet creation.

The Create Cabinet dialog box is invoked.

 

6. Select the Cabinet Type. It can be a Document database, an Image server database or Both.

7. Select the Database Type. It can be MSSQL, Oracle, PostGres or MSDE. Depending upon the database selection the information input varies for the user.

 

If MSSQL is selected then the following procedure is valid.

Note:

  • Enable FTS option is enabled, only if the FTS server is installed.

  • Enable FTS option is available for MSSQL and Oracle cabinets only.

  • Select the Enable FTS option for enabling full text search.

 

8. Specify the Device Size and Log File Size in MB. This section changes according to the selected Database Type.

Note:

  • In case you have selected Oracle Cabinet:

Oracle Information has to be provided.

8. Specify the Service Name where the Oracle Server is running and the Port on which it is listening the client's requests.

 

  • In case you have selected PostGres Cabinet:

PostGres Information has to be provided.

8. Specify Port on which the server is listening to client's requests.

 

  • In case you have selected MSDE Cabinet:

MSDE Information has to be provided.

8. Specify the Device Size and Log Size in MB.

 

 

Cabinet Information

9. Specify the name for cabinet in Cabinet Name.

10. Specify the IP address of the server where the cabinet has to be created, in Server Name.

11.Specify the User Name and Password for connecting to the specified server.

12. The Database path is automatically considered as the cabinet name. You can change the name of the DAT file, in the Database Path.

Note

In case of Oracle Cabinet, ORA file is created.

 

13. Specify the CD Key which was earlier used for the OmniDocs Server setup.

14. Select the Security Level for the cabinet. It can be on the Cabinet, Object Level or No Security.

  • No Security implies no validation of rights on any object, for any user.

  • Cabinet Level implies that the rights are applicable at the cabinet level. All the rights on an object for a user would be inherited from the cabinet.

  • Object Level implies object level security, which means that all the rights and privileges are assigned to all the objects of OmniDocs.

15. Click OK to create the cabinet.

 

The cabinet is created but it will not be visible in the Cabinet list till is associated.

Creating and Registering Process

To create a process:

Click Start Menu, point to Programs, then point to OmniFlow and then click OmniFlow Process Modeler.

The OmniFlow Process Modeler screen is invoked.

1. Click File menu and then click New,

A blank canvas is invoked for creating a process.

a) The Workstep Tool Box appears on the left-hand side of the screen.

b) Select the required Workstep from the toolbox and paste it on the canvas area.

To link two worksteps,

d) Place the mouse pointer on the edge of the required Workstep and drag it to the desired Workstep. An arrow is displayed implying that a link is established between the selected worksteps.

A user can now place the required worksteps and link them to form a continuous process.

The worksteps can be linked with one another and later set the configuration settings.   

 

2. After creating the process, it has to be saved. To saving the current process as a new process.

3. Click the File Menu and then click Save As.

The Save As dialog box is invoked.

4. Specify the new name for the process in File Name (Mdb file) and click Save to save the process.

 

5. After the process is made, you need to register the process. Process is registered in a database.

 

To register a process:

6. Open a local mdb file of the process.

7. Click Admin menu and then click Register Process.

The Login dialog box is invoked.

Note

The Register Process option is enabled only if user is connected with a remote database. Click Connect in File menu to connect with an remote database.

 

8.Specify Login Id and Password to register the process on the database to which you are connected.

9.The database name is displayed in Database on where the process is registered.

10. Specify process name that has to be registered in Process Name.

11. Click Connect to connect to database else click Cancel.

The Registration Status dialog box is invoked displaying the registration status of the process.

Note

If a process with the same name is already registered in the database, a message box is invoked prompting for the creation of a new version of the process. Click Yes to proceed else click Cancel.

 

12. If a document type is defined in the process, a message box is invoked prompting for the creation of process folders in the DMS.

13. Click Yes to create folders. The Login dialog box is invoked.

a) The cabinet drop down list consists of the names of all the cabinets registered on the local machine of OmniDocs.

b) Select the required cabinet.

c) Specify user id and password for the selected cabinet.

d) To create the folders click OK.

 

14. If it is the first version of the process in the current database i.e. if no process exists in the database with the same name then a message box is invoked prompting for the creation of a default queue definition for the process. It will ask for creating the default Queue definition for the process. Click Yes to create the Default queues. You can create / modify queues from the Monitoring Desktop also.

15. If it is a second or higher version of the process then Queue Association dialog box is displayed containing the name of all queues associated with the previous version of the same process in the Defined Queue drop down list, and all the activity in the Activity Name drop down list.

a) Select any queue from the Defined Queue and then select any activity from the Activity Name. It populates the name of all the associated streams of the selected activity with the selected queue in the Associated Streams and all other streams of the same activity in the other List box ( LHS list box).

b) Associate / Disassociate the streams with the queue and click Apply to save the changes.

c) Click OK to save changes for the current selected queue and activities and exit.

16. To save the registration details, click Save As. The Save dialog box is invoked. You can save the information as RTF or TXT file.

17. Click Close to close the Registration Status dialog box.

The process with the specified name is registered.Registering Process

Registering OmniFlow Server

To register OmniFlow Server:

1. Click Start, point to Programs, then go to OmniFlow 3.0 and then click Utilities and then click Configuration server.

2. The Configuration Server is invoked.

 

3. Click Connect Menu and then click Workflow Server option.

The OmniFlow Server Administration screen is invoked.

4. Click Register button, the Register New Server dialog box is invoked.

 

5. Specify the IP address on which the Server is running.

6. Specify the Admin Port on which the WFS server is running.

By default the admin port is set as 9999.

7. Click OK to connect to that particular Server, else click Cancel.

After the Registration of the OmniFlow Server, it appears in the Server list and is in Disconnected State.

8. You can unregister the server as well. Select the server, which has to be unregistered and then click Unregister.

The Server is unregistered and is no longer visible in the Server list.

9. After the successful registration, you need to connect to that particular server. Select the server and click Connect.

10. After the connection the server state changes to Start. And the Connect button changes to Disconnect.

 

Registering Processing Server

The Process Server is responsible for routing the workitems to the next workstep in the Process. It also executes the Entry and Exit rules for the processing workitems. Thus whenever the user creates a workitem or perform any activity on the workitems in his queue, they have to be routed to the next workstep in the Process through Processing Server.

CS allows the remote access to the Processing Server. Through CS you can start /stop the Processing Server on the Machine and also displays the count of the workitems processed.

To register the Processing Server on CS:

1. Click Register from the Configuration Server dialog box.

The Registration dialog box is invoked.

 

2. Select the Application Type as ‘Process Server’.

3. Specify the Application name.

4. Specify Listener IP Address and Listener Port. By default, Listener Port is 2501.

5. Specify the Application Path, where the Process Server is installed in the system.

6. Click Next to proceed.

 

7. Select the OmniFlow Server on which your process is saved from the WFS list. This list displays all the OmniFlow server that are registered

8. Click Connect to connect to that server.

9. The list of Cabinet registered on the selected OmniFlow Server is displayed in the Cabinet Name. Select the Cabinet Name.

10. According to the selected cabinet, the list processes defined for that cabinet is displayed in the Process Definition combo box. Select the process.

11. According to the process selected, the list of Activities defined for that process are displayed in the Selected Activity List. By default, all the activities are selected for the process that is to be routed by Processing Server.

12. To de-select the activities, select them and click >>. The current Processing Server does then not rout these activities.

13. To generate the log for each activity performed on Processing Server, select the option Enabling Log.

To enable the processing server to perform load balancing for the different activities select Load PS. Load Balancing means that the PS starts processing the activity with the maximum load. The activity is processed for the Load value specified. After that, the PS searches for an activity with the next maximum load. In this manner, the processing continues.

14. Whenever the workitem is initiated or processed in the Process, there is a column in Configuration Server Dialog box, which shows the number of workitems processed. It is updated and reflects the total count for the workitems, which are processing. For monitoring the status on CS select the Monitor Status option. If this property is not selected, then the ‘number of workitems processed’ column is marked as 0.

15. Click Finish to register the Processing Server, else click Exit.

16. The registered Process Server is displayed in the Configuration Server dialog box.

 

17. Currently, the status is stopped. To start the Processing Server for routing the workitems, click Start.

18. The status column is updated and the number of workitems processed is displayed.

19. To stop the selected application, click Stop. The message box is invoked for the confirmation.

a) Click Yes to stop the selected application.

20. To unregister the selected application, click Unregister. The Application will no longer be displayed in CS.

a) Click Yes to unregister the selected application.

21. To view the Properties of the selected application, click Properties.

22. The properties dialog box is invoked.

23. You can select \deselect Enabling Log and the Monitor Status option through properties.

 

Enabling Process through Process Manager

For administration purpose, you need to login the Process Manager. Only members of the supervisory group will be able to login.

To login the Process Manager:

1. Specify the http: path for the Process Manager in the browser.

The login screen is invoked.

2. Specify the User Name, Password and select the Database name for which the connection has to be made.

3. Select the option Remember my user id and Cabinet, if you don’t want to specify user name and select the cabinet every time you login.

If this option is selected the user name and current cabinet is displayed at the time of next login. The user just has to specify the password.

4. Click Login to start the session.

 

To change the process status:

1. Click Admin Icon from the Process Manager screen.

2. The Admin Option screen is invoked. Click Process Management link.

3. The Manage Process screen is displayed. Click Modify Process(s) Status link.

The Process Status screen is invoked.

4. If you select the process from Enabled Process list, and click Change State.

The selected process will become disabled. It will be displayed in the Disabled Process list, this process will not appear in the Desktop Client and Web Client.

5. If you select the process from Disabled Process list, and

5. If you select the process from Disabled Process list, and click Change State.

The selected process will become enabled. It will be displayed in the Enabled Process list, this process will appear in the Desktop Client and Web Client.

6. Click Close to exit from the Process Status screen.
