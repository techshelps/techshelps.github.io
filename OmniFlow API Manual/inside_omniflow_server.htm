Inside OmniFlow Server

OmniFlow Server is a multi-tired platform independent solution, built using robust server-side Java and J2EE technologies. It supports the following configurations:

  • MS-SQL Server and ORACLE on Windows

  • ORACLE, DB2 on Linux

  • ORACLE on Unix

  • J2EE-based application servers such as BEA WebLogic™ and IBM WebSphere™  

  • Windows NT and 2000 Active Directory, iPlanet and NDS services.

System Architecture

 

The data can be entered for the OmniFlow server through the following interfaces either on LAN or WEB.

  • OmniFlow Windows Desktop,

  • OmniFlow Web Desktop,

  • Monitoring Desktop and

  • Work Builder.

The data interacts with the OmniFlow Server which can be further divided into various Processing Servers.

 

XML interface for application integration

System provides XMLs for complete set of workflow calls. It exposes consolidated callality-level XML-based API for easy integration with client applications

Load balancing

System supports load balancing on Group Queues. When workitems arrive in these Queues they are diverted to individual users of the work queues as per the defined load balancing criteria defined. While distributing the workitems System also takes care of users current workload and availability.

Designed for handling high volume mission critical business processes

OmniFlow is designed using server side Java, and can be deployed on multiple servers, to handle thousands of workitems per day. It can cater to the needs of small as well as big enterprise. Contains all major workflow callalities like queue management, sequential and parallel routing, queue monitoring, load balancing, tracking, routes versioning, powerful searches, Access rights and administration
