Introduction

Workflow Engine

It provides the facility to automate business process for increased productivity, efficiency and control.

Engine is based on server side JAVA based technology. It is designed to serve small as well as big corporate. It is capable of handling thousands of transactions per day. It supports ORACLE, MSSQL and DB2. It has its own application server, which handles session management, database license management etc.

WfMC compliant server

System exposes the calls given under interface 2 and 3 of WfMC specification. Being WfMC compliant you can run any application, which uses WfMC client interface to interact with workflow server.

Deployable on any EJB Application Servers  

System provides EJB for the complete logic of workflow. These beans can be deployed on application servers like Weblogic, JBOSS etc.

XML interface for application integration

System provides XMLs for complete set of workflow calls. These XMLs can be used for integrating workflow with any system.

Load balancing

System supports load balancing on Group Queues. When workitems arrive in these Queues they are diverted to individual users of the work queues as per the defined load balancing criteria defined. While distributing the workitems System also takes care of users current workload and availability.

Designed for handling high volume mission critical business processes

OmniFlow is designed using server side Java, and can be deployed on multiple servers, to handle thousands of workitems per day. It can cater to the needs of small as well as big enterprise.

 

Seamless integration with existing systems

It provides various mechanisms for integration with other existing systems. You can call external programs from server for updating existing system on some event.
