Message Functions

The message functions send and receive messages and manipulate message aliases. The message functions are:

NetMessageBufferSend
NetMessageNameAdd
NetMessageNameDel
NetMessageNameEnum
NetMessageNameGetInfo

A message is a buffer of text data sent to a user or application on the network. To receive a message, a user or application must register a message alias in a computer's table of message names. This can be done by using NetMessageNameAdd. A message name table contains a list of registered message aliases (users and applications) permitted to receive messages.

The messenger service must be running on the receiving computer, which will display a popup message when the message is received. In addition, the Workstation service must be running on the local computer. Netbios is the transport mechanism used between the sender and receiver.

The aliases registered in the message name table are case insensitive. NetMessageNameDel deletes a specific message alias from the message name table. The NetMessageNameEnum function lists all the aliases stored in the message name table. The NetMessageNameGetInfo function retrieves information about a particular message alias in the message name table.

To send a message, an application calls NetMessageBufferSend.

Message functions are available at two information levels:

MSG_INFO_0
MSG_INFO_1

MSG_INFO_1 exists only for compatability. The Windows NT messenger will not forward names or allow names to be forwarded to it.
