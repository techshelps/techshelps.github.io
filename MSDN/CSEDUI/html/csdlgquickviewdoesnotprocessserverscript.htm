Quick View does not process server script

   

You are switching to Quick View while editing an ASP file. Quick view cannot run server script.

If your page contains server script, controls, or objects that normally run on the server, they will not be active in Quick View. These include:

  • Any script between <% and %> delimiters.

  • Script in <SCRIPT> blocks with the attribute RUNAT=SERVER.

  • Visual InterDev design-time controls whose target platform is Server.

  • Server-based controls such as those found on the Server Objects tab of the Toolbox.

  • ActiveX controls installed on the server and referenced in your page using an <OBJECT> block.

Client script in your ASP page will run correctly. Design-time controls whose target platform is Client and Java applets will also run correctly.

To proceed, choose one of the following options:

  • Switch to Quick View   Proceed to Quick view. If your page contains server script, it is ignored and does not appear on the page.

  • Switch and don't warn again   Proceed to Quick view and turn off this warning message for the remainder of your current Visual InterDev session. Server script is ignored. The next time you start Visual InterDev, this warning message is reactivated.

  • Return to preview view   Return to the view you were in before you chose Quick View.
