UNDI FORCE INTERRUPT

Op-Code: PXENV_UNDI_FORCE_INTERRUPT
Input: ES:DI points to a t_PXENV_UNDI_FORCE_INTERRUPT parameter structure that has been initialized by the caller.
Output: PXENV_EXIT_SUCCESS or PXENV_EXIT_FAILURE will be returned in AX, with the CF set accordingly. The status field in the parameter structure will be set to one of the values represented by the PXENV_STATUS_xxx constants.
Description: This call forces the network adapter to generate an interrupt. When a receive interrupt occurs, the network adapter driver usually queues the packet and calls the application&rsquo’s callback receive routine with a pointer to the packet received. Then, the callback routine either can copy the packet to its buffer or can decide to delay the copy to a later time. If the packet is not immediately copied, the network adapter driver does not remove it from the input queue. When the application wants to copy the packet, it can call the PXENV_undi_force_interrupt routine to simulate the receive interrupt.
