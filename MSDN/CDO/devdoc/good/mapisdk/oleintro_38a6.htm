About Installation

The Collaboration Data Objects (CDO) Library version 1.2 is installed with the Microsoft® Exchange Client and Server. The setup programs register the CDO Library for subsequent use by automation controllers, that is, applications that support Automation.

Note  Microsoft® Outlook™ version 8.03 includes the predecessor of CDO version 1.2, namely Active Messaging 1.1. Future versions of Outlook are currently planned to include CDO 1.2.

When you use the CDO Library with an automation controller, verify that the tool has referenced the CDO Library. For example, when you are using Microsoft® Visual Basic® version 4.0, choose the References command from the Tools menu, and select the check box for Microsoft CDO 1.2 Library.

When the CDO Library is available, the following flag is set in the WIN.INI file: 

[Mail] 
OLEMessaging=1

The OLEMsgPersistenceTimeout registry setting controls how quickly the CDO Library shuts down and unloads from memory after all messaging objects are released by client applications. On Win32® systems, the setting appears at the following registry location: 

HKEY_LOCAL_MACHINE\Software\Microsoft\Windows Messaging Subsystem

For 16-bit Microsoft® Windows® systems, the OLEMsgPersistenceTimeout setting appears within the [MAPI] section of the WIN.INI file.
