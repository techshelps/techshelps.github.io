Spilling the Beans About Microsoft Visual J++

First Java Development Environment to Support COM Objects

by Paul Johns

Microsoft’s just pulled a fresh cup of Java from its espresso machine. It’s called Microsoft Visual J++, and it’s a lot more than just another round of caffeinated puns. It’s just what the good doctor ordered. Visual J++ is a strong and hearty brew—more so because with Visual J++, Microsoft has poured a double shot.

First shot: Visual J++ is rich and tasty enough to set a new standard for convenience and power in Java development and debugging systems. Second shot: As the first Java development environment to support creation and use of COM objects, Visual J++ gives you a real kick: unprecedented power to create solutions for use on the Internet, on your Intranet, or as standalone applications.

Java integrated

Visual J++ is Microsoft’s implementation of Java integrated into the Microsoft Developer Studio Integrated Development Environment (IDE). If you’re familiar with Developer Studio, you already know how its integration of a family of tools—including the Microsoft Developer Network, Visual SourceSafe, Visual C++, Visual Test, and Microsoft Fortran PowerStation—makes tough development tasks easy. And you know how the world-class debugger makes it a snap to find bugs (assuming your code has any). If you haven’t used Developer Studio, you’re in for a treat—and incredible productivity. (It’s better than a double tall mocha!)

Real Java lovers want to drink their espresso anywhere—they want their Java applets to run on any platform. That’s why the code generated by Visual J++ is compatible with all Java execution environments, including Netscape Navigator.

Visual J++ comes with Microsoft Internet Explorer 3.0, which includes Microsoft’s Java execution environment, the reference-standard Java execution environment for Windows platforms. The execution platform includes the high-performance Just in Time (JIT) compiler, which runs Java many, many times faster than the fastest Java interpreter. Finally, it is the first Java execution environment to allow seamless integration of Java applets and ActiveX components.

Java and ActiveX

Java is strong technology all by itself, but when you combine it with ActiveX, you can quickly and easily do things that’ll make your boss say, "Wow! That’s cool—and useful!" (Hint: that’s a great time to ask for a raise.)

Visual J++ is the first tool to let developers quickly and easily use ActiveX components (also known as COM objects) directly from within Java programs. With Visual J++, using ActiveX components is as easy and natural as using a Java class. Components currently available include those providing database access, distributed systems, and high-performance multimedia. Because ActiveX components can be written in any language (including Java), using ActiveX components from within your Java program provides a great way to reuse code written in another language, including legacy code. That sure beats having to grow your own beans by writing everything from scratch in Java. And, of course, you can use Visual J++ to automate any application that provides a COM automation interface—including such applications as Microsoft Excel.

As noted, Visual J++ makes it easy for you to write ActiveX components. These components can be used anywhere that any COM component can be used—by ActiveX-enabled browsers, by applications that can use COM objects (including many Microsoft applications), and by programs written in any language. And with distributed COM (DCOM), you can even execute your Java COM components on a server and communicate with clients over a network.

I hear you asking, "But isn’t ActiveX proprietary? Won’t I lock myself into Windows if I use or write ActiveX components?"

In a word, "No."

Open standard

Microsoft is turning over ownership of ActiveX specifications—including COM and DCOM—to a neutral standards organization (of which Microsoft will be but one of many members). So ActiveX will soon be compatible with all leading coffeemakers and computing platforms—a true open standard.

Also, Microsoft and partners are working hard to implement ActiveX on the Macintosh and UNIX platforms. Microsoft will ship ActiveX-enabled Internet Explorer 3.0 for these platforms soon.

Now that you know what you can do with Visual J++, let’s look at how easy it is to brew great Java applets and applications using Developer Studio. (If you already use Developer Studio, you already know about most of these features.)

Sifting through the beans: Developer Studio Project Workspace window

The Developer Studio Project Workspace window makes it easy to find what you need to see through its three views: ClassView shows the classes, methods, and variables in your project; FileView shows all the files in your project—whether they’re Java source code, HTML, graphics, or other formats; and InfoView allows users to browse and search online documentation. From any view, a double-click on the icon that represents what you want to see brings it up in the right-hand window—much like Windows Explorer. Right-clicking on those same icons gives you a menu of common things you might want to do—for instance, right-clicking on a class name allows you to quickly add methods or variables.

Wizards give you perfect espresso the first time

To make brewing your first Java applet—and every applet after that—a snap, Microsoft has extended its Wizard technology to Visual J++. The Java applet AppWizard provides a wide variety of options from which you can choose:

Explanatory comments

Multithreading support

Simple animation support

Support for parameters passed to the applet from the Web page

Running the applet as an application

Setting the size of the applet

Handling common events such as mouse clicks and mouse movements

   

     Java Applet Wizard

When you’ve chosen your option, the Java Applet Wizard generates the skeleton of the applet for you. You can even run it before you add any code. For a complete applet, all you have to do is add code to perform the actions you need.

If you don’t want to start from scratch, it’s easy to make an empty Visual J++ project and to import an existing Java program. This allows you to use the full power of Visual J++, including the debugger, on projects someone else created.

The Resource Wizard converts Windows-based dialog box and menu resources into standard Java source code, letting you reuse existing dialog box templates and menu resources. The ability to use Developer Studio’s visual dialog box and menu editors makes laying out forms and menus much easier.

And you don’t have to use Microsoft coffee with Visual J++: it supports AppWizards built by third-party companies.

Debugging in Visual J++

The Visual J++ debugger advances the state of Java applet debugging. First, it allows you to debug an applet while it’s running in Internet Explorer 3.0. In fact, you can debug multiple applets on the same Web page simultaneously. And, of course, you can debug multithreaded applets and ActiveX components written in Java. You can even debug applets you download from the Web, including source-level debugging (if you have access to the source and to a version of the applet with standard-format debugging information—most applets on the Web include debugging information).

The Visual J++ debugger provides the same functionality as Developer Studio, including the Call Stack window, which allows you to see how you got where you are, and the Watch window, which displays variables’ values. DataTips let you merely point to an expression to display its value.

Unique to Visual J++, the Variables window features a tree-view hierarchy of classes and three panes: a "locals" pane with information on variables in scope; an "auto" pane, which is automatically updated to show expressions of interest based on the current statement; and a "this" pane, which displays the variables of the current class object.

Editing code

Visual J++ source editor gives developers lot of flexibility, custom key bindings and emulations of the popular BRIEF and Epsilon editors. It even extends syntax coloring to HTML files!

The popular resource editor from Visual C++ helps you quickly create menus and form-based applications. Just create a Windows dialog box template and/or menu resource using a visual resource editor, and then run the Resource Wizard to generate the Java code.

Visual J++ also includes an image editor for creating Java-compliant .GIF and JPEG files and converting bitmaps into .GIF and JPEG files.

Finally, Developer Studio allows you to have projects with many types of source files—Visual J++, Visual C++, HTML.

How to make great Java

The Visual J++ development system provides developers with an incredible amount of information to help them become productive immediately, including:

Learn Java Now, by Randy Davis (300 pages)

A multimedia tutorial, "Introduction to Visual J++"

F1 Help—select a word (such as a method name) and then press F1 to get instant Help

Tip of the Day—a different tip every time you start Visual J++

Books Online documentation, including a "Test Drive" section for new users, Sun’s Java language specification and Java application programming interface (API), search capability, bookmarks, annotation, and integration with MSDN

Samples galore—Visual J++ includes the basic Java samples from Sun’s Java Development Kit as well as samples of using Visual J++ to use and write ActiveX components
