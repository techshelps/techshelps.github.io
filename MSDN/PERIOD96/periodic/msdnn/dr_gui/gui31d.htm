Dr. GUI replies:

Doing things remotely! More cost savings for Dr. GUI's HMO!

Remote Automation originally shipped with the Enterprise Edition of Visual Basic 4.0, which is why you see most of the articles detailing Remote Automation using Visual Basic as the default language.

That is rapidly changing. Visual C++ 4.2 Enterprise Edition now also ships with Remote Automation, as does Visual FoxPro 5.0. Also important is the recent release of Windows NT 4.0, which includes the Distributed Component Object Model (DCOM).
