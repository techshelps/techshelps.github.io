SQLColAttribute (Results Menu)

If you enter a FieldIdentifier that is not recognized by ODBC Test, it is assumed to be a driver-specific descriptor type. So that ODBC Test can correctly display the value returned for this option, you must specify the value's C data type in the fFieldIdentifier Type box. If the description type is SQL_C_ULONG, ODBC Test assumes the value is returned in the NumericAttributePtr argument; otherwise, it assumes the value is returned in the CharacterAttributePtr argument.
