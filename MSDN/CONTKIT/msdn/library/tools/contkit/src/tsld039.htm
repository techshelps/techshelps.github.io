Intranet Subtleties (Simplicity)

  • If no specific benefits on client, keep non-UI, internal components on the server

    • Avoid code signing issues
    • Avoid packaging of control licenses (.LPK)
    • Avoid size focus during component development
      • Actual size of .OCX & dependent files
        • Simpler tool can be used (MFC, VB5)
      • Avoid needing .CAB and .INF (for remote users)
      • File size usually not an issue on intranet
        • Fast LAN and client side configuration control
