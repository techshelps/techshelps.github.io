Test Case: C1.00.00

Title: Registering Controls through Control Toolbox

Platforms: Windows NT 3.51, 4.0, Windows 95
Requirements: Office97 app, OLE control. Container must have Control Toolbox
Sample Apps: Microsoft Word 97, DBGrid Control
Test Category: CONTROL

Overview:

This test verifies that the control can be registered via the control toolbox. There is no fundamental difference between registering a control this way as opposed to RegSvr32 or the OLE Control Test Container, or using the development environment itself.

Steps:

  • Start Office97 app
  • Create new document
  • Choose View Toolbars and ensure Control Toolbox is visible
  • Click on bottom right button for "More Controls"
  • Scroll to bottom of list and choose "Register Custom Control"
  • Browse to find the OCX or DLL to register
  • Click OK
  • Note: there is no visible feedback unless the control is unable to be registered
  • Verify the control has been registered by choosing Register Custom Control again and searching for control in the list.

Summary:

This test verifies that the control can be registered through Office. If there are problems, try registering with REGSVR32 (REGSVR32 myctrl.ocx), or the OLE Control Test Container that ships with Visual C++.
