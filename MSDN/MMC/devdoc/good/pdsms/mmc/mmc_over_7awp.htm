Snap-In Components and the Registry

[This is preliminary documentation and subject to change.]

Following is the registry layout used by the Microsoft Management Console. Two registry keys, each with a set of subkeys, are used as the basis for all required entries. The first is called the SnapIns key and it contains the CLSID of the snap-in. The second key is called NodeTypes and contains the GUID of the snap-in's node type.
