Some project files need updated source control settings

   

The Microsoft development environment has identified inconsistencies between the source control settings found in your project or solution files and those stored in your local cache. The system attempted to resolve the problems by checking out and fixing the project and solution files but was unable to do so. This may indicate that someone else exclusively checked out the project and solution, or it may indicate another problem with the source control system.

If you have not recently branched or forked your project, the development environment will be able to resolve the situation and you do not need to do any further action.
