Using the Idle Disconnect Feature

Idle disconnect reduces the cost of remote access carrier service and can be used to reduce the total capacity of dial-in communications links, based on the reduction of wasted remote connect time. This feature can be configured on remote clients that use Windows NT–based dial-up networking.

This new feature automatically terminates a RAS connection to your server after a certain period of time if there has been no activity over the remote dial-up communications link. The user or administrator can specify the amount of time before this feature is activated.
