EBLC: Bitmap Data Locations

The 'EBLC' table identifies the sizes and glyph ranges of embedded bitmaps, and keeps offsets to the bitmap data in 'EBDT'.

Requirements

  • 'EBLC' format 2 (constant metrics) or 5 (constant metrics with sparse glyph codes) is required for bitmap data using 'EBDT' format 5.
Recommendations

  • It is possible for multiple sizes in 'EBLC' to index the same data in 'EBDT', if the 'EBDT' data is stored in format 5. In this way, multiple sizes can use the same bitmap image with different metrics (more or less whitespace surrounding the glyph).
