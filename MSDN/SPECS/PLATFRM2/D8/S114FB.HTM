ISV Designed for Microsoft BackOffice Logo Requirements:

Server Application Logo Requirements:

  • Run as a Windows NT™ Server service
  • Network independence (support both IPX/SPX and TCP/IP using RPC, OLE, Named Pipes, or WinSock) If you have a Internet Server Application it only needs to support TCP/IP.
  • Support Windows NT unified logon
  • Installable using Systems Management Server (write PDF, support Uninstall and unattended install in setup program, etc.)
  • If your Server Application is integrated with Internet Information Server, you must do the following in addition to the above requirements:
  • Internet Servers
  • Support ISAPI as a means of extending WWW functionality
  • Internet Information Server Extensions
  • Implemented as ISAPI function, or provide an ISAPI connector to a logo'd server
Client Application Logo Requirements:

Product Category

Requirements

All Client Applications

Must meet Client requirements for one or more appropriate client applications. (SNA, Messaging, Systems Management Server, SQL or IIS).

Installable using Systems Management Server (write PDF, support Uninstall and unattended install in setup program, etc.)

Must have at least a 16-bit Windows based client, Win32

client preferred. If you have at least a Win16 client, will allow logo to be used on other clients (Mac, UNIX, OS/2, MS-DOS, etc.) as well.

Use the APIs appropriate to the server application (e.g., ODBC, MAPI)

SNA Client

Use WOSA APIs (WinAPPC, WinCPIC, WinRUI, WinSLI) or use FMI

Use SNA Server client APIs (support SNA Server user record)

Use SNA Server compatibility APIs for IBM PC Support or Client Access/400

Messaging Client

Use MAPI for send and post OR use OLE/Messaging and/or OLE/Scheduling.

SQL Server™ Client

Use ODBC and/or DBLib, or Embedded SQL to store data in a SQL Server DB, or to access data stored in a SQL Server DB

Systems Management Server

Access Systems Management Server Database using SMS APIs or ODBC

Extend data in the database by providing a MIF file

Internet Information Server

Support session keep alives

Support NT challenge response


Development Tools Logo Requirements:

  • Must be able to create a server or client application which meets the requirements above.
  • Must be able to generate a 32-bit application.
