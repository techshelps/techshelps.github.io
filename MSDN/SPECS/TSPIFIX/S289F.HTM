1.3.1 Function Names

Function names consist of several descriptive words, concatenated into a single word-description of the function. To help you remember the names, they are constructed within a prefix/verb/noun model, where:

  • The prefix "TSPI_line" or "TSPI_phone" indicates that the function belongs to the Telephony SPI. "TSPI_line" prefixed operations work on the connection to the network. "TSPI_phone" prefixed operations work on a telephone set. This is a phone-type device that is somehow associated with the PC running the client of the TSPI, and controlled via the TSPI. The mechanism used to establish the actual association is up to the different implementations. Function prefixes are always written in lowercase, except for the "TSPI_" part.
  • The verb tells you what the function does. Function verbs always begin with an uppercase letter.
  • The noun describes the target of the action. The noun may be absent in cases where the verb makes it obvious what the noun should be. For example: TSPI_lineAnswer is used instead of the longer TSPI_lineAnswerCall. Conversely, some function names contain more than one word for the "noun". Function nouns, like verbs, always begin with an uppercase letter. As an example of a function name, TSPI_lineMakeCall is a line function initiates a telephone call.
