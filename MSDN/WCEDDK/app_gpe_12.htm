GPE::AllocSurface

This method is called when the driver must allocate storage for surface pixels. The surfaceFlags field may contain GPE_REQUIRE_VIDEO_MEMORY or GPE_PREFER_VIDEO_MEMORY or neither. If GPE_REQUIRE_VIDEO_MEMORY is set, the call must return E_NOT_ENOUGH_MEMORY if there is insufficient video memory available or E_INVALID_PARAMETER if the allocation is inappropriate for video memory (e.g. wrong pixel format). For GPE_PREFER_VIDEO_MEMORY, the driver is expected to allocate the surface from video memory if possible, otherwise it should use system memory.

Syntax

SCODE GPE::AllocSurface( GPESurf **ppSurf, int width, int height, EGPEFormat format, int surfaceFlags )

For video memory allocation, it is expected that the driver will implement a class derived from GPESurf to contain any extra information such as a pointer to the allocator to be used when freeing the memory. Refer to the Node2D class for information on a supplied memory allocator which may be used for rectangular allocation requirements.

For system memory allocation, GPE::AllocSurface should use the following code segment:

    *ppSurf = new GPESurf( width, height, format );
    if( *ppSurf )
        return S_OK;
    return E_NOT_ENOUGH_MEMORY;
