Code Generated by the Snap-In Wizard

Once you have determined the name and options for your snap-in object, the Snap-In wizard generates the necessary files and adds them to the existing ATL project.

Every generated snap-in object is implemented with three files. For this discussion, MySnapIn is the short name of the snap-in and the default options are accepted. The generated files are as follows:

  • MYSNAPIN.CPP

    Defines the GUIDs used by data members of the snap-in data class (CMySnapInData).

  • MYSNAPIN.H

    Contains the declarations for the five classes that implement the ATL snap-in object (in this case, MySnapIn):

    Class Implements
    CMySnapInData Data class of the snap-in
    CMySnapInComponent Icomponent interface
    CMySnapIn IcomponentData interface
    CMySnapInAbout IsnapInAbout interface
    CMySnapInPage Property page of the snap-in

  • MYSNAPIN.RGS

    Registers the snap-in object with the operating system upon the first usage.
