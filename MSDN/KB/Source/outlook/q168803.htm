OL97: Custom Form Icon Change Does Not Display in Table View

 Last reviewed: February 27, 1998
Article ID: Q168803
The information in this article applies to:
  • Microsoft Outlook 97

SYMPTOMS

In Microsoft Outlook 97, a large or small icon change included in a custom form definition does not show up in a table view of the received message. Instead, you see the standard message icon.

CAUSE

This behavior is by design. The form is not stored where you get the icon property. The icon property is an index to the resource and is only available if the form comes from the forms registry.

MORE INFORMATION

Steps to Reproduce Problem

  1. Open a message form and enter design mode.

  2. Click the form's Properties tab.

  3. Click, Change Large Icon or, Change Small Icon and choose a different icon.

  4. On the Properties page click to select, "Save form definition with item."

  5. Exit design mode and publish the form to your personal forms.

When you compose a message based on this form, the recipient will not see the new icon in the table view.
