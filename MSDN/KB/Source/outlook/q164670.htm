OL97 ErrMsg: Opening Office Resource Kit File Forms2.pst

 Last reviewed: January 19, 1998
Article ID: Q164670
The information in this article applies to:
  • Microsoft Outlook 97
  • Microsoft Office Resource Kit 97

SYMPTOMS

When you follow the instructions in the Office 97 Resource Kit (ORK) Readme.doc file to open the Sample Forms file (Forms2.pst), you may receive the following error message: 

   File access is denied. You do not have the permission require to access
   the file C:\Program Files\Microsoft Office\Office\FORMS2.PST.

CAUSE

This problem may occur if the Forms2.pst file has an attribute of Read- only. Outlook 97 cannot open a Read-only Personal Folders file.

The original file on the ORK Compact Disc has the Read-only attribute set and when you follow the instructions to copy that file to your computer's hard drive the copy inherits the Read-only attribute.

RESOLUTION

To resolve this problem you must remove the Read-only attribute from Forms2.pst. Follow these steps to remove the Read-only attribute:

  1. Click the Start button, point to Find, and then click Files or Folders.

  2. In the Named box, type Forms2.pst.

  3. Make sure there is a check in the Include subfolders check box and click OK.

  4. Right-click the file, Forms2.pst, and from the context sensitive menu click Properties.

  5. Click to clear the Read-only check box and then click OK.

  6. On the File menu, click Close.

Now you can proceed with the instructions in the Office Resource Kit without receiving the above error message.

REFERENCES

For more information about how to perform this task in Windows, see your Windows printed documentation or online Help.
