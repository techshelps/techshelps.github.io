OL97: Remove All Setup Leaves Some Files in System Folder

 Last reviewed: January 19, 1998
Article ID: Q174121
The information in this article applies to:
  • Microsoft Outlook, versions 8.01, 8.03

SYMPTOMS

When you select Remove All, in Maintenance Mode Setup of Microsoft Outlook 97, version 8.01, the following files remain in the System folder of Microsoft Windows 95 or the System32 folder in Microsoft Windows NT Workstation 4.0, SP3: 

   Ccmail.hlp
   Cnfnot32.exe
   Emsmdb32.dll
   Emsapb32.dll
   Emsui32.dll
   Emsuix32.dll
   Fm20.dll
   Fm20env.dll
   Mlctrl.dll

STATUS

Microsoft has confirmed this to be a problem in Microsoft Outlook 97, version 8.01. The problem is corrected in Outlook 97, version 8.03.

MORE INFORMATION

In Outlook 97, version 8.03, the following files remain after removing the program: 

   Mapi32.dll
   Mapi.dll
   Wmsui.dll
   Wmsui.dll
These files remain so users can install an earlier client and have the correct set of dynamic-link library (DLL) files available.
