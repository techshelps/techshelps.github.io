OL97: Password Error Connecting to Internet Mail Provider

 Last reviewed: February 4, 1998
Article ID: Q164975
The information in this article applies to:
  • Microsoft Outlook 97

SYMPTOMS

Using Outlook 97 you may get the following error message when dialing your Internet Mail Service: 

   The computer you are dialing in to cannot establish a dial-up
   networking connection. Check your password and then try again.

CAUSE

The User Name and/or Password are incomplete or incorrect in either the Internet Mail service profile or in the Dial-Up Networking connectoid.

RESOLUTION

Make sure the User Name and Password are either both correct or both empty in the Login As dialog box in the Internet Mail service profile. If the User Name and Password are empty, also make sure the User Name and Password are both correct in the Connect To dialog box of the Dial-Up Networking connectoid.

NOTE: The Login As information in the Internet Mail service profile will take precedence over the information in the Dial-Up Networking connectoid. However, if the User Name and Password information in the Login As dialog box is blank, this information will be taken from the Dial-Up Networking connectoid.

MORE INFORMATION

Steps to Verify the Internet Mail Service "Login As" Information

  1. In Outlook click Services on the Tools menu.

  2. Click Internet Mail service then click Properties.

  3. In the Internet Mail dialog box, click the Connection tab.

  4. Make sure the, "Connect using the modem" option is selected.

  5. Make sure the proper Dial-Up Networking connectoid is selected in the, "Dial using the following connection", list then click Login As.

  6. On the Login As dialog box, make sure that the User Name and Password fields are both correct.

  7. Click OK to exit all dialog boxes.

Steps to Verify the Dial-Up Networking "Connect To" Information

  1. Double-click the My Computer icon on your desktop.

  2. Double-click the Dial-Up Networking icon.

  3. Double-click the dial-up connectoid used to connect to your Internet provider.

  4. On the Connect To dialog box, make sure that the User Name and Password fields are both correct.
