OL97: Running Clean Mailbox Agent Can Result in Lost Messages

 Last reviewed: January 19, 1998
Article ID: Q169503

The information in this article applies to
  • Microsoft Outlook 97

SYMPTOMS

When you use the Mailbox Cleanup Agent to delete messages from a server- based folder containing Microsoft Outlook 97 messages moved from another folder, you might lose messages that were supposedly archived.

CAUSE

When you move or change an Outlook message you update its "Modified" date attribute, used by the Archive program. Moved messages do not appear to the archive program to be as old as they were before the move, therefore may not be archived as soon as expected. The Mailbox Cleanup Agent, on the other hand, uses the "Created" date attribute (in this case a date older than the Modified date) to determine the message's age, and therefore may delete messages that are not yet archived.

RESOULTION

Before running the Mailbox Cleanup Agent on a folder that contains messages moved from another folder, you should check the Modified date of the messages to see if they have been archived.

More Information

To Set AutoArchiving in Outlook

  1. On the Tools menu, click Options.

  2. On the AutoArchive tab, click to select AutoArchive Every and then set the timeframe.

  3. Complete the other AutoArchive options and click OK.

To Set AutoArchiving for a Specific Folder

  1. Right-click the desired folder.

  2. On the shortcut menu, click Properties.

  3. On the AutoArchive tab, click Items Older Than and set the timeframe.

  4. Complete the other AutoArchive options and click OK.

To Check a Message's Modified Date

  1. Open the message.

  2. On the File menu, click Properties.
