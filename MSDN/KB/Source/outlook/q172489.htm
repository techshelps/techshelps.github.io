OL97: File Locking Issue With Novell Client 32

 Last reviewed: January 19, 1998
Article ID: Q172489
The information in this article applies to:
  • Microsoft Outlook 97

SYMPTOMS

When attempting to open a file on a Novell NetWare network share, you may receive the following error message: 

   The file \<path>\filename.xxx could not be accessed because another
   workstation has modified it.

CAUSE

This error occurs under the following conditions:
  • You are using Novell's NetWare Client 32 for Windows 95 version 2.11.
  • An program marked for recording in the Microsoft Outlook 97 Journal, saved the file.

This behavior occurs even if Outlook is closed, because the Journal feature monitors activity for the program at all times.

RESOLUTION

In Outlook, turn off Journaling for the program in question. 

 -or-
Wait until the Novell Client releases the Duplicate Handle Function before reopening the file. 

 -or-
Install the Microsoft Client for NetWare Networks.

MORE INFORMATION

Turn Off Journaling

  1. Click Options on the Tools menu.

  2. On the Journal tab, in "Also record files from" click to clear the check box for the program in question, and then click OK.

Wait for the Duplicate Handle Function to Release

There is a period of time after which the Novell client releases the Duplicate Handle Function. After this time the problem does not occur. We have not determined this period.

Install the Microsoft Client for NetWare Networks

  1. In the Control Panel, double-click the Network icon.

  2. On the Configuration tab, click Add.

  3. Click to select Client, and then click Add.

  4. Click to select Microsoft, click to select Client for NetWare Networks, and then click OK.
