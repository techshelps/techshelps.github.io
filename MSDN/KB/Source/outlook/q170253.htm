OL97 ErrMsg: Invalid Page Fault in Keyboard.drv

 Last reviewed: January 19, 1998
Article ID: Q170253
The information in this article applies to:
  • Microsoft Outlook 97

SYMPTOMS

When you start Microsoft Outlook 97 and you have Outlook support for Lotus cc:Mail in your profile, you may receive the following error message: 

  Invalid Page Fault in Keyboard.drv.

CAUSE

A problem with the dual-mode Vendor Independent Messaging (VIM) file, shipped with cc:Mail Release 6 and 7, and an e-mail message in your cc:Mail inbox can cause the error. The oldest unread message is usually the "bad" message.

RESOLUTION

The new VIM DLL files (6.02) combined with the Outlook support for Lotus cc:Mail version 1.2 (build 48), fix this problem. The new Outlook support for Lotus cc:Mail file can be downloaded from the following URL: 

   http://www.microsoft.com/OfficeFreeStuff/Outlook/dlpages/CCMailSp.htm
If you are using Windows 95 and Outlook 8.0 or 8.01, you need to obtain the current version of the 16-bit VIM files (vdlwin.zip) from Lotus.

If you are using Windows NT Workstation and Outlook 8.0 or 8.01, you need to obtain the current version of the 32-bit VIM files (vdlw32.zip) from Lotus.

The cc:Mail transport that ships with Exchange Server 5.0 Service Pack 1 and with Office 97 Service Release 1 uses the 32-bit VIM files (vdlw32.zip) for both Windows NT Workstation and Windows 95.

The files may be found at: 

   http://www2.support.lotus.com/ftp/pub/comm/ccmail/dev_tools/
Download, unzip, and copy the updated VIM files (Version 6.02) to your \WINDOWS\SYSTEM folder if you are using Windows 95, or your \WINNT\SYSTEM32 folder if you are using Windows NT, then restart your computer.

If you move the bad message to a cc:Mail folder using the cc:Mail client, the problem will be resolved after you quit and restart Outlook. You can then import the bad message.

This issue is fixed in Microsoft Outlook 97, Service Release 1.

MORE INFORMATION

The VIM association's Vendor Independent Messaging (VIM) Application Programming Interface (API) is a specification based on a proposal from Lotus and supported by Microsoft, Novell, Apple, Borland, and others. The VIM API is provided over a number of platforms; this enables the development of programs, which can be used on a number of platforms and can operate over a number of messaging services/protocols.

The latest version of VIM is VIM 6.02. It supports both 16-bit and 32-bit platforms; it supports both the DB6 and DB8 postoffices.

The cc:Mail transport, version 1.2 (build 48), uses 32-bit VIM DLLs for both Windows 95 and Windows NT. Previous cc:Mail transport versions use 16-bit VIM DLLs for Windows 95. Build 48 of the cc:Mail transport can be downloaded from the Microsoft Web site at the following URL: 

   http://www.microsoft.com/OfficeFreeStuff/Outlook/dlpages/CCMailSp.htm

To determine the cc:Mail transport version, follow these steps:

  1. On the Tools menu, click Services.

  2. Click Outlook support for Lotus cc:Mail, then click About.

  3. Click ccmxp32.dll and note the version number. The last two digits indicate the build.
