FIX: F1001: XX-2 line 539, Allocating 43 Arrays in 1 Statement

 Last reviewed: September 11, 1997
Article ID: Q67231
5.00 5.10 | 5.00 5.10 
MS-DOS    | OS/2
kbtool kbfixlist kbbuglist kberrmsg

The information in this article applies to:

  • Microsoft FORTRAN for MS-DOS, versions 5.0 and 5.1
  • Microsoft FORTRAN for OS/2, versions 5.0 and 5.1

SYMPTOMS

When you allocate 43 or more arrays with one ALLOCATE statement in Microsoft FORTRAN version 5.0, the following error is produced, where XX = ASCII characters 2 and 254: 

   fatal error F1001:  Internal Compiler Error
   (compiler file "XX-2," line 539)
With Microsoft FORTRAN version 5.1, the following error is produced: 

   fatal error F1001:  Internal Compiler Error
   (compiler file '', line 404)

RESOLUTION

Breaking the ALLOCATE statement into two separate statements eliminates the error.

STATUS

Microsoft has confirmed this to be a problem in FORTRAN versions 5.0 and 5.1. This problem was corrected in FORTRAN PowerStation.

MORE INFORMATION

The size of the arrays has no effect on the error. If only 42 arrays are defined and allocated, the error is not generated.
