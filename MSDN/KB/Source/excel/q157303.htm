XL97: Extended Characters in Text or Property Inaccurate

 Last reviewed: January 15, 1998
Article ID: Q157303
The information in this article applies to:
  • Microsoft Excel 97 for Windows

SYMPTOMS

A search for files in the File Open dialog of Microsoft Excel 97 may return inaccurate or unexpected results if you use extended ANSI characters in the Text or property field.

STATUS

This is by design in Microsoft Excel 97.

MORE INFORMATION

The ANSI character set contains 255 symbols. An extended character is a character with a value greater than or equal to 128. Extended characters can be typed into any edit boxes of the Microsoft Excel 97 File Open dialog using the ALT+<numeric keys> keystrokes. For instance a diacritical mark can be entered by typing ALT+0136. However, because these characters are extended characters outside the standard ANSI code page, when the characters are filtered for the search they may be stripped from the query string and produce unexpected or inaccurate searches. Microsoft recommends that you not use extended characters when you search for files within the File Open dialog of Microsoft Excel 97.
