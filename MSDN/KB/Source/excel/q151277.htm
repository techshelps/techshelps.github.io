MSQuery: "Table or View Does Not Exist" Uing Visigenic Driver

 Last reviewed: September 12, 1996
Article ID: Q151277
The information in this article applies to:
  • Microsoft Query for Windows 95, version 2.0

SYMPTOMS

You may receive the following error message in Microsoft Query version 2.0 while attempting to view a table from an ORACLE database using the Visigenic 32-bit ORACLE ODBC driver: 

   Table or view does not exist

CAUSE

This problem occurs if you log on to an ORACLE database as a user who is not the owner of the table being accessed.

RESOLUTION

If you need to access a specific table and receive the error message mentioned in the "Symptoms" section, log on to the ORACLE database using the user ID of the owner of the table.

STATUS

This problem occurs when you use Microsoft Query for Windows 95, version 2.0 and the Visigenic 32-bit Oracle ODBC driver version 1.03.0009.
