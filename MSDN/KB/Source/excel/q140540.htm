MXL5: X Axis Labels Incorrect in XY Chart with Dates in X Axis

 Last reviewed: February 2, 1998
Article ID: Q140540
The information in this article applies to:
  • Microsoft Excel for the Power Macintosh, version 5.0a

SYMPTOMS

In an xy (scatter) chart with an x axis that plots dates or numbers that are longer than two characters, the values that you align vertically on the x axis may appear to be incorrectly aligned when you view the chart in Microsoft Excel for the Power Macintosh, version 5.0a.

CAUSE

When you format dates or numbers on the x axis in any of the vertical orientation options, Microsoft Excel aligns the text according to the selected orientation option. However, Microsoft Excel does not expand the width of the x axis label, and values on the x axis appear to be broken into segments that are aligned vertically.

Numbers that contain three or four digits appear to be broken into segments that are arranged vertically, where each segment contains no more than two characters. Numbers that have more than four digits appear as exponential values and also appear to be broken into segments.

WORKAROUND

To work around this problem, use the following steps:

  1. Remove the axis labels. To do this, select the x axis, click Format Selected Axis on the Format menu, and click the Patterns tab.

  2. Under Tick-Mark Labels, click None.

  3. For each tick mark on the x axis, manually create a label and position it on the chart.

For more information about creating a text label on a chart, click Search in Microsoft Excel Help, type the following text 

   chart data labels
and then double-click the selected text to go to the "Adding data labels" topic.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Excel version 5.0a for the Power Macintosh. The problem was corrected in Microsoft Excel 98 Macintosh Edition.
