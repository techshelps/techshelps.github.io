Text not Formatted Pasting to Word for Windows 2.0

 Last reviewed: February 7, 1995
Article ID: Q82949
The information in this article applies to:
  • Microsoft Excel for Windows, version 3.0

Summary:

Linking a query window from Q+E into Microsoft Word for Windows version 2.0 will result in the text being pasted in as unformatted text. The same procedure using Word for Windows version 1.1 will result in the text being pasted into a table.

More Information:

While Q+E does meet the standards for doing Dynamic Data Exchange (DDE), not all of the features of Object Linking and Embedding (OLE) are offered. As a DDE server, Q+E simply puts the information on the clipboard that is necessary for a query to be performed.

Q+E is not written to include formatting information; in this case, Q+E queries are pasted as tab delimited data. Word for Windows version 1.1 made assumptions about the data that was coming in and formatted it as a table.

Steps to Reproduce Problem

  1. Open a file in Q+E.

  2. Select the entire window and choose Copy Special from the Edit menu.

  3. Choose SQL Text for the Link Formula, and either Selected Area or Entire Window for the Link Data.

  4. Open Word for Windows 2.0 and choose Paste Special from the Edit menu.

  5. The only option available in the Paste Special dialog box will be unformatted text.

  6. With Word for Windows 1.1, select Paste Link from the Edit menu.

  7. The text will paste into the Word for Windows document in the table format.
