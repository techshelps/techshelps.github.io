Q+E Err Msg: Duplicate Field Names in Table: XA

 Last reviewed: September 12, 1996
Article ID: Q98572
The information in this article applies to:
  • Q+E for Microsoft Excel for Windows, versions 3.0 and 3.0a
  • Q+E for Microsoft Excel for OS/2, version 3.0

SUMMARY

In Q+E for Microsoft Excel, when you attempt to save a query as an Excel or dBASE file, if the field names in the query window have numeric values followed by an alpha character you may receive the error message 

   Duplicate Field Names in Table: XA

MORE INFORMATION

Q+E will interpret a sequence of field names that are defined as a numeric value immediately followed by an alpha character as duplicate entries. For example, a sequence of field names defined as: "1a," "1b," "2a," and "2b" will generate the above error message when you attempt to save the query as a Microsoft Excel or dBASE file.

Workaround

For either Microsoft Excel or dBASE, modify the field names to be all alpha characters. For example, you could redefine "1b," "2a," and "2b" to be "Oneb," "Twoa," and "Twob."

MORE INFORMATION

REFERENCES

"Q+E for Microsoft Excel User's Guide" version 4.0 (included with version 4.0 of Microsoft Excel for Windows), pages 122 and 143
