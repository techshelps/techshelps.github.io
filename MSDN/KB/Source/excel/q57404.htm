Excel: Custom Data Form Not Behaving as Expected

 Last reviewed: November 2, 1994
Article ID: Q57404

SUMMARY

As with custom dialog boxes, the custom data form has a limit of 32 items that can take or return arguments. Furthermore, the custom data form can display only the first 32 fields in the database. Displaying fields beyond the 32nd can cause unpredictable results, including loss of functionality in some or all buttons in the form, data displaying incorrectly, or data not displaying at all.

Only the functionality of the form is lost; no data in the database is damaged. To correct the malfunctioning form, simply rearrange the database so that the custom data form only accesses the first 32 fields.

For more information on custom data forms, see pages 163-166 in the "Microsoft Excel Reference" manual for version 2.20. For more information on the limits of custom data forms and custom dialog boxes, see page 178 of the "Microsoft Excel: Functions and Macros" manual for version 2.20.
