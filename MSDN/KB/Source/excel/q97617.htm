XL: Err Msg: Cannot Open Consolidation Source File

 Last reviewed: February 2, 1998
Article ID: Q97617
The information in this article applies to:
  • Microsoft Excel for Windows, versions 3.0, 4.0, 5.0, 5.0c
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft Excel 97 for Windows
  • Microsoft Excel for the Macintosh, versions 3.0, 4.0, 5.0, 5.0a
  • Microsoft Excel 98 Macintosh Edition

SUMMARY

In Microsoft Excel, if you use the Consolidate command to consolidate closed files that are saved in the text, CSV, SYLK, DIF, or dBASE (DBF 2, DBF 3, or DBF 4) file formats, you receive the following error message 

    Cannot open consolidation source file <filename>
for each file you want to consolidate. This error message is followed by the following error message: 

   No data was consolidated

WORKAROUND

If the source files are formatted in the text, CSV, SYLK, dBASE (DBF 2, DBF 3, or DBF 4), or DIF file formats, open the files before you run the Consolidation command.

In order for the consolidation feature to work properly on a closed file, the file must be saved in a Microsoft Excel file format (Normal) or a Lotus 1-2-3 (WKS, WK1, or WK3) file format.

REFERENCES

"Microsoft Excel Users Guide 1," version 4.0, pages 374-383 "Microsoft Excel Users Guide," version 3.0, pages 328-334
