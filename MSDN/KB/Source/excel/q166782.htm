OFF97: Error Inserting Wang Image Control into Document

 Last reviewed: February 12, 1998
Article ID: Q166782
The information in this article applies to:
  • Microsoft Excel 97 for Windows
  • Microsoft Word 97 for Windows
  • Microsoft Access 97 for Windows
  • Microsoft PowerPoint 97 for Windows
  • Microsoft Office Binder 97 for Windows
  • Microsoft Windows NT version 4.0

SYMPTOMS

In any of the programs listed above, when you use the Control Toolbox to insert a control (an .ocx file), you may receive an error message similar to the following: 

   An application error has occurred and an application error log is
   being generated.

   <PROGRAM.exe>
   Exception: access violation (0xc0000005), Address: 0x02b70141
where <PROGRAM.exe> is one of the following: 

   EXCEL.exe
   WINWORD.exe
   MSACCESS.exe
   POWERPNT.exe
Note that the address may vary.

CAUSE

This problem will occur if all of the following conditions are true:

  • You are inserting one of the following controls: 

          Wang Image Admin Control
      Wang Image Annotation Control
      Wang Image Edit Control
      Wang Image Scan Control
      Wang Image Thumbnail Control


        -and-
  • You are inserting the control into a location other than a UserForm, for example, a worksheet, a document, or a presentation. 

        -and-
  • You are running Windows NT, version 4.0, on the computer. 

        -and-
  • You have not installed Service Pack 2 for Windows NT 4.0.

NOTE: This problem does not occur if you are running Microsoft Windows 95 or Windows NT 4.0 with Service Pack 2.

RESOLUTION

To eliminate this problem, install Service Pack 2 for Windows NT 4.0 on your computer. To download Service Pack 2, use your Web browser to go to the following address: 

   ftp://ftp.microsoft.com/bussys/winnt/winnt-public/fixes/usa/nt40/ussp2/

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. Installing Service Pack 2 for Windows NT 4.0 eliminates the problem.
