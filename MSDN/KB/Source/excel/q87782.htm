Excel: Vertical Print Titles Are Incorrect When Scaling

 Last reviewed: November 30, 1994
Article ID: Q87782
The information in this article applies to:
  • Microsoft Excel for Windows, version 4.0

SUMMARY

Scaling a multi-page worksheet with columns set as print titles produces unexpected results in version 4.0 of Microsoft Excel for Windows if you are printing to a non-PostScript printer.

MORE INFORMATION

The first page of output and any subsequent pages containing the same rows as page one print correctly. Any pages that print columns to the right of the columns printed on the first page include, as print titles, columns not originally selected as print titles. Using rows as print titles does not result in this problem.

Workaround

Removing either the columns as print titles or setting scaling to 100 percent eliminates this problem.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in a later version of Microsoft Excel.

REFERENCES

"Microsoft Excel User's Guide 1," version 4.0, pages 533-534
