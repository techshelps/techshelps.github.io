XL5: Message Always Sent from cc:Mail as "Low Priority"

 Last reviewed: September 12, 1996
Article ID: Q112507
The information in this article applies to:

- Microsoft Excel for Windows, version 5.0

SYMPTOMS

When you use Lotus Development Corporation's cc:Mail to send a mail message containing a workbook from Microsoft Excel version 5.0 for Windows, all of your mail messages are sent with a low priority setting.

CAUSE

This prioritization problem is caused by a problem contained in the version of the MAPIVIM.DLL file that is shipped with Microsoft Excel version 5.0.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in later versions of Microsoft Excel for Windows.

WORKAROUND

To work around this problem, you must use a MAPIVIM.DLL file version 1.0.0.63 or later.

This version of the file is included with the following applications:

  • Microsoft Word for Windows version 6.0a
  • Microsoft PowerPoint version 4.0
  • Microsoft Office version 4.2
  • International versions of Microsoft Excel version 5.0

If you do not have one of the above applications, you can obtain an appropriate version of this file by accessing the application note WE1009 "MAPI-to-VIM Translation .DLL MAPIVIM.DLL Version 1.0.0.63." For more information about accessing this Application Note, query on the following words in the Microsoft Knowledge Base: 

  AppNote and WE1009 and translation

MORE INFORMATION

The MAPIVIM.DLL file allows Microsoft applications, such as Microsoft Excel and Microsoft Word for Windows, to insert documents into mail messages and then send those messages using the cc:Mail program.

The MAPIVIM.DLL file that ships with Microsoft Excel version 5.0 for Windows is version 1.0.0.53, dated 12/14/93. This version of MAPIVIM.DLL contains a problem that causes all messages to be sent with a low priority setting, even if you choose to send the message at a higher priority setting.

If you have Microsoft Windows for Workgroups 3.11 installed on your computer, do the following to check the version of MAPIVIM.DLL that is installed on your system:

  1. In Microsoft Windows File Manager, switch to the WINDOWS\SYSTEM directory.

  2. In the File list, select the MAPIVIM.DLL file.

  3. From the File menu, choose Properties.

  4. In the "Properties for MAPIVIM.DLL" dialog box, you should see a line that reads 

          Version: 1.0.0.xx

    where xx is a number.

If the version is earlier than 1.0.0.63, or if the date of the file is earlier than 1/14/94, you will probably want to obtain a later version of the MAPIVIM.DLL file.
