MXL4: Only Part of Cell's Contents Printed with Wrap-Text

 Last reviewed: February 2, 1998
Article ID: Q108906
The information in this article applies to:
  • Microsoft Excel for the Macintosh, version 4.0

SYMPTOMS

Under certain circumstances, when a cell is formatted with the text- wrapping attribute, only the last word is printed in the cell.

In print preview, the document is displayed properly until zoomed. When zoomed or printed, some cells formatted with the wrap-text attribute display only the last word in the cell.

CAUSE

This behavior occurs when the text in a cell is exactly equal to the column width; that is, when the number of characters in the cell and the number in the Format Column Width dialog box are the same.

WORKAROUND

Add a space to the end of any cell that is exhibiting this behavior. This extra space should resolve inconsistencies between the wrapping criteria used by the print driver and the wrapping criteria used by Microsoft Excel.

STATUS

This is a known problem that occurs in Microsoft Excel 4.0 for the Macintosh when you use certain printer drivers.

MORE INFORMATION

Example

To see an example of the problem, follow these steps:

  1. Create a new worksheet. (On the File menu, click New. Select Worksheet in the dialog box.)

  2. On the Format menu, click Style. When the dialog box appears, click Define.

    A new dialog box is displayed.

  3. Click Change Font, and change the font to Helvetica 10-point. Click OK.

  4. Click Change Alignment, and select the Wrap Text check box. Click OK.

  5. Click OK to accept changes in the current style.

  6. Click the column label of column A to select the entire column.

  7. On the Format menu, click Column Width. Set the column width to 30 and click OK.

  8. In cell $A$1, type the following text exactly as it appears (including the period at the end of the sentence): 

          From power on to first user traffic passed.

  9. On the File menu, click Print Preview. Click Zoom.

Only the word "passed" is visible in cell $A$1.
