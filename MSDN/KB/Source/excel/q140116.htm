XL7: Page Fault in Module Excel.exe with Quattro Pro File

 Last reviewed: April 24, 1997
Article ID: Q140116

The information in this article applies to:
  • Microsoft Excel for Windows 95, version 7.0

SYMPTOMS

In Microsoft Excel for Windows 95, you may receive an illegal operation error when you re-open a Quattro Pro version 5.0 for Windows file (a file with the .wb1 extension) that you previously saved in Microsoft Excel. If you click Details, a message similar to the following will appear: 

   EXCEL caused an invalid page fault in module EXCEL.EXE at 0137:500513b1.

CAUSE

This problem occurs when both of the following conditions are true:

  • You use Microsoft Excel to open a Quattro Pro (.wb1) file that contains a worksheet with a reference to any worksheet other than the first one in the file.

    -and-

  • You save the file in the Microsoft Excel Workbook (.xls) format, close the file, and reopen it in Microsoft Excel.

The problem does not occur if the Quattro Pro file contains references that only refer to the first worksheet the notebook.

WORKAROUND

To work around this problem, do the following:

  1. Open the Quattro Pro file in Microsoft Excel and save it in the Lotus 1-2-3 (.wk4) format. Note that you must open the original Quattro Pro file.

  2. Close the file.

  3. Open the file and save it in the Microsoft Excel Workbook (.xls) format.

    NOTE: When you reopen the file, cells that contain data are formatted with fuchsia colored shading.

  4. To remove the fuchsia shading, select the cells that contain data, click Cells on the Format menu, click Patterns and click None under Color.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Microsoft Excel for Windows 95, version 7.0a, and in Microsoft Excel 97 for Windows.
