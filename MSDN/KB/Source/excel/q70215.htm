Item Limitation for Custom Data Forms

 Last reviewed: November 3, 1994
Article ID: Q70215
The information in this article applies to:
  • Microsoft Excel for Windows, versions 2.1 and 3.0

SUMMARY

When creating a custom data form, if the number of rows or items in the definition table exceed 64, you will receive a "Data Form Not Correct" error message when you try to use the Data menu's Form command. The only workaround in this case is to reduce the number of rows in the custom data form definition table to 64 or less.

MORE INFORMATION

A single custom data form can contain:

  1. 64 items

  2. 32 items that can take or return arguments

  3. 1,024 text characters

If these limitations are exceeded, the error message "Data_Form Not Correct" will display in an alert box.

Additional causes of the "Data_Form Not Correct" error message can include:

  1. Invalid numbers in the cells for item numbers, field coordinates, height, or width.

  2. Data form definition is less than seven columns wide.

  3. The cell range in the Refers To box (available by choosing Define Name from the Formula menu) may be incorrect.
