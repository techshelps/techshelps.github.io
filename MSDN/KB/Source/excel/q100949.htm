XL: Default Settings Not Used when Embedding Excel Object

 Last reviewed: February 2, 1998
Article ID: Q100949

The information in this article applies to:
  • Microsoft Excel for Windows, versions 4.0, 4.0a, 5.0
  • Microsoft Excel for the Macintosh, version 4.0

SUMMARY

In Microsoft Excel, default settings, such as the default font, can be changed for worksheets, charts, and macro sheets by creating a default template (SHEET.XLT, CHART.XLT, or MACRO.XLT).

However, when you embed a worksheet, chart, or macro sheet object in an application that supports object linking and embedding (OLE), the original default template is used instead of the template that you created.

For example, if you change the default font in Microsoft Excel for Windows from MS Sans Serif 10 point to Arial 10 point, an embedded worksheet object in Microsoft Word for Windows will still use MS Sans Serif 10.

WORKAROUND

In Excel for Windows, adding a FONT= line to the [Microsoft Excel] section of the EXCEL4.INI (EXCEL5.INI, if using Microsoft Excel version 5.0) file will change the default font for embedded Excel worksheets, charts and macro sheets in other applications.

Example:

FONT=Arial,10
