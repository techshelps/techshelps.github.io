MSQuery: Error Occurs Editing Paradox 5.x Timestamp Field

 Last reviewed: September 12, 1996
Article ID: Q141781
The information in this article applies to:
  • Microsoft Query for Windows 95, version 2.0

SYMPTOMS

The following error may occur when you work with a Paradox 5.x database in Microsoft Query 

   Numeric value out of range on column number <column#>(<column name>)
where <column#> is the number of the column, and <column name> is the name of the affected column.

CAUSE

These symptoms occur when you perform one of the following actions on a Timestamp field:

- Include a Timestamp field that contains values to the result set of a 

  query.

   -or-
- Add a new value or record to an empty Timestamp field.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft Paradox ODBC Driver version 2.99.1731. This problem was corrected in the Microsoft Paradox Driver version 3.00.2111.

MORE INFORMATION

Version 3.40.22204 of the ODBC drivers is shipped with Microsoft Office Professional for Windows 95 and Microsoft Access for Windows 95, version 7.0. Version 3.40.22204 of the ODBC drivers is also available in the Application Note "Microsoft ODBC Drivers for 32-bit Programs" (GE1263). For information about how to obtain this Application Note, please see the following article(s) in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q140255
   TITLE     : GE1263: Microsoft ODBC Drivers for 32-Bit Programs
