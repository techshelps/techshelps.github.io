Rounding Inconsistencies Fixed with Excel Version 3.00

 Last reviewed: November 2, 1994
Article ID: Q68630

SUMMARY

You may experience an inconsistency when Excel versions 2.1x round your three-digit decimal number to a two-digit decimal number, by using the Format Number command and selecting 0.00. For example, enter the following into an Excel 2.x worksheet (make sure your formatting is set to General):

  1. Type each of these in a single cell: 

          =2993*.285      The answer will be 853.005
      =6993*.285      The answer will be 1993.005

  2. Highlight the two results.

  3. From the Format menu, choose Number, and select the 0.00 option.

  4. The final result will be: 

        853.00
    1993.01

This causes problems when you need to balance particular money figures.

Note: Selecting Precision As Displayed will not change the answer.

MORE INFORMATION

This problem does not occur in Excel 3.00. Follow the same steps above in an Excel 3.00 worksheet and notice the final result: 

    853.01
   1993.01
