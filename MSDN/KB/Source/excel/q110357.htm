MSQuery 1.0: Paradox 4.x File Format Not Supported

 Last reviewed: October 6, 1997
Article ID: Q110357
1.00 WINDOWS kb3rdparty kbusage

The information in this article applies to:

  • Microsoft Query, version 1.0
  • Microsoft Excel for Windows, version 5.0

SYMPTOMS

If you attempt to use the Paradox ODBC driver that ships with Microsoft Query version 1.0 to connect to a Paradox for MS-DOS 4.x file or Paradox for Windows 1.x file, the ODBC driver will not be able to properly recognize the file format. You may be prompted to enter a password for the file, even if one has not been set, and you will not be able to connect to the file.

CAUSE

Microsoft Query version 1.0 does not currently ship with Open Database Connectivity (ODBC) drivers for Borland International's Paradox 4.x for Windows or MS-DOS. Because of this lack of an appropriate driver, you may experience unexpected behavior if you try to use the Paradox ODBC driver that ships with Microsoft Query to connect to a Paradox 4.x file.

WORKAROUND

To connect to a file created by Paradox for MS-DOS 4.x, use either of the following methods.

Method 1: Use the Paradox 3.x and 4.x driver that is available in the ODBC 

          Driver Pack 2.0. For more information about this Driver Pack, see
          the following article in the Microsoft Knowledge Base:

             ARTICLE-ID: Q125235
             TITLE     : INF: ODBC Driver Pack 2.0 Fulfillment Kit Q&A

          To obtain the ODBC Driver Pack 2.0, call the Microsoft Order Desk
          at (800) 360-7561 and request the ODBC Driver Pack 2.0
          Fulfillment Kit. If you are outside the United States, contact
          the Microsoft subsidiary for your area. To locate your
          subsidiary, see the Microsoft World Wide Offices Web site at:

             http://www.microsoft.com/worldwide/default.htm
Method 2: Save the file in the dBASE 3 or dBASE 4 format using File Export 
          in Paradox. Then use the dBASE ODBC driver that ships with
          Microsoft Excel to access the file. For Paradox for Windows 1.x,
          open the file in Paradox and save it in the Paradox 3.5 file
          format. The Paradox for MS-DOS 3.5 file format is supported by
          the Paradox ODBC driver that ships with Microsoft Query.
