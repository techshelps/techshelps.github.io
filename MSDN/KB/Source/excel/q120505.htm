XL5: Macintosh ODBC SQL Server Doesn't Support Sybase

 Last reviewed: February 2, 1998
Article ID: Q120505
The information in this article applies to:
  • Microsoft Query for the Macintosh, version 1.0
  • Microsoft Excel for the Macintosh, version 5.0
  • Microsoft Excel for the Power Macintosh, version 5.0

SUMMARY

You may experience problems when you access Sybase SQL Server (any version) using Microsoft Query for the Macintosh and the SQL Server ODBC driver supplied with Microsoft Excel version 5.0. Specifically, your computer may stop responding (hang) or it may display various error messages.

The ODBC SQL Server driver that is included with Microsoft Excel supports Microsoft SQL Server for Windows NT or Microsoft SQL Server for OS/2. It does not support Sybase SQL Server for UNIX.

Page 441 of the "Microsoft Excel User's Guide," version 5.0, implies that Microsoft Excel supports Sybase 4.x. This information is not correct in all situations.

MORE INFORMATION

The ODBC SQL Server driver was developed and tested using Microsoft SQL Servers for NT and OS/2, and was not tested with any Sybase SQL Servers.

To obtain a driver for a Sybase SQL Server, contact any of the following:

  • Sybase Software at (510) 596-3333
  • Intersolv at (919) 859-2220

    NOTE: Intersolv (formerly Q+E/Pioneer) produces the following ODBC drivers for the Macintosh: SYBASE, ORACLE, Ingress, and single-tiered ISAM drivers such as dBASE and Fox.

The Sybase SQL Server driver is manufactured by Sybase Software, a vendor independent of Microsoft; we make no warranty, implied or otherwise, regarding this product's performance or reliability.
