Excel: Rotated Text in Automatic Sized Text Box Doesn't Print

 Last reviewed: November 29, 1994
Article ID: Q86628
The information in this article applies to:
  • Microsoft Excel for Windows, version 4.0

SYMPTOMS

In version 4.0 of Microsoft Excel for Windows using the Windows 3.0 Hewlett-Packard (HP) LaserJet III driver, versions 30.3.85 and 30.3.86, text rotated downwards in an automatically sized text box will not print.

More Information:

In Excel 4.0, selecting Automatic Size in the Format Text dialog box will cause a text box to size in proportion to its contents. When this option is selected along with the text rotated in the downward direction, the text will preview okay but will not print to a HP LaserJet III printer using the printer drivers listed above.

STATUS

This problem does not occur with versions of the driver earlier than 30.3.85. Directly formatting the text to be one of the Windows 3.0 vector fonts, Modern, Roman or Script, will allow the text to print correctly.

Microsoft has confirmed this to be a problem in the HP LaserJet III printer driver versions 30.3.85 and 30.3.86.

This problem does not occur in Windows version 3.1 with the HP III driver version 31.3.89.

REFERENCES

"Microsoft Excel User's Guide 1," version 4.0, page 232
