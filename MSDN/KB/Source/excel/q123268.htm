XL5: Formula in WK4 File Displays #VALUE! Error Value

 Last reviewed: September 12, 1996
Article ID: Q123268
The information in this article applies to:
  • Microsoft Excel for Windows, versions 5.0, 5.0c

SYMPTOMS

In Microsoft Excel, when you open a Lotus 1-2-3 WK4 file, a cell on the worksheet may display the #VALUE! error value even though the formula contained in the cell is displayed correctly in the formula bar.

CAUSE

This behavior occurs when the formula is converted properly (displays the correct formula in the formula bar), but initially evaluates to the #VALUE! error value. For example, if your Lotus 1-2-3 WK4 file contains the following formula 

   @@(A1:A1)
where A1 contains the formula @@COORD(1,2,1,8)&" "&@@COORD(1,2,1,8) and cell A2 is blank, the cell that contains the formula displays the #VALUE! error value when you open the file in Microsoft Excel.

WORKAROUNDS

To display the correct value in a cell that displays the #VALUE! error value under the above conditions, use the appropriate method below.

Method 1: To display the value in an individual cell: 

          a. Select the cell that contains the formula that is displaying
             the #VALUE! error value.

          b. Press F2.

          c. Press ENTER.
Method 2: To display the value in multiple cells: 

          a. Select the worksheet that contains the #VALUE! Error values.

          b. From the Edit menu, choose Replace.

          c. In the Find What box, type "=" (without the quotation marks).
             In the Replace With box, type "=" (without the quotation
             marks), and choose Replace All.
