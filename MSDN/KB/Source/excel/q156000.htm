XL97: Additional Controls Dialog may not use Long Filenames

 Last reviewed: February 18, 1998
Article ID: Q156000
The information in this article applies to:
  • Microsoft Excel 97 for Windows

SYMPTOMS

When you use Microsoft Visual Basic for Applications in Microsoft Excel 97, the Additional Controls dialog box (located on the Tools menu) may not use long file names to display the location of certain controls. Although short file names display, all controls function normally.

CAUSE

Microsoft Excel reads the path information directly from the Windows Registry. Many of the controls are installed during the initial Microsoft Windows installation (such as WordPad or Bitmap Image) and therefore do not use long file names.

MORE INFORMATION

The Additional Controls dialog displays a list which allows you to add controls or insertable objects to the Toolbox.

REFERENCES

For more information about the Additional Controls dialog, click the Index tab in Microsoft Help, type the following text: 

   Additional Controls
and then double-click the selected text to go to the "Additional Controls dialog box" topic.
