SBFM97: Setup Could Not Find Microsoft Excel Installed

 Last reviewed: January 9, 1998
Article ID: Q170613
The information in this article applies to:
  • Microsoft Excel 97 for Windows
  • Microsoft Small Business Financial Manager 97 for Microsoft Excel 97
  • Microsoft Windows NT Workstation version 4.0
  • Microsoft Windows NT Server version 4.0

SYMPTOMS

When you run the Setup program for the Microsoft Small Business Financial Manager 97 for Microsoft Excel 97, the following error message appears: 

   Setup could not find Microsoft Excel 97 installed
   on your system. Please install Microsoft Excel 97
   before installing this application.
When you click OK, the Setup program does not continue.

CAUSE

This problem occurs when you install Microsoft Small Business Financial Manager 97 on a computer running Microsoft Windows NT version 4.0.

WORKAROUND

To work around this problem, register Microsoft Excel 97 by using one of the following methods.

Reinstall Microsoft Excel

Depending on which program is installed, run the Setup program for Microsoft Office 97 or Microsoft Excel 97. During this maintenance mode installation, click Reinstall when you are prompted for the type of installation.

Rebuild the Registry Settings

To rebuild the original Microsoft Office or Microsoft Excel registry entries, use the following command 

   <drive>:\setup /y
where <drive> is the drive that contains the Microsoft Office Setup disk. This parameter forces the Setup program to use maintenance mode. When you are prompted for the type of installation, click Reinstall. The Setup program does not copy any files during this kind of installation. Instead, it updates the registry based on the previous installation.

NOTE: the /y switch restores all of the Microsoft Office program defaults.
