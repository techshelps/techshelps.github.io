XL: Spinner Is Not Printed Exactly as It Appears on the Screen

 Last reviewed: February 5, 1998
Article ID: Q176005

The information in this article applies to:

  • Microsoft Excel for the Macintosh, versions 5.0, 5.0a
  • Microsoft Excel 98 Macintosh Edition

SYMPTOMS

When you print a worksheet, dialog sheet, or UserForm that contains a Spinner control, the following problem may occur:

  • The tip of each arrow in the control is "capped" by a small square. As a result, the arrow is not tapered to a fine point.
  • In Microsoft Excel 98 Macintosh Edition, the horizontal divider line in the Spinner control is not printed and does not appear in print preview.

    NOTE: There are no divider lines in Spinner controls in Microsoft Excel 5.0 for the Macintosh.

Note that this problem does not occur when you print a Spinner control in Microsoft Excel for Windows.

MORE INFORMATION

To see an example of this problem, follow these steps:

  1. In Microsoft Excel for the Macintosh, create a new workbook.

  2. Display the Forms toolbar. To do this in Microsoft Excel 5.0, click Toolbars on the View menu, click the Forms check box, and then click OK. In Microsoft Excel 98 Macintosh Edition, point to Toolbars on the View menu, and then click Forms.

  3. Click the Spinner tool on the Forms toolbar and draw a large spinner on the worksheet.

  4. On the File menu, click Print.

The printed Spinner arrows are capped by small squares (the bigger the Spinner control, the more noticeable the squares). In Microsoft Excel 98 Macintosh Edition, the horizontal divider line is not printed.

If you perform these steps in Microsoft Excel for Windows, the arrows are not printed with small squares at the top, and the divider line is printed.
