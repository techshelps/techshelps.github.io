Excel: UAE with Bitstream FaceLift and File Page Setup

 Last reviewed: November 1, 1994
Article ID: Q78341

SUMMARY

Bitstream FaceLift versions 1.x produce an Unrecoverable Application Error (UAE) when you choose Page Setup from the File menu in Microsoft Excel version 3.0 if both of the following conditions are met:

  1. You chose Printer Setup from the File menu and selected either the IBM Graphics or the Toshiba P351 as your printer.

  2. The printer selected is using FaceLift Fonts.

MORE INFORMATION

Steps to Reproduce Problem

To do the workaround, it is necessary to understand the steps to reproduce the problem. The workaround follows the "Steps to Reproduce Problem" section.

  1. Run the Bitstream FaceLift program.

  2. Select the Printers icon from within the Bitstream FaceLift desktop.

  3. Select either the IBM Graphics printer or the Toshiba P351 under Installed Printers.

  4. Select Assign Shell-Driver. An asterisk is added to the left of the selected printer.

  5. Choose OK to close the Printers dialog box.

  6. Start Excel and choose Page Setup from the File menu.

At this point a UAE will occur.

Workaround

According to Bitstream Technical Support, the only workaround for this problem is the following:

  1. Run the Bitstream FaceLift program.

  2. Select Unassign The Shell-Driver (opposite of step 4 above).

  3. Return to Microsoft Excel and choose Page Setup from the File menu and make any desired modifications.

  4. Return to FaceLift and select Assign Shell-Driver (same as step 4 above).

  5. Return to Microsoft Excel.

MORE INFORMATION

Step 4 in the "Steps to Reproduce Problem" section allows Windows to use FaceLift fonts for a particular printer. FaceLift fonts are sent to the printer by means of a shell driver, which intercepts printer commands that normally go to the printer driver.

Bitstream is aware of this problem, and problems should be reported to Bitstream technical support by calling (617) 497-7514.
