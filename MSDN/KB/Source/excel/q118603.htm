XL5: GP Fault Using Paste Name Dialog Box in Print Area

 Last reviewed: September 12, 1996
Article ID: Q118603
The information in this article applies to:
  • Microsoft Excel for Windows, version 5.0

SYMPTOMS

In Microsoft Excel, if you use the Paste Name dialog box (by pressing F3) to enter a defined name in the Print Area box in the Page Setup dialog box, you may either receive

- a general protection (GP) fault 

  -or-
- the error message "Margins do not fit page size."

CAUSE

This error occurs when you select the Margins tab in the Page Setup dialog box, and you then select another tab in the Page Setup dialog box, after using the Paste Name dialog box (by pressing F3) to enter the defined name. This is not a problem when you type a defined name in the Print Area box, or when you use the Insert menu to paste the name. Also, this problem does not occur after you choose OK in the Page Setup dialog box.

For more information about using a defined name in the Print Area box, query on the following words in the Microsoft Knowledge Base: 

   print and area and names and dynamic

WORKAROUND

To avoid receiving the GP fault, do either of the following:

  • Insert the defined name by using the following steps:

    1. Click the Print Area box in the Page Setup dialog box.

    2. From the Insert menu, choose Name, and then choose Paste.

    3. From the Paste Name list, select the defined name that you want to 

          insert and choose OK.

    -or-

  • Insert the defined name by typing the name in the Print Area box instead of using the Paste Name dialog box.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Microsoft Excel for Windows version 5.0c.

REFERENCES

For more information about Pasting A Name Into A Formula, choose the Search button in Help and type: 

   names, pasting into formulas
