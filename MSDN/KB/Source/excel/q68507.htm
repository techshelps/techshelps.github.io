Only Vector Fonts Change Size When Copied Pictures Are Sized

 Last reviewed: November 3, 1994
Article ID: Q68507
The information in this article applies to:
  • Microsoft Excel for Windows, versions 3.0, 4.0

SUMMARY

When you move any item from Microsoft Excel into Word for Windows using the Edit Copy Picture command, and you scale the object using the Microsoft Word Format Picture command, the text in the copied item should be formatted using the Modern, Script, or Roman font. Otherwise, it will not change in size.

This applies to Microsoft Excel charts or sections of worksheets.

MORE INFORMATION

In Microsoft Excel, wWhen an image is put on the Clipboard using the Copy Picture command of the Edit menu, the lines, arcs, shading, etc., in the image are copied as a graphics metafile; the text in the copied selection is copied as a text record in the metafile. When the metafile is printed, all items in the metafile are printed scaled. However, due to the way that Windows metafiles handle text, a font of a given point size will still be printed at that point size -- no matter how it is scaled. Printing an object that contains text at a reduced scale will result in "squished" text; the letters will be closer together, but no smaller.

However, the Modern, Roman, and Script fonts are themselves composed of metafiles, one metafile for each character. These metafiles are scaled in the same manner as other graphic objects; thus, when they are scaled up or down, they change size appropriately.
