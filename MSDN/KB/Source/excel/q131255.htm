Cannot Add Criteria to Memo Field Using Access 2.0 ODBC Driver

 Last reviewed: April 23, 1997
Article ID: Q131255
The information in this article applies to:
  • Microsoft Query for Windows, version 1.0

SYMPTOMS

If you use the Access 2.0 driver (version 2.00.2317) and try to add the memo field to the criteria using the Add Criteria dialog box, you will receive the following error message: 

   Column <field name> can't be used with the operator <operator name>
And, when you select the memo field from the Criteria Field drop-down box in the Criteria Pane, you will receive this error message: 

   Column <field name> can't be used in the criteria

WORKAROUND

To avoid these error messages, add the memo field to the criteria by dragging the memo field from the Table pane and dropping it in the Criteria pane. Once the field has been added to the criteria this way, you can enter the value for the memo field directly in the Criteria pane.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Query version 1.0 for Windows. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
