Excel: Macro Error at NEW Command in Startup Macro

 Last reviewed: November 4, 1994
Article ID: Q80486
The information in this article applies to:
  • Microsoft Excel for the Macintosh, versions 3.0, 4.0, 5.0

SUMMARY

In the versions of Microsoft Excel listed above, templates and macrosheets in the Excel Startup Folder are recognized in alphabetical order.

As each template is recognized, it's added to the list of document types available when New is chosen from the File menu. As each macrosheet is recognized, any Auto_Open macros are executed.

If an Auto_Open macro on one of these macrosheets attempts to create a new document based on a template which occurs later in alphabetical order, the macro will halt with the message: 

   Macro error at cell
To avoid this problem, make sure that template filenames alphabetically precede the filenames of the Auto_Open macros that use them. This will assure that the templates are recognized prior to any Auto_Open macros which may refer to them.

NOTE: Most of the SHIFT characters above numbers on a keyboard (!@#$%&*) when placed at the beginning of a filename will alphabetize that filename before filenames beginning with a letter. The caret (^) and the tilde (~) characters alphabetize files after the letter Z.
