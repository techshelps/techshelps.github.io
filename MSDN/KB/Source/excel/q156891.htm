XL97: Undo Does Not Always Restore Previous Value

 Last reviewed: February 12, 1998
Article ID: Q156891
The information in this article applies to:
  • Microsoft Excel 97 for Windows

SYMPTOMS

After you use the Undo command, a cell value that you changed may not return to it previous value.

CAUSE

This behavior occurs when the cell contains dependencies and/or formulas that require recalculations. This behavior may occur in the following cases:

Case 1

When a cell is dependent on other cell for a value, Microsoft Excel may not update the dependency dynamically until it is forced to do so. Undoing an entry that is a precedent to another cell does not force the link to update to its previous value.

Case 2

When you turn on manual calculation, link one cell to another, and then use the Undo command, Microsoft Excel does not force the link to update to its previous value.

Case 3

When a cell contains a formula that uses a function that refers to another cell (for example, the RAND function), the cell does not revert back to its previous value when you use the Undo command to undo the value of its precedent.

MORE INFORMATION

Microsoft Excel 97 allows 16 levels of undo operations. You can reverse an action by clicking Undo on the Edit menu or by clicking the Undo button on the Standard toolbar.
