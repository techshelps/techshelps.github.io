PC WRmt Ext: Err Msg: Postoffice Has Not Accepted Mail Item

 Last reviewed: June 5, 1996
Article ID: Q127001
The information in this article applies to:
  • Microsoft Mail Remote for Windows, version 3.2

SYMPTOMS

When Mail Remote for Windows has mail queued up and dials into the External machine at the postoffice, the following error may be displayed after a connection has been established: 

   Postoffice has not accepted mail item
External will receive the message in its entirety, but will not process it. It will log the following error in the SESSION.LOG file on the External postoffice. 

   Unable to send message text

CAUSE

This error may be displayed when a mail message contains more than 35 attachments.

STATUS

Microsoft has confirmed this to be a problem in EXTERNAL.EXE version 3.2a, included with Microsoft Mail for PC Networks. This problem was corrected in EXTERNAL.EXE version 3.2.16. If you do not have version 3.2.16 (or later), you can find EXTUPD.EXE, a self-extracting file, on the following services:

  • Microsoft's World Wide Web Site on the Internet

      Click Knowledge Base, and select the product.

          Enter kbfile EXTUPD.EXE (size: 531665 bytes) 
                             , and click GO!
      Open the article, and click the button to download the file.


  • Internet (anonymous FTP)

          ftp ftp.microsoft.com
      Change to the Softlib/Mslfiles folder.
      Get EXTUPD.EXE (size: 531665 bytes)

MORE INFORMATION

 


The External program will also generate this error when receiving messages
with 35 or more attachments from users on another postoffice connected via
modem.
