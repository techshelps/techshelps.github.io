X400: Slow Message Transfer

 Last reviewed: March 9, 1998
Article ID: Q111946
3.00 3.20 MS-DOS kb3rdparty kbfile kbbug3.00 kbbug3.20 kbfix

The information in this article applies to:

  • Microsoft Mail Gateway to X.400, versions 3.0 and 3.2

SYMPTOMS

Microsoft Mail Gateway to X.400 may process messages slowly.

CAUSE

Slow message transfer may result from a large (4000 or more names) custom address map. The X.400 gateway reads the custom address map file (MAPTBL.OUT) in 1K blocks. If a large custom address map exists, it takes time to read the entire file. Other X.400 gateway read activity is handled in 4K blocks.

RESOLUTION

Microsoft has made two changes to the X.400 gateway to increase the search speed when you use a custom mapping table:

  • The algorithm used to search the custom mapping table has been changed to search first on the X.400 O/R attribute most likely to be unique (the surname attribute).
  • A parameter is now available in the X400GATE.INI file that allows you to move the mapping table to the local gateway computer. The table can then be loaded into a RAMDrive for quick access. You should add the new parameter to the [Microsoft X.400 gateway] section, following the example below: 

          [Microsoft X.400 gateway]
      LocalCustomMappingFile=c:\maptbl.out

For more information about the custom mapping table, please see the X.400 Gateway "Administrator's Guide." For more information about the RAMDrive program, please see your Microsoft MS-DOS documentation.

STATUS

Microsoft has confirmed this to be a problem in versions 3.0 and 3.2 of Microsoft Mail Gateway to X.400. This problem is corrected in X400GATE.EXE version 3.2.13. If you do not have version 3.2.13 (or later), you can download X400GA.EXE, a self-extracting file, on the following services:
