PC DB: How to Tell If a CONTROL.GLB File Is Corrupted

 Last reviewed: February 4, 1998
Article ID: Q87388

The information in this article applies to:
  • Microsoft Mail for PC Networks, versions 2.1e, 3.0, and 3.2

SUMMARY

The following are some tests to determine whether a CONTROL.GLB file is corrupt:

  1. The file size must be 8 bytes in length.

  2. The contents of the nmailbag and nletter fields do not look abnormally large. This is not necessarily a corruption that will cause the mail system not to work; instead, it is an indication that there was some file corruption in the database and that there may be future problems.

MORE INFORMATION

Offset  Size  Name                Description


0       4     long nmailbag       For generating user or gateway
                                  associated filenames
4       4     long nletter        For generating mail, attachment and
                                  P1 filenames
