1.00 MS-DOS

The information in this article applies to:

  • Microsoft Mail Gateway to 3Com 3+Mail, version 1.0

Please use the checklist in this article to collect information about your system before calling Microsoft Product Support Services. You will be able to work more effectively with a Microsoft Support Engineer if all the information needed to trace and solve your problem is available. Include any troubleshooting steps you have taken in an effort to resolve the problem. Please be as complete and specific as possible when you fill out this form.

Files needed

Microsoft Product Support Services may need to see these system files to isolate your problem. Extract them, send them electronically if possible:

  • GATEWAY.INI
  • \(GATEWAY)\LOGS\ERROR.LOG
  • \(GATEWAY)\LOGS\AUDIT.LOG
  • Are GATEWAY.EXE, COURGET.EXE, COURPUT.EXE in the FFAPI path location (see GATEWAY.INI)? YES/NO

Troubleshooting

  • Are trailing backslashes ending EVERY path reference in the 
       GATEWAY.INI file?   YES/NO
  • Are files left in \(GATEWAY)\3C-OUT? YES/NO
  • Are files left in \(GATEWAY)\MFFPUT? YES/NO
  • Are files left in \(GATEWAY)\MFFGET? YES/NO
  • Are files left in \P1? YES/NO
  • If you can purge those files (messages will be lost) and restart 
       Gateway, will it run?   YES/NO

Suggestions

  • Look through the 3Com Gateway snapshot for anomalies.
  • Browse for 3COM: articles in the Microsoft Knowledge Base.
  • Do a quick check for complete installation: use ADMIN.EXE, select GATEWAY. You should see SMTP address list. If you don't see SMTP, reinstall until you do.
  • Do a quick check for executables. They should all be in the GATEWAY directory created for install: 

        - COURGET.EXE
    - COURPUT.EXE
    - GATEWAY.EXE
    - GATEWAY.INI
  • If any EXEs are missing, copy them from the floppy. If the .INI is missing, reinstall.

3Com Conversion Utilities

If you are using Microsoft Mail 3.0, turn OFF enhanced security. Get disk one of Microsoft Mail Server software and run UPDATE.EXE. Say NO to enhanced and YES to remove it.
