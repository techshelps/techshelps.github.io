Setup Fails When You Install Lotus Notes Files

 Last reviewed: September 10, 1996
Article ID: Q151038
The information in this article applies to:
  • Microsoft Office for Windows 95, version 7.0
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft PowerPoint for Windows 95, version 7.0
  • Microsoft Word for Windows 95, version 7.0

SYMPTOM

When you install Office 7.0, the Setup program may fail after the program installs the Lotus Notes .dll files.

CAUSE

This problem occurs when the following conditions are met:

  • Your Lotus Notes folder (directory) exists on the server.

    -and-

  • You do not have write permission to the Notes folder on the server.

    -and-

  • You install Office 7.0.

The behavior occurs because you are running a shared installation of Lotus Notes and you have read-only access to the shared Lotus Notes folder. During installation, Setup detects Lotus Notes, and attempts to install four .dlls to the Lotus Notes directory. Because you do not have write permission, Setup cannot install the files, and Setup fails.

WORKAROUND

To work around this problem, have someone who has both read and write permissions to the Lotus Notes folder install Office 7.0. You can then install Office to your workstation successfully.

MORE INFORMATION

The Setup program attempts to install the following files: 

   _Msimp32.dll
   _Ixls.dll
   _Xxls.dll
   Mscthunk.dll
