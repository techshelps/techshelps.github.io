PRA: MS Office/Access CCP Does Not Detect Personal ORACLE 7.2

 Last reviewed: May 15, 1997
Article ID: Q161399
The information in this article applies to:
  • Microsoft Access 97
  • Microsoft Office Pro 97

SYMPTOMS

The Compliance Checking Program (CCP) in the upgrade versions of Microsoft Office 97, Professional Edition and Microsoft Access 97 fails to recognize Personal ORACLE 7.2 as an application that qualifies for the competitive upgrade program.

RESOLUTION

Call Microsoft Technical Support for assistance installing your upgrade version of Microsoft Access 97 or Microsoft Office 97 Professional Edition. To obtain the support phone number for your product, please visit Microsoft's "Support Options & Phone Numbers" Web page at the following address: 

    http://www.microsoft.com/supportnet/

STATUS

Microsoft has confirmed this to be a problem in Microsoft Access 97 and Microsoft Office 97, Professional Edition. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

If you own certain database applications, you qualify for the Microsoft Access competitive upgrade. That means you can buy an upgrade version of Microsoft Access at a reduced price. The Setup program checks for qualifying products before it installs Microsoft Access.

Personal ORACLE 7.2 is one of the products that qualifies for the competitive upgrade program. However, the Setup program fails to recognize it as such.
