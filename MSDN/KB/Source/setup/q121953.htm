Office: Error Installing to Diskless Workstation

 Last reviewed: September 9, 1996
Article ID: Q121953
The information in this article applies to:
  • Microsoft Office for Windows, version 4.0

SYMPTOMS

When you run the Microsoft Office Setup program and you install the files to a diskless workstation, you receive the following error messages: 

   Not enough space on drive C:.
   The following options have been selected for installation on drive C:\

   Options:
   Microsoft Office Shared Components

   Space required on drive C:  4K
   Space available on drive C: 0K
   Space needed on drive C:    4K
If you choose the Change Options button, or the Install Now button, you receive one or both of the following error messages:
  • Could not create C:\AUTOEXEC.TMP
  • Not enough memory on Drive C:

CAUSE

This behavior occurs because the Microsoft Office Setup program creates a file called AUTOEXEC.TMP on drive C. It does this because the Setup program adds a line to the AUTOEXEC.BAT file to load the SHARE.EXE program. If the Setup program cannot create the AUTOEXEC.TMP file on the C drive, you receive the error message described above.

Note that when you receive the AUTOEXEC.TMP error message, Setup has already copied and updated all of the necessary files. You can complete the remaining Setup tasks by manually creating Program Manager groups and icons for the Microsoft Office applications. As long as the SHARE.EXE program is loaded on your computer, the Microsoft Office applications function correctly.

For additional information, please see the following article(s) in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q106499
   TITLE     : XL5 Err Msg: "Must Exit Windows and Load SHARE.EXE
You do not receive an error message when you perform an administrative installation (setup /a) to a diskless workstation.

WORKAROUND

To complete the Setup program and to avoid receiving this error message, you can map drive C to a valid drive that contains the file AUTOEXEC.BAT.
