Need Read Permission to Urlcache Directory

 Last reviewed: October 6, 1997
Article ID: Q170855
The information in this article applies to:
  • Microsoft Proxy Server versions 1.0 and 2.0

SYMPTOMS

The browser may return the following error message when it goes through the Proxy Server: 

   The requested header was not found.

CAUSE

When you use the web proxy with caching, the cache is kept in the Urlcache directory. If you do not have Read permission to this directory then the web proxy returns the above error because it cannot access the cache.

WORKAROUND

Make sure all users have Read permission to this directory through the IUSR_machinename security credentials.
