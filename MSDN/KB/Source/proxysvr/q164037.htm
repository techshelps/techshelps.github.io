IIS and Proxy Are Dependent on Each Other

 Last reviewed: October 6, 1997
Article ID: Q164037
The information in this article applies to:
  • Microsoft Proxy Server versions 1.0 and 2.0

SYMPTOMS

When you stop the WWW service after you stop the Web Proxy service (or vice versa), you will get the following error message: 

   The service has not been started.
-or-

When you start the WWW service after you start the Web Proxy service (or vice versa), you get the following error message: 

   An instance of the service is already running.

CAUSE

If Proxy Server is installed, stopping or starting either the Web Proxy service or the WWW service causes the other service to also be stopped or started.

MORE INFORMATION

When either the Web Proxy service or the WWW service is stopped or started, the other service is also stopped or started. However, the Internet Service Manager (ISM) does not report the other service as being stopped or started. To refresh the ISM screen, press the F5 key. ISM will then display both services as stopped or started.
