Proxy Server Fails if IP is Installed but Not Bound

 Last reviewed: October 6, 1997
Article ID: Q164029
The information in this article applies to:
  • Microsoft Proxy Server versions 1.0 and 2.0

SYMPTOMS

When you use Chkwsp32.exe to test a connection between a WinSock client, and the Proxy Server using IPX, Chkwsp32 reports the following message: 

   Server is Down.

CAUSE

TCP/IP is installed on the client computer but is disabled on the internal network card. For example, if IP is installed for a Dial Up Adapter, but not bound to the network card.

WORKAROUND

Do one of the following:

  • Enable TCPIP on the client's network adapter along with IPX.
  • Edit the Mspclnt.ini file on the server and remove the Servers IPAddress section, then update the client computer in Control Panel, WSP Client.
  • Completely remove TCPIP from the client computer.
