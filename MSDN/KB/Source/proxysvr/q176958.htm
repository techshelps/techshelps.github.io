Socks Port Number Always Shows as Zero (0) in Permission List

 Last reviewed: December 17, 1997
Article ID: Q176958

The information in this article applies to:

  • Microsoft Proxy Server version 2.0

SYMPTOMS

Socks permissions will be displayed incorrectly as port 0 in the Socks permissions list.

CAUSE

This problem is caused by a bug in the configuration to user interface translation that is done by the Socks service. The port number is incorrectly displayed but the actual Socks permission is unaffected by this problem.

RESOLUTION

The original port number defined when the permission was created is still used by the Socks service. To confirm which port numbers are defined, open the Socks.cfg file located in the \Msp directory with a text editor, such as Notepad. The Socks permissions can be edited in the user interface or in the Socks.cfg file.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Proxy Server version 2.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
