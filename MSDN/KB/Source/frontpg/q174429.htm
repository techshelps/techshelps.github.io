FP98: Refresh Does Not Update Shared Borders w/ Multiple Users

 Last reviewed: March 18, 1998
Article ID: Q174429
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows

SYMPTOMS

When multiple users are editing a single Web, refreshing an edited page does not update Shared Borders.

CAUSE

The insertion point is located in the main section of the page. Refreshing the page will only show changes to this portion of the page.

RESOLUTION

To see the updated shared border, follow these steps:

  1. Place the insertion point within a shared border.

  2. Press F5 to refresh the page.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article.
