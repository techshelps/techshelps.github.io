FP98: Form Name Overwritten if Field Validation is Specified

 Last reviewed: March 18, 1998
Article ID: Q174427
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows

SYMPTOM

When you apply validation to a form field, FrontPage overwrites the custom form field name you entered for the form field.

CAUSE

When you apply field validation to a form field, FrontPage automatically assigns a predefined name to the form to insure that the name being used is unique. These names are passed to the FrontPage Server Extensions at browse- time. Note that if you apply validation to a form field, you will not be able to assign a custom name to the form.
