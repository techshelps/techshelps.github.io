FP1: Two or More Minute Delay Opening FrontPage Web on IIS

 Last reviewed: March 17, 1998
Article ID: Q154681
The information in this article applies to:
  • Microsoft FrontPage for Windows, version 1.1

SYMPTOMS

When you open a FrontPage Web from an author-level account on a Microsoft Internet Information server, you may encounter a delay of two minutes or more.

NOTE: This delay is unrelated to the size of the Web you are opening. The same Web can be opened from an administrator-level account without delay.

CAUSE

When the author-level account used to log on to the Web is not a member of any group in Windows NT, the server needs more time to process the security identifier for that account.

RESOLUTION

To resolve this problem, add the FrontPage author account to a Windows NT group. (You can add the account to any group: that is, the group need not be granted any special rights, nor need it be granted any authoring permission to the Web content.)

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
