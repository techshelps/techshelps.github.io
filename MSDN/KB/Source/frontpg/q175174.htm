FP: Hyperlink Not Updated in HTML Source if Linked Page Renamed

 Last reviewed: March 18, 1998
Article ID: Q175174
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows
  • Microsoft FrontPage 97 for Windows with Bonus Pack

SYMPTOMS

In FrontPage Explorer, if you rename a page that is the destination of a hyperlink in a page that you have open in HTML view in FrontPage Editor, the hyperlink will not be updated. This behavior occurs even though you clicked Yes when FrontPage Explorer prompted you to update the pages so that the hyperlinks would not be broken.

CAUSE

This problem occurs because hyperlinks are not updated in HTML view in FrontPage Editor.

WORKAROUND

To work around this problem, use either of the following methods.

Method 1: Click the Normal Tab before You Rename a Linked File

To update the HTML source of the file that is open in FrontPage Editor, click the Normal tab before you switch to FrontPage Explorer and rename the file that is the destination of a hyperlink.

Method 2: Refresh the Page before You Edit the HTML Source Code

If you are viewing the HTML source in FrontPage Editor when you rename the file in FrontPage Explorer, do not modify the HTML code until you have refreshed the page. To do this, press F5 or click Refresh on the View menu before you make any changes to the HTML code.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article.
