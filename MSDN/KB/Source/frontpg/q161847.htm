FP: Permissions Change After Reinstalling Extensions

 Last reviewed: March 18, 1998
Article ID: Q161847
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows
  • Microsoft FrontPage 97 for Windows with Bonus Pack
  • Microsoft FrontPage for Windows, version 1.10

SYMPTOMS

When you uninstall and then reinstall the FrontPage Server Extensions from a Windows NT server running Microsoft Internet Information Server (IIS) or a Windows NT 4.0 Workstation running Peer Web Services or the Microsoft Personal Web Server on Windows 95, author and administrator accounts are reset to browse access (or end user access if you are using FrontPage version 1.1).

CAUSE

When you remove the FrontPage Server Extensions, the files and folders under the \_vti_bin directory are replaced. The permission settings that exist for the current root establish who can browse the Web. This information is created during the installation of the server extensions and is stored in the Shtml.dll file. However, the permission settings that pertain to the author and administrator accounts, which are stored in the Author.dll and Admin.dll files, are reset to allow browsing the Web.

WORKAROUND

FrontPage 97 and 98

To modify the permissions level of users or groups, follow these steps:

  1. Start FrontPage Explorer.

  2. On the Tools menu, click Permissions, and then click the Users tab.

  3. Select the account you want to modify, and then click Edit.

  4. On the Edit Groups tab, under User Can, select one of the following options: 

          Author And Browse This Web
      Administer, Author, And Browse This Web

  5. Click OK.

FrontPage 1.1

To modify the permissions level of users or groups, follow these steps:

  1. Start the FrontPage Explorer.

  2. On the Tools menu, click Permissions, and then click the End Users tab.

  3. Select the account you want to change, and then click Remove.

  4. Click the Administrators tab or the Authors tab.

  5. Click Add.

  6. In the New Author dialog box, type the author's name in the Name field. Type the author's password in the Password and the Confirm Password boxes. Click OK.

  7. Click OK.

NOTE: When you install the FrontPage 97 Server Extensions on IIS version 1.0 or the FrontPage version 1.1 server extensions on any version of IIS, the Shtml.dll, Author.dll, and Admin.dll are replaced by Shtml.exe, Author.exe, and Admin.exe, respectively.
