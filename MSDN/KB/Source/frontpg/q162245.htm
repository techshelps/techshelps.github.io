FP: IDC Queries Fail When Run From IIS 2.0, MSPWS 1.0 Servers

 Last reviewed: March 18, 1998
Article ID: Q162245
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows
  • Microsoft FrontPage 97 for Windows with Bonus Pack
  • Microsoft FrontPage for Windows, versions 1.0, 1.1

SYMPTOMS

When you issue a query from an Internet Information Server (IIS) version 2.0 server using the Internet Database Connector (IDC) and an Access database using Open Database Connectivity (ODBC), the IIS or Microsoft Personal Web Server stops responding (hangs).

On an IIS server, you receive the following error message in the Dr. Watson log: 

   The server returned an invalid or unrecognized response.
On the Microsoft Personal Web Server, you receive an illegal operation and the program quits. If you click the Details button when you receive the illegal operation error, the following error appears: 

   INETSW95 caused an invalid page fault in module KERNEL32.DLL.

CAUSE

Microsoft has confirmed this to be a problem with the Microsoft Jet 3.0 Msjt3032.dll file. This problem was corrected in version 3.00.3213 of the Msjt3032.dll file.

RESOLUTION

To correct this problem, upgrade the Msjt3032.dll file to version 3.00.3213 or later. To do this, download the MSjtwng.exe self-extracting archive from the Microsoft Software Library or one of the online services listed in the "Instructions for Downloading" section of this article.

For additional information, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q143163
   TITLE     : ACC95: Problems Running Microsoft Jet 3.0 on Windows NT
               4.0
