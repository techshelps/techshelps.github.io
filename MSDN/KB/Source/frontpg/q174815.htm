FP98: Setup Uses More Disk Space than Expected

 Last reviewed: March 18, 1998
Article ID: Q174815
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows

SYMPTOMS

When you install Microsoft FrontPage 98, you may notice a greater amount of disk space used than the 36 megabytes (MB) FrontPage requires for a Typical installation.

CAUSE

On a hard disk that uses the FAT file system, the smallest amount of space a file can use is equal to the size of one cluster. This means that a 1- kilobyte (KB) file stored on the hard disk may actually use up to 32 KB of physical disk space, rather than 1 KB of disk space per file. FrontPage installs a large number of small files (1 KB to 2 KB), including images (GIF and JPG) and HTML templates.

NOTE: This behavior does not occur on FAT32 file system or on NTFS file systems, because they do not have a 32-KB limitation.

MORE INFORMATION

FrontPage 98 requires 36 MB of free disk space for a Typical installation, but the program actually uses about 33 MB of physical disk space. On a Windows 95 computer using the FAT16 file system, FrontPage 98 will use about 55 MB,

The type of setup you choose will determine how much disk space will be needed to successfully install FrontPage. You will see the greatest impact on disk space if you choose a Custom installation and install the additional clip art images. For example, if you choose a Typical installation, FrontPage setup will install about 200 clip art images. However, if you choose a Custom installation and select the additional clip art option, FrontPage setup will install an additional 1200 clip art images.

For additional information about cluster sizes on a FAT file system, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q67321
   TITLE     : FAT Type and Cluster Size Depends on Logical Drive Size
