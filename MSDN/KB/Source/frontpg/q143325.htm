FP1: Personal Web Server Out of Memory Errors

 Last reviewed: March 17, 1998
Article ID: Q143325
The information in this article applies to:
  • Microsoft FrontPage for Windows, version 1.0
  • Microsoft FrontPage Personal Web Server, version 1.0

SYMPTOMS

When you run the FrontPage Personal Web Server under Windows 95, you may experience out of memory errors. In addition, the TCP/IP Test fields are not filled in or the Explain Results button is activated even though the TCP/IP test is not finished.

STATUS

Microsoft has confirmed this to be a problem in Microsoft FrontPage Personal Web Server version 1.0. Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

RESOLUTION

To resolve this problem, upgrade to the new version of the 32-bit Personal Web Server and TCP/IP Test. These programs are combined into one zip file, Vhttpd32.zip, which you can download from the following location: 

   ftp://ftp.vermeer.com/pub/patches/
Also, download the Vhttpd32.txt file for instructions on how to install the patch.

NOTE: You will need PKUnzip version 2.04g or later to extract the Vhttpd32.zip file.

MORE INFORMATION

The new TCP/IP Test better handles cases where the TCP/IP stack is incorrectly installed, namely cases where any TCP operation takes several minutes to fail. In the old TCP/IP Test, the Explain Results button was activated even when tests were incomplete. The new TCP/IP test enables the Explain Results button only after all tests are complete.
