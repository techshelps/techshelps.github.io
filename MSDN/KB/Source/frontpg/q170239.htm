FP: Subweb on Virtual Server Fails Authentication on IIS

 Last reviewed: March 18, 1998
Article ID: Q170239
The information in this article applies to:
  • Microsoft FrontPage 98 for Windows
  • Microsoft FrontPage 97 for Windows with Bonus Pack

SYMPTOMS

When you attempt to open a FrontPage Web on a multi-homed Internet Information Server (IIS), you are repeatedly prompted to enter a valid username and password.

CAUSE

This behavior occurs when the authentication response is directed to a default server rather than the requested virtual server while attempting to validate users on IIS.

RESOLUTION

Turn the default server into a virtual server, using these steps:

  1. Uninstall the FrontPage Server Extensions from the default port.

    The default port is 80.

  2. Start the Internet Service Manager.

  3. Double-click WWW Service.

  4. Click the Directories tab.

  5. In the "WWW Service Properties for <machine name>" dialog box, delete all _vti_* mappings which refer to the default port. (They will not have an address in the Address column.)

  6. Double-click the default server.

  7. In the Edit Properties dialog box, click to select the Virtual Server check box. In the Virtual Server IP Address box, type the IP address, and then click OK.

  8. Close the Internet Information Server Manager.

  9. Install the server extensions to the IP address which used to be the default server.

MORE INFORMATION

For more information about configuring FrontPage on an Internet Information Server, please see the following Microsoft Web site: 

   http://www.microsoft.com/frontpage/documents/iis
