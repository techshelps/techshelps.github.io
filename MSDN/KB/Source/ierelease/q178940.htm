Startup Programs Not Run After Installing Internet Explorer

 Last reviewed: January 7, 1998
Article ID: Q178940
The information in this article applies to:
  • Microsoft Internet Explorer versions 4.0, 4.01 for Windows NT 4.0

SYMPTOMS

When you log on to Windows NT without restarting your computer, programs in your Startup folder may not be run. You may experience this behavior after installing Internet Explorer.

CAUSE

This behavior can occur if a service uses different security credentials than those of the user logging on to Windows NT.

RESOLUTION

To work around this issue, use the appropriate method:

  • Stop the service that uses different security credentials, and then log on.
  • Configure the service that uses different security credentials to log on as the System account.

NOTE: You must log on as a user with administrative privileges to configure services.
