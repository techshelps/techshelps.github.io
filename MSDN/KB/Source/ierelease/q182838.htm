Internet Explorer Deletes Files from Share with Long File Name

 Last reviewed: March 21, 1998
Article ID: Q182838
The information in this article applies to:
  • Microsoft Internet Explorer version 3.03 for Windows NT 3.51

SYMPTOMS

When you use Internet Explorer version 3.03 for Windows NT 3.51 to view a Microsoft Office document on a Hypertext Markup Language (HTML) page that is stored on a share that uses a long file name, the Office document may be deleted from the share instead of being opened.

RESOLUTION

To resolve this issue, get the following fix from Microsoft: 

   Mshtml16.dll version 3.03.2932 (dated 3/6/98 or later) 1,079,104 bytes

STATUS

Microsoft has confirmed this to be a problem in Microsoft Internet Explorer version 3.03 for 16-bit platforms running on Windows NT version 3.51. A supported fix is now available, but has not been fully regression tested and should be applied only to computers experiencing this specific problem. Contact Microsoft Technical Support for more information.
