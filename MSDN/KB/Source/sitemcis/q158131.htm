INS Runs Multiple Feeds with No Error Message

 Last reviewed: February 3, 1998
Article ID: Q158131
The information in this article applies to:
  • Microsoft Commercial Internet System, version 1.0

SYMPTOMS

After you run a feed in Internet News Server (INS) from the command line, it is possible to run the exact same feed again. You can run the same feed multiple times without being notified of your error by the program.

For example, the following feed can be run over and over again from the command line: 

   rfeed -t a -f p -x 15 -r mynews -n * -d * -c
Each new instance adds a concurrent feed connection to mynews. You will not receive an error message from the program about this error.

CAUSE

INS does not check to see if the same feed is being executed more than once.

WORKAROUND

If you use the INS GUI administrator program, you can tell whether a duplicate feed has been set up by looking on the Feeds tab. The problem is not noticeable from the command line.

To avoid this problem you should always use the INS GUI to add feeds. you may continue to use the command line for adding feeds provided you type the command multiple times.

STATUS

Microsoft has confirmed this to be a problem in Commercial Internet System INS version 1.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
