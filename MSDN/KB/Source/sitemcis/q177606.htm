FIX: Posting Acceptor Not Reading All IE 4.0 Post Data

 Last reviewed: January 27, 1998
Article ID: Q177606
The information in this article applies to:
  • Microsoft Site Server version 2.0

SYMPTOMS

Large files fail when posted to Microsoft Posting Acceptor 1.0 by Internet Explorer (IE) 4.0.

CAUSE

Internet Explorer 4.0 will close a connection with an Internet Information Server computer upon an HTTP 401 response. An open connection is required to succeed with NTLM authentication, which is also required to upload via Posting Acceptor. If you are uploading a file larger than 48 KB, the message must be split into multiple packets to be processed by Posting Acceptor. Authentication information is also split over the packets. Between packets, a 401 response will be sent back to the client before NTLM succeeds, thus causing IE 4.0 to fail.

RESOLUTION

This problem has been corrected in Microsoft Posting Acceptor 1.01, which is available as part of the Windows NT 4.0 Option Pack, available at http://www.microsoft.com/iis.

STATUS

Microsoft has confirmed this to be a bug in Microsoft Site Server version 2.0. This bug has been corrected in Microsoft Posting Acceptor 1.01.
