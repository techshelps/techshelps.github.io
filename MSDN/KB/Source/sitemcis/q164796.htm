Automatic Replications Fail to Replicate

 Last reviewed: February 3, 1998
Article ID: Q164796

The information in this article applies to:

  • Microsoft Commercial Internet System, version 1.0

SYMPTOMS

Automatic replication projects with Transaction Method turned off will fail to replicate some files when large amounts of files are deleted from the source directory and then replaced with the same content. Further symptoms of this problem are that the Content Replication System (CRS) log produces "access denied" error messages when you try to delete the directories that the restored files are now trying to replicate to.

CAUSE

This condition is most often caused by users deleting entire content trees and then quickly rebuilding them with similar content.

WORKAROUND

To work around this problem:

  • Turn on Transaction Method.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Commercial Internet System version 1.0.
