NetMeeting Clients Can't Connect to Multihomed ILS

 Last reviewed: February 3, 1998
Article ID: Q161643
The information in this article applies to:
  • Microsoft Commercial Internet Server version 1.0

SYMPTOMS

NetMeeting 1.0 clients are unable to connect to your multihomed computer running Internet Locator Server (ILS), but NetMeeting 2.0 or other clients are able to connect with no problems.

CAUSE

NetMeeting 1.0 uses the ULP interface, which is not compatible with multihomed ILS servers.

WORKAROUND

Internet Service Providers (ISPs) who are running multihomed ILS servers should advise all customers to upgrade to Netmeeting 2.0. Because the new Netmeeting client uses the LDAP interface, customers will find faster performance and the ability to interact with other users through both audio and video interfaces over the Internet.

STATUS

Microsoft has confirmed this to be a problem with Microsoft Commercial Internet Server (MCIS).

MORE INFORMATION

A multihomed server is a server with more then one NIC or IP.
