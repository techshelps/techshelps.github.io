INS: Feed Sessions Dropped Due to Failure Opening Articles

 Last reviewed: February 3, 1998
Article ID: Q176672
The information in this article applies to:
  • Microsoft Commercial Internet Server version 1.0

SYMPTOMS

Feed sessions may be dropped by Internet News Server (INS).

CAUSE

INS drops the feed session due to failure opening articles in queue.

STATUS

Microsoft has confirmed this to be a limitation in Microsoft Commercial Internet Server version 1.0. A supported fix is now available, but has not been not fully regression tested and should be applied only to systems experiencing this specific problem. Unless you are severely impacted by this specific problem, Microsoft recommends that you wait for the next Service Pack that contains this fix. Contact Microsoft Technical Support for more information.

MORE INFORMATION

With the fix applied, the state machines for feeds open the article they are offering to the remote server before issuing the first part of the transfer (ihave <msgid> or xreplic group:number) to ensure that when it comes time to transmit the article the file can be opened. This allows the normal error handling to skip over missing articles in the queue instead of dropping the session each time a miss occurs.
