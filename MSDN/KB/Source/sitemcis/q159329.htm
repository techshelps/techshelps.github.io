Same Autostart Channel on Two Servers Causes Failure

 Last reviewed: February 3, 1998
Article ID: Q159329

The information in this article applies to:

  • Microsoft Commercial Internet System, version 1.0
BUG #: 14253

SYMPTOMS

A persistent channel that follows an autostart persistent channel may also be marked as an autostart channel. This causes a bad server-to-server message to be generated (due to the erroneous autostart entry) and the portal links between servers is terminated.

CAUSE

There is an channel identification problem in the version 1.0 Chatsvc.dll which is fixed in the updated Chatsvc.dll file.

RESOLUTION

Upgrade the system with Service Pack 1 (SP1) for Microsoft Commercial Internet System version 1.0.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Commercial Internet System, version 1.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
