ACLs Set Wrong by CRS if MBS Tokens Edited on Target Computer

 Last reviewed: February 3, 1998
Article ID: Q164759
The information in this article applies to:
  • Microsoft Commercial Internet System, version 1.0

SYMPTOMS

When replicating MBS Token mapped ACLs, CRS intermittently fails to automatically create Windows NT groups and/or map existing MBS tokens to the Windows NT groups on the target computer.

RESOLUTION

This bug has been fixed in Service Pack 1 for MCIS. CRS will properly create Windows NT groups and set the correct ACL based on the MBS Token relationship on the target computer.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Commercial Internet System, version 1.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
