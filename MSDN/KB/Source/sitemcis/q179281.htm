CRS Can Modify Files if FastMode Used and TransactionMethod=No

 Last reviewed: March 6, 1998
Article ID: Q179281
The information in this article applies to:
  • Microsoft Commercial Internet System version 1.0 - Content Replication System
  • Site Server version 2.0 - Content Replication System

SYMPTOMS

When you use Content Replication System (CRS) in fast mode with TransactionMethod off, some destination files are not identical to the source files. If you use XCOPY or Windows NT Explorer to copy the files, they are copied correctly.

WORKAROUND

  • Use frame mode and turn TransactionMethod on.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Commercial Internet Server version 1.0 and Site Server version 2.0. This problem has been corrected in the latest U.S. Service Pack for Site Server 2.0 Service Pack 1.
