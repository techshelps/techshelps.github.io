Channel Services May Fail to Load in Distributed Server Setup

 Last reviewed: February 10, 1998
Article ID: Q172622
The information in this article applies to:
  • Microsoft Commercial Internet System, version 1.0 - Microsoft Chat Server

SYMPTOMS

When you are using multiple Microsoft Chat Server servers with Channel Services, if the first member of a channel joins from a Microsoft Chat Server server that is not running the Channel Service, the Channel Service will not load.

CAUSE

The Channel Service Globally Unique Identifier (GUID) is not being sent over the server-to-server protocol, causing the failure of the Channel Service to load on a remote Microsoft Chat Server server.

RESOLUTION

Chatsvc.dll has been modified to add the service GUID to the AddChannel server-to-server message.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Commercial Internet System, version 1.0. A supported fix is now available, but is not fully regression-tested and should be applied only to systems experiencing this specific problem. Unless you are severely impacted by this specific problem, Microsoft recommends that you wait for the next Service Pack that contains this fix. Contact Microsoft Product Support Services for more information.
