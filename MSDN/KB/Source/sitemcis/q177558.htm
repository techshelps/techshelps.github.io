Cookies Not Generated When Using IP Address to View IIS Site

 Last reviewed: January 27, 1998
Article ID: Q177558
The information in this article applies to:
  • Microsoft Site Server version 2.0

SYMPTOMS

When you are using the Microsoft Internet Information Server and accessing a site using the site's IP address, no cookie information is generated. If you view a log file created using Microsoft Usage Analyst, no cookie information will be present for the visit to the site.

CAUSE

This is by design. IP addresses can be moved from site to site. Cookie creation is based on the domain name of the site, because the domain name should remain constant, and the IP address may not.

WORKAROUND

Make sure that the site has a domain name that may be used for access.
