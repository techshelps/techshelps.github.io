Proper Load Order of MCIS MIBS for Use with SNMP

 Last reviewed: February 3, 1998
Article ID: Q156973
The information in this article applies to:
  • Microsoft Commercial Internet System, version 1.0

The following list is the order that you should add MIBs to your SNMP management console to enable access to SNMP objects on Microsoft Commercial Internet System (MCIS) server nodes:

  1. Mib_ii.mib 2. Lmmib2.mib 3. Mssql.mib 4. Wins.mib 5. Dhcp.mib 6. Inetsrv.mib 7. Ftp.mib 8. Gopherd.mib 9. Http.mib
10. Gateway.mib 11. Pop3.mib 12. Smtp.mib 13. Nntp.mib 14. Perf2mib.mib

NOTE: For item 14 you need to make sure you also run Perf2mib\Setup.bat on each system you want to poll with objects defined in this MIB, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q139488
   TITLE     : How to Setup Windows NT SNMP Performance Counter
               Agent Extension
Afterwards all Microsoft specific entries will reside under the SNMP database branch: 

   iso.org.dod.internet.private.enterprises.microsoft.software
