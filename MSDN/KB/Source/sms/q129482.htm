Systems Management Server Inventory Via RAS Causes STOP Screen

 Last reviewed: April 23, 1997
Article ID: Q129482

The information in this article applies to:
  • Microsoft Windows NT Server version 3.5
  • Microsoft Systems Management Server versions 1.0 and 1.1

SYMPTOMS

When you start the Systems Management Server inventory management from a remote computer over a Remote Access Service (RAS) connection, a STOP message screen appears.

CAUSE

The SMSLS.BAT runs INVWIN32.EXE for inventory collection. The asyncmac driver fails to forward an I/O request packet (IRP) down the NDIS chain correctly.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.5. This problem has been corrected in the latest U.S. Service Pack for Windows NT version 3.5. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
This problem was corrected in Windows NT version 3.51.
