SMS: STOP 0x0000000A During Hardware Inventory

 Last reviewed: March 11, 1997
Article ID: Q163675
The information in this article applies to:
  • Microsoft Systems Management Server, version 1.2

SYMPTOMS

A computer running Windows NT Server running the Systems Management Server Inventory Agent either as a service or as an executable may generate the following Blue Screen STOP message during the hardware inventory phase: 

   STOP: 0x0000000A (0xC0002000,0x00000002,0x00000000,0x80129041)
   IRQL_NOT_LESS_OR_EQUAL
NOTE: This message was generated by a computer running Windows NT Server 3.51 Service Pack 5.

This problem is specific to the IBM PCI Token Ring Card 16/4, reference 75H9800, or 75H9802, using the following driver: 

   Driver properties : IBMTRP.SYS, 89728 bytes, 5/21/96, 4:31p.

WORKAROUND

To work around this problem, obtain a new version of the Ibmtrp.sys driver from IBM at the following location: 

   http://www.networking.ibm.com/nes/nestoken.htm
Download the following image (PCI Token Ring Adapter family): 

   PCI Token Ring NDIS Drivers Diskette
   Diskette 2 of 5
   [TRP1642.EXE | Version 1.2 | 1-30-97 | 518,330 bytes]
The updated driver is dated 11/04/96, and its size is 90,944 bytes. Any version of the driver dated before this date causes the error to occur.
