Domain Local Groups Added By Clicking Search in File Manager

 Last reviewed: April 10, 1997
Article ID: Q148639
The information in this article applies to:
  • Microsoft Windows NT Workstation version 3.51
  • Microsoft Windows NT Server version 3.51

SYMPTOMS

When you attempt to assign file or share permissions to a resource on a Windows NT Workstation or Windows NT Server (member server), the local groups created on the domain controllers are not available.

In addition, as a member of the domain's local group only, you are able to connect to a Windows NT Workstation or Windows NT Server (member server), but an "Access Denied" error message appears when you view the contents of the resource.

WORKAROUND

Add the users that you want to access resources to the global groups on the domain. Add the global groups to the local groups of the computers involved.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.51. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
