Mac Clients Must Log On to Trusted Domains - No Pass-through

 Last reviewed: March 25, 1997
Article ID: Q127103
The information in this article applies to:
  • Microsoft Windows NT Advanced Server version 3.1
  • Microsoft Windows NT Server version 3.5 and 3.51
  • Microsoft Windows NT Server version 4.0

SYMPTOMS

From a Macintosh client, if you log into a Windows NT master domain and then access a resource in a Windows NT domain that trusts the master domain, you must re-enter your user ID and password.

This occurs even when Services for Macintosh is configured to use the Microsoft User Authentication Module.

CAUSE

Windows NT clients are able to take advantage of Windows NT pass-through validation, where when you have an account on the master domain, you can access any trusting domain's resources without having to re-enter your name and password. However, the design of the Macintosh redirector and the AppleTalk protocol does not allow for participation in pass-through validation. Thus, you must re-enter your user ID and password.
