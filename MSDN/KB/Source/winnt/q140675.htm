Windows NT Doesn't Show All Files on OS/2 Server

 Last reviewed: March 15, 1996
Article ID: Q140675
The information in this article applies to:
  • Microsoft Windows NT Workstation version 3.51
  • Microsoft Windows NT Server version 3.51

SYMPTOMS

On your Windows NT version 3.51 computer, when you connect to an OS/2 1.3 LAN Manager version 2.2b server and view a directory in File Manager, not all files are displayed. At the command line, if you execute the DIR command on this directory, a "File not found" error message appears.

This only happens when there are many files in the directory (>1000) and only with a distinct number of files. The number of files differ from one Windows NT boot to the next.

RESOLUTION

This problem has been corrected in the latest Service Pack for Windows NT version 3.51.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.51. This problem was corrected in the latest Windows NT 3.51 U.S. Service Pack. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
