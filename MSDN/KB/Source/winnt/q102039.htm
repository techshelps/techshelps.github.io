Setup May Fail on Future Domain SCSI Controller

 Last reviewed: May 8, 1997
Article ID: Q102039

The information in this article applies to:
  • Microsoft Windows NT operating system, version 3.1
  • Microsoft Windows NT Advanced Server, version 3.1

SYMPTOMS

You are using a 16-bit Future Domain SCSI adapter and Windows NT Setup stops responding (hangs) when the graphical portion of Setup starts.

CAUSE

Under MS-DOS you can configure the 16-bit Future Domain SCSI adapters so they do not use an interrupt for the adapter. This configuration is not supported by Windows NT.

RESOLUTION

Configure the 16-bit Future Domain SCSI on an interrupt. This can be changed using a jumper setting on the adapter card.
