Retail SP3 Clients Cannot Connect to SP3 Beta 1 Servers

 Last reviewed: November 12, 1997
Article ID: Q141381
The information in this article applies to:
  • Microsoft Windows NT Workstation version 4.0 SP3 Beta 1
  • Microsoft Windows NT Server version 4.0 SP3 Beta 1

SYMPTOMS

When you log on to a Windows NT 4.0 computer running the released version of Service Pack 3 (SP3) and then attempt to connect to a Windows NT 4.0 server running the Beta 1 version of Service Pack 3, you may receive an error message similar to the following: 

   Logon failure: the user has not been granted the requested logon type
   at this computer.

CAUSE

Service Pack 3 for Windows NT 4.0 contains a feature that allows additional Server Message Block (SMB) session security between Windows NT servers and Windows NT workstations. The Beta 1 version of Service Pack 3 has an SMB security feature enabled by default that is incompatible with the released version of Service Pack 3.

RESOLUTION

To resolve this problem, you must upgrade all Windows NT computers running the Beta 1 version of Service Pack 3 to the released version of Service Pack 3.
