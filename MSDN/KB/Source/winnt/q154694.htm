New Policy Available to Hide Go To on Tools Menu

 Last reviewed: September 10, 1997
Article ID: Q154694
The information in this article applies to:
  • Microsoft Windows NT Workstation version 4.0
  • Microsoft Windows NT Server version 4.0

SYMPTOMS

Once you remove drives from Windows NT Explorer using the NoDrives registry entry through a system policy or the TweakUI utility, you can still jump to the drives by clicking Go To on the Tools menu in Windows NT Explorer.

CAUSE

This problem occurs because a policy for this dialog was not created when the Windows NT system policies were first defined.

RESOLUTION

To resolve this problem, obtain the hotfix below, or wait for the next service pack. Once you apply the hotfix, the new System Policy is called "Remove Tools->Goto Menu from Explorer" in the category "Windows NT Shell\Restrictions for Users and Groups".

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 4.0. A supported fix is now available, but has not been fully regression-tested and should be applied only to systems experiencing this specific problem. Unless you are severely impacted by this specific problem, Microsoft recommends that you wait for the next Service Pack that contains this fix. Contact Microsoft Technical Support for more information.
