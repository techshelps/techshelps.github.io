Corel Draw 5.0 Running in Second VDM Loses Data

 Last reviewed: September 21, 1994
Article ID: Q120062
The information in this article applies to:
  • Microsoft Windows NT workstation version 3.5
  • Microsoft Windows NT Server version 3.5

SUMMARY

If you run separate versions of Corel Draw 5.0 in separate VDMs, Corel's temp file algorithm may cause data loss. Corel Draw is currently working on a fix (as of 8/30/94).
