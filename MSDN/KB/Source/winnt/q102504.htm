Aldus Freehand 3.1 Install Beeps Several Times on Each Disk

 Last reviewed: April 4, 1997
Article ID: Q102504

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

The Aldus Freehand version 3.1 installation program beeps several times after each disk under Windows NT.

CAUSE

This occurs because Aldus Freehand 3.1's installation program calls the MessageBeep(0) application program interface (API) functions several times in a row when prompting for the next disk. On Windows version 3.1, each MessageBeep call results in a very short tone, so several MessageBeeps calls in a row sound like a longer tone. On Windows NT, each MessageBeep emits a longer tone, resulting in several clear beeps.
