STOP: 0x0000000A or STOP: 0x0000001E in Tcpip.sys

 Last reviewed: February 4, 1998
Article ID: Q146095
The information in this article applies to:
  • Microsoft Windows NT Server version 4.0
  • Microsoft Proxy Server version 2.0

SYMPTOMS

Your computer running Microsoft Windows NT Server version 4.0 running Microsoft Proxy Server version 2.0 may display one of the following blue screens error messages: 

   STOP: 0x0000000A
   IRQL_NOT_LESS_OR_EQUAL
-or- 

   STOP 0x0000001e
   Unhandled Kernel exception

RESOLUTION

To resolve this problem, obtain the following fix or wait for the next Windows NT service pack.

This fix should have the following time stamp: 

   08/29/97  12:29 PM               140,528 Tcpip.sys (Intel)
   08/29/97  12:27 PM               258,448 Tcpip.sys (Alpha)
NOTE: Service Pack 3 must be applied to Windows NT 4.0 prior to applying this fix.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 4.0. A supported fix is now available, but has not been fully regression tested and should be applied only to systems experiencing this specific problem. Unless you are severely impacted by this specific problem, Microsoft recommends that you wait for the next Service Pack that contains this fix. Contact Microsoft Technical Support for more information.
