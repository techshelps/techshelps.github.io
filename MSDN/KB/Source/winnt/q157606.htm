Using NWCONV with Duplicate Group Name

 Last reviewed: March 20, 1997
Article ID: Q157606
The information in this article applies to:
  • Microsoft Windows NT Server versions 3.51 and 4.0

SYMPTOMS

Before migration, on a Novell server, a group, for example, named GROUP1 is a member of "print server operator." On the migration destination, which is a Windows NT server, a group with a duplicate name already exists. After you choose the "add prefix: nw" option for duplicated group names with the NetWare Migration Tool, you find, after migration, that GROUP1 belongs to the "print operator" group, but NWGROUP1 does not belong to the "print operator" group.

CAUSE

Under investigation.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.51 and 4.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
