Deleted User Profiles Visable In Setup

 Last reviewed: May 12, 1997
Article ID: Q106217
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When you choose Delete User Profiles from the Options menu in Setup, users may be displayed with blank user names. These blank entries refer to accounts that have been deleted using User Manager after the accounts have been logged onto successfully at least once.

WORKAROUND

Select the entries with blank user names and delete them manually.

STATUS

Microsoft has confirmed this to be a problem in Windows NT and Windows NT Advanced Server version 3.1. We are researching this problem and will post new information in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. Log onto an account with administrative privileges. 2. Create a new account with User Manager. 3. Log off and then on using the newly created account. 4. Log off and back onto an administrative account. 5. Use User Manager to delete the newly created account. 6. Exit User Manager and start Setup. 7. From the Options menu, choose Delete User Profiles.

Note the blank user name for the account you just deleted.
