Windows NT Backup Does Not Support Special-Character Filenames

 Last reviewed: June 15, 1995
Article ID: Q131615
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

The Windows NT Backup 3.1 allows you to backup files with special characters (such as ~ [umlaut]) in the file name. However, if you attempt to restore the file, the special characters in the file name is replaced with non-printable characters.

WORKAROUND

To work around this problem, upgrade to Windows NT 3.5 or 3.51.

The problem does not occur when you backup and restore the files with Windows NT Backup under Windows NT 3.5 and 3.51.
