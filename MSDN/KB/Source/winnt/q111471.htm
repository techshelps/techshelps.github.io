Access Violation in SPOOLSS.EXE

 Last reviewed: May 5, 1997
Article ID: Q111471

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

If the Hewlett-Packard (HP) DeskJet 1200c PostScript driver is installed on a Windows NT machine and a non-PostScript job is sent to the spooler, SPOOLSS.EXE disappears from memory (you can use PVIEW.EXE to verify). A DRWatson may be generated with the following stack trace: 

   RetAddr  FramePtr Param#1  Param#2  Param#3  Param#4
   77e1be8d 00c4fe2c 00000000 77c02690 00000004 0095a100 strncmp
   77bf920c 00c4fe4c 00950002 0095a140 00c4fe68 00c40001 TranslatePjl
   77bf929a 00c4fe9c 0095c401 00950002 0095a140 77bf4436 DecodePjl
   77bf48b7 00c4feac 0095a100 00c4ffec 000000ff 00000000 DecodeIFrame
   77bf4436 00c4ffb4 00000000 000000ff 00000000 00000000 ReadThread

CAUSE

The PostScript translator was not checking for NULL characters.

STATUS

Microsoft has confirmed this to be a problem in Windows NT and Windows NT Advanced Server version 3.1. This problem has been corrected in the latest U.S. Service Pack for Windows NT and Windows NT Advanced Server version 3.1. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
