CTRL+X CTRL+M Does Not Create OS/2 Slick Shell

 Last reviewed: May 7, 1997
Article ID: Q101705

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

Entering the CTRL+X CTRL+M key combinations do not create an OS/2 Slick shell.

CAUSE

Windows NT does not support the OS/2 KbdRegister() function.

STATUS

Windows NT does not provide a method to work around this restriction.

MORE INFORMATION

Even though CTRL+X CTRL+M activates another Command Prompt, it cannot be used as a Slick buffer. When you end your Slick session, you must close the Command Prompt.
