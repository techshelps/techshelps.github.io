Add Printer Wizard Error When Adding Multiple Printer Drivers

 Last reviewed: November 5, 1997
Article ID: Q158753
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1
  • Microsoft Windows NT Workstation versions 3.5, 3.51, and 4.0
  • Microsoft Windows NT Server versions 3.5, 3.51, and 4.0

SYMPTOMS

When you add a new printer by clicking the Start button, pointing to Settings, clicking Printers, double-clicking Add Printer and choose simultaneously several alternate printer drivers to install from different versions and/or platforms of Windows NT, one or both of the following errors appear: 

   Printer Driver for Windows NT <version> <platform> could not be
   installed. An unexpected error occurred.
-or- 

  Operation could not be completed. The printer driver is unknown.
Adding the printer fails, no printer is added, although some files may be copied into the appropriate directory under the spool\drivers directory.

NOTE: This problem does not occur if you add alternate printer drivers for an installed printer.

CAUSE

You inserted the compact disc from Windows NT version 3.x instead of Windows NT 4.0 because Windows NT 4.0 fails to specify that you insert the Windows NT version 4.0 CD.

RESOLUTION

Run the Add Printer Wizard again and insert the Windows NT 4.0 CD when a message prompts you to insert a CD without specifying the version of Windows NT.

The install wizard installs all drivers selected in the order that they appear in the list box, adding the 4.0 driver first. Therefore, if the wizard prompts you the first time or prompts you without specifying the version to insert a CD, insert the Windows NT 4.0 CD. When you are prompted for a specific version, insert the CD of that version.

MORE INFORMATION

When you copy the drivers for the 3.5x and 3.1 versions, the install program requests those CDs by name and version.

When you insert the CD of a previous version and if the printer driver name is the same as in the new version but Windows NT is looking for the version 4.0 driver, Windows NT attempts to install the older driver as the 4.0 version driver. The install may succeed to some degree, actually copying some files and even prompting for the other CDs, but will eventually fail.
