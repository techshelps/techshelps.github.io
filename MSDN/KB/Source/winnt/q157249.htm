Windows 95 Client Computer Cannot See the Print Queue List

 Last reviewed: October 2, 1997
Article ID: Q157249
The information in this article applies to:
  • Microsoft Windows NT Workstation versions 3.51 and 4.0
  • Microsoft Windows NT Server versions 3.51 and 4.0

SYMPTOMS

When more than 120 jobs held in the print queue and you connect to the printer from a client computer running Microsoft Windows 95, the job list does not appear when you double-click the printer icon.

CAUSE

The cause of this problem is currently under investigation.

WORKAROUND

To work around this problem, wait until the number of print jobs is less than 120 in the Windows NT print queue. From the Windows 95 client computer, connect to the printer again, and you should be able to see the print queue.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Windows NT Server and Workstation versions 3.51 and 4.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
