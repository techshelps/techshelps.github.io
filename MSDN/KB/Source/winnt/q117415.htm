STOP: 0x0000000A with Future Domain SCSI Controller

 Last reviewed: May 16, 1997
Article ID: Q117415
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When you write data to a Micropolis 2217 or 2210 hard disk drive, using a Future Domain TMC 1650 SCSI controller, the following STOP message may appear: 

   STOP: 0x0000000A ( 0x00000000, 0x00000010, 0x00000001, 0x[address] )
   IRQL_NOT_LESS_OR_EQUAL
The second parameter points to IRQL 16, the third parameter indicates a write operation that failed, and the forth parameter points to HAL.DLL.

WORKAROUND

At this time, the only workaround is to use a SCSI controller card from a different manufacturer or a different hard disk.
