Seagate Tape Will Not Autoeject After NTBackup Finishes

 Last reviewed: March 16, 1998
Article ID: Q178556
The information in this article applies to:
  • Microsoft Windows NT Workstation version 4.0
  • Microsoft Windows NT Server version 4.0

SYMPTOMS

After you install Service Pack 3 on a system running Windows NT 4.0 and having a Seagate TR4 tape drive (model CTT8000-A), the tape does not Autoeject after finishing the backup. This function works fine with Service Pack 1 and Service Pack 2.

NOTE: The CTT8000-A is an Atapi device and connects to the floppy drive.

CAUSE

The problem is within NTBackup in Service Pack 3.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 4.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
