Bugcheck 0000001e with Exception Code of 80000004

 Last reviewed: August 14, 1997
Article ID: Q148978
3.51 WINDOWS

The information in this article applies to:

  • Microsoft Windows NT Workstation version 3.51
  • Microsoft Windows NT Server version 3.51

When a bugcheck 0000001e error occurs with an exception code of 80000004, it may indicate that there is faulty hardware on the computer. This problem seldom occurs.

80000004 is an exception code that may occur when a POPFD is executed. 80000004 represents the status of STATUS_SINGLE_STEP, which is debugging one instruction at a time. POPFD pops the top of the stack into the EFLAGS register. When the 0x00000100 bit is set in the EFLAGS register, the next instruction will be trapped and the exception code will be 80000004.

A faulty secondary cache may be the cause of this problem. NT!_KeSetPriorityThread+0x3e Somehow POP'd.
