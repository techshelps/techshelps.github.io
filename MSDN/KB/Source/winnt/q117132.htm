Terminal Doesn't Download Multiple Files Using Kermit

 Last reviewed: May 16, 1997
Article ID: Q117132
The information in this article applies to: 

 -Microsoft Windows NT operating system version 3.1
 -Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

Some bulletin board systems allow you to flag multiple files before you download them. When you select Kermit as the binary transfer protocol, the first file transfers as usual, but when Terminal attempts to download the second file, the following error message appears: 

   Terminal could not receive the binary file xxxx.xxx. Verify that you are
   using the correct protocol and try again.

CAUSE

This problem appears to be a limitation of Terminal using Kermit protocol. When you use XModem, you do not have the one-file-per-download limit.

RESOLUTION

The only known workaround at this time is to use XModem as an alternate download protocol.

STATUS

This feature is under review and will be considered for inclusion in a future release of Windows NT.
