How to Use WINMSDP.EXE

 Last reviewed: May 8, 1997
Article ID: Q102468
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

WINMSDP.EXE is a utility written to be spawned by the WINMSD.EXE GUI application for reporting purposes. The WINMSD.EXE program spawns the WINMSDP.EXE utility with the /A switch (Print All Settings).

You can also execute the WINMSDP.EXE program from the command line. If you run the utility, the following menu appears: 

   C:\winnt\system32>winmsdp.exe

   /A Print All Settings
   /O OsVer
   /D Drives
   /S Services
   /R Drivers
   /M Memory
   /I Interrupt Resources
   /Y Memory Resources
   /P Port Resources
   /U DMA Resources
   /W Hardware
   /E Environment
   /N Network
Each of the switches (except /A and /N) corresponds to a particular section of the MSDRPT.TXT file.

The first time WINMSDP.EXE is run, the following message is displayed: 

   C:\winnt\system32>winmsdp /n
   Generating WinMSD Report of your system ....
   WinMSD has saved your system report in msdrpt.TXT
The next time WINMSDP.EXE is run, it saves the previous MSDRPT.TXT file as a file named MSDRPT00.TXT (and so on). For example: 

   C:\winnt\system32>winmsdp /n
   Generating WinMSD Report of your system...

   backed up your previous MSDRPT.TXT file to: MSDRPT00.TXT

   WinMSD has saved your current system report in MSDRPT.TXT
The /N parameter generates a report of the network parameters listed in the Windows NT Registry database. This report contains valuable information for PSS when diagnosing network-related problems.

NOTE: The /N switch is not used if you use the /A switch; these switches must be used separately.
