New Option Name Appears as **iinvalid** in DHCP Manager

 Last reviewed: December 18, 1997
Article ID: Q135128
The information in this article applies to:
  • Microsoft Windows NT Server versions 3.5 and 3.51

SYMPTOMS

After you add a new Dynamic Host Configuration Protocol (DHCP) option with a long integer data type in DHCP Manager, the new DHCP option name appears as follows: 

   **iinvalid**
When you attempt to add a long integer value for the new DHCP option, the following error message appears: 

   The parameter is incorrect

STATUS

Microsoft has confirmed this to be a problem in Windows NT versions 3.5 and 3.51. This problem was corrected in Windows NT Server version 4.0.
