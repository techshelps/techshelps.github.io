DHCP Stops Assigning IP Addresses to Clients

 Last reviewed: August 13, 1997
Article ID: Q153476
3.50 3.51 WINDOWS kbnetwork

The information in this article applies to:

  • Microsoft Windows NT Server versions 3.5 and 3.51

SYMPTOMS

The DHCP server stops assigning IP addresses, and Event Viewer logs this event: 

     DHCP Event 1014
     The JET Database call returned the following Error
     with a hex data status of "ff fd ff ff (0xfffffdff = -513)."

CAUSE

The volume that contains the DHCP database is full. There is no more room to expand the Dhcp.mdb file.

RESOLUTION

Free up hard drive space on the partition that contains the Dhcp.mdb file.
