PowerBuilder Preferences Dialog Box Has No Text

 Last reviewed: February 7, 1995
Article ID: Q120428
The information in this article applies to:
  • Microsoft Windows NT workstation version 3.5
  • Microsoft Windows NT Server version 3.5

SYMPTOMS

No text appears in the PowerSoft PowerBuilder Preferences dialog box.

CAUSE

This is caused by an application problem.

RESOLUTION

Obtain a new PBSYS030.DLL in PowerSoft Technical Bulletin 1-3, tracking number 81947.

STATUS

Microsoft is researching this problem and will post new information in the Microsoft Knowledge Base as it becomes available.
