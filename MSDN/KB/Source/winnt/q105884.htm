Eject Tape Option in Windows NT Backup

 Last reviewed: May 12, 1997
Article ID: Q105884
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

The Eject Tape option on the File menu of the Windows NT Backup program functions correctly only on tape drives that support the SCSI command to eject the tape. In some cases, the tape drive may appear to support the command but will not actually eject the tape when the option is selected. Instead, it will rewind/reset the tape. When this occurs, the Backup program will assume that the tape has been correctly ejected and not recognize that a tape is still in the drive until the tape has been manually ejected and replaced in the drive.
