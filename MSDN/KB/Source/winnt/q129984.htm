SWITCH.INF Script Unavailable as Option in Remote Access

 Last reviewed: March 25, 1997
Article ID: Q129984
The information in this article applies to:
  • Microsoft Windows NT Workstation version 3.5
  • Microsoft Windows NT Server version 3.5
  • Microsoft Windows NT Server version 4.0
  • Microsoft Windows NT Workstation version 4.0

SYMPTOMS

You are unable to select a script from your SWITCH.INF file in the Before Dialing or After Dialing fields of the Remote Access (RASPHONE.EXE) Security Settings dialog box.

CAUSE

One or more scripts from SWITCH.INF may not appear as options in RASPHONE.EXE if:

  • You have not restarted RASPHONE.EXE since modifying the SWITCH.INF file.
  • You saved the SWITCH.INF file as a UNICODE TXT file.
  • You used incorrect syntax in the SWITCH.INF file.

RESOLUTION

If one or more scripts from SWITCH.INF do not appear as options in RASPHONE.EXE:

- Restart RASPHONE.EXE. After making any change to the SWITCH.INF file, 

  you must restart RASPHONE.EXE before the changes are available.
- Confirm that SWITCH.INF is saved as a standard TXT file. If you have 
  saved the SWITCH.INF file as a UNICODE TXT file, none of the scripts in
  SWITCH.INF will be available in the Security Settings dialog of
  RASPHONE.EXE. NOTEPAD.EXE has the option of saving a file as a standard
  TXT file or as a UNICODE TXT file.
- Confirm that you used proper syntax in the SWITCH.INF file. Each script 
  in the SWITCH.INF file is designated by the name of the script enclosed
  in square brackets (for example, [MyScript]). If the script name is not
  enclosed in square brackets, the script will not be available in the
  RASPHONE.EXE Security Settings dialog box.

  Also, if a semi-colon precedes the script name on the same line of
  SWITCH.INF, RASPHONE.EXE will treat the line as a comment and the script
  will not be an option in the Security Settings dialog box.
