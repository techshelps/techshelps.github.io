RAS Client Cannot Browse RAS Server With IPX Only

 Last reviewed: February 4, 1998
Article ID: Q148154
The information in this article applies to:
  • Microsoft Windows NT Workstation versions 3.5, 3.51, and 4.0
  • Microsoft Windows NT Server versions 3.5, 3.51, and 4.0

SYMPTOMS

When you make a PPP connection using the IPX/SPX Compatible Transport protocol from a Remote Access Service (RAS) client to a Windows NT RAS server only, the RAS server does not appear on the RAS client in the Map Network Drive dialog box in Windows NT 4.0 or the Connect Network Drive dialog box in earlier versions of Windows NT, even if you have full administrative permission to the RAS server.

However, you can connect properly to the shared resources on the RAS server using Windows NT Explorer, File Manager, or the MS-DOS Command Prompt (CMD.EXE) provided that you know the specific names of the shared resources.

CAUSE

This behavior is by design.

RESOLUTION

To work around this problem, install the NetBEUI protocol in addition to IPX/SPX Compatible Transport protocol.
