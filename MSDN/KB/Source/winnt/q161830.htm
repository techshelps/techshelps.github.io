Message from Unix Using Smbclient w/ Long Username Crashes

 Last reviewed: May 19, 1997
Article ID: Q161830
The information in this article applies to:
  • Microsoft Windows NT Workstation version 4.0
  • Microsoft Windows NT Server version 4.0

SYMPTOMS

When you use the SAMBA utility, smbclient, to send messages from computers running UNIX to computers running Windows NT 4.0, Services.exe may stop responding if the username is greater than 15 characters.

CAUSE

The long username overflows a buffer that is used to store the message for Winpopup.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 4.0. This problem was corrected in the latest Microsoft Windows NT 4.0 U.S. Service Pack. For information on obtaining the service pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
