Err Msg: Fatal Error on Drive 1 w/NTBackup and Compaq SCSI

 Last reviewed: May 16, 1997
Article ID: Q118400
The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When you are using the Windows NT backup program, NTBACKUP.EXE, the backup process may stop and the following error message may appear: 

   Fatal error on drive 1

CAUSE

This problem occurs with tape backup devices connected to Compaq SCSI Compression or Option adapters.

RESOLUTION

There is a new version of the Compaq Support Software Disk (version 1.04, Rev. C) that provides a fix for this problem. The fix can be downloaded from Compaq BBS at (713) 378-1418. The name of the archive file to download is SP0802.ZIP.

MORE INFORMATION

The file, SP0802.ZIP, unzips into SP0802.EXE, which is a self-extracting archive file. When the EXE is executed, it explodes into several other files.

One of these files is 145911-H.DOC which contains instructions on how to create the SSD. Other relevant files are SCSICOMP.SYS and OEMSETUP.INF located in the SCSI\SCSICOMP directory on the SSD.

You can backup using this driver only if the backup hardware vendor provides class drivers for the tape backup devices that can be attached to the COMPAQ SCSI Compression or Option Adapter. The three tape devices that the COMPAQ SCSI Compression and Option Adapter support are: COMPAQ 320/525 MB tape drive, the COMPAQ 320/525 MB ACA tape drive and the COMPAQ 1.3/2.0 GB DAT drive.
