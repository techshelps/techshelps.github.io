SFM: Err Msg: Address Error

 Last reviewed: May 15, 1997
Article ID: Q116012
The information in this article applies to:
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

When you drag a file from the Microsoft UAM volume to the System folder of a Macintosh and then use Chooser to select a file server using AppleShare, the Macintosh stops responding (hangs) and an "Address Error" message appears.

This problem occurs when you are running Services for Macintosh with a Macintosh SE with 4 megabytes (MB) of memory and running Apple System 7.1.

STATUS

Microsoft has confirmed this to be a problem in Windows NT Advanced Server version 3.1. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
