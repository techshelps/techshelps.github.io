Hangs and Network Problems with NE3200 Adapters (RISC Only)

 Last reviewed: May 12, 1997
Article ID: Q105327

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

On a RISC-based computer with multiple NE3200 network adapters installed, your computer may stop responding (hang) as Windows NT starts network services. If your computer does not hang, you may not be able to get one or more network adapters to work.

CAUSE

The NE3200 driver and Windows NT incorrectly report that all adapters are in the same slot in the computer bus.

STATUS

Microsoft has confirmed this to be a problem in Windows NT and Windows NT Advanced Server version 3.1. This problem was corrected in the latest U.S. Service Pack for Windows NT and Windows NT Advanced Server version 3.1. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
