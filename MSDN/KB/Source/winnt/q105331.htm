NTLDR Err Msg: IoAllocateAdapterChannel: ...Adapter in Use

 Last reviewed: May 12, 1997
Article ID: Q105331

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

During system startup, NTLDR displays the following message about 40-50 times: 

   IoAllocateAdapterChannel: Called while Adapter in use
Your system appears to start as usual after these messages appear.

CAUSE

The disk buffer in NTLDR is too small to store the number of hard disk drives you have on your computer.

STATUS

Microsoft has confirmed this to be a problem in Windows NT and Windows NT Advanced Server version 3.1. This problem was corrected in the latest U.S. Service Pack for Windows NT and Windows NT Advanced Server version 3.1. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
