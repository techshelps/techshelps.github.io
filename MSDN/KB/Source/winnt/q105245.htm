ClipBook May Not Be Able to Save Files to Large Directory Name

 Last reviewed: May 12, 1997
Article ID: Q105245

The information in this article applies to:
  • Microsoft Windows NT operating system version 3.1
  • Microsoft Windows NT Advanced Server version 3.1

SYMPTOMS

In Windows NT, if you attempt to save a file from ClipBook Viewer to an NTFS directory with a 128-character or larger name, the file will not be saved and you may receive the following error message: 

   An error has occurred. ClipBook Viewer cannot complete this procedure.

RESOLUTION

Save the file to another directory and then copy the file to the directory with the long name using File Manager or another utility.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.1 and Windows NT Advanced Server version 3.1. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. Use the File Manager to create a directory with a 128-character or larger name on an NTFS partition.

  2. Copy some text to the Clipboard using a text editor, such as Notepad.

  3. Open the ClipBook Viewer and attempt to save the contents of the clipboard to the long NTFS directory.

The above error message appears.
