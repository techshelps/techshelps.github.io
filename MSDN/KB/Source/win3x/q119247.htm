WFWG: Pause and Resume Events Are Not Logged

 Last reviewed: November 21, 1994
Article ID: Q119247
The information in this article applies to:
  • Microsoft Windows for Workgroups version 3.11

SYMPTOMS

When you print to a shared printer on a machine with Event Log enabled, Pause and Resume events for the queue are not logged.

CAUSE

Windows for Workgroups does not log Pause and Resume events by design. The following printing-related events are logged:

  • Someone connecting to the print share
  • Print job spooled successfully
  • Print job completed
  • Individual print jobs are paused or resumed
