Write Err Msg: '<Filename> Could Not Be Found' and WFWG Hangs

 Last reviewed: November 23, 1994
Article ID: Q91127
The information in this article applies to:
  • Microsoft Windows for Workgroups versions 3.1, 3.11

SYMPTOMS

If you Windows Workgroups Write, you may receive the "<filename> could not be found" error message if any of the following situations occur:

  • You connect to a server in File Manager.
  • You drag and drop a document into Write.
  • The network connection to that server is lost.

If you choose the OK button, the message returns. At this point, you are in a loop and cannot exit Write.

WORKAROUND

The only way to exit Write (and Windows for Workgroups) is to restart the computer.

STATUS

We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
