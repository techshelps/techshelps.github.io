Divide by Zero Error with Ventura Publisher Version 3.0

 Last reviewed: November 23, 1994
Article ID: Q71841
The information in this article applies to:
  • Microsoft Windows operating system versions 3.0, 3.0a

SYMPTOMS

You receive a divide by zero error in Ventura Publisher version 3.0 when you print if all of the following is true:

  • You are using the PostScript driver from the Microsoft Windows version 3.0 Supplemental Driver Library (SDL).
  • The PostScript driver is configured for a Linotronic 300 version 47.1, with the paper size set to "Tabloid 11 x 17."
  • The PostScript driver is connected to the FILE port.
  • In Ventura Publisher, the paper size is set to "11 x 17."

STATUS

Xerox has confirmed this to be a problem with Ventura Publisher and is researching this problem.

NOTE: This information applies to Microsoft Windows versions 3.0 and 3.0a. This information does not apply to later versions of Windows.

MORE INFORMATION

For more information, please contact Xerox.
