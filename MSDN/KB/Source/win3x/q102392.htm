Windows Loses GDI Resources with Micrografx 3.1 Object

 Last reviewed: January 23, 1995
Article ID: Q102392
The information in this article applies to:
  • Microsoft Windows operating system versions 3.1, 3.11
  • Microsoft Word for Windows, versions 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c
  • Microsoft Publisher 2.0

SYMPTOMS

Microsoft Windows version 3.1 may lose GDI resources when your Microsoft Word for Windows document contains an embedded object from any of the following Micrografx applications: 

   Application                      .EXE File Date
   -----------------------------------------------

   Designer 3.1 plus OLE            5/28/92

   Windows Draw 3.0 plus OLE        5/28/92

   Charisma 2.1                     7/23/92
The loss of GDI resources occurs when you scroll through the document or switch to print preview. You cannot restore the lost resources by exiting Microsoft Word.

STATUS

Micrografx, Inc., has confirmed this to be a problem with the above three products.

RESOLUTION

To correct this problem, obtain the GDIFIX.EXE patch from Micrografx Technical Support. This patch modifies the following Micrografx files: 

   Filename         File Date
   --------------------------

   MGXLIB.DLL       01/13/93
   MGXWMF.DRV       02/24/93
   MGXFNT.DLL       10/31/92
For more information, or to obtain the GDIFIX.EXE patch, contact Micrografx Technical Support.

NOTE: You need a password to download the GDIFIX.EXE patch from the Micrografx bulletin board service (BBS). Contact Micrografx Technical Support to obtain the current password.

WORKAROUND

You can temporarily work around this problem by inserting Micrografx graphics using the Picture command on the Insert menu in Word. In many cases, this method prevents the GDI resources leak so that you can work in the file until you receive the GDIFIX.EXE patch. If this method doesn't work, delete the Micrografx objects from your document until you install the GDIFIX.EXE patch, then embed the objects again.
