INFO: GetKBCodePage Returns Different Value Than GetOEMCP

 Last reviewed: January 15, 1998
Article ID: Q175391
The information in this article applies to:
  • Microsoft Windows Software Development Kit (SDK)
  • Microsoft Win32 Software Development Kit (SDK)

SUMMARY

Under Windows 95 Western European versions, the 16-bit function GetKBCodePage returns 437 ( MS-DOS Latin US code page), but the 32-bit function GetOEMCP returns 850 ( MS-DOS Latin 1 code page). This behavior is by design. There is no programmatic way to change it.
