FIX: VC++ 1.5 Setup Erroneously Replaces Newer GRID.VBX

 Last reviewed: September 18, 1997
Article ID: Q123037
1.50 WINDOWS kbsetup kbfixlist kbbuglist

The information in this article applies to:

  • Microsoft Visual C++ for Windows, version 1.5

SYMPTOMS

The version of GRID.VBX that ships with Visual C++ version 1.5 for Windows is older than the version of GRID.VBX that shipped with Visual Basic version 3.0 for Windows. If you installed Visual Basic version 3.0's GRID.VBX before you installed Visual C++ version 1.5, the setup program for Visual C++ overwrote Visual Basic's newer version of GRID.VBX in the WINDOWS\SYSTEM directory. Visual Basic does not work with the older Visual C++ version of GRID.VBX.

RESOLUTION

To allow Visual Basic to work correctly, replace GRID.VBX from Visual C++ with GRID.VBX from Visual Basic by copying the file manually.

The following table can be used to determine which version of GRID.VBX is installed. 

   VBX           Version     File Size
   -----------------------------------
   Visual C++    2.00.0910   54,048
   Visual Basic  3.00.0537   44,656
You can also check the version of GRID.VBX by using File Manager. With the GRID.VBX file selected, choose Properties from the File menu.

STATUS

Microsoft has confirmed this to be a bug in the Microsoft product listed at the beginning of this article. This bug was corrected in Visual C++ for Windows, version 1.51. Visual C++ version 1.51 ships with version 3.00.0538 of GRID.VBX.
