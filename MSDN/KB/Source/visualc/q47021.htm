VRES Video Modes Have 30 Lines w/ Graphics Text Functions

 Last reviewed: July 17, 1997

The information in this article applies to:

  • The C Run-time (CRT), included with: 

        - Microsoft C for MS-DOS, versions 5.1, 6.0, 6.0a, and 6.0ax
    - Microsoft C/C++ for MS-DOS, version 7.0
    - Microsoft Visual C++ for Windows, versions 1.0 and 1.5

SUMMARY

When using the text routines in the Microsoft run-time library, the two graphic modes, _VRES2COLOR and _VRES16COLOR, have 30 text lines by default because of their increased vertical resolution. Most of the other graphics modes have 25 text lines when using these routines.

MORE INFORMATION

The following graphics functions are the routines concerned with text placement and output: 

   _displaycursor()
   _gettextcolor()
   _gettextposition()
   _outtext()
   _settextposition()
   _settextcolor()
   _settextwindow()
   _wrapon()
The line numbers stated above are based on the default 8 x 8 font size of the IBM ROM BIOS. Note that when using fonts, the number of text lines in any video mode depends on the height of the font. For information concerning changing from the default font size, see the interrupt 10, function 11 entries in "IBM ROM BIOS," a Microsoft Press book by Ray Duncan.
