BUG: "Help Topic Does Not Exist" for VBX Control in App Studio

 Last reviewed: July 22, 1997
Article ID: Q116166
1.00 1.50 WINDOWS kbtool kbbuglist

The information in this article applies to:

  • The App Studio, included with: Microsoft Visual C++ for Windows, versions 1.0 and 1.5

SYMPTOMS

The Help topics for VBX controls do not appear when Help is accessed from the Install Controls dialog box within App Studio. The same is true when the Help button is chosen from the properties box for a VBX control inserted into a dialog box. In both situations, attempting to access Help causes the message "Help topic does not exist" to be displayed twice by Windows Help.

RESOLUTION

Help on using VBX controls in App Studio can still be accessed from the App Studio Help menu. To do this, use the following steps:

  1. Choose the Help menu item from the App Studio main menu.

  2. Choose the Search button in Windows Help after the contents of the App Studio Help file have been displayed.

  3. Type the string "VBX" (without the quotes) into the Search word field. The topics related to using VBX controls can now be selected.

STATUS

Microsoft has confirmed this to be a bug in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available

MORE INFORMATION

To reproduce this problem, do the following:

  1. Start App Studio.

  2. From the File menu, choose Install Controls.

  3. Choose the Help button in the Install Controls dialog box. Help will start and the message "Help topic does not exist" will be displayed twice.

  4. After installing a VBX control, place it inside a dialog box.

  5. Double-click on the VBX control to bring up the properties box.

  6. Select the "?" button to start Help. Help will start and the message "Help topic does not exist" will be displayed twice.
