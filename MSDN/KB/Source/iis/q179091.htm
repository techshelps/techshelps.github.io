Multi-threaded Java Programs Stop on Multi-processor Computers

 Last reviewed: January 15, 1998
Article ID: Q179091
The information in this article applies to:
  • Microsoft Internet Information Server version 3.0

SYMPTOMS

A multi-thread Java application may stop responding when running on a multi- processor computer. The same application will run without problem on a single processor computer.

CAUSE

An internal Java Virtual Machine (VM) thread manipulation function is not multi-thread safe.

RESOLUTION

An updated version of Msjava.dll has been created to correct the problem. This hotfix can only be applied on top of Internet Explorer 4.0 or Java Software Development Kit (SDK) 2.0.

This fix should have the following timestamp: 

   9/30/97  5:54 PM               747,792

STATUS

Microsoft has confirmed this to be a problem in Microsoft Virtual Machine for Java. A supported fix is now available, but has not been fully regression-tested and should be applied only to systems experiencing this specific problem. Unless you are severely impacted by this specific problem, Microsoft recommends that you wait for the next Service Pack that contains this fix. Contact Microsoft Technical Support for more information.
