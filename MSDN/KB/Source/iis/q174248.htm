Index Server 1.1 Does Not Index German Compound Words

 Last reviewed: January 22, 1998
Article ID: Q174248

The information in this article applies to:

  • Microsoft Index Server version 1.1

SYMPTOMS

When you index German documents containing compound words with "ä", "ö", "ü", these compound words will not be indexed by Index Server version 1.1. When you search for those documents, you will get the following error message: 

   No documents found.

CAUSE

The wordbreaker DLL that splits compound words into single words only indexes these compound words.

RESOLUTION

Use the following files from Index Server version 2.0:

  • Sccfa.dll
  • Sccfi.dll
  • Sccifilt.dll
  • Sscut.dll

Copy them in the \Winnt\System32 directory and then re-index everything.
