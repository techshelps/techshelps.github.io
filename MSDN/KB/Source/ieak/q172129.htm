Security Options Not Set In Internet Explorer

 Last reviewed: February 10, 1998
Article ID: Q172129
The information in this article applies to:
  • Microsoft Internet Explorer Administration Kit, version 3.2a

SYMPTOMS

Security settings created in the Microsoft Internet Explorer Administration Kit (IEAK) wizard for a Win16 version of Internet Explorer are not set during the installation of Internet Explorer. The options affected by this are the Active Content settings for ActiveX and Java. These options cannot be deselected using the IEAK.

CAUSE

When the IEAK version 3.2 was released the Win16 browser did not have an .ini setting exposed to allow these settings to be made. This will be corrected in a future version of IEAK.

WORKAROUND

To work around this problem, manually edit the Iexplore.ini file to create the settings, and then distribute the Iexplore.ini file to the users.

STATUS

Microsoft has confirmed this to be a problem in IEAK version 3.2a. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
