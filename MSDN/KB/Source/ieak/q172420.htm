Active Setup Does Not Find Secondary Download Sites

 Last reviewed: February 9, 1998
Article ID: Q172420
The information in this article applies to:
  • Microsoft Internet Explorer Administration Kit 4.0

SYMPTOMS

Active setup does not find secondary download sites if the primary site is down or not present. For example:

  1. Create an install package with three download URLs.
2. Finish the wizard and move the files to a second download site, 
   but not the first and third.
3. When you run setup, it will fail.

CAUSE

Setup cannot find the .dat file for the first site, which causes the failure. Because setup can only get the download site list from one site, if it fails on the primary site, it cannot get the list from the secondary sites.

STATUS

Microsoft has confirmed this to be a problem in Internet Explorer Administration Kit version 4.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
