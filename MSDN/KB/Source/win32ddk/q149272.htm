DOC: UNICODE_STRING.Length Is in Characters, Not in Words

 Last reviewed: April 7, 1997
Article ID: Q149272
3.50 3.51 WINDOWS NT kbref kbdocerr kbdocfix

The information in this article applies to:

  • Microsoft Win32 Device Development Kit (DDK) for Windows NT, version 3.5, 3.51

SUMMARY

The Network Reference in the online documentation for the UNICODE_STRING structure states that its Length member specifies the current string length in words. This is incorrect; the length is in characters.

This documentation error has been corrected in the Windows NT DDK version 4.0 online documentation.
