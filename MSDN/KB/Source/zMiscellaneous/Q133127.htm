Crash Saving Document with TOC Field as WordPerfect 5.x Format

 Last reviewed: April 17, 1997
Article ID: Q133127
6.00 7.00 WINDOWS kb3rdparty buglist6.00 buglist7.00

The information in this article applies to:

  • Microsoft Word for Windows 95, version 7.0
  • Microsoft Word for Windows NT, version 6.0

SYMPTOMS

When you save a Word 6.0 for Windows NT or Word 7.0 document as a WordPerfect 5.x document, a general protection (GP) fault or an application error may result if the document contains a table of contents (TOC) field.

CAUSE

The above error occurs if the document contains a TOC field that was created using styles. This behavior does not happen if the TOC field was created using Table Entry (TC) fields. If both TC fields and styles were used, the error will still occur.

STATUS

Microsoft has confirmed this to be a problem with the WordPerfect converter supplied with Word version 6.0 for Windows NT and Word 7.0. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUND

When you have finalized the table of contents, select the table of contents and press CTRL+SHIFT+F9 to unlink the TOC field and cause the TOC results to become normal text.

NOTE: Once the TOC field is unlinked, it becomes normal text instead of a field, and you will not be able to update it.
