XL5: GPF/Type 11 Error Adjusting Margins While Recording Macro

 Last reviewed: December 1, 1997
Article ID: Q130389
The information in this article applies to:
  • Microsoft Excel for Windows, versions 5.0, 5.0c
  • Microsoft Excel for Windows NT, version 5.0
  • Microsoft Excel for the Macintosh, versions 5.0, 5.0a

SYMPTOMS

While you are recording a macro using the Visual Basic, Applications Edition, language in Microsoft Excel 5.0, if you adjust the document margins in print preview, one of the following may occur:

Microsoft Excel for the Macintosh, version 5.0

You receive the following error message: 

   Sorry, a system error occurred.
   "Microsoft Excel"
     error type 11

Microsoft Excel for Windows, version 5.0

You receive a general protection (GP) fault in module EXCEL.EXE. (Note that the memory address that appears in the GP fault error message varies depending on the printer driver that you are using.)

CAUSE

This problem occurs if you adjust the margins on a worksheet in print preview while a Visual Basic macro is being recorded. This problem does not occur if you record this action using the Microsoft Excel 4.0 macro language to record the macro. Additionally, this problem does not occur if you use the Page Setup dialog box to adjust the margin settings while the macro recorder is running.

WORKAROUND

To avoid receiving a GP fault or Type 11 error when you adjust the margins while recording a macro, use the Margins tab in the Page Setup dialog box (File menu) to change the margins instead of changing the margins in print preview.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Microsoft Excel for Windows 95, version 7.0 and later.
