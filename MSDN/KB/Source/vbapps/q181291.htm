WD98: InsertSymbol Result Not Translated Between Mac and ANSI

 Last reviewed: March 3, 1998
Article ID: Q181291
The information in this article applies to:
  • Microsoft Word 98 Macintosh Edition

SYMPTOMS

If you run a Visual Basic for Applications macro that inserts a character, a different character may be inserted when the macro is run on the Macintosh than when it is run on Windows.

CAUSE

Word currently does not support conversion of Macintosh CharNum values to equivalent ANSI values, and vice versa.

When a Visual Basic for Applications macro that includes the InsertSymbol Method is exchanged between Word 98 Macintosh Edition or later for the Macintosh and Word 97 or later for Windows, the numeric value of the CharNum or CharacterNumber command is not translated from its Macintosh character set value to its ANSI equivalent, and vice versa.

WORKAROUND

Edit the macro on the target platform, replacing the character value with the value that gives correct results.

In the following example, replace the number 203 with 192.

(ANSI 192, Capital A Grave, is the equivalent of Macintosh character 203.)

The following macro instructions insert the Capital A Grave character: 

   ' Word for Windows instruction
   Selection.InsertSymbol CharacterNumber := 192, Font:= "(normal text)"

   ' Macintosh Word instruction
   Selection.InsertSymbol CharacterNumber := 203, Font:= "(normal text)"

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

The following Word 98 Macintosh Edition example macro instruction, which inserts the character Capital A Grave (Macintosh character 203) in a Word document, is transferred unchanged to Word 97 for Windows: 

   Selection.InsertSymbol CharacterNumber := 203, Font:= "(normal text)"
If the above instruction is run in Word 97 for Windows, a CE Diaeresis (ANSI 203) is inserted instead of the intended Capital A Grave.
