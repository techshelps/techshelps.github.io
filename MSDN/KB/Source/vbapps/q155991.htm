WD97: Title Bar File Name Not Updated Using ActiveWindow.Caption

 Last reviewed: August 27, 1997
Article ID: Q155991
The information in this article applies to:
  • Microsoft Word 97 for Windows

SYMPTOMS

When you use the ActiveDocument.ActiveWindow.Caption statement in a Microsoft Visual Basic for Applications macro, Word fails to add the saved file name to the title bar in the document. Word saves the document, however, it does not update the title bar.

When you close and reopen the document, the title bar displays the correct file name.

WORKAROUNDS

To work around this behavior, use one of the following methods:

  • Do not use the Caption argument to set a caption on the title bar.

    -or-

  • Close and reopen the file.

STATUS

Microsoft has confirmed this to be a problem in the products listed at the beginning of this article. Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
