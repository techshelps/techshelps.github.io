PRB: Grouping by Wide Character Field Names Hangs Windows

 Last reviewed: July 14, 1997
Article ID: Q166379
The information in this article applies to:
  • Microsoft FoxPro for Windows, version 2.5b

SYMPTOMS

Using wide character field names for a grouping band or a Print When expression in a report created with a localized version of FoxPro for Windows 2.5b hangs the system when performing a report preview under Windows 3.2 (Simplified Chinese Version) and Windows 3.1 (Traditional Chinese Version).

RESOLUTION

Alter the wide character (Double Byte) field name into an English field name if the particular field is for grouping or for a Print When expression.

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Create a table under localized FoxPro 2.5b for Windows and Localized Windows 3.1

  2. Name the field with wide characters.

  3. Create an index based on that field.

  4. Create a report with a grouping based on that field.

  5. System hangs after choosing Report Preview from the Report menu pad.
