BUG: RUN Command Locks Windows NT Version 3.5 with GSNW

 Last reviewed: June 2, 1995
Article ID: Q124718
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.6a

SYMPTOMS

While running FoxPro for MS-DOS from a Novell Netware server under Windows NT version 3.5, issuing a RUN or ! command will lock up the NTVDM. The Process Viewer under Windows NT will not be able to kill this process. FoxPro will remain locked up until the computer is turned off and on to reboot it.

CAUSE

This behavior is related to a problem with the Redirector Services of the Windows NT 3.5 operating system.

STATUS

Microsoft has confirmed this to be a bug in Microsoft Windows NT version 3.5. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
