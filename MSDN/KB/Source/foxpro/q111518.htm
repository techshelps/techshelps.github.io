BUG: Extra Picture Characters Result in Garbage Characters

 Last reviewed: June 26, 1995
Article ID: Q111518
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5, 2.5a, 2.5b, 2.6, 2.6a

SYMPTOMS

If you put a field on a report form that has more picture characters in the format string then the actual string length of the field, garbage characters will be printed.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. USE any table.

  2. Create a memory variable by issuing the following command: 

          xx = "This is Text"

  3. Create a report form by issuing the following command: 

          CREATE REPORT test

  4. Add a field to the report's Page Header band. In the Report Expression dialog box, make the expression your memory variable (xx). For the format, use more "x"'s than the length of your string; in the case of xx, 15 "x"'s is sufficient.

  5. Preview the report form, and note the extra characters that are printed.
