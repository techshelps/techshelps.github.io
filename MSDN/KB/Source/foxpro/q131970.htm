BUG: Opening control3 Gives 'Memo file is missing/invalid'

 Last reviewed: December 3, 1996
Article ID: Q131970
The information in this article applies to:
  • Microsoft FoxPro for UNIX, version 2.6

SYMPTOMS

When opening control3.scx in /usr/lib/FoxPro/tutorial, the error message, 

'Memo file is missing/invalid' appears.

CAUSE

The corresponding memo file (control3.sct) is damaged.

RESOLUTION

Obtain a new copy of control3.scx and control3.sct from Microsoft's FTP site (/developr/fox/public/fpu/controls/).

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Start FoxPro for UNIX.

  2. In the Command window, type:

    MODIFY SCREEN /usr/lib/FoxPro/tutorial/control3.
