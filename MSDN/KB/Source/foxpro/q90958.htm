PRB: Workstation Reboots When Loading FoxPro from Server

 Last reviewed: December 22, 1997

The information in this article applies to:

  • Microsoft FoxPro for MS-DOS, version 2.0

SYMPTOMS

When FoxPro version 2.0 is loaded from the server, the workstation reboots.

CAUSE

The FOXPRO.OVL file on the local workstation has become corrupt.

RESOLUTION

Delete the copy of FOXPRO.OVL from the local workstation. Copy the FOXPRO.OVL file from the server to the workstation.

MORE INFORMATION

Having a copy of FOXPRO.OVL on the workstation can speed up performance.
