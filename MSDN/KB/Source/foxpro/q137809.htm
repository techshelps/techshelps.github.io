FIX: Insufficient Memory on a Computer That Has 512M of RAM

 Last reviewed: October 29, 1997
Article ID: Q137809
3.00 WINDOWS kbenv kbfixlist kbbuglist

The information in this article applies to:

  • Microsoft Visual FoxPro for Windows, version 3.0

SYMPTOMS

The error "Insufficient Memory" is displayed when you attempt to start a Visual FoxPro 3.0 session. The computer includes 512 megabytes of RAM and should have enough memory for Visual FoxPro to start.

NOTE: Using Visual FoxPro 3.0b, the insufficient memory message occurs on machines with 1G of RAM. The message does not occur on machines with 756M of RAM or less.

WORKAROUND

If you are using Windows NT, you can limit the amount of memory used by the operating system. You can create a boot (in the Boot.ini file) that uses a special configuration for Visual FoxPro. To limit the amount of memory Windows NT uses, use the MaxMem switch. Using a trial and error method, determine the maximum amount of memory you can use before you receive the "Insufficient Memory" error when launching Visual FoxPro.

For more information about multi-boot and the MaxMem switch, please see the Windows NT documentation.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Visual FoxPro 3.0b for Windows.

MORE INFORMATION

Steps to Reproduce Problem

  1. Install Visual FoxPro on a computer that has 512 megabytes of RAM.

  2. Start Visual FoxPro. The "Insufficient memory" error appears, and Visual FoxPro cannot start.
