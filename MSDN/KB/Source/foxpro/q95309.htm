How _PDPARMS Array Is Created for a Printer Driver Setup

 Last reviewed: April 29, 1996
Article ID: Q95309
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5, 2.5a
  • Microsoft FoxPro for Windows, versions 2.5, 2.5a

SUMMARY

The _PDPARMS array holds the escape sequences and font information for a printer driver setup. The _PDPARMS array is created when one of the following events occurs:

  • A SET PDSETUP TO <printer driver setup name> command is issued.
  • A _PDSETUP = <printer driver setup name> command is issued.
  • A printer driver is set in the Printer Driver Setup dialog box.
  • FoxPro is started with a default printer driver set beforehand.
  • A printer driver is set for a report (.FRX) and the report is run with the PDSETUP clause.
  • A printer driver is set for a label program (.LBX) and the label program is run with the PDSETUP clause.

If FoxPro does not have enough memory, the _PDPARMS array might not be created.
