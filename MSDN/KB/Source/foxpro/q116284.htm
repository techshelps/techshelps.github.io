BUG: "System Integrity" Error Occurs When Installing FoxPro

 Last reviewed: April 18, 1995
Article ID: Q116284
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.6

SYMPTOMS

A "System Integrity" error message occurs when you are installing FoxPro 2.6 for MS-DOS.

CAUSE

This error can be caused by a conflict with terminate-and-stay-resident (TSR) programs or by low-memory conditions.

RESOLUTION

If you are performing the installation from within an MS-DOS session window under Microsoft Windows, do one of the following:

  • Increase the amount of memory provided to the virtual MS-DOS session by editing the PIF file used to create the MS-DOS session.

    -or-

  • Quit Windows and then perform the installation at the MS-DOS prompt.

If neither of these options resolves the problem, perform a "clean boot" of the computer. For information about how to perform a clean boot, see the "Troubleshooting" topic in the FoxPro version 2.6 for MS-DOS online Help file, or see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q95314
   TITLE     : How to Perform a "Clean Boot" for FoxPro

STATUS

Microsoft has confirmed this to be a problem in FoxPro version 2.6 for MS- DOS. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

This error may also occur when a network installation is performed from a copy of FoxPro that was installed as an administrative installation.
