BUG: Certain Non-US Characters Do Not Display Properly in VFP

 Last reviewed: August 6, 1997
Article ID: Q167675
The information in this article applies to:
  • Microsoft Visual FoxPro for Windows, versions 3.0, 3.0b, 5.0, 5.0a on the following platforms: NT, Win95

SYMPTOMS

Certain non-US characters do not display properly in Microsoft Visual FoxPro. When these fonts are chosen in the display, Microsoft Visual FoxPro displays them in an English font.

STATUS

Microsoft has confirmed this to be a bug in the Microsoft products listed at the beginning of this article. We are researching this bug and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

The following information is specific to a particular font but other non-US fonts exhibit the same behavior. Fonts that are known to have display problems are listed at the end of this article. This is by no means meant to be a comprehensive list.

When storing information entered using the Greek language support in Windows 95 and Windows NT 4.0, Microsoft Visual FoxPro does not display the Greek characters properly.

The following instructions assume proper installation of Greek language support on your Windows 95 or Windows NT 4.0 system.

Steps to Reproduce Behavior

  1. Create a new project with a new database.

  2. Create a new table with two character fields; the first to store English characters, the second to store Greek characters.

  3. Create a form and place this table in the DATAENVIRONMENT.

  4. Drag each of the fields to the form to create text boxes.

  5. Click on the TextBox displaying the Greek characters.

  6. In the Properties box, change the FontName property to Arial Greek.

  7. Save and Run the form.

  8. Switch the international settings in Windows 95 to Greek and type in the Greek TextBox. Note that the characters do not display correctly.

  9. Highlight the text you typed into the Greek TextBox and copy it.

  10. Paste the text into Microsoft Word and change the font to Arial Greek. This will display properly.

Fonts That May Exhibit Display Problems

  • Greek
  • Ukrainian
  • Russian
  • Wingdings
