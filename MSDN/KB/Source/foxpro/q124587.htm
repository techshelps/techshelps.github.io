BUG: BROWSE LAST NOMODIFY May Be Modified After Second Command

 Last reviewed: June 27, 1995
Article ID: Q124587
The information in this article applies to:
  • Microsoft FoxPro for Windows, version 2.6a

SYMPTOMS

A memo field may be edited when a BROWSE LAST NOMODIFY command is issued and the memo edit window is left open when the Browse window is closed and a second BROWSE LAST NOMODIFY is issued.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Use the salesman database.

  2. Issue the BROWSE LAST NOMODIFY command.

  3. Open the memo window. Move or resize it, if you want. Note that the contents cannot be edited.

  4. Select the main Browse window, leaving the memo window open.

  5. Close the Browse window any way you like (press ESC or choose CLOSE).

  6. Issue a second BROWSE LAST NOMODIFY command.

  7. Select the memo window and edit the data.
