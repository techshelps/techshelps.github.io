PRB: GPF Or Stack Fault When Issuing DBConnect() to Oracle

 Last reviewed: April 30, 1996
Article ID: Q129398
The information in this article applies to:
  • Microsoft FoxPro for Windows, version 2.6a

SYMPTOMS

When connecting to Oracle using SQL*NET for TCP/IP, either a Stack Overflow or a general protection (GP) fault occurs with the DBConnect() command.

RESOLUTION

This may be resolved by upgrading your software to one of the following:

  • Upgrade FoxPro and its Connectivity Kit (CK) to version 2.6a.
  • Upgrade the SQL*NET for TCP/IP software to version 1.1.7.11.
  • If the Connectivity Kit is being used to connect to Oracle 7, then obtain the latest Oracle 7 ODBC Driver from Oracle or Microsoft. The latest Desktop Driver pack comes with an Oracle 7 Driver.

If the problem still occurs, follow the trouble shooting steps for a GP fault described in the FoxPro for Windows Help file.

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
