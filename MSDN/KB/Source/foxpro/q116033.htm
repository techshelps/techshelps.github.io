FIX: Can't Create Windows Help File from Word 6.0 RTF File

 Last reviewed: October 20, 1997
Article ID: Q116033
2.5x 2.6x WINDOWS kbtool kbprg kbfixlist kbbuglist kbfile

The information in this article applies to:

  • Microsoft FoxPro for Windows, versions 2.5x, 2.6x

SYMPTOMS

If you are using the Windows Help compiler that is included with the Distribution Kit, you cannot create a Windows Help file when you are using a Word 6.0 for Windows rich-text format (RTF) file.

CAUSE

Word 6.0 RTF files are not supported by the version of the Help Compiler that is shipped with the Distribution Kit.

RESOLUTION

Use a Word 2.0 RTF file or obtain an updated version of the Help Compiler.

NOTE: Microsoft Word 7.0 for Windows works correctly when making .rtf files for the help compiler.

To get version 3.10.505 of the Help Compiler, download VBHC505.EXE, a self- extracting file, from the Microsoft Software Library (MSL) on the following services:

  • Microsoft Download Service (MSDL) 

          Dial (206) 936-6735 to connect to MSDL
      Download VBHC505.EXE (size: 231170 bytes)
  • Internet (anonymous FTP) 

          ftp ftp.microsoft.com
      Change to the \SOFTLIB\MSLFILES directory
      Get VBHC505.EXE (size: 231170 bytes)

STATUS

Microsoft has confirmed this to be a problem in version 3.10.504 of the Windows Help Compiler. This problem was corrected in version 3.10.505 of the Help Compiler, which is shipped with Visual FoxPro 3.0 for Windows.

MORE INFORMATION

Version 3.10.504 of the Help Compiler is shipped with the Distribution Kit for the versions of Microsoft FoxPro listed above. This version of the Help Compiler does not support Word 6.0 RTF files. Version 3.10.505 of the Help Compiler does support Word 6.0 RTF files. To create a Help file with a Word 6.0 RTF file, you must obtain version 3.10.505 of the Help Compiler. To create a Help file with version 3.10.504 of the Help Compiler (the version included with the Distribution Kit), you must use a Word 2.0 RTF file.
