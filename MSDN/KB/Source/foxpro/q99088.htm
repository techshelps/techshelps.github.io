PRB: NOOVERWRITE Keyword Not Consistently Displayed in RQBE

 Last reviewed: April 29, 1996
Article ID: Q99088
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5, and 2.5a

SYMPTOMS

Using RQBE's Quick Report and Overwrite File options does not show the NOOVERWRITE clause in the See SQL window.

However, if the Do Query button is chosen, and immediately afterwards the See SQL button is used to look at the SQL statement, the NOOVERWRITE keyword will be present. If the Options dialog box is opened again, the Overwrite File box will be clear.

STATUS

This behavior is by design.

MORE INFORMATION

This design was chosen so that creating a quick report, making changes to it, then running the query would not overwrite the changes by default. Overwriting the file must be specifically requested each time.
