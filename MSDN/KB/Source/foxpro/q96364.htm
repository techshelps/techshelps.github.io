Including a .CDX File in a Project Gives Error

 Last reviewed: June 27, 1995
Article ID: Q96364
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5, and 2.5a
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

When a .CDX file is marked as included in a project while the database is excluded and an .EXE file is created from the .CDX file, the error message "File Access Denied" is displayed when you attempt to open the database with the USE command.

In the project, make sure to mark both the database and the index files as either included or excluded. To do this, select (highlight) the file in the project, and choose Include or Exclude from the Project menu. A circle with a line through the middle will appear next to the excluded files, which must be distributed with the .EXE file. If only one file is marked as included or excluded, the "File Access Denied" error message will be displayed.

The error occurs in this case because files that are included in a project are read-only, whereas excluded files are not.
