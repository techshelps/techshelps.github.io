READBORDER Setting Determined by NOLOGO Field in PJX File

 Last reviewed: August 28, 1995
Article ID: Q121633
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5x, 2.6, 2.6a
  • Microsoft FoxPro for Macintosh, versions 2.5x, 2.6a

SUMMARY

The Border For GETs check box under the More section in the Generate Screen dialog box places a T or F in the logical field called NOLOGO in the .PJX database.

MORE INFORMATION

Records of type "S" (without the quotation marks) are screen set records in the project. The logical field NOLOGO for these records will contain a T if the GETs for that screen set have borders or an F if the GETs for that screen set have no borders. The setting for this field is determined by the Border For GETs check box under the More options for the screen set.

The NOLOGO setting for fields of type "S" has no effect on the appearance of the logo when an executable file is run. The NOLOGO field for the record of type "H" controls this.
