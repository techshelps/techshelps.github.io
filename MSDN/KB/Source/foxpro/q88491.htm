Temporary Files Created by FoxPro

 Last reviewed: April 17, 1995
Article ID: Q88491
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5x, 2.6, 2.6a

Microsoft FoxPro for MS-DOS creates a number of temporary files for its own use. Each of these files is deleted when the Quit command is used to terminate FoxPro for MS-DOS normally. While some of these files are created when FoxPro for MS-DOS first starts, others are created throughout the session. The temporary files are as follows:

  • One file created for each Debug window
  • One file created for each Browse window
  • Two files created for FOXUSER.DBF and FOXUSER.FPT resource files
  • Two files created for FOXHELP.DBF and FOXHELP.FPT help files if help is used
  • One file created for commands entered in the Command window
  • One file created for each text or program file edited, in addition to the text or program file itself
  • One file created for each database
  • Two files created for each database with a memo field
  • Three files created for each database with a memo field and a structural index
