PRB: Extended FoxPro Gives "Incompatible Memory Manager" Error

 Last reviewed: April 17, 1995
Article ID: Q87686
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5 and 2.5a

SYMPTOMS

Although the standard version of Microsoft FoxPro runs without any problems, the extended version gives an "Incompatible memory manager" error.

CAUSE

This error has two possible causes:

  1. The memory manager is not completely compatible with the Lotus/Intel/Microsoft (LIM) Expanded Memory Specification (EMS) version 4.0.

  2. The memory manager is configured incorrectly.

RESOLUTION

Ensure that your memory manager creates LIM-4.0-compatible EMS memory. Please refer to the "Developer's Guide" and the FoxPro on-line help file for more information about the extended version of FoxPro and memory managers.
