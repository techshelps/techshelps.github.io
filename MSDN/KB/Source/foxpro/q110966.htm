FIX: Simultaneous Appends from a Browse May Corrupt Data

 Last reviewed: October 20, 1997
Article ID: Q110966
2.50 WINDOWS kbprg kbfixlist kbbuglist

The information in this article applies to:

  • Microsoft FoxPro for Windows, version 2.5

SYMPTOMS

Under certain rare circumstances, multiple users appending blank records from a browse at exactly the same time causes data corruption. This corruption is caused by the end-of-file (EOF) marker (ASCII character 26) being written into the record's delete marker field.

STATUS

Microsoft has confirmed this to be a problem in FoxPro 2.5 for Windows. This problem was corrected in FoxPro 2.5a for Windows.

MORE INFORMATION

The programs, databases, and macros used to reproduce this problem are too large to include in this article.
