PRB: Choosing to Print When No Printer Makes Computer Hang

 Last reviewed: May 16, 1996
Article ID: Q123801
The information in this article applies to:
  • Microsoft FoxPro for Macintosh, version 2.6a

SYMPTOMS

When you attempt to print a document, the computer hangs (stops responding).

NOTE: This behavior does not occur in Visual FoxPro 3.0b for Macintosh.

CAUSE

There is no printer selected in the Macintosh Chooser.

RESOLUTION

Make sure you have selected an active printer from the Chooser in the Apple menu before you attempt to print a document.

STATUS

This behavior is by design.

MORE INFORMATION

This behavior is not limited to Microsoft FoxPro for Macintosh. This behavior happens in any application that prints to a printer port with no physical connection.

Steps to Reproduce Behavior

  1. Choose Chooser from the Apple menu.

  2. Select a printer from the Chooser that is not on the system making sure to set the Connect To option to a port that is not in use by a printer.

  3. Close the Chooser, and start TeachText.

  4. Try to print any text from the Simple Text document.
