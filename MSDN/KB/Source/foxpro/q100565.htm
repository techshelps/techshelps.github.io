Unhiding the Command Window in FoxPro Ver 2.5 for Windows

 Last reviewed: June 27, 1995
Article ID: Q100565
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

SUMMARY

The Command window must be unhidden differently in FoxPro for Windows than in FoxPro version 2.x for MS-DOS, as demonstrated in the example below.

MORE INFORMATION

In the example below, before hiding the Command window, an ON KEY LABEL command is set to run the program UNHIDE, which will restore the window. Normally, this program would be called programmatically from a menu option or valid routine.

  1. In the Command window, type: 

          ON KEY LABEL F12 DO unhide

  2. Create the program, named UNHIDE.PRG: 

          ACTIVATE WINDOW hidecomm
      ACTIVATE WINDOW command IN SCREEN
      RELEASE WINDOW hidecomm

  3. Follow the standard method of hiding the Command window, as in FoxPro for MS-DOS: 

          DEFINE WINDOW hidecomm FROM 1,1 TO 3,3
      ACTIVATE WINDOW command IN hidecomm

  4. When you are satisfied that the Command window is hopelessly irretrievable, press F12 to restore the window.
