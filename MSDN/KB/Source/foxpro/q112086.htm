BUG: "SQL Internal Error" If BETWEEN() in WHERE Clause

 Last reviewed: June 27, 1995
Article ID: Q112086
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5, 2.5a, 2.5b, 2.6
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5, 2.5a, 2.5b, 2.6
  • Microsoft FoxPro for Macintosh, version 2.5b

SYMPTOMS

Executing a SELECT statement that has the BETWEEN() function in the WHERE clause will generate a "SQL Internal error" message under these conditions:

  • There is a function nested in the BETWEEN() function.
  • The parameters of the BETWEEN() function are from two or more tables.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

Execute the following code from a program: 

   CREATE TABLE main1 (num C(2))
   INSERT INTO main1 (num) VALUES ('05')
   SELECT 0
   CREATE TABLE rel1 (num C(2))
   INSERT INTO rel1 (num) VALUES ('01')
   SELECT * FROM main1,rel1 WHERE ;
        BETWEEN(ALLTRIM(main1.num),rel1.num,rel1.num)
