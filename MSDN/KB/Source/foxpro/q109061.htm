PRB: All Layout Commands on the Label Menu Are Disabled

 Last reviewed: April 18, 1995
Article ID: Q109061
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 1.02, 2.0, 2.5, 2.5a, and 2.5b

SYMPTOMS

The Layout, Save Layout, and Delete Layout commands on the Label menu are disabled.

CAUSE

The RESOURCE statement is set to OFF. The Label Writer uses the resource file (usually FOXUSER.DBF and its associated memo file, FOXUSER.FPT) to store label layouts. If no resource file is in use, the Label Writer cannot load or delete existing layouts, or store new ones.

RESOLUTION

To use the default resource file, issue the following command: 

   SET RESOURCE ON
To set a specific resource file, issue the command: 

   SET RESOURCE TO <filename>

MORE INFORMATION

Steps to Reproduce Problem

  1. Start FoxPro for MS-DOS.

  2. Issue the following commands: 

          SET RESOURCE OFF
      CREATE LABEL

  3. Open the Label menu.

Note that the Layout, Save Layout, and Delete Layout commands are disabled.
