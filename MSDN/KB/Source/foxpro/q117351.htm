PRB: Page Number Doesn't Reset When at Top of Report in FPW

 Last reviewed: June 27, 1995
Article ID: Q117351
The information in this article applies to: 

  - Microsoft FoxPro for Windows, version 2.5

SYMPTOMS

When you are printing or previewing a report with a data group and the "New Page (Page No. = 1)" check box is selected and the _PAGENO system memory variable is placed in the Page Header band, the first group will start with page 1 but the other groups will begin with page 2. If _PAGENO is in the Page Footer band, it will reset correctly.

RESOLUTION

Upgrade to FoxPro version 2.5a or 2.5b for Windows. This problem does not occur in FoxPro version 2.5a or later for Windows.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Create a new report.
2. Open the INVOICES database from the TUTORIAL subdirectory. 3. Create a quick report. 4. Create a group on the CNO field. 5. Select the "New Page (Page No. = 1)" check box. 6. Drag the page number field from the Page Footer to the Page Header. 7. From the Report menu, choose Page Preview, and then choose Next.
