PRB: "Object File Is the Wrong Version" Error

 Last reviewed: June 27, 1995
Article ID: Q93335
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a
  • Microsoft FoxPro for MS-DOS, versions 2.5 and 2.5a

SYMPTOMS

An attempt to run an object file (with the .APP, .EXE, or .FXP extension) in FoxPro version 2.5 fails with an "Object file <filename> is the wrong version" error message.

CAUSE

The object code format for FoxPro versions 2.5 and 2.5a differs from that of earlier versions.

This error can also occur if the FoxPro version 2.0 Distribution Kit is installed after FoxPro version 2.5 has already been installed and you try to run the application after successfully building the executable file.

RESOLUTION

To work around this problem, obtain the source code files (for example, the .PRG and .SCX files) and rebuild the application for FoxPro version 2.5 or 2.5a.
