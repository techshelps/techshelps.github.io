FIX: CTRL+LEFT/RIGHT ARROW Doesn't Work in Screen Prompts

Article ID: Q113321

The information in this article applies to:

  • Microsoft FoxPro for Windows, versions 2.5x, 2.6x
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5x, 2.6x
  • Microsoft FoxPro for Macintosh, versions 2.5x, 2.6x

SYMPTOMS

Pressing the CTRL+LEFT ARROW or CTRL+RIGHT ARROW key combinations in the Prompt text box of a push button, radio button, or popup in the Screen Builder does not move the insertion point one word to the left or right.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Visual FoxPro 3.0 for Windows.

MORE INFORMATION

NOTE: The CTRL+LEFT ARROW or CTRL+RIGHT ARROW key combinations do work properly in the Check Box Prompt text box.

The SET KEYCOMP command does not affect this behavior.

Steps to Reproduce Behavior

  1. Type the following in the Command window to start the Screen Builder: 

          CREATE SCREEN

  2. Place a push button on the screen. (In FoxPro for Windows and Macintosh, select the Push Button tool from the toolbox. In FoxPro for MS-DOS, choose Push Button from the Screen menu.)

  3. In the text box below Push Button Prompts, type "Test text" (without the quotation marks).

  4. Press CTRL+LEFT ARROW.

Note that the insertion point does not move to the left one word as expected.
