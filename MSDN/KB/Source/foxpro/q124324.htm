BUG: Project Manager "Save As Default" Won't Save All Settings

 Last reviewed: June 27, 1995
Article ID: Q124324
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.5, 2.5a, 2.5b, 2.6, 2.6a
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5, 2.5a, 2.5b, 2.6, 2.6a
  • Microsoft FoxPro for Macintosh, version 2.5b, 2.5c, 2.6a

SYMPTOMS

When you define default settingd in the Option dialog box in Project Manager by using the 'Save As Default' button, the following settings are not saved as the default:

  • The "Display Logo When Application is Run" check box in FoxPro for Windows.
  • The "Logo" check box in FoxPro for MS-DOS.
  • The Home Directory setting in FoxPro for Windows, MS-DOS, and Macintosh.

NOTE: The rest of the settings are saved as expected. FoxPro for the MacIntosh does not have a Logo check box.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. Start FoxPro, and bring up a new project manager.

  2. Click the Project menu pad, and select the Option menu option.

  3. In the Option dialog box, select the Home Directory. Then change to a different directory such as C:\FPW26\TUTORIAL.

  4. In the Project Option group, Clear the Display Logo When Application is Run check box in FoxPro for Windows or the Logo check box in FoxPro for MS-DOS.

  5. Click the Save As Default button to save this setting as the default. Then click OK to return to the Project Manager.

  6. Save the project with a name of your choosing and close it.

  7. Reopen a new Project Manager, and go into the Option dialog box. You'll see that the check box for the Display Logo When Application is Run is checked again, and the Home Directory is back to the C:\FPW26.
