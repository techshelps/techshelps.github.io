PRB: FoxPrint Character Descenders Not Displayed in Text Box

 Last reviewed: February 23, 1996
Article ID: Q147211
The information in this article applies to:
  • Microsoft Visual FoxPro for Windows, versions 3.0 and 3.0b
  • Microsoft FoxPro for Windows, Versions 2.6 and 2.6a

SYMPTOMS

Text that is formatted for FoxPrint and is displayed in a text field in a Visual FoxPro 3.0 form or a Get field in a FoxPro version 2.6 or 2.6a for Windows Screen will not display the descender of lowercase letters. The letters with descenders are q, y, p, g, and j.

STATUS

Microsoft is researching this behavior and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Create a new form, and place a text box on the form.

  2. Change the Value property of the text box to agajaqapay.

  3. Change the font to FoxPrint.

  4. Save and run the form. Notice that the descenders are not displayed. The font size does not seem to have an effect on this behavior.

Additional reference words: 3.00 3.00b VFoxWin 2.60 2.60a
