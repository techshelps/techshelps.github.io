PRB: Alias Shows Work Area Letter, Not Database Name

 Last reviewed: June 27, 1995
Article ID: Q93326
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5, and 2.5a
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

SYMPTOMS

Choosing View from the Window menu displays the work area alias instead of the actual database name. The same results occur with the ALIAS() function.

CAUSE

The following situations may cause this to occur:

  • A database was opened as follows: 

          USE <database name> AGAIN
  • The database name not begin with an alphabetic character.
  • The database name contains a hyphen (-) instead of an underscore (_).
  • The database name contains one of the following special characters: 

           ! @ # $ % ^ ( )

RESOLUTION

Use the corresponding method below to address this situation:

  • Use the ALIAS modifier to specify an alias for the database, as follows: 

          USE <database name> AGAIN ALIAS <alias name>
  • Rename the database so that the first character of the name is alphabetic.
