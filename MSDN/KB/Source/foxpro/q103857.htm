FIX: Running FoxPro Extended 2.5 with MS-DOS 6.0 and EMM386

 Last reviewed: September 22, 1997
Article ID: Q103857
2.50 MS-DOS kbprg kbfixlist kbbuglist

The information in this article applies to:

  • Microsoft FoxPro for MS-DOS, version 2.5

SYMPTOMS

When you are running MS-DOS version 6.0 and EMM386, the Extended version of FoxPro for MS-DOS can take over 1 minute to start up.

CAUSE

This problem is caused by differences in the ways that EMM386 and FoxPro allocate and report extended memory (XMS). This does not cause any problems other than slow startup.

RESOLUTION

To avoid slow startup, you can do one of the following:

  • Eliminate EMM386. FoxPro can still find and use all available memory.

    -or-

  • Upgrade to FoxPro 2.5a for MS-DOS.

STATUS

Microsoft has confirmed this to be a problem in FoxPro 2.5 for MS-DOS. This problem was corrected in FoxPro 2.5a for MS-DOS.
