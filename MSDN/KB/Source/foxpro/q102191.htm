FIX: Unable to Link Word for Windows Documents

 Last reviewed: September 22, 1997
Article ID: Q102191
2.50 2.50a WINDOWS kbinterop kbprg kbbuglist kbfixlist

The information in this article applies to:

  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

SYMPTOMS

You receive the error message "Word files must be embedded" when using the APPEND GENERAL LINK command to link a Microsoft Word 2.0 for Windows document. However, when you attempt to insert a Word document and link the file through the interface, the file is embedded without an error message being displayed.

RESOLUTION

Use Word 6.0 for Windows instead of Word 2.0 for Windows. Word 6.0 documents can be pasted and paste linked both as Word 6.0 objects or pictures. The APPEND GENERAL <general field name> FROM <Word filename> LINK works with Word 6.0.

The documents do not appear as Word icons. The face of the document appears in the general field, as do those of Microsoft Excel documents.

STATUS

Microsoft has confirmed this to be a problem in Word 2.0 for Windows. This problem was corrected in Word 6.0 for Windows.
