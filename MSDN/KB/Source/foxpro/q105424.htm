PRB: Unable to Install Printer Driver

 Last reviewed: April 18, 1995
Article ID: Q105424
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5, and 2.5a

SYMPTOMS

When you are attempting to SET a printer driver setup, the error message "Unable to Install Printer Driver" appears. This problem may occur in any one of the following situations: 

   _PDSETUP=<setup name>
   LABEL FORM <label> TO PRINT PDSETUP
   REPORT FORM <report> TO PRINT PDSETUP

     -or-

   On startup of FoxPro

CAUSE

After a printer driver setup has been established, the contents of the P_NAME field of the P_CODES database have been changed and GENPD.APP has been rebuilt so that GENPD can no longer find the correct printer name.

RESOLUTION

To resolve this problem, do one of the following:

  • Re-create the printer driver setup (choose Printer Setup from the File menu, choose Printer Driver Setup, and then choose Edit).

    -or-

  • Change the name of the printer in P_CODES.DBF in GENPD.PJX to the previous name and rebuild GENPD.APP.
