BUG: Error When Printing MS-DOS Labels Under Windows

 Last reviewed: April 30, 1996
Article ID: Q114140
The information in this article applies to:
  • Microsoft FoxPro for Windows, versions 2.6

SYMPTOMS

Labels created in the FoxPro for MS-DOS Catalog Manager cannot be run in FoxPro for Windows without first being transported using the FoxPro for Windows Catalog Manager. If you try to print or preview a label created in FoxPro for MS-DOS, the following error will occur: 

   Report File WIZSTCPCO invalid

WORKAROUND

The FoxPro for MS-DOS label must be transported to FoxPro for Windows. For example:

  1. Open the Catalog Manager in FoxPro for Windows and choose Open Catalog from the File menu.

  2. Use the MS-DOS sample catalog in the FPD26\SAMPLE\CATALOG subdirectory.

  3. Select the Customer label and choose the Modify button.

    The Transport dialog box will appear.

  4. Choose Transport And Open.

    The file is transported to FoxPro for Windows and opened.

  5. Choose Print and either Print or Preview.

The error will not occur.

STATUS

Microsoft has confirmed this to be a problem in FoxPro 2.6 for Windows. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

This behavior occurs only with labels. Report files do not return an error.

Steps to Reproduce Problem

  1. Open the Catalog Manager in FoxPro for Windows and choose Open Catalog from the File menu.

  2. Use the MS-DOS sample catalog in the FPD26\SAMPLE\CATALOG subdirectory.

  3. Select the Customer label and choose the Print button.

  4. Select either Page Preview or Printer, and then choose OK.

The error "Report File WIZSTCPCO invalid" appears.
