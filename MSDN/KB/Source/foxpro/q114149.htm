FIX: Modifying Wizard-Created Item Not Invoking Wizard

 Last reviewed: October 20, 1997
Article ID: Q114149
2.60 MS-DOS kbtool kbbuglist kbfixlist

The information in this article applies to:

  • Microsoft FoxPro for MS-DOS, version 2.6

SYMPTOMS

When you modify a wizard-generated screen, report, or label, the item is opened in the Screen Builder, Report Writer, or Label Designer, respectively, rather than in the wizard that was initially used to create the item, even if no modifications have been made to the item outside the wizard.

STATUS

Microsoft has confirmed this to be a problem in FoxPro 2.6 for MS-DOS. This problem was corrected in FoxPro 2.6a for MS-DOS.

MORE INFORMATION

Steps to Reproduce Problem

  1. In the Catalog Manager of FoxPro for MS-DOS, create and save a new screen, report, or label using the appropriate wizard.

  2. In the Catalog Manager, select the screen, report, or label you just created and choose Modify.

The appropriate FoxPro power tool, not the wizard, will be launched for the modification of the item.
