Error in Documentation for UPDATE Command

 Last reviewed: June 27, 1995
Article ID: Q105145
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5, 2.5a
  • Microsoft FoxPro for Windows, versions 2.5, 2.5a

The UPDATE command updates fields in the current table/.DBF with data from a table/.DBF open in another work area. A portion of the syntax, as stated on page L3-1104 of the "Language Reference," is as follows: 

     UPDATE ON <field> FROM <expN> | <expC> ...
As shown above, the FROM table can be specified with the work area number of the table. Doing this will, however, cause a syntax error. For example, if the CUSTOMER table has been opened into work area 1, the following command will result in a syntax error: 

     UPDATE ON <field> FROM 1 ...
To avoid this problem, specify the alias name of the table instead of the work area number, as follows: 

     UPDATE ON <field> FROM customer ...
