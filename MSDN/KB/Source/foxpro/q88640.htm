FIX: "Record Is Not Locked" Exiting Numeric GET Field

 Last reviewed: April 17, 1995
Article ID: Q88640
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, version 2.0

SYMPTOMS

In Microsoft FoxPro version 2.0 for MS-DOS, if a user exits a numeric GET field by pressing the ENTER key, a zero is placed in the field and the error message "Record is not Locked" is displayed.

CAUSE

This error would only happen if the value in the GET field was not the full width of the field and the code option READ NOLOCK was marked in the Screen Builder.

STATUS

Microsoft has confirmed this to be a problem in Microsoft FoxPro version 2.0 for MS-DOS. This problem was corrected in Microsoft FoxPro version 2.0 for MS-DOS dated March 25, 1992.
