BUG: Application Generator Doesn't Default to USEd Table/DBF

 Last reviewed: June 27, 1995
Article ID: Q114675
The information in this article applies to:
  • Microsoft FoxPro for Windows, version 2.6
  • Microsoft FoxPro for MS-DOS, version 2.6

SYMPTOMS

The Application Generator does not default to the table/.DBF file that is already being USEd in Catalog Manager.

CAUSE

The Application Generator calls the FOXAPP.APP program, which does not automatically use an open table/.DBF file.

STATUS

Microsoft has confirmed this to be a problem in FoxPro version 2.6 for Windows and FoxPro version 2.6 for MS-DOS. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. If the Catalog Manager is not displayed, choose Catalog Manager from the File menu.

  2. If you are prompted to choose a catalog, choose New. If a catalog opens automatically, choose New Catalog from the File menu.

  3. Save the catalog as TEST.FPC.

  4. Select the Table tab. Choose Add.

  5. Select the CUSTOMER.DBF file from the TUTORIAL subdirectory in the main FOXPRO directory. Choose Add, then choose Use.

  6. Select the Program tab. Choose New.

  7. Choose Application Generator.

    The first dialog box in the Application Generator requests the name of a table to use.
