PRB: "Sharing Violation on Drive X:" with Virus Protection

 Last reviewed: April 17, 1995
Article ID: Q93352
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5

SYMPTOMS

When virus-protection software is installed and the user runs an executable file created with FoxPro, a "Sharing violation on drive X:" error message occurs.

RESOLUTION

There are two ways to address this problem:

  • Use the MS-DOS ATTRIB command to set the Read-Only attribute for the executable file.

    -or-

  • Disable the anti-virus software by editing the AUTOEXEC.BAT file to remove VSAFE.COM or by editing the CONFIG.SYS file to remove VSAFE.SYS.
