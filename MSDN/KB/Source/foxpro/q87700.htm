PRB: "Out of Memory" When Running FoxDoc

 Last reviewed: April 17, 1995
Article ID: Q87700
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.0, 2.5x, 2.6, 2.6a

SYMPTOMS

An "Out of memory" error occurs while you are running FoxDoc from within FoxPro for MS-DOS.

CAUSE

FoxDoc uses only conventional memory, so this error means that the conventional memory (memory below 640K) is full.

RESOLUTION

To avoid this problem, modify the CONFIG.SYS and AUTOEXEC.BAT files to free conventional memory by removing unneeded drivers and other programs that may be loading in conventional memory.

If you cannot free enough conventional memory, try running FoxDoc as a stand-alone .EXE file instead of running it from within FoxPro for MS-DOS.
