Space Missing in Program Example for \|\\ Command

 Last reviewed: June 27, 1995
Article ID: Q96340
The information in this article applies to:
  • Microsoft FoxPro for MS-DOS, versions 2.5 and 2.5a
  • Microsoft FoxPro for Windows, versions 2.5 and 2.5a

The example for the \|\\ command on page L3-15 of the "Language Reference" is missing a space between the month and day of the date. The example appears as follows: 

   USE customer
   SET TEXTMERGE ON
   SET TEXTMERGE TO letter.txt
   \<<CDOW(DATE())>>, <<CMONTH(DATE())>>
   \\<<DAY(date())>>, <<YEAR(DATE())>>
   \
   \
   \Dear <<contact>>
   \
   \Additional Text
   \
   \Thank you,
   \
   \XYZ Corporation
   CLOSE ALL
   MODIFY FILE letter.txt NOEDIT
In the program example above, the day of the date will be printed immediately after the month; for example, January20, 1993. In order for the date to print out in the correct format, a space must be added right before <<DAY(DATE())>>. The fifth line in the program example above should be as follows: 

   \\ <<DAY(DATE())>>, <<YEAR(DATE())>>
    (^ note the added space).
