FIX: Searching Online Books Is Slow in Developer Studio

 Last reviewed: October 21, 1997
Article ID: Q168372
The information in this article applies to:
  • Microsoft Visual Studio 97
  • The development environment included with: - Microsoft Visual Studio 97 - Microsoft Visual C++, 32-bit Editions, version 5.0 - Microsoft Visual J++, version 1.1 - Microsoft Visual InterDev, version 1.0

SYMPTOMS

Performing a query in the online Books is slow in Developer Studio.

CAUSE

There is a bug that causes search time to increase excessively as more books are added to the online content. The search degradation can be significant if all of the online Books are installed (installing the MSDN library).

RESOLUTION

Search in subsets instead of the entire contents of the Books online. 

 -or-
Install the April 97 release of the MSDN Library CD-ROM.

STATUS

Microsoft has confirmed this to be a bug in the Microsoft products listed at the beginning of this article. This bug has been corrected in MSDN Library CD, April 97 release.

MORE INFORMATION

The April 97 MSDN library has two changes that, together, fix this problem:

  • an update to the search software.

    -and-

  • a new query file on the CD containing references to all of the text.

Neither of these changes is any good without the other. When you install the MSDN April 97 release, it will update the search software in InfoViewer to look for and use the IVQ file, and it also includes the IVQ file on the new MSDN CDs. The IVQ (InfoViewer Query) file is a new file format introduced in the April 97 release.
