INF: Sample ODBC.INF for ODBC Drivers Shipped in Access 7.0

 Last reviewed: September 9, 1996
Article ID: Q139373
The information in this article applies to:
  • Microsoft Open Database Connectivity, versions 2.5 and 3.0

SUMMARY

Microsoft SQL Server Driver 2.50.0126, and Desktop Database Drivers 3.0.2111 ship with Microsoft Access 7.0, but there is no ODBC 2.0-compliant ODBC.INF file in the package. This article gives a sample ODBC.INF designed for those drivers.

COMMENTS:

  1. All the files mentioned in an ODBC.INF file should be copied into the redistribution package. Most of the files mentioned in the above sample ODBC.INF can be found in the following directory on your Access 7.0 CD: 

          \OS\SYSTEM

    You can search for the remaining file names in ACC95.INF, which will give you the directory containing the file you are looking for.

  2. If you do not want to write your own driver setup program, you can "borrow" _BOOTSTP.EXE, _MSSETUP.EXE, ODBCST32.EXE, SETUP.EXE, and SSETUP.LST from SQLSRV32.EX via the following services:

    - Anonymous FTP via the Internet 

         ftp ftp.microsoft.com
     Change to /bussys/sql_odbc/odbc-public directory
     Get SQLSRV32.EXE

  3. Please refer to the "ODBC 2.0 Programmer's Reference and SDK Guide" for documentation on ODBC.INF files.
