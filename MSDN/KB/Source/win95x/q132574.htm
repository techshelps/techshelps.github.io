File Does Not Open When You Drag the File to a Shortcut

 Last reviewed: August 8, 1996
Article ID: Q132574
The information in this article applies to:
  • Microsoft Windows 95
  • Microsoft Windows NT Server version 4.0
  • Microsoft Windows NT Workstation version 4.0

SYMPTOMS

When you drag a file from File Manager to an application shortcut on the desktop, you receive no error, but the file does not open and the mouse pointer does not change to indicate that the action is invalid.

CAUSE

This is a limitation of File Manager; File Manager is a Windows 3.1-based application and does not support the enhanced features of OLE drag and drop found in Windows 95.

RESOLUTION

Make sure the program on the desktop is open before you drag a file from File Manager to the shortcut.

-or-

Drag the file from Windows Explorer to the shortcut.
