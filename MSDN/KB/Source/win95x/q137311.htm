VLM Client Generates SetDrivePath Error 85

 Last reviewed: September 26, 1995
Article ID: Q137311
The information in this article applies to:
  • Microsoft Windows 95

SYMPTOMS

When you try to assign trustee rights or create a user directory on a NetWare server using the Syscon tool, you may receive the following error message: 

   SetDrivePath returned error 85

CAUSE

This problem can occur if you are using an outdated version of the VLM network client.

RESOLUTION

To correct this problem, obtain and install the latest VLM client files (version 4.1 or later) from Novell.
