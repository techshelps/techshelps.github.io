OFFICE: No ExcelFile/dBASEFile Options Under Q+E File Save As

 Last reviewed: March 12, 1997
Article ID: Q119025
4.00 WINDOWS kbusage

The information in this article applies to:

  • Microsoft Office for Windows, version 4.0
  • Q+E for Microsoft Excel for Windows, version 3.0a

SUMMARY

When you run Q+E version 3.0a, installed from Microsoft Office 4.0 Standard or Professional edition, the ExcelFile and dBASEFile Options are not listed in the Destination box of the Save As dialog box (the dialog box that appears in Q+E when you choose Save As from the File menu). The options listed in the Destination box are the following: 

   MailingLabels
   PrintToFile
   QueryFile
   TextFile
NOTE: The ExcelFile and dBASEFile options are listed correctly when you run Q+E 3.0a installed from the stand alone Microsoft Excel versions 4.0 and 4.0a.

WORKAROUND

If you have purchased Microsoft Office 4.0, you are entitled to receive Microsoft Excel 5.0 which includes Microsoft Query, a replacement for Q+E. Similar to Q+E, Microsoft Query is a tool that you can use to organize and retrieve data from a variety of sources.

If you want to use Q+E, you should request stand-alone Microsoft Excel 4.0 disks and install Q+E from these disks. Both version 4.0 and 5.0 disks can be obtained from the Microsoft Sales Information Center by calling (800) 426-9400.

MORE INFORMATION

Steps to Reproduce Behavior

  1. In Program Manager, double-click the Q+E icon in the Microsoft Office program group.

  2. Open a .DBF, .QEF, or .XLS file.

  3. From the File menu, choose Save As.

  4. Click the down arrow in the Destination box.

The list contains MailingLabels, PrintToFile, QueryFile, TextFile. This list should also contain ExcelFile and dBASEFile.
