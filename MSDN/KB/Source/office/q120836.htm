Office 4.3 Err Msg: "Clipart Gallery File Cannot Be Read..."

 Last reviewed: March 12, 1997
Article ID: Q120836
4.30 WINDOWS kbtools kbole kberrmsg

The information in this article applies to:

  • Microsoft Office for Windows, version 4.3
  • Microsoft ClipArt Gallery

SYMPTOMS

When you add clip art to the Microsoft Clipart Gallery after installing Office 4.3, you will receive the following error message: 

   Sorry, the file x:\MSOFFICE\OFF43ART.IDX cannot be read. This
   file may be installed incorrectly, someone may be editing this
   file, or you may be having hardware problems.

CAUSE

On the disks, the files OFF43ART.IDX and OFF43ART.THM have been incorrectly named OFF42ART.IDX and OFF42ART.THM, respectively.

RESOLUTION

Workaround #1 (Stand Alone)

After you complete the Office installation, rename OFF42ART.IDX and OFF42ART.THM to OFF43ART.IDX and OFF43ART.THM, respectively. Once you rename these files, you should be able to add Clipart to the Clipart Gallery without an error. These files are located in the MSOFFICE\CLIPART directory.

Workaround #2 (Server Installation)

To stop this error from occurring when you install from a server, you need to apply Workaround #1 and then modify the SETUP.INF of the server installation.

WARNING: Modification of the SETUP.INF file as described in this article is at your own risk. Microsoft does not support modifications of the SETUP.INF file to suit specific customer requirements.

  1. Make a back-up copy of SETUP.INF called SETUP.BAK.

  2. In Microsoft Write, open SETUP.INF, and select the No Conversion option. In SETUP.INF, search for OFF42ART. You should find the following text: 

          21,OFF42ART.IDX,,,,,,,NEVER,,,,,,,31104,,,,,
      21,OFF42ART.THM,,,,,,,NEVER,,,,,,,170048,,,,,

    Modify these lines to read: 

          21,OFF43ART.IDX,,,,,,,NEVER,,,,,,,31104,,,,,
      21,OFF43ART.THM,,,,,,,NEVER,,,,,,,170048,,,,,

  3. Save the file as Text Only, and quit Microsoft Write.
