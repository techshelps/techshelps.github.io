OFF97: Error Using Image Document Object under Windows NT 4.0

 Last reviewed: April 30, 1997
Article ID: Q167723
The information in this article applies to:
  • Microsoft Office 97 for Windows
  • Microsoft Excel 97 for Windows
  • Microsoft Word 97 for Windows
  • Microsoft PowerPoint 97 for Windows
  • Microsoft Windows NT Workstation version 4.0

SYMPTOMS

In any of the programs listed at the beginning of this article, when you insert or double-click an Image Document object, you may receive one or more of the following error messages: 

   Microsoft Word 97
   -----------------

   Word cannot edit the Image Document.

   -or-

   The server application, source file, or item cannot be found. Make
   sure the application is properly installed, and that it has not been
   deleted, moved, or renamed.

   Microsoft Excel 97
   ------------------

      Cannot start the source application for this object.

   Microsoft PowerPoint 97
   -----------------------

      This presentation contains open embedded objects that can't be closed
      automatically. Close the objects and try again.

      -or-
   
      The server application, source file, or item can't be found. You
      should reinstall the server application.
You may also receive an error message similar to the following: 

   An application error has occurred and an application error log is
   being generated.

   wangimg.exe

   Exception: access violation (0xc0000005), Address: 0x760436dc
Note that the address may also be 0x7608ce06.

You may not be able to switch to or quit the program that contains the Image Document.

CAUSE

The problems described in this article may occur when the following conditions are true:

  • You insert or double-click an Image Document object in any of the programs listed at the beginning of this article.

    -and-

  • You are running Microsoft Windows NT version 4.0.

The problems occur even if Service Pack 2 for Windows NT 4.0 is installed on the computer.

These problems do not occur when you use any of the Office programs under Microsoft Windows 95.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

you should be aware of the following:

  • When you use a computer running Windows NT 4.0, you can edit documents that contain Image Document objects. However, if you activate any of the objects or insert a new Image Document object, you may encounter the problems described in this article.
  • If you cannot quit a program, you can use the Windows NT Task Manager to quit the program.

    To do this, press CTRL+ALT+DEL in the Windows NT Security dialog box, click Task Manager, click the name of the program you want to quit, and then click End Task.

In the programs listed at the beginning of this article, you can insert an Image Document object by doing the following:

  1. On the Insert menu, click Object. Click the Create New tab.

  2. In the list of Object Types, click Image Document. Then, click OK.
