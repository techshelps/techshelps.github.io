Office Setup Doesn't Remove All Files Using Remove All

 Last reviewed: October 21, 1997
Article ID: Q129285
4.20 4.20-CD 4.20c 4.30 4.30-CD 4.30c-CD WINDOWS kbsetup

The information in this article applies to:

  • Microsoft Office for Windows, versions 4.2, 4.2-CD, 4.2c, 4.3, 4.3-CD, 4.3c, 4.3c-CD

SYMPTOMS

When you run the Microsoft Office setup program in maintenance mode, and you choose the Remove All button to remove all of the Microsoft Office files, some of the files are not removed.

In general, when you run an application's Setup program in maintenance mode, and you choose the Remove All button, a file is not removed if either of the following is true:

  • The file was not installed by the setup program 

        -or-
  • The file was installed by the setup program, but the file is used by another application.

WORKAROUND

To remove these files from your hard disk drive, you must delete the files manually from File Manager or from the MS-DOS prompt. The following table contains a list of the individual files that you will need to delete manually:

Files Not Removed By Using Remove All

NOTES: 

   The following table includes only files that Microsoft Office Setup
   installs. Files that are created when you run an application, such as
   the Microsoft Word NORMAL.DOT (located in the MSOFFICE\WINWORD\TEMPLATES
   directory) are also not removed; however, these files are not included
   in the following list.

   In the following table, WINDOWS is your Microsoft Windows directory.

   The "X" indicates that the file is installed by Microsoft Office Setup,
   but it is not removed when you run Setup in maintenance mode and choose
   the Remove All button. The "*" indicates that due to inconsistent
   results in testing, the file may or may not be removed.

   If you remove the file DVA.386, you must also remove the corresponding
   entry in the SYSTEM.INI file or you will not be able to start Windows
   again. To remove the entry, do the following:

   1. Open the SYSTEM.INI file in Notepad or another text editor.

   2. Locate the following line in the [386Enh] section:

         device=dva.386

   3. Remove the line, and save the changes to your SYSTEM.INI file.
