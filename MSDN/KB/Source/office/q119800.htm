Setup Could Not Write to File While MOM is Running

 Last reviewed: September 16, 1996
Article ID: Q119800
The information in this article applies to:
  • Microsoft Office for Windows, version 4.2

SUMMARY

When you run the Microsoft Office Setup program, you receive the following error message if the Microsoft Office Manager (MOM) is running: 

   Could not write to the file named:

   C:\MSOFFICE\MSOFFICE.EXE'.

   This file is currently being used by another
   application. Please shut down your other
   applications.

CAUSE

This behavior is by design of Microsoft Office. This error message appears when you run the Microsoft Office version 4.2 Setup program while the Microsoft Office Manager version 4.0 is running. The Microsoft Office file, MSOFFICE.EXE, cannot be updated while MOM is running.

Note that if you run the Microsoft Office Setup program while one of the Microsoft Office applications is running, such as Microsoft Excel, you receive a warning that the Setup program detected the application, and that you should close the application before running the Setup program. However, you do not receive this warning when MOM is running.

WORKAROUND

To continue with the Setup program when you receive one of the above error messages, do the following: 

   1. Activate the Microsoft Office Manager.

   2. Close the Microsoft Office Manager by choosing Exit from the
      Microsoft Office menu.

   3. In the Setup Error dialog box, choose Retry.
