OFF97: "Doing Business on the Internet" Is Not Completely Loaded

 Last reviewed: October 14, 1997
Article ID: Q170806
The information in this article applies to:
  • Microsoft Office 97 Small Business Edition for Windows
  • Microsoft Internet Explorer version 3.02 for Windows 95
  • Microsoft Internet Explorer version 3.02 for Windows NT 4.0

SUMMARY

When you view the "Doing Business on the Internet" Web page located on the Microsoft Office 97 compact disc, the Web page may not be completely loaded or may appear to stop responding (hang). The following error message is displayed in the status bar of Microsoft Internet Explorer: 

   exception: com.ms.applet.AppletSecurityExtension: security.Couldn't
   connect to  with origin from file

CAUSE

This problem occurs when both of the following conditions are true:

  • You are running Microsoft Internet Explorer 3.02.

    -and-

  • You are viewing the "Doing Business on the Internet" Web page in the Microsoft Office 97 ValuPack.

RESOLUTION

To resolve this problem, install the latest update to Microsoft Java Virtual Machine (VM) for Internet Explorer (version 1517 or later). To obtain this update, follow the steps below:

  1. Visit the following Microsoft Web site: 

          http://www.microsoft.com/java/download.htm

  2. Click on the "Microsoft virtual machine for Java" hyperlink.

  3. Read the End-User License agreement and then click the "I Accept: Download the VM for Java" hyperlink.

  4. Click the hyperlink that applies to your particular Operating System. The Java Virtual Machine for Internet Explorer 3.02 components will automatically begin to install.

NOTE: Because the Microsoft Web site is constantly updated, the site address may change without notice. If this occurs, link to the Microsoft home page at the following address: 

   http://www.microsoft.com/
For information about downloading the current version of the Java Virtual Machine, see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q163637
   TITLE     : INFO: Availability of Current Build of Microsoft VM for
               Java

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Microsoft Win32 Virtual Machine for Java version 1517 or later.

MORE INFORMATION

This behavior occurs because of a security fix that restricts local file Uniform Resource Locator addresses in version 1516 of the Microsoft Virtual Machine for Java. This restriction prevents an untrusted Java program from being able to open a file on the local computer.

For more information about Java program conflicts with Internet Explorer 3.02, see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q170469
   TITLE     : Java Program Does Not Work with Internet Explorer 3.02
