OFF95: Dial-Up Network Connection Button on Shortcut Bar Fails

 Last reviewed: March 27, 1997
Article ID: Q148915
7.00 WINDOWS kbnetwork kbenv

The information in this article applies to:

  • Microsoft Office for Windows 95, version 7.0
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft PowerPoint for Windows 95, version 7.0
  • Microsoft Word for Windows 95, version 7.0

SYMPTOMS

When you add a Dial-Up Networking button to the Office Shortcut Bar, clicking the button has no effect.

CAUSE

This problem occurs when you do the following:

  • You install the Dial-Up Networking Server service that is included with Microsoft Plus! for Windows 95.

    -and-

  • You create a new dial-up connection.

    -and-

  • You drag a shortcut to the connection to the desktop.

    -and-

  • You drag the shortcut to the Office Shortcut Bar.

For additional information, please see the following article(s) in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q143271
   TITLE     : OFF95: Shortcuts That You Drag and Drop to OSB Do Not Start

WORKAROUND

To work around this problem, use the right mouse button to click the button and click Connect.

This problem occurs because the Office Shortcut Bar does not start a service when you click the button on the bar. The Office Shortcut Bar does correctly open files and folders when you click their corresponding buttons.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Microsoft Office for Windows 95, version 7.0a.

MORE INFORMATION

The Dial-Up Networking Server component in Microsoft Plus! lets you share network resources with others who connect to your computer from remote locations.
