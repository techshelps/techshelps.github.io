OFF97: Buttons on OSB Launch Quick View

 Last reviewed: November 24, 1997
Article ID: Q163647
97 WINDOWS kbenv kbprb

The information in this article applies to:

  • Microsoft Office 97 for Windows, all editions

IMPORTANT: This article contains information about editing the registry. Before you edit the registry, you should first make a backup copy of the registry files (System.dat and User.dat). Both are hidden files in the Windows folder.

SYMPTOMS

When you click a button on the Microsoft Office 97 Shortcut Bar (OSB), the program may not start. Quick View may start instead.

CAUSE

This problem will occur if the computer's registry contains the following key: 

   HKEY_CLASSES_ROOT\*\QuickView

RESOLUTION

WARNING: Using Registry Editor incorrectly can cause serious problems that may require you to reinstall Windows 95. Microsoft cannot guarantee that problems resulting from the incorrect use of Registry Editor can be solved. Use Registry Editor at your own risk.

For information about how to edit the registry, view the Changing Keys And Values online Help topic in Registry Editor (Regedit.exe). Note that you should make a backup copy of the registry files (System.dat and User.dat) before you edit the registry.

To resolve this problem, you must remove the HKEY_CLASSES_ROOT\*\QuickView registry key. To do this, use the following steps:

  1. On the Start menu, click Run.

  2. Type "regedit" (without the quotation marks), and click OK.

  3. Select the following registry key: 

          HKEY_CLASSES_ROOT\*\QuickView

  4. On the Registry menu, click Export Registry File.

  5. Type a new file name and click Save.

  6. On the Edit menu, click Delete.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
