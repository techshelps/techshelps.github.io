Empty Desktop Folder Left After Exiting Admin Setup

 Last reviewed: November 17, 1997
Article ID: Q125786
The information in this article applies to:
  • Microsoft Office for the Macintosh, version 4.2

SYMPTOMS

When you run an administrative installation of Microsoft Office, by holding down the COMMAND key while double-clicking the Setup file on the Install Disk 1 floppy disk, an empty Desktop Folder is left in the Install Disk 1 directory on the network server if you exit the Setup program after Setup prompts you to insert Install Disk 2.

CAUSE

This behavior occurs when you choose to allow users to install Microsoft Office from the network server (Mode 1). A Desktop Folder is used temporarily during Setup. When you complete Setup normally, this folder is removed from the network drive. However, if you exit Setup before it is completed, this folder may be left in the Install Disk 1 folder in the directory to which you are installing Microsoft Office.

WORKAROUNDS

To work around this behavior when you exit Setup, drag the temporary Desktop Folder to the trash, or drag the entire folder to which you were installing Microsoft Office on the network server to the trash.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
