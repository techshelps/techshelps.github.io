OFF95: Workstation Installation Switches Don't Work

 Last reviewed: March 27, 1997
Article ID: Q140243
7.00 WINDOWS kbsetup kbnetwork

The information in this article applies to:

  • Microsoft Office for Windows 95, version 7.0

SYMPTOMS

When you use the /n and /o switches during a quiet mode workstation installation of Microsoft Office Standard or Professional, the switches do not work as expected.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem has been corrected in Microsoft Office 7.0b for Windows 95.

MORE INFORMATION

The following command line shows how these switches might be used: 

   d:\setup.exe /q /t "<location of SETUP.STF file" /n "<username>" /o
   "<organization name>"
The /n switch specifies the username and the /o switch specifies the organization name for Microsoft Office. These values are stored in the System registry when you install Microsoft Office.

/n Switch

The /n switch does not work properly in a quiet mode workstation installation. Depending on the program you are using, you will get different results for the information in the About dialog box (on the Help menu, click About). The /n switch only works when there are no values in the registry for Username and Organization.

/o Switch

The /o switch is not supported in a quiet mode workstation installation and only works when the following conditions are true:

  • There are no values in the registry for Username and Organization

    -and-

  • You are running a local installation of Office.
