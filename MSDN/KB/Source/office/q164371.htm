OFF97: Shared Components Deleted with Local Installation

 Last reviewed: March 13, 1998
Article ID: Q164371
The information in this article applies to:
  • Microsoft Office 97 for Windows

SYMPTOMS

When you install Microsoft Office 97 on a computer running an earlier version of Microsoft Office that is shared over a network, the Setup program deletes the shared components on the server. This behavior affects other users who attempt to use these shared files.

CAUSE

This problem occurs when the following conditions are all true:

  • You perform a workstation installation of Microsoft Office that is earlier than Microsoft Office 97 for Windows on the computer. 

        -and-
  • The network software you are using is Novell Netware version 4.10 

        -and-
  • You have Delete rights on the server that contains the shared components of the earlier version of Microsoft Office. 

        -and-
  • You run the Microsoft Office 97 Setup program and click Yes when you are prompted to remove files for the earlier version.

WORKAROUND

To prevent this behavior from occurring, use any of the following methods.

Method 1

When you are prompted to remove earlier versions of Microsoft Office, click No.

Method 2

Assign the read-only attribute to all the Microsoft Office files on the server.

Method 3

Log on to the network with user rights that do not allow you to delete files on the network server, and then run the Microsoft Office Setup program. See your network administrator to change your logon rights.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Office 97 for Windows. This problem was corrected in Microsoft Office 97 SR-1 Enterprise Update.

For additional information about SR-1, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q172475
   TITLE : OFF97: How to Obtain and Install MS Office 97 SR-1

MORE INFORMATION

When you install Microsoft Office locally on your computer, the Office 97 Setup program detects earlier versions of Office files and deletes these files by design. However, when you install Office in a shared network environment, all users who rely on the shared files are no longer able to use these components.
