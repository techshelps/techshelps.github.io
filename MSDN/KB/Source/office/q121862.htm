MOM Serial Number Isn't Changed After Installing Upgrade

 Last reviewed: September 16, 1996
Article ID: Q121862
The information in this article applies to:
  • Microsoft Office for Windows, versions 4.3c, 4.3c-CD

SYMPTOMS

When you run the Microsoft Office Setup program, either from floppy disks or from CD, and you install the Microsoft Office files to a directory that contains Microsoft Office version 4.3, the About Microsoft Office dialog box displays the serial number for your previous version of Microsoft Office.

CAUSE

This behavior occurs because the Microsoft Office Manager (MOM) file (MSOFFICE.EXE) is not updated when you install Microsoft Office version 4.3c to a directory that contains Microsoft Office version 4.3. Because the version of MSOFFICE.EXE (version 4.3) that ships with Microsoft Office version 4.3c is the same as the version that ships with Microsoft Office version 4.3, the file is not replaced. No changes have been made to this file between versions 4.3, and 4.3c; the only way to distinguish this file is by the file date.

WORKAROUND

Although there is no functional reason that you should replace the Microsoft Office Manager file that ships with Microsoft Office version 4.3 with the version that ships with version 4.3c, you can use the following steps to do this:

  1. Run the Microsoft Office Setup in maintenance mode and choose the Add/Remove button.

  2. Clear the Microsoft Office Manager check box in the Options list.

  3. Complete the Setup.

  4. Run the Microsoft Office Setup in maintenance mode again. Choose the Add/Remove button, and this time, select the Microsoft Office Manager check box in the Options list.

These steps remove the MSOFFICE.EXE file, and then add it again, this time with the serial number for Microsoft Office version 4.3c.

To view the About Microsoft Office dialog box, choose About Microsoft Office from the Microsoft Office menu on the MOM toolbar.
