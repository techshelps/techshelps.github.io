Updating Individual Office 4.0 Applications on a Network

 Last reviewed: September 10, 1996
Article ID: Q113623
The information in this article applies to:
  • Microsoft Office for Windows, version 4.0
  • Microsoft Word for Windows, version 6.0a
  • Microsoft Excel for Windows, version 5.0
  • Microsoft PowerPoint for Windows, version 4.0

SUMMARY

There is no way to keep the integrated setup of Office 4.0 when you update individual Office applications in a network installation. This process requires that the updated application be installed in a separate subdirectory on the server, rather than in the MSOFFICE directory.

If you install the updated application in the network directory of Office, the Office 4.0 SETUP.STF file is overwritten, preventing you from future workstation installation/removal of the Office package.

MORE INFORMATION

To update your server installations of the Office 4.0 applications:

  1. Run setup /a to install the updated application either in a subdirectory of the Office directory (for example as MSOFFICE\EXCEL) or in a completely separate directory on the server.

  2. From the individual workstation, run the workstation installation from the directory of the updated application. (This will update the workstation to use the new version of the Office application.)

The old version of the updated application will still be on the MSOFFICE server subdirectory. There is no way to remove these files.

It is a good idea to run the Setup/Install program from the workstation to remove the earlier version of the application before you install the updated version. If you do not remove the earlier version of the application, old object linking and embedding (OLE) references and icons for shared applications (such as Microsoft Graph) will remain on each workstation. These old references and icons may confuse users.

If you need to install a new workstation both with Office 4.0 components and the updated application, do the following:

  1. Run the Office 4.0 workstation installation and install all components EXCEPT the updated application.

  2. Run the updated application's workstation installation.
