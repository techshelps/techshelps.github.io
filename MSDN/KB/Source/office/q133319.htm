Office 4.2.1: File Location on Setup Disks and CD

 Last reviewed: February 2, 1998
Article ID: Q133319
The information in this article applies to:
  • Microsoft Office for the Macintosh, versions 4.2.1, 4.2.1a

SUMMARY

The information in this article provides the location of each file on the Microsoft Office for the Macintosh and Power Macintosh, versions 4.2.1 and 4.2.1a, Setup disks and CD. For the versions that are on CD, the files are located in folders on the CD, where there is a folder for each of the install disks. (Note that most of these files cannot be decompressed or extracted individually from the disks or CD, they can only be installed using the Setup program.)

Because Microsoft Office for the Macintosh and Power Macintosh, versions 4.2.1 or 4.2.1a CD, does not ship with User's Guides, all of the necessary documentation is in the online documentation. You can access this documentation by choosing Online Documentation from the Microsoft Office Manager (MOM) menu. The Microsoft Office Online Documentation is actually a collection of .BMP files in a folder titled Microsoft Office Manuals:Images which is in the Install Disk 1 folder on the Microsoft Office CD. There are also subfolders titled Excel, Office, PowerPt, and Word, in this Microsoft Office Manuals:Images folder. The total number of .BMP files is approximately 2300 in all of the folders (these files will not be listed in this article).

Also in the Install Disk 1 folder of the Microsoft Office CD, there is a Tips & Tricks folder, that contains five Microsoft Word documents with information about setting up and running the Microsoft Office applications. These files are not installed by Setup, but can be opened once you have Microsoft Word installed on your machine.

NOTE: The following information may be useful when you are troubleshooting a Setup issue. For example, if Setup cannot read a particular disk or file, or the disk appears to be damaged, you can use the following information to determine which item in Setup to clear, so that Setup does not request, or use, the disk in question.
