Screen Fails to Redraw with Office 7.0 Program in Windows NT

 Last reviewed: March 11, 1997
Article ID: Q149578

The information in this article applies to:

  • Microsoft Office for Windows 95, version 7.0
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft Word for Windows 95, version 7.0
  • Microsoft Access versions 7.0
  • Microsoft PowerPoint for Windows 95, version 7.0
  • Microsoft Windows NT Workstation version 3.51
  • Microsoft Windows NT Server version 3.51

SYMPTOM

When you use Microsoft Excel, Microsoft Access, Microsoft Word, or Microsoft PowerPoint under Microsoft Windows NT, the screen may fail to redraw correctly.

CAUSE

This behavior may occur when you switch between Microsoft Office 7.0 programs, such as Word and Excel, or between a Microsoft Office application and any other 16-bit program multiple times.

This problem does not occur if you use Windows 95.

WORKAROUND

To work around the problem, resize or minimize the 16-bit program window and restore it.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 3.51. This problem was corrected in the latest Windows NT 3.51 U.S. Service Pack. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
