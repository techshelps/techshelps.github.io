OFF95: Install to NT Fails from Novell Netware Server

 Last reviewed: March 12, 1997
Article ID: Q138428
The information in this article applies to:
  • Microsoft Office for Windows 95, version 7.0
  • Microsoft Excel for Windows 95, version 7.0
  • Microsoft Project for Windows 95, version 4.1
  • Microsoft Access, version 7.0

SYMPTOMS

When you perform a workstation installation of the programs listed at the beginning of this article to a computer running Windows NT version 3.51 from a Novell Netware version 3.x or version 4.x server, you may receive any of the following error messages 

   Setup could not read from the file:
   \\servername\sharename\msoffice\msvcrt20.dll

   -or-

   Setup could not read from the file:
   \\servername\sharename\msoffice\odbccp32.dll

   -or-

   Setup could not read from the file:
   \\servername\sharename\msoffice\odbcint.dll

   -or-

   DAO3032.DLL was unable to register itself in the system registry.
followed by: 

   Microsoft <product name> setup was not completed successfully.

CAUSE

This behavior occurs when you use the Microsoft Client for Netware, which is a networking component of Windows NT version 3.51.

WORKAROUND

To avoid receiving these error messages, obtain Service Pack 5 for Windows NT version 3.51 and update your system.

For information on how to download Service Pack 5, refer to the Network.txt file on Disk 1 of the Microsoft Office for Windows 95, version 7.0, disks or on the Microsoft Office for Windows 95 compact disc.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem has been corrected in the U.S. Service Pack 5 for Windows NT version 3.51. For information about obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
