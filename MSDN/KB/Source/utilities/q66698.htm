FIX: L1089 Error, Response Filename Length > 32 Characters

 Last reviewed: September 11, 1997
Article ID: Q66698
5.01.21 5.03 5.05 | 5.01.21 5.03 5.05 
MS-DOS            | OS/2
kbtool kberrmsg kbbuglist kbfixlist

The information in this article applies to:

  • Microsoft LINK for MS-DOS and OS/2, versions 5.01.21, 5.03, and 5.05

SYMPTOMS

An attempt to link an application fails and Microsoft LINK displays the following message: 

   LINK : Fatal error L1089 : D:\C600\FILES\PROJECT\TEST\MYFIL :
      cannot open response file

CAUSE

The length of the path and filename for the response file exceeds 32 characters.

RESOLUTION

Move the file to a different directory or specify a relative file name to make the path and filename length less than or equal to 32 characters.

STATUS

Microsoft has confirmed this to be a limitation in LINK versions 5.01.21, 5.03, and 5.05 for MS-DOS and OS/2. This limit was increased to 255 characters in LINK version 5.1 for MS-DOS and OS/2.
