EPS Graphic Prints with Box Around It to HP LaserJet IV

 Last reviewed: February 5, 1998
Article ID: Q110864
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows 95, version 7.0
  • Microsoft PowerPoint for Windows, version 4.0
  • Microsoft Excel for Windows 95, version 7.0

SYMPTOMS

When printing an EPS graphic with a TIFF header to a Hewlett-Packard (HP) LaserJet IV printer in non-PostScript mode, the graphic may print with a box or border around it. The graphic appears correctly (without the box) on the screen and in print preview.

This problem has also been reported when printing to a Tektronics Phaser II printer using the PostScript printer driver version 3.58.

CAUSE

This problem occurs when using the HP LaserJet IV driver version 1.20 set to HPGL graphics mode. When the printer is not in PostScript mode, it is actually printing the TIFF header rather than interpreting the encapsulated PostScript code. This issue was corrected with the HP IV driver version 1.26. This new driver is available from HP’s Fulfillment Center, which you can reach by calling (303) 353-7650.

WORKAROUND

HP LaserJet IV

To work around this problem until you receive the new driver:

Word 7.0

  1. On the File menu, click Print.

  2. Click Properties.

  3. Click the Graphics tab.

  4. At the bottom of the dialog box, select Use Raster Graphics.

  5. Click Apply and click OK.

Word 6.0

  1. On the File Menu, click Print.

  2. Click Printer.

  3. Click Options.

  4. Click Options again.

  5. In the Graphics Mode box, select Raster.

  6. Click OK twice and then click Close.

  7. Click OK to return to the document.

The graphic should now print correctly.

The HP products included here are manufactured by vendors independent of Microsoft; we make no warranty, implied or otherwise, regarding these products' performance or reliability.

Tektronics Phaser II

From the Windows Control Panel, choose Printers and change the printer selection to AppleLaserWrite II NT.
