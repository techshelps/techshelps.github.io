Dr. Watson Causes Errors in Help Examples & Demos Modules

 Last reviewed: February 6, 1998
Article ID: Q110216
The information in this article applies to:
  • Microsoft Word for Windows, version 6.0

SYMPTOMS

When the Dr. Watson program is running in the background on your computer, Dr. Watson Clues error messages or a general protection (GP) fault may occur while you run some of the Word for Windows examples and demonstrations tutorials. (To run the examples and demonstrations, choose Examples and Demos from the Help menu.)

STATUS

Microsoft has confirmed this to be a problem in Word for Windows. This problem was corrected in version 6.0a of Word for Windows.

To obtain pre-sales information about new or updated Microsoft products, call the Microsoft Sales Information Center at (800) 426-9400. If you are outside the United States, contact your local subsidiary. To locate your subsidiary, see the Microsoft World Wide Offices Web site at: 

   http://www.microsoft.com/worldwide/default.htm

WORKAROUND

To work around this problem, use the following steps:

  1. Exit the Dr. Watson utility before you run the Examples And Demos command on the Help menu.

  2. Make sure the SHARE command in your AUTOEXEC.BAT file is using the following parameters: 

          <PATH>\SHARE.EXE /L:500 /F:5100

MORE INFORMATION

Microsoft has received reports of similar error messages ("exception error 13") when other diagnostic programs, such as WinSpectra by Borland, are running in the background.

These instances are usually resolved by disabling the diagnostic program and loading SHARE with the following parameters: 

   <PATH>\SHARE.EXE /L:500 /F:5100
The error message you receive sometimes varies depending on the video driver you are using. For example, if you are using the Super VGA 800 x 600 video driver, you may receive an error in PLAY.EXE.
