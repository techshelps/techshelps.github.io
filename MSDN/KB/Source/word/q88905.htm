Corel Draw! 3.0 TrueType Fonts Display Spaces as Squares

 Last reviewed: February 5, 1998
Article ID: Q88905
The information in this article applies to:
  • Microsoft Word for Windows, versions 1.0, 1.1, 1.1a, 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c, 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows 95, versions 7.0, 7.0a
  • Microsoft Word 97 for Windows

SUMMARY

In Microsoft Word for Windows, if you use a Corel Draw! version 3.0 TrueType font, spaces are displayed as square characters when you choose the Show/Hide Nonprinting Characters button on the ribbon or select the Spaces check box or the All check box in the Options dialog box. When you print your document, these squares do not appear.

MORE INFORMATION

When you select one of the options to display spaces, Word for Windows usually displays a dot character (ANSI character code 0183) to represent each space. When you use a Corel TrueType font, ANSI character code 0183 displays a nonprinting square.

Workaround

Word Versions 2.x - 97

  • Turn off the Show/Hide Nonprinting Characters button on the ribbon.

    -or-

  • From the Tools menu, choose Options. Under the View category, clear the Spaces check box in the Nonprinting Characters area, then choose the OK button.

Word Versions 1.0, 1.1, and 1.1a

  • Turn off the Show All button on the ribbon.

    -or-

  • From the View menu, choose Preferences. Clear the Spaces check box, then choose the OK button.
