Section Margins and Page Size Changes in Embedded Word Object

 Last reviewed: February 5, 1998
Article ID: Q116267
6.00 6.00a WINDOWS kbusage

The information in this article applies to:

  • Microsoft Word for Windows, versions 6.0, 6.0a

SYMPTOMS

If you embed a Word Document Object that contains multiple sections and multiple margin settings in a container (client), Word for Windows changes the margin settings to 0 (zero) inches in all sections.

CAUSE

By design (to avoid unwanted white space), Word changes the page size of an embedded Word Document Object to 6 inches and changes the margins to 0 (zero) inches. When Word does this in a multisection document with different section margin settings, it sacrifices the position of each section in relation to one another.

STATUS

Microsoft has confirmed the sacrifice of relative margin settings between sections in an embedded Word Document Object to be a problem in Word versions 6.0 and 6.0a for Windows. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUND

After you embed the Word Document Object, activate the object and change the margin settings for each section. Word retains these settings when you update the object.

REFERENCES

"Microsoft Word User's Guide," version 6.0, pages 602-606
