Word Setup Corrupts FAT Under WFW or with Adaptec Controller

 Last reviewed: February 6, 1998
Article ID: Q113733
6.00 6.00a WINDOWS 
 kb3rdparty
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a
  • Microsoft Windows for Workgroups versions 3.1, 3.11

SYMPTOMS

After you run the Word for Windows Setup program, the file allocation table (FAT) on your computer may be damaged (corrupt) if your computer uses an Adaptec disk controller card or if you are running under Windows for Workgroups 3.11.

CAUSE

The FAT corruption usually occurs if the computer is running either a pre- release (beta) copy of Windows for Workgroups version 3.11 or the Adaptec device driver ASPI4DOS.SYS version 3.0 or earlier, or both.

Adaptec's 1542B SCSI controller card requires ASPI4DOS.SYS driver version 3.0a or later to operate correctly with version 3.1 of Microsoft Windows or Windows for Workgroups.

For more information about this problem when you are running Windows for Workgroups version 3.1 or 3.11, query on the following words in the Microsoft Knowledge Base: 

   ASPI4DOS.SYS and adaptec and workgroups and 3.11

RESOLUTION

If the computer is running a pre-release copy of Windows for Workgroups version 3.11 (determined by choosing About from the Help menu of Program Manager), obtain and install the released version.

If the computer is running a version of ASPI4DOS.SYS prior to version 3.0a, contact Adaptec to obtain an upgrade.

For more information on ASPI4DOS.SYS, contact Adaptec technical support.
