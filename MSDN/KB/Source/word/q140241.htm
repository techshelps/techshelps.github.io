WD:TAB Key Doesn't Move to Next Table Cell in Protected Document

 Last reviewed: February 5, 1998
Article ID: Q140241
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for the Macintosh, versions 6.0, 6.0.1
  • Microsoft Word for Windows NT, version 6.0
  • Microsoft Word for Windows 95, versions 7.0, 7.0a
  • Microsoft Word 97 for Windows
  • Microsoft Word 98 Macintosh Edition

SYMPTOMS

When you press the TAB key in a table, Word inserts a tab character instead of moving the insertion point to the next cell. This behavior occurs in the unprotected section a document that is protected for forms.

WORKAROUND

To work around this problem, use either of the following methods before you use the TAB key to move from cell to cell within your table.

  • Click in another cell of the table, and then click in the first cell of the table. 

        -or-
  • Insert an empty paragraph above the table in the unprotected section of your document. Then, click in the first cell of the table.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
