ViewFootnotes Command in Exit Macro Causes GP Fault

 Last reviewed: February 5, 1998
Article ID: Q108252
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c

SYMPTOMS

If your protected Word for Windows form contains an exit macro that runs the ViewFootnotes WordBasic command, a general protection (GP) fault error may occur when you press TAB to exit that FORM field.

CAUSE

The GP fault occurs if you are in normal view. It does not occur if you are in page layout view.

STATUS

Microsoft has confirmed this to be a problem in the versions of Word listed above. This problem was corrected in Word version 7.0 for Windows 95.

WORKAROUND

Modify your exit macro so it switches to page layout view before running the ViewFootnotes command.

REFERENCES

"Microsoft Word User's Guide," version 6.0, Chapter 14, "Forms," "Running Macros Automatically" section
