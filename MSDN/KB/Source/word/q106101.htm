Word 2.x Object Opens in Word 6.0 with Wrong Size Margins

 Last reviewed: February 6, 1998
Article ID: Q106101
The information in this article applies to:
  • Microsoft Word for Windows, version 6.0

SYMPTOMS

Embedded Word 2.x for Windows objects, when activated from an OLE client application, will open into Word 6.0 for Windows with a page size based on the size of the graphic presentation of the client. In the case of Word 2.x objects, this is based on the size of the Word icon in the container--usually about .5-inch wide.

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUND

To get the Word 2.x for Windows object to open in Word 6.0 at the correct page size, resize the graphic image in the container to match the desired page size.

If you are using Microsoft Mail, the best way to preserve the proper page size is to open the object in Word 6.0, adjust the page size using the Page Setup command on the File menu, save the object as a document, return to Mail, and attach the document.
