Cannot Resize Objects in Converted Word 2.x Documents

 Last reviewed: February 6, 1998
Article ID: Q120281
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c

SYMPTOMS

Embedded objects in documents you convert from Word version 2.x for Windows either do not respond to changes made in their size or they respond incorrectly.

Specifically, any of the following behaviors may occur:

  • When you drag a corner of a Microsoft Graph object or a Microsoft WordArt object with the mouse, the object returns to its original size when you release the mouse button.
  • If you change the scaling by choosing Picture from the Format menu, the dimensions of the object may change drastically. After changing the scaling percentage, a different size is shown in the "Original Size" area of the Format Picture dialog box. The actual original size is not shown.

CAUSE

This problem happens only to those objects that contain the \S switch in their field codes. In Word 2.x, when you insert a Graph, Microsoft Excel, or WordArt object, Word inserts "\s" in the field. This switch preserves the original size of an object; if an object contains the switch in its field code, you can resize the object using the mouse, but the object will return to its original size when it is updated.

STATUS

We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUND

  1. Select the object.

  2. Press SHIFT+F9 to display the field code. For example, a Graph object would reveal the following field: 

          {EMBED MSGraph \s \* mergeformat}

  3. Remove the "\s" from the field.

  4. Press SHIFT+F9 to return to the result of the field code.

You can now resize the object.
