WD: Word 97 Import Converter: How to Do an Administrative Setup

 Last reviewed: February 5, 1998
Article ID: Q176017
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows 95, versions 7.0, 7.0a

SUMMARY

This article describes how to perform an administrative installation of the Microsoft Word 97 import converter for Word 6.x and 7.x.

What Is the Converter?

In order to open a Word 97 document in an earlier version of Word (6.x or 7.x), you must use the Word 97 import converter for Word 6.x and 7.x. This converter works with the earlier versions of Word, allowing them to read the newer Word 97 file format.

The most recent version of the converter (Wrd97cnv.exe) is available in the ValuPack folder included with Office 97 SR-1 Enterprise Update (the CD version of SR-1). This file is also available for download from the Microsoft Software Library: 

 ~ Wrd97cnv.exe (size: 984958 bytes)
For more information about downloading files from the Microsoft Software Library, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q119591
   TITLE     : How to Obtain Microsoft Support Files from Online Services

MORE INFORMATION

There are two methods you can use to place the converter into a central location so that it can be easily installed on multiple computers.

Method 1: Expanding Files to the Network Server

The following procedure allows you to expand the converter files to a network location. Then, from each workstation, you can run Setup from the server location and choose either to install the converter files locally or to run them from the server location. To do this, follow these steps:

  1. Copy the Wrd97cnv.exe file to the network location on which you want to install the converter files.

  2. On the Start menu, click Run, and type 

          <drive>:<location>\wrd97cnv.exe /A

    where <drive> and <location> points to the directory containing Wrd97cnv.exe.

    The following files are extracted into the source directory: 

          admin.flg
      cnvhlp32.exe
      emfimp.flt
      emfimp32.flt
      iffjpeg.dll
      jpegim32.flt
      jpegimp.flt
      license.txt
      msconv97.dll
      msword8.cnv
      mswrd832.cnv
      pictim32.flt
      pictimp.flt
      png32.flt
      pngimp.flt
      setup.exe
      wrd97cnv.doc
      wrd97cnv.inf

  3. Set the share permissions to "Change" (RWXD), and then set the individual file permissions to "Read-Only" (RX) if desired.

    NOTE: Do not set the share permissions to "Read-Only" or you will be unable to run Setup.

  4. From each workstation, run Setup.exe from the network server location.

    The converter files will either be copied to the local machine or will remain on the network server depending on the installation option you choose.

Method 2: Running Wrd97cnv.exe from the Server

NOTE: This installation method does not allow you to run the converters from the network location (if you want to do this, use Method 1).

In the following procedure, you place the self-extracting executable, Wrd97cnv.exe, in a common location. Then, from each workstation, you can run the executable file from the network server. This procedure will download the converter files to the workstation and install them. To do this, follow these steps:

  1. Copy the Wrd97cnv.exe file to the network location.

  2. Set the share permissions to "Change" (RWXD), and then set the individual file permissions to "Read-Only" (RX)if desired.

    NOTE: Do not set the share permissions to "Read-Only" (RX) or you will be unable to run the Setup program.

  3. From each Workstation, run Wrd97cnv.exe from the network server location.

The converter files are copied and installed to the local machine.
