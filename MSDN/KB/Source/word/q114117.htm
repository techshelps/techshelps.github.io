Return to Command Prompt or GP Fault in Module WIN87EM.DLL

 Last reviewed: February 5, 1998
Article ID: Q114117
6.00 6.00a 6.00c WINDOWS kbenv kberrmsg

The information in this article applies to:

  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Windows operating system version 3.1

SYMPTOMS

When you run Word for Windows under Windows version 3.1 on a 386-based computer with an Intel i387 math coprocessor, Windows may unexpectedly exit to the command prompt, or the following error message may occur: 

   Winword Caused a General Protection Fault in module WIN87EM.DLL
   at 0001:0290.

CAUSE

These errors occur because of a timing problem between the Windows WIN87EM.DLL file and some 80387 Intel math coprocessors. This problem does not happen if you are using the Intel 80486DX and 80486DX2 processors.

STATUS

Microsoft has confirmed this to be a problem in the WIN87EM.DLL file supplied with Windows 3.1. This problem was corrected in an updated WIN87EM.DLL file, which you can obtain by mail (with application note WW0548, "WIN87EM.DLL Patch for Intel 80387"), and online services. For more information about obtaining this application note, query on the following words in the Microsoft Knowledge Base: 

   win87em.dll and WW0548 and 80387
