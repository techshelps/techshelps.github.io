Word Hangs When You Select a Table Column

 Last reviewed: February 5, 1998
Article ID: Q111080
6.00 WINDOWS kbusage buglist6.00 fixlist6.00a

The information in this article applies to:

  • Microsoft Word for Windows, version 6.0
  • Microsoft Windows operating system version 3.1

SYMPTOMS

Word for Windows may hang (stop responding) when you select a column in a table that contains a field.

CAUSE

Word hangs when you select a portion of a table that contains a field if you previously deleted cells in the table.

STATUS

Microsoft has confirmed this to be a problem in Word 6.0 for Windows. This problem was corrected in version 6.0a of Word for Windows. To obtain Word 6.0a, call the Microsoft Order Desk at (800) 360-7561 and request Word version 6.0a for Windows. If you are outside the United States, contact your local subsidiary. To locate your subsidiary, see the Microsoft World Wide Offices Web site at: 

   http://www.microsoft.com/worldwide/default.htm

MORE INFORMATION

For example, the following procedure hangs Word:

WARNING! These steps will hang your computer.

Steps to Reproduce Problem

  1. From the Tools menu, choose Options and select the View tab.

  2. In the Field Shading list, select When Selected, and then choose OK.

  3. In a Word document, insert a four-column by four-row table.

  4. In the table, select cells B3 through D4.

  5. From the Table menu, choose Delete Cells.

  6. Select the Shift Cells Left option and then choose OK.

  7. Position the insertion point in cell D2 and press CTRL+F9 to insert a blank field.

  8. Position the insertion point in column C and then choose Select Column from the Table menu.

At this point, Word stops responding. If you press CTRL+ALT+DEL, Windows notifies you that WINWORD.EXE has stopped responding to the system.
