Saving Documents from NT Workstation Causes Errors

 Last reviewed: July 31, 1997
Article ID: Q152768
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Windows NT Workstation, versions 3.5 and 3.51
  • Microsoft Windows NT Server, versions 3.5 and 3.51
  • Microsoft Gateway Service for NetWare

SYMPTOMS

When you save a Word document to a Novell drive through an NT server with Gateway Services for NetWare (GSNW) installed., Word does not save the file or you get one of the following Error messages: 

   There has been a network or file permission error. The network
   connection may be lost. <file name>

   You cannot save while the file is in use by another process. Try
   saving the file with a new name. <file name>
This is evident by no change to the file name in the title bar of the application. This behavior occurs only when you use 16-bit Word for Windows, versions 6.0, 6.0a, and 6.0c, on a Windows NT workstation.

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

WORKAROUND

Use one of the following methods to correct this problem.

Method 1: Obtain an Updated Version of Word

Obtain an updated copy of Word for Windows NT version 6.0 or Word for Windows 95.

To obtain pre-sales information about new or updated Microsoft products, call the Microsoft Sales Information Center at (800) 426-9400. If you are outside the United States, contact the Microsoft subsidiary for your area. To locate your subsidiary, see the Microsoft World Wide Offices Web site at: 

   http://www.microsoft.com/worldwide/default.htm
Method 2: Connect Directly to the Novell Drive

Connect directly to the Novell drive by mapping a drive on the workstation to the document share. This will exclude the use of the Gateway Services for NetWare. To do this, in Control Panel double-click Network. Install Client Services for NetWare and the NW Link protocol to access the Novell network drives.

MORE INFORMATION

Gateway Services for NetWare (GSNW) allows Microsoft networking clients to access files that reside on NetWare servers without having to load NetWare client software.
