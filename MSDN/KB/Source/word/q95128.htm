WD:Second Macro Can't Call Subroutine or Function in First Macro

 Last reviewed: February 2, 1998
Article ID: Q95128
The information in this article applies to:
  • Microsoft Word for Windows, versions 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c, 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows NT, version 6.0
  • Microsoft Word for Windows 95, versions 7.0, 7.0a
  • Word for the Macintosh, versions 6.0, 6.0.1, 6.0.1a

SYMPTOMS

In a WordBasic macro, you can call a subroutine or function from another macro. However, the subroutine or function you call cannot call back to a subroutine or function in the initial running macro.

CAUSE

This occurs by product design in the versions listed above. This problem was corrected in Word 97 for Windows and Word 98 Macintosh Edition Visual Basic for Applications.

REFERENCES

"Word for Windows 2.0 Macros" by Russel Borland, Microsoft Press, 1992, page 223

"Using WordBasic" by WexTech Systems, Inc. and Microsoft Corporation, Microsoft Press, 1992, pages 58-60, 157
