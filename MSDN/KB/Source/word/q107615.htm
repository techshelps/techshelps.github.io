WD: Word Hangs Sending Mail Using Lotus CC:Mail

 Last reviewed: October 20, 1997
Article ID: Q107615
The information in this article applies to:
  • Microsoft Word for Windows, version 6.0
  • Microsoft Windows operating system version 3.1

SYMPTOMS

An hourglass pointer may appear and your computer may hang (stop responding) if you choose the Address button in a New Message window after you start Lotus CC:Mail version 2.0 using the Send command on the File menu in Word for Windows. If you then press CTRL+ALT+DEL, Windows displays the following error message: 

  winword.EXE has stopped responding to the system.

CAUSE

Your computer hangs because the Lotus CC:Mail file VIM.DLL conflicts with the MAPIVIM.DLL file supplied with Word.

Use the following procedure to check which version of MAPIVIM.DLL is installed on your computer:

  1. In Windows File Manager, switch to the \WINDOWS\SYSTEM subdirectory.

  2. Select the MAPIVIM.DLL file.

  3. From the File menu, choose Properties.

  4. In the "Properties for MAPIVIM.DLL" dialog box, you should see a line that reads: 

          Version: 1.0.0.xx

    where xx is a number.

WORKAROUND

To prevent this error from occurring, you must use a MAPIVIM.DLL file whose version is 1.0.0.63 or later, with a file date of 1/14/94 or later. This file is included with the following programs:

  • Microsoft Word for Windows versions 6.0a and later
  • Microsoft PowerPoint version 4.0
  • Microsoft Office version 4.2
  • International versions of Microsoft Excel version 5.0

Or, you can obtain a copy of the .dll from a Microsoft Excel Application Note. For additional information, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q112730
   TITLE     : XL5: AppNote: MAPI-to-VIM Translation .DLL MAPIVIM.DLL
Once you have downloaded the new MAPIVIM.DLL file, copy it to your WINDOWS\SYSTEM folder and then restart Microsoft Windows to start using the new file.

STATUS

Microsoft has confirmed this to be a problem in Word for Windows. This problem was corrected in versions 6.0a and later of Word for Windows.

MORE INFORMATION

If you have the version of MAPIVIM.DLL that shipped with the U.S. domestic version of Microsoft Excel 5.0 or with Word 6.0, you can prevent Word from trying to retrieve the address list by adding the line 

   MAPIVIMDisableABScan=1
to the [Mail] section of your WIN.INI file.
