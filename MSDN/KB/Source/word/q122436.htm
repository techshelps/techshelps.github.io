WD: SetSelRange Causes WordBasic Error 512: "Value Out of Range"

 Last reviewed: February 3, 1998
Article ID: Q122436
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows 95, versions 7.0, 7.0a
  • Microsoft Word for the Macintosh, version 6.0
  • Microsoft Word for Windows NT, version 6.0

SYMPTOMS

When you run a WordBasic macro that contains the SetSelRange command, the following error may occur: 

   WordBasic error 512: Value Out of Range

CAUSE

This error occurs when the second argument (Pos2) of the SetSelRange command selects the last paragraph mark in the document.

For example, the following sample macro selects the last paragraph mark in a document, so WordBasic error 512 occurs: 

   Sub main
      EditSelectAll
      StartOfDocument
      SetSelRange GetSelStartPos(), GetSelEndPos()
   End Sub

To avoid the error, create a subroutine that traps the "Out of Range" error, resets the selection so it does not include the last paragraph mark, and then selects the last paragraph mark using a different method. The following sample WordBasic subroutine demonstrates this workaround: 

   Sub SetSelRangeSafe(Pos1, Pos2)
      On Error Goto OutOfRange
      SetSelRange Pos1, Pos2
   OutOfRange:
      If err = 512 Then
         SetSelRange Pos1, Pos2 - 1
        CharRight 1, 1
      End If
   End Sub

STATUS

Microsoft has confirmed this to be a problem in the versions of Microsoft Word listed at the beginning of this article. This problem was corrected in Word 97.
