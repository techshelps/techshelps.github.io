WD: Callout Text Box Not Affected by Line Style Settings

 Last reviewed: February 2, 1998
Article ID: Q110280
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows NT, version 6.0
  • Microsoft Word for Windows 95, versions 7.0, 7.0a
  • Microsoft Word for the Macintosh, versions 6.0, 6.0.1, 6.0.1a

SYMPTOMS

When you insert a callout and then format the drawing object to select a line style in Word, the line style affects only the polyline portion of the callout. The border portion of the callout's text box remains unaffected.

This behavior occurs with the Style, Color, and Weight attributes specified under the Line tab in the Format Drawing Object dialog box.

WORKAROUND

To have the line attributes affect the text box portion of the callout, turn on the Text Border option for the callout.

To turn on the Text Border option, click Drawing Object on the Format menu, select the Size and Position tab, click Callout and then click to select the Text Border check box.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. This problem was corrected in Microsoft Word 97 for Windows and Microsoft Word 98 Macintosh Edition.

MORE INFORMATION

In Microsoft Word 97 and later, the Text border option is turned on by default. If the border does not appear around the text box when you insert your Callout (AutoShape), follow these steps:

  1. Click to select the Callout.

  2. On the Format menu, click AutoShape.

  3. Click to select the Text Box tab, and then click Format Callout.

  4. In the Format Callout dialog box, click to select Text border, and then click OK.
