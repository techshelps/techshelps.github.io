Word for Windows NT: Supported Characters in Filenames

 Last reviewed: February 5, 1998
Article ID: Q118583
6.00 WINDOWS kbusage kbenv

The information in this article applies to:

  • Microsoft Word for Windows NT, version 6.0

SUMMARY

Characters for filenames of documents saved in or opened from Word 6.0 for Windows NT must exist in both the Windows ANSI character set and the OEM character set to be valid. The U.S. version of Windows NT uses the IBM Extended character set (code page 437) as the OEM character set.

MORE INFORMATION

Characters that are supported:

  • All ASCII alphabetic characters (both uppercase and lowercase)
  • All ASCII numeric characters
  • Spaces
  • All of the following ASCII punctuation characters: 

          ! # $ % & ' ( ) + , - . ; = @ [ ] _ ^ ` { } ~
  • The following characters with grave accent: a e i o u
  • The following characters with acute accent: E a e i o u
  • The following characters with circumflex: a e i o u
  • The following characters with tilde: N n
  • The following characters with diaeresis: A O U a e i o u y
  • All of the following extended characters:
