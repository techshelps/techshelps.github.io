WD98: Save As RTF Adds Extra Language Property to Normal Style

 Last reviewed: February 3, 1998
Article ID: Q179019
The information in this article applies to:
  • Microsoft Word 98 Macintosh Edition

SYMPTOMS

When you create a document that uses the Normal style with the default language property of English(US) and you,

  • Save the document as Rich Text Format,

    -and-

  • Close and re-open the document,

    -and-

  • View the properties of the Normal style,

the Language property for the Normal style has changed from English(US) to this: 

   (FE) English (United States), (Other) English (US)

WORKAROUND

This problem does not occur using custom styles.

Although this problem has no adverse effect on your document, to work around this problem, use a custom style in place of the default Normal style.

Creating a Custom Style

The quickest way to create a new paragraph style is to format a paragraph, select it, and then base the new style on the formatting and other properties applied to the selected text. To do this, follow these steps:

  1. Select the text that contains the formatting you want to include in your style.

  2. On the Formatting toolbar, click in the Style box.

  3. Type over the existing style name to create the name for the new style.

  4. Press ENTER.

For more information about creating custom styles, click the Office Assistant, type "creating styles," click Search, and then click to view "Create new styles."

NOTE: If the Assistant is hidden, click the Office Assistant button on the Standard toolbar. If Microsoft Help is not installed on your computer, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q120802
   TITLE     : Office: How to Add/Remove a Single Office
               Program or Component

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
