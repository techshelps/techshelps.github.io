Limitations of the Cardtext Number Format

 Last reviewed: July 30, 1997
Article ID: Q35596
The information in this article applies to:
  • Microsoft Word for Windows versions 1.0, 1.1, 1.1a, 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c, 6.0, 6.0a, 6.0c
  • Microsoft Word for Windows 95, versions 7.0, 7.0a
  • Microsoft Word 97 for Windows

The Cardtext field number format has the following limitations: 

Maximum number:    999,999
Minimum number:    0 (no negative numbers)
Decimals:          Rounded to the nearest number. Cardtext cannot
                   express decimals.
If these limitations are exceeded, Word will display the message: 

   Error! Number cannot be represented in specified format.
Some examples of the Cardtext number format are as follows: 

{=\* Cardtext 15}      Fifteen
{=\* Cardtext 20+15}   Thirty-Five
{=\* Cardtext 12.75}   Thirteen
