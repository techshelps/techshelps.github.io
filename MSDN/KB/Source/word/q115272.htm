General Protection Fault at Module EHNWNNET.DRV

 Last reviewed: August 4, 1997
Article ID: Q115272
6.00 6.00a 6.00c WINDOWS kbnetwork

The information in this article applies to:

  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Windows operating system version 3.1

SYMPTOMS

The following error message can be generated on a PC that contains a 5250 emulation board for communication to an IBM AS/400: 

   MSWord caused a General Protection Fault in module EHNWNNET.DRV.

STATUS

IBM is aware of this error and has available a Program Temporary Fix (PTF SF14263) for the AS/400. IBM also offers two documents, II06590 and II05474, which can be downloaded through the IBM ECS network.

NOTE: PTFs are dated, and therefore the PTF number may change. If this is the case, you should get the updated PTF from IBM.

The IBM Technical Support number is (800) 772-2227. The IBM Customer Service number is (800) 426-2468.

WORKAROUND

Be sure that any memory being used by the 5250 is excluded both in the memory manager loaded in your CONFIG.SYS file (EMM386.EXE, 386MAX, and so forth) and with an EMMExclude statement in the [386ENH] section of your SYSTEM.INI file.

MORE INFORMATION

If DRWATSON.EXE was running at the time of the general protection (GP) fault, the entry in the DRWATSON.LOG will read: 

   WINWORD had an 'Exceed Segment Bounds (Read)' fault at EHNWNNET 3:02d6.
