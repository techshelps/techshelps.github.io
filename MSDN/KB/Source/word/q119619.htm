File I/O Problems After Installing Video 1.1 for Windows

 Last reviewed: February 6, 1998
Article ID: Q119619
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c

SYMPTOMS

If you run Word for Windows after you install Microsoft Video 1.1 for Windows on your computer, various file I/O (input/output) problems may occur in Word, including the following error messages:

  • The disk is full or too many files are open.
  • Cannot Read from Drive C
  • Error: Cannot Save or Create File. Make sure the disk is not full or write protected.

CAUSE

Video 1.1 installs some shared OLE .DLL files, especially STORAGE.DLL, that conflict with file I/O in Word.

IMPORTANT: Before you conclude that Video 1.1 is causing your file I/O problems, be sure you first use standard troubleshooting methods to rule out other causes (for example, make sure your SHARE.EXE, VSHARE.386, and IFSHLP.SYS files are correctly installed, configured, and running).

STATUS

Microsoft has confirmed this to be a problem in Video version 1.1 for Windows. This problem was corrected in version 1.1a of Video for Windows. For information on obtaining Video 1.1a, see the the following article(s) in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q128227
   TITLE     : WV1160: Video 1.1e for Windows Run-Time Edition
For more information about troubleshooting Setup problems in Word for Windows, please see the following article(s) in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q110420
   TITLE     : Troubleshooting Word 6.0 for Windows Setup Problems

WORKAROUNDS

Method 1: Delete the following .DLL files, installed by Video 1.1, and then 

          reinstall Word:

             COMPOBJ.DLL          OLE2NLS.DLL
             OLE2.DLL             OLE2PROX.DLL
             OLE2CONV.DLL         STORAGE.DLL
             OLE2DISP.DLL
