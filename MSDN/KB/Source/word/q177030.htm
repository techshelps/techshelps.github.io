WD98: Space Added Before Number After Pasting to Text

 Last reviewed: February 17, 1998
Article ID: Q177030
The information in this article applies to:
  • Microsoft Word 98 Macintosh Edition

SYMPTOMS

When you paste a numbered list as unformatted text (using the Paste Special command on the Edit menu), a space appears before numbers greater than 1, but the space does not appear before the first item, as in the following example: 

   1. Apples
    2. Oranges
    3. Bananas
    4. Grapes

CAUSE

This behavior occurs when the following conditions are true:

  • The "Use Smart Cut and Paste" option is selected.

    To locate the "Use Smart Cut And Paste" option, click Preferences on the Tools menu, and click the Edit tab.

    -and-

  • The Number Position is set to Right.

    To set the number position, click Bullets And Numbering on the Format menu, click a number style, and click Customize.

    -and-

  • You apply numbering to the text using the Bullets And Numbering feature.

WORKAROUND

Method 1

Turn off the "Use Smart Cut and Paste" option before pasting the numbered list. To do this, follow these steps:

  1. On the Tools menu, click Preferences, and click the Edit tab.

  2. Click to clear the "Use Smart Cut and Paste" check box.

Method 2

If the "Use Smart Cut and Paste" option is turned on, paste the numbered list as Formatted text. To do this, follow these steps:

  1. Select the list.

  2. On the Edit menu, click Copy.

  3. Position the insertion point where you want to paste the list.

  4. On the Edit menu, click Paste Special.

  5. Select Formatted Text (RTF) and click OK.

This method may not start your new numbering scheme with the number one. To start the new list with the number one, follow these steps:

  1. Select the new list of items.

  2. On the Format menu, click Bullets And Numbering.

  3. Under List Numbers, click Restart Numbering, and then click OK.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
