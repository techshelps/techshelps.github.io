WinWord 2.0: How Word Uses SHARE.EXE

 Last reviewed: July 30, 1997
Article ID: Q91218
The information in this article applies to:
  • Microsoft Word for Windows, versions 2.0, 2.0a, 2.0a-CD, 2.0b, 2.0c
  • Microsoft Windows operating system versions 3.0 and 3.1

SUMMARY

Microsoft Word for Windows uses the MS-DOS file sharing utility (SHARE.EXE) to allow more than one instance of itself to run simultaneously. SHARE accomplishes this by enabling Word for Windows to access the same file more than once. When two Word for Windows sessions are running on a system, both can simultaneously access the NORMAL.DOT and WINWORD.INI files if SHARE is installed.

MORE INFORMATION

SHARE allows two different Word users on a network to open the same file. Without SHARE loaded, the system returns an error message if two users try to open the same file at the same time. With SHARE loaded, Word informs the second user that someone else has the file open and asks whether or not the second user wants to open a copy of that file.

Many networks, such as Novell, include a utility that handles file sharing; therefore, SHARE is not usually necessary for sharing Word for Windows files over a network.

Word for Windows operates correctly without SHARE loaded; however, without SHARE, Word loses the ability to run two sessions simultaneously and the ability to allow two people to access the same file at once.

For more information on SHARE, refer to pages 565-566 of the "Microsoft MS-DOS User's Guide and Reference," version 5.0.

Reference(s):

"Microsoft MS-DOS User's Guide and Reference," version 5.0, pages 565-566
