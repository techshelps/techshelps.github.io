Mail Merge with Access 1.0 Database Starts at Record 0 (Zero)

 Last reviewed: February 5, 1998
Article ID: Q106475
The information in this article applies to:
  • Microsoft Word for Windows, versions 6.0, 6.0a, 6.0c
  • Microsoft Access for Windows, version 1.0

SYMPTOMS

When you use a Microsoft Access 1.0 database through DDE as the data source in a Word 6.0 for Windows mail merge, the record selector will initially show record 0 (zero), even though the data merged is from record 1.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Access version 1.0. This problem was corrected in Microsoft Access version 1.1.

RESOLUTION

As soon as you choose another record, Word will display the correct record number.
