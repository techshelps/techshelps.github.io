Server Installation Does Not Install RISC Versions

 Last reviewed: February 6, 1998
Article ID: Q125030
The information in this article applies to:
  • Microsoft Word for Windows NT, version 6.0

SYMPTOMS

After you perform an Admin (server) installation of Word for Windows NT from an Intel-based Windows NT system, MIPS and Alpha AXP versions of Word do not exist in the Word server directory.

CAUSE

The Setup program included with Word for Windows NT installs only the necessary program files for the platform that the Setup program is running from, including Admin installations. Thus, when running the Setup program on an Intel-based Windows NT workstation, only Intel-based Word Admin installations can be performed.

RESOLUTION

To create Admin installations for MIPS and Alpha AXP platforms, repeat the Admin setup process from MIPS and Alpha AXP Windows NT workstations, respectively. When you perform the Admin install for the other platforms, select a separate directory for each platform. This will result in three directories, one for each platform.
