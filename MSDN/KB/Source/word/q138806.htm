Some Files Remain After Choosing "Remove All" Option in Setup

 Last reviewed: August 5, 1997
Article ID: Q138806
The information in this article applies to:
  • Microsoft Word for Windows 95, versions 7.0, 7.0a

SYMPTOMS

When you choose the Remove All option in Word for Windows 95 Setup, not all files are removed.

CAUSE

Certain files are left in place because they are shared by other programs. This article lists those files that remain.

MORE INFORMATION

The following folders are not removed: 

   C:\MSOffice
   C:\MSOffice\Winword
   C:\MSOffice\Winword\Setup
   C:\Windows\Msapps
   C:\Windows\Msapps\Proof
The files listed in the following table are not removed. 

   File name         Folder
   ----------------------------------------------

   Graph5.ini       C:\Windows
   Ttembed.ini      C:\Windows
   Msfntmap.ini     C:\Windows
   Commtb32.dll     C:\Windows\System
   Custom.dic       C:\Windows\Msapps\Proof
   Dir1tx~1.lnk     C:\Windows\Recent
   Dir2tx~2.lnk     C:\Windows\Recent
   Docobj.dll       C:\Windows\System
   Hlp95en.dll      C:\Windows\System
   Mfcans32.dll     C:\Windows\System
   Mso5enu.dll      C:\Windows\System
   Mso95.dll        C:\Windows\System
   Mso95fx.dll      C:\Windows\System
   Msothunk.dll     C:\Windows\System
   Mstool32.dll     C:\Windows\System
   Openenu.dll      C:\Windows\System
   Pcdlib32.dll     C:\Windows\System
   Pubole32.dll     C:\Windows\System
   Scp.dll          C:\Windows\System
   Sdm.dll          C:\Windows\System
   Ttemb32.dll      C:\Windows\System
   Visx.exe         C:\Windows\System
   Visxutil.dll     C:\Windows\System
