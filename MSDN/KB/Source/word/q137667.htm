IA: Document Views with HTML Tags Displayed

 Last reviewed: February 5, 1998
Article ID: Q137667
The information in this article applies to:
  • Microsoft Internet Assistant for Word, versions 1.0, 1.0z, 2.0
  • Microsoft Word for Windows, versions 6.0a, 6.0c
  • Microsoft Word for Windows 95, version 7.0
  • Microsoft Word 97 for Windows

SYMPTOMS

The HTML source code is displayed even though you open the document as HTML. This behavior occurs when you open the HTML document as Text Only and then save the text-only version as HTML.

CAUSE

Internet Assistant converts documents to HTML based on the styles applied to the text or the special HTML markup that you insert in your document. When you open an HTML document as text only, the Normal style is applied to all of the text. In addition, the HTML source code is converted to ordinary characters that are literal representations of HTML tags. So, when you save the text version of the document as HTML, Internet Assistant does not convert the literal text back to HTML code, and it applies only the Normal style to the document.

RESOLUTION

To correct the document that contains the HTML source, do the following:

  1. Open the document as Text Only.

  2. On the Edit menu, click Select All.

  3. Click the Copy button on the Standard toolbar.

  4. Create a new document based on the Normal template.

  5. In the new document, click the Paste button on the Standard toolbar.

  6. In the Save File As Type list, click Text Only.

  7. Change the file name extension to .htm.

  8. Click OK and return to the document.

  9. Close the document.

  10. Click No when the following message box appears:

    This document contains formatting that cannot be saved in text format. Do you want to save changes to <file name>?

You can now reopen the document as an HTML document. (The Confirm Conversions option can remain selected; make sure that you use the HTML converter.)
