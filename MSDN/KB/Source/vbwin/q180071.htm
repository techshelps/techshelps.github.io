FILE: MSVBVM50.EXE Installs Visual Basic 5.0 Run-time Files

 Last reviewed: January 29, 1998
Article ID: Q180071
The information in this article applies to:
  • Microsoft Visual Basic Control Creation, Learning, Professional, and Enterprise Editions for Windows, version 5.0

SUMMARY

MSVBVM50.EXE installs the latest versions of the Visual Basic run-time files that all applications created with Visual Basic 5.0 need in order to run.

MORE INFORMATION

The following file is available for download from the Microsoft Software Library: 

 ~ Msvbvm50.exe (size: 1307480 bytes) 
                  1.24MB
For more information about downloading files from the Microsoft Software Library, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q119591
   TITLE     : How to Obtain Microsoft Support Files from Online Services
MSVBVM50.EXE installs the following files, which ship with Visual Basic Service Pack 2 and Service Pack 3: 

   FILE           VERSION
   ------------   ----------
   MSVBVM50.DLL   05.00.4319
   OLEAUT32.DLL    2.20.4118
   OLEPRO32.DLL     5.0.4118
    STDOLE2.TLB    2.20.4118
   ASYCFILT.DLL    2.20.4118
     COMCAT.DLL    4.71
These files are the base dependencies for any component or application created in Visual Basic 5.0. Microsoft has made MSVBVM50.EXE available to developers in order to minimize the size of their downloads. If these files are the only dependencies of a Visual Basic 5.0 application, the developer can simply distribute the executable file and this Microsoft Knowledge Base article. Users who already have the run-time dependencies can use the application right away; otherwise they can download MSVBVM50.EXE and install them.

Many applications created with Visual Basic 5.0 will also require other specific ActiveX controls or DLLs. Developers should include instructions for obtaining those components if they are not included with their downloads.

REFERENCES

Users of Microsoft Internet Explorer can download some additional ActiveX Controls and DLLs frequently required by applications built with Visual Basic 5.0. See the following Microsoft Knowledge Base article for more information: 

   ARTICLE-ID: Q172991
   TITLE     : INFO: CAB Files Distributed with Visual Basic 5.0
