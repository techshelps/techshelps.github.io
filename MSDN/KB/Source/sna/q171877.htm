LU Creation Wizard Does Not Accept HEX Data When in HEX Mode

 Last reviewed: March 19, 1998
Article ID: Q171877
The information in this article applies to:
  • Microsoft SNA Server for Windows NT, version 3.0 and 3.0 Service Pack 1

SYMPTOMS

When you use the LU Creation Wizard to create a range of LUs, you can select hexadecimal as the Base setting, but the Wizard will not accept hexadecimal data in the First Extension field.

CAUSE

Snawiz.dll does not accept hexadecimal data even when the hexadecimal mode is selected.

RESOLUTION

A hotfix to SNA Server for Windows NT, version 3.0 (post SP1) is available to correct this problem.

STATUS

Microsoft has confirmed this to be a problem in SNA Server 3.0 and 3.0 Service Pack 1 (SP1). This problem was corrected in the latest SNA Server version 3.0 U.S. Service Pack.
