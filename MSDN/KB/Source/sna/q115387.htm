Bugs fixed:

2194 Fixes to SNA Server OS/2 client:

  • TRANSFER_MS_DATA verb does not work from OS/2 client
  • Invokable TP's do not work correctly on OS/2 clients
  • Autostarted TP's not working correctly due to SNA.INI parsing problem
  • "comm start nap \" hangs OS/2 client

2118 Case sensitivity causes 3270 groups from working on local domain

If an OS/2 user was configured as a member of a Windows NT group, and that group was given access to 3270 LU's using SNA Server, the OS/2 client would not retrieve the 3270 LU list. This was due to an OS/2 client problem handling case sensitive group information from the Windows NT server, in response to a NetUserGetGroups call.

Note: OS/2 clients must be running in the same domain as SNA Server. SNA Server 2.0 doesn't support OS/2 clients from a remote domain.

2193 ASCII to EBCDIC conversion problem with "_" character (Patch #40)

While IBM PC Support (and the SNA Server MS-DOS router) support the "_" character in an AS/400 username, the SNA Server Win16 and Win32 CSV convert verb would reject this character when converting using the AE conversion table. So, with this update applied, the 5250 applets will now support an AS/400 username which includes an underscore character.

Update Procedure:

- Stop the OS/2 client using the "COMM START NAP" command.

- Apply the updated OS/2 client DLL's to the target OS/2 machine.
