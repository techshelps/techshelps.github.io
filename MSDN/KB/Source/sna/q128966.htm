SNA Server X.25 PVC incoming calls.

 Last reviewed: November 1, 1995
Article ID: Q128966
The information in this article applies to:
  • Microsoft SNA Server for Windows NT, versions 2.0 and 2.1

SYMPTOMS

If an SNA Server X.25 Permanent Virtual Circuit (PVC) connection is configured for incoming calls only, the connection is never established.

CAUSE

The SNA Server Admin program should not allow this type of configuration. Outgoing calls should always be configured for PVC connections.

WORKAROUND

To work around this problem, always configure an X.25 PVC connection to support outgoing calls.

STATUS

Microsoft has confirmed this to be a problem in SNA Server versions 2.0 and 2.1. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
