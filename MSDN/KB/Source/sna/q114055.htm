Bugs fixed:

1786 Applet truncates screen prints after column 77

For international customers using A4 paper size with Windows, the emulator was truncating screen data after column 77.

1858 Trap in 3270 emulator

The 3270 emulator was not protecting the message queue used to receive messages from the server, causing the emulator's buffer pool to be corrupted, leading to an access violation.

1861 Newline key in 3270 applet does not work

SHIFT+ENTER was acting as an Enter key, and not a NEWLINE key.

1944 Memory leak during file transfer

Very large file transfers would fail over time due to a memory leak when the file transfer process bar was being updated (however, once the file transfer completed, the memory would be freed).

Update Procedure:

- For Windows 3.x clients, copy the above files into the client machine's 

  <snaroot> directory (default = C:\SNA).
- For Windows NT clients, copy the above files into the machine's 
  <snaroot>\SYSTEM directory.
