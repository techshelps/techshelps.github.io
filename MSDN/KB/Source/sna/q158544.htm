IBM Multiprotocol SDLC Adapter Not Work on HP Vectra XU 6/15

 Last reviewed: April 17, 1997
Article ID: Q158544

The information in this article applies to:
  • Microsoft SNA Server for Windows NT, versions 2.0, 2.1, 2.11, 2.11 Service Pack 1, and 3.0

SYMPTOMS

Notes:

  • The information in this article has been tested by Microsoft running Microsoft Windows NT Server version 4.0. This has not been confirmed by Hewlett-Packard; however, they did conclude that the IBM MPA/A is not listed as a supported or tested device on their Vectra series computers.
  • The following assumes you have disabled SERIAL.SYS, parallel ports, and any COM ports that use interrupts that could cause conflicts when you use this card.

After you configure an IBM Multiprotocol SDLC Adapter on a Hewlett-Packard Vectra XU 6/150 computer, the connection status may remain in an inactive state after you try to activate the service.

RESOLUTION

Use a different SDLC card. Supported adapters can be found in the SNA Server Companion Product Catalog and in Isvcatal.doc in the \Collatrl\General directory on the SNA Server compact disc.

The IBM and Hewlett-Packard products discussed here are manufactured by IBM and Hewlett-Packard Corporation respectably, vendors independent of Microsoft; we make no warranty, implied or otherwise, regarding these products performance or reliability.

Failed to Open device driver \Device\MPCA_1. Windows NT rc = c000003e EXPLANATION: An error occurred during an attempt to open the SDLC device driver. The return code from the OpenFile call is shown.
