Product: Microsoft SNA Server for Windows NT v2.0 Update: Patch #8 Status: Confirmed

- Includes Patch #3 fix

Module Updated: 

  SYSTEM\SNADMOD.DLL   12/02/93  12:47a  491520 (2.0.0.132)
  CLIENTS\WINNT\SYSTEM\SNADMOD.DLL  (same file as above)
Date: December 2, 1993

Bugs fixed:

1737 RECEIVE_ALLOCATE on APPC service TP name never attaches (Patch 3)

If APPC invokable TP's are being used with SNA Server, and the first character of a TP name is less than EBCDIC hex 0x40, the incoming allocate will fail.

1755 SNA Server hangs if unable to access configuration file

If SNA Server cannot access the configuration file located on the primary SNA Server machine, the server will go into a tight loop trying to open the file, giving the appearance that the SNA Server service is hung.

Update Procedure:

On the Windows NT machine where the invokable APPC TP is to run:

- go to a command prompt and enter "NET STOP SNABASE". This will stop 

  all SNA Server processes on the machine (and will force off all current
  SNA Server users if entered at the server).
- copy the attached module to the <SnaRoot>\SYSTEM directory and 
  <SnaRoot>\CLIENTS\WINNT\SYSTEM directory.
- restart by typing "NET START SNABASE".
