Microsoft SNA Server over SDLLC

 Last reviewed: July 30, 1996
Article ID: Q154031
The information in this article applies to:
  • Microsoft SNA Server for Windows NT, versions 2.0, 2.1, and 2.11

SUMMARY

Microsoft SNA Server can be configured to connect to a mainframe or AS/400 using SDLLC (SDLC routed to Token Ring or Ethernet). A Cisco router will only support PU 2.0 (XID format "0").

MORE INFORMATION

An SDLLC connection should be configured like any SDLC connection on the SNA Server. Information on the configuration properties can be found in 

Q112159 or in Appendix D of the MS SNA Server  Administration Guide.
Information for configuring 3270 device emulation (PU 2.0) to an AS/400 can be found in Q145954.
