Bugs fixed:

1736 Win32 Allocate hangs due to CSV, Management verb conflict

Depending on the order in which an application issues an APPC management verb and a common service verb, a subsequent APPC Allocate call may never return.

Update Procedure:

On the Windows NT, Windows 3.x or OS/2 client machine where the APPC program is to run:

- go to a command prompt and stop the SNA Server client software

- copy the appropriate SNA Server client module over the existing 

  version:

  Windows NT clients: <SnaRoot>\SYSTEM directory and
     <SnaRoot>\CLIENTS\WINNT\SYSTEM directory on the server

  Windows 3.x clients: <Winroot>\SYSTEM directory

  OS/2 clients: <SnaRoot>\DLL directory
- restart the SNA client software and the application
