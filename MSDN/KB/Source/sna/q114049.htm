Product: Microsoft SNA Server for Windows NT v2.0 Update: Patch #20 Status: Confirmed

PKZIP 2.04g format -- uncompress using the "-d" option.

Modules updated: 

 clients\win3x\wdmod.dll        01/21/94  04:46p  60928
 clients\win3x\wnap.exe         01/26/94  09:08p  38400
 clients\win3x\trace\wdmod.dll  01/21/94  04:47p  286034
 clients\win3x\trace\wnap.exe   01/26/94  09:09p  180558
Date: February 4, 1994

Bugs fixed:

1840 WNAP/NWCLI problems with signon & restart using 5250

- When running the Windows client (WNAP) on Netware, the login dialog box 

  will always be on top, unless the following entry is added to the [WNAP]
  section of WIN.INI:  AlwaysOnTop= NO
- Do not allow WNAP to be terminated when the 5250 emulator is running.

Update Procedure:

- Stop the SNA Server client software on the client machine.

- Copy updated modules over the previous versions: 

   <winroot>\SYSTEM\WDMOD.DLL
   <snaroot>\WNAP.EXE

  (keep the traced versions of this update for diagnostic purposes only --
  you may want to place them in a separate directory on SNA Server, such
  as \clients\win3x\trace\*.*)
