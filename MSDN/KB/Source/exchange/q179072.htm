XCON: cc:Mail Post Office Gets NFT Error

 Last reviewed: February 13, 1998
Article ID: Q179072
The information in this article applies to:
  • Microsoft Exchange Server version 5.0

SYMPTOMS

A cc:Mail post office gets the NFT error at random with following information in the event log: 

   Event ID: 59

   Lotus cc:Mail IMPORT from file *.ccm returned result code 1.  Program
   did not complete successfully.

   Event ID: 68

   Lotus cc:Mail EXPORT to file msg4987.ccm returned result code 1.  Could
   not access the Lotus cc:Mail Post Office.

CAUSE

The Microsoft Exchange Connector for Lotus cc:Mail redirects the import/export console messages to a temporary file. To do this, the Connector process creates a second process and uses the DuplicateHandle function to duplicate its file handles in the second process. The second process then uses those handles to perform the redirection. However, because of the way that this particular function operates, the duplicated handles are not registered in the handle table of the new process and this causes the redirected console messages to be written to random parts of the cc:Mail post office files.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Exchange Server version 5.0. This problem has been corrected in the latest U.S. Service Pack for Microsoft Exchange Server version 5.0. For information on obtaining the Service Pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
