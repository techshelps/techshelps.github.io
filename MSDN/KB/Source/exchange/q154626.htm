XADM: Err Msg: Extension SMTP Could Not Be Loaded

 Last reviewed: February 13, 1998
Article ID: Q154626
The information in this article applies to:
  • Microsoft Exchange Server, version 4.0

SYMPTOMS

You receive the following error message when attempting to look at the property page for the Internet Mail Connector or the MS Mail Connector in the Exchange Administrator program: 

   Extension 'SMTP' could not be loaded.

   The specified module could not be found.

   Microsoft Windows NT
   ID no:  0xc002007e

   Abort, Retry, Ignore
You receive the following error message when attempting to look at the properties on the MSMail Connector: 

   Extension 'MS' could not be loaded.

   The specified module could not be found.

   Microsoft Windows NT
   ID no:  0xc002007e

CAUSE

This occurs when the Exchsrvr\Add-Ins subdirectory is not shared or does not have the correct permissions.

MORE INFORMATION

When Microsoft Exchange Server is installed, some of the subdirectories of the Exchsrvr directory are shared. For more information on the default Microsoft Exchange Server shares and permissions, please see the following article in the Microsoft Knowledge Base:

Article ID: Q147376 

TITLE     : XADM: Names and Permissions of Exchange Server Shares
This error can also occur if the Microsoft Exchange Service Account or the Everyone group has had the "Access this computer from network" user rights policy removed in the Windows NT User Manager for Domains.

RESOLUTION

Ensure that the Exchsrvr\Add-ins subdirectory is shared and that the proper permissions are granted.
