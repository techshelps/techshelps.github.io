XCLN: Public Folders with Unread Messages Not Bold

 Last reviewed: September 23, 1997
Article ID: Q137208
The information in this article applies to:
  • Microsoft Exchange Windows client, versions 4.0 and 5.0
  • Microsoft Exchange Windows NT client, versions 4.0 and 5.0
  • Microsoft Exchange Windows 95 client, versions 4.0 and 5.0

Public folder names are not displayed in bold letters when there are unread messages in the public folders. This is contrary to the behavior of private folders that are displayed in bold letters when there are unread messages in them.

To be notified that a public folder contains unread messages, create a shortcut to that particular public folder. When the public folder contains unread messages, the shortcut to the public folder will then be displayed in bold letters.

To update the public folders instead of just the shortcuts would be prohibitively uneconomical in terms of CPU and network bandwidth, as there could be thousands of public folders and this would have to be done for each user.
