XADM: Error Running ISINTEG on a Cluster Server

 Last reviewed: February 6, 1998
Article ID: Q178311
The information in this article applies to:
  • Microsoft Exchange Server, version 5.5

SYMPTOMS

When you run ISINTEG -patch on a Microsoft Cluster Server, you may receive the following error: 

   The private store could not be updated
   Reason: JET_errKeyDuplicate

WORKAROUND

Before you run the ISINTEG -patch on a Microsoft Exchange Server computer running on a Microsoft Cluster Server, you must set the environment variable _CLUSTER_NETWORK_NAME_ to the network name of the cluster. For example, if your cluster network name is EXCLUSTER, type the following at the command prompt: 

   SET _CLUSTER_NETWORK_NAME_=EXCLUSTER
