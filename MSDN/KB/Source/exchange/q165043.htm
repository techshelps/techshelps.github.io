XADM: How to Install Exchange 4.0 SP1 in Site with 5.0 Servers

 Last reviewed: October 21, 1997
Article ID: Q165043
The information in this article applies to:
  • Microsoft Exchange Server, version 4.0
  • Microsoft Exchange Server, version 5.0

SUMMARY

When you have installed at least one Microsoft Exchange Server, version 5.0, in a Microsoft Exchange Site that previously only contained version 4.0 Servers, the Schema was upgraded in order to support new object types such as NNTP, LDAP, and so forth. This directory information was then replicated throughout the Site. If you now attempt to install a new version 4.0 Server in this Site, it will fail with the following error message: 

   An internal error has occurred during directory replication. Stop and
   then restart the directory service. Then try directory replication
   again.

   Microsoft Exchange Server Setup
   ID no:c1030b0d

MORE INFORMATION

To successfully add a version 4.0 Server to this Site, you must first make sure that you are attempting to install the Service Pack 1 (SP1) release of Microsoft Exchange Server, version 4.0. If the file dates on the compact disk are earlier than July 1996, you do not have the SP1 version. Once you have the correct compact disk, follow these steps:

  1. Copy the \setup\<platform> directory, and all of its sub-directories from the Microsoft Exchange Server, version 4.0 SP1 compact disk to an accessible location such as a local drive or network share.
2. Copy Setup.exe from the Microsoft Exchange Server, version 5.0 compact 
   disk in the \support\Exch40\<platform> directory to the installation
   tree that you created during step 1, overwriting the existing Setup.exe.
Because the new Setup.exe accounts for the Schema differences, it will successfully complete the replication portion of the installation process.
