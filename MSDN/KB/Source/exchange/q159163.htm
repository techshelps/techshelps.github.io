XCLN: Exchange Client PST Limitations

 Last reviewed: October 20, 1997
Article ID: Q159163
The information in this article applies to:
  • Microsoft Exchange Windows 3.x client, version 4.0
  • Microsoft Exchange Windows NT client, version 4.0
  • Microsoft Exchange Windows 95 client, version 4.0
  • Microsoft Outlook 97 for Windows 95, version 8.0
  • Microsoft Outlook 97 for Windows 95, version 8.01
  • Microsoft Outlook 97 for Windows 95, version 8.02
  • Microsoft Outlook 97 for Windows NT, version 8.0
  • Microsoft Outlook 97 for Windows NT, version 8.01
  • Microsoft Outlook 97 for Windows NT, version 8.02

SYMPTOMS

You are unable to add any items (posted messages or files) to an existing personal folder (PST file)in the Microsoft Exchange client. When attempting to do so, one of the following error messages may occur: 

   An item could not be copied. The action could not be completed.

   An item could not be moved. The action could not be completed.

   Changes to the item could not be saved. The action could not 
   be completed.

   General Microsoft Exchange failure. Make sure Microsoft Exchange
   is installed correctly then try again.
In Microsoft Outlook 8.0x, you may receive one of the following errors when attempting to copy or move an item to a folder in a .PST file that contains more than 16,383 items: 

   Could not make another copy of the item. The original was either moved
   or deleted, or access was denied.

   The item could not be moved. It was either moved or deleted, or access
   was denied.

CAUSE

PST files are limited to items of 16,384 in a table. This means PST files can accept 16,384 folders of up to 16,384 items in a folder. The maximum size a PST can be is 2GB total file size.

RESOLUTION

To store more information, create additional personal folders (PST files) or delete some information from the existing PST file.
