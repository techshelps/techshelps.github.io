XCLN: Post Button Enabled Even If User Doesn’t Have Permissions

 Last reviewed: January 29, 1998
Article ID: Q169851
The information in this article applies to:
  • Microsoft Outlook Web Client version 5.5

In Microsoft Outlook Web Client version 5.5 the option to post to a public folder is enabled even if a user does not have write access to the public folder.

If the user attempts to post a message, the following error is generated: 

   You do not have the necessary permissions to post to this folder.
   Contact your Exchange Administrator for assistance or choose another
   folder.
The message is not posted if the user does not have write access. This behavior is by product design.
