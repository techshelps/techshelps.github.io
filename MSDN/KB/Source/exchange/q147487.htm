XCLN: Forward Doesn't Close Message if Attachment is Open

 Last reviewed: March 19, 1997
Article ID: Q147487
4.00 WINDOWS kbusage kbbuglist

The information in this article applies to:

  • Microsoft Exchange Windows 3.x client, version 4.0

SUMMARY

If you use the Microsoft Exchange client for Windows 3.x, after you open a message and then open a document attached to it, when you attempt to forward the message, the original message will not close. After you have completed forwarding the message, when you close the original message you will receive a dialog box with the following text: 

   Do you want to save changes to this item?
You will be able to choose either Yes, No, or Cancel.

MORE INFORMATION

Choose No to close the message without saving it.

This behavior will only occur if an attachment is opened in the message before forwarding the message.

STATUS

Microsoft has confirmed this to be a problem in version 4.0 of the Microsoft Exchange client for Windows 3.x. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
