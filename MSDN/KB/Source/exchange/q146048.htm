XCLN: Changing Fonts for Richedit Field Doesn't Resize Field

 Last reviewed: March 31, 1997
Article ID: Q146048

The information in this article applies to:
  • Microsoft Exchange Windows client, versions 4.0 and 5.0
  • Microsoft Exchange Windows 95 client, versions 4.0 and 5.0
  • Microsoft Exchange Windows NT client, versions 4.0 and 5.0

SYMPTOMS

When you drop a new rich entry field onto a newly created forms canvas and change the font size property of that rich entry field, when you reposition this field, the horizontal size of the field does not correctly resize to reflect the new font size. The height property of the field does correctly show the new size related to the new font size.
