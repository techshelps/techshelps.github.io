XCLN: Internet Tab Missing on Message Properties Send Options

 Last reviewed: February 16, 1998
Article ID: Q181029
The information in this article applies to:

- Microsoft Outlook Windows 95 client, versions 8.00, 8.01, 8.02, 8.03 - Microsoft Outlook Windows NT client, versions 8.00, 8.01, 8.02, 8.03 - Microsoft Exchange Windows 3.x client, versions 4.0, 5.0, 5.5 - Microsoft Exchange Windows 95 client, versions 4.0, 5.0, 5.5 - Microsoft Exchange Windows NT client, versions 4.0, 5.0, 5.5

SYMPTOMS

When you create a new message, the Internet tab is not visible on the Send Options property page.

CAUSE

When the Microsoft Exchange or Microsoft Outlook client starts, it checks the Gateway Address Routing Table (GWART) update time and checks to see whether the GWART has changed since the last logon. If the GWART has changed, the client requests from the Microsoft Exchange Information Server to download all of the organization's unique address types and associated option templates. The Microsoft information store downloads these address types and associated option templates in alphabetical order. The client then stores these address types and option templates in the profile within the registry.

Because of a size limitation for registry keys in Windows 95, the number of message types and associated option templates that can be stored in the profile is limited to 16.

If a Microsoft Exchange organization has more than 16 unique address types that start with letters "A" through "R", the Simple Mail Transfer Protocol (SMTP) address type and option template cannot be stored in the profile. The option template contains the necessary information to build the Internet tab on the Send Options Property dialog box. Because it is missing from the profile, the dialog box will not be built and is not seen by the user.

STATUS

Microsoft has confirmed this to be a problem in the clients listed at the beginning of this article. A supported fix is now available, but has not been fully regression-tested and should be applied only to systems experiencing this specific problem. Unless you are severely impacted by this specific problem, Microsoft recommends that you wait for the next Service Pack that contains this fix. Contact Microsoft Technical Support for more information.
