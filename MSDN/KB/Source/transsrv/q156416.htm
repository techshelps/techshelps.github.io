FIX: Allows Your Own Machine Name for a Remote Component

 Last reviewed: October 29, 1997
Article ID: Q156416
The information in this article applies to:
  • Microsoft Transaction Server, version 1.0

SYMPTOMS

Microsoft Transaction Server Explorer allows you to specify the local machine name for a remote computer. As a result, there are cases where you will receive a REGDB_E_CLASSNOTREG on CoCreateInstance when you try to create an object from one of the local packages.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Transaction Server version 1.0. This problem was corrected in Microsoft Transaction Server version 1.1. For information on obtaining version 1.1, please see the following article the Microsoft Knowledge Base:
