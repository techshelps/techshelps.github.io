XL5: GP Fault Converting Object in Worksheet Object

 Last reviewed: September 9, 1996
Article ID: Q120160
The information in this article applies to:
  • Microsoft Excel for Windows, version 5.0

SYMPTOMS

When a Microsoft Excel 5.0 Worksheet object is open for in-place editing in a client application and you insert an object in the worksheet object, if you choose the Object Convert command while the object is not selected, you receive a general protection (GP) fault.

CAUSE

This problem occurs only when you are editing a Microsoft Excel 5.0 Worksheet object in an application that supports in-place editing (such as Microsoft Word version 6.0). When you select the inserted object (such as a Microsoft Graph chart) in the Microsoft Excel 5.0 Worksheet object, you can choose Chart Object from the Edit menu, and then choose Edit, Open, or Convert to modify the object.

However, if you select a cell in the worksheet object, so that the chart object is no longer selected, and you choose the Edit menu again, the Chart Object command is still available. If you then choose Convert, you receive a GP fault. If you choose either Edit or Open, the object is not activated, but you do not receive a GP fault.

This problem occurs because the Chart Object command is still available, even though the chart object is no longer selected.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
