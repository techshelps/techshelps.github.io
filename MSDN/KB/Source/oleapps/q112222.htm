MS Graph: Linked Data Overwrites Row and Column Headings

 Last reviewed: January 20, 1997
Article ID: Q112222
The information in this article applies to:
  • Microsoft Graph for Windows, version 5.0
  • Microsoft Graph 97 for Windows

SYMPTOMS

When you import data to a Microsoft Graph datasheet using the Paste Link command, the row and column headings on the datasheet are overwritten.

WORKAROUNDS

To include row and column headings when you link data to a Microsoft Graph datasheet, do the following:

  1. In the source application, include row and column headings when you select the data to import. Copy the data you want to import.

  2. In the Microsoft Graph datasheet, choose Paste Link from the Edit menu.

  3. Click OK to overwrite the existing data.

  4. If you are using Graph 5.0, the ChartWizard--Step 1 of 1 dialog box appears. (Assuming Data Series in Rows) If you are importing row headings, under Use First Row For, select the Category (X) Axis Labels option. If you are importing column headings, under Use First Column For, select the Legend Text Option. Choose OK.

    If you are using Graph 97, the ChartWizard does not run.

Note that when you use the Paste option to import data, the existing row and column headings on the datasheet are not overwritten.

MORE INFORMATION

When you paste link data from a source application to a Microsoft Graph datasheet, a dialog box appears with the message that the existing data will be overwritten. This includes existing row and column headings on the datasheet.

Any changes you make to the Microsoft Graph datasheet after you link data are overwritten when you update the link. To make changes to the datasheet, change the data in the source file.
