ClipArt Gallery Cannot Find <FILENAME.PCS>

 Last reviewed: February 25, 1998
Article ID: Q114373
The information in this article applies to:
  • Microsoft PowerPoint for Windows, versions 4.0, 4.0a
  • Microsoft ClipArt Gallery 1.1

SYMPTOMS

Microsoft PowerPoint for Windows ships with a version of Microsoft ClipArt Gallery 1.1 that can work with files in the PictureStore (.PCS) format. When you use ClipArt Gallery 1.1 to insert an image from a .PCS file, you may receive one of the following error messages 

   The file <PATH><FILENAME.PCS> cannot be found. Do you want to
   remove the thumbnail for this picture from the ClipArt Gallery.

   -or-

   The ClipArt Gallery could not read the file <PATH><FILENAME.PCS>.

   -or-

   The file <PATH><FILENAME.PCS> could not be read.
where <FILENAME.PCS> is the name of the .PCS image file installed by PowerPoint 4.0 and <PATH> is the location of the image file.

If you try to save a presentation, you may also receive the first error message along with the error messsage: 

   Too many files open.

RESOLUTION

If <FILENAME.PCS> is in the correct location and is the right size (see the "More Information" section in this article below) follow these steps.

NOTE: The following steps assume you have Windows 3.1 installed in the WINDOWS directory on drive C, and your floppy disk drive is drive A. If Windows 3.1 is loaded in a different drive/directory, substitute the correct location. If you are using floppy disk drive B instead of drive A, substitute B for A.

  1. Retrieve the PowerPoint 4.0 Setup disks.

  2. Exit Windows.

  3. Rename the current PICSTORE.DLL file to PICSTORE.OLD by typing the following command and pressing ENTER: 

          REN C:\WINDOWS\SYSTEM\PICSTORE.DLL C:\WINDOWS\SYSTEM\PICSTORE.OLD

    (If Windows is installed into a different directory than C:\WINDOWS, modify the above command as necessary.)

  4. Place PowerPoint 4.0 Setup disk 1 in drive A.

  5. At the MS-DOS command prompt, type the following and press ENTER: 

          a:\decomp a:\picstore.dl_ c:\windows\system\picstore.dll

  6. Return to Windows and try using ClipArt Gallery again.

If the error message is regenerated after step 6, Windows may be experiencing an OLE conflict. At this point, back up all .PPT files, delete the following files and directories, and reinstall PowerPoint 4.0.

NOTE: If this is a workstation installation, the location of these files will vary. Consult you PowerPoint Help file for the locations of these files on a network. 

   Files                        Location
   --------------------------------------

   All Files                    WINDOWS\MSAPPS\ARTGALRY

   ARTGALRY.INI                 WINDOWS

   All Files                    POWERPNT\SETUP

   PICSTORE.DLL                 WINDOWS\SYSTEM
   PUBOLE.DLL
   COMPOBJ.DLL
   OLE2.DLL
   OLE2CONV.DLL
   OLE2DISP.DLL
   OLE2NLS.DLL
   OLE2PROX.DLL
   STORAGE.DLL
   TYPELIB.DLL
NOTE: Deleting ARTGALRY.INI prevents other applications' clip art from being represented in the ClipArt Gallery. Also, deleting ARTGALRY.IDX and ARTGALRY.THM causes the loss of any other images previously registered in ClipArt Gallery. If you have any other applications that use the ClipArt Gallery, you will need to reinstall them to ensure that their thumbnails are represented in the ClipArt Gallery.

To resolve the error message "Too many files open" you should:

  1. Edit your AUTOEXEC.BAT file to include the following:

    SHARE.EXE /L:500 /F:5100

  2. Set FILES=60 in your CONFIG.SYS file (unless you have a software application that requires a higher setting).
