OFF4: Error When Patching Older Version of ClipArt Gallery

 Last reviewed: February 3, 1998
Article ID: Q150578
The information in this article applies to:
  • Microsoft ClipArt Gallery, version 1.1
  • Microsoft Office for Macintosh, versions 4.2, 4.2.1
  • Microsoft PowerPoint for Macintosh, version 4.0
  • Microsoft Works for Macintosh, version 4.0

SYMPTOMS

When you attempt to patch the ClipArt Gallery program you may see a message similar to the following: 

   "ClipArt Gallery" (version 0.1.2) is older than version 0.1.4.

   This patch cannot be applied.
The only option button available is the Cancel button.

CAUSE

The patch was designed to work with version 0.1.4. You will not be able to patch any other version of the ClipArt Gallery.

RESOLUTION

To resolve this problem, use the following steps to remove the old ClipArt Gallery and then re-install ClipArt Gallery version 0.1.4.

Remove the ClipArt Gallery

  1. Back up your original ClipArt Gallery to a disk, and then delete the ClipArt Gallery from your computer.

    NOTE: The ClipArt Gallery program is normally located in the Microsoft folder.

  2. Run Microsoft Office Setup or Microsoft PowerPoint Setup.

    Setup should be started in maintenance mode.

  3. Click the Add/Remove button.

  4. Under Installed Components, click the option triangle for Shared Applications.

    This reveals a list of all installed Shared Applications. If ClipArt Gallery does not appear in this list, skip the remaining steps and go to the "Add the ClipArt Gallery" section.

  5. Clear (remove the X from) all items within the Shared Application section with the exception of ClipArt Gallery.

    After you complete this step, you should see marks in only two check boxes: ClipArt Galley and Shared Applications.

  6. Click the Continue button to remove ClipArt Gallery and its registration information.

  7. Double check that no ClipArt Gallery application exists on the hard disk drive.

Add the ClipArt Gallery

  1. Run Microsoft Office Setup or Microsoft PowerPoint Setup again.

  2. Under Components Not Yet Installed, click the option triangle for Shared Applications.

  3. Clear (remove the X from) all items within the Shared Application section with the exception of ClipArt Gallery.

    When you have completed this step, you should see marks in only two boxes: ClipArt Galley and Shared Applications.

  4. Click the Continue button to install the ClipArt Gallery.

Attempt to run the patch again. If the version of ClipArt Gallery on your installation disks is version 0.1.4, the patch should now work.

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

When the patch mentions ClipArt Gallery version 0.1.2, it is not referring to the version displayed in the Get Info window. It is referring to the version listed in the vers resource. This vers resource is only visible after you open the ClipArt Gallery with a special development tool.

After the patch is applied to the ClipArt Gallery, the version listed in the Get Info window will change from 1.1 to 1.1a.

TIP: Enable Show Version within the Views Control Panel. When you view the Microsoft folder by Name, Size, Kind, Label, or Date, the short version string (0.1.2 or 0.1.4) will be visible.
