Err Msg After Installing Norton 1994 Upgrade: Unable to Launch

 Last reviewed: September 9, 1996
Article ID: Q117908
The information in this article applies to:
  • Microsoft Publisher for Windows, version 2.0
  • Microsoft Draw version 1.0
  • Microsoft ClipArt Gallery versions 1.x
  • Microsoft WordArt version 2.0

SYMPTOMS

Installing the Norton Compatibility Upgrade 1994 for the Norton Desktop results in Microsoft applications such as WordArt, ClipArt Gallery, and Microsoft Draw no longer working in OLE version 2.0 applications. These applications still function properly in OLE version 1.0 applications. (For example, they do not work in Publisher 2.0, but do work in Microsoft Write for Windows.)

After you install the Norton Compatibility Upgrade 1994 for the Norton Desktop, you receive the following error message: 

   Unable to launch or communicate with server application...

CAUSE

The Norton Compatibility Upgrade installs the following two dynamic-link libraries (DLLs): 

   COMPOBJ.DLL 4/8/94
   STORAGE.DLL 4/8/94
These files make up only a portion of OLE 2.0. By updating these two files without updating the other OLE 2.0 files, functionality can be impaired.

WORKAROUND

To allow Publisher and the shared applications to function correctly, do the following:

  1. Exit Windows.

  2. Change to the Windows SYSTEM subdirectory by typing the following, pressing ENTER after each line: 

          cd windows
      cd system

  3. Rename the two DLLs with the following commands, pressing ENTER after each command:

    rename compobj.dll compobj.old rename storage.dll storage.old

  4. Restart Windows.

  5. Reinstall Publisher 2.0, choosing the "Custom" setup option. When asked which Publisher components you want to install, choose Program Files, and clear all other options.

Publisher is now fully installed.
