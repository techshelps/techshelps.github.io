Restoring Missing Axis Labels in Microsoft Graph 5.0

 Last reviewed: September 10, 1996
Article ID: Q117664
The information in this article applies to:
  • Microsoft Graph for Windows, version 5.0

SYMPTOMS

Alternating labels of a Graph 5.0 chart do not display if long text labels on the x-axis overlap. Changing the font size or stretching the graph to provide enough room for the text to fit as currently wrapped does not always fix the overlap problem. A change in either the font size or the graph size changes the wrap formatting, but the text still does not display.

WORKAROUND

To specify a position for wordwrap or prevent a word from wrapping from the second line back to the first, you can insert a carriage return in the text.

  1. Double-click the text cell in the datasheet to edit the contents.

  2. Position the cursor and remove unwanted spaces.

  3. Press the ALT+ENTER key combination.

This procedure inserts a carriage return in the text and keeps the text on multiple lines.

MORE INFORMATION

Some long text labels on the x-axis do not wordwrap properly. Resizing the graph or changing the font size allows the text to fit as long as the position at which the wordwrap occurs does not change. However, words on the second line often wrap back to the first line. This prevents the labels from displaying. Placing a carriage return in the text prevents wordwrapping and allows the text to fit.
