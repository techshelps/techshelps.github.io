Error Message: "Divide by Zero" with Corrupted DEFAULT.GRA

 Last reviewed: September 10, 1996
Article ID: Q84273
The information in this article applies to:
  • Microsoft Graph for Windows version 3.0

SUMMARY

In Microsoft Graph, if you receive the error message "Divide by Zero," your default datasheet and chart file, DEFAULT.GRA, is probably corrupted. To correct this problem, you must delete the C:\WINDOWS\DEFAULT.GRA file.

MORE INFORMATION

After you delete the DEFAULT.GRA file, Graph returns to the system Datasheet and Chart settings. You can use these defaults or create new default settings by re-creating DEFAULT.GRA.

If the "Divide by Zero" error message persists after you delete DEFAULT.GRA, you should delete the files in your WINDOWS\MSAPPS\MSGRAPH subdirectory and reinstall Graph using the Word for Windows version 2.0 Setup program.

Microsoft has confirmed this to be a problem in Graph version 3.0. We are researching this problem and will post new information here as it becomes available.

REFERENCES

"Microsoft Graph User's Guide," version 3.0, page 10
