Missing or Incorrect Characters in WordArt Object

 Last reviewed: August 6, 1997
Article ID: Q93793
The information in this article applies to:
  • Microsoft WordArt version 1.0

SUMMARY

If you try to create an object with Microsoft WordArt, the preview area of the WordArt dialog box may remain blank, contain incorrect characters, or correct characters formatted with an incorrect font. Choosing Exit and Return from the File menu returns a WordArt object that matches the preview. Any combination of these problems may occur if the Fontfx.fnt file is corrupted.

Fontfx.fnt is the file containing the polygons that WordArt uses to generate its letters and numbers. It is located in the Msapps\WordArt subdirectory of the Windows directory. Its file size should be 385,536 bytes. The file date depends on the which product you received WordArt with.

Microsoft WordArt ships with the following products:

  • Microsoft Publisher for Windows, version 1.0
  • Microsoft Publisher Design Pack version 1.0
  • Microsoft Word for Windows, version 2.0

MORE INFORMATION

To correct this problem, follow these steps:

  1. Rename or delete the current copy of Fontfx.fnt.

  2. Run Setup for the program with which you received WordArt. To do this, insert Disk 1 in drive A or B, and type A:\SETUP (or B:\SETUP) at the command prompt.

  3. Choose to perform a custom setup when prompted.

  4. A checklist of the various program components appears. If there is an option for WordArt alone, select that option only. If there is no option for WordArt alone, select the Program Files option.

  5. Place the appropriate disks in the appropriate drive when prompted by the Setup program.

Some programs, such as Microsoft Publisher, have methods for manually decompressing files from the distribution disks. If the program supports this functionality, you can decompress Fontfx.fnt from the disks and copy them to the WINDOWS\MSAPPS\WORDART directory. For more information, query on the following words in the Microsoft Knowledge Base: 

   <the program's name> and manual and decompress and file
