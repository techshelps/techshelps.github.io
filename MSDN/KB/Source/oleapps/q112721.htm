ClipArt Gallery: Cannot Create Index File Error

 Last reviewed: February 25, 1998
Article ID: Q112721

The information in this article applies to:

  • Microsoft ClipArt Gallery for Windows, version 1.x
  • Microsoft Office for Windows, version 4.x
  • Microsoft PowerPoint for Windows, version 4.x
  • Microsoft Publisher for Windows, version 2.x
  • Microsoft Works for Windows, version 3.x

SUMMARY

The first time you try to insert clipart using Microsoft ClipArt Gallery for Windows, you may receive the following error message: 

   ClipArt Gallery: Cannot Create Index File

MORE INFORMATION

The three main causes of this error are described below.

Cause 1: A Lack of Free Disk Space for Modifying the Artgalry.idx File

In File Manager, check the amount of free space on the hard disk. Make sure there is at least 5 megabytes (MB) of available space on the hard disk. (This is not the amount of space required for the index file (.idx), but it does help Windows run more reliably.)

Cause 2: Invalid Paths to the .icx Files in C:\Windows\Artgalry.ini

Use Notepad to view the Artgalry.ini file. Make sure that the path for each entry in the Artgalry.ini file is valid. For example, if the [Import] section of the Artgalry.ini file includes the entry 

   File=C:\Winapps\Works3\,<collection>.idx,...
where <collection>.idx is the name of the index and thumbnail files associated with a collection of clip art graphics files, such as the ones that ship with several Microsoft products. For example, an entry for Microsoft Works might look like: 

   File=C:\Winapps\Works3\,Wks3art.idx, Wks3art.thm,...
Make sure the path is correct for all the file names in the [Import] section of the Artgalry.ini file, especially the line that has the name of the index file that appears in the error message.

Cause 3: A Corrupted <collection>.idx or Artgalry.idx...

To establish whether or not an .IDX file is corrupted, use Notepad to edit the C:\Windows\Artgalry.ini file. In the [Import] section, you should find the following entry: 

   File=C:\Msworks\Clipart\,<collection>.idx,<collection>.thm,...
where <collection>.idx is the name of the index file that appears in the error message.

Place a semicolon (;) at the front of this line (for example, ;File=c:) to disable it. Save the file and quit Notepad.

Now try inserting clip art using ClipArt Gallery. If you do not receive the error message, the .idx file is probably corrupted, and you will need to either manually add the clip art images, or remove and then reinstall the clip art images using the Setup program.

To manually add the clip art do the following:

  1. In ClipArt Gallery, click Options.

  2. Click Add.

  3. Switch to the directory that contains the clip art graphics. For example, the location of the clip art graphics that come with Microsoft Works might be the C:\Msworks\Clipart directory.

  4. Click Select All. Click OK.

At this point, you are asked whether or not you want to categorize your clip art. If you click No, the process is much faster, but both options are valid. The clip art is then added to ClipArt Gallery.

If the clip art graphics are stored as Picture Store (*.PCS) files, you will not be able to add the clip art manually and you will have to use the Setup program for the application that provided the clip art to remove the clip art that is causing the error and them run Setup again to add the clip art images back in.
