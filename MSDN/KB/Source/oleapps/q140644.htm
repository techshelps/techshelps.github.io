CG: ClipArt Gallery Requires Write Access to Registry

 Last reviewed: March 11, 1998
Article ID: Q140644
The information in this article applies to:
  • Microsoft ClipArt Gallery version 2.0
  • Microsoft Office for Windows 95, version 7.0
  • Microsoft Windows NT Workstation version 3.51
  • Microsoft Windows NT Server version 3.51

SYMPTOMS

If you run a Microsoft Office for Windows 95 program, such as PowerPoint, and you attempt to insert clip art while logged on to a Microsoft Windows NT 3.51 computer as a normal user, the following ClipArt Gallery error message may appear: 

   The ClipArt Gallery requires write access to the system
   registry. You are currently logged onto this machine as a
   guest, which does not allow this access. Log on as a user or
   administrator to use the ClipArt Gallery.
You then receive the following Microsoft PowerPoint error message: 

   The server application, source file, or item cannot be found.
   You should reinstall the server application.

CAUSE

This problem occurs because the Office 95 Setup program did not provide write access to the ClipArt Gallery portion of the registry to the Users group.

RESOLUTION

To resolve this problem, obtain the Cag20a.exe file. This file is available for download from the Microsoft Software Library: 

 ~ cag20a.exe
For more information about downloading files from the Microsoft Software Library, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q119591
   TITLE     : How to Obtain Microsoft Support Files from Online
               Services
