DRW97: Draw 97 Setup Requires Mso97.dll and Mso7enu.dll

 Last reviewed: February 23, 1998
Article ID: Q181522
The information in this article applies to:
  • Microsoft Draw 97 for Windows

SYMPTOMS

When attempting to install Microsoft Draw 97, you may receive the following error message: 

   Setup could not find at least one Microsoft Office 97 application
   installed on your system. Please install at least one Microsoft Office
   97 application before installing this application.

   -or-

   Setup could not find a matching language version of Office 97 installed
   on your system. Please install a matching language version of Office 97
   before installing this application.
followed by: 

  Microsoft Draw 97 was not completed successfully.

CAUSE

Draw 97 requires Mso97.dll and Mso7enu.dll. If the Draw 97 setup program is unable to find both of these dynamic-link library (DLL) files, it does not continue.

RESOLUTION

To resolve this problem, you must have Microsoft Office 97 (or at least one of the Office programs) installed on your computer prior to installing Draw 97.

STATUS

This behavior is by design. To install Draw 97, you must have at least one Office program installed on your computer.

MORE INFORMATION

Draw 97 only installs if it finds the Mso97.dll file and the matching language DLL (for example, Mso7enu.dll). Setup uses the following logic when attempting to locate Mso97.dll: 

   1. Looks in the Windows registry for Mso97.dll.

   2. If not in the registry, it searches local disk drives. It also
      searches for Mso7enu.dll on local disk drives.
NOTE: If Office 97 is installed on a network drive or removable disk drive, Draw may not work correctly.
