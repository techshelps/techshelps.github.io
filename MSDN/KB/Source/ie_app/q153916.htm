Cannot Delete Source File of Message Attachment

 Last reviewed: September 2, 1997
Article ID: Q153916
The information in this article applies to:
  • Microsoft Internet Mail and News version 1.0 for Windows 95
  • Microsoft Internet Mail and News version 1.0 for Windows NT 4.0

SYMPTOMS

When you try to delete the source file for a file that was attached to an open new message by dragging the file to the message, you may receive one of the following error messages.

In Microsoft Windows 95

   Cannot delete <filename>: Access denied.

   Make sure the disk is not full or write-protected and that the
   file is not currently in use.

In Microsoft Windows NT 4.0

   Cannot delete mail attachment: There has been a sharing violation.
   The source or destination file may be in use.

RESOLUTION

To work around this issue, use one of the following methods:

  • Delete the source file after the new message is closed.
  • Attach the file by using the File Attachment command on the File menu, rather than by dragging the file to the message. Using the File Attachment command allows the source file to be deleted while the new message is still open.
