Error Message "HTTP/1.0 404" When Administering Web Pages

 Last reviewed: August 27, 1997
Article ID: Q162338
1.00 1.00a WINDOWS kberrmsg kbnetwork

The information in this article applies to:

  • Microsoft Personal Web Server versions 1.0, 1.0a for Windows 95

SYMPTOMS

When you are administering Microsoft Personal Web Server version 1.0 for Windows 95, you may receive the following error message: 

   HTTP/1.0 404 Object Not Found.

CAUSE

The Htmla folder may be missing or not shared. This folder contains files necessary to administer the Personal Web Server Web pages, and needs to be shared for Personal Web Server to operate properly.

RESOLUTION

If the Htmla folder is missing, then you must reinstall Personal Web Server. If the Htmla folder is not shared, then you must share this folder to restore proper operation of the program. To do so, follow these steps:

  1. Double-click My Computer.

  2. Double-click the C: drive (where C: is the drive that Personal Web Server is installed on).

  3. Double-click the Program Files folder

  4. Double-click the Websvr folder.

  5. Right-click the Htmla folder, and then click Sharing.

  6. Click Shared As, and then click Web Sharing.

  7. Click the following check boxes to select them: 

        - Share Folder For HTTP
    - Read Only
    - Execute Scripts

  8. Click OK, and then click OK again.
