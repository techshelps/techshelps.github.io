Problems Using Internet Explorer with a Proxy Server

 Last reviewed: October 2, 1997
Article ID: Q165826
The information in this article applies to:
  • Microsoft Internet Explorer version 3.01, 3.02, 4.0 for Windows 95

SYMPTOMS

IMPORTANT: This article contains information about editing the registry. Before you edit the registry, you should first make a backup copy of the registry files (System.dat and User.dat). Both are hidden files in the Windows folder.

When you use any of the versions of Internet Explorer listed at the beginning of this article with a proxy server to browse the Internet, you may experience any of the following symptoms:

  • Slow connections.
  • Inconsistent downloads.
  • An inability to view non-Microsoft sites.
  • Other odd behavior.

CAUSE

This behavior can occur when you are using the Novell NetWare 32-bit client software.

RESOLUTION

To resolve this problem, verify that you are using the Novell NetWare 32- bit client and that you are connecting to the Internet using a proxy server.

Step 1: Verify That You Are Using the Novell Client

a. Click Start, point to Settings, and then click Control Panel.

b. Double-click the Network icon, and click the Configuration tab.

c. Verify that Novell 32 appears in the "The following network components 

   are installed" box.
Step 2: Verify That You Are Connecting Using a Proxy Server

a. In Internet Explorer, click Options on the View menu, and click the 

   Connection tab.
b. Look at the "Connect through a proxy server" check box and the entries 
   under "Address of proxy to use."
If the check box is selected and if the "Address of proxy to use" boxes contain entries, Internet Explorer is set up to connect to the Internet through a proxy server.

Step 3: Add DontUseDNSLoadBalancing to the Internet Settings Registry Key

If you are using the Novell NetWare 32-bit client and are connecting to the Internet using a proxy server, use the following information to resolve the issue.

WARNING: Using Registry Editor incorrectly can cause serious problems that may require you to reinstall Windows 95. Microsoft cannot guarantee that problems resulting from the incorrect use of Registry Editor can be solved. Use Registry Editor at your own risk.

NOTE: For information about how to edit the registry, view the Changing Keys And Values online Help topic in Registry Editor (Regedit.exe). Note that you should make a backup copy of the registry files (System.dat and User.dat) before you edit the registry.

To resolve the issue, create a new binary value named 

   DontUseDNSLoadBalancing (make sure to type this name exactly as shown)
under the following registry key: 

   HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\
   Internet Settings.
Set the value of the new value to: 

   01 00 00 00
After you make these changes, restart your computer.
