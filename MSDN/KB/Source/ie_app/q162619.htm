No Messages Exported from Internet Mail to Microsoft Exchange

 Last reviewed: September 9, 1997
Article ID: Q162619
The information in this article applies to:
  • Microsoft Internet Mail and News version 1.0 for Windows 95

SYMPTOMS

When you use Internet Mail to export messages from Internet Mail to Microsoft Exchange, the export function may seem to work, but the exported messages may not be present when you start Microsoft Exchange. When this occurs, no error message is generated.

CAUSE

This behavior can occur for either of the following reasons:

  • The export function in Internet Mail exports mail only to the Windows Messaging Update for Microsoft Exchange.
  • There is no set of personal folders configured in any profile in Microsoft Exchange.

RESOLUTION

Use the appropriate resolution:

  • Download and install the Windows Messaging Update for Microsoft Exchange. For information about obtaining the Windows Messaging Update for Microsoft Exchange, see the following article in the Microsoft Knowledge Base: 

          ARTICLE-ID: Q141127

          TITLE     : SFS Post Office Shared Folders Not Available with 
                  Microsoft Exchange


  • Configure a set of personal folders in a profile in Microsoft
   Exchange. To do so, follow these steps:

    
   1. Click Start, point to Settings, and then click Control Panel.

    
   2. Double-click Mail. (If the icon is labeled Mail And Fax, install the 

          update.)

    
   3. Click Add.

    
   4. Click Personal Folders, and then click OK.

    
   5. In the File Name box, type the path to your Personal Folders (.pst) 

          file. If a Personal Folders file does not already exist, type the  
      path and name of the Personal Folders file to create. For example, 
      to create a Personal Folders file called Test in the Windows folder, 
      type the following:

    

             c:\windows\test.pst

   6. Click Open.

   7. Configure your Personal Folders file as appropriate, click OK, and 
      then OK again.


STATUS

 


Microsoft has confirmed this to be a problem in Internet Mail. We are
researching this problem and will post new information here in the
Microsoft Knowledge Base as it becomes available.
