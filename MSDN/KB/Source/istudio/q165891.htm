PRB: Using OSR2 of Win95 Yields HTTP Error 500

 Last reviewed: December 9, 1997
Article ID: Q165891
The information in this article applies to:
  • Microsoft Visual InterDev, version 1.0

SYMPTOMS

Using Microsoft Windows 95 release OSR2 as a server with Personal Web Server that ships with OSR2, files larger than approximately 4K cannot be added or saved to a project. The error message encountered is: 

  "HTTP Error 500, cannot connect to server"
Examination of the PWS log SENDRECV.OUT shows an error code of 87, which means "invalid parameter."

CAUSE

The version of Personal Web Server installed with the Windows 95 OEM Service Release (build 1111) does not support the addition of files exceeding 4K in size to Visual InterDev projects.

RESOLUTION

Visual InterDev includes a more recent release of Personal Web Server that has no such limitation. Remove PWS from the Control Panel's Network Properties, restart the system as prompted, and install the version of PWS included with Visual InterDev.

REFERENCES

For the latest Knowledge Base articles and other support information on Visual InterDev and Active Server Pages, see the following page on the Microsoft Technical Support site:
