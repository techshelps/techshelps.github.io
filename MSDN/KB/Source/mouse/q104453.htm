Err Msg: "FILE ERROR: Cannot Find/Copy CTL3D.DLL" with Mouse 9

 Last reviewed: September 16, 1996
Article ID: Q104453
The information in this article applies to:
  • Microsoft Mouse driver for Windows, version 9.0

SYMPTOMS

After you install the Microsoft Mouse driver version 9.0 to a network installation of Windows, the following error occurs when you try to run the Mouse Manager: 

   FILE ERROR: Cannot find CTL3D.DLL

   Application Execution Error: Cannot find file MOUSEMGR.EXE (or one
   of its components). Check to ensure the path and filename are correct
   and that all required libraries are available.
Or, if you install the Microsoft Mouse driver version 9.0 to a network shared installation of Windows, you receive the following error: 

   CANNOT COPY FILE CTL3D.DLL

CAUSE

The file CTL3D.DLL is designed to install to the Windows SYSTEM subdirectory. When the 9.0 Setup program does not find a SYSTEM subdirectory under Windows, it does not install the file CTL3D.DLL, which is required to run Mouse Manager.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Mouse driver version 9.00. This problem was corrected in Microsoft Mouse driver version 9.01.

RESOLUTION

Copy CTL3D.DLL from the Mouse Setup disks to your Windows directory by using the following syntax: 

   copy a:ctl3d.dll c:\<your directory>\ctl3d.dll

   -or-

   If you cannot complete the mouse setup, do the following:

   1. Create a SYSTEM subdirectory in your local Windows directory.

   2. After a successful installation, copy the CTL3D.DLL from the SYSTEM
      subdirectory to the local Windows directory.

   3. Delete the CTL3D.DLL file from the SYSTEM subdirectory and then
      remove the SYSTEM subdirectory.
