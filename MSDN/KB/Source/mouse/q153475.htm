Msinput Folder Remains After Uninstall

 Last reviewed: September 12, 1996
Article ID: Q153475
The information in this article applies to:
  • Microsoft IntelliPoint software version 1.1
  • Microsoft IntelliType software version 1.1

SYMPTOMS

When you uninstall both IntelliType version 1.1 and IntelliPoint version 1.1, the Msinput folder is not removed.

CAUSE

The Msinput folder is not removed because it is not empty. The uninstall programs for IntelliType and IntelliPoint leave the Kbdctl32.dll file in the Msinput folder if both programs are installed in the Msinput folder. The Kbdctl32.dll file is common to both programs.

RESOLUTION

After both programs have been uninstalled, delete the Msinput folder using Windows Explorer.

MORE INFORMATION

This problem occurs regardless of the order in which the two programs are installed or uninstalled.

Note that the Kbdctl32.dll file and Msinput folder are properly removed by uninstall when only one of the two programs is installed.
