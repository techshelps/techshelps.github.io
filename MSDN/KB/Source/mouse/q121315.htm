IntelliPoint MS-DOS Mouse Driver Is Incompatible with LVMD.386

 Last reviewed: September 12, 1996
Article ID: Q121315
The information in this article applies to:
  • Microsoft IntelliPoint software version 1.0
  • Microsoft Natural Keyboard with IntelliType software version 1.0

SYMPTOMS

After you install the Microsoft IntelliPoint or IntelliType software and restart your computer, the computer stops responding (hangs) or the following error message is displayed when Windows is restarting: 

   Insufficient memory or address space to load Windows in 386 Enhanced
   mode.

   Quit one or more applications and try again.

CAUSE

This error occurs on systems that are loading the MS-DOS version of the IntelliPoint mouse driver (MOUSE.EXE) and using the Logitech virtual mouse driver (LVMD.386). The Logitech virtual mouse driver replaces the Mouse= line in the [386Enh] section of the SYSTEM.INI with LVMD.386. IntelliPoint does not modify this line.

Your computer hangs after you install IntelliType 1.0 because it is not compatible with LVMD.386.

RESOLUTION

To correct this problem, change the Mouse= line in the [386Enh] section of the SYSTEM.INI file to Mouse=*VMD.

MORE INFORMATION

Compaq and Acer systems ship with Logitech mouse driver.
