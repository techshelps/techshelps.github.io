FILE: Developer Java Classes for Microsoft VM for Java

 Last reviewed: January 29, 1998
Article ID: Q169800
The information in this article applies to:
  • Microsoft Win32 Virtual Machine for Java on the following platform: - x86
  • SDK for Java 2.0
  • Visual J++ 1.0, 1.1

SUMMARY

Classd.exe installs the developer classes for the Microsoft Virtual Machine for Java. Invoke the self-extracting executable to install the developer classes.

Note this version of the debug classes is intended for use with the 1518 build of the Microsft Virtual Machine for Java. If you require the developer classes for the lastest Java VM, click the download link from http://www.micrsoft.com/java Also note, the developer classes are also included in the SDK for Java under <sdk-installdir>\bin directory. They correspond to the version of the Java VM that is in the SDK.

These classes contain debug information in the class files and many of the Java source files for the system classes. A utility from Visual J++ and the SDK for Java 2.0, called Javasrc.exe, can be used to extract the Java source files onto the local drive for use with a Java debugger. If you do not have access to Visual J++ or the SDK, you can use WinZip or PKZip to extract the sources.
