INFO: Availability of Current Build of Microsoft VM for Java

 Last reviewed: January 29, 1998
Article ID: Q163637
The information in this article applies to:
  • Microsoft Visual J++, versions 1.0, 1.1 on the following platform: - x86
  • Microsoft Internet Explorer (Programming), versions 3.0, 3.01, 3.02, 4.0, 4.01
  • Microsoft Win32 Virtual Machine for Java on the following platform: - x86
  • Microsoft SDK for Java, versions 1.51, 2.01

SUMMARY

The current version (or build number) of the Microsoft Win32 Virtual Machine (VM) for Java is 2339. The Microsoft VM for Java is available at http://www.microsoft.com/java/.

Using Internet Explorer (IE) navigate to the page, then click Downloads, and click Microsoft Virtual Machine for Java. It is important that the VM is not currently in use when attempting to update. If you have run a Java applet in this session of IE, please exit IE and navigate directly to the page above. If you are using IE 4.x with active desktop enabled, you may need to restart your machine to unload the Java VM.

The download page causes IE to download the most recent version of the Microsoft VM for Java and install it on the client machine.

MORE INFORMATION

Typing 'jview' in a Command window will display the version of the currently installed Java VM.

If you suspect one or more of these files are out of date, simply navigate to the URL listed above and complete the setup.

If you receive an error during setup or the process does not solve the problems you are having with the VM, the next step is to backup the files in the list above and remove the originals from your system. Then, reboot and navigate back to the URL listed above.

If you have installed a newer version of the Microsoft Java VM, you may be prompted to decide whether to overwrite newer files in the Java VM. Answer 'Yes' to all dialogs to fully reinstall this retail version of the Java VM.
