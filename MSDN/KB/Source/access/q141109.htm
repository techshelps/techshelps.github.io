ACC: No Prompt to Save Changes When Closing Report in Preview

 Last reviewed: May 30, 1997
Article ID: Q141109
The information in this article applies to:
  • Microsoft Access versions 7.0, 97

SYMPTOMS

Novice: Requires knowledge of the user interface on single-user computers.

If you modify a Print Setup setting while you are previewing a report, Microsoft Access does not ask you to confirm the changes; the changes are automatically saved. This behavior differs from Microsoft Access version 2.0, which prompts you to save any changes you make while you are previewing a report.

STATUS

This behavior is by design.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Open the sample database Northwind.mdb.

  2. Click the Reports tab, select the Customer Labels report, and on the File menu, click Print Preview.

  3. On the File menu, click Page Setup.

  4. In the Page Setup dialog box, click the Page tab.

  5. Click to select the Use Specific Printer option, and then click OK. Note that you do not receive a message asking you to save the changes; the changes are saved automatically.
