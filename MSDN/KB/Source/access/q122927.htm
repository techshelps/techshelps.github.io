WX1124: Microsoft Access Version 2.0 Service Pack

 Last reviewed: October 21, 1997
Article ID: Q122927
The information in this article applies to:
  • Microsoft Access version 2.0

The Microsoft Access version 2.0 Service Pack contains files that replace existing files in your current Microsoft Access version 2.0 installation. The Service Pack includes the latest versions of:

  • The Microsoft Jet database engine (version 2.5).
  • The OLE dynamic-link libraries (version 2.02).
  • The Btrieve, Paradox, and xBASE installable ISAM drivers.

You can obtain this Application Note from the following sources (note that this file appears under two names, WX1124 and Accsvc.exe):
  • Microsoft's World Wide Web Site on the Internet
  • The Internet (Microsoft anonymous ftp server)
  • Microsoft Download Service (MSDL)
  • Microsoft Technical Support

For complete information, see the "To Obtain This Application Note" section at the end of this article.

TEXT OF WX1124

The Microsoft Jet database engine has been modified to enhance database repair functionality and to correct the erroneous setting of an internal database flag that in some cases results in the message "Couldn't open SYSTEM.MDA." The OLE dynamic-link libraries (DLLs) have been modified to allow the use of OLE custom controls created by third-party vendors and also to correct problems with diminished system resources when printing reports containing embedded Microsoft Graph objects. The installable ISAM drivers have been modified to correct various minor problems.

If you are using the ADT and you use templates with the Setup Wizard, you should review the following KB article before installing the latest version of the Setup Wizard: 

   ARTICLE-ID: Q124183
   Title:      ADT2: Setup Wizard Templates Lost Installing Service Pack.
If you will use the Service Pack with the Microsoft SQL Server ODBC driver and you are trying to attach long names with underscores, you might receive the error message, "Couldn't find object '<tablename>'." For more information, see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q124239
   TITLE:      ACC2: "Couldn't Find Object '<Name>'" with SQL Server Table.
