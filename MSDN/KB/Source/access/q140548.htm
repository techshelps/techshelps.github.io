ACC: List of ODBC Drivers Installed by Product

 Last reviewed: January 9, 1998
Article ID: Q140548
The information in this article applies to:
  • Microsoft Access versions 1.0, 1.1, 2.0, 7.0, 97

SUMMARY

Moderate: Requires basic macro, coding, and interoperability skills.

This article contains a listing of all the Open Database Connectivity (ODBC) drivers installed by Microsoft Access versions 1.0, 1.1, 2.0, 7.0, 97, the Microsoft ODBC 3.0 drivers for Windows 95 and Windows NT, the Microsoft ODBC Fulfillment Kit, the Microsoft Windows NT Desktop Driver Pack, the Microsoft Upsizing Tools, and the Microsoft Data Access Pack, Dataacc.exe, included with the Microsoft Access 97 ValuPack.

NOTE: The Data Access Pack is also available in Application Note WX1350. For more information about this Application Note, or to download WX1350.exe (which contains Dataacc.exe), please see the following article here in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q159674
   TITLE     : WX1350: ODBC Desktop Database Drivers 3.5 for 32-Bit
               Programs
This article is in two parts. First are diagrams of how ODBC interacts with other software in a networked PC environment. Second is a listing of all the ODBC drivers installed by the following products:
  • Microsoft Access versions 1.0, 1.1, 2.0, 7.0, and 97
  • Data Access Pack (Dataacc.exe)
  • Microsoft ODBC 3.0 Appnote (32-bit) drivers for Windows 95 and Windows NT
  • Microsoft ODBC Fulfillment Kit (Desktop Driver Pack 2.0)
  • Microsoft Windows NT 3.51 Desktop Driver Pack
  • Microsoft Upsizing Tools.

Installing support for ODBC installs the following DLLs into the WINDOWS/SYSTEM directory.
