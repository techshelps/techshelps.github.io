ACC2: Tabular, Groups/Totals Report Wizards Lose Sort Order

 Last reviewed: May 29, 1997
Article ID: Q123152
The information in this article applies to:
  • Microsoft Access version 2.0

SYMPTOMS

Moderate: Requires basic macro, coding, and interoperability skills.

When you create a report using the Tabular or Groups/Totals Report Wizard, and the report is based on a query containing a sorted field, the field is unsorted when you view the report.

RESOLUTION

Sort the field in the report in addition to, or instead of, sorting it in the query. To sort the field in the report, open the report in Design view, choose Sorting And Grouping from the View menu, select the field to sort, and select an ascending or descending sort order.

STATUS

This behavior no longer occurs in Microsoft Access version 7.0.

MORE INFORMATION

This behavior only occurs in reports created with the Tabular or Groups/ Totals Report Wizard. Reports created with the other Report Wizards do retain an underlying query's sort order.

Steps to Reproduce Behavior

  1. Open the sample database NWIND.MDB.

  2. Create a new query based on the Products table. Include all the table fields in the query, and sort the query on the Supplier ID field in ascending order. Save the query as Query1 and then close it.

  3. Use either the Tabular or the Groups/Totals Report Wizard to create a new report based on the Query1 query. Include all the fields in the report. Do not specify a sort order. If you are using the Groups/ Totals Report Wizard, do not specify a field to group by. Choose the Finish button to create the report.

    Note that the Supplier ID field is not sorted in ascending order.

REFERENCES

For more information about Report Wizards, search for "Report Wizards" then "Creating a Report with a Wizard" using the Microsoft Access Help menu.
