ACC97: Certain ISAM Drivers Not Appearing in Files Of Type Box

 Last reviewed: February 17, 1998
Article ID: Q171955
The information in this article applies to:
  • Microsoft Access 97

SYMPTOMS

Moderate: Requires basic macro, coding, and interoperability skills.

In Microsoft Access 97, on the File menu, you point to Get External Data, and then Import, but one or more of the following applications do not appear in the Files Of Type list box:

  • dBASE III
  • dBASE IV
  • dBASE V
  • Text Files
  • Microsoft Excel

However, if you rerun the Microsoft Office 97 or Microsoft Access 97 setup program, the ISAM drivers for the applications are checked as installed.

CAUSE

This behavior can be caused by invalid information in the Windows registry, or by a missing or corrupted dynamic-link library (.dll) file.

RESOLUTION

You must uninstall (remove) and then reinstall the ISAM drivers so that they can be reregistered properly. The following steps demonstrate how to remove and then reinstall the dBASE, Text, and Microsoft Excel drivers so that they will appear in the Files Of Type list box:

  1. Start the Microsoft Access 97 or the Microsoft Office 97 Professional Edition Setup program by double-clicking the Add/Remove Programs Icon in Control Panel.

  2. On the Install/Uninstall tab, select Microsoft Office 97, Professional Edition or Microsoft Access 97 in the list of applications, and then click Add/Remove.

  3. When prompted, place your compact disc into the CD-Rom drive and click OK.

    NOTE: If you are running setup from a network, you will not need to place the compact disc into the CD-ROM drive.

  4. When the first screen of the Setup program appears, click Add/Remove.

  5. In the Options box, select Data Access, and then click Change Option.

  6. In the Data Access Options box, click Database Drivers, and then click Change Option.

  7. Click to clear the check box for each ISAM driver that does not appear in the Files Of Type list box in Microsoft Access, click OK twice, and then click Continue.

    NOTE: If the check box for the ISAM driver is unavailable, click Cancel three times, click Exit Setup twice, click OK to exit the Setup Program, and proceed to Step 10.

  8. Click Yes when asked if you want to remove the components.

  9. After Setup has finished, click OK to the message that the Setup program has completed successfully.

  10. While the Add/Remove Program Properties dialog box is still open, start Windows Explorer (or File Manager) and search for the following file names: 

        File Name       Import/Export ISAM For
    --------------------------------------
    MSEXCL35.DLL    Microsoft Excel
    MSTEXT35.DLL    Text
    MSXBSE35.DLL    dBASE

  11. Rename the files. Close Windows Explorer.

  12. Repeat steps 2-6 from the beginning of this section.

  13. Click to select the check box for each ISAM driver that you want to reinstall.

  14. Click OK twice, and then click Continue.

  15. When the Setup program has finished updating your system, click OK. Close the Add/Remove Programs Properties box.

  16. Return to Microsoft Access and on the File menu, point to Get External Data, and then click Import. The additional ISAM drivers should now appear in the list.

If the application that you want to import from still does not appear in the list:

  1. Rename the files described above in steps 10-11.

  2. Repeat steps 1-3 from the beginning of the Resolution section.

  3. When the first screen of the Setup program appears, click Reinstall.

  4. When the Reinstall process has finished successfully, repeat step 16. 

        NOTE: If the application that you want to import from still does not
    appear in the list, you will need to uninstall Microsoft Access 97
    or Microsoft Office 97 Professional Edition and then reinstall the
    application. For information about how to successfully uninstall
    and reinstall Microsoft Access 97 or Microsoft Office 97 Professional
    Edition, please see the following articles in the Microsoft Knowledge
    Base:

        TITLE     : OFF97: How to Completely Remove Microsoft Office 97

    TITLE     : ACC97: How to Completely Uninstall Microsoft Access 97


MORE INFORMATION

 



Steps to Reproduce Behavior

 


  1. Start Microsoft Access 97 and open a new database.


  2. On the File menu, point to Get External Data, and then click Import.


  3. In the Import box, click the Files Of Type list box.

    
   The application that you want to import from does not appear in the
   list.


  4. Follow steps 2-6 in the "Resolution" section.

    
   Note that the ISAM drivers appear to be selected. In some cases, the
   word "INSTALLED" appears in the list where the file size normally
   appears.
