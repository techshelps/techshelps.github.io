ACC: Setting Height Property in Print Setup Is Ignored (1.x/2.0)

 Last reviewed: May 14, 1997
Article ID: Q143238
The information in this article applies to:
  • Microsoft Access versions 1.0, 1.1, 2.0

SYMPTOMS

Novice: Requires knowledge of the user interface on single-user computers.

When you set a form or report section's height in the Print Setup dialog box under Item Size of the More option, the height of the form or report section does not change. The Print Setup Height setting is ignored, and the section remains at the height specified in the form or report section's Height property setting.

RESOLUTION

  1. Open your form or report in Design view.

  2. Click the detail section and open the detail section's property sheet.

  3. Set the detail section's Height property to the number of inches or centimeters you want (the maximum setting is 22" or 55.873cm).

STATUS

Microsoft has confirmed this to be a problem in Microsoft Access versions 1.0, 1.1, and 2.0. Microsoft is researching this problem and will post new information here as it becomes available.

MORE INFORMATION

Steps to Reproduce Problem

  1. Open the sample database NWIND.MDB.

  2. Open the Customer Mailing Labels report in Preview.

  3. In the Country Filter dialog box, click OK. Note the current white space between the labels.

  4. On the File menu, click Print Setup.

  5. Click the "More >>" button.

  6. Under Item Size, click to clear the Same As Detail check box, set Height to 4 inches, and then click OK. Note that the preview of the report doesn't change.

  7. Close the Customer Mailing Labels report, and then re-open it in Design view.

  8. Click the detail section and open the detail section's property sheet. Set the detail section's Height property to 4 inches.

  9. Preview the report. Note the significant white space between labels.
