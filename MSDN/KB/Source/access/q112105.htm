ACC2: Microsoft Access and ORACLE ODBC Drivers

 Last reviewed: June 3, 1997
Article ID: Q112105
The information in this article applies to:
  • Microsoft Access version 2.0

SUMMARY

Novice: Requires knowledge of the user interface on single-user computers.

Microsoft Access version 2.0 contains an ODBC driver for connecting to SQL Server. However, Microsoft Access 2.0 does not contain the ORACLE 6 or ORACLE 7 ODBC driver because customer research showed that relatively few people used the ORACLE driver that was shipped with Microsoft Access version 1.1.

If you have the ORACLE ODBC driver supplied with Microsoft Access version 1.1, however, you can use it with Microsoft Access version 2.0. The ORACLE ODBC driver included with Microsoft Access version 1.1 supports ORACLE 6 functionality and ORACLE 6 functionality in ORACLE 7, but does not support full ORACLE 7 functionality.

If you want full ORACLE 7 functionality, you can obtain the ORACLE 7 driver by purchasing the Microsoft ODBC Drivers Pack 2.0 Fulfillment Kit. This kit contains 32-bit ODBC drivers for Microsoft SQL Server 4.2 and 16-bit drivers for ORACLE 6 and ORACLE 7.

You can call Microsoft Sales at (800) 426-9400 (or (425) 882-8080 from outside the United States) to order the Microsoft ODBC Drivers Pack 2.0 (part number 273-054-0300). The price is $10.25 (to cover the cost of materials and shipping) for any customer who owns one of the following Microsoft products:

  • Microsoft Word
  • Microsoft Excel
  • Microsoft Office or Office Professional
  • ODBC Drivers Fulfillment Kit version 1.0
  • Microsoft Access version 2.0
  • Microsoft FoxPro version 2.6
  • ODBC Desktop Drivers version 1.0
  • Microsoft Visual Basic version 3.0
  • Microsoft Visual C++ version 2.0 (or earlier)
