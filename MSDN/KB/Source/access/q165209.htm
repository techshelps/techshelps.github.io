ACC: Table Wizard Loses Selection When You Scroll Backward

 Last reviewed: April 1, 1997
Article ID: Q165209
The information in this article applies to:
  • Microsoft Access versions 7.0, 97

SYMPTOMS

Novice: Requires knowledge of the user interface on single-user computers.

When you create a table using the Table Wizard, and you select table properties or you create relationships for the table, the options you select are lost if you scroll back through the Table Wizard screens.

For example, if you select a sample table and some sample fields on the first page of the Table Wizard and you click Next and then click Back, the sample table and sample fields that you picked are no longer selected.

CAUSE

The Table Wizard does not remember the answers you provide to the following questions if you scroll back through the Table Wizard screens: 

   Which of the sample tables listed below do you want to use to create
   your table?

   Is your table related to any other tables in your database?

RESOLUTION

If you want to modify the relationships you have added to the Table Wizard or add new fields to the table, you should make those changes in Design view of the table when you have finished using the Table Wizard.

MORE INFORMATION

Steps to Reproduce Behavior

  1. Open the sample database Northwind.mdb in Microsoft Access 7.0 or 97.

  2. Create a new table using the Table Wizard.

  3. In the Sample Tables box, click Customers, in the Sample fields box, double-click CustomerID, and then click Next.

  4. Click Back to scroll back to the previous page, and then click Next to return to the Sample Tables screen. Note that the sample table and sample fields you picked are no longer selected.

REFERENCES

For more information about using the Table Wizard, search the Help Index for "Table Wizard," and then "Create a table using the Table Wizard," or ask the Microsoft Access 97 Office Assistant.
