Microsoft Excel 97 Causes a Windows NT Access Violation

 Last reviewed: December 29, 1997
Article ID: Q163891

The information in this article applies to:

  • Microsoft Windows NT Workstation version 4.0
  • Microsoft Windows NT Server version 4.0

SYMPTOMS

Microsoft Excel 97 may cause an access violation when you create OLE links with a Word document. You will receive the following error message: 

   Excel.exe - Application Error
   The instruction at "0x30053409" referenced memory at "0x06460504".
   The memory could not be "read'.
   Click ok to terminate the application.
document; mso97!MsoPeekMessage+0x10f2

RESOLUTION

To resolve this problem, obtain the hotfix mentioned below.

STATUS

Microsoft has confirmed this to be a problem in Windows NT version 4.0. This problem was corrected in the latest Microsoft Windows NT 4.0 U.S. Service Pack. For information on obtaining the service pack, query on the following word in the Microsoft Knowledge Base (without the spaces): 

   S E R V P A C K
