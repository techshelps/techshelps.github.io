BUG: Memory Error During Bulk Load Not Rollback Last Batch

 Last reviewed: April 29, 1997
Article ID: Q112699

The information in this article applies to:
  • Microsoft SQL Server Programmer's Toolkit, version 4.2
BUG# 9871 (4.2)

SYMPTOMS

If a client application encounters a DB-Library (DB-Lib) out of memory error, 

   OSERROR 0004 LocalAlloc Failed
while bulk loading data into SQL Server, rows from the last batch will be committed.

An out of memory error can occur when using DB-Library 4.20.50 and bulk loading character data into a table containing datetime fields. The expected behavior is that none of the rows in the failing batch will be loaded into the target table.

CAUSE

Memory allocation errors may be encountered when bulk loading data using DB-Library 4.20.50. Out of memory errors should not occur using other versions of DB-Library.

STATUS

Microsoft has confirmed this to be a problem in DB-Library version 4.20.50. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
