BUG: SQL Transfer Manager Error with DBO Login

 Last reviewed: April 30, 1997
Article ID: Q119802

The information in this article applies to: 

  - Microsoft SQL Server Programmer's Toolkit, version 4.21
BUG# 9978 (4.21)

SYMPTOMS

The SQL Transfer Manager utility refuses to complete a database transfer, even though you may have logged in as the database owner (dbo) for the destination database, with the error: 

   You must be 'sa' of destination server, or 'dbo' of
   destination database.

CAUSE

SQL Transfer Manager does not properly recognize the database owner login and the transfer will not proceed.

WORKAROUND

When using the SQL Transfer Manager utility, log into the destination database as the sa.

STATUS

Microsoft has confirmed this to be a problem with the SQL Server Transfer Manager utility version 4.21. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
