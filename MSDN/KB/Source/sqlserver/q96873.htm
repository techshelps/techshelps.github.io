PRB: "Login Incorrect" Error from Q+E under NetWare NIK

 Last reviewed: April 28, 1997
Article ID: Q96873

The information in this article applies to:
  • Microsoft SQL Server Programmer's Toolkit, version 4.2

SYMPTOMS

When using Q+E to log in to SQL Server through the Network Integration Kit for NetWare (NIK) under Windows, you may receive the following error message: 

   Login incorrect
But in fact, the log in ID and password are correct.

CAUSE

This log in problem is caused by using an incorrect version of DB-Library (DB-Lib) with Q+E. The NIK documentation states that it only supports 4.x DB-Lib clients. However, the DB-Lib file W3DBLIB.DLL supplied with Microsoft Excel 4.0a (or Q+E 3.0a) is version 1.x, and may be loaded when running Q+E, which will cause DB-Lib communication problems. Certain DB-Lib front-end applications such as Microsoft SQL Administrator for Windows can handle the Net Manager's error messages, and display the following error when this occurs: 

   Login failed. Network Manager only supports 4.x clients.

WORKAROUND

To work around this problem, search the Q+E or Excel directory for W3DBLIB.DLL, and replace it with the 4.2 version. Also, search the entire disk for other old DB-Lib files.
