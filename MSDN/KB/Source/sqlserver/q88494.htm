BUG: SYS2070 Error Using 4.20 BCP with Older DB-Library

 Last reviewed: April 28, 1997
Article ID: Q88494

The information in this article applies to:
  • Microsoft SQL Server version 4.2 for OS/2
BUG# OS/2: 7260 (4.2)

SYMPTOMS

When the SQL Server 4.2 bulk copy program (BCP) is used to copy data either in to or out of a table, OS/2 returns the following error: 

   SYS2070: The system could not demand load the application
   segment. PDBLIB is in error.

CAUSE

This error appears if the PDBLIB.DLL file from SQL Server version 1.1 or 1.11 is loaded instead of the one from version 4.2. The BCP included with version 4.2 of SQL Server does not function correctly with older versions of DB-Library (DB-Lib) and BCP incorrectly checks that the current PDBLIB.DLL is the proper version.

WORKAROUND

To resolve this problem, be sure to use the version of PDBLIB.DLL included with SQL Server 4.2. To verify that you are using the correct version, you can execute the PDBVER.EXE utility, which returns the db-lib version currently being used.

STATUS

Microsoft has confirmed this to be a problem in SQL Server version 4.2 for OS/2. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
