FIX: OPENDSNT.DLL Makes Thread AV Server-Wide

 Last reviewed: December 19, 1997
Article ID: Q132423
4.21a WINDOWS kbother kbbug4.21a kbfix4.21a.sp4

The information in this article applies to:

  • Microsoft SQL Server, version 4.21a

BUG# NT: 9175 (4.21a)

SYMPTOMS

The retail x86 4.21a Service Pack 2 and Service Pack 3 OPENDSNT.DLL may cause access violations that were previously limited in scope to a single thread to instead be server-wide.

WORKAROUND

Use the 4.21a OPENDSNT.DLL with SQLSERVR.EXE Service Pack 2 or Service Pack 3, or use SQL Server Service Pack 4, which does not experience this problem.

STATUS

Microsoft has confirmed this to be a problem in the OPENDSNT.DLL for SQL Server version 4.21a Service Packs 2 and 3 for x86 Windows NT. This problem was corrected in the latest U.S. Service Pack for SQL Server version 4.21a. For information on obtaining the Service Pack, please contact your primary support provider.
