FIX: SEM Can Take Excessive Time to Generate a Script

 Last reviewed: June 16, 1997
Article ID: Q136389

The information in this article applies to:
  • Microsoft SQL Server, version 6.0
BUG# NT: 11273 (6.00)

SYMPTOMS

When you generate a script from SQL Enterprise Manager(SEM) for a single object, it could take as long as generating a script for the entire database.

CAUSE

The script generation process in SEM only enumerates dependencies at the database object level no matter what is to be scripted.

STATUS

Microsoft has confirmed this to be a problem in Microsoft SQL Server version 6.0. This problem has been corrected in Microsoft SQL Server version 6.5. For more information, contact your primary support provider.
