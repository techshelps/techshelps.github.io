BUG: Cannot Use a Blank for Field Terminator with BCP

 Last reviewed: April 25, 1997
Article ID: Q70845

The information in this article applies to:
  • Microsoft SQL Server version 4.2 for OS/2
BUG# OS/2: 203 (4.2)

SYMPTOMS

The following error message is received when using the BCP utility: 

   dblib error: column number out of range

CAUSE

This error message occurs when using a blank for a field terminator within the format file.

WORKAROUND

Instead of a format file, use the -t option on the command line. For example: 

   bcp dbase..table out file.out -SServer -Usa -P -r\r\n -c -t"  "

STATUS

Microsoft has confirmed that this problem exists in SQL Server version 4.2 for OS/2. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
