BUG: ESQL-C: Precompiler Doesn't Generate Correct #Line Number

 Last reviewed: May 5, 1997
Article ID: Q152153
The information in this article applies to:
  • Microsoft Embedded SQL for C Programmer's Toolkit, version 4.22

SYMPTOMS

Nsqlprep.exe does not generate correct #line numbers in the resulting C file. This makes debugging the application difficult because setting break points and stepping in a debugger do not work properly.

When debugging and setting break points, you may encounter messages indicating that there is not executable code at the break point specified. If the file is small, stepping in the debugger may place you at the end of the file.

STATUS

Microsoft has confirmed this to be a problem in Microsoft Embedded SQL for C Programmer's Toolkit version 4.22. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
