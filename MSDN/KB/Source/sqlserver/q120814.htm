INF: Using TCP/IP Socket Net-Library in Windows for Workgroups

 Last reviewed: April 30, 1997
Article ID: Q120814

The information in this article applies to: 

  - Microsoft SQL Server version 4.2

SUMMARY

To connect to Microsoft SQL Server using TCP/IP sockets in Microsoft Windows for Workgroups version 3.11 with the 32-bit protect mode TCP/IP stack (TCP/IP-32), you must upgrade to the latest TCP/IP Sockets Network Library (Net-Library) DBMSSOC3.DLL.

MORE INFORMATION

This version of Net-Library is included with Microsoft SQL Server version 4.21a. You can use the Client Configuration Utility to verify its version (4.21.05), date (7/15/94), and size (9520 bytes).

This new version of DBMSSOC3.DLL is also supported on both WFW TCP/IP 1.0 and the WOW subsystems of Microsoft Windows NT versions 3.1 and 3.5. Please see the SQL Server readme file for more details.
