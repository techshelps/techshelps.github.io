BUG: Default Query Timeout for Replication Tasks Incorrect

 Last reviewed: April 9, 1997
Article ID: Q155582

The information in this article applies to:
  • Microsoft SQL Server, versions 6.0 and 6.5

SYMPTOMS

The SQL Server 6.0 and 6.5 documentation states that the default query timeout of replication tasks (for both logreader and distribution) is 30 seconds. This is incorrect. All replication tasks (logreader and distribution) have a default query timeout of 0 (no timeout) in Microsoft SQL Server 6.0. In Microsoft SQL Server 6.5, the logreader does not have any default timeout. The distribution process has a default timeout of 30 seconds.

STATUS

Microsoft has confirmed this to be a problem in Microsoft SQL Server versions 6.0 and 6.5. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.
