Changes to Store Pipeline Configuration File Do Not Take Effect

 Last reviewed: July 16, 1997
Article ID: Q158797
The information in this article applies to:
  • Microsoft Commerce Server, version 2.0

If you make changes to your store's pipeline configuration (.pcf) file, but do not see the changes take effect, save the Global.asa file of your store. Doing this causes the new pipeline configuration to be loaded. You can also close and open the store using the Host Administrator to force the changes to take effect. You do not, however, need to stop and restart Internet Information Server (IIS).
