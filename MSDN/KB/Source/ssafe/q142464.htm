FIX: "Too Many File Handles" in VB Enterprise SourceSafe

 Last reviewed: August 25, 1997
Article ID: Q142464 
4.00    | 4.00
WINDOWS | WINDOWS NT kbusage kbfile kbbuglist kberrmsg kbfixlist kbsample

The information in this article applies to:

  • Microsoft Visual SourceSafe version 4.0
  • Enterprise Edition of Microsoft Visual Basic for Windows, 16-bit only, version 4.0

SYMPTOMS

You get a "Too Many File Handles" error while trying to use the Visual SourceSafe that ships with Visual Basic Enterprise Edition.

WORKAROUND

The 0831 version of SSVB16.DLL and SSSCC16.DLL will fix this problem. These files have been placed in a self-extracting file that you can download.

The following file is available for download from the Microsoft Software Library: 

 ~ Vss0831.Exe (size: 514595 bytes)
For more information about downloading files from the Microsoft Software Library, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q119591
   TITLE     : How to Obtain Microsoft Support Files from Online Services
After downloading Vss0831.Exe, run it in an empty directory to extract the two .DLL files. Copy SSVB16.DLL and SSSCC16.DLL into the SourceSafe (client version) WIN directory (for example, C:\VSS\WIN).

STATUS

Microsoft has confirmed this to be a problem in Microsoft Visual Basic Enterprise Edition's Visual SourceSafe version 4.00 for Windows (release 0811). This problem was corrected in Visual SourceSafe (standalone) version 4.00 for Windows (release 0831). The problem was discovered after Microsoft shipped Visual Basic Enterprise 4.00. It is fixed in the Visual SourceSafe 4.0 stand alone product.
