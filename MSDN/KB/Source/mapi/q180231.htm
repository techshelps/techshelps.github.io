PRB: ITnef::ExtractProps Does Not Extract All Properties

 Last reviewed: February 3, 1998
Article ID: Q180231
The information in this article applies to:
  • Extended Messaging Application Programming Interface (MAPI), version 1.0
  • Exchange Development Kit (EDK), version 5.0

SYMPTOMS

When the PR_DISPLAY_TO of a Transport-Neutral Encapsulation Format (TNEF) message contains more than 1024 bytes, and the TNEF message becomes an attachment of another message, ITnef::ExtractProps() does not process all properties of the embedded (attached) message. For instance, the PR_BODY property of the attached message is missing.

STATUS

Microsoft is researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

MORE INFORMATION

The TNEF process is used to encapsulate Extended Messaging Application Programming Interface (MAPI) properties for messages sent to systems that are not MAPI based.

Gateways and transport providers use the ITnef::ExtractProps() method to decode properties from the encapsulation of a message and attachment that was encapsulated through OpenTnefStream.

REFERENCES

For more information, please see the following topics in the Microsoft Developer Network Library:

  • ITnef::AddProps
  • ITnef::SetProps
  • OpenTnefStream
  • OpenTnefStreamEx
  • STnefProblemArray
