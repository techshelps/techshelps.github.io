PRB: Client Extensions Do Not Display Changes to Messages

 Last reviewed: January 30, 1998
Article ID: Q180174
The information in this article applies to:
  • Exchange Development Kit (EDK), versions 4.0, 5.0
  • Microsoft Exchange Windows NT client, versions 4.0, 5.0
  • Microsoft Outlook, versions 8.00, 8.01, 8.02, 8.03

SUMMARY

Client extensions may make changes to the message currently displayed. Once changes are made, you may want to refresh the window to display the new information.

There is no process or technique that tells the client to refresh the screen to display newly added or changed properties. So, although it is possible for the client extension to add information to the message, new properties and/or new attachments, it is not possible for the view of the standard IPM.Note form to be refreshed to reflect the changes. The Microsoft Exchange and Microsoft Outlook clients do not expose this functionality for programmatic control.
