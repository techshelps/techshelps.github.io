PRB: Application Hangs Calling Fetch of VB5 MAPIMessage Control

 Last reviewed: November 14, 1997
Article ID: Q176484
The information in this article applies to:
  • Microsoft Visual Basic Professional and Enterprise Editions for Windows, version 5.0

SYMPTOMS

When a Visual Basic 5.0 application using MSMAPI32.OCX version 5.00.3721 or 5.01.4319 is installed on a Windows 95 system, the application hangs when it uses the Fetch method of MAPIMessage (if there are any messages for MAPIMessage to find).

This would presumably occur with later versions of this control as well.

CAUSE

The version of the MAPI .dll's installed are too out of date for this control to function successfully.

RESOLUTION

Download and install the Windows Messaging System (WMS) from the following site: 

   ftp://ftp.microsoft.com/Softlib/MSLFILES/
Exupdusa.exe is a self-extracting Windows 95 specific .exe file that installs WMS on a Windows 95 system.

You may also install either Microsoft Exchange 5.0 or Microsoft Outlook client and the installation updates the underlying MAPI .dll's.

STATUS

This behavior is by design.
