PRB: Error 80010010 in the NT Application Log on DAPIStart

 Last reviewed: June 5, 1997
Article ID: Q169552
The information in this article applies to:
  • Exchange Development Kit (EDK)
  • Microsoft Exchange Server, versions 4.0, 5.0

SYMPTOMS

When using DAPI on a machine that has not had the Exchange Admin program installed on it, the error 80010010 may occur. The error is written to the NT Application Log and is generated on the DAPIStart method.

CAUSE

The error occurs because there are components that are required to run DAPI and one of these is missing.

RESOLUTION

The items that are required to run DAPI must be set up on the machine. For more information on the required components for DAPI, please refer to the following article:

For additional information, please see the following article in the Microsoft Knowledge Base: 

   ARTICLE-ID: Q169551
   TITLE     : INFO: Items Required to Use DAPI

STATUS

This behavior is by design.

MORE INFORMATION

DAPI is a DAPI supplied with Exchange that allows reading and writing to the Exchange Directory. It is typically used to create and delete mailboxes and for directory synchronization.
