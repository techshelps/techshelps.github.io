PRB: Win95 Hangs when Using ADO with Active Server Pages

 Last reviewed: February 19, 1998
Article ID: Q164479
The information in this article applies to:
  • Microsoft Active Server Pages, version 1.0
  • Microsoft ActiveX Data Objects

SYMPTOMS

ActiveX Data Objects (ADO) cause Windows 95 to stop responding with Personal Web Server (PWS) under heavy loading conditions.

CAUSE

This behavior is caused by a problem in the Windows 95 operating system, which may present itself when multiple requests are made simultaneously to ADO.

RESOLUTION

Currently the workarounds are as follows:

  • Use ADO on Windows 95 only under light loads.

    -or-

  • Run Active Server Pages on Windows NT when using ADO.

STATUS

Microsoft has confirmed this to be a problem in the Microsoft products listed at the beginning of this article. We are researching this problem and will post new information here in the Microsoft Knowledge Base as it becomes available.

REFERENCES

For the latest Knowledge Base articles and other support information on Visual InterDev and Active Server Pages, see the following page on the Microsoft Technical Support site:
