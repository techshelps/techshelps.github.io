Package and Deployment Wizard — DAO Drivers

The Package and Deployment Wizard has determined that you are using Data Access Objects (DAO) and lets you select the driver(s) required by your application.

Note   When you add a data control to a form or other object in your project, Visual Basic adds a reference to Data Access Objects (DAO). If you later remove the data control, Visual Basic does not remove the reference to DAO. The Package and Deployment Wizard recognizes the reference whether or not you use data controls in your application. If your project used to contain data controls but no longer uses DAO, be sure to remove the reference to DAO.

Options

Available Drivers

Lists available drivers you can include. When you include a driver, a special file(s) is added to your distribution list to support the driver.

Included Drivers

Lists the drivers you are including in your package. You can use the buttons to include or remove driver(s) from this list.

Moves the selected driver(s) in the Available Drivers list to the Included Drivers list.

Moves all of the drivers in the Available Drivers list to the Included Drivers list.

Moves all of the drivers in the Included Drivers list to the Available Drivers list.

Moves the selected driver(s) in the Included Drivers list to Available Drivers list.

Help

Displays the help topic for this step. You can also press F1 for help.

Cancel

Cancels your previous actions and closes the packaging portion of the Package and Deployment Wizard.

Back

Moves you to the previous step.

Next

Moves you to the next step.

Finish

Creates a package using the settings you selected and displays a report at the end of the process. Available only when you have provided enough information for the Package and Deployment Wizard to successfully create a package.

If you chose a pre-existing script, the Finish button is available because the Package and Deployment Wizard assumes that the script contains all of the information needed to create the package.
