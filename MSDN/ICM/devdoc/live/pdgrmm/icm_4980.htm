Installing ICM 2.0

[This is preliminary documentation and subject to change.]

Note  ICM 2.0 is not available under Windows 95 or Windows NT 4.0.

In order to run programs that use ICM 2.0, you must have the correct version of Windows running on your system. See ICM 2.0 Availability for details.

Downloading and Installing ICM 2.0

The ICM 2.0 beta documentation can be downloaded from the following World Wide Web site on the Internet:

http://www.microsoft.com/msdn/sdk/icm20.htm

From this site, you can download Wicm20.exe which is a self-extracting installation file. After you have downloaded this file to your local development system, you can install the Programmer's Reference by running it as a program.

Your test machine must be running the beta 1 release of either Windows 98 or Windows NT 5.0. Copy the updated ICM 2.0 dynamic-link libraries to the test machine after you have installed the Programmer's Reference on your development system, using the following procedure.

Updating your Windows Beta Machine

Note  This Programmer's Reference requires the beta 1 release of Windows 98 or the beta 1 release of Windows NT 5.0.

    To Update Dynamic-Link Libraries on your Windows Beta Machine
  1. Copy the following file from the \Lib folder of the ICM 2.0 Programmer's Reference into the Windows system folder on the beta machine:
    • Icmui.dll
  2. Copy the file Icm.inf from the ICM 2.0 Programmer's Reference root folder to the desktop or to a temporary folder on your beta machine.
  3. Right click on the Icm.inf file and select Install from the menu that appears. This updates the registry on your beta machine to take advantage of the new DLL files.
