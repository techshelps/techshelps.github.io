Code Generation Wizard: Step 3b – Specify Base Class Substitutes

See Also

Allows you to substitute user-defined classes to use instead of the modeler-specified mappings to Visual FoxPro classes.

To specify a substitute class to use

  1. Right-click the class you want to remap.

  2. Click Select a class.

  3. Locate the library file that contains the class and select the class to substitute.

You can click Reset to Default to restore the original mapping.

You can also load or save class mappings by clicking one of the profile buttons and specifying the file name of the profile. Modeler profiles are saved as records in the Foxuser.dbf resource file. You can use saved profiles in other models.
