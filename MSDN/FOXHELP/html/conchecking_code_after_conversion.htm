Checking Code After Conversion

See Also

When you convert projects or files, check .PRG files for code you can modify to take full advantage of the Visual FoxPro event model. You might relocate code or procedures to events or methods in containers or objects, or use new Visual FoxPro commands or functions.

From within the Project Manager, you can check all project files for the following situations:

  • Check for UDFs or other code that duplicates new Visual FoxPro functions. For more information, see Differences Between Visual FoxPro and Previous Versions.

  • Check .PRG files for @ ... GET code that creates overlapping controls. You might need to reorder your @ ... GET code and change the ZOrder Method of controls to maintain FoxPro 2.6 functionality.

    Tip To access properties of controls in .PRG files, use the _SCREEN system variable, and the ActiveForm, and ActiveControl properties.

  • Look for symbol name conflicts with new reserved words.

  • Check for conflicting configurable key combinations and navigation keys. For more information, see Keystroke Differences in Interface Differences.

  • Change CLEAR READ commands in forms or form sets to RELEASE THISFORMSET or RELEASE THISFORM, or to CLEAR EVENTS.

  • Change SHOW GET and SHOW GETS statements to object.Refresh statements.

  • Change SHOW GET and SHOW GETS statements in list and combo boxes to object.Requery statements.

  • Move Foundation READs into the Visual FoxPro event model by using READ EVENTS and CLEAR EVENTS commands.
