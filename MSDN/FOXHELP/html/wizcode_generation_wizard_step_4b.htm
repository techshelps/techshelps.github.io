Code Generation Wizard: Step 4b – Resolve Inheritance Conflicts

See Also

Allows you to reconcile conflicts raised in Visual FoxPro by multiple inheritance in the imported model, which is not supported in Visual FoxPro. Resolve instances of multiple inheritance to a single parent class.

If you encounter inheritance conflicts here, you should stop the wizard and first resolve these conflicts in the Visual Modeler tool, then restart the Code Generation Wizard. You can use the information displayed here to resolve the conflicts using Visual Modeler. The Code Generation Wizard cannot update the original model from inheritance changes you make here.

To resolve inheritance conflicts

  1. Highlight the class containing the conflict.

  2. Uncheck the parent class or classes you want to exclude from the model.

You cannot import the model without resolving inheritance conflicts.
