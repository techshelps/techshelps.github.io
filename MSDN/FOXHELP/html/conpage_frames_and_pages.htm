Page Frames and Pages

See Also

In FoxPro 2.6 READ compatibility mode, each form set contains at least one page frame that contains one page for each read level of the original screen. Each page contains all the controls that participate in that READ level. Visual FoxPro destroys the participating controls and the page whenever a READ terminates.

Visual FoxPro creates a page frame, if one does not already exist, and a page to handle the READ level for any @ ... GET commands in a FoxPro 2.6 program. The page and its controls become active at the READ and terminate when the READ terminates.
