Sample Transport Provider

This sample uses files and directories to transmit and receive messages. It implements and registers a very simple preprocessor that appends a line of text to each outbound message. It illustrates how to split message content between TNEF and text. It also supports all configuration options (property sheets, wizard, and programmatic configuration) and message options. It does not support the remote transport interfaces.

Quick Info

Executable: SMPXP.DLL
Directory for source code: PEER.XP
Written in: C
Supports: All platforms

Supported Features

  • Basic features:

    Sending, receiving and polling for new messages

  • Interactive and programmatic configuration
  • IMAPIStatus interface except property setting
  • Thread safety
  • Event logging to a text file

    The file is automatically limited to a specified size. All transport sessions use the same file.

Unsupported Features

  • Asynchronous detection of incoming messages
