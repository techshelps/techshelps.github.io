Mapping the STATUS Clause

[This is preliminary documentation and subject to change.]

Each SNMPv1 and SNMPv2C MIB object, whether in a scalar or table collection, contains a STATUS clause. Where the STATUS clause defines an "obsolete" value, the MIB object is discarded from the mapping. Otherwise, the associated CIM property is augmented with a status qualifier using the property qualifier status.

For SNMPv1, it is preferable that the value of the status property qualifier be either "mandatory" or "optional" but it is not considered an error for this qualifier to contain some other value.

For SNMPv2C, it is preferable that the value of the status property qualifier be either "current" or "deprecated" but it is not considered an error for this qualifier to contain some other value.
