The SNMP provider returns an empty result set for a given instance because the SNMP agent is returning key information that is inconsistent with the CIM MOF definition.

[This is preliminary documentation and subject to change.]

Further description:

The SNMP provider uses a GET-NEXT operation to retrieve a particular instance of a class from a device. The device returns property information that is inconsistent with the MOF definition of the instance being retrieved. The SNMP providers drop these erroneous instances.

Solutions:

  • Check that the MOF class definition is consistent with the SNMP MIB definition used to derive it.
  • Check with the SNMP agent vendor to determine the expected typing for these properties.
