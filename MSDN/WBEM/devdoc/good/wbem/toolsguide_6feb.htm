Adding Users

[This is preliminary documentation and subject to change.]

To add a user
  1. Click the Add User button.

    —or—

    Select Add New User from the User menu. The User Properties dialog box appears:

  1. Type the User Name and the Domain name where the user's account resides. User names must be unique and cannot be shared with group names. If you do not specify a domain name, the current domain is the default name.
  2. Under Attributes, select all applicable access privileges for the user:

    Enabled

    Accounts are enabled by default. When this option is not selected, the account is disabled. A login is not permitted against a disabled account.

    Execute Methods

    Allows the user to execute object methods exported from the CIM Object Manager.

    Schema Access Level

    Select one of the following access levels:

    Read Only

    Allows read-only access to CIM schema objects. Users with this privilege can execute queries or retrieve instances and classes, but cannot create, delete, or modify CIM objects.

    Write Instance

    Allows full read/write/delete access to instances. Users with this privilege have read-only access to classes.

    Write Class

    Allows full read/write/delete access to all CIM objects, classes, and instances.

  3. Under Security, select Edit Security if you want to give the user read/write access to the ROOT\SECURITY namespace. Users with this privilege can edit security information. Users without this privilege have no access to ROOT\SECURITY.
  4. Click OK. This returns you to the WBEM User Manager window. The new user appears in the list of users in the top portion of the window.
  5. Select Save changes from the User menu to save the changes and remain in the WBEM User Manager.

    —or—

    Select Exit with changes from the User menu to save the changes and close the WBEM User Manager.

    To close the WBEM User Manager without saving the changes, select Exit without changes from the User menu.
