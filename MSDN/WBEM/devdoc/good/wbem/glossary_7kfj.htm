O

[This is preliminary documentation and subject to change.]

object
A unit of data that is an instance of a class; a set of property values that can be uniquely identified by a key.
object path
A formatted string used to access namespaces, classes, and instances. Each object on the system has a unique path which identifies it locally or over the network. Object paths are conceptually similar to Universal Resource Locators (URL).
object provider
A COM-based component that communicates directly with managed objects and forwards the information that it receives to CIMOM for integration and interpretation. Also see class provider, instance provider, and property provider.
operation
The name, return type, and parameters supported by a method.
optional property
A property that need not have a value.
