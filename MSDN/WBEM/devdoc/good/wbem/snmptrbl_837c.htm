The set of classes returned by the SNMP providers changes intermittently.

[This is preliminary documentation and subject to change.]

Further description:

The SNMP class provider uses a correlation procedure to determine the set of class definitions supported by an SNMP device. The correlation process may return different results depending on the state of the network nodes connecting the SNMP providers and the SNMP agent. If additional MIBs are compiled into the SMIR, the correlation process may return additional classes that have not been available previously from the SMIR.

Solutions:

See the following topics:

  • The SNMP protocol data units generate ICMP-host-unreachable packets.
  • The SNMP protocol data units generate ICMP-net-unreachable packets.
  • The SNMP agent receives an SNMP protocol data unit but does not return a response.
  • The SNMP agent does not respond to an SNMP protocol operation because the SNMP operation protocol data units are being dropped before reaching their destination.
  • The SNMP agent does not respond to an SNMP protocol operation because the SNMP response protocol data units are being dropped before reaching their destination.
  • The SNMP agent does not respond to an SNMP protocol operation because the SNMP agent is dropping the SNMP protocol data units.
