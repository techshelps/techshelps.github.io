The SNMP agent does not respond to an SNMP protocol operation because the SNMP response protocol data units are being dropped before reaching their destination.

[This is preliminary documentation and subject to change.]

Further description:

Some entity between the destination and source is dropping packets due to network saturation.

Solutions:

  • Increase the value of the AgentRetryCount qualifier of the SNMP device namespace instance or SNMP-compatible class.
  • Decrease the value of the AgentFlowControlWindowSize qualifier of the SNMP device namespace instance or SNMP-compatible class. This reduces the number of outstanding protocol data units sent to that device and, therefore, the number of responses being returned.
