Transport Protocol Operation

[This is preliminary documentation and subject to change.]

The following topics describe errors associated with the transport protocol operation.

  • SNMP device namespace instance or SNMP-compatible class contains an illegal transport address.
  • The AgentAddress qualifier of the SNMP device namespace instance or SNMP-compatible class is correct but the operation hangs for a period of time before successfully returning.
  • The AgentAddress qualifier of the SNMP device namespace instance or SNMP-compatible class is a resolvable host name but the operation hangs for a period of time before unsuccessfully returning.
  • The AgentAddress qualifier of the SNMP device namespace instance or SNMP-compatible class is an IP address but the operation hangs for a period of time before unsuccessfully returning.
  • The SNMP protocol data units generate ICMP-port-unreachable packets.
  • The SNMP protocol data units generate ICMP-host-unreachable packets.
  • The SNMP protocol data units generate ICMP-net-unreachable packets.
  • The SNMP agent receives an SNMP protocol data unit but does not return a response.
  • The SNMP agent does not respond to an SNMP protocol operation because the SNMP operation protocol data units are being dropped before reaching their destination.
  • The SNMP agent does not respond to an SNMP protocol operation because the SNMP response protocol data units are being dropped before reaching their destination.
  • The SNMP agent does not respond to an SNMP protocol operation because the SNMP agent is dropping the SNMP protocol data units.
  • The SNMP agent responds to an SNMP protocol operation after the SNMP provider times out of the protocol operation response.
  • The SNMP provider sends out multiple packets with the same protocol request identifier even when the agent is responding to the SNMP protocol operation.
  • The SNMP provider returns a subset of the properties present for a given class definition because some entity is dropping a percentage of SNMP packets between the source and destination.
  • The SNMP provider returns a subset of the instances expected for a class definition because the SNMP agent device is returning GET-NEXT variable bindings that are not lexicographically ordered.
  • The SNMP provider returns a subset of the properties present for a class definition because the SNMP agent does not return properties that are defined as optional.
  • The SNMP provider returns a subset of the properties present for a class definition because the SNMP agent experiences some internal error processing those properties.
  • The SNMP provider returns a subset of the properties present within an instance of a class because the SNMP agent is returning key information that is incompatible with the CIM MOF definition.
  • The SNMP provider returns an empty result set for a given instance because the SNMP agent is returning key information that is inconsistent with the CIM MOF definition.
  • The SNMP provider returns an empty result set for a given instance because the SNMP agent is expecting key information that is inconsistent with the CIM MOF definition.
  • The SNMP provider returns an error result because the qualifier values of the SNMP device namespace instance or SNMP-compatible class do not match the expected type or values.
  • The set of classes returned by the SNMP providers changes intermittently.
  • The set of instances returned by the SNMP providers changes intermittently.
