SNMP device namespace instance or SNMP-compatible class contains an illegal transport address.

[This is preliminary documentation and subject to change.]

Solutions:

  • Verify that the AgentAddress qualifier is of type string.
  • If the AgentAddress qualifier is a dotted decimal Internet Protocol (IP) address, then the address must be a valid unicast IP address.
  • If the AgentAddress qualifier is a host-name-resolvable string, verify that the host name is resolvable using the Windows executable file:

    ping.exe <hostname>

    where hostname is the host-name-resolvable string.
