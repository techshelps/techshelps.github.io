Mapping Subtype Enumeration Definitions

[This is preliminary documentation and subject to change.]

The following table defines the mapping where the SYNTAX clause of the SNMPv1 and SNMPv2C OBJECT-TYPE macro refers to an SNMP primitive type, named type, or textual convention containing a subtype definition of the type enumerated INTEGER.

Table 4: Mapping SNMPv1 and SNMPv2C Subtype Enumeration Definitions

Constrained MIB type Variant type Textual convention qualifier Encoding qualifier Cimtype qualifier Default value
Enumerated Integer VT_BSTR enumeratedinteger INTEGER string Named value of first element of enumeration

The CIM named qualifier textual_convention defines the textual representation of the SYNTAX clause of the OBJECT-TYPE macro.

The CIM named qualifier object_syntax defines the named type definition present within the SYNTAX clause of the OBJECT-TYPE macro. The object_syntax is copied from the SYNTAX clause of the OBJECT-TYPE macro up to but not including the enumerated value assignments, if present.

The CIM named qualifier encoding defines the SNMP type used when encoding SNMPv1 and SNMPv2C protocol frames.

The CIM property qualifier enumeration defines the enumerated values. The enumeration qualifier is represented as a string containing a comma-separated list of signed 32-bit integer values.

For example:

Status := INTEGER {

up(1),
down(2),
testing(3)
}

This example maps to:

enumeration("up(1),down(2),testing(3)")
