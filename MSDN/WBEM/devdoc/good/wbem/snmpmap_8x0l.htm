Mapping the SNMPv2C AUGMENTS Clause

[This is preliminary documentation and subject to change.]

In SNMPv2C it is possible to replace the use of the INDEX clause with the AUGMENTS clause. The AUGMENTS clause indicates that the following table collection can be considered an extension of a referenced table collection. Where an AUGMENTS clause is specified, the set of collections referred to by that AUGMENTS clause can be combined with the referencing table collection to form one collection. The resultant collection shares the primary key properties specified in the last table collection in that chain. The previous mapping rules for the INDEX clause should be applied to the last table collection in that chain. The collection of objects then defines one CIM class definition.
