Registering Users for Your FrontPage Web

You can create a form that registers users for your FrontPage web. When a user visits a FrontPage web that requires registered users, the Web server prompts the user for a name and password before it releases the FrontPage web for display in a Web browser.

The FrontPage form handler that adds a user and password to the list of users with permission to access a FrontPage web is the Registration Form Handler. You must create forms handled by Registration Form Handlers in the root FrontPage web (the FrontPage web named “<Root Web>” by default).

A user registers for access to a FrontPage web by supplying a name and password to a form handled by a Registration Form Handler. Along with adding the user and password to the list of users with permission to use the FrontPage web, Registration Form Handlers can save other information from the form in the same way as the default form handler.

By default, all FrontPage webs on a Web server have the same permissions for administrators, authors, and users as the root FrontPage web. When you create a FrontPage web that will require users to register before using it, you must open the FrontPage web and specify that it does not inherit its permissions from the root web, but instead has its own set of permissions. You can do this by opening the FrontPage web for which you are registering users and choosing the Permissions command in the Tools menu of the FrontPage Explorer. In the Settings tab, click Use unique permissions for this web and then click the Apply button. Next, in the Users tab, click Only Registered Users Have Browse Access and click the Apply button.

Once you have configured your FrontPage web to accept only registered users, you can add the user registration form to any page in your root web. (If you do not add the form on the home page of the root web, make sure you have a hyperlink to it from the home page, so users can find it.) Along with Push Button fields for submitting and resetting the form, a registration form should have the following fields:

  • a one-line text box field for entering a user’s name

  • a one-line text box field for entering a user’s password

The easiest way to add a user-registration form to the root web is to create a new page and select the User Registration template. This template contains a complete user registration form along with explanatory text that you can modify. Once you have created a user registration page, you must open the page, open the Form Properties dialog box, click the Options button, and specify the FrontPage web for which you are registering users. In the FrontPage web name field, type in the name of the FrontPage web that requires registered users, and click OK twice to close all dialog boxes.
