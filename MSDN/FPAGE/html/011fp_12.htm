Processing Forms

A form is processed by a form handler that is associated with the form. The way a form handler processes a form depends on the form handler’s design. Some form handlers enter form field data into a database. Others do some processing and generate HTML that the Web browser displays. For example, a form that is a mathematical calculator could accept numbers from a user and send them to a form handler which calculates a result, formats it in HTML, and sends it back to a Web browser for display.

FrontPage includes a set of form handlers that process forms for most common uses. You can use FrontPage form handlers to collect information from your form and store it in a variety of formats, register users for your FrontPage web, or support a discussion group on your web. Advanced users can create their own form handlers and assign them to forms in FrontPage.

When you create a form, FrontPage assigns its default form handler to your form. This general purpose form handler saves the form’s information to a file in your FrontPage web. See “Processing Forms”, for details.

FrontPage also includes a set of advanced FrontPage form handlers. You select one of these form handlers using the Form Properties dialog box. Along with the default form handler, you can assign the following FrontPage form handlers to a form:

  • Registration Form Handler, to register users so that they can access a service implemented as a Web site.

  • Discussion Form Handler, to support participation in an online discussion.

Note that FrontPage automatically saves form results only on Web servers containing the FrontPage Server Extensions. These extensions are available for most popular Web servers at http://www.microsoft.com.

If you do not want to use one of the FrontPage form handlers, you can assign a custom form handler that you have created. FrontPage supports custom form handlers that have been created using one of the following methods:

  • Internet Server Application Program Interface (ISAPI)

  • Netscape Server Application Program Interface (NSAPI)

  • Common Gateway Interface (CGI)

  • Active Server Page (ASP)

You use the Form Properties dialog box to change the settings of your current form handler or to change form handlers. To open this dialog box, right-click anywhere in the form and choose Form Properties from the shortcut menu.

When Send to is selected in the Form Properties dialog box, the default form handler is in use and the results file, which is the file that will contain the saved form information, is displayed in the File Name field. To send form information in e-mail, type the e-mail address in the E-mail Address field. To select an advanced form handler, click Send to other and choose the form handler from the drop-down list.

To configure the form handler, click Options. This opens the configuration dialog box for the form handler you selected. The default form handler’s configuration dialog box is called Options for Saving Results of Form.
