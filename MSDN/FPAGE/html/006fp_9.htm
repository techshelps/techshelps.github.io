Hyperlink Status View

For a report on the status of the hyperlinks in the current FrontPage web, choose this view. Internal hyperlinks point to pages or files in the current FrontPage web, while external hyperlinks point to pages, sites, or files outside of your FrontPage web or local file system.

When you switch to the Hyperlink Status view, internal hyperlinks are verified at once and displayed only if they are broken. External hyperlinks, however, are not immediately verified because they point to pages outside of your FrontPage web and, depending on network traffic, can take a long time to verify.

Broken hyperlinks are shown with a red “Broken” status and unverified hyperlinks are shown with a yellow “Unknown” status. When an unverified or broken hyperlink has been verified or repaired, it is displayed in the list with a green “OK” status. You can use the Verify Hyperlinks command on the FrontPage Explorer’s Tools menu to verify external hyperlinks in your FrontPage web.

Once you have verified all of the hyperlinks in your FrontPage web, you can use the Hyperlink Status view to repair any broken hyperlinks. You can edit each hyperlink directly, edit the page containing the hyperlink, or add a web task to fix the hyperlink. These options are available from the Edit Hyperlink dialog box, which is displayed when you double-click a broken hyperlink in the FrontPage Explorer’s Hyperlink Status view.

For more information, see “Creating Hyperlinks and Bookmarks”.
