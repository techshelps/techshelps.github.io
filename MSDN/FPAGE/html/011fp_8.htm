Form Field Validation

When you set the properties of a form field you can set field validation rules that limit what a user can enter in the field. For example, in a one-line text box, you can make a rule that a user must type an integer into the field with a value in the range 0 – 100. You set field validation rules for a form field by opening its properties dialog box, clicking Validate to open the field’s validation dialog box, and setting rules that limit what a user can type into the field.

The FrontPage Editor can generate validation scripts in the JavaScript or Visual Basic Script (VBScript) languages, which both run on the Web browser computer. You can specify the type of validation script that the FrontPage Editor should generate in the Advanced tab of the FrontPage Web Settings dialog box in the FrontPage Explorer.

The validation script checks the contents of the field when the user submits the form in a Web browser and, if they do not conform to the rules that you set, displays an error message. A backup script will also run on the Web server computer to validate the fields, in case the Web browser computer does not support the scripting language you choose.

You can validate the following types of fields:

  • One-line text box

  • Scrolling text box

  • Radio button

  • Drop-down menu
