Tasks View

This view displays a list of tasks associated with a FrontPage web. Tasks are items that may need your attention before you publish your FrontPage web. If you are working in a multi-user environment, such as an intranet, you can assign tasks to other authors.

You can add tasks to the Tasks view, as well as modify task names, descriptions, and who the task is assigned to. Certain FrontPage Explorer web wizards and some of the FrontPage Editor’s commands can add tasks and link them to the appropriate page or file. For example, when you check the spelling in the pages in your FrontPage web, you can choose to add a new task for each page containing misspellings. You can open the pages that require corrections by double-clicking a task, and then clicking Do Task in the Task Details dialog box.

To monitor all tasks in a FrontPage web — including completed tasks — you can enable Task History from the FrontPage Explorer’s Views menu while in the Tasks view.

While it is not necessary to complete every task before you publish your FrontPage web, it is a good idea to review the list of tasks when you are finished making changes to your FrontPage web. The Tasks view helps you manage your FrontPage web by flagging important reminders for you.

Shared Borders and Navigation Bars

Shared borders are a useful way to present recurring information in a consistent manner. In the FrontPage Explorer, you can specify that one or more shared borders be included on all pages that are part of the FrontPage web’s navigational structure (as displayed in the Navigation view). You can also add shared borders for individual pages, using the FrontPage Editor.

If you enable shared borders, FrontPage will automatically create navigation bars that help users navigate your Web site.
