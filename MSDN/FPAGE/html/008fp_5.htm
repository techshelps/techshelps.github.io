Managing Hyperlinks in the FrontPage Explorer

FrontPage automatically manages hyperlinks in your FrontPage web. For example, when an author moves a page from one folder to another in a FrontPage web or renames a page, FrontPage automatically update all hyperlinks to that page from every other page and Microsoft Office document in the FrontPage web. Along with automatic hyperlink management, the FrontPage Explorer includes the Hyperlink Status view, in which you can test and repair hyperlinks in a FrontPage web.

The Hyperlink Status view displays a list of all the internal and external hyperlinks in the current FrontPage web. (An internal hyperlink links to a page or file within the current FrontPage web. An external hyperlink points to a page or file outside of your FrontPage web.) You can select the Hyperlink Status view by choosing Hyperlink Status on the View menu.

When you first select the Hyperlink Status view, the FrontPage Explorer checks all internal hyperlinks before displaying the hyperlink list. The FrontPage Explorer does not check external hyperlinks at first because it can take a long time to access each page or file over the Internet.

The status of each hyperlink is displayed in the Hyperlink Status view:

  • OK indicates that the hyperlink is good. Note that good hyperlinks inside your FrontPage web are not displayed by default. You can view them by selecting Show All Hyperlinks in the View menu.

  • Broken indicates that the hyperlink is bad. A bad hyperlink indicates either an incorrect URL or a missing page or file.

  • Unknown indicates that the hyperlink is external and has not been checked yet.

  • Added task indicates that the hyperlink is either broken or has not been verified yet, and a related task has been added to the Tasks view.

You can use the Hyperlink Status view to repair broken hyperlinks:

  • To edit a selected broken hyperlink, choose Edit Hyperlink from the shortcut menu, and repair the hyperlink. Click Browse to browse the World Wide Web for the hyperlink target.

  • To verify one or more selected hyperlinks, choose Verify from the shortcut menu.

  • To edit the page containing a selected hyperlink, choose Edit Page from the shortcut menu. This opens the page in the FrontPage Editor.

  • To add a task to the Tasks view to fix or verify one or more selected hyperlinks, choose Add Task from the shortcut menu.

Why are images displayed in the Hyperlink Status view? When you insert an image on a page in the FrontPage Editor, you are not placing the image directly on the page. Instead, you are creating a hyperlink from the page to the image. A hyperlink to an image can be broken, for instance if you delete the image from your FrontPage web. You can use the commands in the Hyperlink Status view to repair broken links to images in the same way that you repair other types of hyperlinks.
