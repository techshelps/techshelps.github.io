Formatting the Search Form

Before you can test the results page, you must also design the page that will be used to search your database for criteria specified by a user.

Û To Format the Search Form

  1. In the FrontPage Editor, open or create the page you will use to obtain criteria input from the user, then position the cursor where you want the search form to be displayed.

  2. On the FrontPage Editor’s Insert menu, point to Form Field, then click One-Line Text Box.

    By default, text box form fields are placed on the page with Submit and Reset buttons.

  3. Double-click the one-line text box to open the Text Box Properties dialog box.

  4. In the Name field, replace the default with a proper name value.

    The name value you enter here must match the name that you entered in the second step of the Database Region Wizard. In our example, the name used was CategoryName.

  5. Text box fields should also have a descriptive label that tells users what information it will accept. Place the insertion point just before the left border of the text box, then click the mouse button to activate the cursor.

  6. Type the words Enter the Product Category:, then press the spacebar.

    You can use more than one parameter field in the same form, provided that the SQL text was modified accordingly to accept more than one parameter in the query.

Next, you must configure the form so that it will correctly link the search field to the database file and display the results using your Active Server Page.

Û To Configure a Custom Form Handler

  1. In the FrontPage Editor, right-click the one-line text box and choose Form Properties from the shortcut menu.

    Normally, the default FrontPage form handler can process data submitted through form fields. For ODBC database integration, however, you must specify your Active Server Page as the form handler.

  2. In the Form Properties dialog box, select Send to Other and choose the Custom ISAPI, NSAPI, CGI or ASP Script option.

  3. Next, click the Options button to specify additional options for this custom form handler.

  4. In the Action field, type results.asp — the file name of the results page you previously created.

  5. Ensure that the Method field contains “POST,” then click OK to close the Options for Custom Form Handler dialog box.

  6. Click OK once more to close the Form Properties dialog box.

Before you can use the search form on this page, you must save it to your current FrontPage web as an HTML file. On the FrontPage Editor’s File menu, choose Save As. In the Save As dialog box, type the file name for this page is the URL field. For this example, name the page “search.htm.” In the Title field, enter a descriptive page title, such as “Database Lookup.” Click OK to save the page.
