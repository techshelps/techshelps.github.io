Integrity Enhancement Facility Conformance

The Desktop Database Drivers support the same optional Integrity Enhancement Facility (IEF) features (as contained in the X/Open and SAG SQL CAE specifications) that Microsoft Jet exposes. A call to SQLGetInfo with an fInfoType argument of SQL_INTEGRITY (SQL_ODBC_SQL_OPT_IEF in ODBC 2.x) returns an “N,” indicating that the drivers do not support the full IEF syntax contained in the X/Open and SAG SQL CAE specifications. However, the Microsoft Access driver supports the IEF features supported by Microsoft Jet by using Microsoft Jet syntax. For instance, the Microsoft Access driver supports the same syntax for creating a primary key that Microsoft Jet does.
