Replication by Using DAO

The DAO programming interface provides methods and properties you can use to implement replication and manipulate replicas. DAO provides access to all the replication functionality available from the Briefcase or from within Microsoft Access, as well as partial replicas, to any application that supports Microsoft Jet and DAO.

Note Not all applications that support Microsoft Jet use its features in the same way. For example, although Microsoft Excel cannot replicate a database, it can be used to update a database that has been made replicable by another product. Any changes made to the database through Microsoft Excel will be correctly synchronized with other replicas.

You can use DAO to:

  • Convert a database to a Design Master.

  • Create and populate additional replicas.

  • Create partial replicas.

  • Synchronize replicas.

  • Get and set specific properties of a replicated database.

  • Handle or resolve conflicts or errors.
