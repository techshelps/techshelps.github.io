Specific Testing Requirements For Office Compatible 97 Products

Tables 1 and 2 list the specific testing requirements for file-based and non-file-based standalone applications. For more information about a particular requirement, read the technical reference that is listed for the feature.

Table Legend:

M = Mandatory. The feature must be supported as Office does.

MIS = Mandatory If Supported. The feature is not mandatory, but applications that include it (or a substantially comparable entry) must use the standard name, location, and/or accelerator key, as appropriate.

MDA = Mandatory Design Adaptation. A suitably adapted version of the Office standard is mandatory. In other words, if the standard Office version of the feature is not appropriate, then an alternate design (derived from the Office standard) must be used instead. For more information, read the technical reference that is cited for the feature.
