<File> was checked out from <Local Folder>, not from the current folder. Continue anyway? (Y/N)

This warning has the following cause and solution:

  • When a file is checked out, VSS records where the file was copied to.  When a file is checked in from the command line, VSS, by default, looks in the current folder for the file.  If a copy of the file exists in the current folder, but that not in the location the file was checked out to this message will appear.

    Type Y if you want to check in the file that exists in the current folder.

    Type N if you want to abort the operation and not check in the local folder.  If you want to check in a file that doesn’t exist in the current folder, use the –GL switch of the checkout command to tell VSS what folder to check in the file from.
