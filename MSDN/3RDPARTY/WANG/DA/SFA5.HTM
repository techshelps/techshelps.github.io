The Object Hierarchy

The Imaging OLE Automation object model includes one top-level object (the Application object) and one document object (the ImageFile object). The Application object is added to the system registry the first time you start the Imaging application. In addition, there are two supporting objects, Page and PageRange, for the ImageFile object.

Imaging OLE Automation exposes only the Application object for creation. Other programmable objects are created or deleted by referencing the Application object.
