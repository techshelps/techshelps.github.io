Adjusting an Engine's Pronunciation of a Word

Typically, it should not be necessary to adjust the pronunciations of words in the engine's native language. However, engines are likely to mispronounce foreign names or invented words (such as those that commonly occur in game software). An application can adjust an engine's pronunciation of a word in any of the following ways:

· If the engine has a pronunciation lexicon, the correct pronunciation can be added to the lexicon and the incorrect pronunciation (if any) removed. For more information, see the ILexPronounce interface in this section.

· The application can provide a text-to-speech engine with phonemes instead of ASCII text to be spoken. For more information, see the ITTSCentral::TextData member function in the section, "Low-Level Text-to-Speech API."

For example, suppose a text-to-speech engine mispronounces a word that is not in the engine's lexicon. Correcting the pronunciation would involve the following steps:

1. Determine what phonemes the engine is using by calling the ITTSCentral::Phoneme member function with the text of the word. For example:

pITTSCentral->Phoneme(CHARSET_IPAPHONETIC, //IPA character set
NULL, // text is not tagged
dtext, // text of the word
pdPhoneme); // function fills in pointer to phonemes

The function fills in the SDATA structure pointed to by the pdPhoneme parameter with a pointer to a WCHAR array of Unicode values, which contains the phonemes used to pronounce the word.

2. Check the phonemes to determine which ones should be changed.

3. Store the correct pronunciation in the engine's lexicon by calling the ILexPronounce::Add member function, or pass the corrected phonemes to the engine by calling the ITTSCentral::TextData member function and specifying CHARSET_IPAPHONETIC for the eCharacterSet parameter.

If an engine doesn't support the IPA or the ILexPronounce interface, it may still provide a dialog box that allows an end-user to adjust pronunciation, typically by supporting the LexiconDlg function of the ISRDialogs interface (for a speech-recognition engine) or the ITTSDialogs interface (for a text-to-speech engine). For more information, see the section, "Low-Level Speech Recognition API" or "Low-Level Text-to-Speech API."
