Notification Interface

When an application calls the IVoiceText::Register member function to register its voice-text site, the application must also provide a pointer to its IVTxtNotifySink interface. The voice-text object calls this interface to notify the application of events related to voice text. To receive notifications, the application must create a Component Object Model (COM) object that supports the IVTxtNotifySink interface. The application developer determines how an application responds to notifications.

The voice-text object sends a variety of notifications to the IVTxtNotifySink interface. When calling Register, an application indicates whether the object should send all notifications or just a subset. A voice-text object can send the following notifications.

Notification

Description

AttribChanged

Sent when an application changes an attribute for a voice-text site. The notification includes a flag that identifies the attribute that changed.

Speak

Sent when an application puts text into the playback queue for the voice-text site.

SpeakingDone

Sent when speaking is finished and no text remains in the playback queue.

SpeakingStarted

Sent when the engine has started speaking text for a site.

Visual

Sent when the engine is speaking text for a site. The notification identifies the phoneme that the engine is currently speaking and the type of sentence that contains the phoneme (for example, a question, command, or statement).
