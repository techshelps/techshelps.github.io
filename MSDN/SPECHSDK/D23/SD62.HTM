IVTxtNotifySink::Visual

HRESULT Visual(WCHAR cIPAPhoneme, TCHAR cEnginePhoneme, DWORD dwHints,

PTTSMOUTH pTTSMouth);

Notifies all applications on a voice-text site about the phoneme that is currently being spoken.

· Return value is ignored.

cIPAPhoneme

[in] International Phonetic Alphabet (IPA) phoneme that is being spoken. Note that this parameter is always a wide character (WCHAR) because IPA symbols are available only in the Unicode character set.

cEnginePhoneme

[in] Engine-specific phoneme that is being spoken.

dwHints

[in] Type of sentence in which the phoneme is being spoken. This parameter can be one of these values:

Value

Meaning

TTSNSHINT_COMMAND

The phoneme is part of a command.

TTSNSHINT_EMPHASIS

The phoneme is part of a word being emphasized.

TTSNSHINT_EXCLAMATION

The phoneme is part of an exclamation.

TTSNSHINT_QUESTION

The phoneme is part of a question.

TTSNSHINT_STATEMENT

The phoneme is part of a statement.


pTTSMouth

[in] Address of a TTSMOUTH structure that contains information about the mouth position that corresponds to the current phoneme. If the mouth position is unknown or not supported, this parameter is NULL.

This notification may or may not be supported by an engine. If it is supported, this notification is sent only to applications that set the dwFlags parameter of the IVoiceText::Register member function to the VTXTF_ALLMESSAGES value when the application registered to use voice text on the site.

An application can use the information returned by Visual to synchronize character animation with spoken text.
