Writing an Audio Source/Destination Object

Applications can write their own audio source or destination objects to rout the audio as they desire. Some typical uses might be:

Speech Recognition Accuracy Test. Write an audio source that gets its audio from a wave file.

Speech Recognition Over the Network. The audio source gets audio from a network pipe rather than a wave device.

Cache Text-to-Speech Output. Write an audio destination that writes its audio to a wave file so that the application can save processing on text-to-speech phrases it speaks a lot.

DirectSound Output. Write an audio destination object that send its audio to the DirectSound driver rather than the wave device.

Echo, Chorus, and other Effects. The audio destination object does some digital effects on the audio before passing it down to another audio destination object.

Text-to-Speech Over the Network. Pipe the audio from the audio destination through a network pipe to another computer.

Get notified ahead of time. If some of the timing notifications come right when the text-to-speech audio is being played then the animation will always lag behind the audio. An application can write an audio destination object that passes all of the data down to another audio destination object except that it re-maps bookmarks so they happen a bit sooner than the text-to-speech engine thinks.

Writing an audio source or destination object is straight forward. Look on the speech SDK for a sample.
