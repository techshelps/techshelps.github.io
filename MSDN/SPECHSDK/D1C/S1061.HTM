IAudio::PassNotify

HRESULT PassNotify(PVOID pNotifyInterface, IID IIDNotifyInterface);

Passes the address of a notification interface to the audio object so that the object can notify the engine when the amount of available data changes or other events occur.

· Returns NOERROR if successful, or one of these error values:

E_INVALIDARG
E_OUTOFMEMORY
AUDERR_INVALIDNOTIFYSINK

pNotifyInterface

[in] Address of the notification interface through which the audio object notifies the engine about audio information.

Because passing the address to the audio object does not transfer ownership of the notification interface, the audio object must call the interface's AddRef member function before returning from the call to PassNotify. The audio object must also call the IAudio interface's Release member function when it closes. The calling application must release any reference counts it holds on the notification interface after calling PassNotify, unless it needs the notification object to be valid when the audio object releases it.

IIDNotifyInterface

[in] Interface identifier of the interface used for notification. The identifier must be IID_AudioDestNotifySink for an audio-destination object or IID_AudioSourceNotifySink for an audio-source object.

Initially, an audio object has NULL for a notification sink.

An engine can use the notification interface to get information about the status of audio data, such as whether or not there is space available in the object's internal buffer.

If PassNotify is called more than once, Release is called on the previous notification interface pointed to by pNotifyInterface before AddRef is called on the new interface pointed to by pNotifyInterface.
