Low-Priority Input

The audio source object uses a feature of some sound card drivers, called "low priority." Low priority allows the speech recognition to be listening all of the time, but it allows other applications to play or record sounds. When the other application play or record sounds, the audio to the audio source object is temporarily suspended, and resumed when the other application stops playing/recording.

(New for 3.0)

However, some sound cards have bugs in their low priority support that will cause problems for end users. The audio source object has a list hard-coded in, but new drivers come out every day. An application can override whether or not low priority is used by writing to:

HKCU\Software\Microsoft\SpeechAPI, UseLowPriority=<0 or 1>

A DWORD value of 1 causes low priority to always be used, and 0 to never be used. If there is no value then the object uses its default choice.
