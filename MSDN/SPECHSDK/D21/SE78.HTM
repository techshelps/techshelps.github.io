IVDctText::AutoLockSet

HRESULT AutoLockSet(BOOL fLock);

Sets the value of the automatic locking feature.

· Returns NOERROR if successful, or one of these error values:

E_INVALIDARG
E_OUTOFMEMORY

fLock

[in] Turns the automatic locking feature on/off.

If automatic locking is on, then after command notifications the text buffer will be locked so that the engine pauses recognition until the buffer is unlocked. This allows the application to do accurate synchronization on machines that are behaving slowly. For example, the sequences of utterances "this" "app-command" "is" will make sure that "app-command" will be processed before "is" is recognized.
