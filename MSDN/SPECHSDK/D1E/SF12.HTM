Text-to-Speech Error Values

The following errors may be returned by member functions of the text-to-speech objects. If no error occurs, the member functions return NOERROR.

Error value

Description

<<TTSERR_INVALIDCHAR>

The text contains an invalid character.

<<TTSERR_INVALIDINTERFACE>

The interface is invalid.

<<TTSERR_INVALIDKEY>

The registration key is invalid.

<<TTSERR_INVALIDMODE>

The speech mode is invalid.

<<TTSERR_INVALIDWINDOW>

The window is invalid.

<<TTSERR_NOTSUPPORTED>

The function is not supported.

<<TTSERR_OUTOFDISK>

There is no more disk space.

<<TTSERR_QUEUEFULL>

The text queue is full. This error is returned when the application fails to wait for a ITTSBufNotifySink::TextDataDone notification before calling TextData, and the engine's queue is full.

<<TTSERR_VALUEOUTOFRANGE>

The value is out of range.

<<TTSERR_WAVEDEVICEBUSY>

The wave device is busy.

<<TTSERR_WAVEFORMATNOTSUPPORTED>

No wave formats are supported.


In addition to the error values described in the preceding list, member functions can return error values generated by lower-level component objects or calls to the Win32 API. For example, a member function of the ITTSEnum interface might return an AUDERR_ value generated by the audio-destination object. For more information, see "Object Levels and Error Values."
