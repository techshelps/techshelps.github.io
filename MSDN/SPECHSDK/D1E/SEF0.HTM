ITTSBufNotifySink::TextDataDone

HRESULT TextDataDone(QWORD qTimeStamp, DWORD dwFlags);

Notifies the application that the text in a particular text-to-speech buffer has been sent to the audio-destination object or the application called the ITTSCentral::AudioReset member function.

· Return value is ignored.

qTimeStamp

[in] Time, in bytes, when the engine finished sending all of the text in a text-to-speech buffer to an audio-destination object. An application can ignore this parameter.

dwFlags

[in] Reason that the text data is finished. If this parameter is NULL, all of the text in the text-to-speech buffer has been sent to the audio-destination object. If this parameter is the TTSBNS_ABORTED value, the application called the ITTSCentral::AudioReset member function.

This notification is sent when the text-to-speech buffer is emptied, either by sending its data to the audio-destination object or by the application calling the ITTSCentral::AudioReset member function to stop sending the text and empty the buffer. The ITTSBufNotifySink interface is released by the engine after an pending BookMark or WordPosition notifications are sent.

Because the notification is sent after the audio finishes processing, the application may receive the notification before the audio in the buffer actually finishes playing.
