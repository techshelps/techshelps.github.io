Dialog Boxes

A text-to-speech engine provides a set of dialog boxes that enable the user to configure the engine. By using the ITTSDialogs interface provided by a text-to-speech engine object, an application can direct the engine to display one of its dialog boxes. The controls and capabilities in the dialog boxes are determined solely by the engine. An application retrieves the address of the ITTSDialogs interface by calling the IUnknown::QueryInterface member function on the engine object with the IID_ITTSDialogs interface identifier.

When an application calls a member function of the ITTSDialogs interface to display a dialog box, it specifies the handle of the parent window for the dialog box. By specifying NULL instead of a window handle, the application can determine whether the engine supports the given dialog box. The engine returns the NOERROR value if it supports the dialog box or the TTSERR_NOTSUPPORTED error value if it does not.

The application also specifies the title of the dialog box by specifying the address of a title string. If the application specifies NULL instead of a title string address, the engine uses the default title.

About Dialog Box

An engine's About dialog box contains information that identifies the text-to-speech engine and its manufacturer and contains the manufacturer's copyright notice. An application displays the About dialog box by calling the ITTSDialogs::AboutDlg member function.

General Dialog Box

The General dialog box gives the user control of the general features of the text-to-speech engine. An application displays the General dialog box by using the ITTSDialogs::GeneralDlg member function. Depending on the engine, the General dialog box may allow the user to perform the following tasks:

· Set the maximum amount of disk space that the text-to-speech engine can use. The amount of disk space affects the playing of concatenated wave files.

· Set the speaker preferences supported by the engine.

· Provide the user with access to any other dialog boxes provided by the engine.

Lexicon Dialog Box

A text-to-speech engine maintains a database of pronunciations called a pronunciation lexicon, and an engine may allow an application to collect new or corrected pronunciations from the user. The Lexicon dialog box displays the user's pronunciation lexicon and allows the user to edit it. For example, the user can edit the phonetics of mispronounced words and load or unload personal exception dictionaries. An application uses the ITTSDialogs::LexiconDlg member function to display the Lexicon dialog box.

For more information about pronunciation lexicons, see the section, "Common Elements of the Speech Application Programming Interface."

Translation Dialog Box

The ITTSDialogs::TranslateDlg member function causes the engine to display a dialog box that enables the user to control how the engine speaks symbols, currencies, and abbreviations. The user can also control number-translation techniques, such as whether the engine should speak "34" as "thirty-four," "three four," or the old English "four and thirty."
