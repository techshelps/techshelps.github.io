Control Tags

An application can embed control tags in text buffers. By using control tags, the application provides clues about the text, so the engine can do a better job of speaking it, perhaps even changing the speaking voice. An application can use control tags to perform the following tasks:

· Change the voice, language, or speaking style. For example, an application might use a different voice for different data types or conduct a conversation between different voices.

· Insert bookmarks in text. Bookmarks notify an application when the audio for the given text has been reached so that the application can switch to a new page in a book, do some animation, or do any other synchronized event.

· Specify how a sentence should be read. An application can instruct the engine to read a sentence quickly, with excitement, or in a monotone.

If text contains tags, the application should set the TTSDATAFLAG_TAGGED value when it calls the ITTSCentral::TextData member function to place a buffer in the engine's speaking queue. As an alternative, the application can call the ITTSCentral::Inject member function to insert control tags into text as the engine plays it.

For more information about about text-to-speech control tags, see Appendix A, "Text-to-Speech Control Tags."
