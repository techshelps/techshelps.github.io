ITTSNotifySink::AudioStart

HRESULT AudioStart(QWORD qTimeStamp);

Notifies an application when audio data starts playing. Audio may not start playing until several milliseconds after a buffer is placed while the engine computes the first fraction of a second of audio.

· Return value is ignored.

qTimeStamp

[in] Time, in bytes, when the audio data starts playing. An application can ignore this parameter.

If the engine does not actually send out audio information (for example, if the ITTSCentral::TextData member function is called with only a tag and no other characters or spaces), the ITTSNotifySink interface's AudioStart and AudioStop member functions are not called. This allows an application to change the state of an engine without sending output. The exception to this behavior is the \pau=number\ tag, which actually sends out silence.
