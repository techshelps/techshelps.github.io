ISRResSpeaker::Identify

HRESULT Identify(DWORD dwRank, TCHAR *pszName, DWORD dwNameSize,

DWORD *pdwNeeded, LONG *plScore);

Identifies the speaker for a recognized utterance.

· Returns NOERROR if successful, or one of these error values:

E_INVALIDARG
E_OUTOFMEMORY
SRERR_NOTENOUGHDATA
SRERR_NOTSUPPORTED
SRERR_VALUEOUTOFRANGE

dwRank

[in] Rank of the speaker. An engine ranks the candidates with integers, starting with 0 for the most likely speaker. If dwRank is greater than the number of speakers ranked by the engine, the function returns an error.

pszName

[out] Address of a buffer that receives the name of the person identified by dwRank.

dwNameSize

[in] Number of bytes allocated in pszName. If the number is too small, the function copies only part of the name string and fills pdwNeeded with the buffer size needed for the whole string.

pdwNeeded

[out] Address of a location in memory that receives the number of bytes needed to copy the entire name string into pszName.

plScore

[out] Address of a variable that receives the engine's level of confidence that the speaker in pszName actually spoke the utterance. The score is normalized, so that values less than 0 are not considered accurate enough for the engine to commit that the speaker in pszName is speaking.

An application can call Identify to learn the identity of the speaker, querying for the most to least likely choice and learning the confidence of the engine in that choice. The engine gets information about speakers through training and correction done while the ISRAttributes::SpeakerSet member function has set the specific speaker's name string.

Calling Identify can take additional processor time beyond that needed for speech recognition.
