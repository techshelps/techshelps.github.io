ISRResMemory::LockGet

HRESULT LockGet(BOOL *pfLock);

Retrieves the lock status for a speech-recognition results object. A results object that is locked does not deteriorate and lose information.

· Returns NOERROR if successful, or one of these error values:

E_INVALIDARG
E_OUTOFMEMORY
SRERR_NOTSUPPORTED

pfLock

[out] TRUE if the speech-recognition results object is locked and its data does not deteriorate, or FALSE if the object is allowed to deteriorate. The default is FALSE.

Because results objects occupy so much memory, an engine may discard some of a results object's data over time so that memory isn't consumed by results objects. An application can prevent deterioration of a results object by calling the LockSet member function.

The effects of the deterioration are engine-dependent, but they probably involve a loss of data, starting with the largest data first. This means that digital-audio data is lost first, followed by the information necessary for training. If the information has deteriorated, calling any function that uses the missing information returns an error.
