ISRGramCFG::ListGet

HRESULT ListGet(TCHAR *pszListName, PSDATA pdWord);

Retrieves the list of words that the engine is currently using.

· Returns NOERROR if successful, or one of these error values:

E_INVALIDARG
E_OUTOFMEMORY
SRERR_INVALIDLIST
SRERR_NOTSUPPORTED

pszListName

[in] Address of a buffer that contains the name of the list.

pdWord

[out] Address of an SDATA structure that receives information about the list. The dwSize member of the structure receives the number of bytes needed for the list information and the pData member receives the address of a memory object that contains a sequential list of SRWORD structures, one for each word in the list. (This is not an array, because the size of each structure varies with its contents.) It is not necessary to explicitly terminate the list; it is terminated implicitly because no more SRWORD structures fit in the specified amount of memory.

The application allocates the SDATA structure and passes its address to the ListGet member function. ListGet allocates memory (by using the OLE task allocator) for the returned data and sets the pData member of the SDATA structure to point to the memory. If the allocation fails, the pData member is set to NULL and the dwSize member is set to zero. The application must free the memory pointed to by pData, as well as the SDATA structure itself.

This member function allocates memory and fills in a list of words. If a list has not been set or is empty, the memory will be empty and the member function returns SRERR_INVALIDLIST.

If the engine does not support lists, it should return SRERR_NOTSUPPORTED.

The application must free the memory allocated by the member function by using the CoTaskMemAlloc and CoTaskMemFree functions.
