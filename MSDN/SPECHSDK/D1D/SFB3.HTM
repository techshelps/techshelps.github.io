ISRGramCommon::Archive

HRESULT Archive(BOOL fArchiveResults, PVOID pBuf, DWORD dwBufSize,

DWORD *pdwBufNeeded);

Serializes a grammar object into memory, using a memory format that is native to the engine. The native format for the engine is stored in a data buffer.

· Returns NOERROR if successful, or one of these error values:

E_INVALIDARG
E_OUTOFMEMORY
SRERR_INVALIDFLAG
SRERR_NOTSUPPORTED

fArchiveResults

[in] TRUE if the engine should archive all information in existing speech-recognition results objects, or FALSE if no results objects should be archived. Archiving the information in results objects may use a lot of memory, so an application should do this only if it needs the results objects for the next use of the application or document.

pBuf

[out] Address of a buffer that receives the archived data — the engine's native format and results objects, if any. As an alternative to a buffer, pBuf can point to memory in a memory-mapped file.

dwBufSize

[in] Number of bytes available in the buffer specified by pBuf. If the number is too small, Archive returns SRERR_NOTENOUGHDATA and fills pdwBufNeeded with the number of bytes needed.

pdwBufNeeded

[out] Number of bytes needed for the data to be archived.

If the engine does not support the grammar's standard format, Archive returns an error.

If fArchiveResults is TRUE, existing speech-recognition results objects are archived with the grammar. Information about lists is not archived. The application can save the archive data to disk or leave it in the buffer.

To reuse the grammar, the application calls the GrammarLoad member function and passes in the native format, which loads more quickly than standard format. If the archived grammar includes speech-recognition results objects, the engine calls the ISRGramNotifySink::UnArchive member function for each results object present when archive was called so the application can retrieve them.

While a grammar is activated, the speech-recognition engine may modify its engine-specific grammar structure as it acquires more information about the patterns of the speakers' speech. For this reason, the Archive member function should be called each time a grammar is deactivated and just before it is released, to ensure that the engine's modifications are preserved for the next use of the grammar.
