Dialog Boxes

A speech-recognition engine provides a set of dialog boxes that let the user configure the engine. By using the ISRDialogs interface provided by a speech-recognition engine object, an application can direct the engine to display one of its dialog boxes. The controls and capabilities in the dialog boxes are determined solely by the engine.

When an application calls a member function of the ISRDialogs interface to display a dialog box, it specifies the handle of the parent window for the dialog box. By specifying NULL instead of a window handle, the application can determine whether the engine supports the given dialog box. The engine returns NOERROR if it supports the dialog box, or SRERR_NOTSUPPORTED if not.

The application also specifies the title of the dialog box by specifying the address of a title string. If the application specifies NULL instead of a title string address, the engine uses the default title.

About Dialog Box

An engine's About dialog box contains information that identifies the speech-recognition engine and its manufacturer, and it contains the manufacturer's copyright notice. An application displays the About dialog box by calling the ISRDialogs::AboutDlg member function.

General Dialog Box

The General dialog box gives the user control of the general features of the speech-recognition engine. An application displays the General dialog box by using the ISRDialogs::GeneralDlg member function. Depending on the engine, the General dialog box may allow the user to do the following:

· Inform the engine of the speaker's gender or other speaker-specific information

· Control how stringent the engine is about recognition

· Control the maximum amount of disk space allotted for speaker information

· Control optimizations that affect speech processing time

· Export and import speaker information to and from a disk so that the information can be transferred between systems

Lexicon Dialog Box

A speech-recognition engine maintains a database of pronunciations called a pronunciation lexicon, and an engine may allow an application to collect new or corrected pronunciations from the user. The Lexicon dialog box displays a user's pronunciation lexicon and allows the user to edit it. For example, the user can edit the phonetics of mispronounced words and load or unload personal exception dictionaries. An application uses the ISRDialogs::LexiconDlg member function to display the Lexicon dialog box.

For more information about pronunciation lexicons, see the section, "Common Elements of the Speech Application Programming Interface."

General Training Dialog Box

The ISRDialogs::TrainGeneralDlg member function displays the General Training dialog box, which trains a speech-recognition engine by having the user speak a preselected set of words and phrases. After the user speaks the words and phrases, the recognition accuracy should improve for that speaker.

Microphone Training Dialog Box

The Microphone Training dialog box trains the engine for use with a particular microphone so that recognition accuracy improves with that microphone. An application should display this dialog box at the request of the engine or whenever the speaker wants to display it. Training information is associated with both the speaker and the microphone. If the engine does not supply this dialog box, the speaker cannot train it for the microphone; however, the application will still work. The application uses the ISRDialogs::TrainMicDlg member function to display the Microphone Training dialog box.
