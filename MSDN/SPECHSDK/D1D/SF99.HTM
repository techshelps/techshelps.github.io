ISRCentral::Pause

HRESULT Pause(void);

Pauses the speech-recognition engine, incrementing the pause count. The application must call the Resume member function after each call to the Pause member function.

Retrieves information specific to the current speech-recognition mode.

· Returns NOERROR if successful, or E_OUTOFMEMORY if a low-memory condition occurs.

Pausing an engine prevents it from processing future phrases but not from identifying the beginning and ending of utterances.

Calling this function during an utterance pauses the processing of digital audio. This function affects the notifications received through the ISRGramNotifySink interface, and may affect notifications received through the ISRNotifySink::VUMeter member function.

The Pause member function may be called at different times by different applications that share the same engine, or it may be called automatically when a PhraseFinish notification is sent. For this reason, the Pause function maintains a pause count that is incremented any time the function is called. The engine starts with a pause count of 0.

If an engine is paused more than a second on a real-time audio stream, it may start losing information from the audio source because internal buffers aren't large enough to accumulate that much audio.
