Objects

There are two objects involved in doing text-to-speech:

Voice-Text Object. This is the object that handles speech and the only one that the application deals with.

Notification Sink. The notification sink code is optional and is supplied by the application. The voice-text object calls methods in this object when audio starts or stops playing, or with mouth-animation cues.

Each object has a set of interfaces that it supports. These are listed in the sections below.
