About OLE Automation

The voice-command and voice-text objects are exposed as OLE automation objects, allowing them to be accessed and controlled through Visual Basic, Visual Basic for Applications (VBA), and other automation-aware application macro languages. You can take advantage of the OLE automation capabilities of these objects to add voice-command and voice-text capabilities to your Visual Basic applications.

This appendix describes the properties and methods that Visual Basic programmers can use to manipulate the voice-command objects.

Note that any version of Visual Basic or VBA that supports OLE automation can be used with voice-command and voice-text object. However only Visual Basic version 4.0 supports the custom asynchronous notification mechanism. Using previous versions of Visual Basic requires the use of a polling mechanism to periodically check to see if a voice command was recognized or another event occurred.

All OLE automation methods described in this appendix set a generic error #440, "OLE Automation error" if they fail.

For examples of Visual Basic applications that use the voice-command and voice-text objects, see the sample applications included with the Speech Software Development Kit.

For more information about OLE automation, see the Microsoft Win32 Software Development Kit (SDK).
