Object Creation

To create a voice-command object, an application calls the CoCreateInstance function with the CLSID_VCmd class identifier and the IID_IVoiceCmd interface identifier. An application must create a voice-command object for each audio source that it needs to use. For example, if the application needs to recognize commands from a telephone line as well as the computer microphone, it should create two voice-command objects.

After creating a voice-command object, CoCreateInstance returns a pointer to the IVoiceCmd interface for the voice-command object. An application uses IVoiceCmd to register itself with the speech-recognition engine and to create voice-menu objects and voice-menu enumerators. For more information about voice-menu objects and voice-menu enumerators, see "Voice-Menu Object" later in this section and "Voice-Menu Enumerator" later in this section.

Before an application can perform other voice-command tasks, it must register itself by calling the IVoiceCmd::Register member function. By registering, the application tells the engine which site to use as the source of voice commands and passes the voice-command object a pointer to the application's voice-command notification interface. The object calls the notification interface to notify the application about events related to voice commands.
