Object Creation

After an application has registered its site with the voice-command object, it can create a voice-menu object by using the IVoiceCmd::MenuCreate member function. MenuCreate retrieves a pointer to an IVCmdMenu interface for the newly created voice-menu object. An application uses the IVCmdMenu member functions to obtain information about the voice menu, add and remove menu commands, and so on.

When calling MenuCreate, the application specifies the name of the voice menu that the new object is to represent and indicates whether the function should create a new menu or open an existing one from the voice-menu database. If the function creates a new menu, the menu initially contains no commands. The application must add commands by using the IVCmdMenu::Add member function.

A voice menu is stored in the database, unless it was created as a temporary menu. A temporary menu is destroyed when the voice-menu object is released. Typically, you will store a voice menu between invocations of the voice-command object, so you should first call MenuCreate with the VCMDMC_OPEN_EXISTING value. If MenuCreate returns an error, you should call it a second time with the VCMDMC_OPEN_ALWAYS value, which creates a new voice menu for the application.

An application can use the IVoiceCmd::MenuEnum member function to create a voice-menu enumerator object to enumerate the menus currently in the voice-menu database. For more information about voice-menu enumerators, see "Voice-Menu Enumerator" later in this section.
