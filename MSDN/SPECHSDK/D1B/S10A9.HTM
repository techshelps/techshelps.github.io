ISTPhoneConv::SetByText

HRESULT SetByText(LPCWSTR pszText);

Sets the phone character set to use.

pszText

[in] NULL-terminated string that defines the IPA to character set conversion. See below for the format:

· Returns NOERROR if successful, or one of these error values:

E_INVALIDARG
E_FAIL - Buffer is too small.

The text string is a NULL-terminated Unicode string. It contains new-lines (\n) to separate lines. It has a header noted by the string, "[Header]", and then rules for converting "[PhoneToIPA]". The rules are sorted in order of priority, so the first rules take precedence over later rules. Also, rules with longer matches take precedence over rules with shorter matches.

[Header]

Spaces=<1 if spaces occur between phones, 0 if they do not>

CaseSensitive=<1 if the set is case sensitive, 0 if it's case insensitive>

[PhoneToIPA]

<phone #1 text>=<IPA hexsymbol numbers separated by spaces>

<… more phones>

[Examples]

<phone #1 text>=<word using phone, with parentheses surrounding phoneme>
