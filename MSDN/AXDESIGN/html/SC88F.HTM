Supporting Property Pages

ActiveX Designers can support property pages to allow users to change and view properties. Each property page is associated with an object and allows the end user to edit properties for the object. An object can have more than one property page; each page handles a group of related properties.

Users access the property pages through a property frame. The property frame displays the property pages that apply to an object (or a set of objects).

The following Application Programming Interfaces (APIs) and interfaces handle the interactions between objects, their properties, property pages, and property frames:

  • The ISpecifyPropertyPages interface defines the set of property pages that apply to an object.
  • The IPropertyPage interface creates and manipulates an individual property page.
  • The IPropertyPageSite interface associates a site with a property page.
  • The OleCreatePropertyFrame API creates a property frame.
  • The IPropertyPage2 interface navigates to a property on a property page.
