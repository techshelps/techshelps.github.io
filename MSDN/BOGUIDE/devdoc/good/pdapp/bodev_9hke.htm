Supporting Remote Administration

Supporting remote administration allows for single-seat administration of multiple servers within an enterprise environment. Even though servers within an organization may be spread throughout the world on a wide area network (WAN), it is possible to architect applications to administer remote services running on remote servers. To create server applications that can be administrated remotely, you should:

  • Use the Windows NT registry to store configuration parameters.
  • Use event logging to audit actions, failures, errors, and so on.
  • Create a separate user interface (UI) client portion that communicates to a server portion by way of remote procedure call.
  • Support delayed connections.
