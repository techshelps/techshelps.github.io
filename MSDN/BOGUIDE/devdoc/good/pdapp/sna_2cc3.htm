AFTP API Calls

AFTP API calls can be organized into the following functional categories:

  • Creating or destroying an AFTP connection object
  • Establishing and managing a connection to the AFTP server
  • Querying the characteristics of the connection to the AFTP server
  • Transferring files between the AFTP client and AFTP server programs
  • Specifying file transfer characteristics
  • Querying file transfer attributes
  • Listing files on the AFTP server
  • Listing files on the AFTP client
  • Performing directory manipulation to manage directories on the client and server
  • Performing file manipulation to modify files on the AFTP server
  • Querying system information to acquire information about the AFTP client and AFTP server computers
  • Generating message strings, which are used to get text strings for use in messages issued by your application
  • Controlling tracing of AFTP activity
  • Miscellaneous, providing a method of loading the AFTP initialization file
