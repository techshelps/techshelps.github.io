Visual C++ Components Folder

Home |  Overview |  How Do I

This default folder contains shortcuts (links) to the Visual C++ components (.DLL files) registered on your system. These components are either registered automatically by the Setup program, or are components you purchase and register separately by running the vendor’s setup program.

The Gallery regenerates links to registered components every time you open it, based on the current state of the registry. This has certain implications:

  • If you delete a link to a registered component, the link will still appear the next time you open the Gallery.

  • If you delete a registered component (.DLL file), but do not unregister it, the link will still appear the next time you open the Gallery. Of course, the link will be inoperative.

  • If you copy a component to your machine, but do not register the file, the link does not appear because the Gallery does not recognize the component.

To unregister a component, see Deleting a Gallery File or Folder.

Tip   When you use Windows Explorer to drag a component link from a default folder, the Gallery creates a shortcut to the component in the new folder so that you don’t have to worry about maintaining it in more than one location.
