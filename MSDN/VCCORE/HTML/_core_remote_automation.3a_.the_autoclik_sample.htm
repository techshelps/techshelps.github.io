Remote Automation: The AUTOCLIK Sample

Home |  Overview |  How Do I |  Sample

AUTOCLIK is a simple Automation server application that you can use as a base from which to learn more about Remote Automation.

To install the AUTOCLIK project onto your development machine

  1. Select Search from the Help menu. The MSDN Library browser appears.

  2. Type “AUTOCLIK” in the edit box of the Index tab.

  3. Double-click “AUTOCLIK sample” in the list of index entries, or select “AUTOCLIK sample” and click the Display button.

  4. The topic "AUTOCLIK: Automation Server Application" appears in the MSDN Library browser's main window.

  5. Click STEP3 to install the AUTOCLIK project.

To install AUTOCLIK.EXE on two machines and drive it using Remote Automation

  1. Build the Step 3 version of AUTOCLIK.EXE.

  2. Run AUTOCLIK.EXE in standalone fashion and then shut it down. This will register it as an Automation server.

  3. Copy AUTOCLIK.EXE to a remote machine, run it there, and then shut it down.

  4. Build AUTODRIV.EXE and verify that running it starts AUTOCLIK.EXE on the local machine. Read the AutoClik Tutorial to find out more about AUTODRIV.EXE.

  5. On the remote machine, start AUTMGR32.EXE (Automation Manager).

  6. On the remote machine, start RACMGR32.EXE (Remote Automation Connection Manager).

  7. In the Remote Automation Connection Manager, select AutoClik.Document from the OLE Classes list.

  8. Choose a system security policy from the Client Access tab to grant client access to AutoClik.Document.

  9. On the local machine, start RACMGR32.EXE and select AutoClik.Document from the OLE Classes list.

  10. From the Server Connection tab, choose both the network address of the remote machine and the appropriate network protocol.

  11. With AutoClik.Document still selected in the OLE Classes listbox, choose the Remote command from the Register menu.

  12. On the local machine, run AUTODRIV.EXE or the equivalent AUTOCLIK.MAK Visual Basic project if you want to have a Visual Basic, rather than an MFC, client.

On the remote machine, you should now be able to see AUTOCLIK executing commands initiated from the local client.
