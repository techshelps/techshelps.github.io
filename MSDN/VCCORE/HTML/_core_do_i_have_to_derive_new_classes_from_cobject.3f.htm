Do I have to derive new classes from CObject?

Home |  Overview |  How Do I |  FAQ

No, you don't.

Derive a class from CObject when you need the facilities it provides, such as serialization or dynamic creatability. Many data classes need to be serialized to files, so it's often a good idea to derive them from CObject. For an example of a class derived from CObject, see The Document's Data: Class CStroke in the Scribble tutorial.
