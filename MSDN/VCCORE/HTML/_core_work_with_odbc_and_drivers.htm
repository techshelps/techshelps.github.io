ODBC allows you to write applications that are not dependent on just one database management system (DBMS). You can write one application that can operate on multiple DBMSs for which your users have the correct DBMS-specific ODBC driver.

What do you want to do?

  • Determine the schema of an ODBC data source at run time

  • Get a driver description using ODBC Administrator

  • Obtain online help for your ODBC driver

  • Programmatically create a table in an ODBC data source

  • Call ODBC API functions directly in MFC
