Debugging Your Program

You have created your application and resolved the build errors. Now it's time to find and correct those logic errors that keep your application or stored procedures from running correctly. You can do this with the development environment’s integrated debugging functions.

This debugging section explains solutions to common problems in 32-bit programming. You’ll learn how to stop at procedure locations, inspect memory and register values, change variables, observe message traffic, and get a close look at how your code works -- or doesn’t!
