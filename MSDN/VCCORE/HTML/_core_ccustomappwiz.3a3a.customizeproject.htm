Feature Only in Professional and Enterprise Editions   Creating a Custom AppWizard is supported only in Visual C++ Professional and Enterprise Editions. For more information, see Visual C++ Editions.

virtual void CustomizeProject(IBuildProject* pProject)

Parameters

pProject A pointer to the BuildProject object.  You may access methods and properties on the BuildProject object through this pointer to customize the project settings.

Remarks

AppWizard (MFCAPWZ.DLL) calls the CustomizeProject member function just after generating all of the files and creating the project.

CustomizeProject provides a place for you to customize the settings of the project.  For example, you can adjust the compiler and linker settings, and add custom build steps that execute after the target is built.  For details on what aspects of the project may be customized, see BuildProject and Configuration.

It is important to note that IBuildProject and all interfaces you can get from it (such as IConfiguration) are COM interfaces.  You must be careful to release all interfaces you acquire (by calling the Release() method on each of those interfaces).  In accordance with the standard rules of COM, you must not release pProject, unless you explicitly call AddRef on it, since pProject is passed as an "in" parameter to this function.
