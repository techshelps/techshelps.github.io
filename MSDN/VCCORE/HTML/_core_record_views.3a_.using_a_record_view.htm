Record Views: Using a Record View

This article explains how you might commonly customize the default code for record views that the wizard writes for you. Typically, you’ll want to constrain the record selection with a filter or parameters, perhaps sort the records, or customize the SQL statement.

This information applies to both CRecordView (ODBC) and CDaoRecordView (DAO).

Using CRecordView or CDaoRecordView is much the same as using CFormView. The basic approach is to use the record view to display and perhaps update the records of a single recordset. Beyond that, you might want to use other recordsets as well, as discussed in the article Record Views: Filling a List Box from a Second Recordset.
