Translating Message Requests

[This is preliminary documentation and subject to change.]

If a WinSNMP manager application requests functionality that is available under the SNMP version 2C framework (SNMPv2C), but the target entity supports only the SNMP version 1 framework (SNMPv1), the Microsoft WinSNMP implementation attempts to translate the request to the SNMPv1 format. To do this, the implementation uses the procedures defined in RFC 1908, "Coexistence Between Version 1 and Version 2 of the Internet-Standard Network Management Framework." If translation is not possible, the SnmpSendMsg function fails with the extended error code SNMPAPI_OPERATION_INVALID.
