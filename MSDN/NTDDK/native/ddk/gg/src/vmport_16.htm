Setting up NumEmulatorAccessEntries

In x86-based Windows NT machines, nonVGA-compatible miniports always set the NumEmulatorAccessEntries to zero in the VIDEO_PORT_CONFIG_INFO buffer, because a system-supplied VGA or VGA-compatible SVGA miniport driver provides this support for full-screen MS-DOS applications. In other Windows NT platforms, video miniport drivers need not support emulator access entries and should set this member to zero.

A VGA-compatible SVGA miniport driver must set up emulator access entries in the VIDEO_PORT_CONFIG_INFO buffer. For all RISC-based Windows NT platforms, this member of the VIDEO_PORT_CONFIG_INFO structure is ignored.
