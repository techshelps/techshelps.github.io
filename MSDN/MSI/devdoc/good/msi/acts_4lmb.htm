Standard Actions

[This is preliminary documentation and subject to change.]

In the Microsoft® Windows® installer, actions serve the same purpose as functions in a programming language. An action encapsulates a typical function you can perform during installation or maintenance of your application. There is a very close relationship between actions and the installer database. In fact, many of the tables in the database can be viewed as serving the singular purpose of containing parameters used by a corresponding action. Almost all of the standard actions are linked to at least one of the tables.

Examples of installer standard actions include:

  • CreateShortcuts action: Manages the creation of shortcuts to files installed with the current application. This action uses the Shortcut table for its data.
  • InstallFiles action: Copies selected files from the source directory to the destination directory. This action uses the File table for its data.
  • WriteRegistryValues action: Sets up registry information that the application requires in the registry. This action uses the Registry table for its data.

    Note  The need to write registry values into the HKCU and HKLM hives of the registry at installation will diminish in the future as part of the ZAW effort.
