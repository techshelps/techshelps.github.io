Separating an Application into Components

[This is preliminary documentation and subject to change.]

One of the most important aspects of making an application work effectively with the installer is separating the application into features and components. For information on separating an application into features, see Separating an Application into Features. For more information on components, see Components and Features.

If you answer no to any of these questions, you should consider separating the application into components differently:

  • Do your application's features and components have a close relationship?
  • Do all parts of the component map to a common feature?
  • Do all files of a component have a common installation location?
  • Do all parts of the component have a common setup logic?
