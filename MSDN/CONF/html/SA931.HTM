Query Manager

The query manager is the glue that binds the application to the remote agent. It is part of the application. It performs the following functions:

  • Calls agent and obtains results
  • Handles requests in parallel or spools query requests
  • Activates and populates appropriate forms upon receipt of data

Although a query manager is not required, it allows simpler coding and more features. A key feature should be the ability to resume an application. Other features are:

  • Cleans up obsolete MDBs
  • Reports failures and time-outs
  • Retrieves data from cache (old MDB) instead of submitting new queries
