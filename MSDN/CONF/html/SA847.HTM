Conclusion

The SDIS demonstrates how complex solutions can be developed using the component architecture of the Microsoft Workgroup Strategy. The solution includes the following technologies:

  • Integrating Visual C++ components with Visual Basic solutions.

  • Using OLE architecture includes containers, objects, automation, and the structured storage file.

  • Integrating API's such as MAPI and the Schedule+ Access Libraries.

  • Taking advantage of powerful servers such as SQL Server and Mezzanine.

The leverage these technologies provide drastically reduce the development and maintenance associated with complex solutions.
