Description

The architecture incorporates the following characteristics:

  • A SQL Server-based back-end (4.2+) running on Microsoft Windows NT 3.5 operating system.

  • A front-end built in Visual Basic 3.0 Professional Edition.

  • The front-end uses Microsoft Excel 5.0 spreadsheet pivot tables to present multidimensional views of user-selected result sets.

  • All client-side SQL is based on SQL Server stored procedures.

  • Client-side GUI elements (e.g. menus, buttons) dynamically enable/disable based on SQL Server administrative security changes.

  • A two-tiered dedicated DSS data warehouse, separate from the transaction processing environment, that supports generalized tables and denormalized application-specific tables.

  • No C-based development.
