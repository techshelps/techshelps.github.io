The Magician's Hat

The key to our solution is a layered architecture. The query moves from application to agent to communications link, such as RAS or e-mail. The received query moves from the communications link to agent to database. The result set returns along the same route to the application or client.

The use of an agent also permits dynamic client configuration for connections. Considering the connections' properties, the agent can fulfill the query request by:

  • Direct link on high bandwidth
  • RAS agent on low bandwidth with high priority
  • MAPI agent on low or no bandwidth
