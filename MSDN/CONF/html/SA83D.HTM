Running Test Plans

The Test Plan is submitted to the testing lab where the assigned Test Engineer conducts the Test Plan. A Test Bar, displayed at the bottom edge of the application, displays the task which should be executed. If the task succeeds, it is accepted; if not, a defect description is entered.

Figure 10: Test Bar.

The Test Bar floats above the application currently being tested. A button on the bar allows the tester to determine if the bar anchors to the top or bottom of the application.

After all tasks are executed, the Test Plan is closed. The script can be conducted several times until all tasks execute successfully.
