DAY 2 (TUESDAY, NOVEMBER 5)

On day 2, you can find out more about the store for all the security principles in the new Windows NT Security model—the Windows NT Directory Service. You can also attend a talk to find out more about how security is layered on top of the Windows NT Security APIs—the Microsoft Security Support Provider Interface (SSPI). SSPI is the common API for obtaining integrated security services for authentication, message integrity, message privacy, and security quality of service for any distributed application protocol. Application protocol designers can take advantage of this interface to obtain different security services without modification to the protocol itself.
