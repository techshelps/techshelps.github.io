Integrating Interfaces

By integrating the various interfaces, gateways and services available, information from various BackOffice and third party repositories can be accessed and made available to the desktop. Below are some data repositories that can be accessed and made available to the BackOffice family of products:

ODS Gateways to a variety of data formats

  • ORACLE, DB2, Informix, Ingres, Btrieve, VSAM, IMS

    MAPI connects to various back-ends

  • Fax, CompuServe, Microsoft Mail, MS Net, Internet (Messaging and Netnews)

    SNA Server APIs accesses information on IBM platform

  • CICS, VSAM, SQL on IBM Host and AS400, DB2

By integrating these services between the BackOffice family of products, below are some examples of additional functionality available:

MAPI integration with SQL Server provides:

  • performing message-based data transactions

  • support for writing remote query applications

  • custom message types can access data (supply order form)

Systems Management Server integration with SQL Server provides:

  • developers can query configuration data using Systems Management Server

  • ODS and VBA allows Front Office applications to read Systems Management Server data

Systems Management Server integration with any BackOffice application

  • Create PDFs

  • Remotely deploy software and manage versions and program groups

  • Remote helpdesk

SNA integration with SQL Server provides:

  • access to IBM host data

SNA integration with any BackOffice application:

  • RAS over SNA

  • Provides APPC API for migration of legacy applications

  • access to IBM host data

  • Integration of data and messaging on host for staged migration

  • Support popular emulators for HLLAPI applications

Systems Management Server Integration with SNA provides:

  • SNA Sender for Systems Management Server

  • Systems Management Server can operate over SNA WANs

SNA Integration with SQL

  • ODBC/DRDA Driver

    DB2 in MVS

    SQL/DS in VMS

    DB2/400 in AS/400

    SQL Integration with any BackOffice application

    • Use Extended Stored Procedures to execute any program on the Win32 platform offering Net Send Broadcast

    • Use Extended Stored Procedures to integrate with any device on Win32 platform offering services such as fax any document or alert a pager
