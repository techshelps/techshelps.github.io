IADsLocality

The IADsLocality interface enables a COM object in Active Directory to manage the data representing geographical location of an accout. This interface is one of several that provide support for directory services that organize accounts by country, locality (geographical location), organization (company), and organizational unit (department). Properties accessed by IADsLocality are defined on the ADSI Locality Object.

When to Implement

Implement IADsLocality when you are providing an Active Directory implementation for your directory service that organizes directory entries by geographical location.

When to Use

Call the methods on this interface when you are expanding a directory that organizes accounts by physical location.
