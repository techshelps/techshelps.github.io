About Authentication

You can supply credentials to a directory service using the IADsOpenDSObject::OpenDSObject method or the ADsOpenObject function. Both these routines include user-name and password parameters. A flags parameter can indicate binding option preferences when you want to use secure authentication or encryption options, but the directory service may override these flags. Because the flags are provider specific, the directory service may not support them at all. For example, the Microsoft® NDS-compliant ADSI provider always uses strong authentication. For more information, see ADSI System Providers.
