ADSI Object Definitions

The Active Directory object definitions included in this document are examples of how COM objects can be used to support the Active Directory Service Interfaces. They are supplied here as a convenience.

All the properties accessed by property methods supported on Active Directory Service Interfaces are defined in this section.

ADSI client applications should be aware that ADSI providers may choose to support provider-specific interfaces on ADSI objects and may provide access to the underlying namespace data store attributes or properties using the late-bound methods for saving and retrieving properties by name: IADs::Get, IADs::GetEx, IADs::Put, and IADs::PutEx.

The following ADSI objects are grouped by function. There are several categories of ADSI objects:

  • Standard ADSI objects that are implement by all ADSI provider software. See Standard ADSI Objects.
  • ADSI objects that represent specific elements in the underlying directory service. See Example ADSI Objects.
