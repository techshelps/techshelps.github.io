ADSI WinNT Provider

To access the directory services of a Windows NT 4.0 node on your network using the Microsoft implementation of Active Directory Service Interfaces, you must have ADSI installed. ADSI registers the case-sensitive programmatic identifier (progID) "WinNT" so that all ADsPath names that begin with "WinNT" will be routed to this implementation.

The WinNT provider object definitions follow the ADSI Provider specifications as described in Microsoft's Platform SDK, Networking and Distributed Services, Active Directory Service Interfaces, ADSI for Providers.
