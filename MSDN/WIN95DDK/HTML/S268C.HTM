About Windows 95 Multilingual IME

In Windows® 95, IMEs are provided as Dynamic Link Libraries (DLLs), in contrast to IMEs of Windows 3.1 Far East Edition, and each IME runs as one of the multilingual keyboard layouts. Compared to Windows 3.1 IME, the Windows 95 multilingual IMEs provides the following advantages:

  • Runs as a component of the multilingual environment
  • Provides multiple input contexts for each application task
  • Provides one active IME for each application thread
  • Gives information to an application through its message loop (message order is not changed)
  • Provides strong support for both IME-unaware and IME-aware applications

To take full advantage of IME, an application needs to support the Windows 95 IME application I/F.

This document describes the application I/F of the Windows 95 IME architecture.
