Windows NT 3.5 32-Bit LAN Manager APIs

In Windows NT version 3.5, the multinet layer was formalized to be a Multiple Provider Router (MPR) and the API set was expanded to provide network browsing capability using Win32 interfaces. The API set was also expanded to allow for some differences between networks (for example, different naming conventions and the NetWare server security model).

The 32-bit LAN Manager APIs exported from Netapi32.dll in Windows NT version 3.5, are not supported in Windows 95 because they are Unicode-only APIs.
