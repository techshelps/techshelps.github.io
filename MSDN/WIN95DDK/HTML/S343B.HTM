PNP_NEW_DEVNODE 


include vmm.inc
include configmg.inc

mov     ebx, hDevnode
mov     edx, LoadType
mov     eax, PNP_NEW_DEVNODE
VMMcall System_Control

Notifies a dynamically loadable VxD that a new device node has been loaded. Uses the C calling convention. This is a directed system control message. The virtual device may modify the EAX, ECX, EDX, and Flags registers, but it must preserve the EBX, EDI, ESI, and EBP registers.

  • The virtual device returns a configuration manager return value (CR_) in EAX. The carry flag must be set if a value other than CR_SUCCESS is returned. It doesn't hurt to return carry set even if you are returning CR_SUCCESS, so it's safest to always return carry set.

hDevnode

Handle of the new device node.

LoadType

Type of functionality that the VxD should handle. Can be one of these values:

Value

Meaning

DLVXD_LOAD_DEVLOADER

Device loader

DLVXD_LOAD_DRIVER

Device driver

DLVXD_LOAD_ENUMERATOR

Device enumerator
