QueryPhysLock

This service is used to query if the current process owns the physical volume lock on the int 13h drive passed in. The IFS manager checks its internal data structures to see if this process is also the owner of the physical lock.

[EAX]

Supplies the int 13h unit number for the disk the operation is being done on.

  • Carry clear, current process owns the volume lock.

    Carry set, current process does not own the volume lock.

Registers Used  
EAX, ECX, EDX, Flags.
