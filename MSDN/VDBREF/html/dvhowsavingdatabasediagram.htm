Saving a Database Diagram

Save your database diagram to update the database with the changes you made to the diagram layout and the object definitions in the diagram.

If you want to discard your changes to the diagram layout but keep the option to save your changes to your object definitions at a later time, you can save a change script. You can then run the change script at some other time to update the database. You can also edit the change script’s SQL code in a text editor before you run it.

For more information about change scripts and how to use them, see Saving a Change Script.

To save a database diagram

  1. Click the Save button on the toolbar.

  2. In the Save New Database Diagram dialog box, type a name for your diagram and choose OK.

  3. If you have made changes to tables in your diagram, the Save dialog box appears and displays a list of tables that will be saved in the database when you save your diagram. Choose Yes to update the database to match your diagram.

Database diagrams are saved in the dtproperties or MicrosoftDTProperties table of your database.
