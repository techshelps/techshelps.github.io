Customizing the Open Data Services ODBC Driver

There are two places in which you can customize the behavior of the Open Data Services ODBC driver to suit your gateway application: the gateway resources DLL and the ODBC.INI file.

The gateway resources DLL at the client computer contains information about the gateway's capabilities. These capabilities include:

  • the datatypes that are supported by the back-end DBMS
  • gateway support for the various ODBC escapes
  • support for transaction isolation levels
  • support for scalar functions
  • support for SQL Server statement preparation optimizations

The capabilities information is stored as a Windows string resource, with one string for each capability.

In addition, a new section in ODBC.INI or Registry has been added to those required by the SQL Server driver, specifying the file path of the resources DLL and optional initialization strings.
