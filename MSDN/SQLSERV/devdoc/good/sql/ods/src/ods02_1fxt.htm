System Requirements

This chapter describes the system requirements for using Microsoft Open Data Services. For information about installing a completed Open Data Services server application, see Network Setup Options for Open Data Services Server Applications.

Microsoft Open Data Services requires a computer that includes the following software components:

  • Microsoft Windows NT Workstation version 3.5 or later (version 3.51 or later is recommended)

    Or

    Microsoft Windows NT Server version 3.5 or later (version 3.51 or later is recommended)

  • Microsoft Visual C++ version 2.0 or later, or 100% compatible compiler and linker

Microsoft Open Data Services is available for computers with the following microprocessor architectures:

  • Alpha AXP™
  • Intel® 32-bit x86-based
  • MIPS®

The computer must meet all of the hardware requirements for Microsoft Windows NT and for a minimum version of Microsoft Visual C++, such as processor type, available disk space, and recommended RAM.

If the computer is also running Microsoft SQL Server, be sure to meet the hardware and software requirements described in the Microsoft SQL Server Setup.

Important Existing Open Data Services applications must be recompiled and rebuilt using Open Data Services 6.0. This is required because of changes to Open Data Services data structures, and because of the removal of certain constants, such as SRV_CONNECTSPERTHREAD, SRV_MINNETWORKTHREADS, and SRV_THREADPOOLING used by srv_config. Applications should not use LocalFree on the errtext and oserrtext parameters within the error handler (registered using srv_errhandle) because Open Data Services frees these now.
