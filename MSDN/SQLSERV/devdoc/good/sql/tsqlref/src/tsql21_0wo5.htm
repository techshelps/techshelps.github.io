sp_dropremotelogin System Stored Procedure

Removes a remote user login.

Syntax

sp_dropremotelogin remoteserver [, loginame [, remotename]]

where

remoteserver
Specifies which server has the remote login to be dropped.
loginame
Is the local server's username that is associated with the remote server in the sysremotelogins table.
remotename
Is the remote username that gets mapped to loginame when logging in from the remote server.

Remarks

Executing sp_dropremotelogin drops a user login ID from a remote server, deleting the user's entry from master.dbo.sysremotelogins.

To add local server users, use the sp_addlogin system stored procedure. To drop local server users, use the sp_droplogin system stored procedure.

Examples

A.    Drop a Remote Server

This example drops the entry for the remote server ACCOUNTS.

sp_dropremotelogin ACCOUNTS
B.    Drop a Login Mapping

This example drops the entry for mapping remote logins from the remote server ACCOUNTS to the local user named Albert.

sp_dropremotelogin ACCOUNTS, Albert
C.    Drop a Remote User

This example drops the login for the remote user Chris on the remote server ACCOUNTS that was mapped to the local user salesmgr.

sp_dropremotelogin ACCOUNTS, salesmgr, Chris

Permission

Execute permission defaults to the system administrator and cannot be transferred.

Tables Used

master.dbo.sysremotelogins, master.dbo.sysservers
