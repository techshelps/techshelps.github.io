Installing the MS DTC Client Utility

To install the MS DTC client utility, run the SQL Server setup program on a client computer. The MS DTC client utility includes only basic client support that provides remote access to an MS DTC server.

You do not need to install the MS DTC client utility on a computer that already has the SQL Server server components installed on it. The client utilities are automatically installed with the server components.

For a detailed discussion of installing Microsoft SQL Server client utilities, including supported hardware platforms and hardware and software requirements, see What's New in SQL Server 6.5 and Microsoft SQL Server Setup.

Note Unlike other Microsoft SQL Server client utilities, the MS DTC client utility does not run on 16-bit computers.

  • The setup program:
  • Adds the Microsoft SQL Server 6.5 program group to Program Manager.
  • Adds the MS DTC client configuration program to Control Panel.
  • Updates the Windows Registry with information about the MS DTC files.

The MS DTC client utility files are installed in the \BINN subdirectory of the MSSQL root directory.

After the MS DTC client utility is installed, you must run the MS DTC Client Configuration program on the client computer to configure the name and network protocol of the default commit coordinator. For details about using the MS DTC Client Configuration program and configuring the MS DTC client software, see Configuring the MS DTC Client Utility.

For details about how to set configuration options for other SQL Server client utilities, see What's New in SQL Server 6.5 and the Microsoft SQL Server Administrator's Companion.
