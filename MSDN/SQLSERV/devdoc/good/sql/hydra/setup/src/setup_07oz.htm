Keywords and Character Sets

The following words, reserved in SQL Server 6.0, are keywords in SQL Server 6.5:

AUTHORIZATION LEFT
CASCADE OUTER
CROSS PRIVILEGES
DISTRIBUTED RESTRICT
ESCAPE RIGHT
FULL SCHEMA
INNER WORK
JOIN

The following character sets were added in SQL Server 6.0 and were listed in the README.TXT file:

  • CP1250Central European
  • CP1251Cyrillic
  • CP1253Greek
  • CP1254Turkish
  • CP1255Hebrew
  • CP1256Arabic
  • CP1257Baltic

For these character sets, three generic sort orders are defined:

  • Binary: binary sort order
  • Diction: dictionary, case-sensitive sort order
  • Nocase: dictionary, case-insensitive sort order

In a given character set, these sort orders should be used for all languages that do not have a specific sort order. Specific sort orders are provided in two variants: case-sensitive and case-insensitive.

The following character sets are new with SQL Server 6.5:

  • CP932Japanese
  • CP936Chinese (simplified)
  • CP949Korean
  • CP950Chinese (traditional)

Two new sort orders are supported for all character sets:

  • Binary
  • Windows NT Provided

All character set files are located in \MSSQL\CHARSETS.
