SQL Mail and Microsoft Exchange Server Configuration Requirements

Some Microsoft® Exchange Server configurations might cause SQL Mail to process messages incorrectly. If Microsoft Exchange Server is configured to deliver mail to a local folder, SQL Mail messages and subsequent Microsoft Exchange Server messages are queued in the outbox of the local folder and never sent. These messages can be removed only by stopping SQL Mail and Microsoft Exchange Server and re-starting Microsoft Exchange Server. This may require rebooting your system. To prevent this problem, specify that Microsoft Exchange Server deliver mail to the mail server.

    To deliver mail to the mail server
  1. Open Microsoft Exchange.
  2. From the Tools Menu, select Options.

    The Options dialog appears.

  3. Click the Delivery tab.
  4. From the Deliver new mail to the following location option, choose the Mailbox - Username option.
  5. Click OK.
