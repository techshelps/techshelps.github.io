Replication Restrictions for ODBC Subscribers

The following restrictions apply to replication to all ODBC subscriber types:

  • Batched statements to ODBC subscribers are not supported (because the distribution task -c option is ignored).
  • The ODBC DSN must conform to the SQL Server naming conventions (because the DSN is stored in the sysservers table).
  • The publication option to truncate before synchronization is not supported if the ODBC DSN is not a SQL Server DSN. ODBC subscribers will not be allowed to subscribe to publications that have this option selected.
  • The target server's quoted identifier character (as reported by the ODBC driver) is used.
  • The Character Format bulk copy method must be selected for synchronization (in the Edit Publications dialog box). ODBC subscribers cannot subscribe to publications that have selected the Native Format bulk copy method for synchronization.
