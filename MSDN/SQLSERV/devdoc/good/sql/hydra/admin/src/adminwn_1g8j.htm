Installing the MSSQL-MIB and the SNMP Extension Agents

If the Windows NT-based SNMP service is already installed on the computer when the SQL Server setup program is run, then MSSQL.MIB and SQLSNMP.DLL are installed when SQL Server is installed. SQL Server SNMP support is automatically enabled.

If the Windows NT-based SNMP service is not installed on the computer when the SQL Server setup program is run, then MSSQL.MIB and SQLSNMP.DLL are installed when SQL Server is installed, but SQL Server SNMP support is not enabled. In this situation, to set up SNMP support you must:

  1. Install the Windows NT-based SNMP service.

    To install the Windows NT-based SNMP service, use the Network application in Control Panel. For more information, see your documentation for Windows NT or the online Help for the Network application.

  2. Enable SQL Server support of SNMP.

    To enable SQL Server support of SNMP, run the SQL Server setup program, select Set Server Options and choose Continue, and select the Enable SNMP Support option. For more information, see the online Help for SQL Server Setup.
