Configurable Packet Size

Packet size can be configured in ISQL/w. Configuring the packet size gives you a choice between responsiveness and overall throughput. Larger packets provide less packet overhead and provide better end-to-end throughput: an entire task completes more quickly with larger packets. Smaller packets are more responsive for those interested in intermediate results. Overall, a smaller packet size provides quicker response time, but usually an entire task takes longer to complete with smaller packets than with larger packets.

    To configure packet size
  1. From the File menu, choose Configure.

    The Configure ISQL/w dialog box appears.

  2. Choose the Connection tab and in the Packet Size box, type a value.
  3. To accept the new packet size, choose OK. The new packet size will take effect the next time you reconnect.
