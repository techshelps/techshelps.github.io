Alerts Management Interface

The SQL Enterprise Manager alerts management interface includes these changes:

  • A Send Mail button has been added to the New Operator and Edit Operator dialog boxes. (Both dialog boxes are opened from the Manage Alerts window, which appears after choosing Alerts from the Server menu). The Send Mail button is enabled when the local mail client is already running.

    Choosing the Send Mail button opens the Send Note window from the underlying mail system.

    The Send Mail button is useful for notifying operators about changes in their alert-handling responsibilities.

  • The SQLALRTR.EXE command includes parameter -M, with which you can supply parameters to errors. The SQLALRTR.EXE command requires a comma (,), but no spaces, between each error message parameter. Strings must be delimited with single quotation marks ('). Note that any messages containing the format string %.*s will require two parameters to be supplied: the length and the string. Separate these parameters with a comma (,).

    The following example specifies that message 50001 will receive three parameters: 

    'add'
5
5,'alpha'

    This is the message string associated with message 50001: 

    Testing the %s procedure for the %ldth time. Using test %.*s.

    This is the example command and the message returned: 

    sqlalrtr -E 50001 -M 'add',5,5,'alpha'
  
Testing the add procedure for the 5th time. Using test alpha.
  • SNMP traps can be raised as part of an alert response. For more information, see SNMP Support, later in this document.
  • New task parameters support alert response tasks. For more information, see Task Scheduling Interface.
