Defining Trap Destinations

If you will be raising SNMP traps in alert responses (as discussed in Raising SNMP Traps in Alert Responses) you must use the Network application in Control Panel to specify the workstations to receive SNMP trap notifications.

    To define trap destinations
  1. From Control Panel, choose the Network application.

    The Network Settings Dialog Box appears.

  2. From the Installed Network Software list, select SNMP Server.
  3. Choose Configure.
  4. In the SNMP Service Configuration dialog box, select a community name (for example, public).

    Each server that runs SNMP must belong to a group of hosts that has a community name. The community name is placed in the SNMP packet when the trap is sent. You can specify one or more communities to which the current computer will send traps.

  5. Add the computer names or IP addresses of the workstations to receive SNMP trap notifications:
    • Type the IP address or the computer name in the Address box, and then choose the Add button to move the entry to the Trap Destination box.
    • Repeat for each workstation to receive SNMP trap notifications.
  6. Choose OK.

For more information, see the online Help for the Network application.
