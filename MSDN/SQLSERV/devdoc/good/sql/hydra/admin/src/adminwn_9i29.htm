Configurable Packet Size

With SQL Enterprise Manager, you can configure packet size. Small packet sizes are best if responsiveness is an important consideration. Larger packet sizes require less overhead and provide better end-to-end throughput of data. When large data sets are sent, increase the packet size so that the task will complete more quickly.

    To configure packet size
  1. In SQL Enterprise Manager, from the Tools menu choose Preferences/Configure

    The Configure SQL Enterprise Manager dialog box appears.

  2. Choose the Connection tab.
  3. In the Packet Size box, enter a value.
  4. To accept the new packet size, choose OK. The new packet size will take effect the next time you reconnect.
