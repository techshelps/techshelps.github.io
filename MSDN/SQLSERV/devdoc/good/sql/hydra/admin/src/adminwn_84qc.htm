Running the SQL Server Web Assistant

The SQL Server Web Assistant is automatically installed with Microsoft SQL Server 6.5, and can be optionally installed on 32-bit clients.

The SQL Server Web Assistant works with databases created with SQL Server version 6.5.

To function properly, the SQL Server Web Assistant requires the following privileges:

  • CREATE PROCEDURE privileges in the database you are using
  • SELECT privileges on columns
  • The account in which SQL Server is running must have adequate privilege to create files
    To run the SQL Server Web Assistant

  • From the SQL Server program group, double-click the SQL Server Web Assistant icon.

    The SQL Server Web Assistant Login screen appears.

The login settings are identical to those for SQL Server logins.

    To log in to SQL Server
  1. Enter a SQL Server name.
  2. Enter a login ID.
  3. Enter a password.
  4. To continue, choose the Next button.

Or

  1. Enter a SQL Server name.
  2. Select the Use Windows NT Security To Log In Instead Of Entering A Login ID And/Or A Password option.
  3. To continue, choose the Next button.
