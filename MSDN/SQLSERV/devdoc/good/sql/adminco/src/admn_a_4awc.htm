Running SQLOLE42.SQL

Before you can administer a SQL Server version 4.2 server from SQL Enterprise Manager, you must run the SQLOLE42.SQL script on the SQL Server version 4.2 server. SQLOLE42.SQL is a Transact-SQL script that creates SQL-DMO stored procedures for SQL Server 4.2 servers. (You must install these on a SQL Server 4.2 server before SQL-DMO objects can connect to that server.)

The appropriate SQLOLE42.SQL file can be found in the distribution directory (\ALPHA, \I386, or \MIPS) containing the software compatible with the hardware platform’s processor architecture.
