Managing Statement Permissions

A system administrator or database owner can assign statement permissions that allow individual users or groups to perform various database tasks, such as:

  • Creating a database
  • Creating rules and defaults on a database
  • Creating tables and views on a database
  • Creating stored procedures
  • Backing up the database

Each database has its own independent permissions system. Being granted permission to use a certain statement in one database has no effect on other databases. Only a system administrator (SA) or a database owner (DBO) can administer statement permissions.
