Transaction Logs and the recovery interval Option

The recovery interval configuration option defines the maximum number of minutes that SQL Server needs per database to complete its recovery procedures in case of a system failure. If the activity in the database is such that recovery would take longer than the recovery interval, SQL Server issues a checkpoint. Because the checkpoint process examines a particular database only every 60 seconds, enough logged activity can occur during this interval that the actual recovery time required exceeds the time specified in the recovery interval option.
