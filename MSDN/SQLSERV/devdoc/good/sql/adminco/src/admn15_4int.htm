Recovery

In many cases, SQL Server replication can automatically recover from events that in the past would have been considered a catastrophic failure. Recovery considerations vary based on the replication role that a server performs: publication server, distribution server, or subscription server. This section discusses both automated and manual replication recovery for each type of server.

Important This recovery process can be further assisted by appropriate scheduling of database backups and cleanup retention periods. (For more information, see Replication Maintenance.)
