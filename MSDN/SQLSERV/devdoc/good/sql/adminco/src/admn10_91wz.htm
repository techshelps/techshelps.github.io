Dropping Remote Users

Dropping a remote login deletes the remote login ID from the sysremotelogins table and prevents that user from accessing the remote server through his or her local server.

    To drop a remote login
  • Type:

    sp_dropremotelogin remoteserver [, loginame [, remotename]]

    where

    remoteserver
    Specifies which server has the remote login to be dropped.
    loginame
    Is the local server's username associated with the remote server in the sysremotelogins table.
    remotename
    Is the remote username that gets mapped to loginame when logging in from the remote server.

For example, to drop the remote user chris from the remote server ACCOUNTS who was mapped to the local user salesmgr:

sp_dropremotelogin accounts, salesmgr, chris

For more information about sp_dropremotelogin, see the Microsoft SQL Server Transact-SQL Reference.
