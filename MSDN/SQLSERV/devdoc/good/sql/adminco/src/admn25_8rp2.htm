Error 5006

Severity Level 16

Message Text

Could not find enough space on disks to extend database %.*s

Explanation

This error occurs when an alter database statement references a device that is completely full.

This error message does not usually appear in the error log. But if the device takes up the space on a physical disk drive as well as the space allocated for SQL Server, messages stating that the file system is full will appear in the error log. If this occurs, you must correct the file-system problem as well as correct the SQL Server space problem.

Note If the device is not full but the ALTER DATABASE statement requests more space than is available on that device, the database will be extended and fill up the available space on the device. No error message will be displayed.

Action

These are your options:

  • In the ALTER DATABASE statement, specify a device that is large enough. Use the sp_helpdevice and sp_helpdb system stored procedures to identify devices that have enough room.
  • Add a new device to SQL Server by using the DISK INIT statement. Then reference this device with the alter database statement.
  • There is no performance benefit to keeping a database entirely on a single device. However, if this is necessary, follow these steps:
    1. Back up all the databases on the device by dumping them to a valid dump device with the dump database statement.
    2. Drop the device by using the sp_dropdevice system stored procedure.
    3. Initialize a larger device with the disk init statement.
    4. Re-create the old databases on the new device. It is important to create the databases exactly like the old ones or errors will occur.
    5. Reload each database with the load database statement.

For information about how to safely create, dump, load, and re-create databases, see the Microsoft SQL Server Transact-SQL Reference and Part 5, "Data Management."
