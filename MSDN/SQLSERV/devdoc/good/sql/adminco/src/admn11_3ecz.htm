Starting SQL Transfer Manager and Connecting to Servers

Before starting SQL Transfer Manager and connecting to source and destination servers, make sure that the SQL Server service is started on both servers. Review a service's status¾and if necessary start it¾using SQL Service Manager from the Microsoft SQL Server 6.0 program group.

A source server can be a Windows NT-based Microsoft SQL Server, a Microsoft SQL Server for OS/2, or a non-Microsoft-based SQL Server. A destination server can only be a Windows NT-based SQL Server.

    To start SQL Transfer Manager and connect to source and destination servers
  1. From the Microsoft SQL Server 6.0 program group, double-click the SQL Transfer Manager icon.

    The SQL Transfer Manager Connect dialog box appears.

  2. Specify the source server.
    • In the Server box, type the name of the SQL Server to export objects and data from.
    • In the ID box, type a valid login identification for the source server (for example, SA).
    • In the Password box, type the password for that login ID on the source server.
  3. Specify the destination server.
    • In the Server box, type the name of the SQL Server to import objects and data to.
    • In the Login ID box, type a valid login identification for the destination server.
    • In the Password box, type the password for that login ID on the destination server.

    Note If the source server is a Windows NT-based SQL Server, the destination server can be the same server as the source server. The supplied login ID must be the database owner of the destination database

  4. Review the setting of the ANSI to OEM option.

    The ANSI to OEM box is checked if the DB-Library "AutoAnsiToOem" option is set on in your system. If this box is checked, TEXT values will be converted during the transfer. Clear the check box if you do not want TEXT values to be converted.

  5. Choose the Connect button to connect SQL Transfer Manager to the specified servers.

    The SQL Transfer Manager window appears.

    For more information, see Transferring Objects and Data Using SQL Transfer Manager, next in this chapter.
