Managing Remote Servers

Remote servers are set up in pairs. To set up a pair of remote servers, you will:

  • Configure both servers to recognize each other as remote servers.
  • Verify that certain configuration options are properly set for both servers, so that SQL Server allows remote users to execute procedure calls.

Note If you have set up servers in your enterprise for SQL Server 6.0 replication, then each pair of publication and subscription servers has already been set up as remote servers for each other. For replication server pairs (displayed as servers linked together in the Replication Topology window of SQL Enterprise Manager), it is unnecessary to perform the procedures for adding a remote server that are described in this section.
