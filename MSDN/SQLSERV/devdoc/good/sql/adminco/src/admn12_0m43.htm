Setting Backup Task Options

Scheduling a backup creates a backup task. While scheduling the backup, you can set task options that will:

  • Send an e-mail notification to an operator when the backup fails or succeeds.
  • Write an entry to the Windows NT Application log when the backup fails or succeeds.
  • Specify the number of retry attempts when a backup fails, and the delay between each retry.
    To set the backup task options
  1. Define the backup as described in Backing Up a Database or Transaction Log, earlier in this chapter.
  2. From the Database Backup/Restore window, choose the Schedule button.

    The Schedule Backup dialog box appears.

  3. Choose the Options button.

    The Task Options dialog box appears.

  4. To notify an operator by e-mail of the status of the backup task execution, select a name from the list in the Operator box, and then select the e-mail On Success and/or On Failure check boxes.

    If necessary, you can add a new operator by selecting <New Operator> in the Operator box and completing the New Operator dialog box that appears. For information about creating new operators, see Chapter 17, Managing Alerts.

  5. To record information in the Windows NT Event log about the backup task execution, select the NT Event Log On Success and/or the On Failure check boxes.
  6. To specify the number of retry attempts for a failed task, type a number between 0 and 256 in the Retry Attempts box, and another number¾representing the number of minutes to wait before retrying a failed task¾in the Retry Delay box.

    The default for both values is 0.

  7. Choose OK.

    The options are set.
