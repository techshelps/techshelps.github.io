Setting Task Options

When creating or modifying a task, you have the option of setting options for administrator email notification upon task failure or success, Windows NT Event log entries upon task failure or success, and task retry parameters.

    To set task options
  1. While creating a task (using the New Task dialog box) or while modifying an existing task (using the Edit Task dialog box), choose the Options button.

    The Task Options dialog box appears.

  2. To notify an operator by email of the status of the task execution, select a name from the list in the Operator box, and then select the Email On Success and/or the On Failure check boxes.

    If necessary, you can remove an assigned operator by choosing <No Operator> in the Operator box. Or, you can add a new operator by selecting <New Operator> and completing the New Operator dialog box that appears.

    For more information about creating new operators, see Chapter 17, Managing Alerts, or the online Help for SQL Enterprise Manager.

  3. To record information about the task execution in the Windows NT Event log, select the NT Event Log On Success and/or the On Failure check boxes.

    Note that regardless of this setting, information about task execution is always logged in the Task History table. For more information, see Viewing Task History, later in this chapter.

  4. To specify the number of retry attempts for a failed task, type a number in the Retry Attempts box, and another number¾representing the number of minutes to wait before retrying a failed task¾in the Retry Delay box.
  5. For LogReader and Distribution tasks only you can specify a remote target server and database by typing the server name in the Server box, and the database name in the Database box.
  6. Choose OK.

    The options are set. The previous dialog box (New Task or Edit Task) returns.
