EAP Interface with Windows NT

[This is preliminary documentation and subject to change.]

Microsoft® Windows NT® RAS interacts with EAP implementations through the use of function calls that must be exported by the third-party EAP DLL. Another component of EAP is its ability to interface with authentication providers, whether built into Windows NT (such as Windows NT domain authentication or Radius), or provided as a third-party authentication DLL. Some structures defined in EAP header files have members that are used only if an authentication provider is being used. Details on these structures and their use are provided in the individual structure reference pages.
