Semantics of DBGUID_DEFAULT

OLE DB defines a special language dialect identifier, DBGUID_DEFAULT, which denotes the default language dialect. In the context of OLE DB for OLAP, the following semantics apply to the use of this identifier:

  • If the provider is both a TDP and an MDP, then it must interpret the command string as SQL when DBGUID_DEFAULT is passed as the dialect identifier.

  • If the provider is an MDP, then it should interpret the command string as MDX when DBGUID_DEFAULT is passed as the dialect identifier.

Note   MD consumers are strongly discouraged from using DBGUID_DEFAULT for creating datasets. They must use DBGUID_MDX.
