Persisting Local Cubes – Creating the Cube File

Microsoft® PivotTable® Service enables client applications to persist data cubes in local files (default extension is ".cub"). Local data cubes are stored in structured (compound document) files that are the local equivalent of Microsoft OLAP Server objects. More than one local data cube can be stored in a .cub file. A local data cube file name can be specified in DBPROP_INIT_DATASOURCE, and if the specified .cub file doesn't exist, the file will be automatically created when the data cube is initialized with the CREATE CUBE statement.
