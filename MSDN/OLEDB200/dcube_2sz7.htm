Defining Cubes

Microsoft® PivotTable® Service can be used to define data cubes for local storage only; when connected to a Microsoft OLAP Server, creation of data cubes on the server is not allowed. The CREATE CUBE command is used to define the logical data structure of a data cube, and the INSERT INTO statement populates the cube. The syntax of the Data Definition Language (DDL) statements used to define a cube are described in the section "Microsoft PivotTable Service MDX Reference." The source of data used to create a local cube must be a tabular data provider, such as a relational database or Microsoft OLAP Server. Creation of local cubes using a local cube file as the data source is not implemented in this release.
