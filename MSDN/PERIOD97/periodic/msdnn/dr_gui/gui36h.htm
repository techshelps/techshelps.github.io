To OCX or not

Dear Dr. GUI,

One more time! Under Windows NT I get an error message such as "Unable to locate DLL" when attempting to register an OCX. Under Windows 95 I do not get that message. Why not?

Thanks,
Alan Kemp

Dr. GUI replies:

The ActiveX specification states that controls need to export a DLLRegisterServer() function for registration. When a tool such as RegSrv32.exe is used to register an ActiveX control, it must load the DLL using LoadLibrary() in order to call the DllRegisterServer() function. If the ActiveX control is dependent upon other DLLs, they would need to be registered and available on the system in order for the LoadLibrary() call to be successful. Hence, if you simply copy an OCX to another machine it fails to register.

There are a couple of things you can do. If you are using Microsoft Visual Basic, version 4.0, you can create a setup program that makes use of the control in question. When you do this, Visual Basic will create a setup.lst file, which you can use to determine what the dependencies are. Visual Basic 5.0 makes it easier by allowing you to create a dependency file.
