Idle NAK

An idle negative acknowledgment, or Idle NAK, is a request from the receiver to the sender saying that no packets are being received but the transfer is not complete. When a sender gets Idle NAKs from a receiver, it waits to hear from any other receivers, which could still send NAKs requesting packets that need to be resent.

Idle NAK requests are part of the Fully Reliable service.
