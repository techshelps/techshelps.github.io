Important
  • The referenced ASX file can contain one or more Entry elements.
  • Any EntryRef elements in the referenced ASX file is ignored. This means that ASX files can be nested one level only.
  • The Entry elements of the ASX referenced ASX file are included as Entry elements in the original ASX file. The Entry elements are then processed as any other Entry element. The parser keeps track of whether the contents of this Entry element originated with the current ASX file or the ASX file referenced in the EntryRef element in order to adhere to the SkipIfRef attribute of the Entry element.
  • The ASX element and its attributes in the included ASX file are ignored.
  • If a user  bookmarks the piece of content defined in the included ASX file, or adds a Favorites entry to it, the URL used to define it is the URL to the ASX file in the EntryRef element. If the value of the ClientBind attribute is “no,” the user cannot bookmark this URL, or add it to the Favorites list.
