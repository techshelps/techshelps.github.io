Negative Acknowledgement (NAK)

A negative acknowledgement (NAK) is a message sent from the receiver to the sender indicating that packets were dropped. An NAK is a request to resend the packets. Using NAKs is an efficient way to add reliability, because the receiver needs to request only the packets in a sequence that were not received.

NAK requests are part of the Fully Reliable service.
