AdoEnums (ADO/WFC Syntax)

To use AdoEnums, specify the fully qualified name of the enumerated constant. For example, the first constant in the AdoEnums.Schema class is AdoEnums.Schema.PROVIDERSPECIFIC.

Each topic lists the values of the constants. However, use the constant names, not the values. The values are subject to change between releases.

The tables that follow contain three headings that pertain to the core ADO documentation.

  • Topic   This heading is a link to the ADO documentation topic where the enumerated constant is described.

  • Prefix   The names of the ADO/WFC enumerated constants are derived from their ADO counterparts. However, when you read the ADO documentation, it may not be obvious which is the corresponding ADO constant.

    The last qualifier in the ADO/WFC constant is an identifier, written in upper-case. The ADO constant consists of a prefix, followed by the identifier written in mixed-case.

    For example, the equivalent of the ADO/WFC constant, AdoEnums.Schema.PROVIDERSPECIFIC, is the ADO constant, adSchemaProviderSpecific. In this case the prefix, adSchema, is prepended to the identifier, ProviderSpecific.

    This heading lists the prefix you should append to the ADO/WFC identifier to generate the name of the ADO enumerated constant.

  • Declared   ADO COM interface files declare each set of enumerated constants. This heading shows the name of that declaration.

    The name of the COM declaration can be derived from the ADO/WFC constant by suffixing Enum to the second qualifier. For example, the COM declaration associated with AdoEnums.Schema.* is named SchemaEnum.
