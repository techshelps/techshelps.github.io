Exceptions Table

In some cases, it is necessary to specify exceptions to recurring items. For example, you may want to set up a recurring meeting for every Monday at 4 p.m., except for Mondays that fall on a holiday. To handle exceptions, items in the RecurringAppointments table have an Exceptions property, which is a table that contains all exceptions for the recurring item. Each exception is a separate item in the Exceptions table.

The easiest way to create an exception is to delete an instance of a recurring item in the merged table. Schedule+ automatically places the deleted instance in the Exceptions table under the recurrence pattern. For more information on creating exceptions, see Creating Exceptions to Recurring Appointments, Creating Exceptions to Recurring Events, and Creating Exceptions to Recurring Tasks.
