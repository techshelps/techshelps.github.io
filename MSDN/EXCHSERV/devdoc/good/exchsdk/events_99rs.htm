New Agent Dialog Box

The following dialog box is displayed when you choose New or Edit on the Agents property page.

This dialog box has the following controls:

  • Agent Name This is a text edit box through which the author can name the agent. The string entered is used as the PR_DISPLAY_NAME property of the agent message that is written to the folder.
  • When the following event(s) occurs This is a check box-style list box that contains the following event-type choices:
    • A scheduled event occurs
    • A new item is posted in this folder
    • An item is changed in this folder
    • An item is deleted in this folder

    If at least one event is checked in this list box, the agent specified in the Run the following action section will start when that event occurs. Otherwise, no events are monitored in this folder.

  • Schedule This is available only if A scheduled event occurs is checked. Choosing the Schedule button displays the Scheduled Event dialog box, which you use to schedule timer events.
  • Logs Choosing this button displays the last 32KB of log messages resulting from this particular event, in this particular folder. See About Event Logs.
  • Run the following action These controls are used to edit the agent to be invoked when a matching event is triggered.
    • Script Choosing this option button causes the agent to default to the Microsoft Exchange Scripting Agent that handles scripts.
    • Edit Script Choosing this button starts Microsoft Notepad as your script editor. To use Microsoft Visual Studio 97 to edit scripts, refer to the Release Notes for Exchange Server version 5.5.
    • Other (option button) Choosing this option button sets the agent to the one listed in the drop-down list box.
    • Other (drop-down list box) This control displays the list of IExchangeEventSink-compatible objects installed on the server to which the script was bound on the Agents property page.
  • Properties Choose Properties to configure the properties of a COM object (a custom agent). This requires that the actual DLL or EXE files be installed on the local client computer as well as the server, so that the normal ActiveX property pages can be displayed.
  • OK/Apply Choosing OK or Apply registers your folder to be monitored by the Event Service so that your new agent is triggered when the event occurs. Specifically, this action
  1. Writes a message to the application folder. This hidden message contains the script, if the action chosen was to handle scripts with the Microsoft Exchange Scripting Agent. The script, now stored in the application folder, travels with the folder if it is moved.
  2. Writes an event-binding message to the EVENTS_<server name> hidden public folder. This message holds information for use by the Event Service, such as which folder to monitor (it contains the folder's identifier), the name of the server on which to run the action, the identifier of the script message to be run, and any needed COM object properties.
