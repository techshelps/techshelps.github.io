Building the Sample Applications

Once you have installed Windows NT Server, Microsoft Visual C++, the Microsoft Exchange Client, Microsoft Exchange Server, and the Microsoft Exchange Server SDK, you are ready to begin development work. For more information, see Development Environment.

At this point you can also use Visual C++ to build the Microsoft Exchange Server sample applications. You can do this in either of two ways:

  • From a command prompt See Building an Individual Sample Application, or see the sample application's description in Sample Applications.
  • Using the Visual C++ development environment See Building Sample Applications in the Visual C++ Development Environment.

Approximately 1 GB of hard disk space is adequate for setting up the development environment and building the sample applications.

Build Version Types

You can build two different versions of each sample application: DEBUG and RETAIL. See Building the Set of Sample Applications.

Tip Installing the Microsoft Exchange Server SDK creates the \BKOFFICE\BIN directory, copies a complete set of compiled sample applications into it, and adds this directory to your search path. To run a sample application you have built or modified instead of the compiled version in the search path, do one of the following:

  • Use the full pathname.
  • Modify the search path.
  • Run the sample application from the directory containing the object files.
