RPTDRIV: Generating Reports

The RPTDRIV sample application uses REPORT.LIB functions, which generate delivery, nondelivery, receipt, and nonreceipt reports. RPTDRIV logs on to the messaging system as a gateway so you can send mail messages to it. RPTDRIV reads each message and generates a report, sending the report to the originator of the message.

    To build RPTDRIV
  1. Change to the directory containing the RPTDRIV source files:

    \BKOFFICE\SAMPLES\EXCHANGE\RPTDRIV directory.

  2. Type NMAKE.

    To run RPTDRIV
  1. Install the sample gateway. See Installing the Sample Gateway.
  2. Create a custom recipient with an address type similar to the gateway.
  3. Type START RPTDRIV NOTSERV <gateway-name>.
  4. Log on to a mailbox profile using the Microsoft Exchange Client.
  5. Submit a message to the custom recipient with one of the subject lines in the following table. RPTDRIV reads the subject line of the message and returns the corresponding report to the originator. The subject must exactly match one of the choices for the report to be generated.
Subject line Corresponding report type
NDR Nondelivery report
DR Delivery report
NRN Nonreceipt notification
RN Receipt notification
