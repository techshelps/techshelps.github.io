PR_EMS_AB_TOMBSTONE_LIFETIME

The PR_EMS_AB_TOMBSTONE_LIFETIME property contains the number of days that a tombstone will exist before expiring.

Quick Info

Header file: EMSABTAG.H
Applies to: Directory-Cfg objects (directory objects). These objects appear to MAPI as messaging_user objects, which are accessible through the MAPI IMailUser interface.
Identifier: 0x8145
Property type: PT_LONG
Access: Read/write

Remarks

Tombstones are created whenever you delete an object from the directory. It is the tombstone that is replicated to other servers, effectively deleting the corresponding directory entry on the other server. The number that you enter here should be large enough to account for the time required to replicate the tombstone throughout the organization and the length of time required between server backups.
