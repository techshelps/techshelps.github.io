Handling Message Conflicts

Because public folders and the messages they contain can be replicated to multiple servers, users can edit replicated copies of the same message on different locations. A message conflict arises at the time of the next replication. When Microsoft Exchange Server detects a message conflict, it takes actions to resolve the conflict. The specific actions taken are determined by the value of the PR_RESOLVE_METHOD folder property, which is set on the folder that contains the message in conflict.
