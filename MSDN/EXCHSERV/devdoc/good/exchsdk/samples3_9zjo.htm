Rule Actions Recognized by RULEEDIT

Rule Actions and Parameters Recognized by RULEEDIT

Rule Action Parameters
MOVE | COPY Only one of these actions may be specified for any command. The parameter for both is the full pathname of target folder. For example:

ACTION=MOVE /o=MICROSOFT/ou=NORTHWEST /cn="Public Folders" /cn="Next Week's Menu"
DELETE No parameters.
REPLY | MSGID Only one of these actions may be specified for any command. For REPLY, the parameter is replying text, for MSGID, the parameter is reply message identifier. For example:

ACTION=REPLY "This is the last transmission."
FORWARD | DELEGATE Only one of these actions may be specified for any command. The parameter for both is target recipient list. For example:

ACTION=FORWARD "Eva Corets"
BOUNCE Parameter is string value indicating reason for bounce. String values include:

BOUNCE_MESSAGE_SIZE_TOO_LARGE | BOUNCE_FORMS_MISMATCH | BOUNCE_ACCESS_DENIED

Only one bounce action can be contained in a single command. For example:

ACTION=BOUNCE BOUNCE_ACCESS_DENIED
TAG Property assignment. For example, ACTION=TAG PR_EMS_AB_DISPLAY_NAME="Shane S. Kim"
MARKREAD No parameters.

Note If the rule being set must carry out several actions, you may specify several actions in a command section.
