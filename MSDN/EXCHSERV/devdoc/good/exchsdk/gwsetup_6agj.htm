Files Used with Directory Objects

When Microsoft Exchange Server is installed, it creates the following publicly exported directory shares:

\\Server-name\ADD-INS

\\Server-name\ADDRESS

\\Server-name\CONNECT$

Under these directories, subdirectories called Extension-Name\Platform designate the name of the Administrator extension object or Addr-Type object being installed and the platform on which it runs. (For information about platform designations, see Installing the Administrator Extension DLL.) The gateway setup program must create the Extension-Name\Platform subdirectories, along with the directory for the gateway’s executable file.

Use these directories as follows:

  • Files for Administrator extension DLLs Gateways that use a DLL to insert extension data into the directory should install their public files in \\Server-Name\ADD-INS\Extension-Name\Platform.
  • Files for address types If the gateway creates a new address type, files used by the Addr-Type object should be installed in \\Server-Name\ADDRESS\Address-Type\Platform. Note that the address type appears in the PR_ADDRTYPE property in the recipient table of a message, as well as in other message properties.
  • Other files Other public files, such as those that provide server-specific configuration information, should be installed in \\Server-Name\CONNECT$\Extension-Name\Platform.
