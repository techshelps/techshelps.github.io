POPPST: Creating a Customized Information Store

The POPPST sample application demonstrates how to create a new personal folder store (PST) and populate it with one or more test messages (optional). When POPPST creates a new PST, it does so by using a temporary profile that it deletes after PST creation. To create messages in this PST, you must provide an existing profile name for the PST (see examples that follow). For more information about adding information stores, see Opening an Information Store.

POPPST is a command-line program that uses input from an .INI file. POPPST requires one command-line argument, which is the name of the .INI file. For a description of the required format of the .INI file used with POPPST, see POPPST .INI File Format. For general information about .INI and other file formats, see File Formats.

The POPPST application also demonstrates functions of the NEWPST library.

    To build POPPST
  1. Change to the directory containing the POPPST source files:

    \BKOFFICE\SAMPLES\EXCHANGE\POPPST

  2. Type NMAKE.
