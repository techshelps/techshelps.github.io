Files Used with Directory Objects

When Microsoft Exchange Server is installed, it creates the publicly exported directory shares \\SERVER\ADD-INS, \\SERVER\ADDRESS, and \\SERVER\CONNECT$. Under these directories, subdirectories called Extension-Name and Platform designate the name of the application being installed and the platform it runs on. (For information about platform designations, see Installing the Administrator Extension DLL.) The Extension-Name and Platform subdirectories, along with the directory for the application's executable file, must be created by the server application setup program.

These directories are used as follows:

  • Files for Administrator extension DLLs Applications that use a DLL to insert extension data into the directory (including mailbox agents) should install their public files in the \\SERVER\ADD-INS share:

    \\SERVER\ADD-INS\EXTENSION-NAME\PLATFORM

  • Files for address types If the server application creates a new address type, files used by the Addr-Type object should be installed in the \\SERVER\ADDRESS share:

    \\SERVER\ADDRESS\EXTENSION-NAME\PLATFORM

  • Other files Other public files, such as those that provide server-specific configuration information, should be installed into the \\SERVER\CONNECT$ share:

    \\SERVER\CONNECT$\EXTENSION-NAME\PLATFORM
