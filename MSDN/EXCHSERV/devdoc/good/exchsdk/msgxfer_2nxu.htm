Foreign System to Gateway Message Transfer

The following steps are performed when the foreign system transfers a message to the gateway:

  1. The foreign system assigns a message ID to the message being transferred.
  2. The gateway creates:
    • A MAPI message (the content message) to contain the content of the incoming message.
    • An MTE message to contain the content message.
  3. The MAPI message identifier of the content message is assigned to the MTE’s PR_CONTENT_IDENTIFIER property.
  4. The message ID from the incoming message is assigned the MTE’s PR_FOREIGN_ID property.
  5. If the gateway builds a delivery or nondelivery report, the PR_FOREIGN_SUBJECT_ID property from the MTE is assigned to the subject ID of the report.
