Message Identifier Tracking for Inbound Messages

The following topics discuss message identifier tracking when a message is transferred from a foreign system to Microsoft Exchange Server and when a delivery or nondelivery report is returned from Microsoft Exchange Server to the foreign system.

There are several steps in this process:

  1. The gateway receives a message from the foreign system.
  2. The gateway translates the message into a MAPI message and delivers it to the server.
  3. The server delivers the message to the recipient.
  4. The server generates a report and delivers it to the gateway
  5. The gateway translates the report to the foreign system’s format and delivers it to the foreign system.
