About Agents

An agent maps a folder, one or more events, and the code (usually in a script) that processes those events. There are two types of agents:

  • A script, combined with the Microsoft Exchange Scripting Agent and a folder in which events are monitored
  • Custom agents

An agent is like a rule in that is has these characteristics:

  • It is tied to a specific folder.
  • It runs when a certain event triggers it, such as when a message with specific properties appears in that folder.
  • It has specific behavior.

Agents differ from rules in that, while any Microsoft Exchange recipient can set a rule, you need special folder permissions to set up agents. For information on creating agents, see Configuring Events and Actions.

About the Microsoft Exchange Scripting Agent

The Microsoft Exchange Scripting Agent is an agent that runs folder-resident scripts and provides CDO session, folder, and message objects to the script. It can work in conjunction with Microsoft Transaction Server. This is the default agent installed as part of Microsoft Exchange setup.
