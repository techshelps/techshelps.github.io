Microsoft Exchange Server to Gateway Report Transfer

The following steps are performed when Microsoft Exchange Server transfers a delivery or nondelivery report to the gateway:

  1. The value of the PR_CONTENT_CORRELATION property of the original message is returned with the report.
  2. A new MAPI message is created in the gateway’s MTS-OUT folder for the report, and a PR_SEARCH_KEY is created for it.
  3. The report’s PR_MTS_REPORT_ID, PR_PARENT_KEY, and PR_MTS_SUBJECT_ID properties are assigned the values of the PR_MTS_REPORT_ID, PR_CONTENT_CORRELATION, and PR_MTS_SUBJECT_ID from the Microsoft Exchange Server report, respectively.
  4. The gateway finds the original message by matching the report’s PR_MTS_SUBJECT_ID property with the PR_MTS_ID properties of its archived messages.
  5. The gateway assigns the PR_FOREIGN_SUBJECT_ID property’s value from the PR_FOREIGN_ID property in the original message.
