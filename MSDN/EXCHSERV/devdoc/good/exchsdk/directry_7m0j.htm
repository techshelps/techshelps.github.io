Rules for Importing Attributes

The BatchImport and DAPIWrite import functions follow these rules when using template objects for importing attributes:

  1. If an attribute value is specified explicitly in an import file or in the pAttributes parameter, this value is imported, even if it preempts user template data.
  2. If an attribute value is not specified in an import file or in pAttributes, but is specified in the template object, this value is imported unless it is a personal attribute.
  3. Personal attributes such as a recipient's Display-Name or e-mail address are not inherited from the template object. To determine which attributes are excluded, see the Duplicate User option of the Administrator program; it excludes the same attributes. Excluded personal attributes not given in an import file or in the pAttributes parameter are left blank in the destination object.
