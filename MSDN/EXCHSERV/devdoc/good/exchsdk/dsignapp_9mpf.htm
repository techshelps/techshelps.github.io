Troubleshooting the Primary Files

If you can’t find anything wrong with the primary file lines with error reports, trace errors with the following procedure:

    To check for errors in a primary file
  1. Make a backup copy of the original migration files.
  2. Edit the primary file to remove all lines with error reports.
  3. Import the edited files.

    Error reports can now be eliminated. However, if errors are still reported, the errors are in the previous lines. The errors in the previous lines did not stop the importing, but did cause problems in the next lines.

  4. Make a copy of the backup migration files.
  5. Edit the primary file, removing the lines previous to the lines with error reports.
  6. Import the edited files.

    The error reports can now be eliminated.

  7. If errors are still reported, repeat steps 4 through 7, removing lines previous to the lines removed in the last test.

Once you identify the problem lines, see how they differ from lines that were successfully imported, or how they differ from the proper migration file format.
