Using the ~SERVER Substitution String

You can use the ~SERVER string to specify the current server against which the import operation is being run. During an import operation, the DAPIWrite and BatchImport functions substitute the name of the actual home server for this string. You can use this string not only to import the correct current value for the Home-Server attribute; but in any part of the import string, such as the Comments attribute.

Do not place quotation marks around ~SERVER. This string can be specified in upper- or lower-case letters (~SERVER or ~server).
