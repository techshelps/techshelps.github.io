Agents Property Page

Folder owners use this property page to create a new agent. This page is also used for binding an agent to a server.

This property page is used to connect events, scripts, and servers. Collectively, these components are displayed as properties of an agent.

As with the Permissions page, this page is visible only to owners of the folder. When the folder owner clicks the Agents tab, the property page displays a list of agents, as shown in the illustration Agents Property Page (with Agents Displayed).

The Agents property page contains the following controls:

  • Agents for this folder This is a list box control that uses the check box style. It lists the current agent messages installed in this folder, displaying the property PR_DISPLAY_NAME. Double-clicking one of the items in the list box is the same as choosing the Edit button.
  • New Choosing New causes a new agent message to be created. See New Agent Dialog Box.
  • Copy Use Copy to copy agent messages between folders.
  • Edit Use the Edit button to modify the existing agent configuration data. See New Agent Dialog Box.
  • Delete Choosing Delete deletes the current agent messages. Deletion then requires user confirmation.
  • Run these agents on this server Use this control to bind an agent to a server. This is a drop-down list of servers on which the script should be run. It is populated from the list of folders that are under the NON_IPM public folder tree and whose name follows the format EVENTS_<server name>. One such folder exists for each server that has the Event Service installed. Each of these folders is created when the Event Service is installed on a particular computer.

    The server choice defaults to the home MDB server of the current folder, assuming that this server has the Event Service installed. If the home MDB server does not have the Event Service installed (and thus has no corresponding EVENTS_<server name> folder), the choice defaults to the first server on the list. The list is sorted alphabetically.

    The set of agents for a given folder must be guaranteed to run on only one server. This helps to ensure that the common set of ICS bookmarks for this folder (stored in the agent message in the folder) are manipulated by only a single server (ICS client).
