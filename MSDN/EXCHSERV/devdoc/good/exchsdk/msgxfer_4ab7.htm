Detecting Outgoing Messages

A gateway can determine when an outgoing message arrives in its MTS-OUT folder by polling the information store periodically or by waiting for an Advise event from the information store. If your gateway uses the Advise event, it is important to note that the information store may not immediately send an Advise to the gateway when it sends a message. This is because MAPI does not immediately flush the request for the Advise. The Advise request does not become active until the cache is flushed. If your gateway uses the Advise event, you can cause the any pending Advise events to be flushed by:

  • Performing a IMAPITable::QueryRows operation on a the MTS-OUT folder’s contents table.
  • Requesting a noncached property of the MTS-OUT folder.
