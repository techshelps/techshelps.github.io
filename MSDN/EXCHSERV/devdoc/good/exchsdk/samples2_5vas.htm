Installing a Mailbox Agent with MBINST

    To install a mailbox agent with MBINST
  1. To install the SampleMBA mailbox agent at the Microsoft Northwest site with the server EDKDOCS20, type the following at a command prompt:

    MBINST /SITEDN=/O=MICROSOFT /OU=NORTHWEST /DISPLAY_NAME=SMBAGENT /RELATIVE_DN=/CN=RECIPIENTS /CN=SAMPLEMBA /SERVER=EDKDOCS20
    /EXTNAME=SAMPLEMBA

  2. Use the SERVINST sample application to register the service so it can be managed through the Services dialog box of the Control Panel. See SERVINST: Installing a Service.
  3. Use the PROFINST sample application to install a profile that allows you to log on as the service and view the messages in the mailbox. See PROFINST: Installing a MAPI Profile.
  4. If the mailbox agent is to use an Administrator extension DLL, create a new directory and copy the DLL into the directory. Then use the EXTNINST sample application to install the DLL. See EXTNINST: Installing an Administrator Extension.
  5. To verify that you have correctly installed the mailbox agent, look at the Recipients container in the Administrator program. It should contain an entry for the mailbox agent.

This procedure creates a Mailbox-Agent object in the directory, and optionally assigns an Administrator extension DLL to it. A mailbox is created for the mailbox agent when a profile is created for it and it logs on for the first time.
