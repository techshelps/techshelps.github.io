Directory Access Function Files

By default, import and export files used with the directory access functions are comma-separated ASCII text files. These files can have any extension, but the Microsoft Exchange Server Administrator program expects files with the .EXP extension for use with the Directory Import command of the Tools menu, and it expects .CSV files for use with the Directory Export command of the Tools menu.

An import file contains the attributes of the objects being imported, as well as the data of those attributes. Any attribute associated with an imported object can be manipulated by the directory access import functions. Moreover, depending on the object being handled (and the mode of the import operation), certain attributes may be required to be imported. This section explains these import requirements.

Note  Characters used in a distinguished name (DN) or in any string used to build a DN (such as pszBasePoint and pszContainer) must be in the range 0x0020 to 0x007E.
