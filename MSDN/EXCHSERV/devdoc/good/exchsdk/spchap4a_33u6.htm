AlarmAmountAgain

The AlarmAmountAgain property specifies the default amount of time to wait when the user sets the snooze option.

Applies To

Appointment, Event, RecurringAppointment, RecurringEvent, RecurringTask, and Task objects

Data Type

Long

Access

Read/write

Remarks

The AlarmAmountAgain property is used in combination with the AlarmTypeUnit property to specify the default amount of time to wait when the user sets the snooze option. It causes an alarm to ring again after the specified time interval. Because users set this value in the Schedule+ user interface, most applications will use this property only to set a default value that appears in the appointment, event, and task Reminder dialog boxes.
