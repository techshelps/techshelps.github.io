Objects

You gain access to Microsoft Exchange Server through MAPI objects. Therefore, it’s important to understand how MAPI interfaces and properties are implemented by Microsoft Exchange Server MAPI providers.

In addition to supporting standard MAPI interfaces, Microsoft Exchange Server provides several interfaces of its own. These interfaces are generally accessed when a property (which can itself be opened as an object) is defined on an object and returns its own interface.

The two main categories of Microsoft Exchange Server objects are:

  • Directory objects
  • Information store objects

Note that the information provided in this section is not intended to take the place of the MAPI documentation. Rather, it explains how MAPI objects, interfaces, and properties are implemented by components of Microsoft Exchange Server. For complete information on MAPI objects, interfaces, and properties, see the MAPI Programmer’s Reference. For a discussion of Microsoft Exchange Server information stores, see About the Information Store and Information Store Operations.
