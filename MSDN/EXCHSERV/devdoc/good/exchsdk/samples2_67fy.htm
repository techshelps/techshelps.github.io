FIMPORT: Importing Folder Information

The FIMPORT sample application demonstrates how to import information into one or more Microsoft Exchange Server folders using bulk import methods. It can be used to:

  • Import bulk data into one or more folders on Microsoft Exchange Server.
  • Create folder content for testing and general use.
  • Provide a framework for building other folder import tools.
  • Demonstrate how to create and use of named properties on messages.
  • Demonstrate how to use the MS_EXCHANGE_01 property on messages. For more information, see Using the MS_EXCHANGE_01 Property.

FIMPORT creates folder content from an external .CSV file that contains a column for each property to be set on the messages in the folder. Each line in the file corresponds to a message to be changed. The file can be used to append, replace, or update the contents of the folder. FIMPORT opens the destination folder in the information store and updates each message. A progress display is updated after each item.

Two sample input files are included with the FIMPORT sample application. The FIMPORT1.CSV file shows how FIMPORT can be used to update any kind of information in a public folder, in this case the price and availability of auto parts. The FIMPORT2.CSV file shows how FIMPORT can be used to update messages in a public or private folder.

    To build FIMPORT
  1. Change to the directory containing the FIMPORT source files:

    \BKOFFICE\SAMPLES\EXCHANGE\FIMPORT

  2. Type NMAKE.
