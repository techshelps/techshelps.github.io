Creating Event Objects

The WS2_32.DLLprovides facilities for event object creation to both applications and service providers, although in most instances event objects will be created by applications. Event object services are made available to Windows Sockets service providers via WPUCreateEvent simply as a convenience mechanism for any internal processing that may benefit from same. Note that the event object handle is only valid in the context of the calling process. In Win32 environments the realization of event objects is via the native event services provided by the operating system.
