Microsoft SNA Server for Windows NT—SNA Connectivity Made Easy

Microsoft Corporation

Created: February 2, 1993

Abstract

This article explains how Microsoft® SNA Server for Windows NT® combines client-server architecture with the power and productivity of Windows® to deliver advanced connectivity between LAN workgroups and SNA networks.

Overview

Microsoft SNA Server for Windows NT makes enterprise-wide connectivity easy and economical. SNA Server offers PC users reliable and secure access to IBM host-computing with the familiar user interface of desktop PCs and the flexibility of modern LAN systems.

By employing advanced client-server architecture to distribute the communications processing, SNA Server maximizes the power of your host computers and desktop PCs. Each PC uses the LAN to connect to one or more servers using standard LAN protocols. The servers then provide shared links to host computers using SNA protocols and carry the heaviest workload, thus reducing the storage and processing requirements on desktop PCs. SNA Server offers advanced tools for easy system setup and use, regardless of the client PC operating system, LAN operating system, host connection, or host type.

SNA Server makes access to corporate and government data immersed within today's IBM SNA networks easier than ever before.

SNA Server provides flexible SNA connectivity between LANs and IBM host computers. SNA Server can support multiple concurrent SDLC, X.25 (QLLC), DFT, and 802.2 connections between the server and IBM host systems.

*Macintosh is supported as a “downstream physical unit” through the SNA Server.

Complete IBM SNA connectivity

  • SNA Server is a key part of Microsoft’s client-server product family that connects products such as Microsoft Mail and SQL Server to IBM host mail and database systems.

  • Delivers full implementation of WOSA SNA APIs for the Windows and Windows NT operating systems.

  • Supports popular 3270 and 5250 emulators for Windows, MS-DOS®, Windows NT, OS/2, and Macintosh® operating systems from ISVs such as Computer Logics, DCA, Eicon Technology, FutureSoft, NSA, and Wall Data.

  • Provides each user with multiple 3270 and 5250 sessions for concurrent terminal and printer emulation, including file transfer and EHLLAPI applications.

  • Supports APPC, CPI-C, CSV, and LUA APIs for advanced SNA applications in each client environment.

Advanced Client-Server Architecture

  • The SNA Server(s) off-loads communications processing from clients, allowing efficient use of client memory and server processing power.

  • Provides excellent SNA session security and fault tolerance.

  • Supports client PCs across LAN and WAN bridges, routers, and Microsoft’s Windows NT Remote Access Service.

  • Automatic load balancing distributes users evenly across multiple host connections and servers for optimum throughput.

  • Supports the following LAN protocols: NetBEUI, TCP/IP, IPX/SPX, and 802.2 (for DSPU clients).

Comprehensive Support for Distributed SNA Applications

  • APPC: For developing applications that communicate peer-to-peer with other APPC applications.

  • CPI-C: For developing applications that communicate peer-to-peer with IBM SAA™ applications.

  • CSV: For developing applications that enable tracing of API calls and communicate with NetView.

  • LUA: For developing applications (using LUA/RUI or LUA/SLI APIs) that need direct access to LU 0, 1, 2, and 3 data streams.

  • EHLLAPI: For developing applications that interface with existing 3270 or 5250 applications (EHLLAPI offered by ISV emulators).

Flexible Host Connectivity

  • Allows LU pooling for flexible host access, enabling an authorized user to use any server transparently.

  • Supports any combination of LU, PU, and data link protocols concurrently.

  • Supports downstream SNA nodes as secondary PUs (DSPU).

  • Supports (via the SNADIS interface) a wide variety of SNA communications adapters from IBM, DCA, Eicon, Atlantis, and others.

Intuitive Setup and Administration

  • Allows remote administration across bridges and routers, through Remote Access Service, and from NetView.

  • Includes an easy-to-use graphical administration tool for configuring SNA parameters.

  • Supports monitoring and controlling of all SNA user terminals, connections, and servers on the network from the same tool.

  • Supports comprehensive tracing and self-diagnostics of APIs, SNA protocols, and connections.

Integration with Windows NT

  • Shares the scalability, performance, reliability, and flexibility of Windows NT.

  • Uses the C2-level security of Windows NT, making connectivity to corporate data safe.

  • Benefits from the Windows NT SMP architecture through a multithreaded implementation.

  • Supports the same hardware platforms that Windows NT does (currently, Intel; later, MIPS and Alpha).

Full WOSA Compliance

  • Microsoft’s Windows Open Services Architecture (WOSA) provides a consistent interface to enterprise computing environments, thus hiding the complexities of connectivity from applications and application developers.

  • All the SNA Server’s APIs (APPC, CPI-C, LUA, and CSV) are 100-percent compatible with WOSA’s SNA API standard, endorsed by 20 leading SNA providers, including IBM.

SNA Server provides flexible SNA host connectivity for LAN-connected PCs. For example, an executive of a major U.S. company can work on an important proposal from her hotel room in Boston using sales data from the Finance SQL Server in New York, and production capacities from the Manufacturing AS/400 in Dallas, then process the profit-margin analysis in the Corporate ES/9000 also in Dallas. She can then pull the results to her notebook running Windows and be ready to close the deal next day. She knows that the functionality and the operating environment are exactly the same whether in the office or on the road. The fact that the physical connection goes through Remote Access Service over TCP/IP LAN and WAN, and across a Token-Ring LAN to the IBM host systems does not concern her. The system is fully transparent—to her it is just a click of an icon: “Information at your fingertips”.

3270 display emulation provides PCs running Windows with convenient access to your mission-critical host applications.

Network administrators can monitor the status of multiple servers using the SNA Server’s administration tool, or even “zoom in” on selected connections to view active users, such as those using a Token-Ring connection to the host.

Technical Features

Advanced Server Functionality

  • Supports up to 500 LU sessions per server.

  • Supports up to 200 users per server.

  • Supports up to 50 simultaneous host connections per server.

  • Supports up to 50 downstream PUs per server.

  • Provides automatic server and connection fault tolerance.

  • Supported PU and LU protocols:
    • PU 2.0, PU 2.1, LEN, and DSPU

    • LU 0, 1, 2, 3, and LU 6.2.
  • Supported data-link protocols:
    • 802.2: Token-Ring or Ethernet connections via IBM 37xx front-end processor or 3x74 cluster controller.

    • SDLC: Leased or switched telephone line connections.

    • X.25/QLLC: Public or private packet-switched X.25 connections.

    • DFT: Coax or twisted-pair connections via IBM 3x74 cluster controller.

Comprehensive NetView support

  • API support for bi-directional communications with NetView.

  • Notifies NetView of communications problems through automatic data-link alerts.

  • Supports RTM and user-defined alerts for ISV 3270 emulators.

  • Sends application-defined Windows NT event-log messages to NetView.

  • Supports execution of Windows NT commands from the NetView console.

Applications can take advantage of the EHLLAPI functionality provided by third-party 3270 or 5250 emulators that are supported by SNA Server’s 3270 EIS, or APPC APIs. Advanced LU 6.2 applications can use either CPI-C APIs or APPC APIs. LUA applications can use either LUA/RUI or LUA/SLI APIs. Diagnostic programs can use CSV APIs for API tracing and for NetView connectivity. The open SNA adapter support is provided through the SNADIS interface.
