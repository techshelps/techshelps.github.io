Add Item Dialog Box

   

Use the Add Item dialog box to add machines, event logs, and filters to your Visual Studio Analyzer project (the New tab) or to open an event log or filter in an existing Visual Studio Analyzer project (the Existing tab).

Options

Event Log

Creates a new empty event log to contain Visual Studio Analyzer events. By default, this new event log is set to record events next time you run your application. Make sure you activate a filter as well; the filter determines what events are collected in the event log.

Filter

Creates a new filter to identify Visual Studio Analyzer events to collect, and opens the filter in the Filter Editor. If you want to use this new filter to determine which events are collected the next time you run your application, make sure to activate the filter by clicking Set Record Filter on the shortcut menu.

The predefined filters shipped with Visual Studio Analyzer also appear in the Add Item dialog box. Click any of the predefined filters to add them to your project; you can use the Filter Editor dialog box to modify them to suit your purposes. The predefined filters are shown in the following list.

  • Advanced Events

  • Basic Events

  • Intermediate Events

  • All Events Except Performance

  • Basic Performance Events

  • Basic COM (and MTS) Events

  • Basic Database Events

  • Basic Object Events

  • Basic Web Events

Machine

Displays the Connect to Machine dialog box, where you can specify the name of a machine you want to monitor for Visual Studio Analyzer events. Make sure that the Visual Studio Analyzer Server components are installed on any Windows machine you want to monitor.

Sample

Adds the sample event log to your project. This event log is shipped with Visual Studio Analyzer to provide a way for you to familiarize yourself with Visual Studio Analyzer's features without connecting to other machines or recording any data.
