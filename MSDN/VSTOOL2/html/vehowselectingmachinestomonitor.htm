Connecting to Machines for Monitoring with Visual Studio Analyzer

You can identify the machines you want to monitor with Visual Studio Analyzer. References to these machines are saved as part of your Visual Studio Analyzer project. Make sure the Visual Studio Analyzer Server components are installed on any machine you plan to monitor. Installing the Visual Studio Analyzer Server components is part of the Visual Studio setup. On Windows 95 you need to start the server components manually on any remote machine from which you plan to collect data. See Starting the Visual Studio Analyzer Server on Windows 95 for instructions on how to start and stop the Visual Studio Analyzer Server components.

To add machines to a Visual Studio Analyzer project

  1. In the Project Explorer, select the Visual Studio Analyzer project.

  2. Right-click to display the shortcut menu, and then click Connect to Machine.

    You can also click the Connect to Machine button on the Visual Studio Analyzer toolbar.

  3. Type the name of the machine you want to connect to, or click Browse to navigate to a list of available machines and select the machine you want.

    Note   You can type or omit the two backslashes (\\) from computer names. You can also use IP addresses (123.4.56.7) to refer to machines on your network if appropriate. However, you cannot select the My Computer node to refer to your local machine.

  4. Click Connect.
