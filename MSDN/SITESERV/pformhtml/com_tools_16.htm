Setting Stage Properties in Expert Mode

Using the Win32-based Pipeline Editor in Expert mode, you can set the following properties of a pipeline stage:

  • υGUID. The globally unique identifier is a number that uniquely identifies this stage.
  • υMode. The Mode field is provided for compatibility with Commerce Server 2.0 pipelines, in which the number identified which stages of a pipeline should be run by the RunProduct, RunPlan, and RunPurchase methods of the Page object.
  • υError level. When any component in the stage generates an error at or above the level specified in the Error level box, the stage stops running and generates an error.
Caution

In standard mode, the Pipeline Editor has built-in safeguards that prevent some actions that could disable a site’s pipeline. In Expert mode, these protections do not exist. It is recommended that you do not run the Pipeline Editor in Expert mode unless you specifically need the additional functionality it provides.

To set stage properties in Expert mode
  1. Start the Win32 Pipeline Editor in Expert mode, and then open the pipeline whose stage you want to edit.
  2. Right-click the stage, and then, on the shortcut menu, click Properties.
  3. For GUID, type a unique identifier for this stage.
  4. If this is a Commerce Server 2.0 pipeline, type a value for Mode.
  5. For Error level, type a number that represents the error level at which you want to stage to stop execution.
  6. Save the pipeline and close the Pipeline Editor.
