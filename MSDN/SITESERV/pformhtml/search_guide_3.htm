Configuring the Catalog Build Server

The Catalog Build Server, obtained by the BuildServer property, serves two purposes:

Obtains a catalog definition collection for adding, removing, and configuring catalog definitions. This collection is obtained by the BuildCatalogs property.

  1. Provides the following configuration options, such as:
    • Specifes crawl settings such as sites that have hit frequency restrictions, the user agent used for crawling, timeout values, the default content access account name for crawling other hosts, and the email address to log while crawling.

    • Specifies default settings for the access account and other administrative settings.

    • Specifies locations for catalog defintions, logs, and temporary files.

    • Saves and load catalog definitions and server configurations.

    • Sets the server’s performance level.

    • Specifies proxy server addresses and settings.
