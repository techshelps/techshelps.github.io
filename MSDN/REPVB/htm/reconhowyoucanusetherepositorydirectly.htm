Using Repository Directly

Because Repository is implemented as a set of Automation objects, you can directly manipulate its contents from Visual Basic programs. You manipulate a repository directly to perform the following:

  • Create a new information model.

  • Extend an existing information model.

  • Retrieve information from any information model.

  • Store information using any information model other than the MDO Model.

  • Store information using any extensions of the MDO Model.

Note The Repository Add-in for Visual Basic populates the MDO Model with information about your Visual Basic projects. Do not store information directly in the MDO Model unless you extend it.
