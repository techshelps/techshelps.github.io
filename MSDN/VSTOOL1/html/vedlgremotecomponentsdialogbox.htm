Remote Components (Setup Properties Dialog Box)

   

Use this dialog box to make your setup distribution unit enter the appropriate registry entries in the target computer to enable applications to use remote objects through DCOM.

The distributed Component Object Model (DCOM) is a protocol that enables software components to communicate directly over a network in a reliable, secure, and efficient manner.

Options

Always prompt user for server

Specifies that your component's user will be asked during the installation which remote server the remote DCOM objects are running on. If you added a remote server to the Use Remote Server box, that server will be supplied as a default.

Remote Components

Displays a list of the remote DCOM files that you can select to include in the setup distribution unit. When the setup distribution unit is run on the target computer, appropriate entries are made in the registry of that target computer to enable that computer to use the remote objects through DCOM.

Use Remote Server

Provides a place to enter or edit your remote server computer's network name. The remote server is the computer that will run the remote component.
