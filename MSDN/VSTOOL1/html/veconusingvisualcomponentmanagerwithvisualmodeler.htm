Using Visual Component Manager with Visual Modeler

With Visual Component Manager, you can publish models created in Visual Modeler, and import and export these models to and from your Visual Basic project.

To import a class or logical package from the Visual Component Manager to a Visual Modeler model

  1. In Visual Modeler, open the model to which you want to import the class or package.

  2. In Microsoft Visual Basic, open the Microsoft Visual Component Manager as described in Installing and Starting Visual Component Manager, and click on the Visual Models folder. The published models are displayed in the contents pane.

  3. Select the model you want to import.

  4. On the shortcut menu, click Browse. A hierarchical list showing the logical packages, associations, and classes of the selected model is displayed in the properties pane.

  5. Expand the browser list to display the class or logical package you want to import.

  6. Drag the class or package from the list and drop it on an open class diagram or on a logical package in Visual Modeler.

Visual Modeler imports the selected class or logical package into the logical view of the current open model.

Publishing a Visual Modeler Model to the Microsoft Repository

To publish the entire logical view of the current open model to the Visual Component Manager

  1. Save the model.

  2. On the Tools menu, click Publish to VCM.

For more information about using Visual Component Manager with the Visual Modeler, search for Visual Component Manager in online Help for Visual Modeler.
