Nested Setups (Setup Properties Dialog Box)

   

Use this dialog box to include any self-installing setup file you want to include in a setup file for execution when the setup is run on the target computer.

Simply adding these files to the setup file with the Additional Files feature will not enable the libraries to run and install on the target computer as part of the setup.

For example, the Microsoft® Visual Basic® Setup Wizard creates a setup file for any ActiveX controls authored in Visual Basic. This setup can be included in a setup file as a nested setup (a setup file inside of a setup file). When the entire nested setup is run on the target computer, the nested setups run, with the innermost setup file running first.

If these nested setup files are deployed, there are no HTML reference link updates to any but the topmost setup file.

Options

Select the programs to execute when the installer is run

Displays a list of the files you can select to nest inside your current setup distribution unit. When the entire nested setup distribution unit is run on the target computer, the innermost setup files are extracted first, and then the outermost setup distribution unit.
