Specifying External Dependencies for a Cabinet (.cab) Distribution Unit

Cabinet files contain links to other cabinet files on the intranet or Internet. You can publish reusable components, such as system libraries, to the intranet or Internet, and then reference the reusable components in your cabinet files when needed.

When you specify that one cabinet file has a dependency on another cabinet file, the contents of the first cabinet will only be run once the contents of both cabinets are installed on the customer's computer. This is a feature of Internet Explorer code download.

You can add references to any self-installing cabinet file by specifying the URL, friendly name, and version of the cabinet file.

To specify an external dependency file for a cabinet distribution unit

  1. Display the property pages of the distribution unit.

  2. Click the External Dependencies tab.

  3. In the External Dependencies dialog box, click Add to place a new entry in the Dependencies on the Web list. Specify the URL, friendly name, and version of the required cabinet file. Click OK.

  4. Click Edit to change any entries in the Dependencies on the Web list.

  5. Click Delete to remove any entries in the Dependencies on the Web list.

  6. Click OK.
