Deployment Target (Deployment Target Settings Dialog Box)

   

Use this dialog box to change the deployment target URL, verify contact with (or "ping") the network server, and enable or disable automatic modification of HTML code within a deployed file to reference distribution units (cabinet or .zip files) created in the Microsoft development environment.

If you need to get new information on a network server, such as a newly installed server component capability, you can click OK on this dialog box, and the development environment will contact that server for fresh information on the server status and supported services. Click Cancel to exit this dialog box without contacting the network server.

Options

Deploy to (URL)

Changes the URL of the target Web server for the current deployment target. You can change the protocol to HTTP or HTTPS, depending on what your network server supports. You can enter or edit the URL for the network server.

For More Information   For step-by-step instructions on setting up your network server, see Configuring an IIS Server to Be a Deployment Target.

Modify deployed Web pages so that applet and object tags reference deployed project outputs

Enables automatic modification of Web pages to reference distribution units (cabinet or .zip files) created in the development environment. This check box is selected by default, so that when a file containing HTML references is deployed, this feature will update any APPLET and OBJECT tags in HTML that reference components that are packaged into distribution units.

For More Information   For more information on this link updating, see What is Deployment?.

You can disable this feature so that the HTML will not be modified when it is deployed, such as when you want to update the HTML code in your project by hand, or you are certain that your HTML code does not need updating.
