Component Attributes

   

In order to help organize components so that you can easily locate and reuse them, Visual Component Manager maintains attribute information about each component stored in its repository database. These attributes are displayed in the properties pane of Visual Component Manager window.

Attribute Description
Component Name The name by which the component is recognized.
Type The component type, as described in the topic Repositories, Folders, and Items.
Description A brief description of the component's functionality, usage, interface(s) and any other useful information about the component.
Keywords Words or phrases used to identify the component for searching.
File Name The names of all files included with the component.
Author The name of the component's author.
Date Published Date the component was first added to Visual Component Manager.
Usage Count Number of times the component has been reused from the current repository.
Has Source Code True if source code is included, otherwise False.
Has Documentation True if textual documentation is included, otherwise False.
Has Help File True if online Help file(s) are included, otherwise False.
Has Sample Code True if sample False.

Note that you enter most of these properties when you publish the component, by supplying information in the Publish Wizard. You can modify them by right-clicking the component in the contents pane and clicking Properties on the shortcut menu.

When you publish a component, you should be careful to enter complete and well-organized information in its attributes, since this information will be used by others to locate the components they need.

For More Information   For more information about using component attributes to organize and search for published components, see Finding Components.
