Installing Shared Files in the System Folder

Sometimes programs share the same files or components. If all the shared components are stored in the same place, uninstalling one application won't uninstall a file needed by another application.

With the setup distribution unit, you can specify that shared files be installed to the Windows System folder.

If your project contains Visual J++ elements, build the project first, so that all project outputs are identified. Adding additional files will increase the file list in the System Directory Installation tab of the Setup Project Properties dialog box.

To install a shared file into the Windows System folder when the setup file extracts

  1. Display the property pages of a distribution unit.

  2. Click the System Directory Installation tab.

  3. In the System Directory Installation (Setup Project Properties) dialog box, select the files in the list that you want to install to the Windows System folder of the target machine (after the setup executable extracts).
