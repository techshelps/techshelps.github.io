Server Component Installation (Deployment Target Settings Dialog Box)

   

Use this dialog box to change the default directory/folder for the deployment target, Server Component Installation service node. This is the location where all the server-side components are installed and registered on the network server machine when the project is deployed.

Note   Microsoft® Visual InterDev™ projects are arranged hierarchically, which places the server-side components in many different folders when deployed. The Deploy to Folder feature in the Microsoft development environment places all the server-side components in one folder instead. If the server-side components are not in the same directory tree as the content (on the network server you deploy to), you can't get state information on the components for debugging.

Options

Deploy to Folder

This is the location on the network server machine where all the server-side components are installed and registered. You can change this folder from the default listed here. If necessary, a new folder will be created on the network server.

Note   You are not notified if the component fails to register on the server.
