Using an Accelerator-Table Resource

The most common way to add accelerator support to a Win32-based application is to include an accelerator-table resource with the application's executable file and then load the resource at run time. The steps involved in using an accelerator-table resource are as follows:

  • Create an accelerator table in a resource-definition file. Compile it and add the resulting resource to your application's executable file.
  • Include a call to the LoadAccelerators function to load the accelerator-table resource and receive a handle to the accelerator table.
  • Add the TranslateAccelerator function to the message loop associated with the accelerator table.
  • Process the WM_COMMAND messages generated when the user uses the accelerator.
