Grouping and Sorting Dialog Box (Object Browser)

   

Allows you to group or sort members and classes by type or access in the Object Browser. By default, no grouping is applied.

Options

Classes

Group By List

Allows you to select the way you want to group the classes for sorting

  • <No Grouping>   Creates no group or, if you already have a grouping, removes it.

  • Group by Class Access   Arranges classes into groups based on their access type. Changing the order of values in the Value list changes the order of the groups in the view.

  • Group by Class Type   Arranges classes into groups based on their type. Changing the order of values in the Value list changes the order of the groups in the view.

Values

Displays the development-specific values for the selection in the Group By list.

Members

Group by List

Allows you to select how you want to group the members for sorting

  • <No Grouping>   Creates no group or, if you already have a grouping, removes it.

  • Group by Member Access   Creates a group based on the member access selected in the Values list.

  • Group by Member Type   Creates a group based on member type selected in the Values list.

Values

Displays the development-specific values for the selection in the Group By list.

Moves the selected item up one entry in the list each time you click it.

Moves the selected item down one entry in the list each time you click it.

Group with Headers in Main List

When selected, displays the sorted classes or members in a tree structure. The list displays only values relevant to the packages and libraries you are browsing. For example, if no COM libraries are currently selected in the browsed set, no COM class and member types and access types appear in the value lists.

Apply

Applies the settings to the current data in the Object Browser.
