Common Network Interface Properties

Common network interface properties are data values that describe all network interfaces regardless of their type and are static values stored in the cluster database. Common properties can be read-only or read/write. To access common resource properties, applications typically call ClusterNetInterfaceControl with one of the following values for the dwControlCode parameter:

  • CLUSCTL_NETINTERFACE_ENUM_COMMON_PROPERTIES to retrieve the names of the common network interface properties.
  • CLUSCTL_NETINTERFACE_GET_COMMON_PROPERTIES to retrieve the names and values of common network interface properties that can be modified.
  • CLUSCTL_NETINTERFACE_GET_RO_COMMON_PROPERTIES to retrieve the names and values of common network interface properties that cannot be modified.
  • CLUSCTL_NETINTERFACE_SET_COMMON_PROPERTIES to update the values of common network interface properties.
  • CLUSCTL_NETINTERFACE_VALIDATE_COMMON_PROPERTIES to validate common network interface properties.

For the get, set, and validate operations, ClusterNetInterfaceControl either requires as input or returns as output a property list that contains an entry for every property. Each entry has two parts: a property name and a property value. The property name contains a syntax value, a string describing the name, the length of the string, and padding if necessary to properly align the name. The property value is similar to the name in that it also contains a syntax value and a length. The data portion varies depending on the type of property. For most properties, a cluster structure such as CLUSPROP_SZ or CLUSPROP_DWORD is used to store the property value in the property list.

The Quick Info tables that appear with each property describe the following pieces of information:

  • Property type.
  • Property access level.
  • Functions that can be called to access the property such as ClusterNetInterfaceControl.
  • Structures that are used as input to or output from a property operation.
  • Control codes that can be passed to ClusterNetInterfaceControl to retrieve the property.
