Common Network Properties

Common network properties are data values that describe all networks and are values stored in the cluster database. Common properties can be read-only or read/write. To access common network properties, applications call ClusterNetworkControl with one of the following values for the dwControlCode parameter:

  • CLUSCTL_NETWORK_ENUM_COMMON_PROPERTIES to retrieve the names of the common network properties.
  • CLUSCTL_NETWORK_GET_COMMON_PROPERTIES to retrieve the names and values of common network properties that can be modified.
  • CLUSCTL_NETWORK_GET_RO_COMMON_PROPERTIES to retrieve the names and values of common network properties that cannot be modified.
  • CLUSCTL_NETWORK_SET_COMMON_PROPERTIES to update the values of common network properties.
  • CLUSCTL_NETWORK_VALIDATE_COMMON_PROPERTIES to validate common network properties.

For the get, set, and validate operations, ClusterNetworkControl either requires as input or returns as output a property list that contains an entry for every property. Each entry has two parts: a property name and a property value. The property name contains a syntax value, a string describing the name, the length of the string, and padding if necessary to properly align the name. The property value is similar to the name in that it also contains a syntax value and a length. The data portion varies depending on the type of property. For most properties, a cluster structure such as CLUSPROP_SZ or CLUSPROP_DWORD is used to store the property value in the property list.

The Quick Info tables that appear with each property describe the following pieces of information:

  • Property type.
  • Property access level.
  • Functions that can be called to access the property such as ClusterNetworkControl.
  • Structures that are used as input to or output from a property operation.
  • Control codes that can be passed to ClusterNetworkControl to retrieve the property.
