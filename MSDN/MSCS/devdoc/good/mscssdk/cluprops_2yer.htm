Common Resource Type Properties

Common resource type properties are data values that describe all resource types. They are static values stored in the cluster database. To access the common resource type properties, applications typically call ClusterResourceTypeControl with one of the following values for the dwControlCode parameter:

  • CLUSCTL_RESOURCE_TYPE_ENUM_COMMON_PROPERTIES to retrieve the names of the common resource type properties.
  • CLUSCTL_RESOURCE_TYPE_GET_COMMON_PROPERTIES to retrieve the names and values of common resource type properties that can be modified.
  • CLUSCTL_RESOURCE_TYPE_GET_RO_COMMON_PROPERTIES to retrieve the names and values of common resource type properties that cannot be modified.
  • CLUSCTL_RESOURCE_TYPE_SET_COMMON_PROPERTIES to update the values of common resource type properties.
  • CLUSCTL_RESOURCE_TYPE_VALIDATE_COMMON_PROPERTIES to validate common resource type properties.

For the get, set, and validate operations, ClusterResourceTypeControl either requires as input or returns as output a property list that contains an entry for every property. Each entry has two parts: a property name and a property value. The property name contains a syntax value, a string describing the name, the length of the string, and padding if necessary to properly align the name. The property value is similar to the name in that it also contains a syntax value and a length. The data portion varies depending on the type of property. For most properties, a cluster structure such as CLUSPROP_SZ or CLUSPROP_DWORD is used to store the property value in the property list.

The Quick Info tables that appear with each property describe the following pieces of information:

  • Property type.
  • Property access level.
  • Functions that can be called to access the property such as ClusterResourceTypeControl.
  • Structures that are used as input to or output from a property operation.
  • Control codes that can be passed to ClusterResourceTypeControl to retrieve the property.
