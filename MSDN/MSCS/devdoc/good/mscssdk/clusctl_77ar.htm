CLUSCTL_RESOURCE_TYPE_VALIDATE_PRIVATE_PROPERTIES

The CLUSCTL_RESOURCE_TYPE_VALIDATE_PRIVATE_PROPERTIES control code is used to check whether or not specific properties are valid private properties for a resource type.

Quick Info

Type: External
Access: CLUS_ACCESS_READ
Input: Property list
Output: None
Global: CLUS_NOT_GLOBAL
Modify: CLUS_NO_MODIFY
Code: CLCTL_VALIDATE_PRIVATE_PROPERTIES

Notes to Callers

When you call ClusterResourceTypeControl with the CLUSCTL_RESOURCE_TYPE_VALIDATE_PRIVATE_PROPERTIES control code, pass a property list with the names and values for the private read/write properties to validate. Private properties can include:

  • Dynamic values generated as needed.
  • Static values stored in the cluster database.
  • Static values not stored in the cluster database.

To build a property list, use the property list utility functions.

Notes to Resource DLL Writers

Resource DLLs should handle the CLUSCTL_RESOURCE_TYPE_VALIDATE_PRIVATE_PROPERTIES control code in their ResourceTypeControl entry point functions. They should check that the properties specified in the InBuffer parameter are valid for the resource.

If you cannot support this control code, the Resource Monitor validates that the property list is formatted correctly. It does not actually validate the data.
