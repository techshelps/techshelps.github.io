ODL File Ignored

Dear Dr. GUI:

For some reason ClassWizard refuses to put anything in my .ODL file. After I created the project I added seven or so methods and properties, and they are in my .ODL file. However, when I try to add any new methods or properties, the .ODL file is ignored and the "Dispatch and event IDs" section in my include file is empty: 

// Dispatch and event IDs
public:
   enum {
   //{{AFX_DISP_ID(CTickCtrl)
   //}}AFX_DISP_ID

It looks as if somehow ClassWizard "forgot" that I wanted a typelib. What can I do to make it "remember" again?

Dr. GUI replies:

Here's what you can do.

1. Manually add the appropriate DISP_IDs for methods and properties after the enum keyword.

2. Exit Visual C++ 2.0 and delete the .CLW file.

3. Start Visual C++ 2.0 again, and regenerate the .CLW file by opening the .RC file and pressing ctrl+w.

4. Exit Visual C++ 2.0 again, and open the .CLW file in Notepad. Add the following line to the [General Info] section:

ODLFile=foo.odl

5. Load the project, and it works!
