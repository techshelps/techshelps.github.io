Dynamic, Resource-Free Dialogs

Dear Dr. GUI:

I am building an application that requires dynamically configured dialog boxes. I am developing under the Windows API (as opposed to the Foundation classes), and I have not found any mechanism for creating such dynamic dialog boxes "on the fly." My current implementation uses CreateWindow to build the window, but with that approach I don't get the default dialog processing. (This approach also produces several other problems and some nonconforming GUI behaviors.)

Is there an easy way to create arbitrary modal and nonmodal dialog boxes dynamically without using resources?

Bob Kuczewski

Dr. GUI replies:

You can use the CreateDialogIndirect function to create dialog boxes dynamically. This function uses the CreateWindowEx function to create a modeless dialog box from a dialog template in memory. CreateDialogIndirect then sends a WM_INITDIALOG message (and a WM_SETFONT message if the template specifies the DS_SETFONT style) to the dialog box procedure. The function displays the dialog box if the template specifies the WS_VISIBLE style. Finally, CreateDialogIndirect returns the window handle of the dialog box.

After CreateDialogIndirect returns, the application displays the dialog box (if it is not already displayed) by using the ShowWindow function. The application destroys the dialog box by using the DestroyWindow function.
