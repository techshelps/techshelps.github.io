Windows NT 3.51 Has International Flair (and Fonts)

Lori Hoerth

September 1, 1995

Windows NT 3.51 added support for the PowerPC hardware platform as well as a number of other features, including some very important international functionality. Combined with the functionality that already existed in Windows NT 3.5, this new international functionality gives users the ability to run localized applications on the English product—for example, the Greek version of Microsoft Excel on the U.S. version of Windows NT.

Windows NT 3.51 provides four main new features important to international customers:

Locale-sensitive fonts: When the language and country are set in the International application in Control Panel, all fonts—including console fonts, system fonts, and application fonts—are reset to match the newly set locale. This allows an English-language system to run the Russian version of Windows 3.x applications and display all characters correctly when the locale is set to Russian.

Keyboard switching application: Microsoft now ships, as part of every version of Windows NT 3.51, an application that allows for quick switches of the systemwide active keyboard layout between a primary and a secondary layout. The keyboard switching application can be activated by running KBDSEL.EXE, as shown in Figure 1.

Large content application fonts: The Arial, Times New Roman, and Courier New fonts that ship with Windows NT 3.51 are now multicharacter set fonts. These fonts contain glyphs for all European languages (including Greek, Turkish, and English). These same fonts ship as part of the multilingual support on Windows 95. They give users of Windows NT 3.51 more choices of fonts they can use, while still being able to see multiple scripts within the Windows NT shell (that is, File Manager), as shown in Figure 2.

Figure 2.

"Tagged font" mapping: Windows NT 3.51 has added font mappings to enable document compatibility with existing Greek, Central European, and Eastern European documents that have used tagged font face names in the localized Windows 3.x environment. For example, an existing Microsoft Word document that was created on the Greek version of Windows 3.1, using the Greek version of Word 6.0, contains Greek text that was created using a font named Arial Greek. This document is displayable on a Windows NT system, because Windows NT maps Arial Greek to the Greek portion of the multicharacter set font named Arial, thus being able to display Arial Greek using the Arial font that contains Greek glyphs.

The next release of Windows NT, a shell update planned for beta release this fall, will contain all of the new multilingual functionality in Windows 95, as well as a number of enhancements to allow simpler creation of multilingual Unicode applications. A subsequent version of Windows NT, codenamed "Cairo," will increase the number of international- and language-based features in Windows NT by adding content indexing and language-based queries.

Lori Hoerth is a senior program manager for the Windows NT team in Microsoft's Business Systems Division. Her areas of responsibility include international features and functionality, NLS APIs, multilingual computing, Unicode issues and implementation, and character encodings and mappings.
