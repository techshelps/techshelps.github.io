Accessing Data With ASP

ASP with IIS offer the Web developer flexible, easy to use, scaleable methods to interact with ODBC compliant databases for Internet or intranet applications. Using ASP and the Microsoft Data Access Components (MDAC) as part of your development not only means that you can initially develop a solution using Microsoft Access and scale up to Microsoft SQL Server 6.5, but that you can access any other databases that are ODBC compliant. With the additional power of Microsoft Transaction Server (MTS), you can create highly scalable multi-tier applications that support Web-based transactions.
