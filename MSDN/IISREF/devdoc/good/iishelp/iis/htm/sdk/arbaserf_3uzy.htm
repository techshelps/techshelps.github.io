MD_LEVELS_TO_SCAN

This identifier specifies the number of virtual directory levels to scan in the metabase from the top of the tree (i.e., the ROOT virtual directory for a server) to build a list of directories to be monitored for file system changes. If a virtual directory is not included in the list then the IIS cache will not be flushed when a file change occurs in the directory. This list is also used for downlevel administration from previous versions (1.0 to 3.0) of IIS.

Attributes

Data type DWORD
Default inheritance Not inheritable
User type IIS_MD_UT_SERVER

Values

Value Range
0 – 0xFFFFFFFF
Default Value
2

Metabase Keys

This identifier is available at the following metabase keys:

Metabase Path Key Type
/LM/MSFTPSVC IIsFtpService
/LM/W3SVC IIsWebService
