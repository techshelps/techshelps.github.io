Testing Online without a BDS connection

Follow the following instructions to configure MS Money for Online Testing (without requiring a connection to the Branding Directory Server).

-------------------------------------

To test online stuff without going to the branding server, you have to (a) tell the code that it's not going online, and (b) provide all of the information locally. The former is accomplished by setting the BDS's "Online" registry value:


[HKEY_LOCAL_MACHINE\Software\Microsoft\BDS\1.0]
Online="0"

The latter is accomplished by creating a local "default" tree that mirrors what bds.dll expects to find internationally. Assuming the Money install directory is "C:\Program Files\Microsoft Money", that means the following:

  • Create the directory hierarchy "C:\Program Files\Microsoft Money\system\bds\default".
  • Create a "default.ini" in the default directory. (A "summary.rtf" should probably be created as well.)
  • Create a directory and branding info for each "provider" listed in the default.ini.

Click here for a sample "default.ini" and "summary.rtf"

To use them directly, the subdirectories "test1" and "test2" should be created in the "default" directory, and populated with branding info. By the end of all of this, the local machine should have the following files:

  • C:\Program Files\Microsoft Money\system\bds\default\default.ini
  • C:\Program Files\Microsoft Money\system\bds\default\summary.rtf
  • C:\Program Files\Microsoft Money\system\bds\default\test1\bank.ini
  • C:\Program Files\Microsoft Money\system\bds\default\test1\other branding files
  • C:\Program Files\Microsoft Money\system\bds\default\test2\bank.ini
  • C:\Program Files\Microsoft Money\system\bds\default\test2\other branding files
