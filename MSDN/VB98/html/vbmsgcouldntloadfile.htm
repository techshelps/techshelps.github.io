'item' could not be loaded

In addition to other causes, such as if the file is corrupt, or you don’t have read permission to the file, this error occurs when you attempt to add a file containing an existing Form, MDIForm, UserControl, PropertyPage, or UserDocument to a project in which you have selected the Unattended Execution option (accessed from the General tab of the Project Properties dialog box).

A project marked for unattended execution cannot support any form of user interaction, so the module types it can contain are severely restricted.

For more information, see "Scalability and Multithreading," in "Building Code Components" in Creating ActiveX Components.
