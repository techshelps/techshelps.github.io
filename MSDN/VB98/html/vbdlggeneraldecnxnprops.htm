General Tab (Connection Properties Dialog Box)

Using the Connection Properties dialog box you can set the required and optional properties of a Connection object. The properties are divided between three tabs: General, Authentication, and Miscellaneous.

Note   The Connection Properties dialog box is only used by Visual InterDev.

On the General tab, specify the source of data and set the required properties of the Connection object.

Connection Name

The name of the Connection object. This name references an individual Connection object within a DataEnvironment object. The default is "Connection" plus a unique number (for example, Connection1). It is recommended that you change this to a more meaningful name, such as the name of the target database.

Source of Connection

This frame contains the control group used to specify the type of connection to use. This can be an ODBC data source name (the default), a Data Link file, or a connection string.

Use Data Link File

Specifies that you are using a custom connection string that connects to the data source. When this is selected, you can click Browse to access the Organize Data Sources dialog box, from which you can select your Data Link file.

Use ODBC Data Source Name

Specifies that a system-defined data source name (DSN) is used for the connection string. You can access a list of all system-defined DSNs via the combo box.

When this option is selected, you can click New to access the Create New Data Source wizard dialog box to add to or modify DSNs on the system.

Use Connection String

Specifies that you are using a connection string to access data. If the Use Connection String text box is empty, the wizard appears, or you can click Build to access the Data Link Properties dialog box. Use this dialog box to specify the connection, authentication, and advanced information required to access data using an OLE DB provider.

Test Connection

Once the connection source is selected, click this button to test the specified connection, ensuring that the properties are correctly set in the Connection Properties dialog box.

Note   If the connection is invalid, ensure that all properties are specified correctly on each tab of the dialog box, and then retest the connection.
