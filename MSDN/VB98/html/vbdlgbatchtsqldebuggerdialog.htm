Visual Basic Batch T-SQL Debugger Dialog Box

Allows you to create, edit, and interactively debug stored procedures written in Microsoft SQL Server’s Transact SQL dialect within the Visual Basic development environment.

To view this dialog, add the T-SQL Debugger to the development environment via the Add-In Manager command on the Add-Ins menu. Then, with the debugger loaded, select T-SQL Debugger on the Add-Ins menu.

Use the T-SQL debugger to:

  • Display the SQL call stack, local variables, and parameters for a SQL stored procedure.

  • Control and manage breakpoints.

  • View local variables and parameters.

  • View global variables.

Note   In order to use the T-SQL debugger, you must have SQL Server version 6.5 with Service Pack 3 or later installed as your database server.
