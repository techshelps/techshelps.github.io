Microsoft Data Report Window (Design Time)

Allows you to determine where information appears in each section, by placing Data Report designer controls, such as labels and text boxes.

Window Elements

Report Header

Appears once at the beginning of a report. Use for items such as a logo, report title, or print date. The report header is printed before the page header on the first page of the report.

Page Header

Appears at the top of every page. Use to display items such as column headings, page numbers, or the report title.

Group Header

Associated with a DataMember of the DataSource, each group header delineates a repeating section of the final report. Data-bound TextBox controls placed in the group header can be bound to fields in the DataMember associated with the group header. Each group header is matched with a group footer.

Detail Section

The innermost repeating section of the report is, like group headers/footers, associated with a DataMember. Like group headers and footers, the detail section can contain controls bound to fields in the associated DataMember. If the DataSource is the Data Environment, the detail section is bound to the lowest Command object in the hierarchy.

Group Footer

Like group headers and the Detail section, each group footer delineates a repeating section of the final report. Data-bound TextBox controls in a group footer can be bound to fields in the DataMember associated with the group footer. Function controls can also be placed in the group footer section.

Page Footer

Appears at the bottom of every page in the report. Use to display items such as page numbers.

Report Footer

Appears once at the end of the report. Use to display items such as report totals. The Report Footer is the last section in the report design but appears before the Page Footer on the last page of the printed report.
