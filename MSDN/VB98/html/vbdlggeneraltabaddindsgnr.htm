General Tab (Add-In Designer)

Allows you to begin the development process for an add-in by registering it, and setting properties.

To display the Add-In Designer

  1. Select the AddInDesigner Object on the Designers tab of the Components dialog on the Project menu.

  2. Select the AddInDesigner Object from the Add ActiveX Designer command on the Project menu.

Window Elements

Add-In Display Name

Enter the display name of the add-in. This name identifies the add-in in the Add-In Manager Dialog. You can also add a resource ID here by entering a pound sign (#) followed by an integer. Enter the filename of the .dll containing the resource on the Advanced tab of this designer.

Add-In Description

Enter a description of the add-in. You can also add a resource ID here by entering a pound sign (#) followed by an integer. Enter the filename of the .dll containing the resource on the Advanced tab of this designer.

Application

Select the add-in's host application. The dropdown lists registered applications capable of hosting add-ins.

Application Version

Enter the version number of the add-in's host application.

Initial Load Behavior

Select the add-in's load behavior.

  • None - No load behavior is specified.

  • On Startup - The add-in loads on normal Visual Basic startup

  • Command Line - The add-in loads when Visual Basic starts from the DOS prompt

  • Command Line/On Startup - The add-in loads whenever Visual Basic starts, whether from the command line or not.

Note   The options for load behavior depend on the host selected under Application. For example, if Microsoft Office is selected you'll see a different list.
