Unable to write Designer cache file 'item'.  Will just use regular files on Load

Your ActiveX Designer was not able to write information to a cache file, so the designer will load using uncached information.

An ActiveX Designer writes a cache file to improve performance the next time you start Visual Basic. When this error occurs, the designer was unable to write to the cache file due either to insufficient disk space or invalid permissions to the drive (such as write privilege). Make sure you have enough disk space available and that you have network access (if applicable) and write permission to the drive. If the problem persists, contact the ActiveX Designer's vendor.
