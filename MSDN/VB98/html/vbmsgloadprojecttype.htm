Current public/instancing property values are not valid in this type of project (<name>). The property values have been changed.

The values currently set in the Public and/or Instancing properties aren't valid for this type of project, so the property values were changed to values that will work.

This error occurs when you add a file to an existing project, but that project type doesn't support the setting of the Instancing and/or Public properties of the module the file contains.

For example, you get this error if you add a class module from an ActiveX Exe project whose Instancing property is set to MultiUse to a Standard Exe project. This is because Standard Exe projects support only the value Private for the Instancing property.

Similarly, this error occurs if you add a UserControl whose Public property is True to a Standard Exe project, because controls in Standard Exe projects can be private only.

You don't need to take any action as a result of this error, because Visual Basic automatically resets the Instancing or Public property to a value supported by the project type.
