Procedure ID must be a negative number or zero

You are only allowed to enter Standard Procedure IDs in the Procedures Attributes dialog that are zero or a negative number.

Valid ranges are:

  • -999 to -500 for controls.

  • -2499 to -2000 for Visual Basic version 5.

  • -3899 to -2500 reserved for future expansion.

  • -3999 to -3900 for Dynamic HTML.

  • -5550 to -5500 for Dynamic HTML.

  • -5499 to -5000 for ActiveX accessibility.

  • Also allowed is a special range of 0x80010000 to 0x8001FFFF for control containers such as Visual Basic version 4.

The above values are ID numbers that are invoked through an internal Visual Basic interface called IDispatch. It is your choice as to what range of numbers to use in your application, although you should use a set of numbers that are contiguous and from the appropriate group. That is, if your application is an ActiveX control, you might choose a value (or values) from the range -5499 to –5000.
