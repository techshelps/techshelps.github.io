Chapter 2: Embedded SQL Programming

Embedded SQL programs incorporate Transact-SQL statements into C source code. But because you cannot directly submit Embedded SQL programs to a C compiler, Embedded SQL source programs must be processed by a precompiler that produces source code that is acceptable to a C compiler.

Embedded SQL programming is a multistep development process that converts your original Embedded SQL source code into a SQL Server application—an executable file that is compiled for the appropriate operating system.
