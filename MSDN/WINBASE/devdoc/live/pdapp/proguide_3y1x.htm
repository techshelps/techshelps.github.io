Updating the Restored Database

[This is preliminary documentation and subject to change.]

The restored database might not be synchronized with the current state of media. The following table describes some reasons that the RSM database might not be synchronized and how your application can resolve each problem.

Problem Solution
A specified medium is deallocated by an application after the backup but before the disaster and remains in the application media pool. RSM recognizes this medium as allocated and puts it in the application media pool. The application can deallocate this medium.
A specified medium is allocated by an application after the backup but before the disaster. RSM puts the medium in the import media pool. The application can import the medium to the scratch media pool.

When the restored database is updated, you can use the RSM system to complete the recovery process.
