Recovering the Database

[This is preliminary documentation and subject to change.]

When the RSM database has been restored, the recovery application can use the ImportNtmsDatabase function. This function makes the changes necessary for RSM to import the restored database and inventory the online libraries on the next restart. Recovery applications should offer the administrator the option to restart RSM or the system.

Typically, recovery applications also restore the registry and reboot the system after it has been restored. Applications can restore the registry and import the RSM database in any order.
