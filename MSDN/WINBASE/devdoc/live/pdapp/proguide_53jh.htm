Inserting media into a library

When a medium is introduced into a library, either by an insert command or a door access command, it is examined to determine its destination media pool. The identification process involves two IDs: a bar code if available or an on-media identifier (OMID). The following diagram shows the process and resulting destination media pool for the medium.

The following table shows what happens when media is moved from or to different media pools.

From/To Scratch Import Foreign Application
Scratch Not allowed Not allowed Retains Scratch label until allocated.
Import Write MTF scratch label Not allowed Not allowed
Foreign Write MTF scratch label Not allowed Not allowed
Application Write MTF scratch label Not allowed Not allowed Retains current label. Allocated media may not be moved from a pool.
