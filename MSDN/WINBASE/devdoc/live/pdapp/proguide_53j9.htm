RSM Database

[This is preliminary documentation and subject to change.]

The RSM database contains information about the objects in an RSM system. RSM maintains the data in the database and updates it whenever the administrator or an application makes a change in an RSM system. For example, if the administrator wants to mount a piece of media in an online library, RSM mounts the media and updates the database to reflect the change.

The database is the mechanism used by RSM to track media-related system components. The database stores the status of these system components and maintains media inventories. The database includes the following information:

  • Library, drive, and media configuration and state
  • Media pool configuration and contents
  • Library work list
  • Operator requests
