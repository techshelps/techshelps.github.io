Backing Up the RSM Database

[This is preliminary documentation and subject to change.]

To back up the RSM database, a backup application must open an RSM session and use the ExportNtmsDatabase function. This function creates a replica of the current database so that the recovery application can back up the database without stopping RSM. RSM creates the replica in the %SYSTEMROOT%\System32\NTMSData\Export subdirectory of the RSM database directory.
