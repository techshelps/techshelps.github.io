Recovery with Non-RSM Applications

[This is preliminary documentation and subject to change.]

RSM opens the RSM database when it needs it and closes the database immediately after usage. This allows a non-RSM recovery application to back up the RSM database. To recover the database using a non-RSM application, restore the database to a temporary location, shut down RSM, copy the restored database files to the %SYSTEMROOT%\System32\ntmsdata directory, and restart RSM.
