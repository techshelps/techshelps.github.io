NTMSIDX

[This is preliminary documentation and subject to change.]

The NTMSIDX (index) file stores the indexes for looking up records in the database. The following two distinct indexes are stored within this file:

  • An index of records organized by type, for finding records by type.
  • An index using a globally unique identifier (GUID) hash, for locating records by their IDs.

The index file can be fully recreated from the NTMSDATA file and is therefore not critical to integrity. In fact, RSM rebuilds the index on startup if an integrity problem exists.
