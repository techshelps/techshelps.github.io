Optional Replica Synchronization

[This is preliminary documentation and subject to change.]

The RSM database can add an option to provide for immediate replica synchronization on commit completion. This option allows the replica to be up-to-date prior to commit completion and therefore available as an exact duplicate of the main data file. Although this does not provide additional integrity, it can help failure one or more sectors within the data file are bad and the one or more files within the journal are bad.
