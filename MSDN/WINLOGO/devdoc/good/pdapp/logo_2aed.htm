Accessibility and User Interface

[This is preliminary documentation and subject to change.]

These are guidelines that relate to providing consistent, up-to-date support for accessibility under Microsoft® Windows®, and for other user-interface standards:

  • Be compatible with the High Contrast option. (Required)
  • Provide keyboard access to all features. (Required)
  • Document the keyboard user interface. (Required)
  • Provide notifications of the keyboard focus location. (Required)
  • Do not rely on sound alone. (Recommended)
  • Do not rely on color alone. (Recommended)
  • Let other software identify and manipulate screen elements. (Recommended)
  • Support Active Accessibility. (Best Practice)
  • Do not hard-code font sizes smaller than 10 points. (Best Practice)
  • Allow users to customize UI timings. (Best Practice)
  • Support system font and resolution changes. (Best Practice)
  • Allow users to set font names and sizes. (Best Practice)
  • Handle changes in pointer or keyboard focus. (Best Practice)
  • Test using multiple displays. (Best Practice)
