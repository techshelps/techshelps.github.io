Development Tools

[This is preliminary documentation and subject to change.]

Development tools are products such as languages and compilers that create executable files or interpreted code modules that mimic the action of executable files when used with a run-time engine of the developing application.

These products must meet all the general requirements, with the following exceptions, and with the following additional requirements and recommendations:

Special Requirements for Development Tools

  • Make a development tool capable of generating logo-compliant applications. (Required)
  • Submit a sample application for testing of your development tool. (Required)
  • Make it easy to create OLE containers and/or object servers with your development tool. (Required)
  • Advise customers of potential logo-requirement problems in your development tool. (Required)

Special Exceptions for Development Tools

  • OLE drag-and-drop support is not required within the tool's design environment.
  • Development tools that create Java applications do not need to be capable of generating applications that can meet all of the Designed for Microsoft® Windows® Logo requirements (see Java Applications).
  • Development tools are temporarily exempt from the requirement to expose the keyboard focus. Vendors should be aware that this exemption will be removed in the next update of the Logo Handbook.
  • Java applications must use and re-distribute the 32-bit virtual machine for Java.
  • Add-on products must be hosted by a 32-bit product that is logo-compliant.
