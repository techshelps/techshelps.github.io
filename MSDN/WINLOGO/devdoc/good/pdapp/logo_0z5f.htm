Take Advantage of COM+ Run-Time DLL Functions

[This is preliminary documentation and subject to change.]

Best Practice   (Designed for Microsoft® Windows® Logo)

Your application supporting COM+ objects does not have to expose any of the following COM+ DLL API functions that are exposed by the COM+ run time:

  • DllRegisterServer
  • DllUnregisterServer
  • DllGetClassObject

See Also

For information about COM+, see http://www.microsoft.com/com.
