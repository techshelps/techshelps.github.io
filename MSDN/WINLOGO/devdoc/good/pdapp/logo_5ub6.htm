Use a Product-Specific DS Object in the Configuration/Services Container

[This is preliminary documentation and subject to change.]

Best Practice   (Designed for Microsoft® Windows® Logo)

Your application should both publish global configuration information to a product-specific DS object created in the Configuration/Services container and obtain global configuration information from the Configuration/Services container.

Remarks

This container is replicated to every Domain Controller in the enterprise. It is the preferred way to establish enterprise-wide configuration defaults for applications.
