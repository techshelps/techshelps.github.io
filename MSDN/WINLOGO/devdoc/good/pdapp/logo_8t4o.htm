Explicitly Link to a TAPI32 Library If Using TAPI 2.x

[This is preliminary documentation and subject to change.]

Requirement   (Designed for Microsoft® Windows® Logo)

If the application uses TAPI 2.x functions, it must either explicitly link to TAPI32.dll (by calling GetProcAddress) or be linked to theTAPI32.lib static-link library so that it will load on both Windows 95 (TAPI 1.4) and Windows NT 4.0 (TAPI 2.0).
