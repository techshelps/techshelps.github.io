Be Easy to Install and Remove

[This is preliminary documentation and subject to change.]

Installation is one of the most important and sensitive areas for a user or administrator. First impressions are lasting, and a trouble-free installation is the best kind of first impression to make. Also, a user or administrator needing to remove an application expects to be provided an easy, automated way to do so.

Designed for Microsoft® Windows® Logo recommendations and best practices that relate to this goal are listed below. The list is just a summary. To comply with the recommendations and best practices, you must follow all stipulations of the reference text to which the lists refer.

Installing and Removing Applications

  • Maintain user settings across version changes. (Best Practice)
  • Avoid rebooting the system. (Best Practice)

Installing and Removing Individual Components

  • Do not write to Windows system directories. (Recommended)
  • Use Microsoft update packs to upgrade system components. (Recommended)
  • Do not install globally shared DLLs. (Recommended)
  • Set count correctly for pre-existing unregistered shared components during installation. (Best Practice)
