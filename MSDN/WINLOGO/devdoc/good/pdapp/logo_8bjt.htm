Provide Separate Fields with Defaults for Phone Number Entry

[This is preliminary documentation and subject to change.]

Best Practice   (Designed for Microsoft® Windows® Logo)

User entry of phone numbers should be in separate fields for country code, area/city code, and local number. The country code and area code fields should default to the user's current location country code and area code obtained from the lineGetTranslateCaps function. The country code selected can consist of a drop-down list of countries obtained from the lineGetCountry function.
