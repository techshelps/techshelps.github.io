Special Requirements

[This is preliminary documentation and subject to change.]

Several types of application have special requirements and exemptions when applying to use the Designed for Microsoft® Windows® Logo. Designed for Microsoft® Windows® Logo requirements relating to particular types of applications are listed below.

These lists are just summaries. To qualify for the Designed for Windows Logo, you must comply with all stipulations of the requirement reference text to which the lists refer.

Development Tools

  • Make a development tool capable of generating logo-compliant applications.
  • Submit a sample application for testing of your development tool.
  • Make it easy to create OLE containers and/or object servers with your development tool.
  • Advise customers of potential logo-requirement problems in your development tool.

Utilities

  • Provide meaningful functionality on both Windows 95 and Windows NT.
  • Extensively test any utility that uses the Exclusive Volume Locking function.

Games and Multimedia Applications

  • Use DirectX compatibly with Windows NT Versions.
  • Use Direct3D and DirectSound compatibly with Windows NT 4.0 and Windows 95.
  • Use DirectX 5.0 for multimedia capabilities.

Java Applications

  • Use and redistribute the 32-bit Virtual Machine for Java Applications.

Separate Add-On Products

  • Host an add-on product on 32-bit products that are logo-compliant.
