Pretesting for Stability on Windows 95

[This is preliminary documentation and subject to change.]

The following is one of the tests we will run to test the stability of your product. You can use it to pretest your product prior to submitting for the Designed for Microsoft® Windows® Logo testing.

  1. Find the Stress application in the Platform SDK (\mstools\bin\win95\stress.exe).
  2. Select the following options:

    User Heap
    GDI Heap
    Wind32 Heap
    Menu32 Heap
    GDI32 Heap

    Set the Level to 2.

  3. Start the Stress program.
  4. With Stress running, start your product from its icon.
  5. Exercise all of your product's functions, and make sure they are stable even when the system is under stress.
  6. Switch video modes "on the fly," and make sure your product is still operational.
  7. Test that printing and all other input/output functions supported.
  8. Close Stress, and then start one popular 16-bit application and one popular 32-bit application to see whether your product will still run without problems while these are active.
