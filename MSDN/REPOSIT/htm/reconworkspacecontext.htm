Workspace Context

When you operate within a workspace, the workspace provides a specific context for your work. That context includes the workspace's Versions-of-Workspace collection and its Checkouts-of-Workspace collection. If an object version is not part of either of these collections, it is not part of a workspace's context.

When you invoke a Repository method, the method's behavior can vary depending on whether you invoked the method from within the context of a workspace or from within the more general context of a Repository instance. For example, if you materialize the ObjectInstances collection for a class, the collection can consist of two different sets of items:

  • If you are operating within the context of a workspace, the collection contains one item for each object in that workspace conforming to that class.

  • If you are operating within the more general context of a Repository instance, the collection contains one item for each object in the Repository conforming to that class.

Similarly, if you invoke get_Object, two different things can happen:

  • If you are operating within the context of a workspace, Repository's resolution strategy yields whatever version of that object is present in the workspace.

  • If you are operating within the more general context of a Repository instance, Repository's resolution strategy yields the latest version of the object.
