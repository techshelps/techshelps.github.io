Introducing Microsoft Repository

Microsoft Repository has two major components: a set of ActiveX® interfaces that a programmer can use to define open information models, and a Repository engine that is the underlying storage mechanism for these information models. The Repository engine sits on top of either Microsoft SQL Server™ or Microsoft Jet.

Microsoft Repository targets software tool vendors who want to use open information models to support the development, deployment, and reuse of component software. Corporate customers who will benefit from component reuse and tool interoperability will be the primary users of these tools.
