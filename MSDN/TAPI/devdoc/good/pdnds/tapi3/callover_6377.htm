Provider-Specific Interfaces

[This is preliminary documentation and subject to change.]

TAPI service providers may provide additional interfaces through TAPI 3.0 objects to implement provider-specific features.

The provider-specific interfaces should be optional and intended for applications written with a specific provider in mind. An application must be able to make connections and access standard media streams without using the provider-specific interfaces.

For TAPI 3.0, raw data streams should be exposed through provider specific interfaces.
