Remote SP Compatibility with Service Providers

The developer should be aware of the following Remote SP compatibility issues:

  • Remote SP is only compatible with TAPI version 2.0 service providers.
  • Remote SP must use the Telephony user interface SPI (TUISPI) for UI and communication between the client and telephony server.
  • The service provider that is installed on the telephony server must use the TUISPI interface for any UI it wants to display. If the service provider does not use the TUISPI interface or incorrectly assumes that its UI will display on the machine that the service provider is installed on, the service provider is incompatible with Remote SP.
  • Remote SP currently only handles call control. Any media streams must be handled by the service provider to work in a client/server environment.
