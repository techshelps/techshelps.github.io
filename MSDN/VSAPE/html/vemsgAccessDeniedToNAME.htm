Access denied to <NAME>

   

The remote computer <name> did not allow APE to run.

Try the following:

  • Confirm that the remote computer is still turned on and logged on to the network.

  • Confirm that a valid APE (client or server) installation exists on the computer.

  • If the remote computer is logged on correctly, confirm that AutMgr32.exe is running to support Remote Automation calls and that the necessary security rights have been set to allow for the use of distributed COM.

For More Information   For information on installing APE and configuring security rights, see Installing Application Performance Explorer.
