Failed to create AEServerMgr.ServerMgr on the remote machine

   

APE requires the file, AESvrMgr.exe, to support the calling of server processes.

Try the following:

  • If the error text directs you to the lack of response or failure to connect to a specified computer, check that the computer is turned on and logged onto the network.

  • If the remote computer is logged on correctly confirm that AutMgr32.exe is running on it to support Remote Automation calls and that the necessary security rights have been set to allow for the use of distributed COM.

  • Confirm that the installation of all local APE components and any remote installations being used in the current profile were successful. If not, you may need to reinstall either the APE client or server, as appropriate.

For More Information   For information on installing APE components, see Installing Application Performance Explorer.
