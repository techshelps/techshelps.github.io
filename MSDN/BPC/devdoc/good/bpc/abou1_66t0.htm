Further Information on Television Services for the Client

[This is preliminary documentation and subject to change.]

  • For more information about WM_POWERBROADCAST messages and the Advanced Configuration and Power Interface (ACPI), see the Power Management Messages section in the Platform SDK and the http://www.microsoft.com/hwdev/desinit/default4.htm Web site.
  • For information about the GetActiveObject function used when connecting to Microsoft® WebTV® for Windows®, see the Automation section of the Platform SDK.
  • For information about the idle processes used by ITVControl, see the OnIdle function topic in Microsoft® Foundation Classes (MFC) documentation supplied with the Microsoft® Visual C++ development system or with MSDN Level 2.
  • For information about the IOleObject interface and the IOleObject::DoVerb method used by the Broadcast Architecture ITVControl interface, see the COM section of the Platform SDK.
  • For information on the QueryDef query definition, see the DAO SDK documentation.
  • For more information on using the Jet database, as required by some functions of Television System Services (TSS), see the Data Access Objects (DAO) SDK documentation in the Platform SDK.
  • For information on the ITaskTrigger interface, the TaskTrigger object, and the TASK_TRIGGER structure used with TSS and with show references, see the Task Scheduler section of the Platform SDK.
  • For information on the IDispatch Component Object Model (COM) interface used by the TSS object library and Microsoft WebTV for Windows applications, and on the IErrorInfo Automation interface, see the Automation section of the Platform SDK.
  • To view the Winerror.h header file, which includes error codes returned by methods of the ITelevisionServices and IScheduledItems interfaces, among others, see the Reference section of the Platform SDK.
  • For access to the Dbdaoerr.h header file, which includes error codes returned by some methods of the ITelevisionServices and IScheduledItems interfaces, see the DAO SDK documentation.
  • For information about using the Jet Installable Sequential Access Module (ISAM), see the DAO SDK documentation.
  • For information on the SAFEARRAY and VARIANT data types, see the Automation section of the Platform SDK.
  • For information about the IEnumVARIANT interface used in enumerating collections, see the Automation section of the Platform SDK.
  • For information about ITask interface, see the Task Scheduler section of the Platform SDK.
