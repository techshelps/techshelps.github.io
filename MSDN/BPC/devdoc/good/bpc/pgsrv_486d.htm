Moving Data to the Program Guide

[This is preliminary documentation and subject to change.]

For any given broadcast service, one or more sources may provide Program Guide data. Because of the number of sources and formats for Program Guide data, it would be unwieldy for the Program Guide to directly receive Program Guide data. There are simply too many variations. To address this problem, Broadcast Architecture uses database loader programs that can be written for any given data source. Each tuning space has an associated Guide database loader. For more information on Guide database loaders, see Guide Database Loaders.
