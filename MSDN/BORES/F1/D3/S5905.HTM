Associating Connections with Security Domains

Since the user may have different host accounts in several host security domains, SNA Server must be able to choose the appropriate host account for single sign-on when the user opens a session. Therefore, an SNA connection defined in SNA Server Manager can be assigned to a host security domain. The administrator uses SNA Server Manager to assign the connection using the Insert menu or by dragging a connection to a host security domain folder. A host security domain can be assigned an unlimited number of connections, but each connection can be a member of only one host security domain.

When the client chooses an LU resource (alias or pool), the client and server perform resource location to determine which connection will support the session, and, based on that determination, from which host security domain the host account information will come.
