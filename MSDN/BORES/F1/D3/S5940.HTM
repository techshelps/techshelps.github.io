RandomSponsor Setting

SNA Server allows clients to randomly choose a sponsor server. This provides load balancing across SNA Server computers to evenly distribute the available resources.

This setting can only be configured in the Registry for Windows NT clients and in the WIN.INI file for Windows 3.x and Windows for Workgroups clients. The RandomSponsor is not configurable for the Windows 95 client as remote sponsor servers are always tried in random order.

The RandomSponsor setting is enabled by default for Windows 3.x and Windows NT clients. It can be disabled using the Registry and WIN.INI files.

Windows NT Clients

To disable the RandomSponsor setting for Windows NT clients, use the following Registry entry:

HKEY_LOCAL_MACHINE
SYSTEM
CurrentControlSet
Services
SnaBase
Parameters

RandomSponsor:REG_DWORD:0

After the Registry is modified, the client SnaBase service must be restarted. If the RandomSponsor setting is disabled, the client will try the remote servers in the order of configuration.

Windows 3.x or Windows for Workgroups Clients

In the WNAP section of the WIN.INI file modify the RandomSponsor line:

RandomSponsor=NO
