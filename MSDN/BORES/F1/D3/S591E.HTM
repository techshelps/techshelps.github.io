VTAM Definitions

Each remote system must be defined to VTAM in its table of Network Addressable Units, called NAUs, using LU definitions (see Table 7.12).

Note that if a LOGAPPL parameter is specified on the LU definition naming the CICS system, CICS will attempt to establish a session to that LU whenever it becomes active. This is necessary if TPs residing in CICS do not use the ALLOCATE verb with RETURN_CONTROL=WHEN_SESSION_ALLOCATED to establish sessions. There must also be a VTAM mode table definition present for LU 6.2, although CICS does not use the values specified here.

TP Coding Requirements

To work with CICS, TPs must be coded to specify a SYSID parameter on the ALLOCATE request that matches the defined SYSIDNT parameter configured in the DFHTCT tables. This request must be in the following format:

EXEC CICS ALLOCATE SYSID('DC11')

The SYSID field must match the SYSIDNT field specified for a CICS program by the DFHTCT definition in the TCT. The TCT then uses the NETNAME parameter to refer to a specific LU in the VTAM table of NAUs for use by the TP to communicate with the remote system.
