Message 31

(value) SNA Server Manager command failure, rc = value

EXPLANATION: A command issued by SNA Server Manager failed.

SUBCODE INFORMATION:

X'06'

COMMAND_IN_PROGRESS: This code should not normally occur.

X'07'

COMMAND_NOT_RECOGNIZED: This code should not normally occur.

X'09'

SESSION_ACTIVATION_FAILED: An attempt to start an APPC session via SNA Server Manager failed. This log is accompanied by one of the APPC session activation failure messages (message 7, 14, 15, 17, 18, 22, 24, 25, or 26). These messages provide more information about the failure.

X'0A'

COMMAND_RACE_REJECT: A change number of sessions (CNOS) request (either Activate or Deactivate APPC sessions) made through SNA Server Manager failed because the partner LU was also trying to change the session limit for the specified mode. Retry the command.

X'0B'

SESSION _LIMIT_CLOSED: A CNOS request made through SNA Server Manager failed because the partner LU will not allow the session limit for the specified mode to be raised above zero. This may not be a permanent condition; try the CNOS request later. If the problem persists, contact support personnel for your network.

X'0C'

SESSION_LIMIT_NOT_ZERO: An activate session request made through SNA Server Manager has failed because the session limit was not zero. This may occur because the partner LU initialized the session limit immediately before SNA Server Manager attempted to do so. No action is required.

X'0D'

LU_SESSION_LIMIT_EXCEEDED: SNA Server Manager issued an activate session request that violates the session limit for the LU.

In most cases, the LU session limit should be larger than the sum of the session limits of all the modes associated with the LU (remembering to allow for SNASVCMG modes). If this is not the case, some modes may not available, or may only allow a reduced number of sessions.

X'0E'

REQUEST_EXCEEDED_MAXIMUM_ALLOWED: This code should not normally occur.

X'0F'

RESOURCE_FAILURE_NO_RETRY: A CNOS request failed to complete because of a permanent resource failure. This error is caused by the partner LU violating the CNOS protocol. Obtain a trace of the error, and contact your network support personnel with the trace file and the event log file(s) related to SNA Server.

X'10'

ALLOCATION_ERROR: A CNOS request failed because no SNASVCMG session could be allocated to perform the CNOS exchange. This may occur because the connection required has not been activated, or the partner LU rejected the BIND request. If the error occurred because the connection was not active, activate it and retry the request. If the BIND was rejected, contact your network support personnel with the event log file(s) related to SNA Server.

X'11'

UNRECOGNIZED_MODE_NAME: A CNOS request failed because the partner LU does not recognize the specified mode name. This error signifies a configuration mismatch between the local LU and the partner LU.

X'12'

PARAMETER_ERROR_BAD_LU_NAME: This code should not normally occur.

X'13'

PARAMETER_ERROR_BAD_RESOURCE NAME: This code should not normally occur.

X'14'

PARAMETER_ERROR_MISSING_PARAMETERS: This code should not normally occur.

X'16'

PARAMETER_ERROR_BAD_ADDRESS: This code should not normally occur.

X'17'

PARAMETER_ERROR_SECURITY_ERROR: This code should not normally occur.

X'18'

PARAMETER_ERROR_MIN_GT_MAX: This code should not normally occur.

X'1A'

RESOURCE_ACTIVE: SNA Server Manager tried to activate a connection when it was already active. This may occur as part of normal operation, for example, if an application activates the connection immediately before SNA Server Manager issues the command. No action is required.

X'1C'

RESOURCE_INACTIVE: SNA Server Manager tried to deactivate a connection when it was already inactive. This may occur as part of normal operation, for example, if the connection is deactivated because of an outage immediately before SNA Server Manager issues the command. No action is required.

X'1D'

NO_LINK: SNA Server Manager tried to activate a switched connection, but all the appropriate link services are being fully used. Deactivate unused connections using the appropriate link services and retry the command.

X'1E'

NO_PRIVILEGE: This code should not normally occur.

X'1F'

PARAMETER_ERROR_FORCE_ERROR: This code should not normally occur.

X'22'

ABEND_PROG_ERROR: This code should not normally occur.

ACTION: Provide network support personnel with the event log file(s) related to SNA Server, the SNA Server configuration file, internal trace file(s), and the number and description of the displayed return code. For information about SNA Server log files, configuration files, and trace files, see the Microsoft SNA Server Administration Guide.
