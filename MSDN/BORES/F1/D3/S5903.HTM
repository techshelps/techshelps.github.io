Creating a Host Security Domain

The administrator defines a host security domain using SNA Server Manager in a Windows NT resource domain containing SNA Server computers. If the same host security domain is defined in multiple SNA Server subdomains, it must be defined with the same name and consistent properties in each subdomain.

When SNA Server Manager creates the host security domain, it sends a message to the primary host account cache in the Windows NT domain and it creates a local group in the Windows NT user database. Properties of the host security domain, including the checkboxes Allow User to Override, Mapped, and Replicated, are stored in the Host Account Cache and not in the SNA Server configuration file.
