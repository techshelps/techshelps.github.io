Table 7.2 defines key parameters shown in Table 7.1.

Table 7.2 VTAM Parameter Descriptions for Token Ring

VTAM parameter

Corresponding SNA Server parameter


Description

IDBLK=

First three digits of Local Node ID (basic connection setting)

A three-digit hexadecimal value that, together with IDNUM, identifies the SNA Server. The values 000 and FFF cannot be used; these values are reserved.

IDNUM=

Last five digits of Local Node ID (basic connection setting)

A five-digit hexadecimal value that, together with IDBLK, identifies the SNA Server.

MAXDATA=

(in the PU definition for SERVER01)

Max BTU Length (advanced 802.2 setting)

The maximum length of a basic transmission unit (BTU) sent or received; BTUs are also known as I-frames. Max BTU Length should be less than or equal to MAXDATA.

MACADDR= in the PORT definition

Remote Network Address (basic 802.2 setting)

The media access control address, a 12-digit hexadecimal address.

The Remote Network Address (MACADDR) shown in Table 7.1 is 400001701100.

SAPADDR= in the PU definition

Remote SAP Address (advanced 802.2 setting)

The system access point address, a two-digit hexadecimal number, a multiple of 4, between 04 and EC.

The Remote SAP Address shown in Table 7.1 is 04, the default value. This value is recommended for most installations.

LOCADDR= in the LU definition

LU Number

The LU local address at the PU. For independent LUs that communicate with a host, the LOCADDR parameter must be set to 0. For dependent LUs with VTAM, the LOCADDR must be at least 2. Also, for dependent LUs, the LU Numbers set in SNA Server must match the LOCADDR values on the host.

The LU Numbers range from 002 to 006 in Table 7.1.


The GRPNM value shown in the sample in Table 7.1 is a group name; group names must match the label of a VTAM group definition. The DLOGMOD values shown in the sample are the logon mode table entries from the mode table. For descriptions of the group macros used in these samples, and a complete listing of default mode table entries, see the documentation for your IBM product.
