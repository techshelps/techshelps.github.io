Chapter 8 Registry and Environment Variables for SNA Server

This chapter describes the Registry variables and environment variables that can be set for SNA Server.

In most situations, it is recommended that you leave Registry and environment variables at the default settings. Specifying different values for variables allows you to:

  • Communicate with a host that implements unusual variants of SNA protocols. Changing the Registry and/or environment variables allows for workarounds with such hosts.
  • Use special default values for such settings as timers.
  • Set a variable for an application when the application could not use a standard API call to set that variable.

Important Change Registry and environment variables only if your specific situation makes it appropriate. Changing Registry and environment variables inappropriately can disable SNA Server functionality. In most situations, these variables can be left at default settings.

The variables used to configure an invokable TP are not documented in this chapter. For information about these variables, see Chapter 2 of the Microsoft SNA Server APPC Programmer's Guide, or Chapter 2 of the Microsoft SNA Server CPI-C Programmer's Guide.
