SnaBase: SNA Servers of Variables

For SNA Server or a Windows NT-based client, the Registry variables that affect the SnaBase are found in the following location:

HKEY_LOCAL_MACHINE
SYSTEM
CurrentControlSet
Services
SnaBase
Parameters

These are the variable names and descriptions:

BufferAuditInterval:REG_DWORD:number

This variable causes SNA Server computers and client computers to carry out audits of the buffer pool at regular intervals, and sets the interval length in seconds. (SNA Server rounds the number of seconds upward to the nearest 5 seconds.) The range for number is 1 through 0xFFFFFFFE. The parameter must be configured individually on server and client computers.

Normally (that is, when this variable is not set), SNA Server only carries out buffer audits when a buffer pool is extended and more than half of the buffer pool is in use.

When SNA Server carries out a buffer audit, the information is placed in message 684 in the application event log.

LOCALLUS:REG_MULTI_SZ:localLUname

This entry is no longer used in SNA Server 3.0.

OtherServers:REG_MULTI_SZ:server1 [server2][ ...]

This variable specifies one or more servers outside of the client's subdomain to which the client's requests for LUs will be sent if the requests cannot be satisfied by servers in the client's subdomain. If this variable is not set, only servers in the client's subdomain will receive the client's requests for LUs.

When specifying the variable, substitute one or more server names for
server1 [server2][ ...]. Separate multiple server names with spaces.

COMPOP12:REG_SZ:{ YES | NO } Default: NO

When this variable is set to YES, level 12 warnings do not display as pop-up messages on the local system. (Level 12 messages are not as severe as level 16 messages.)

SnaBackupDomain:REG_SZ:domain

This variable specifies a backup subdomain for a client configured to run in local mode. When the client software is starting, if it cannot connect to its primary sponsor subdomain, the client software will attempt to connect to the backup subdomain.

BackupSponsors:REG_MULTI_SZ: server1 [server2][ ...]

This variable specifies one or more backup sponsor servers for a client configured to run in remote mode. When the client software is starting, if it cannot connect to its primary sponsor servers, the client software will attempt to connect to the backup sponsor servers. If the SnaBackupDomain variable is present, the BackupSponsors variable is ignored.

When specifying the variable, substitute one or more server names for
server1 [server2][ ...]. Separate multiple server names with spaces.

Note that this variable is of type REG_SZ in the Windows 95 Registry.
