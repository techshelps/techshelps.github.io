SnaBase

These are the variable names and descriptions:

For Windows version 3.x:

AutoTerminate={YES | NO} Default: NO

When this variable is set to YES, wnap is automatically closed when the Windows 3.x application session is closed. Wnap must be automatically started for the automatic termination to be enabled. If wnap is manually started, wnap will not terminate, even if set to Yes.

For MS-DOS and Windows version 3.x:

LOCALLUS=localLUname, localLUname[, ...]

This entry is no longer used in SNA Server 3.0.

OtherServers=server1 [server2][ ...]

This variable specifies one or more servers outside of the client's subdomain, to which the client's requests for LUs will be sent, if the requests cannot be satisfied by servers in the client's subdomain. If this variable is not set, only servers in the client's subdomain will receive the client's requests for LUs.

When specifying the variable, substitute one or more server names for
server1 [server2][ ...]. Separate multiple server names with spaces.

This entry is no longer used in SNA Server 3.0.

BackupSponsors=\\server1 [\\server2][ ...] or BackupSponsors=subdomain

This variable specifies a backup subdomain for a client configured to run in local mode or backup sponsor servers for a client configured to run in remote mode. If the client software cannot connect to its primary sponsor subdomain or sponsor servers when it is starting, it will attempt to connect to the backup subdomain or servers. If the string supplied as the value of the variable begins with a double backslash (\\), it is expected to be one or more server names; otherwise, it is treated as a subdomain name.

For OS/2:

COMLUASS={ YES | NO } Default: NO

When this variable is set to YES, LUs on this system are assigned on a per-workstation basis, using the LAN computer name. Otherwise, SNA Server attempts to assign LUs on a per-user basis, using the currently logged-on LAN user identification.

For MS-DOS, Windows version 3.x, and OS/2:

COMPOP12={ YES | NO } Default: NO

When this variable is set to YES, level 12 warnings do not display as pop-up messages on the local system. (Level 12 messages are not as severe as level 16 messages.)

For MS-DOS:

OverrideDefaultLU={ YES | NO } Default: NO

When this variable is set to YES, SNA Server will use the default APPC LUs configured for PC Support; these LUs are specified on the RTLN line of the CONFIG.PCS file. Otherwise, SNA Server will use the default APPC LUs configured in SNA Server; these LUs are assigned to users or groups through SNA Server Manager or through the SNA Server command-line interface (snacfg).

For more information about PC Support, see the following sections in Chapter 3 of the Microsoft SNA Server Getting Started:

  • "PC Support Used with Windows for Workgroups"
  • "Installing an MS-DOS – Based Client," which includes the subsection "PC Support Router Replacement."
