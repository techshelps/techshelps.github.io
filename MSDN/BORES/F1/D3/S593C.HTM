Adding Server Names to Windows NT, Windows 95, and Windows 3.x Clients

When running Setup for clients, the program prompts for two remote server names. You can add additional remote server names using the Registry (for Windows NT and Windows 95 clients) or by modifying the WIN.INI file (for Windows 3.x clients).

Windows NT Client

On a Windows NT client, you can add server names to the following Registry variable:

HKEY_LOCAL_MACHINE
SOFTWARE
Microsoft
SnaBase
Parameters

Sponsors: REG_MULTI_SZ: server1 server2 server3

Enter the server names with a space between each name. The only limit to the number of servers that can be added is the number of servers in the subdomain.

Windows 95 Client

On a Windows 95 client, you can add server names to the following Registry variable:

HKEY_LOCAL_MACHINE
SOFTWARE
Microsoft
SnaBase
Parameters: Sponsors: server 1 server2 server3

Enter the server names with a space between each name. The only limit to the number of servers that can be added is the number of servers in the subdomain.

Windows 3.x Clients

Add server names (preceded by "\\") on the Remote= line in the WNAP section of the WIN.INI, separating the server names with spaces. When connecting over TCP/IP, the server's IP address may be can be specified:

Remote= \\server1 \\server2 \\124.55.69.45
