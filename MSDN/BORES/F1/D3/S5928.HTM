SNA Server Shared Folders Gateway Service:
SNA Servers of Variables

The variables that affect the Shared Folders Gateway service are found in the following Registry location:

HKEY_LOCAL_MACHINE
SYSTEM
CurrentControlSet
Services
SFGW
Parameters

AllocateTimeout: REG_DWORD: Default: INFINITE

This specifies the number of seconds the Shared Folders Gateway waits for an ALLOCATE verb to complete before reporting an error. Other values are 1 to 4294966 seconds.

By default, the Shared Folders Gateway will wait forever for the ALLOCATE verb to complete. If the value is changed, the Shared Folders Gateway will cancel the ALLOCATE verb if it does not complete in the configured time and will also report an error.

AnsiPage: REG_DWORD: Default: 0

This specifies the ANSI code page used in the EBCDIC/ASCII conversion.

ContextCacheSize: REG_SZ: Default: FolderCacheSize + FileCacheSize

This parameter configures the size of the Shared Folders Gateway file/folder context cache size. A context is a data structure that holds the state, such as current file offset, of an open file/folder.

EbcdicPage: REG_DWORD: Default: 37

This specifies the EBCDIC code page used in the EBCDIC/ASCII conversion.

FileCacheSize: REG_SZ: Default: 16

FileCache is a cache that stores currently unused FileObjects. This enhances performance as it is faster to allocate and release FileObjects to and from the cache rather than create a new object and then delete it. The maximum value for this parameter is 256.

FoldersCacheSize: REG_SZ: Default: 16

This is the internal folder cache size. The FolderCache parameter stores unused FolderObjects in the cache in the same manner as FileCacheSize. The maximum value is 256.

ForceUpperCase: REG_SZ: Default: YES

If YES, the Shared Folders Gateway converts user names and passwords to upper case before sending them to the AS/400.

MaxConversations:REG_DWORD: Default: 8

This specifies the maximum number of conversations per AS/400 that the Shared Folders Gateway allocates. The minimum number of conversations allowed is one, the maximum is 64.

MaxWorkBuffers: REG_SZ: Default: 32

This specifies the maximum number of work buffers allowed. These buffers are used to pass data between the shared folders file system driver and the gateway service itself.

MaxWorkThreads: REG_DWORD: Default: 4

This specifies the maximum number of worker threads per processor. The maximum value for this is 64. Worker threads process file system requests such as open file, read file, write file, close file and many others.

Mode:REG_SZ: Default: QPCSUPP

This specifies the APPC mode name. For more information about modes see "What are Modes and How Do I Use Them?" in Part 4 of the Microsoft SNA Server Administration Guide.

RemoteTPName:REG_BINARY: Default: 0x30 0xF0, 0xF4 0xF6

This specifies the partner TP name.

TPName:REG_SZ: Default: SFGW

This specifies the local Shared Folders Gateway TP name. It can be any ASCII string of eight characters. The default is SFGW.

UnloadDriver:REG_DWORD Default: 1

This will unload the shared folders files system driver (SFFSD.SYS) when the Shared Folders Gateway service terminates. If it is set to 0, it will not unload.
