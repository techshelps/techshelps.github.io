Message 659

Invalid configuration file

EXPLANATION: The configuration file running when SNA Server was asked to start a service is invalid. It might be corrupted, or might not be a configuration file.

ACTION: Check to see that the COM.CFG file exists in the directory specified by the COMCFG share on the primary server.

If you need to replace the COM.CFG file, possible sources are the COM.CFG file on a backup server, or files with the extension .SNA, which is the default extension for backup configuration files created with the File menu Backup command. Copy the appropriate configuration file to the filename COM.CFG. Place the configuration file in the SYSTEM\CONFIG subdirectory of the SNA Server root directory.

If the error still occurs, call support services at your site.
