ASSETMIF.EXE: Asset MIF Maker

ASSETMIF collects asset information from clients. ASSETMIF can also be used to assign a user-friendly name to NetWare clients. A LabAsset table is created in the SMS database.

ASSETMIF can be run manually on a client after clisetup.exe has been run, or it can be added to RUNSMS.BAT or smsls.bat after clisetup.exe is run and before inventory is taken.

Before running ASSETMIF.EXE, you must configure ASSETMIF.INI.

Runs On

Alpha, x86

Files Required for ASSETMIF

ASSETMIF.EXE
ASSETMIF.INI

ASSETMIF Syntax

assetmif

ASSETMIF.INI

In this version, the fields below are hard-coded; new ones won't be recognized. ASSETMIF.INI should be in the root of drive C:. It can be a hidden file.

[AssetMif]
IniVersion=1.0

[Identification]
LabAssetNumber=
MachineSerialNumber=
MachineName should match NetBIOS name. If NetWare, use what you like.
MachineName=
Comment=

[Location]
Building=
Room=
Bench=
Comment=

[Attributes]
RAM in kilobytes should match SMS inventory values.
An Alert can compare values.
RamInKbytes=
ExternalHardDriveCount=
InternalHardDriveCount=
Comment=
