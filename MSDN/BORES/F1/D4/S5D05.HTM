NETBOOT.EXE: Network Boot Disk Generator

NETBOOT creates and configures a netboot floppy disk. Such a disk, when inserted in a computer, detects and can use a variety of network cards and network protocols, prompts for a user name and password, and logs you onto the network.

The NETBOOT GUI, NetBoot Administrator, contains folders in the left-hand pane. When you select a folder, the objects it contains are shown in the right-hand pane.

Making a Floppy

By default, all net adapters and all protocols are selected. Therefore, to make a floppy containing all net adapters and protocols, from the Floppy menu, choose Build.

If you want to save space on a floppy disk, you can disable a net adapter or protocol by double-clicking it, which opens its Properties box, and deselecting Use This Device?

Changing Default Settings

The Default Settings object contains global configuration data.

General Tab

The General tab has the following items:

  • User. The contents of this field is placed in the Username field of the logon UI on the floppy. A valid username is expected.
  • Domain. The contents of this field is placed in the Domain field of the logon UI on the floppy. This must be a Windows NT domain name.
  • Protocol. The protocol selected here is used as the default protocol in the logon UI on the floppy. The supported protocols are defined in the Protocols folder.
  • Redirector. The redirector selected here is used as the default on the floppy:
    • BASIC: Does not support specific domain logons.
    • FULL: Requires a domain name.
  • UNC Path to a DOS Environment. This field specifies a complete UNC path to a globally available directory that has two subdirectories: DOS and BIN. The DOS directory contains any MS-DOS 6.22 files that you would like to make available to users of the floppy (you should at least have doskey.exe). The BIN directory contains any other DOS tools you would like to provide.

    This field can be empty, in which case this feature is disabled.

Domain List Tab

  • In the Domain List Tab, enter the names of the Windows NT user account domains available on your network. (This list need not be comprehensive.)

Note Be sure to click Add for each domain you enter.

Changing a Net Adapter's Properties

A Net Adapter object describes a net adapter to the floppy.

Device Properties Tab

The Device Properties tab has the following items:

  • Display Name. The descriptive text displayed to the floppy user.
  • Adapter Name. Used in protocol.ini to represent this adapter. It must be unique.
  • Driver Name. The internal identifier of the driver file. (This can be gleaned from the sample protocol.ini file often provided with a driver.)
  • Driver File. The contents of this field are placed in the value of the netcard line of the [network drivers] section of system.ini.
  • Script File. Indicates that the specified file is a script that performs further adapter-specific configuration.
  • Files to Extract. A list of files that the adapter requires on the floppy. These files must be in the floppy\drivers directory so they can be compressed onto the floppy.
  • Use This Device? Allows an adapter to be left off the floppy to save space, without deleting the information in this Properties box.
Device Settings Tab

In the Device Settings tab, enter the names of the settings needed in protocol.ini to configure this adapter. Then enter the default values for each setting. Use the up and down arrow buttons to sort values. The value at the top of the list is the default value used by the floppy.

Note Be sure to click Add for each setting and value you enter.

Device Detection Tab

The Device Detection tab is for advanced users only. This dialog is not supported.

Changing a Protocol's Properties

A Protocol object describes a network protocol to the floppy.

Device Properties Tab

The Device Properties tab contains the following items:

  • Display Name. The descriptive text displayed to the floppy user.
  • Adapter Name. Used in protocol.ini to represent this protocol. It must be unique.
  • Driver Name. This is the internal identifier of the driver file. (This can be gleaned from the sample protocol.ini file often provided with a driver.)
  • Driver File. The contents of this field are placed in the value of the transport line of the [network drivers] section of system.ini.
  • Script File. Indicates that the file specified is a script that does further protocol-specific configuration.
  • Files to Extract. A list of files that the protocol requires on the floppy. These files must be in the floppy\drivers directory so they can be compressed onto the floppy.
  • Use This Device? Allows a protocol to be left off the floppy to save space, without deleting the information in this Properties box.
Device Settings Tab

In the Device Settings tab, enter the names of the settings needed in protocol.ini to configure this protocol. Then enter the default values for each setting. Use the up and down arrow buttons to sort values. The value at the top of the list is the default value used by the floppy.

Note Be sure to click Add for each setting and value you enter.

Initialization Tab

In the Initialization tab, enter the DOS commands that will initialize this protocol properly. These commands are used in a batch file on the floppy.

Object Management

The left pane of NetBoot Administrator contains folders. Each folder contains objects. Selecting a folder reveals its objects in the right pane of NetBoot Administrator.

To Add an Object to a Folder

Right-click in an empty part of the right pane.

From the pop-up menu, choose New. The new object is called New_Object.

Edit the name by selecting it to highlight it, then clicking the highlighted name and typing. (If the second click occurs too soon after the first, it is interpreted as a double-click, which opens the object's Properties window.)

To Remove an Object from a Folder

Right-click on the object.

From the pop-up menu, choose Delete.

To Open an Object

Double-click on the object.

or

Right-click on the object.

From the pop-up menu, choose Properties.
