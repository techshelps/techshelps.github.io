RSAPPCTL.EXE: Shared Applications Database Resetter

When run on a site server, RSAPPCTL resets the shared applications databases (SADs) of child sites. The SADs are cleared and the shared application information is re-sent from the site server running RSAPPCTL.

Running RSAPPCTL causes the SAD to regenerate itself. The SAD for this site and/or its child sites can be reset.

Runs On

Alpha, x86

Files Required for RSAPPCTL

RSAPPCTL.EXE

RSAPPCTL Syntax

rsappctl {/self | /all | /site:site_code | /siteserver:server}

/self

Resets the shared applications database (SAD) for the site server running RSAPPCTL, but not its child sites.

/all

Resets the SADs for all child sites, but not the site server running RSAPPCTL.

/site:site_code

Resets the SAD for the child site specified by site_code.

/siteserver:server

Resets the SAD for the site server specified by server. Use this option if RSAPPCTL is running on a site other than the one specified by server.

/?

Provides a usage statement.
