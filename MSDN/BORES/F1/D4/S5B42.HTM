Creating the Office Package

When using the SMS program group control to install Office 97, use the PDF files in the Microsoft Office 97 Resource Kit, rather than those included with Office 97. The PDF files in the Microsoft Office 97 Resource Kit support program group control, while the PDFs on the Office 97 CD-ROM do not. If you will be using Program Control Manager to install local copies of Office on client computers, use the PDF files on either the Office 97 CD-ROM or those in the Microsoft Office 97 Resource Kit.

Eight PDF files are included in the Microsoft Office 97 Resource Kit Tools and Utilities:

This PDF

is used for installing

Off97pro.pdf

Office 97 Professional Edition

Off97std.pdf

Office 97 Standard Edition

MSAPPS97.pdf

Office 97 shared components (available with the
Microsoft Office 97 Resource Kit only)

Excel97.pdf

Microsoft Excel 97 stand-alone

Outl97.pdf

Microsoft Outlook 97 stand-alone

Pptsms97.pdf

Microsoft PowerPoint 97 stand-alone

Work97.pdf

Microsoft Word 97 stand-alone

Clipart.pdf

Microsoft Office 97 Clip Gallery


Be sure there is enough room on your distribution servers for the Office 97 package.

To import the package

In SMS Administrator, open the Packages window. From the File menu, choose New.

The Package Properties dialog box appears.

Choose Import.

The File Browser dialog box appears.

Select the PDF you want to use, and click OK.

Choose Workstations.

The Setup Package for Workstations dialog box appears.

In the Source directory box, type the location of the package source directory and click Close.

If you choose to browse through the list of files, edit the resulting path so that it shows the UNC name (which would begin with \\servername\sharename) rather than the relative name (which would begin with the drive letter you have assigned to the server during the current session).

Click OK in all open dialog boxes.

The Office PDFs include the following package options:

Typical

Setup runs in batch mode, installing Office with a Typical installation. For this option, use the command

    setup.exe /Q1 /B1

Custom

Setup runs in batch mode, installing Office with a Custom installation. Use this if you have modified the .STF file. If you do not modify the .STF file, Custom installation run in batch mode is identical to Typical installation.

For this option, use the command

    setup.exe /Q1 /B2

Uninstall

Setup runs in batch mode, removing Office from the client computer. For this option, use the command

    setup.exe /Q1 /U

Manual

Setup runs interactively. If you want users to run Setup interactively, modify the Manual package command to add a /B1 or /B2 command-line option to select a Typical or Custom installation by default.

If you use the Manual package option, users can select a Run from Network Server installation. In order for the Package Command Manager to support a Run from Network Server installation, you must use the Network Installation Wizard included with the Microsoft Office 97 Resource Kit, and then manually edit the .STF file on each distribution server before the package is offered to any clients. If you have not taken these steps, users should be strongly discouraged from selecting a Run from Network Server installation.

For this option, use a command of the form

    setup.exe /Bx

where /B1 specifies a typical installation and /B2 specifies a Custom installation.

In these commands, the /Q1 option directs Setup to run with no user interaction, and the /Bx option specifies the installation type.

To add custom commands to the Office PDF

In the SMS Administrator, open the Packages window, and from the File menu, choose New.

The Package Properties dialog box appears.

Choose Import.

The File Browser dialog box appears.

Select the appropriate PDF.

Click OK.

Choose Workstation if you want to install Office on the client's hard disk.

The Setup Package dialog box appears.

Specify the location of the administrative installation point for Office as the source folder.

To define the new command line, choose New.

In the Command Name box, type a name for the new command.

In the Command Line box, type the command to run your custom
Office Setup.

For example, to run SETUP.EXE from the administrative installation point in batch mode, creating a log file, type:

setup.exe /qt /g  "setuplog.txt"

To indicate that no user input is required to run the package, select the Automated Command Line check box.

In the Supported Platforms box, select the appropriate platforms: Windows 95, Windows NT, or both.

Click OK, and then click Close.
