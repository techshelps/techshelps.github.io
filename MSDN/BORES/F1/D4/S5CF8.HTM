EXECUTE.EXE: Application Executer

EXECUTE runs a two-phase setup program under Package Command Manager (PCM). A two-phase setup is two different programs; the first program starts the second one and then the first one exits. When the first program exits, PCM severs its network connection, leaving the second program without a network connection.

EXECUTE runs the command line for a Windows program, and then polls for its termination. Upon termination, EXECUTE temporarily pauses and waits until no window with the specified window title and window class can be found. (Use SPY.EXE, distributed with Microsoft Visual C++ 1.5, to determine the window class and title for the second phase of a setup application.)

EXECUTE.EXE must reside either in the root directory of the package share, or the logon server MSTEST directory: logon.srv\MSTEST.

Restrictions

EXECUTE is only for 16-bit applications running on Windows 3.1, Windows NT, or Windows 95.

Runs On

Alpha, x86

Files Required for EXECUTE

EXECUTE.EXE

The file EXECUTE.EXE can be found in SMS\SITE.SRV\MAINCFG.BOX\MSTEST? on the SMS site server..

EXECUTE Syntax

execute "command_line" "window_caption" "window_classname"

command_line

The command line to be executed.

window_caption

The string to appear in the title bar of the window running command_line. This can be empty (" "), but not omitted.

window_classname

The name of a window class. This can be empty (" "), but not omitted.

EXECUTE Examples

execute "SETUP.EXE /B4" "Microsoft PowerPoint" "PPApplicationClass"

execute "SETUP.EXE /B4" " " "PPApplicationClass"
