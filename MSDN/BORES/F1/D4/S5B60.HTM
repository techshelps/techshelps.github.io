The Windows NT Predefined Registry Keys

Five keys are predefined in the Windows NT version 4.0 registry: HKEY_CURRENT_USER, HKEY_USERS, HKEY_LOCAL_MACHINE, HKEY_CLASSES_ROOT, and HKEY_CURRENT_CONFIG. When accessing the registry of a remote computer, only two predefined keys, HKEY_USERS and HKEY_LOCAL_MACHINE, appear.

A description of each of the predefined registry keys follows:

HKEY_CURRENT_USER

This key is the root of the user profile for the user who is currently logged on. The user profile contains information that defines the behavior and appearance of the user's desktop, network connections, and other environment settings. HKEY_CURRENT_USER is a subkey of HKEY_USERS.

HKEY_USERS

This key is the root of all user profiles on the computer. It includes HKEY_CURRENT_USER and the default profile.

HKEY_LOCAL_MACHINE

This key contains configuration information about the local computer system, including hardware and operating system data.

HKEY_CLASSES_ROOT

This key contains information that is used to open the correct application when a file is opened indirectly (for instance using file association in the Windows NT Explorer or with Object Linking & Embedding). This key provides compatibility with the Windows 3.1 registration database. HKEY_CLASSES_ROOT is an alias of HKEY_LOCAL_MACHINE\Software\Classes.

HKEY_CURRENT_CONFIG

This key contains information about the current hardware profile of the local computer system. Hardware profiles are sets of changes to the standard configuration of services and devices established by data in the Software and System keys under HKEY_LOCAL_MACHINE. Only the changes appear in this key. HKEY_CURRENT_CONFIG is an alias of HKEY_LOCAL_MACHINE\System\CurrentControlSet\Hardware Profiles\Current. This key was introduced in Windows 95 and Windows NT version 4.0.
