UGMIF.EXE: User Group MIF Creator

UGMIF creates global user group Management Information Format (MIF) files for trusted domains. These MIF files contain either the user groups detected in the domain or, depending on the options used, the differences between what is detected and what exists in the SMS database.

In either case, the MIF files are written by default to the site's SMS Inventory Data Loader in-box, which means the SMS database will be updated automatically with the latest user group information.

Runs On

Alpha, x86

Files Required for UGMIF

UGMIF.EXE
SMSAPI.DLL
OBJECTTY.DLL

SMSAPI.DLL and OBJECTTY.DLL from the BackOffice SDK must be in the UGMIF's path. These files are not shipped with the SMS Resource Kit.

UGMIF Syntax

ugmif [/t] [/v] [/d:domain] [/m:MIF_directory] [/s:site_code] [/u[:SQL_Server[:database[:SQL_account[:SQL_password]]]]]

/d:domain

Specifies a Windows NT domain. User groups will be listed from this domain and all trusted domains of this domain if /t is also used.

/m:MIF_directory

A directory path where the user group binary MIF file will be written. If /m is not specified, the binary MIF file is written to the site's SMS Inventory Data Loader in-box.

/s:site_code

Specifies the site code to use when writing the MIF files. The default is the site code of the site containing the computer running UGMIF.

/t

Includes trusted domains.

/u

Updates an SMS database with MIF files that reflect the difference between the user groups found in the domain and the user groups in the SMS database.

:SQL_Server

The computer running SQL Server containing the SMS database. The default is the site server.

:database

The SMS database. The default is the site database.

:SQL_account

The account to use to access inventory. The default is the SMS SQL Server account.

:SQL_password

The password to SQL_account. UGMIF prompts if this is not specified.

/v

Verbose mode. More output is displayed on the screen.

/?

Provides a usage statement.
