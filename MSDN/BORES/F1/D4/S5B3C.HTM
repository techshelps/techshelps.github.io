Preparing the Clients

When forming your target machine groups, run queries to make sure that each client has enough hard disk space, RAM, and processing capability to run Office. The requirements are outlined in the following table:

Note The SMS inventory records disk space on a per-drive basis, not on a per-computer basis. If the client computers in your organization typically use a small C: partition and a larger D: partition for application and data files, check for disk space on the D: drive.

Before installing Office, run virus-detection, disk-scanning, and defragmentation programs on the client computers. Although the computer may appear to be operating properly, software upgrades often uncover hardware or software problems because of the way they read and write data to the hard disk. Checking the computer before installing Office will help you stay focused on issues related to the installation process.

Be sure to back up critical data and configuration files for the client, in case the installation fails or you need to revert to the old system for some reason. Also, you should create a Microsoft diagnostics report and copy it to a floppy disk by running MSD.EXE on the hard disk. If you will need to restore automatically, consider using a commercial backup program, instead of copying the files by hand.
