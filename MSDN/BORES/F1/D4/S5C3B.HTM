SETCLCFG.EXE: Client Configuration Setter

SETCLCFG forces a client to install the Client Configuration Services, or gets or sets values in the Client Services portion of a client's registry.

Runs On

Alpha, x86

Files Required for SETCLCFG

SETCLCFG.EXE

SETCLCFG Syntax

setclcfg client {/c | /r} component [state]
setclcfg client /f logon_server request_ID
setclcfg client /p parameter [value]
setclcfg client /u remote_computer

Note For any computer name, an asterisk (*) specifies the computer running SETCLCFG.

client

The client to act on.

/c

Displays the Current Installation State of component, or sets component's Current Installation State to state.

/r

Displays the Requested Installation State of component, or sets component's Requested Installation State to state.

component

"Remote Control", "Client Inventory", or "SNMP Trap Translator", and must be typed with quotes.

state

"Installed", "Not Installed", "Started", or "Disabled", and must be typed with quotes.

/f logon_server request_ID

Creates a Client Configuration Request (CCR) on logon_server. The CCR is named request_ID. An asterisk (*) for request_ID causes the request ID to be site_code00001.CCR; for example APE00001.CCR.

/p parameter [value]

Displays the value of parameter, or sets parameter to value. parameter is a ClientServices parameter, such as ClientArchitecture, or Language ID. If the parameter contains spaces, enclose it in quotes (for example, "Log Directory".)

/u remote_computer

Unlocks an access token on remote_computer.

/?

Provides a usage statement.
