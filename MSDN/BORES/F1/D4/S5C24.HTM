MIFNAME.EXE: MIF File Renamer

MIFNAME gives SMS 1.0 and 1.1 MIF files new file extensions based on their content.

The SMS 1.2 dataloader is multithreading for different types of MIF files. The different types are determined by different file extensions on the MIF files. However, SMS 1.0 and 1.1 make no distinction among different types of MIF files (all have a .MIF extension). Therefore, MIF files being passed up a hierarchy from a 1.0 or 1.1 site to a 1.2 site will not take advantage of the 1.2 dataloader's multithreading.

MIFNAME solves this by scanning the current directory (which should be \sms\site.srv\dataload.box\deltamif.col) and changing MIF file extensions according to their content. MIFNAME does this at intervals specified on the command line.

Runs On

Alpha, x86

Files Required for MIFNAME

MIFNAME.EXE

MIFNAME Syntax

cd \sms\site.srv\dataload.box\deltamif.col
mifname [sleep_time]

sleep_time

The number of seconds to wait between checks of the dataload.box directory. If this argument is omitted, only a single check is made. If sleep_time is -1, MIFNAME displays a list of all MIF file types and their quantity, but does not rename any. To rouse MIFNAME from sleep_time mode, press crtl + c.
