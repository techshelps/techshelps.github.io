Deploying Microsoft Office 97 with SMS

You can use SMS 1.2, with Service Pack 1, to deploy Office 97 to clients on your network. A standard PDF file is included with Office 97 to help you install Office locally on client computers using a Typical, Custom, or Complete installation. The Microsoft Office 97 Resource Kit includes a PDF that supports remote Run from Network Server installations, as well as installation modes supported in the standard PDF. While this document assumes you have access to the PDF included in the Microsoft Office 97 Resource Kit, most of the instructions also apply when you are using the PDF on the product CD-ROM.

Before you deploy Office to SMS clients:

  • Review the Office feature set and decide which components to install on the clients.

    You can take advantage of user profiles to require that a mandatory desktop configuration be loaded each time a user logs on. (For more information, see your Microsoft Office 97 documentation.)

  • Decide on configuration and installation options, such as file locations, installation media, and installation method. (For more information, see the Microsoft Office 97 Resource Kit.)
  • Decide which existing files, macros, and custom programs you will need to use in the new software, and identify any extra steps that might be required to use the existing files and macros in Office.
  • Use SMS queries to identify which clients have the hardware required to run Office.
  • Run a test to identify any issues in deploying Office on your network.

The Microsoft Office 97 Resource Kit, from Microsoft Press, is a technical supplement to the Office product documentation. It was written to assist administrators in installing, supporting, and managing Office on corporate networks.
