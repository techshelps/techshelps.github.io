Validate the Windows NT Deployment

After deployment, evaluate the distribution results on the upgraded clients. Note that a Complete job status does not necessarily indicate successful deployment of Windows NT. You must perform additional analysis to determine the results of deployment:

  • Make sure that the SMS client inventory has been updated for all clients in the target group.
  • Use queries to determine whether the upgrade succeeded.
  • Test the clients that were successfully upgraded to ensure that the configuration you specified works as expected on these computers.

Some computers in your target group may not have been successfully upgraded. You can deal with these on a case-by-case basis. Here are a few of the things you can check in these cases:

  • If you marked the Inventory This Package check box in the Setup Package for Inventory dialog box when you created the package, you can examine a copy of the Setup log file created on each target computer. Study this file to determine the cause of any problems. For example, there might have been changes in available disk space or in the availability of some other resource between the time the computer was added to the target group and the time the upgrade was performed.
  • Check the details of job events to troubleshoot problems. For example, if a job has failed because the answer file (for example, UNATTEND.400) was not available where it was expected, this would be reported in the job events.
  • Determine whether any individual clients, which may have failed during the Windows NT upgrade process, are no longer operational.

    Clients that are no longer operational are not updated in the SMS inventory. You might choose to perform the inventory at more frequent intervals before deployment to make sure that only currently operational computers are targeted. In some cases, you might want to develop an SMS query to identify clients that have not had their inventory updated.

  • Determine whether a faulty hardware component caused a failure of the Windows NT upgrade process.
