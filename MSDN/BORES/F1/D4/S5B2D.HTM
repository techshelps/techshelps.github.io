Preparing the Source Directory

Use the following procedure to create and populate a package source directory for your Windows NT Workstation and/or Windows NT Server source files. This directory will also contain the answer files (UNATTEND.TXT) and uniqueness database files (UDFs) that allow you to deploy Windows NT without user intervention at the target computers.

To create a package source directory

Create a source directory for the Windows NT operating system files. Within this source directory, create subdirectories for each processor type (i386, Alpha, or MIPS).

Share the directory using a share name that is identical to the directory name.

From a client computer or from the server containing the package source directory, log on to the network and connect to the share you created.

Insert the Windows NT CD-ROM and use the xcopy command to install the operating system files in the package source directory. Copy all the files for every platform supported in your organization, preserving the directory structure on the CD-ROM or installation disks. For example, if your organization uses x86-based computers but not Alpha or MIPS computers, you would need to copy the \i386 directory tree (but not the \Alpha and \MIPS directory trees) to your package source directory.

Copy the UNATTEND.TXT file from the Windows NT 4.0 CD-ROM to the source directory.

  • If you will be targeting Windows NT systems for upgrade, copy UNATTEND.TXT to NTUPGRD.400.
  • If you will be targeting MS-DOS or Windows 3.1, Windows for Workgroups, or Windows 95 clients for upgrade, copy UNATTEND.TXT to UNATTEND.400.

Modify the UNATTEND.400 and NTUPGRD.400 files for installation as required by your configuration. See the Windows NT Workstation Resource Guide for details. (This book is included online in the Windows NT Server Resource Kit).

In the UNATTEND.400 file, leave the ComputerName entry (in the [UserData] section) blank if you want SMS to fill in this value using the one specified in the SMS.INI file, or if you will be using a Uniqueness Database File (UDF) to tailor the installation to individual computers. For information on creating answer files and UDF files see Part I, "Deployment Guide," in the Windows NT Workstation Resource Guide.

Copy your UDF and any other files used in a customized setup to the package source directory. If you will be installing applications along with Windows NT, the files for those applications also need to be copied to the source directory. Chapter 2, "Customizing Setup," in the Windows NT Workstation Resource Guide explains the directory structure to use for these files.
