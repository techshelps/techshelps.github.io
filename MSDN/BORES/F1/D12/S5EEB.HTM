Using the Internet Mail Service Sample Extension DLL

When initialized, the Internet Mail Service Sample Extension DLL loads a table named Imcroute.cfg from the configuration file you created. This file is in the directory specified in the RootDir Internet Mail Service registry value.

Comparing Domain Names

The Imcroute.cfg table contains a list of domain names and associated SMTP hosts. For each recipient of every inbound message the Internet Mail Service receives, the Internet Mail Service Sample Extension DLL must compare the domain name on the address to the list of domain names in the table. If a match is found, the message for that particular recipient is rerouted to the associated host.

The match is compared to the rightmost part of the domain name on the recipient address. For example, if the domain name in the table is sea.com, it matches an address of x.sea.com, as well as sea.com. The list is searched from top to bottom, and the first match is used. It is recommended that you list more specific patterns before general patterns to ensure that the names are found.

In cases where the associated SMTP host is localhost, domain names in the list represent names that should be handled by the Internet Mail Service. If the recipient address matches one of these domain names, the Internet Mail Service processes the recipient normally and attempts to deliver the message to the Microsoft Exchange Server. If no match is found, the Internet Mail Service Sample Extension DLL reroutes the outbound message to the domain name in the recipient's address. For this reason, it is important to ensure that any domain names that must be routed to the Microsoft Exchange Server computer are listed in the configuration file. In particular, any domain names used in SMTP addresses for Microsoft Exchange users must be defined in the configuration file. Otherwise, reports and replies will not be deliverable and could cause routing loops.

It is possible to introduce a message routing loop if the configuration file contains incorrect or incomplete information. It is recommended that you enable the DisableLoopbackConnections Internet Mail Service registry value to reduce the chance of a message routing loop occurring.

Configuration File Example

The following is an example of a configuration file.

! Internet Mail Service Extension DLL Configuration File

! Event log parameters

EL_EventLogging

ReRouted

EL_EventLogging

NotRouted

Listserv.granite.com

listserv.granite.com

granite.sea.com

localhost

sea.com

mail.sea.com

granite.com

localhost


Lines beginning with an exclamation point (!) or a blank line are comments and are ignored. The keyword EL_EventLogging controls whether events are recorded to the Windows NT Workstation Event Log for messages that are processed. EL_EventLogging must be followed with ReRouted or NotRouted. ReRouted indicates that an event is recorded whenever an outbound message is rerouted. NotRouted indicates that an event is recorded whenever a message is not rerouted.
