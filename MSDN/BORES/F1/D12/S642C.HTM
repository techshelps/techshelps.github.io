Using Mapindl

Before using Mapindl, make sure that one of the following development environments exists on your computer.

  • Microsoft Exchange Client 4.x
  • Lotus Notes Client 4.x

You can use the Mapindl.Attach object provided by the Mapindl tool as part of a LotusScript action in a Lotus Notes database application. Likewise, standard Active Scripting (Microsoft Visual Basic Script or Microsoft Visual Basic for Applications) can be used in the same way from other applications. However, other means must be used to acquire the appropriate Lotus Notes identifier strings.

The following steps are typically required to implement Mapindl.

To enable Microsoft Exchange Server notifications of Lotus Notes applications

Create or identify a Lotus Notes application that you want to enable Microsoft Exchange Server notifications. This application should be located on a server so that the application's links remain valid from multiple computers. However, you can also use a local database for development and testing.

Configure a Microsoft Exchange Server MAPI profile to connect to a valid messaging system. If profile selection is enabled, the profile name can be used in Active Messaging to prevent repeated prompts during log on.

In the Lotus Notes application, create an action in a view or form that can run a LotusScript. In the script, perform the following steps:

  • Obtain the necessary database, view, and document IDs of the link target.
  • Create an Active Messaging session from the MAPI.Session object.
  • Log on to the correct messaging profile by using the .Logon method.
  • Create a new MAPI message in the default mailbox outbox by using the .Outbox.Messages.Add method.
  • Set the desired message properties for the recipient, subject, and body of the message.
  • Attach the desired Lotus Notes DocLink to the message by using the objMAPINDL.AttachLink method.
  • Send the message.
  • End the MAPI session.
