Using Replver

When Replver starts, it searches for a file called Replver.ini in your local directory.

To run the Replver tool

From the Config-Topology menu, choose Add Server.

Provide the appropriate information for your Microsoft Exchange organization, including the site and server name for each server you want to verify.

Start the comparison process by choosing the Start on the Run menu, or by choosing the Preferences-Options menu and changing the default settings of the tool.

Batch-Mode Operation

Replver can also run in a batch mode. For Help on running Replver in batch mode, type the following at the command prompt:

replver /?

The following options are displayed.

Option

Description

/b

batch mode

inifile

input filename (default = replver.ini)

output

output filename (default = replver.out)


Following is the typical command-line syntax usage for running Replver:

ReplVer [/b] [/I inifile] [/o output]

Note File names specified through the command line cannot contain spaces.

Verification Options

There are two parts to the Verification Options window. The first is output, and the second is the criteria used during the verification process. The Output Window Options section controls the amount of information placed into the log file for the tool.

The output options are shown below.

Option

Description

Show Time

Logs the date and time on the output.

Output to file

Writes all logging information to the file specified. Both universal naming convention (UNC) style and drive designator formats are supported when specifying the name and location where the log file will be created.

Overwrite Previous File

Overwrites an existing log file if selected.

Starting Folder

Controls the specific folder where the tool begins its verification.

Compare site folders alone

Compares only the special site folders (Schedule+ free/busy and the offline Address Book). This option verifies that there are no duplicate site folders from any site and that the local site folder's entry identification was obtained. It also correctly matches the local site's folder.

Folder tree root

Compares only folders starting from the root folder of the home server. The default is All Public Folders.

Other

Compares only the specific folder entered for the master server. If the Compare subfolders too option is also selected, then Replvr drills down the folder hierarchy.

Browse

Retrieves the complete public folder hierarchy and takes time to complete.


Comparison Options

Comparison options control what is compared between the home folders and replicated folder. The options are described in the following table.

Option

Description

Compare subfolders too

Starts from the specified starting folder and recursively compares the subfolders found within each subsequent hierarchical level. If it is not checked, only the starting folder is compared.

Compare messages

Compares all messages found in the folders. That is, verifies that the messages found in the home server are also on the replicated servers. However, the contents of the messages themselves are not compared.

Compare MAPI Properties

Ccompares all the supported MAPI properties within the folders and for the messages as well. If this option is not selected, then property-level verification will not be completed, and only folder mismatches will be reported.

Continue after Mismatches

Continues comparing other folders even if it finds mismatches on some folders. If this option is not selected, Replver stops after the first mismatch.


Replver Output Tags Configuration

Enabling different tags makes it possible for the advanced user to control the information to log, both to the Replver window and to the Replver logging file. However, you cannot control them independently.

The tag numbers 00 through 08 are used to specify the relevance of a log entry. The following table lists the tags and their descriptions.

Tag

Description

00: Start of new folder

Displays the next folder that is being compared in the Replver window and in the logging file.

01: Start of new message

Displays information about the messages that are being compared in the folder. Note that this information is only displayed once for all messages in a given folder.

Tag 02: Folder Mismatch

Displays information about folder mismatches.

03: Message Mismatch

Displays information about message mismatches.

04: Folder property dump

Displays a dump of all folder properties into the log file.

Note This setting configures Replver to create an extensive dump of all folder properties and will cause the master server and other servers to become bogged down by the imposed demands of this comparison. This information is provided in very raw form and is usually only valuable for development purposes.

05: Message property dump

Displays a dump of all message properties.

Note This setting configures Replver to create an extensive dump of all message properties and will cause the master server and other servers to become bogged down by the imposed demands of this comparison. This information is provided in very raw form and is usually only valuable for development purposes.

06: Folder property mismatches

Displays file information about folder properties that are mismatched.

07: Message property mismatches

Displays information about the message properties that are mismatched.

08: Summary

Selected by default. Displays a brief summary of all the errors and warnings that Replver found. It is highly recommended that this option remains enabled.

Replver.ini File

The Replver.ini file lists supported flags. The following table includes these flags and their descriptions.

Section

Flag and Setting

Description

[Debug]

Superuser=0 or 1

Default is 0.

If set to 1, Replver attempts to log on as the store owner, which enables you to work around access control list (ACL) restrictions. From that point on, the profile Replver creates also has super user privileges.

ListOnly=0 or 1

Default is 0.

If set to 1, Replver attempts to log on to the master server only and lists the entire public folder hierarchy into a file called Folders.out. This flag is useful for periodically verifying how the folders are growing. If this option is selected, then the Compare site folders alone option is automatically disabled.

TouchAll=0 or 1

It is highly recommended that you do not set this to 1. If you do, Replver affects every specified folder. In other words, Replver makes a change on every folder that will be replicated, and as a result, you will flood the system with replication mail.

NeverCreateProfile=0 or 1

Default is 0.

If set to 1, Replver will use existing profiles rather than trying to create new ones (or it will fail if the profile does not already exist).

[Replicas]

This section controls the replica list changes on public folders. The changes are managed from the master server in the list.

ReplicaAdd=0 or 1, ReplicaRemove=0 or 1, ReplicaSet=0 or 1

Only one (or none) of these should be set to 1. The following line controls which servers get added/deleted/set: ReplicaList=DN1,DN2,DN3.

This line represents a comma-separated list of server DNs. Server DNs are of the form:

/o=Organization/ou=Site/cn=Configuration/
cn=Servers

/cn=ServerName/cn=Microsoft Public MDB

If ReplicaAdd is set, these servers are added to the existing replica list of the folders.

If ReplicaRemove is set, these servers are removed from the existing replica list of folders.

If ReplicaSet is set, these servers are set to be the replica list of the folders (thus replacing the existing replica list).

These replica operations are started from the "Starting folder" specified in the Options property page and applied recursively, even if you have cleared the Compare Subfolders option.
