Saving Organizational Forms

If you have organization-wide forms, follow the procedure below.

To save your organizational forms

While still in the Microsoft Exchange Client, create a new personal folder called "OrgForms."

Select the new folder.

From the File menu, choose Properties.

Select the Forms tab.

Choose Set, and then select Organization Forms. This is the name of the folder that was specified using the Microsoft Exchange Administrator program (under Tools\Forms Administrator).

Select all the forms listed in the list on the left, and with all the forms highlighted, choose Copy. This copies all the selected forms to the list on the right. All the forms in the list are then copied to the OrgForms personal folder that you created in step one.

Repeat these steps for each forms library.

As mentioned earlier, you cannot copy the Pub.edb file from Server A to Server B and expect all the public folders to be available on Server B. After you copy Pub.edb to Server B, and run the ISINTEG troubleshooting utility and the DS/IS consistency adjuster, the public folders on Server A will still not be available on Server B. Only after you display the public information store properties on Server B, select the Instances tab and add all the public folders to be homed on Server B, will these folders be available.

Note The Instances tab is not available with Microsoft Exchange Server, Standard Edition. Because Standard Edition servers are stand-alone servers, they do not have the ability to store replicas of public folders.
