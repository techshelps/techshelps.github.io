Using the Administrative Mailbox Agent

To use the Administrative Mailbox Agent for creating new accounts, send a message to the Administrative Agent with "NEW" as the subject heading. The body of the message must include the Windows NT account name, and the first and last names of the new Microsoft Exchange user.

To create new accounts

In the Microsoft Exchange Client Viewer, open a new message.

In the message, choose To, and then select the name of the Agent from the Address Book. For example, if you chose the name "Admin Agent" for the Agent during installation, then this name will appear in the Address Book.

In the Subject box, type NEW.

If you are administering more than one messaging container, type the name of the container in X.500-style notation. For example, if you want to request a new user for the container "/cn=Recipients/cn=Office1042," type NEW /cn=Recipients/cn=Office1042 in the Subject box.

In the message body, type one or more lines in the following format:

<Windows NT account name>,<Microsoft Exchange first name>,<Microsoft Exchange last name>

If you have more than one container to administer, the message sent will include the container name after the word "NEW" in the Subject box.

After you send your request, the Administrator Mailbox Agent creates Windows NT and Microsoft Exchange accounts with the specified Windows NT account names and corresponding first and last names.

If you have permission to use the Agent, a confirmation message is sent, as shown in the figure below. This message lets you know that the accounts have been created. As mentioned earlier, you must have the appropriate permissions to create accounts with the Administrative Mailbox Agent. If you do not have permission to create accounts, a message is returned stating that fact.

Deleting Accounts

To use the Administrative Mailbox Agent for deleting new accounts, send a message to the Agent with the word "DELETE" in the Subject box and message body text that contains the name of the Windows NT account and the first and last name of the new Microsoft Exchange user.

Granting Access Permissions

The Administrative Mailbox Agent needs to know which administrators have the access permissions required to request the creation or deletion of accounts. You can grant access permissions to other administrators by using the Admins.csv file.

The Admins.csv file is usually located in the directory where the Administrative Mailbox Agent was installed (for example, C:\Admin). Note that the directory where Admins.csv is stored must be indicated somewhere in the system path for Microsoft Exchange Server to find it.

To add the Admins.csv directory to your system path

In Control Panel, double-click the System icon.

Select the Environment tab, and then type the directory path in the Variable box.

To ensure that the directory path is added to your system path, choose Set.

The Admins.csv. file is a text file that contains a record for each administrator in comma-separated-value (CSV) format. This file can be edited in Notepad or any other text editor. You can also edit Admin.csv by using Microsoft Excel (which natively handles CSV files), as shown in the figure below.

Admins.csv Fields

  • Display Name — The first field for each record includes the display name of each administrator. If the administrator's name is "Lee, Bill," then the first field on the line should read "Lee, Bill." The quotation marks are necessary due to the embedded comma, but if you are editing in Microsoft Excel, Microsoft Excel places the quotation marks around the field automatically.
  • Server — The second field displays the name of the Microsoft Exchange Server computer that all the administrators' accounts are stored on.
  • Recipients Container — The third field displays the Recipients container for all users. When working in this field, use X.500-style directory notation. For example, to place users in the top-level Recipients container, type /cn=Recipients.

    To place users in the container called "Office1," type /cn=Recipients/cn=Office1.

  • Windows NT Domain — The fourth field displays the Windows NT domain that you want to create Windows NT accounts in, or link to. If this field is left blank, the current Windows NT domain for the Microsoft Exchange Server computer is assumed.
  • Permissions Levels
    • Level 3: Set the permissions level to 3 if you want to enable an administrator to delete Windows NT and Microsoft Exchange accounts with the Administrative Mailbox Agent
    • Level 2: Set the permissions level to 2 if you want to enable an administrator to create Windows NT and Exchange accounts with the Agent.
    • Level 1: Set the permissions level to 1 if you want to enable an administrator to link to previously created Windows NT and Exchange accounts, but not to create or delete accounts with the Agent.

    Note You can enter the same administrator's name more than once in the Admins.csv file. In fact, you need to do so if the administrator is responsible for more than one container. However, this forces the administrator to send requests that specify which container to use.

To start or stop the Administrative Mailbox Agent

In Control Panel, double-click the Services icon.

Select the Administrative Mailbox Agent from the list of services, and then choose Start or Stop.
