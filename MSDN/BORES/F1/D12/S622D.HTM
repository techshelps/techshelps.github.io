Acctonme Tool

The Acctonme tool completely rebuilds the postoffice address list. It can be used with Microsoft Mail version 2.1 and later postoffices.

Acctonme does not directly access the global address list (GAL) in Microsoft Mail versions 3.0 and 3.2. Because the GAL contains the names from the local postoffice, it requires rebuilding after this process is complete. For more information on rebuilding the GAL, see your Microsoft Mail documentation

When to Use Acctonme

Use Acctonme only when both the Admin.nme and Adminshd.nme files are deleted or irrevocably damaged. If only one of the files is damaged or missing, you can replace the damaged file with a copy of the undamaged file.

Side Effects of Using Acctonme

Running this tool will result in the loss of all external postoffice names, gateway names, and local group names in the postoffice address list. Given the structure of the address list file and its connection to the group subsystem of the database, you must delete all postoffice groups before proceeding.

Note As with all Microsoft Mail utilities, it is highly recommended that you back up your Microsoft Mail database before running Acctonme. You must also back up the Admin.nme file in the NME subdirectory by typing the following at the command prompt in the Microsoft Mail Database subdirectory:

copy \nme\admin.nme \nme\admin.old

As a precaution, print a report of all postoffice groups by starting the Microsoft Mail Administrator program (Admin.exe) and selecting Local-Admin, Group, Print, <File or Printer>. If necessary, you can use this list later to recreate the groups.

To rebuild your postoffice list by using Acctonme

Delete all postoffice groups by starting the Microsoft Mail Administrator program (if you haven't already), and then select the Local-Admin, Group.

Delete each group in the postoffice.

If the group has an associated group folder, you will be prompted to delete the folder.

Switch to the drive and path where the Microsoft Mail database is located and then set the Admin.exe file in the NME subdirectory to zero bytes by typing the following command at the command prompt:

nul > \nme\Admin.nme

When you type this command, you may receive a "File not found" error message. Disregard this message.

From the root directory where the Microsoft Mail database is stored, run the ACCTONME tool. It will rebuild the postoffice address list so that it contains only local mailbox names (no messages are generated upon completion).

Copy the Admin.nme file to Adminshd.nme by typing the following command from the root directory where the Microsoft Mail database is stored:

Copy \nme\admin.nme \nme\adminshd.nme

To recreate the postoffice groups, select Local-Admin, Group, Create in the Microsoft Mail Administrator program. Use the list generated as a guide to recreating the postoffice groups.

To recreate the lost external user names, choose the Address menu option in the Mail Administrator program or use the Import tool.

If you are using Microsoft Mail 3.0 or 3.2, start Rebuild.exe to recreate the global address list.

Restoring Postoffice Address Lists

Any changes in the postoffice address list since the last backup, such as new users, will be lost when you restore the list. Therefore you need to restore the Admin.nme, Adminshd.nme, Access.glb, Access2.glb, and Access3.glb files from the backup files.

Adding a Lost Name into the Address List

If you are rebuilding the Admin.nme file because a record is missing (for example, you see a particular user name from the Microsoft Mail Administrator program, but you do not see the user when you try to compose a message), use the following procedure.

To add a name back into the address list

If there is not a 00000000.nme file in the Nme subdirectory of the Microsoft Mail database, create a new user and substitute that user's eight-digit ID number for 00000000 in the following procedure. The user's eight-digit ID number can be found by running the Listuser.exe program located on the database utilities disk.

In the NME subdirectory of the Microsoft Mail database, copy 0000000.NME to 00000000.old. (If the file is 0 bytes long, do not perform this step.)

In the same directory, copy Admin.nme to 00000000.nme, and then make the current directory the root of the Microsoft Mail database.

Start the Microsoft Mail Administrator program and then select Local-Admin, Modify. Select the user name to be added to the address list and then record the mailbox name and alias. Return to the top level of the Microsoft Mail Administrator program, select Config, Password, and record the network and postoffice names.

Quit the Microsoft Mail Administrator program and start the MS-DOS client.

Log on as the administrative user.

Add the missing user to the personal address list by choosing Address, Enter (select Microsoft Mail Address Type if you have any gateways installed), and then selecting the mailbox, postoffice, network, and alias names.

Quit the Microsoft Mail program and then switch to the NME subdirectory of the Microsoft Mail database. Copy 00000000.nme to Admin.nme and Dminshd.nme.

In the same directory, copy 00000000.old to 00000000.nme or create a 0-byte 00000000.nme file by typing the following at the command prompt:

Type aaa > 00000000.nme

When you type this command, you will receive a "File not found" error message. Disregard this message.

Removing a Record

If you are rebuilding the Admin.nme file because you are trying to remove a record (for example, if you see a particular user when you try to compose a message, but you do not see the user from the Mail Administrator program), use the following procedure:

To remove a record

If there is not a 00000000.nme file in the NME subdirectory of the Microsoft Mail database, create a new user and substitute that user's eight-digit ID number for 00000000 in the following procedure. Then use Listuser.exe (located in the database utilities disk) to determine the ID number for that user.

In the NME subdirectory of the Microsoft Mail database, copy 0000000.nme to 00000000.old. However, do not perform this step if the file is zero bytes long.

In the same directory, copy Admin.nme to 00000000.nme, then make the current directory the root of the Microsoft Mail database.

Start the Microsoft Mail program and log on as the administrative user.

To remove the record, from the Address menu, choose Delete (or Remove if you are running Microsoft Mail). Select the user and then press ENTER.

Quit Microsoft Mail.

Switch to the drive where the Microsoft Mail database is located. In the NME subdirectory, copy 00000000.nme to Admin.nme and Adminshd.nme.

Restore 00000000.nme from 00000000.old, or if the file was zero bytes long, you can zero out the file by typing the following at the command prompt:

aaa > 00000000.nme
