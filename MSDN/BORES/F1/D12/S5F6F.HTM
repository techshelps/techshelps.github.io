Using PKicker

When a proxy is added or updated, only mailboxes that currently have a proxy of that type are updated. The only way to complete this task is to export all the recipients and then import them back in. The PKicker tool enables a user to add a new proxy and then force all the mailboxes to be updated. PKicker can also delete proxies, thus removing the specified proxy from all your mailboxes, not custom recipients.

To run the Pkicker tool

  • Use the following command-line syntax

    PKicker /T<proxy type> [/S<server>] [/F<n>]

The following table lists the command-line options available in PKicker, and their descriptions:

Parameter

Description

/S

Server name. (default: local server)

/T

Type of connector, for example Lotus cc:Mail.

/F

Desired action. The default is ADD where ADD is // Add new proxy to all users, DEL is // Del a current proxy and UPD is // Update the proxy.


Performing Specific Functions with PKicker

The following procedures outline how to use the PKicker tool to perform specific functions:

To access the Help menu

  • At the command prompt, type:

    pkicker /?

To add a Lotus cc:Mail proxy on the current Microsoft Exchange Server computer

  • At the command prompt, type:

    pkicker /TCCMAIL /FADD

To update a Lotus cc:Mail proxy on another server

  • At the command prompt, type:

    pkicker /TCCMAIL /SOtherServer /FUPD

To delete a Lotus cc:Mail proxy on another server

  • At the command prompt, type:

    pkicker /TCCMAIL /SOtherServer /FDEL
