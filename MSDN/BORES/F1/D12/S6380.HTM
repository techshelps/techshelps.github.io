Consistency

To verify the consistency of a database, you can use the EDBUTIL utility with specific switches. The correct syntax is edbutil /c <database name> [options]. The options are zero or more of the switches as detailed in the table below. The switches must be separated by a space.

Switch Option

Description

<Database name>

File name of database to verify, or /ispriv, /ispub, or /ds.

/a

Checks all nodes, including deleted nodes.

/k

Generates key usage statistics.

/p

Generates page usage statistics.

/t<name>

Performs a check on the specified table only (default: checks all tables in the database).

/o

Suppresses the logo.


The consistency checker performs no recovery and always assumes that the database is in a consistent state. If the database is not in a consistent state, an error message is returned. The /ispriv, /ispub, and /ds switches use the registry to automatically set the database name for the appropriate Microsoft Exchange Server store (public or private).
