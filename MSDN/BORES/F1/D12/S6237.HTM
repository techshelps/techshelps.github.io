Dumpidx Tool

The Dumpidx tool generates a diagnostic dump of a folder index file. Dumpidx works with Microsoft Mail 2.1 and later postoffices. However, it does not dump information from Microsoft Mail 3.0 or 3.2 Mail message files (.mmfs).

When to Use Dumpidx

Use this tool to verify the contents and/or the integrity of a folder index file.

Side Effects of Using Dumpidx

There are no side effects from using this tool because it performs a read-only function on the folder index file.

To run Dumpidx

  • Type the following at the command prompt (with an option):

    dumpidx -[ecpns] <index_file name>

The following table lists the command-line options available in Dumpidx, and their descriptions:

Option

Description

-e

Displays the same information as the -c option, with additional information about the associated folder file. This includes the name of the folder, access rights, and the number of items in the folder.

-c

Displays the header information for the selected file. This includes the location in the file, attributes, and file name, comment and owner fields for each record in the index file.

-p

Displays the folder names for each of the records in the index file.

-n

Displays the folder file names associated with each record in the index file.

-s

Displays the same information as the -c option, with size information for each folder file.

<index_file name>

The index file in the current directory for which information is to be dumped.


Example for Using Dumpidx

To dump folder size information for all items in the index file 00000002.idx to the computer screen, type the following at the command prompt:

dumpidx -s 00000002.idx

Typically, .idx files are stored in the Folders subdirectory. Private folder indexes are stored in the Loc\000xxxxx directory, where 000xxxxx is the ID number for that user. Public, group, and shared folders are stored in the Pub subdirectory. Index files can also be stored locally on a user's hard disk drive or on an alternate network location.

To direct the output to a file, use the MS-DOS redirection symbol ">" and a file name. The following command will direct the output of Dumpidx to the file Results.txt:

dumpidx -s 00000002.idx >results.txt
