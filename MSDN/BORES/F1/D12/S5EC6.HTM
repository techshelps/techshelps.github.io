Using Crystal Reports

When you use Crystal Reports, you can save your report in a variety of file formats, including .rpt, .doc, .xls, .rtf, .and .txt. You can also choose to save the report in a variety of locations, including a Microsoft Exchange folder, as an attachment in e-mail, or as a separate file.

To start Crystal Reports

In the Microsoft Exchange Client Viewer, choose the View menu.

Select a report option.

Report Types

The following table lists the reports available in Crystal Reports:

Option

Description

Personal Reports

Displays the currently defined Personal Reports templates that are applicable to any folder as submenu items. By selecting a specific Report template, you start the Crystal Reports Analyzer for the selected Report template.

Folder Reports

Displays the currently defined Folder Reports templates that are defined for this folder as submenu items. By selecting a specific Report template, you start the Crystal Reports Analyzer with the selected Report template.

Define Reports

Displays the folder's property page for Views, Report Templates, and Forms for the currently selected folder.

Crystal Reports Viewer for Microsoft Exchange

The Crystal Reports Viewer opens each time you choose a Folder Report, Personal Report, or Folder Instance file from the Microsoft Exchange Client View menu. The Viewer enables you to view an existing Report Template or Report Instance in a variety of ways without changing its design. While working in the Crystal Reports Viewer, you can export the report to a Microsoft Exchange folder or disk, send it as an attachment, drill down for more details, modify the selection formula, and perform searches.

The Viewer is activated as a result of file association of *.rpt to the cvwxx.exe program. This enables you to start the analyzer for a selected report by double-clicking that report. If you choose several reports from the View menu, the Viewer displays multiple windows, one for each report.

Below is an example of the Crystal Reports Viewer with a sample Helpdesk report. While working in the Viewer, you drill down to view details on specific elements of this report.

In addition to the destination types already provided (disk file, MAPI user, VIM user), a Microsoft Exchange folder is also available as a report destination. When the report is run with its destination set as a Microsoft Exchange folder, the completed report appears as a new item in the indicated folder. This can be any folder you have access to, including a public folder on a Microsoft Exchange Server computer.

To view an existing Crystal Report

  • In the Microsoft Exchange Client Viewer, double-click a report in a folder; or select the report and press enter.

To view report properties

In the Microsoft Exchange Client Viewer, from the Public Folders menu, choose All Public Folders.

Select a public folder, and then right-click on the folder.

From the shortcut menu, select Reports.

You can view properties and options in the following tabs: General, Views, Design Overview, Forms, and Reports. Note that the Reports property page has the same look and feel as the View property page. When a folder design is copied to another folder, its associated views and reports are copied as well.

The Reports property page displays options for either the current Folder Reports for the selected folder or the Personal Reports that have been defined by the user. When a report is highlighted, the report title and description appear in the lower pane.

The following table lists options in the Report property page:

Using the Crystal Reports Designer for Microsoft Exchange

You can also use the Crystal Reports Designer to create or modify reports. The Designer provides a list of Microsoft Exchange columns that you can choose from. After you choose a column, you can drag and drop it into a report. You can then customize your report by adding text and formatting the data, fields, formulas, and summary operations.

To start the Crystal Reports Designer

In the Microsoft Exchange Client Viewer, select a public folder, and then right-click on the folder.

From the shortcut menu, choose Reports, choose New, and then select Crystal Reports Designer.

Specifying a data source

You can use the Crystal Reports Designer to create a Microsoft Exchange-specific report by choosing Microsoft Exchange Data as the data source for a new report. If you are accessing a folder or the Address Book, you must select a MAPI profile for logon.

The following three data sources are provided to the Microsoft Exchange user.

  • Microsoft Exchange folder
  • Address Books
  • Message Tracking Logs

In addition to the destination types already provided (disk file, MAPI user, VIM user), a Microsoft Exchange folder is also available as a report destination. When you run the report with the destination set as a Microsoft Exchange folder, the report appears as a new item in the selected folder.

Microsoft Exchange Server Administrator Reports

Crystal Reports provides the functionality to create reports for two types of Microsoft Exchange Administrator objects: Microsoft Exchange public folder properties and mailboxes on the server. However, this functionality is not included with this compact disc. To view these objects, you must be running Crystal Reports from a Microsoft Exchange Server computer.

Microsoft Exchange public folder properties

Crystal Reports enables you to specify which information you have permissions to, where the public folder is replicated, and the storage limits.

The reporting properties are:

  • Access Control Lists (ACLs)
  • Age Limit for Replicas
  • Folder Name
  • Folder Path Name
  • Issue Warning Storage Limit (K)
  • Last Access Time
  • Newest Message Creation Time
  • Number of Messages
  • Oldest Message Creation Time
  • Replica List
  • Storage Quota (K)
  • Total Message Size (Byte)
  • Use Default Storage Limit
Microsoft Exchange mailbox properties

You can also report mailbox information on the server, permissions, which server is used, and message size limits.

Following is the list of mailbox properties:

  • Home Server
  • Issue Warning Storage Limit (K)
  • Last Logoff Time
  • Last Logon Time
  • Mailbox Name
  • Max Incoming Message Size (K)
  • Max Outgoing Message Size (K)
  • Permissions
  • Primary Windows NT Account
  • Prohibit Send Storage Limit (K)
  • Storage Limit Information
  • Total Message Size (Bytes)
  • Use Default Storage Limit
