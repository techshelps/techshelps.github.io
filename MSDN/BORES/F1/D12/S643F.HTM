Using the Microsoft Mail SMTP Gateway Proxy Migration Tool

To run the Microsoft Mail SMTP Gateway Proxy Migration Tool

  • Type the following at the command prompt:

    MIGSMTP [-?] [-R] [-V] [-Ccontainer] [-Taddresstype] -sserver -pPOpath

The following table lists the command-line options available, and their descriptions:

Option

Description

-?

Displays Help.

-R

Replaces the primary simple mail transfer protocol (SMTP) proxy name. If not specified, the Microsoft Mail SMTP proxy name is appended as a secondary SMTP proxy name.

-V

Verbose progress reporting.

-C

Processes only the recipients in container. If -C is not specified, only the Recipients container will be processed.

-T

Uses addresstype as the type of the generated address instead of SMTP.

-s

server is the name of the Microsoft Exchange Server computer to run the process on.

-p

POpath is the path to the MicrosoftMail postoffice. The Addrmap.cfg and Local.cfg files are required by the proxy generation process. These files must be located in the path specified either in a live database in the directory, or as separate files in the directory.


Migsmtp.exe adds an SMTP secondary proxy for the user that is located by converting the Microsoft Mail address (type MS in Microsoft Exchange Server). The Microsoft Mail address is searched for in the following order:

The target address in a custom recipient

The primary proxies

The secondary proxies

This transformation is not performed on custom recipients with a target address of type SMTP. Address generation for SMTP addresses also takes place for distribution lists and recipients hidden from the Address Book.
