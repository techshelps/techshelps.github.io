Using the EFD Handcode Agent

The EFD Handcode Agent program includes a sample project file (listdemo.efp) that demonstrates how to use handcode to link two list boxes together. This sample file is located in the Listdemo folder.

You can also find EFD-generated source-code files and three additional files in the Listdemo\Vb subfolder.

Warning Do not delete the following files: Handcode.ini, Handcode.bas, and Handcode.rtf.

Handcode.ini contains the list box data.

Handcode.bas contains global variables and subroutines for populating the list boxes from the Handcode.ini file.

Handcode.rtf is the script file that Efdcoder uses to reinsert the necessary handcode into the EFD source files.

To install the Efdcoder form

From the Start menu, choose Run, and then select Browse to search for the directory where you stored the Efdcoder.exe file.

Double-click Efdcoder.exe.

This step starts the Efdcoder script.

Choose Open to open the File Open dialog box.

In the Listdemo\Vb subfolder, select Handcode.rtf.

Choose Run, and then choose Compile.

Your modified sources are recompiled by Microsoft Visual Basic and then converted to an .exe file.

Choose Install to install the form on your Microsoft Exchange Client.

Tips for Using the Efdcoder Form

Following are tips to help you use the Efdcoder form effectively:

  • To single-step through your script file, choose the Step button instead of the Run button.
  • To open a shortcut menu, right-click the top portion of the handcode menu.
    • If nothing is currently selected , a list of verbs appears. Select a verb to insert it into your handcode file at the current insertion point.
    • If you have selected text, then a text format menu appears.
  • To insert previously selected text, choose the Insert button between the two windows.
  • To maximize the handcode window so that it fills the entire screen, choose the Zoom button. This feature is useful when you are building a script file because you can paste or copy source code while working in Visual Basic.
