Using the Microsoft Mail Route Change Program

The Microsoft Mail Route Change Program changes the connection of all external postoffices to a local area network (LAN). If certain postoffices need to be defined as using a modem or an X.25 connection, you must run the Microsoft Mail Admin.exe program and change the external postoffice configuration.

Before running the actual program, the Route Change program backs up the .xtn files to a Bak subdirectory in the Xtn subdirectory. The Network.glb file is also backed up to a Bak subdirectory in the Glb subdirectory.

In the event an error is encountered, the backed up files are restored before the program stops.

Command-Line Options

When running the Microsoft Mail Route Change Program, you can use the following command-line options.

CHGROUTE admin [-D<d|path>] [-L<log file>] -N<network> -O<postoffice> -P<password> [-V] [-Y] [-?]

Option

Description

admin

The mailbox assigned to the Microsoft Mail administrator.

-D<d | path>

Drive D specifies the mail data files drive (the default drive is M). Path specifies the path to the mail data files.

-L<log file>

The file to which logging information is written.

-N<network>

Specifies the network name of the new hub postoffice.

-O<postoffice>

Specifies the postoffice name of the new hub postoffice.

-P<password>

The mailbox logon password of a Microsoft Mail administrator.

-R<I | D>

Specifies the new routing of all external Microsoft Mail postoffices.

I - Indirect routing. The –N and –O options are required

D - Direct routing.

-V

Displays/logs detailed logging information.

-Y

Overrides the confirmation message.

-?

Displays this message.
