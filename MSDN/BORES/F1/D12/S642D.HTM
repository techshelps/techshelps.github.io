Troubleshooting Mapindl

The following are common errors that you may encounter when building systems using Mapindl.

  • "Cannot create automation object" ¾ This error is generated on a call to CreateObject(Mapindl.Attach) and Mapindl.dll is not correctly registered on the client computer making the call. Check the HKEY_CLASSES_ROOT hive in the directory for the correct Mapindl.Attach configuration. Try rerunning RegServ32 to reregister the OLE object.
  • "The source application for the object could not be started"¾This error is generated when a user attempts to activate (double-clicks) the resulting Lotus Notes Link from the Microsoft Exchange Client. Either the linked document no longer exists in the database, or the targeted Lotus Notes application is not present on the user's computer. It typically does not mean that Lotus Notes could not be started.

For more information, see the following files on the Microsoft BackOffice Resource Kit, Part Two compact disc:

  • Mapindl.tlb — The type library
  • Mapindl.nsf — The sample Lotus Notes database
  • Mndlapi.doc — The application programming interface (API) documentation for the object
