Troubleshooting the AT&T Mail Migration Wizard

The following sections describe how to troubleshoot problems that occur during migration.

Unable to Log On to the Migration Wizard

If there are problems logging on to the AT&T Mail Migration Wizard, verify that you can log on to the Microsoft Exchange Client by using the same profile. If the logon process is successful, restart Windows.

Invalid Directory Message Is Displayed

The AT&T Mail Migration Wizard expects to find the source mail messages under the default directory of C:\Attmsg. Specify where the Access PLUS or PMX messages are located when running Attmwiz.exe, or move them to C:\Attmsg.

Not All Drives Appear in the Browse Dialog Box

If you choose Browse and the appropriate network destination does not appear in the list, make sure that:

  • The network is started.
  • Physical network connectivity still exists

Mail Messages Are Not Migrated to the Microsoft Exchange Client

If the source mail system was PMX, verify that when the files were transferred from the PMX server, it was done using binary mode. If the files are transferred in ASCII mode, portions of the files lose formatting. This can result in messages not being transferred or being transferred incorrectly.

More Messages Are in the Old Client Than in the Microsoft Exchange Client

If not all mail messages are transferred, it can be caused by the following:

  • Messages are corrupt.
  • Messages are not transferred correctly from the source system.
  • Messages are queued (they are logged but not migrated).

To obtain a list of messages that were not migrated, see Attmwiz.log, the summary log file located in the Windows directory. The Attmwiz.log file can be viewed in a text editor such as Notepad.

Some Folder Names Are Truncated

Folder names can be truncated if the source mail system is PMX. The AT&T Mail Migration Wizard truncates folder names to eight characters during migration to the Microsoft Exchange Client. You can restore the long folder names after migration by choosing Rename from the File menu.

Messages Are Not Processed

If all messages are not processed, review the Attmwiz.log file in the Windows directory using a text editor such as Notepad. Review the list of messages. If an error description of "File could not be opened" exists, run the migration again. This error may be cause by a network interruption during processing.
