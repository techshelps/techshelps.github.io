Revert Tool

Sometimes corruption in a Microsoft Mail message file (.mmf) can make the "read" and "new" counters indicate that a message has not been read, when in fact it has been read and there are no new or unread messages. The Revert tool resets these counters to correct the problem. After the counters are reset, the Microsoft Mail client recalculates and displays the correct values.

The CheckMMF tool included with version 3.0b or later of Microsoft Mail for Windows can also reset these counters, but this tool can actually cause the file to be corrupted if the .mmf file is not otherwise damaged.

When to Use Revert

Use this program when the "new" or "unread" counters displayed by the Microsoft Mail for Windows client in the message file are not correct. You can also use the Listuser tool to determine the hexadecimal ID of the user whose .mmf needs to be reset. For more information on Listuser, see the "Listuser Tool" section.

Side Effects of Using Revert

There are no side effects to using the Revert tool.

To run Revert

  • Type the following at the command prompt:

    revert <file name>

    where <file name> displays the path to the .mmf that is to be reset. The full path must be specified.
