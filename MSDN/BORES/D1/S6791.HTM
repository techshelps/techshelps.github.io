Allocation Units

SQL Server allocates space to databases in units of 256 continuous pages called allocation units. Each allocation unit is 512K, or 0.5 MB in size. Each allocation unit holds 32 (256/8) extents.

The first page in each allocation unit is an allocation page. The allocation page does not contain any user data; it contains system data (header information) and 32 extent data structures. Each extent data structure:

  • Indicates which object owns the extent.
  • Indicates which pages in the extent are in use and which are not in use.
  • Contains pointers to other extents in the extent chain that are owned by that object.
