When a Character in One Code Page is Unavailable on Another

When a character in one code page is unavailable on another and the conversion occurs, the character gets converted to its closest equivalent character in the other code page. For example, ASCII 224 (a) in code page 437 is converted to ASCII 97 (a) in code page 1252 because this is the most similar character in the 1252 code page. When you convert this ANSI character back to code page 437, the result is ASCII 97 (a), because ASCII 97 (a) exists in both code pages. The original 437 character (a) is lost. This means that incorrect data will be saved in your database if the character exists in one code page but not the other, and AutoANSItoOEM is turned on.

When you save your data on a server with a code page different from the code page that will be used by your clients, be sure to test your data for accuracy. If possible, choose characters that will convert easily between ANSI and OEM. For strategies to avoid these inaccuracies, see "Suggestions for Consistent Data" later in this article.
