Hardware Planning Strategy Summary

When building an optimal SQL Server solution, the choice of hardware should be based on:

  • User and application performance requirements.
  • Knowledge of the Windows NT operating system.
  • Knowledge of SQL Server internals and operations.
  • The logical database design.

All too often, SQL Server solutions are made to fit generically configured hardware platforms, which results in systems that perform poorly. Consideration must be given to proper hardware configuration before the physical implementation of the database. This strategy results in less tuning and augmentation of hardware components, better initial performance, and a savings of time and (potentially) money.
