Implementing SQL Load Simulator

SQL Load Simulator allows you to open one or more SQL script documents and assign between 1 and 64 execution threads to each document. You will see a status table of the threads executing the script. Each row displays the status for an individual thread.

Note The status table refreshes every 10 seconds. This delay can cause the status column to be misleading.

SQL Load Simulator fetches column descriptions for each column in the result set to simulate actual result set processing. It processes each column individually, rather than fetching one or more rows into a buffer, to reduce per-thread memory requirements.

For information about the columns of the status table, see "Column Meanings" in the SQL Load Simulator Help file.

To simulate a server load

Double-click the Sqlls icon.

Enter the requested login information in the ODBC Information dialog box.

The ODBC Data Source box lists all configured data sources. To configure additional ODBC data sources, use ODBC in Control Panel.

On the File menu, click Open, and select a SQL script.

In the Script Execution Information dialog box, enter a value between 1 and 64 in the Number of threads field.

If the number of threads exceeds the number of user connections configured on the SQL Server, you will receive errors.

On the Threads menu, click Start to begin processing the SQL script.

You can simultaneously run SQL Performance Monitor to view activity on the SQL Server.

Click the traffic signal to pause or stop processing the SQL script.
