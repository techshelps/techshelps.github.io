ANSI/OEM Conversion

The AutoANSItoOEM entry in the Registry controls the default conversion behavior when the data is inserted or retrieved. If AutoANSItoOEM is set to ON, conversion is turned on in the following cases:

  • ANSI clients to OEM servers
  • OEM clients to ANSI servers

If AutoANSItoOEM is set to OFF, conversion is turned off for all connections.

All of the Windows-based clients, such as ISQL/w and SQL Enterprise Manager, are considered ANSI clients. Console-based applications such as the isql and bcp utilities are considered OEM clients.

A SQL Server with the default code page of 12x, for example 1252 (ISO 8859-1), is considered to be an ANSI server; with any other code page, it is considered to be an OEM server (for example, 850, 437).

Although default code page values exist for both ANSI and OEM, the client's current operating-system code page will determine conversion values when characters are translated.

Note For more information about ANSI to OEM conversion, see SQL Server 6.5 Books Online.
