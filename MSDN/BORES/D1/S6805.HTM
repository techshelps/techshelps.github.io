Microsoft Windows NT Backup

To verify the names of the files to back up

Make sure SQL Server is running.

Go to a query window.

Set your database to master.

Execute:

select * from sysdevices where cntrltype = 0 or cntrltype = 2

Note the values in the phyname column.

To perform a backup using Windows NT

Stop SQL Server.

On the Start menu, point to Programs and then Administrative Tools (Common). Click Backup.

Back up all the data and log device files, including Master.dat. By default, the files you'll need to back up end in the extension .dat.

Copy all the data and log device files to tape or to a temporary directory other than their original locations and other than \Sql, \Sql60 or any of their subdirectories. This prevents accidental deletion of the SQL directory.

Note This step assumes that you are backing up your data and log device files before upgrading to SQL Server version 6.5. If you choose to back up your SQL Server version 6.5 database, do not copy data and log device files to \Mssql.

It is a good idea to have a recent backup of the Registry. Refer to the section "Backing Up Disk Files to Tape" in the Windows NT documentation to learn how to back up the local Registry to tape. After the upgrade successfully completes, you can delete the temporary directory.
