Types of Data Integrity

Enforcing data integrity involves preserving the consistency and correctness of data by validating the contents of individual columns, verifying column values with respect to one another, validating data in one table as compared to another table, and verifying that a database is successfully and accurately updated for each transaction. Data integrity falls into five categories:

  • Entity integrity preserves the uniqueness of entities (rows) in a table. Entity integrity is usually enforced by using the primary key of a table but can also be enforced through unique indexes or constraints.
  • Domain integrity refers to the valid entries for a given column. Domain integrity is enforced by restricting the data type, format, or range of possible values for a column.
  • Referential integrity maintains consistency between multiple tables of a database—for example, "Don't delete customers with open orders." Referential integrity is enforced by checking the validity of data to ensure that it exists within a range of values in another table. In Microsoft SQL Server, referential integrity can be enforced declaratively, by using constraints defined with the table, or programmatically, by using database triggers and stored procedures.
  • User-defined integrity allows organization-wide business rules to also be actively enforced. SQL Server supports user-defined integrity through its programmable server architecture. Critical business rules, such as "Don't ship to a customer with a bad credit history," can be enforced easily across all applications and users.
  • Distributed data integrity maintains the consistency and accuracy of data that is replicated or partitioned across multiple databases or multiple servers. This is a key requirement for reliable, highly available, distributed systems.

Microsoft SQL Server enforces data integrity actively in the server by using methods that fully conform to the ANSI SQL-92 standard. In addition, Microsoft SQL Server provides sophisticated data integrity extensions such as user-defined data types, defaults, rules, stored procedures, and triggers.
