Moving Data to the Warehouse

The following three steps must be performed to move data into the warehouse:

Extract the data from the source database(s).

Load the data into the data warehouse.

Process the data in the data warehouse.

Microsoft SQL Server has three extraction, or transfer and processing, tools you can use to move data to the warehouse:

  • Bulk Copy Program (bcp) Utility

    The bcp utility is a command-prompt utility that can be used to extract data from the OLTP database and load it into the data warehouse database. The bcp utility will extract and load ASCII data quickly, but it will not process data once it reaches the data warehouse, and it is not as flexible as either of the other two options.

  • SQL Enterprise Manager

    SQL Enterprise Manager can be use to extract data from the OLTP database and load it into the data warehouse database (on the Tools menu, click Database/Object Transfer). Because it is a graphical tool, it is somewhat easier to use than the other two options, but it is an interactive tool and needs an operator present for certain steps. Like bcp, it does not process data once it reaches the data warehouse.

  • SQL Distributed Management Objects

    SQL Distributed Management Objects (SQL-DMO) can be used to write a script that will extract data from the OLTP database, load it into the data warehouse, and process it in the data warehouse. Provided you can program using SQL-DMO, this is the most flexible and complete way to move data into the warehouse.

There are also third-party software products developed to extract, transform, and transfer data from source data systems that generate input data to populate a Microsoft SQL Server data warehouse.
