Installing DB Checker Examples

The files Dbchkcrs.sql and Dbchktbl.sql can be used to generate a secondary isql script that can then be used in isql to generate a check of all databases on the system. If you ran the BackOffice Resource Kit CD-ROM Setup, you can access DB Checker from the Bork\Sql\Dbcheck directory on your computer. Otherwise, install it from the BackOffice Resource Kit CD-ROM.

To install Dbchkcrs.sql or Dbchktbl.sql

Copy either the file Dbchkcrs.sql or Dbchktbl.sql from the Sql\Dbcheck directory on the BackOffice Resource Kit CD-ROM to a directory on your computer.

Sites with SQL Server version 4.2 must use Dbchktbl.sql; all other sites should use Dbchkcrs.sql.

At the command prompt, use the isql utility to create the secondary SQL script that will actually check the integrity of the databases. For example:

C:> ISQL /Usa /Ppassword /Sserver_name /ilocation\DBChkXXX.sql /ooutfile.sql

where:

/Ppassword

Is the password for the SA login.

/Sserver_name

Is the name of the server on which SQL Server resides.

/ilocation

Is the full path to the Dbchkxxx.sql script.

DBChkXXX.sq

Is either Dbchkcrs.sql or Dbchktbl.sql.

/ooutfile.sql

Is the name of the file isql will produce that will contain the script that checks the integrity of all the databases on the system.
