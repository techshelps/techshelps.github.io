Removing SQL Workbook Publisher

There are three ways to remove SQL Workbook Publisher from your computer.

To Remove SQL Workbook Publisher from a SQL Server

This task is for database administrators.

Before removing SQL Workbook Publisher from your SQL Server, you must copy Remove.sql from the BackOffice Resource Kit CD-ROM to the computer running Windows NT Server.

Start Microsoft ISQL/w.

In the Server box, select the server from which you want to remove SQL Workbook Publisher.

In the Login ID box, type a valid login identification for the SQL Server. Be sure the login identification allows the database owner access to the database.

In the Password box, type the password for the login ID, and click Connect.

In the DB box of the Query window, select the database from which you want to remove SQL Workbook Publisher.

Click Load SQL Script, locate Remove.sql, and select it.

After the Remove.sql script appears in the query window, click the Execute Query button.

Repeat steps 5 through 7 for each database on the server from which you want to remove SQL Workbook Publisher.

You can quit Microsoft ISQL/w after all SQL Workbook Publisher objects have been removed.

To Remove SQL Workbook Publisher from an Excel Client

This task is for Excel users.

On the Windows Start menu, point to Settings, and click Control Panel.

Click Add/Remove Programs.

Double-click SQL Workbook Publisher or SQL Workbook Viewer.

To Remove SQL Workbook Publisher from the Intranet

This task is for Webmasters.

Find the IIS directory. The directory default name is Inetsrv on Windows NT Server 3.51 and InetPub on Windows NT Server 4.0.

Delete all SQL Workbook Publisher files from the Scripts\Sqlwpub directory, then delete the Sqlwpub directory.

Delete all SQL Workbook Publisher files from the Wwwroot\Sqlwpub directory, then delete the Sqlwpub directory.

Note If you want to remove references to only certain databases from the intranet, be careful to delete only the files referencing those databases from Scripts\Sqlwpub and Wwwroot\Sqlwpub.
