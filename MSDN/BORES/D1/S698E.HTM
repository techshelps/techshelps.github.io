Implementing the Arcadia Bay Demonstration

The Arcadia Bay demonstration is completely implemented using Microsoft technology. Microsoft SQL Server serves as the data repository. The data is stored in a star schema and uses Active Server Pages (ASP) in combination with Microsoft Internet Explorer to drill down to the attribute sales data stored in the database. All of the operational data is stored in one database (order entry system) and the historical data (data warehouse) is stored in another.

Arcadia Bay Transaction Simulator

This application loads sales transactions into the order entry database. At the same time, it creates scenarios that demonstrate data warehousing functionality. The following scenarios are written into the application:

  • Inventory is low on the best-selling item.
  • Promotional items sell better.
  • Sales increase when the Web server goes live.
  • Differences in sales figures are attributed to employees and customers.
  • Some store locations have better sales figures than others.

To load the order entry database with sales transactions

On the Start menu, click Programs and then click Arcadia Bay Transaction Simulator.

Click Start.

By default, the application is set to automatic mode and will load two days' worth of transactions into the order entry database.

To turn off automatic mode

  • On the File menu, click Automatic.

    In manual mode, you will be queried for the baseline number of transactions to execute per day, the baseline probability for each transaction, and the begin and end dates for the run each time you execute the application.

To set program options

  • On the View menu, click Options.

You can change the program options as you see fit. The following table contains descriptions of the options you can change.

Option

Description

Reset Log Every Program Execution

If this box is checked, the log file will be reset every time the program is run.

Create Error Log

If this box is checked, a log file will be created every time the program is run. If a log file is created, you can choose between capturing database errors only or capturing all errors.

Default Time Period

This value indicates the default number of days for which the application will create transactions. In automatic mode, the application will use this value when it creates the Options window. You can override this value at that time.


To reset the order entry system

On the File menu, click Reset System.

Log on as sa if you are prompted.

Enter values for store and warehouse inventory.

If you increased the baseline number of transactions per day, increase both the store and warehouse values accordingly.

Click OK to reset all of the tables and to create an empty order entry database.

Arcadia Bay Data Transformation Utility

This application moves data from the order entry database into the data warehouse and transforms it. It is very similar in look and feel to the Arcadia Bay Transaction Simulator. In a typical scenario, this operation would occur regularly (nightly, weekly, or monthly) based on the requirements of the application.

To move and transform data

  • Start up the Arcadia Bay Data Transformation Utility. Click Start.

By default, the application is set to automatic mode and will determine the start and end dates. The start date is calculated from the last time the utility was run. The end date is calculated from the data in the order entry database. This process will last several minutes the first time you run this application as the dimension tables are constructed. Subsequent executions will be faster.

To turn off automatic mode

  • On the File menu, click Automatic.

In manual mode, you will be queried for the begin and end dates every time you execute the application. You will also be offered the option of just loading the dimension tables or just loading the fact table. It is not recommended you do this.

To set manual run options

  • On the View menu, click Options.

You can change the manual options as you see fit. The following table contains descriptions of the options you can change.

Options

Description

Log Errors to File

If this box is checked, a log file will be created every time the program is run. If a log file is created, you can choose between capturing database errors only or capturing all errors.

Reset Log File

If this box is checked, the log file will be reset every time the program is run.


To reset the data warehouse

On the File menu, click Reset Data Warehouse.

Log on as sa if you are prompted.

Click Start.

The application indicates the active connection and shows the status of that connection. It indicates the start and end dates, and the status bar illustrates the progress of data as it moves from the order entry database into the data warehouse.
