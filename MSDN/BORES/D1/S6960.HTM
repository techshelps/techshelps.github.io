Suggestions for Consistent Data

The following suggestions will help ensure that the data stored on the SQL Server is stored in accordance with the SQL Server code page instead of the client's code page. Storing data in terms of the client's code page is most often the cause of bad data.

  • For OEM and ANSI clients, if the SQL Server is not using the ANSI code page, change the client's OEM code page to match the SQL Server code page and enable the AutoANSItoOEM conversion.
  • For OEM and ANSI clients, if the SQL Server is using the ANSI code page, enable the AutoANSItoOEM conversion.

Exceptions to the above suggestions are:

  • For the Win32 Bcp.exe client importing from an ANSI SQL Server (BCP out), change the client's code page to match the code page that the data was created with.
  • For the Win32 Bcp.exe client exporting to an ANSI SQL Server (BCP in), change the client's code page to the code page of the SQL Server that will receive the export results. For example, to export to a SQL Server with code page 1252, change the client's code page to 1252.
