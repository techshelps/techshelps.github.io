SQL-DMO Error Message 1722

If, at any time, you have changed the Windows NT computer name but not the SQL Server name and the @@SERVERNAME global variable is not NULL and does not match the Windows NT computer name, then you must change the SQL Server name using sp_dropserver and sp_addserver. For more information, see SQL Server Books Online.

To get the registered SQL Server name, execute the statement: 

SELECT @@SERVERNAME

Or you can view the server name in the SQL Server error log located in the \Mssql\Log directory. The SQL Server name must also conform to the rules for identifiers.
