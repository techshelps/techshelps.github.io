Security Issues

In a client/server relationship, multiple clients and servers communicate across one or more linked and shared networks. SQL Server's security system prevents unauthorized clients from gaining access over shared networks to data stored on SQL Servers.

SQL Server allows the system or database administrator to specify the clients that can log in to the server. The three SQL Server security modes are:

  • Standard security

    SQL Server prompts the client for a username and password each time a connection is requested. Trusted connections are also supported by this mode.

  • Integrated security

    SQL Server obtains the information from the Windows NT security system, removing the need for additional logins. Clients that connect in this mode are said to have "trusted connections."

  • Mixed security

    SQL Server also allows a combination of security methods.

The SQL Server default is standard security mode. Excel clients can connect to SQL Server with a trusted or standard connection. If you decide to implement SQL Workbook Publisher's intranet component, the SQL Server security mode must be set to mixed or standard. If you do not implement SQL Workbook Publisher's intranet component, you can use whichever security mode you think is appropriate for your needs.

Note Your users will receive the following message if your security parameters have been improperly configured or you haven't given them access permissions: Unable to connect to specified server.
