Architecture of a Data Warehouse

A data warehouse is composed of operational and external data sources, one or more databases, and one or more desktop data analysis tools. In the following illustration, data is extracted, scrubbed, transformed, loaded, aggregated, and replicated between the operational and external data sources (1) and the data warehouse (2). A subset of the data is loaded, and data is aggregated and replicated at the data mart level (3). Using desktop analysis tools (4), spreadsheets are built, reports are written, and data is analyzed.

The following is a list of the most popular Microsoft products that can be used to build and maintain data warehouses, as presented in the preceding architecture illustration.

  • SQL Server, TransAccess, and SNA Server can be used to load the data warehouses with operational data.
  • SQL Server relational database management system can be used as a data warehouse.
  • SQL Server, alone or combined with Internet Information Server, Microsoft Exchange, or SNA Server is an effective data mart.
  • Microsoft Access, Excel, Visual Basic, Visual C++, and Internet Explorer are used as desktop data analysis tools.
