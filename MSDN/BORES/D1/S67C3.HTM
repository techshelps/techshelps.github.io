Publishing Snapshots of Database Information

Sometimes you may need to make available to many individuals in your organization information stored in your SQL Server databases. These individuals may not have the appropriate client-side applications installed. A logical solution to this dilemma is to extract information from the database and publish it on your company intranet.

In this type of situation, the data owner or database administrator determines what information to publish, and when to publish it. Users need to view database information, but don't need to alter database records. The results of one query or report are published to many people to save the time of running the query again and again, to reduce the risk of running the wrong query, or to reduce the performance impact that multiple queries would have on the database server.

Microsoft makes three products that address this issue:

  • SQL Workbook Publisher
  • The Web Wizard component of Microsoft Access 97
  • The SQL Server Web Assistant component of Microsoft SQL Server 6.5

Each extends Microsoft's database software by effectively converting query results into HTML pages. Each creates Web pages that any Web server can publish and any Web client can browse.

Note If you have not yet upgraded to Microsoft Access 97, you can download Internet Assistant for Microsoft Access 95 from the Microsoft Web site at http://www.microsoft.com.

Answer these questions to choose the right tool for your needs:

Do you need the Web page to update automatically?

  • If yes, use Web Assistant.
  • Microsoft Access solutions require that you run Microsoft Access interactively. SQL Workbook Publisher requires that you run Microsoft Excel interactively.

Is the ODBC data source Microsoft SQL Server?

  • If yes, use Web Assistant or SQL Workbook Publisher.
  • If no, use a Microsoft Access solution.

Do you want to leverage investments in Microsoft SQL Server, Microsoft Access, or Microsoft Excel 8.0?

  • Web Assistant is supplied with and requires Microsoft SQL Server 6.5.
  • The Web Wizard is supplied with and requires Microsoft Access solutions. Internet Assistant is a no-charge add-in product that runs only with Microsoft Access 95.
  • SQL Workbook Publisher is a no-charge add-in product that runs with Microsoft SQL Server 6.5 and Microsoft Excel 8.0.

Is the Web server an Internet Information Server (IIS)?

  • If yes, the IIS add-in for Microsoft Access can be used in place of the Microsoft Access solutions.
