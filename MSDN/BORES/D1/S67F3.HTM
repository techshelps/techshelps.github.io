Deferred Update

When the deferred method of update is used, the changes are applied to all rows of the table by making log records in the transaction log to reflect the old and new value of the column(s) being modified (in the case of UPDATE operations) or to reflect the values that will be inserted or deleted (in the case of INSERT and DELETE, respectively).

When multiple rows can qualify for the update, if the row can increase in size or if indexed columns are updated, the rows could move out ahead of the current position and be processed again. For example:

UPDATE T1
SET col1 = col1 + 1

In this case, deferred mode must be used. If only a single row can qualify, deferred update is not required.
