tempdb in RAM

If the queries being executed against the SQL Server are using temporary workspace for sort operations, GROUP BY, temporary tables, multitable join operations, and so on, it will be beneficial to move tempdb to RAM. However, to make this move, you must have available memory beyond that required by and allocated to Windows NT Server and SQL Server. Consequently, if the tempdb is currently 25 MB in size, then the total memory required for the system is: 16 MB for Windows NT Server plus total SQL Server memory plus 25 MB for tempdb.
