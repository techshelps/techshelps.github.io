Installing Sqlhdtst

Sqlhdtst is a single executable that can simply be copied to the Winnt\System32 directory on the Windows NT server to be tested. If you ran the BackOffice Resource Kit CD-ROM Setup, you can access the Intel x86 version of Sqlhdtst from the Start menu. Otherwise, install it from the BackOffice Resource Kit CD-ROM.

Note A full Sqlhdtst run should consume most of the processing resources on the server, reducing its ability to respond to other applications. For this reason, do not run Sqlhdtst on a server actively being used as a production server. The server's files should also be backed up prior to running Sqlhdtst.

To install Sqlhdtst

  • The Sql\Sqlhdtst directory on the BackOffice Resource Kit CD-ROM contains the Sqlhdtst version for DEC Alpha servers and the Sqlhdtst version for PowerPC servers, as well as the default Sqlhdtst for Intel servers. Copy the appropriate executable to the server's Winnt\System32 directory.
