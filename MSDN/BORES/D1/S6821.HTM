Boost SQL Server Priority

Boosting SQL Server's priority can improve performance and throughput on single- and multiple-processor hardware platforms because the Windows NT thread scheduler will favor SQL Server threads over threads of other processes. By default, this option is turned off and SQL Server runs at a priority of 7. When selected on a single processor platform, SQL Server runs at priority 15. When selected on an SMP platform, SQL Server runs at priority 24.

When this option is turned on, it can degrade the performance of other processes. Therefore, this option should only be turned on for dedicated SQL Server computers or if slower performance of other processes is acceptable.

Note Boost should generally not be used except in special circumstances.
