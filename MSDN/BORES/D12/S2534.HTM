Microsoft Exchange Server Setup

If Setup indicates that it failed while installing Microsoft Exchange Server, in most cases you should reinstall Microsoft Exchange Server. If Setup fails but Microsoft Exchange Server seems to be working, there may be a problem with the network. After you've verified that the network connection is working correctly, complete the following steps to ensure that Microsoft Exchange Server is correctly installed.

    1. Turn off all Microsoft Exchange Server services.

    2. Restart the server.

    3. Run the Setup program and reinstall any components that Setup did not successfully install during the initial installation.

If Setup fails while you're adding a new server to a site, information about the new server may have been replicated to other servers in the site. To prevent the site's directory from becoming corrupted if directory replication has already been initiated, you may need to remove the new server from the site's directory before reinstalling the server. Complete the following steps to ensure that Microsoft Exchange Server is installed correctly in the site.

    1. In the Administrator program on the remote server you specified during Setup, delete the Server object for the new server.

    2. Run Setup again on the new server, and choose Remove All.

    3. Run Setup again on the new server, and reinstall Microsoft Exchange Server.
