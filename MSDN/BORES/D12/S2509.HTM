Generating Import Files

Load Simulator stresses the server by using simulated users. It obtains the users and, if selected, distribution lists from directory import files. Before you can create simulated users, you must instruct Load Simulator to create the following directory import files.

  • A directory import file (.csv file) for each site defined in your load simulation topology. The names of these files are derived from the site name. For example, Site0.csv, Site1.csv.
  • A simple mail transport protocol (SMTP) remote user import file (-s.csv file) for each organization defined in your load simulation topology. The name of this file is derived from the organization name. For example, Org-s.csv.
  • An X.400 remote user import file (-x.csv file) for each organization defined in your load simulation topology. The name of this file is derived from the organization name. For example, Org-x.csv.

Note

If you are running tests in a single organization, you don't need to use the SMTP and X.400 import files.

To generate import files

  • From the Configuration menu, choose Generate Directory Import Files.

The directory import files are created in the default Load Simulator directory.
