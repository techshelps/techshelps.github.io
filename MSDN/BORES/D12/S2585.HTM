Setting Up a Mailbox to Send As Another Mailbox

If a user has Send As permissions on a mailbox, that user can send a message with that mailbox's return address so that it appears that the message was sent by the mailbox's owner. Send As permissions can be granted to user accounts only with the Administrator program.

Note

Send As permissions override Send on Behalf of permissions. For example, if a user has both Send As and Send On Behalf of permissions for another user, any message sent in the other user's name will have that user's name as the return address.

To grant Send As permissions

    1. In the Administrator window, choose Recipients, and then select the name of the recipient you want to configure.

    2. From the File menu, choose Properties, and then select the Permissions tab.

If the Permissions tab is not available, choose Options from the Tools menu, select the Permissions tab, and then select the Show Permissions page for all objects and Display rights for roles on Permissions page check boxes.

    2. Select the user account that you're granting permissions to, or choose Add to add an additional account.

    3. Under Roles, select Send As, and then choose OK.
