Setting the Type of Test

You can use Load Simulator to specify and run the following tests.

User Initialization

Creates folders and messages in the user mailboxes you specify, one or more at a time.

Public Folder Initialization (optional)

Sets up a public folder hierarchy in your site. This is required only if you have specified public folders in your topology.

Load Simulation

Simulates multiple users performing a wide variety of client actions. It is useful if you're using Load Simulator for capacity planning. The User Initialization and Public Folder Initialization tests are prerequisites for the Load Simulation test.

Configuring the User Initialization Test

When you set user initialization parameters, Load Simulator logs on to user mailboxes and adds items to the mailboxes to simulate real mailboxes. You can specify mailbox properties, including the type and number of messages in the mailbox and the number of messages in the Inbox. Your server's information store should be empty before you start the Initialize Users test.

You can perform the following user initialization tasks:

  • Specify the contents of mailboxes.
  • Select a subset of mailboxes to initialize.
  • Select a range of recipients to address messages to.
  • Select recipient options.

In most cases, the default user initialization values are sufficient for a standard Load Simulation test. You are required to set only the usage level in the Parameters property page.

Before you run the Load Simulation test, you should populate mailboxes with messages and folders. This helps to simulate a realistic usage scenario because mailboxes are not typically empty when users send and receive messages.

To specify the contents of mailboxes

    1. From the Test menu, select User Initialization Parameters.

    2. In the Parameters property page, specify the default settings for usage.

Option

Specifies the number of new folders and messages you wish to create for each user mailbox. This usage definition corresponds with a low volume e-mail user.

You can select recipient options that will be used during your Load Simulation test such as the number of recipients that will receive a message. The options you select can help simulate a realistic usage scenario because how messages are sent and addressed can impact performance.

To select recipient options

    1. From the Test menu, select User Initialization Parameters.

    2. Select the Recipient Options tab, and specify the recipient options that will be used to address messages.

Option

Description

All available recipients

Each message will be addressed to the entire set of recipients specified in the Recipients property page. If this option is selected, local delivery options are ignored.

Use random recipients

Each message will be addressed to the specified number of recipients, which are selected randomly from those specified in the Recipients property page.

Between and average recipients

Specifies the minimum and maximum number of recipients for each message weighted to the average you select.

To local server

The percentage of messages sent to recipients on the same server as the sender. This option assumes you have specified multiple servers in the Recipients property page. For example, if you set the local server percentage to 90, only 10 percent of the mail will be sent to a recipient on another server.

To local site

The percentage of messages sent to recipients in the same site as the sender. This option assumes you have specified multiple sites in the Recipients property page. For example, if you set the local site percentage to 90, only 10 percent of the mail will be sent to a recipient in another site.

To local organization

The percentage of messages sent to recipients in the same organization as the sender. This option assumes you have specified multiple organizations in the Recipients property page. For example, if you set the local organization percentage to 90, only 10 percent of the mail will be sent to a recipient in another site.

Add distribution list to % of messages

The percentage of messages that are addressed to a distribution list if one is defined. This option should only be selected if you've imported distribution lists into the directory.


Note

The server, site, and organization traffic depends on a combination of the local delivery options selected and the composition of the distribution lists used, if any. If none of the local delivery options are selected, recipients are selected randomly from the entire set specified in the Recipients property page.

To simulate realistic usage scenarios, you can select the types and sizes of messages that are used to populate mailboxes. For example, you can select large or small messages with or without attachments. The messages you select should correspond to the type of usage level you're simulating.

To select messages used to populate mailboxes

    1. From the Test menu, select User Initialization Parameters.

    2. Select the Messages tab, and specify the recipient options that will be used to address messages.

Option

Description

List of .MSG files

The list of message files that Load Simulator will use in a test. By default, the list of messages corresponds to the usage definition you selected in the Parameters property page.

Weighting for selected file

Specifies the weight value for a message. The weight value is a relative number that indicates whether the selected message will be used more or less than other messages. For example, if message A has weight 1 and message B has weight 50, message B will be used 50 times more often than message A.

Add File(s)

Adds one or more .msg files.

Remove

Removes an .msg file.


Configuring the Public Folder Initialization Test

When you set public folder initialization parameters, Load Simulator initializes the public folders defined in your topology. To run this test, public folders and user initialization parameters must already be defined.

You can perform the following public folder initialization tasks:

  • Specify public folder initialization parameters.
  • Select the public folders that will be used.
  • Select messages that will be used to populate public folders.

If you want to test public folder usage in your test, you can populate public folders with messages before running the Load Simulation test. This helps to simulate a realistic usage scenario because public folders are not typically empty when they are used.

To set public folder initialization parameters

    1. From the Test menu, select Public Folder Initialization Parameters.

    2. In the Parameters property page, specify the distribution of messages per public folder.

Option

Description

Messages per folder

The minimum, maximum, and average number of messages in a public folder. For example, if you enter 1, 10, and 7 for the minimum, maximum, and average values, each public folder will have up to 10 messages with an average of 7 messages.

Show folder in address book

Each initialized folder is added to the global address list.

Use age limit (Expiry)

Messages will expire in the specified number of days.


If you're using public folders in your test, you should specify which public folders will be used during the test. This will help simulate a realistic usage scenario because some public folders are used more than others.

Configuring the Load Simulation Test

The Load Simulation test is key to testing a Microsoft Exchange Server computer. It is used to calculate server performance for a certain type of user. The user definitions are based on low, medium, and high usage of the Microsoft Exchange Client. You configure the Load Simulation test after you've populated user mailboxes with messages and any public folders.

In most cases, the default values are sufficient for a standard test. You are required to set only the usage level in the General property page.

Note

If you're using public folders, public folder initialization parameters must be defined before you configure public folders in the Load Simulation test.

To configure the Load Simulation test

    1. From the Test menu, select Load Simulation Parameters.

    2. If you are configuring a new Load Simulation test, select Yes to create a new test.

    3. Type the name of the test.

    4. In the General property page, specify the distribution of messages per public folder.

    5. Specify any additional information and review your test's configuration using the Test Report property page.

This property page provides details about the load that will be generated on the server based on settings you've specified.

Option

Description

Test description

The name of the current test. The name you specified will be added to the .sim file.

New

Specify a new test.

Test duration

The amount of time the test will run. To manually stop the test, select Forever. To run multiple tests consecutively without pause, select Iterations. If Iterations is selected, length of day settings are ignored.

Low usage

Specifies default settings that correspond with a low volume e-mail user.

Medium usage

Specifies default settings that correspond with a moderate volume e-mail user. This setting is used by default.

High usage

Specifies default settings that correspond with a high volume e-mail user.

Read Mail

The simulated users will read mail during the test.

Send Mail

The simulated users will send mail during the test.

Schedule+ Actions

The simulated users will perform Microsoft Schedule+ tasks during the test.

Public Folders

The simulated users will use public folders during the test. This option should only be selected if public folders have been defined in your configuration.


Note

There is currently no mechanism for deleting or renaming a test.

When you run a test, it is important to consider how many hours in a day a user will use e-mail so you can simulate realistic usage patterns. The amount of e-mail traffic varies according to how frequently e-mail is used, which can affect your system's performance.

To set the length of a user's day

    1. From the Test menu, select Load Simulation Parameters.

    2. Select the Logoff tab and define the length of a user's day.

Option

Description

Daily schedule

The amount of time during the day or night that a user will use e-mail. This option can be used to increase the stress on the server. For example, if you want to double the amount of load on the server, shorten the user's day from 8 hours to 4 hours. This option does not affect the duration of the test.

Nighttime is specifies the length of the user's night. When a day is over, users log off (optional) and go to sleep for the specified period. Setting Nighttime is to zero disables the feature.

Log on immediately at beginning of the test

All users log on immediately at the beginning of the first day and never log off. No actions are begun until all the users are logged on.

Log off at the end of each day

All users log off at the end of each day.

Empty Deleted Items folder before logging off

The Deleted Items folder is emptied before users log off.

Log off after consecutive errors

After a certain number of consecutive errors, the user assumes that the server is unavailable and logs off. When the server is reactivated, the user automatically logs back on to perform the next scheduled action.

Log off when the next scheduled task is at least away

Users log off when the time between two tasks reaches the specified threshold. Don't select this option unless you're using Load Simulator to generate client performance data.


Because the frequency that users read and respond to mail can affect performance, your test should consider how users read their messages.

To set the properties for reading mail

    1. From the Test menu, select Load Simulation Parameters.

    2. Select the Read Mail tab, and define the properties used to simulate a user checking the Inbox and reading and responding to any new mail.

Option

Description

Read new mail times per day

The number of times per day users read new mail.

Read old mail times per day

The number of times per day users read old mail.

Read mail processing

The percentage of time that users reply, reply all, forward, move, copy, or delete messages. None is the percentage of messages that are read and then remain in the Inbox.

Load Simulator keeps track of how many times a particular message has been replied to (including reply all) or forwarded and reduces the percentage of reply, reply all, and forward actions accordingly. A message that has already been replied to or forwarded once is half as likely to be replied to or forwarded again.

Load % of attachments (on messages with attachments)

The percentage of message attachments that will be opened.

Limit Inbox to messages at any time

The number of messages allowed in a user's Inbox at the end of new mail processing. Any extra messages are moved to the Deleted Items folder.

Read note delay

The length of time a user pauses while reading a message. Load Simulator will use the specified minimum, average, and maximum values to randomly select a delay time when a user reads a message.


Important

When choosing percentage weighting values, you should use realistic values that approximate messaging in the real world. Setting certain percentage weighting values too high can create a message feedback loop that will overload your server and can invalidate your data.
