Single Copy of a Public Folder

You can store the contents of a public folder on a single public folder server. This is the default configuration of a public folder when replication has not been configured.

The advantages of a single copy of a public folder are as follows:

  • Automatic load balancing for user requests
  • Availability in a wide area network (WAN). Users at sites with no direct connectivity can still share data.
  • Fault tolerance

The disadvantages of a single copy of a public folder are as follows:

  • Possible public folder conflicts
  • Latency issues
  • Additional message traffic associated with replication
