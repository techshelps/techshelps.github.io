Dedicated Public and Private Servers vs. Multipurpose Servers

As mentioned previously, you can centralize responsibilities among Microsoft Exchange Server computers by configuring all the Microsoft Exchange Server computers to be either public or private servers. A private server has a private information store (dedicated to private mail storage and user mail), but no public information store.

Configuring separate public and private servers allows you to plan the hardware needs for a particular computer according to its role. In other words, if you know that a public server will have heavy use, you can use a higher speed processor, more RAM, and so on.

Separating public and private servers also concentrates the flow of public folder traffic. In this way, replication of the public folder hierarchy and contents takes place only between the public servers and does not impact the private servers.

Specify Separate Responsibilities for Servers

You can assign different tasks to different servers.

To separate the responsibilities between public servers and private servers

    1. Move all the public folders to the public servers by using the Replicas property page on the public folder or the Instances property page on the target public information store.

    2. For each private server, configure the private information store so that the public folder server is the appropriate public server. To do this, use the General property page on the private information store server object.

This step ensures that all top-level public folders created in the future are located on the new public servers.

    3. Use the Microsoft Exchange Server Administrator program to remove the private information store from the public servers and the public information store from the private servers. To do this, delete the private information store object from the server container in the Administrator program. Also, move mailboxes off of the public-only servers.
