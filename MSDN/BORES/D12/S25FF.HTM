Using Mailbox Statistics

When you use Mailbox Statistics, you set up an administrator's mailbox for collecting results and send the tool to your users to run it against their sets of personal folders. When user's submit a number of results, you can run the tool in administrator mode and export results to a variety of output formats. Before you send the tool to your users, you can test it with a smaller subset of about 10 people to ensure the tool works as expected and to become familiar with the different kinds of results.

Note that mailbox Statistics can detect only Microsoft Exchange Server distribution lists (DLs).

If the Save Items to Sent Mail option is not selected in a user's Microsoft Exchange setup, then Mailbox Statistics cannot determine the number of messages sent per day. You can ask the test group to select this option for at least seven days before the study to collect enough information to use.

To use Mailbox Statistics:

  • Set up an administrator's mailbox that will be used to receive the data
  • Send a message with Storstat.exe attached to the users to be surveyed. Those users double-click an icon in the message to run it.

Note

Storstat.exe is supported only for users on Windows 95 or Windows NT Workstation.

To set up the Administrator's mailbox

    1. Using the Microsoft Exchange Server Administrator program, create a mailbox with the display name StorStat.

    2. Set permissions on this mailbox according to your organization's procedures so that the Microsoft Exchange Server administrator's account has user permission on this mailbox.

Results appear as MAPI messages with the subject "UserName UserResults." You can use the Inbox Assistant to forward or copy results to another folder.

To send mailbox statistics to users

    1. From the Microsoft Exchange Client Compose menu, choose New Message.

    2. In the message body, type instructions for your users.

    3. From the Insert menu, choose File, and then attach StorStat.exe to the message.

It appears as an icon in the message.

Users can be concerned about using the tool if they believe it will alter, damage, copy, or intrude in any way on their sets of personal folders. For this reason, you can include information to reassure them that no information is read or copied, no damage is inflicted on their data, and no changes are made to their data when this tool is running.
