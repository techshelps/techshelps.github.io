Using IMC Configuration Extractor

The IMC Configuration Extractor performs the following operations for collecting data and extracting Internet Mail Connector messages:

  • Looks in the registry to find the location of Imcdata, and then creates a directory called Backup under Imcdata. If the directory already exists, the process quits.
  • Copies all Internet Mail Connector service registry values to a file called Imcsvc.reg.
  • Copies all Internet Mail Connector profile registry values to a file called Imcpro.reg.
  • Uses directory application programming interface (DAPI) to export the gateway object to a file called Imcobj.csv.
  • Logs on to the service in the same manner as Internet Mail Connector.
  • Sets a flag immediately in the information store, which prevents more mail from being routed through Internet Mail Connector.
  • Creates a directory called Imcdata/Backup/Mtsout, and then copies all messages from MTS-OUT into that directory by using the transport-neutral encapsulation format (TNEF).
  • Creates a directory called Imcdata/Backup/Mtsin, and then copies all messages from MTS-IN into that directory.
  • Generates a packing list of all generated files called Imcfls, using an .ini format and code to encode the information.
