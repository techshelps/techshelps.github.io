Customizing Load Simulator

You can customize Load Simulator environment options at any time, including the font used to display output in the Load Simulator window and the path to .msg files. You can also use Microsoft Exchange Client to create your own message files for tests.

To customize Load Simulator

    1. From the Preferences menu, choose Options.

    2. Specify the options.

Option

Description

Show time

Prepends the date and time to each line in the output window.

Show process ID

Prepends the identity of the client process to each line in the output window.

Show thread ID

Prepends the identity of the client thread to each line in the output window.

Output to file

All text in the output window is also written to the specified file.

Browse

Enables you to select an output file.

Overwrite previous file

Overwrites an existing output file if one exists.

Choose Font

Enables you to change the font displayed in the output window.

Message file path

The path to the message files.

Maximum simultaneous client logons

Allows you to control the number of Load Simulator processes that can log on simultaneously.


You can create your own message files to have more control over the size and types of messages you use in your test. For example, if your users typically send larger messages than the default messages provided with Load Simulator, you can create message files that more closely correspond with your scenario.

To create message files using Microsoft Exchange Client

    1. In the Microsoft Exchange Client, create a new message or open an existing message.

    2. Format the message as desired. It is not necessary to address the message because Load Simulator provides its own addresses.

    3. From the message's File menu, choose Save As.

    4. Under Save file as type, select Message Format, name your file, and choose OK.
