Using MS Mail SMTP Gateway Proxy Migration Tool

Run Migsmtp.exe at the command prompt on a Microsoft Exchange Server using the following form.


MIGSMTP [-?] [-R] [-V] [-Ccontainer] [-Taddresstype] -sserver -pPOpath

-? Displays Help.

-R Replace the primary SMTP proxy name. If not specified, the MS Mail SMTP proxy name is appended as a secondary SMTP proxy name.

-V Verbose progress reporting.

-C Processes only the recipients in container. If -C is not specified, only the Recipients container will be processed.

-T Uses addresstype as the type of the generated address instead of SMTP.

-s server is the name of the Microsoft Exchange Server to run the process on.

-p POpath is the path to the MSMail postoffice. The Addr_map.cfg and Local.cfg files are required by the proxy generation process. These files must be located in the path specified either in a live database in the directory, or as separate files in the directory.

Migsmtp.exe adds an SMTP secondary proxy for each user that is located by converting their MS Mail address (type MS in Microsoft Exchange Server). The MS Mail address is searched for in the following order:

    1. The target address in a custom recipient

    2. The primary proxies

    3. The secondary proxies

This transformation is not performed on custom recipients with a target address of type SMTP.

SMTP address generation also takes place for distribution lists and recipients hidden from the Address Book.
