Connections Used for NetView and RTM

SNA Server can make use of NetView, a reporting system that runs on an IBM host (mainframe). NetView sends alerts between the host and the PCs that connect to it. SNA Server can also receive information from RTM, a 3270 and NetView facility that monitors the amount of time it takes for a host to respond during 3270 display sessions.

To send link alerts, link statistics, and application-generated alerts to the NetView program at the host, you must specify the connection on which the alerts will be sent. See SNA Server Manager Help for information about specifying the connection.

RTM data and NetView alerts sent to the host by 3270 users are not necessarily sent via the NetView connection. RTM data is sent on the connection for the session to which it refers, while 3270 user alerts are sent on the connection for the currently selected 3270 session.

For information about configuring SNA Server Manager to work with NetView and/or RTM, and information about configuring the NVAlert and NVRunCmd services, see the Microsoft SNA Server Administration Guide.
