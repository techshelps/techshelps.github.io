Open or Create Web Project Dialog Box

      

Use this dialog box to specify which Web server you want to place the master application on and the name you want to use for your application. Unlike the Project wizard, using this dialog box does not add the script library, search.htm, or global.asa file to your new project.

Options

Project file name

Displays the identifier for the project you are creating.

Web server URL

Type a location for your master Web server. This might be your staging server or your production server, depending on your system setup.

Typically, this name is the first element in the Uniform Resource Locator (URL) that you select to specify your Web application. For example, the Web server identifier in the URL http://MyWebServer/MyWebApplication is "MyWebServer."

Note   If you use a proxy server, entering the number for the IP Address instead of the server name may generate an authentication error. For more information, see Troubleshooting.

Web name

Type an identifier for the application. This identifier determines the name for the virtual root on the master Web server specified above and the physical directory within your local project directory.

Typically, this name is the second element in the URL that specifies your Web application. For example, the application identifier in the URL http://MyWebServer/MyWebApplication is "MyWebApplication."

Options

Open an existing Web application

Select this option to search the master Web server for the Web name specified and set the new project to reference that application. If the Web server or application cannot be located, you have the option to work offline.

Create a new Web application

Select this option to connect to the Web server, create a new virtual root for the new Web application, or set the new project to reference the new Web application.
