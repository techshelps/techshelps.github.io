Add User Dialog Box

      

Use this dialog box to control access to your Web application by specific users.

This dialog box is available from the Users tab of the Permissions dialog box if Use unique permissions for this Web is selected on the Settings tab of the Permissions dialog box and your Web server is not Microsoft® Internet Information Server.

Options

Name

Type a unique identifier for the user. If the user is in a Microsoft® Windows NT® domain, precede the user name with a domain name and backslash, as follows: specialdomain\username.

Password

Type an identifying string for the user. This string displays as asterisks for added protection. The user can use this password to gain access to the Web application as determined by the Web permissions set.

Confirm password

Retype the password to confirm the string you chose. This provides verification that the string typed above is correct.

New user permissions

Browse this Web application

Select to specify that the user is allowed to view the content of the Web application through a browser. By default, all users have browse access to Web applications when they are created. The user is not allowed to create a new project that points to this application.

Author this Web application

Select to specify that the user is allowed to browse, open, and modify the Web application files, including adding or deleting content.

Administer this Web application

Select to specify that the user is allowed to change permissions on the Web application and create new Web projects as well as browse, open, and modify the Web application files.
