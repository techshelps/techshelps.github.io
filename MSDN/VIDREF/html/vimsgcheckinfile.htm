Check In File

There are several ways to handle a merged file in a Web project or database project:

  • To save the merged copy as your working copy, without checking the file into source control, choose Save.

  • To release the merged copy to source control and make your working copy read-only, choose Check in.

  • To return to editing the file without making either of these choices, choose Cancel.

To make the same decision for all merged files, choose Apply to all items.

For more information on working with source control in Microsoft® Visual InterDev™, see Working with Multiple Developers.
