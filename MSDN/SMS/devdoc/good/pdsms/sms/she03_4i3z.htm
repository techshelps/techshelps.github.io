Requirements for Gaining the Microsoft BackOffice Logo

Microsoft has developed the Designed for Microsoft BackOffice logo program to provide developers and users with consistent criteria and standards to use in developing and implementing client/server solutions that live in the Microsoft BackOffice environment. The goal of the logo program is to help users identify applications designed specifically to take advantage of BackOffice components and functionality, and to allow developers to garner the marketing benefits of using the logo on product packaging, advertisements, collateral, and other materials. Meeting the criteria required to obtain a logo ensures that your product uses and benefits from important BackOffice technologies.

To meet the Systems Management Server requirements for this program, all applications must:

  1. Support installation and deinstallation in an unattended mode.

    For information on this requirement, see SMS Requirements for Unattended Installation and Deinstallation.

  2. Provide a package command status MIF file with the required Status and Description entries.

    For information on this requirement, see The Package Command Status MIF File.

  3. Provide a Package Definition File (PDF) with all required entries.

    For information on this requirement, see The Package Directory and Package Definition File Format.

    For an example of a PDF file for a simple application, see Run Command on Workstation Jobs.
