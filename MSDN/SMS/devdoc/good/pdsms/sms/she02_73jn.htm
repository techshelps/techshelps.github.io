MIF File Syntax for Custom Architectures

In the MIF file, an object must have one Component section, an Identification group, an Architecture group, and optionally any additional groups that contain attributes describing the object.

For each attribute, follow a standard format that uses the same spelling for the attribute name, the same data type, and the same access entries.

When you create a MIF file to add an object with a custom architecture, you must ensure that the MIF file follows the DMI Specification version 4.5. See Desktop Management Interface (DMI) Compatibility
