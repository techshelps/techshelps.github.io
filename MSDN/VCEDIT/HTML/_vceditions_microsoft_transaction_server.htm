Microsoft Transaction Server

Feature Only in Enterprise Edition   This feature is supported only in Visual C++ Enterprise Edition. For more information, see Visual C++ Editions.

The Microsoft Transaction Server provides development tools and run-time services for building high-performance scaleable transaction applications. The programming model includes transactional components, instance streams, location transparency, naming, declarative security, thread pooling, instance recycling, and class factory caching. In addition, two or more components can share global variables accessed by name or property index. The ODBC resource dispenser manages pools of database connections for Microsoft Transaction Server components that use the standard ODBC interfaces. This allows quick and efficient database connections.

For more information, see Microsoft Transaction Server.
