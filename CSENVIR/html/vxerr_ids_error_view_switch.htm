In some cases, Edit and Continue will not permit changes immediately to compiled code but will apply these changes later as you continue debugging. The HTML view window shows code to be replaced in gray, and does not let you edit dimmed code.

Note   You cannot change views while you are debugging.

When the current function finishes running and is called next, Edit and Continue will make the changes to your compiled code, and the revised code will be undimmed in the editor.

To correct related errors

  • To open the page in Design view, finish debugging the page, turn off debugging, and then switch views.
  • To turn off HTML Validation, select Text Editor from Tools Options, and under HTML clear the "Enable HTML Validation" check box.
