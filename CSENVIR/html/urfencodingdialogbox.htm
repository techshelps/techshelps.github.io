Encoding Dialog Box

Use this dialog box to:

  • Open existing files (for example, .htm or .txt) that are improperly tagged, lack a character set attribute, or have an encoding that cannot be detected by the editor.
  • Change the character set of files created or modified in an editor.
  • Specify non-default encoding for a new file. If the file is not saved with a Unicode encoding, the Windows system default codepage (determined by the system default locale) will be used. For example, if your Windows system default locale is English, the Save As encoding will default to Windows codepage 1252, the encoding for the Western European language group.

Depending on the task you wish to perform, the Encoding dialog box can be accessed in several ways.

When you open a downloaded HTML file, the editor detects the charset parameter defined in the http meta tag. The Editor uses that information to convert the document to Unicode. However, many HTML pages do not contain this optional character set attribute. If the character set attribute is absent or incorrectly defined in the file you wish to open and if the file was created on a machine whose default encoding differs from the one you use, the file's characters will be incorrectly displayed. If you know the character set used by the author of such files, you can use the Encoding dialog box to select that character set. When notified of the HTML file's native character set, the editor converts the file into a form that is readable on your machine.

Encoding

Use this list box to select an encoding scheme for files. (Auto-detect) is selected by default.
