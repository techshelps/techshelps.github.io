Экспорт проектов

Вы можете выполнять Экспорт Проектов в другой каталог или файл щелкнув по кнопке Экспорт на Ленте - вкладка Главная. Эта возможность позволяет копировать загруженные файлы из одного или нескольких Проектов.

При экспорте загруженных сайтов Offline Explorer Pro изменит ссылки в файлах HTML или Java script согласно новым именам файлов. Вы можете использовать следующие параметры для тонкой настройки операции Экспорт:

Список слева позволяет Вам копировать экспортируемые файлы в указанный каталог или сжимать их в ZIP-файл, MHT-архив, электронную книгу EXE, сжатый файл HTML Help (.CHM) или передавать на FTP-сервер.

Примечание: Экспорт в MS IE Cache, MHT, EXE, CHM-файлы и передача на FTP поддерживаются только в версиях Pro и Enterprise.

MHT-архив не использует сжатия, но сохраняет все экспортируемые сайты в одиночный файл. Кроме того, MHT-архивы могут просматриваться непосредственно без использования MS Internet Explorer.

HTML Help имеет преимущества по сравнению с MHT-файлами, такие файлы занимают меньше места на диске из-за применения сжатия.

Однако и MHT, и HTML Help файлы могут быть просмотрены только на системах с установленным MS Internet Explorer 5.0 или выше.

Если Вы выберете Кэш MS Internet Explorer, Offline Explorer Pro отправит все экспортируемые файлы в кэш Internet Explorer и эти файлы будут доступны в режиме "Автономная работа".

Не создавать подкаталогов - все файлы размещаются в отдельном каталоге. Файлы с одинаковыми именами будут переименованы во избежание конфликтов с их именами (только в версиях Pro и Enterprise).

Если Вы отметите опцию Сделать автозапускаемый CD, то Offline Explorer Pro также создаст файлы autorun.inf и oe_open.exe. Все три файла, если они размещаются в корневом каталоге CD, делают CD автозапускаемым - когда Вы вставляете диск в привод CD-ROM, файл contents.htm автоматически открывается в браузере.

Экспорт только последних загруженных файлов - копирует только файлы, загруженные или обновлённые в ходе последней загрузочной сессии. Если какие-то ранее загруженные файлы не были обновлены, то они будут пропущены. Снимите флажок для экспорта всех файлов, принадлежащих Проекту.

Преобразование имен файлов - вы можете выбрать один из трёх возможных форматов:

  • Как есть - файлы копируются под своими оригинальными именами, что для Вас наилучшее решение при экспорте Проекта в другой каталог.

  • 8+3 - совместимость с MS-DOS - эта опция полезна при экспорте Проектов в Windows 3.11, на дискету, сетевой диск Novell или другие старые файловые системы. Offline Explorer Pro изменит все имена файлов так, чтобы они имели не более 8 символов в имени файла и 3 символов в расширении.

  • Joliet - CD-R 64 символа - подходит для публикации на дисках CD-ROM, так как они требуют, чтобы имя файла содержало не более 64 символов.

Использовать стандартные расширения для известных типов файлов - многие web-сайты содержат стандартные файлы с нестандартными расширениями. Например, HTML-файлы обычно используют расширения .htm или .html. Однако часто случается, что CGI-BIN сценарии, ASP-страницы, сайты Lotus Domino и многие другие, возвращают форматированное HTML содержимое под нестандартными именами файлов. Расширения могут заканчиваться как .shtml, .nsf, или .asp. Множество файлов может вообще не содержать никакого расширения.

Обычные web-браузеры определяют тип содержимого файла по его расширению. Таким образом, файл с нестандартным расширением будет использоваться как неизвестный, и при автономном просмотре без запуска браузера Offline Explorer Pro, браузеры могут просто предложить сохранить файл на диске, вместо того, чтобы показать его.

Для преодоления этой проблемы Offline Explorer Pro добавит к копируемым файлам стандартные расширения, например .htm, .gif, .jpg, .js и другие в зависимости от их действительного типа.

Запретить всплывающие окна - изменяет все ссылки всплывающих окон для открытия в том же окне браузера. Рекомендуется включать эту настройку, когда вы выполняете экспорт в файл формата HTML Help.

По завершении экспорта Проекта, Offline Explorer Pro создаст файл с именем contents.htm в каталоге с экспортированным сайтом. Этот файл содержит ссылки в экспортированном Проекте. Вы можете настроить формат contents.htm в версиях Offline Explorer Pro и Enterprise.

Сохранять онлайн-ссылки - если любой загруженный HTML-файл содержит онлайн-ссылки, то они будут сохранены в оригинальном виде без изменений и не будут преобразованы для автономного просмотра.

Добавлять исходный URL в HTML-файл - добавляет онлайн-URL к каждой экспортируемой web-странице как комментарий к содержанию страницы. Кроме того, это помогает решить проблему совместимости при просмотре экспортированных файлов в Windows XP Service Pack 2 или более поздней версией.

Ускоренный экспорт улучшает скорость анализа файлов во время выполнения экспорта. Однако некоторые ссылки могут работать неверно, особенно когда Вы используете Предотвращать переполнение каталогов (в диалоге Настройки). Также пропускаются перемещённые ссылки и ссылки в .ram-файлах.

Открыть файл после завершения экспорта - автоматически открывает файл после успешного выполнения экспорта, когда вы экспортируете Проект в каталог, ZIP, MHT, EXE или CHM файл.

Загрузка на FTP-сервер

Эта функция требует точного описания сервера. Также вы можете ввести каталог для передачи: ftp://ftp.site.com/path/. Если порт сервера не имеет значение 21, задайте этот путь: ftp://ftp.site.com:8021/

Также могут требоваться имя пользователя/пароль.

Передача на FTP выполняет экспорт в локальный каталог на вашем жёстком диске. Используются также другие настройки экспорта, как: формат имён файлов, стандартные расширения и так далее. По завершении экспорта, запускается утилита передачи файлов, которая отправляет файлы из каталога экспорта на сервер. Также вы можете удалить экспортируемые файлы из локального каталога после завершения передачи. Это не окажет влияния на удалённый сервер.

Offline Explorer Pro и Enterprise позволяют Вам выполнять экспорт Проектов автоматически по окончании загрузки.

Устранение проблем

Если у вас возникли проблемы с экспортом, таких как: файлы не были экспортированы, как вы ожидали, пожалуйста, проверьте карту Проекта - если ли все эти файлы в списке или их там нет. Экспорт работает только с файлами, присутствующими в карте Проекта.

Если карта Проекта неполная или отсутствуют загруженные Проектом и находящиеся на диске файлы, пожалуйста выберите Проект и нажмите на клавиатуре Ctrl + F5. Этим будет запущена загрузка Проекта в режиме "Не загружать существующие файлы". Offline Explorer найдёт все ранее загруженные файлы и добавит их в карту Проекта. Если некоторых файлов нет на диске, они будут загружены.
