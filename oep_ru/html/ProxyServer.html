Настройки: Интернет - Прокси-сервер

Компьютеры, размещённые за файерволом (firewall) или в пределах сети Intranet, вероятно имеют прокси-сервер для доступа к Интернет-серверам. Offline Explorer Pro в настоящее время поддерживает HTTP, SOCKS 4, SOCKS 5, RTSP и User@Site типы прокси.

Для определения типа прокси-сервера запустите Ваш любимый web-браузер и проверьте его параметры настройки. Чтобы установить параметры настройки прокси Вы должны знать адрес прокси-сервера и его порт, на котором он отслеживает запросы. Кнопка Автоопределить получает текущие настройки прокси из браузера MS Internet Explorer.

Могут понадобиться имя пользователя и пароль для доступа к некоторым HTTP прокси-серверам. Если Ваш прокси-сервер требует NTLM-авторизацию, добавьте пожалуйста Ваш домен в поле имя пользователя. Он должен быть отделён символом наклоненной влево черты:

DOMAIN\username

Тип прокси User@Site реализован в WinGate.

Режим PASV используется для FTP-загрузок при нахождении за файерволом.

Если у вас нет уверенности в правильности настроек параметров прокси-сервера, то попробуйте нажать кнопку "Автоопределить", которая поможет получить параметры настройки прокси из MS Internet Explorer. Если это не поможет, то проконсультируйтесь с администратором вашей сети.

Совет: Наиболее распространённые номера портов HTTP-прокси это значения 80, 800, 808, 8000, 8080, 3128. SOCKS прокси-сервер обычно использует порт 1080. User@Site прокси по умолчанию использует порт 21.

Вы можете задать список прокси-серверов. Offline Explorer будет использовать следующую запись для начала закачки каждого нового файла.

Метод альтернативной загрузки использует библиотеку WinInet из MS Internet Explorer. Это даёт лучшую совместимость при загрузке сайтов, имеющих на страницах формы авторизации.

PASV-метод для FTP-соединений является более совместимым способом, не требующим открытия активных портов на стороне клиента для загрузки FTP сайтов.
