Настройки: Интернет - Соединение

Для Интернет-соединения могут быть установлены разные параметры:

  • Количество соединений

    Offline Explorer Pro может загружать от одного до 500 файлов одновременно, делая загрузку простой и быстрой. Обычно используемое количество соединений - от 10 до 20. Меньшее количество соединений может сделать загрузки медленнее, в то время как большее - сделать их неустойчивыми из-за возможно долгого времени ожидания. Если Вы чувствуете, что Ваши загрузки неустойчивы, но Вы не хотите уменьшить количество соединений, то попробуйте увеличить значение таймаута (времени ожидания ответа сервера). Вы можете в любое время свободно изменить эту настройку. Если Вы увеличиваете количество соединений, то новые каналы немедленно заработают. Если Вы уменьшаете количество соединений, то дополнительные каналы не будут сразу удалены, поскольку Offline Explorer Pro будет ожидать окончания загрузки файлов на последнем из этих дополнительных каналов перед их удалением. Внимание: выбор более чем 100 одновременных соединений может вызвать неустойчивость системы, так как Windows не сможет обработать так много открытых сокетов TCP/IP.

  • Количество попыток

    Загрузки файла могут быть неудачными по нескольким причинам: сервер не найден, соединение не было установлено, файл не был передан полностью, выбранное время ожидания ответа сервера истекло, и так далее. Для того, чтобы избавиться от этих неудач, установите Ограничить количество попыток в Настройки | Интернет | Соединения, чтобы указать Offline Explorer Pro, сколько раз пытаться восстановить соединение. Если количество попыток соединения не установлено, то Offline Explorer Pro немедленно их повторяет до достижения успеха или прекращения попыток вручную.

  • Задержка между загрузками

    Для уменьшения потока данных, создаваемого Offline Explorer Pro, Вы можете увеличить в настройках задержку между загрузками. Как только файл загружен, Offline Explorer Pro будет ждать указанное число секунд перед запросом следующего. Вы также можете использовать случайные задержки времени, чтобы Offline Explorer Pro моделировал просмотр страниц человеком вручную. Это может помочь преодолеть защиту от оффлайн-браузеров на некоторых web-сайтах. Просто используйте следующий формат: 2-10. Приведённое значение укажет Offline Explorer Pro выбирать случайные задержки времени в диапазоне между 2 и 10 секундами перед загрузкой каждого файла.

    Вы можете задать нецелые задержки, вроде 0.23 секунды. Или 2.5-5.7.

  • Таймаут

    Это значение поможет тонко настроить Ваши загрузки. Если Offline Explorer Pro ожидает данные из Интернет дольше, чем время (в секундах), указанное в настройках времени ожидания ответа сервера, то соединение будет прервано. Большое число времени ожидания обычно означает непорядок в пути к серверу. Исправьте свехвысокое число времени ожидания или уменьшите количество соединений (количество каналов загрузки), или увеличьте значение времени ожидания.

  • Поддержка cookies

    Отметьте эту опцию для того, чтобы при загрузке web-сайтов использовать MS Iternet Explorer cookies. Это полезно при загрузке некоторых защищённых сайтов, которые хранят информацию о пользователе в cookie-файлах. В этом случае лучше всего зайти на такой сайт используя MS Internet Explorer, и затем загрузить его в Offline Explorer Pro. Некоторые сайты используют временные cookies, так что Вы должны сначала использовать Встроенный браузер в Offline Explorer Pro для входа на сайт. Offline Explorer Pro будет использовать как постоянные, так и временные файлы из внутреннего браузера.

  • Порт встроенного HTTP-сервера

    Offline Explorer Pro включает встроенный сервер HTTP (Web), чтобы облегчить просмотр и совместное использование загруженных страниц. По умолчанию встроенный HTTP-сервер использует порт 800. По желанию Вы можете изменить эту настройку. Для доступа к серверу с компьютера, на котором выполняется Offline Explorer Pro, Вы можете ввести: http://127.0.0.1:800/ в поле URL Вашего web-браузера. Пользователи на других компьютерах должны знать IP-адрес этого компьютера для доступа к загруженным страницам.

  • Идентификация агента

    Иногда необходимо изменить путь, представляемый Offline Explorer Pro самостоятельно другим web-серверам. Это называется "User Agent." По умолчанию это MS Internet Explorer. Offline Explorer Pro попробует самостоятельно идентифицировать версию MS IE для того, чтобы использовать эти данные. Есть несколько предварительно введённых значений пользовательского агента в поле раскрывающегося списка. Действующее значение представлено в поле Идентифицировать как. Если Вы хотите использовать другое, не представленное в списке значение, то выберите Использовать идентификацию и введите желаемый пользовательский агент. Значение Идентифицировать как не имеет значения для большинства web-сайтов. Однако у Вас могут быть проблемы при загрузке некоторых защищённых web-сайтов, или HTML-документы могут быть возращены в неверной кодировке до правильной установки (по мнению web-сайтов) Идентифицировать как.

  • Только местный доступ

    (Только версии Offline Explorer Pro и Enterprise)
    Когда эта настройка включена, Offline Explorer Pro запретит другим компьютерам доступ к встроенному HTTP (Web) серверу. Это может использоваться из соображений безопасности.

    Снятие отметки позволит вам использовать загруженные web-сайты совместно с другими людьми.
