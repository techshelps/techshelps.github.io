Свойства Проекта: Проект

Здесь Вы найдёте общие параметры Проекта.

  • Имя Проекта

    Введите название Проекта и оно станет видно в дереве Проектов.

  • Адреса (URL)

    Offline Explorer Pro начнёт загрузку Проекта, стартуя с этого интернет-адреса.

Версии Professional и Enterprise могут иметь множество URL в каждом Проекте.

Совет: для развёртки поля URL нажмите на клавиатуре клавишу F11. Если Вы хотите увидеть полный список дополнительных команд, нажмите клавишу F12. Ctrl-A выделяет весь текст.

В настоящее время Offline Explorer Pro поддерживает следующие типы URL:

  • http:// - для доступа к web-сайтам

  • https:// - для доступа к защищённым web-сайтам

  • mms:// - для доступа к потокам Microsoft Media Player

  • mmst:// - для доступа к TCP-потокам Microsoft Media Player

  • pnm:// - для доступа к потокам старой версии Real Player

  • rtsp:// - для доступа к потокам Real Player (поддержка протоколов HTTPS, MMS, PNM и RTSP доступна только в версиях Pro и Enterprise)

  • ftp:// - для доступа к FTP-сайтам

  • file:// - для доступа к HTML-файлам, хранящимся локально на Вашем жёстком диске

Форматы URL:

http://web_server_name/path/filename.ext

или

https://secured_server/path/filename.ext

mms://media_server_name/path/filename.ext

file://c:/path/filename.ext

или

file://c:\path\filename.ext

или

file://server/path/file.ext

Пожалуйста не забывайте при вводе URL вводить префиксы http://, https://, ftp://, mms://, mmst://, pnm://, rtsp:// или file://.

Вы можете определить множество URL в Проекте (только в версиях Pro и Enterprise). Пожалуйста вводите каждый URL в отдельной строке.

В поле URL могут быть добавлены комментарии, если вы введёте символ # в начале строки.

Версии Pro и Enterprise поддерживают макрокоманды URL. Макрокоманды полезны для автоматических загрузок по расписанию, когда URL каждый раз меняется.

Например, сайт с информацией о погоде в виде изображения, обновляющий информацию ежедневно. Его URL включает текущий день.

Таким образом, http://www.weather.com/image5.png соответствует 5-му дню текущего месяца, в то время как http://www.weather.com/image12.png соответствует 12-му дню текущего месяца.

В этом случае будет полезна следующая макрокоманда URL: http://www.weather.com/image{:day}.png

Когда Offline Explorer начинает загрузку Проекта, то макрокоманды меняют эти значения. Макрокоманда должна начинаться с символа {: и заканчиваться символом }

Поддерживаются следующие макрокоманды:

{:day} текущий день (1 - 31)

{:0day} текущий день, включающий при необходимости начальный нуль (01 - 31)

{:shortday} день как сокращённое название (Sun-Sat)

{:longday} день как полное его название (Sunday-Saturday)

{:month} текущий месяц как число без начального нуля (1-12)

{:0month} месяц как число с начальным нулём (01-12)

{:shortmonth} месяц как сокращённое название (Jan-Dec)

{:longmonth} месяц как полное его название (January-December)

{:year} текущий год как число из двух цифр (00-99)

{:longyear} год как четырёхзначное число (0000-9999)

{:week} текущий номер недели без начального нуля (1-51)

{:0week} номер недели с начальным нулём (00-51)

{:hour} текущий час без начального нуля (0-23)

{:0hour} час с начальным нулём (00-23)

{:minute} текущая минута без начального нуля (0-59)

{:0minute} минута с начальным нулём (00-59)

{:second} секунда, в которую стартует загрузка, без начального нуля (0-59)

{:0second} секунда с начальным нулём (00-59)

{:5..12} это специальная макрокоманда, которая загрузит URL с числами от 5 до 12. Вы можете также использовать начальный нуль в первом номере.

{:0..100|25} разделитель | определяет шаг перечисления. В этом примере будут сгенерированы числа 0, 25, 50, 75, 100.

{:a..z} загрузка URL с символами от a до z. Вы можете здесь использовать одиночные буквы от A до Z и a до z. Более подробная информация о макрокомандах URL доступна в меню Справка | Содержание | Дополнительные возможности | Использование макрокоманд URL.

  • Уровень

    Значение Уровня задаёт Offline Explorer Pro глубину поиска HTML-документов.

    • Уровень 0 - означает, что Offline Explorer Pro загрузит одну страницу, относящуюся к Адресу (URL) со всеми изображениями, видео, звуками, Java-сценариями, Java-классами, и так далее с этой страницы.

    • Уровень 1 - означает, что Offline Explorer Pro загрузит первую страницу и все страницы, связанные с ней, а также всё с этих страниц – изображения и так далее.

    • Уровень 2 - точно так же, как и для Уровня 1 и все страницы, связанные с загруженными. И так далее... Изображения, фреймы, Java-сценарии, Java-апплеты, звуки, видео – всё это будет загружено с web-страниц. С любым значением уровня, установленным Вами, Вы получите полные web-страницы. Другими словами, уровень – это количество кликов мышью по web-ссылкам, необходимых для того, чтобы перейти от начальной страницы к последней нужной странице. Обычное значение уровня – это число от 0 до 3. Установка более высокого значения уровня может привести к необходимости загрузки слишком большого количества web-страниц. Если Вы хотите загрузить полностью web-сервер, снимите отметку уровень и не забудьте отключить загрузку с других серверов через Фильтры URL | Сервер | Загружать файлы только с начального сервера. Иначе Вы можете пытаться получить содержимое всего Интернет на Ваш жёсткий диск.

  • Проверка изменений файлов

    Определяет – как Offline Explorer Pro обновляет загруженные web-сайты. Доступны следующие параметры настройки:

    • Загружать только изменённые и новые файлы

      Kогда Проект загружается, Offline Explorer Pro проверит каждый загруженный файл для того, чтобы узнать – изменился ли этот файл на web-сервере со времени последней загрузки. Если окажется, что файл тот же самый, то он не будет загружен во второй раз. Примечание: не все web-серверы и файлы поддерживают проверку на обновление. Например, CGI-BIN и ASP приложения, сценарии доступа к базам данных и так далее. Этот метод – лучший для частого обновления загруженных web-сайтов.

    • Проверка размеров файлов

      При загрузке Проекта, Offline Explorer Pro проверит каждый ранее загруженный файл для того, чтобы узнать – изменился ли этот файл на web-сервере со времени последней загрузки. Если окажется, что файл тот же самый, то он не будет загружен во второй раз. Примечание: не все web-серверы и файлы поддерживают проверку на обновление. Например, CGI-BIN и ASP приложения, сценарии доступа к базам данных и так далее. В таких случаях Вы можете также проверить файл на обновление по его размеру. Если размер файла не изменился со времени последней загрузки, то Offline Explorer Pro предположит, что файл не изменялся. В редких случаях может быть возможно, что изменённый файл имеет тот же самый размер. Так что пожалуйста используйте эту возможность осторожно.

    • Пропуск существующих медиа-файлов

      Обычно изображения обновляются на web-сайтах очень редко. Если Вы отметите поле Пропуск существующих медиа-файлов Offline Explorer Pro загрузит только новые изображения, видео- и аудиофайлы, и не будет проверять – обновились или нет ранее загруженные медиа-файлы. Это может значительно уменьшить время проверки обновления web-сайта.

    • Загружать все файлы

      Offline Explorer Pro не будет выполнять никаких проверок на обновления при загрузке web-сайтов. Каждый ранее загруженный файл будет перезаписан поверх. Этот метод может использоваться, если что-то случилось с загруженным web-сайтом и Вы должны полностью заменить его.

    • Не загружать существующие файлы

      Offline Explorer Pro не будет выполнять никаких проверок на обновления при загрузке web-сайтов. Вместо этого каждый уже загруженный файл будет сохраняться и даже его обновлённая копия не будет загружаться. Это лучший способ завершения частично загруженных web-сайтов, например, если предыдущая загрузка была по некоторым причинам прервана.

    • Пропускать существующие файлы на уровне больше

      Этот метод – один из лучших для загрузки часто обновляемых новостных сайтов, и так далее. Offline Explorer Pro проверит каждую страницу на указанных Уровнях и страницы будут загружены только в случае, если они отсутствуют на Вашем жёстком диске. Например, большинство сайтов новостей имеет страницу индексов со ссылками на статьи новостей и многим страницам новостей. Эти статьи никогда не изменяются, они только часто добавляются или удаляются. В этом случае настройка "Пропускать существующие файлы на уровне больше 0" будет подразумевать, что первая страница (на Уровне l 0) будет проверена и загружена, если она изменилась, а затем Offline Explorer Pro будет загружать только отсутствующие статьи новостей. Все ранее загруженные страницы не будут ни проверяться на обновление, ни заново загружаться.

Советы: для развёртки поля URL нажмите на клавиатуре клавишу F11 (только для версий Pro и Enterprise). Если Вы хотите увидеть полный список дополнительных команд, нажмите клавишу F12. Ctrl-A выделяет весь текст.

Для получения дополнительной информации о макрокомандах URL, нажмите здесь.

Поле URL в Offline Explorer Pro и Enterprise может содержать некоторые дополнительные команды. Полный список доступных параметров смотрите здесь.

Вы можете также выполнять следующие задачи, используя поле URL:

Автоматический экспорт или печать после завершения загрузки.

Автоматическое удаление старых файлов Проекта после завершения загрузки.

Определение произвольных referer, cookie или POST-запроса.

Сбор адресов E-mail из загруженных web-страниц.

Индексация загруженных страниц

Поиск ключевых слов в загруженных файлах по содержанию может быть выполнен мгновенно, если вы включите опцию "Индексировать загруженные файлы для быстрого поиска".
