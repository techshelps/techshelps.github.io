Файлы .primary и Descr.wd3

В ходе выполнения загрузки Offline Explorer Pro может создавать файлы с расширениями .primary и descr.wd3 files.

Цель файлов .primary – помощь при обновлении загруженного web-сайта. Когда бы Offline Explorer Pro не предложил загрузить неизменённые файлы, каждый файл будет проверен – изменился ли он с даты последней загрузки. Если нет, то файл не будет загружен. Если это файл HTML, то Offline Explorer Pro должен иметь копию оригинального файла с неизменёнными HTML-ссылками. Файлы .primary – это оригинальные и неизменённые HTML-файлы.

Primary-файлы также используются, когда существующие файлы не загружены. Для следования по ссылкам Offline Explorer Pro анализирует .primary HTML-файлы.

Примечание: файлы .primary не увеличивают поток данных. Offline Explorer Pro загружает только обычные файлы, а файлы .primary локально создаёт позже.

Offline Explorer Pro начиная с версии 2.0 не создаёт файлы .primary, если Вы не добавите следующую строку в поле URL Проекта:

Additional=KeepPrimary

Файлы Descr.wd3 постоянно создаются и находятся в каждом каталоге Offline Explorer Pro; они содержат информацию о всех загруженных файлах в каталоге:

- имя файла

- URL оригинального файла

- дата последнего изменения файла на web-сервере (для проверки на обновления)

- тип файла MIME, поскольку его возвращает web-сервер (это полезно для оффлайн-просмотра)

- подпись, если файл .primary был создан или нет.

Хотя все эти файлы могут быть удалены, это всё же не рекомендуется потому, что это может сделать обновления web-сайта и оффлайн-просмотр менее устойчивыми и надёжными.
