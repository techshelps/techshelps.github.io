Как после загрузки автоматически выполнить экспорт или печать Проекта?

Примечание: эта возможность доступна только в версиях Offline Explorer Pro и Enterprise.

Автоматический экспорт или печать будут выполнены после полного завершения загрузки Проекта.

Введите параметры экспорта в поле URL ниже начального URL (на отдельной строке). Например:

http://www.metaproducts.com/

Additional=AutoExport=c:\export\;1010001000

Замените c:\export\ полным путём к каталогу, в котором будут размещены экспортируемые файлы.

Символ ";" отделяет путь от настроек в окне Экспорт. 1 означает отметить значение, 0 - снять отметку: первый символ - это "Использовать стандартные расширения", второй - "Формат имён файлов" (0 - как есть, 1 - 8+3, 2 - Joliet), третий - "Сохранять онлайн-ссылки", четвёртый - "Помещать файлы в" (0 - каталог, 1 - Zip-архив, 2 - MHT-архив, 3 - кэш MS IE, 4 - HTML Help (CHM), 5 - сжатую EXE-книгу), пятый - не используется, шестой - "Создать Автозапускаемый CD", седьмой - создать файл Contents.htm, восьмой - не создавать подкаталоги, девятый - экспорт только недавно загруженных файлов, десятый - добавление оригинального URL в HTML-файлы, одиннадцатый - быстрый экспорт, двенадцатый - открыть файл после экспорта, тринадцатый - отключить всплывающие окна, четырнадцатый - создать файл отчета report.txt, пятнадцатый - удалить экспортированные файлы после выгрузки на FTP.

Для выгрузки на FTP, используйте команду подобную этой:

Additional=AutoExport=c:\export\;101700100000001;ftp://user:password@ftp.server.com/folder/

Вы также можете использовать Макрокоманды URL в параметрах автоэкспорта. Например:

Additional=AutoExport=c:\export{:day}\;101

Так выполняется экспорт Проекта в каталоги c:\export1\, c:\export2\ ... c:\export31\ в зависимости от даты загрузки.

После добавления строки AutoExport в поле URL-адресов Проекта, Вы можете протестировать его, выбрав Проект в дереве Проектов и нажав

Ctrl+E, чтобы открыть диалог экспорта. Последний будет использовать настройки автоэкспорта Проекта.

Команда печати выглядит так:

Additional=Print

Две другие команды печати:

Additional=PrintWhenUpdated - напечатать все web-страницы проекта только когда будут загружены один или более новых/обновленных файлов.

Additional=PrintUpdatedOnly - напечатать только новые и обновленные web-страницы Проекта.

Вы можете объединить команды экспорта и печати следующим способом:

Additional=Print;AutoExport=c:\export{:day}\;101

Additional=DeleteProjectFiles - удаляет загруженные файлы. Применяется, когда вы хотите выполнить экспорт Проекта автоматически и не сохранять файлы в каталоге загрузки.

Другое дополнение к возможности удаления файлов - это способность удаления некоторых файлов по ключевым словам после того, как файлы загружены и проанализированы:

DeleteAfterParsing=sports*.htm,ads.shtml?param1=*&param2={:10..50}
