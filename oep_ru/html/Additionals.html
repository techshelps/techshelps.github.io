Список всех параметров команды Additional=

Вы можете размещать команды Additional= в отдельной строке после всех URL в поле Адреса диалога Свойства Проекта. Они помогут Вам автоматически выполнять разные операции в загруженном web-сайте после окончания загрузки:

Additional=DeleteOldFiles

Удаление из локальной копии файлов, которых больше нет на сервере.

Additional=CountCRC

Подсчёт CRC загруженных файлов для улучшения проверки обновлений.

Additional=Print

Печать загруженного сайта автоматически сразу после завершения загрузки.

Additional=CollectEMails=c:\dir\file.txt

Поиск всех адресов E-mail в загруженных страницах и сохранение найденных в указанный текстовый файл. Подробная информация здесь.

Additional=AutoExport=...

Автоматическое выполнение экспорта по окончании загрузки. Нажмите сюда для получения дополнительной информации.

Additional=DeleteProjectFiles

Удаление всех загруженных файлов. Может применяться, например, в комбинации с параметром AutoExport.

Additional=DeleteFilesBeforeDownload

Удаление всех загруженных файлов проекта во время старта загрузки, поэтому файлы всегда сохраняются в пустой каталог загрузки.

Additional=DeleteProjectSettings

Удаление настроек Проекта по завершении загрузки.

Для удаления и настроек и файлов Проекта, используйте обе команды сразу: Additional=DeleteProjectFiles;DeleteProjectSettings

Additional=SuspendToFileOnLimit=c:\dir\file.wdq

Приостановка загрузки Проекта в файл, если ограничитель был установлен в Свойствах Проекта | раздел Дополнительно.

Additional=FinalizeAllFilesOnStop

Завершение загрузки и сохранение даже незагруженных полностью файлов Проекта, или когда Проект должен быть остановлен автоматически из-за заданных ограничений.

Additional=DisableScripts

Отключение извлечения ссылок из сценариев Java и VB.

Additional=DisableJava

Отключение извлечения Java-апплетов (.class файлов).

Additional=KeepPrimary

Offline Explorer всегда будет создавать копию каждого HTML-файла, добавляя к нему расширение ".primary". Файлы .primary есть оригинальные исходные HTML-файлы с неизменёнными ссылками, так как они и располагаются на web-сервере. Обычные HTML-файлы имеют все ссылки, изменённые для автономного просмотра.

Additional=ReuseFTP

Использовать для загрузки разных файлов те же FTP-соединения (может не работать на некоторых FTP-серверов).

Additional=ConvertPOSTToFileName

Добавляет содержимое POST-запроса к имени сохраняемого файла после @ символа. Это помогает избежать перезаписи файлов с разным содержанием, когда Вы разрешаете обрабатывать HTML–формы или используете макрокоманды URL в POST.

Additional=SkipIFrames

Пропуск обработки IFRAME-тэгов, обычно содержащих рекламу.

Additional=DepthFirst

Добавление новых URL в самый верх очереди загрузки, так что сначала загружаются изображения, и так далее.

Additional=NoMovedDuplicates

Прекращение обработки 302 Moved HTTP-команды, если этот URL уже был загружен.

Additional=AbsoluteURLs

Другой метод трансляции ссылок - преобразование всех ссылок в абсолютные онлайн URL.

Additional=AddOriginalURL

Добавляет комментарий к каждой загруженной web-странице, который содержит оригинальный URL страницы.

Additional=NoMap

Пропустить добавление загруженных файлов в карту Проекта.

Additional=DoNotParseFiles

Только загрузка файлов, без их анализа и следования по ссылкам.

Additional=DoNotParseExistingFiles

Пропуск получения ссылок из ранее загруженных файлов.

Additional=ParseIncludedScripts

Эта команда включает обработку внешних .js файлов в web-страницах, связанных с ними ссылками. Тогда все сценарии будут целиком анализироваться для получения большего количества ссылок для некоторых сайтов, содержащих сложные сценарии. Необходимо также разрешить обработку сценариев в Свойствах Проекта - раздел Дополнительно.

Эта функция по умолчанию отключена, потому что замедляет обработку загруженных файлов.

Additional=SkipDisposition

Некоторые серверы могут сообщать альтернативное имя файла, отличающееся от URL. Например, об URL http://www.srv.com/img.asp?id=17 могут сообщить как flower.jpg. В этом случае Offline Explorer Pro создает оба файла - img.asp@id=17 и flower.jpg. Первый файл будет содержать только небольшой код, перенаправляющий на второй файл. Эта команда блокирует такое поведение, и альтернативные имена файлов игнорируются. Это полезно, когда разные URL-адреса с разным содержанием сообщают одинаковое имя файла.

Additional=MoveOldFiles=c:\dir\

Не перезаписывать файлы, которые изменились с момента последней загрузки, а переместить их в указанную папку. Однако для сохранения старых копий измененных файлов лучше использовать функцию Копии файла в диалоге Свойства Проекта.

Additional=KeepCurrentURLs

Если Вы запускаете Проект второй раз, то не будут скачиваться ссылки, загружаемые в текущий момент.

Additional=KeepAMP

Сохраняет символы &amp; в ссылках. Иначе они преобразовываются в &.

Additional=ConvertWebArchive

Воссоздаёт оригинальный сайт и восстанавливает структуру папок из копии с web.archive.org.

Для объединения нескольких параметров в одной строке используйте символ точка с запятой:

Additional=Print;DeleteOldFiles

или

Additional=AutoExport=c:\export\;101;DeleteProjectFiles

Вы можете сократить написание команды Additional= до простого Add=:

Add=AutoExport=c:\export\;101;DeleteProjectFiles
