Conclusion

Well, there you have it. As you can see, you can use ADO in Visual Basic .NET under the covers in basically the same way that you did in prior versions of VB.

Just as a parting word of advice for this Appendix: Convert as much of your code to take advantage of ADO .NET as possible. However, if you have several developers and need to crank out a simple application with Visual Basic .NET in a hurry, ADO is still available.
