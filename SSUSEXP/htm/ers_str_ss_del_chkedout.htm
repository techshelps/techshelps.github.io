File <File name> is currently checked out by <user name>.  Delete anyway?

This warning has the following cause and solution:

  • You are attempting an operation that would delete a file currently checked out to another user.

    If you delete this file, the check out information will be lost, even if the file is not permanently destroyed.  In this situation, VSS gives you some options:

    • Click Yes to delete the file.

    • Check Yes All to have your choices in this dialog box apply to all other files checked out to other users.

    • Click No to skip the operation for this file.

    • Click No All to skip the operation for all files checked out to other users.

    • Click Cancel to cancel the operation.
