<File name> stores only the latest version and is already checked out.

This warning has the following cause and solution:

  • VSS only allows one user to check out a file that is set to store only latest version.  This is because VSS can't merge the changes of a second user as the ancestor file has been replaced by the first users check in.

    Identify who has the file checked out and have them perform the Check In or Undo Check Out command.  If you can't find the individual, the Admin user can check in files another user has checked out.

    To identify who has a file checked out, view the Check Out Status tab on the file properties dialog or use the ss status command.
