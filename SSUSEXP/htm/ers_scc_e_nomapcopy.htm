The file <File name> could not be mapped to the VSS project <Project name>.  Copy the file to <Project name>

This error has the following cause and solution:

  • You added a file from a folder other than your project folder (or one of its subdirectories) and VSS can't determine which project corresponds to your new file's home folder.

    VSS must map the local copy into a VSS project path for this operation to work.  Copy the file to a subfolder at or below your IDE project.  If the file is already in VSS in a different project, share the file to your current project and try the operation again.

    You will continue to get this message for the file as long as the name cannot be mapped.  If this happens, you should remove the file from your project file, copy or share it, and then re-add the file.
