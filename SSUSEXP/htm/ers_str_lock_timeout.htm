Timeout locking file: <File name>

This warning has the following causes and solution:

  • VSS can be set up to use its own form of file locking.  This has to be set by the VSS Administrator in the SRCSAFE.INI file.  This error indicates a lock file (.LCK extension) exists in the \DATA\LOCKS folder. This situation occurs under either of the following circumstances:
    • A user has a file locked. For example, a file is locked while it is updated.

    • A VSS lock file has not been deleted from the \DATA\LOCKS folder.

    • A \DATA\LOCKS folder doesn’t exist.

    If a user has a lock on a file then, just retry the command in a few minutes. 

    If the lock hasn’t been deleted from the \DATA\LOCKS folder, go there and delete the .LCK file, however, please make sure everyone is out of VSS before deleting .LCK files or you may affect another users process.

    If this warning appears when you log in to VSS, check to see if a \DATA\LOCKS folder exists.  If it doesn’t, create one and attempt to log in again.
