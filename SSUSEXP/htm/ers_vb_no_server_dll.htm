Unable to load the source code control provider.

This error has the following causes and solutions:

  • The registry strings used to find the source code control provider dynamic-link library for example, SSSCC.DLL aren’t correct, or you have the wrong version of Visual Basic, or the file SSSCC.DLL is missing.

  • You moved the source code control folder after setup without updating the registry.

  • The setup program failed.

    Check your source code provider's setup instructions.


    Rerun the VSS setup program.
