File <File> is exclusively checked out

This warning has the following cause and solution:

  • If Multiple Checkouts is enabled, users in VSS can check out files in two ways.  First they can allow other users to check out the file or they can check it out so no one else can check out that file, namely exclusively.  This message appears when a user attempts to checkout a file that is already exclusively checked out by someone else.  It frequently appears with the "<File> is already checked out, continue?" message.

    You will not be able to check out this file until the user who has the file checks it in or performs an undo check out.  To find out who has the file checked out, you can use the ss status command or look at the Check Out Status tab on the file Properties dialog. The VSS Administrator can also undo checked out files that another user has checked out.
