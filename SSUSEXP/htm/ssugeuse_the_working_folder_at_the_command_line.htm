Use the Working Folder at the Command Line

See Also CmdLine_Comment; Concept_AltCmdlineWords;

When you are working with the VSS command-line interface, commands such as Get and Check Out place files into your current folder, not into your working folder for the project as they do in the Windows interface.

VSS does not, by default, change your current folder when you issue an ss CP command (change current project); nor does it change your project if you move from folder to folder. That is, the working folder has, by default, no effect on the command line and vice versa.

To change the working folder of a project:

  • At the command prompt, type ss workfold $/the project name C:\the working folder name

    For example, to change the working folder for a project $/myprj from C:\default to C:\Mystf\Myfld, type the following:

    ss workfold $/Myprj C:\Mystf\Myfld
