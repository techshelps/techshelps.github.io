Force_Prj Initialization Variable

See Also

Automatically sets the current project, based on the current directory. Applies only to the VSS command-line program (SS.exe).

Syntax

Force_Prj = {Yes|No|Ask}

Example

Instructs VSS not to automatically change the current project:

Force_Prj = No

Remarks

  • Yes First looks at your current directory whenever you execute any VSS command. If this is the working folder for a VSS project, VSS automatically changes to that project before executing the command. You can change directories to move around your projects, without executing the SS CP command.

  • No Does not change to your current project.

  • Ask Prompts before changing your project.

If you set Force_Prj = Yes, it is highly recommended that you do not give two projects the same working directory, because VSS won't know which project to switch into.

You can also set this variable in your SS.INI file by using the Assume Project Based on Working Directory option on the Command Line Options tab in the SourceSafe Options dialog box.
