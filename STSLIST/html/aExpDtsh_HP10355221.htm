About exporting a Datasheet view

You can export from SharePoint Foundation 2010 to Microsoft Access 2010 or Microsoft Excel 2010. For Access 2010, you can export to either a static or a linked table in a new or existing database. For Excel 2010, you can create a query of the Datasheet view in a new or existing worksheet, or a new workbook.

Exporting to Access

When you export from SharePoint Foundation 2010 to Access 2010, you are creating a static database table with data from Datasheet view. This static database table is not linked to the Datasheet view and does not update when changes are made to the data in SharePoint Foundation 2010.

Tracking a list in Access

When you choose Track this List in Access in the Task Pane, you create a table in Access that links dynamically to SharePoint Foundation 2010. If you change the data in Datasheet view, the changes are automatically updated in the Access table. Similarly, if you change the data in the Access table, the changes are automatically reflected in Datasheet view.

 Note   In some types of lists, there are columns that can be modified in the Datasheet view in SharePoint Foundation 2010, but not in linked Access tables.

Querying a list with Excel

When you query a Datasheet view with Excel, you are creating a query in Excel that retrieves data from Datasheet view. Once a Datasheet view has been exported to Excel 2010, any changes made to the data in the Datasheet view are updated automatically in the Excel worksheet. However, changes to the Excel worksheet are not transmitted back to SharePoint Foundation 2010.
