SetACL

Sets the rights for folders, documents or dataclass.

Public Function SetACL(ByVal ObjectId As Long, ByVal ObjectType As String,ByVal GrpUserCount As Long, ByRef GrpUserRights() As Byte) As Long

 

Parameters

In/out

Name

Data type

Description

In

 

ObjectId

Long

 

Index of the Object on which rights are being set

In

ObjectType

String

Type of object for on which rights are being set 'F' for folders, 'D' for Documents and 'T' for dataclass

In

 GrpUserCount

Long

Total number of groups plus users  who have rights on this object

In/Out

 GrpUserRights

Byte[TagACL]

[In]Rights to be set

[Out] Rights retrieved after being set

 

Description

This function is called only for shared cabinet from both admin and desktop. It is called when rights are either set or changed on folders ,documents or dataclass. Object type is set to “T” for dataclass, ”F” for folders and “D” for documents.

 

Return value

Name

Description

 1

In case of success

-1

Invalid cabinet name

-3

Cabinet not connected

-201

Insufficient rights for the current operation

-133

User not logged in

 

Example

Dim udtRights              As TagACL

Dim ObjectId               As Long

Dim ObjectType          As String

Dim GrpUserCount      As Long

Dim bGrpUserRights()   As Byte

Dim lRetVal                As long

Dim gPDb                 As PanDb.pdbCls


Set gPDb = New PanDb.pdbCls

ObjectId = 22

ObjectType = “F”

GrpUserCount = 2

udtRights.GrpUserId = 7

udtRights.GrpUserType =”U”

udtRights.GrpUserRights = “101010”

 

Redim bGrpUserRights(len(udt Rights) -1)

Call CopyMemory(bGrpUserRights(0),udtRights,Len(udt Rights))

lRetVal = gPDb.SetACL(ObjectId,ObjectType,GrpUserCount, bGrpUserRights)

    

If lRetVal <> NG_SUCCESS Then

        MsgBox gPDb.ErrDescription, vbInformation

        Screen.MousePointer = vbNormal

        Exit Sub

  End If
