No application is associated with this file type

This error has the following cause and solution:

  • You have attempted to view a file for which no file association exists. Without this association, VSS doesn't know which editor to use to display the file.

    Use File Manager to add an association from this file to the appropriate editor, then try your command again.

    If you want to override the File Manager association for a specific file type, you can enter specific associations in your SS.INI file, for example:

    .C = C:\VC\VC.EXE
.FRM = C:\WINDOWS\NOTEPAD.EXE
