File <File> has been destroyed, and cannot be rebuilt

See Also

VSS may not be able to reproduce the file you requested. This error has the following cause and solution:

  • You permanently destroyed a file or moved the file to another location. For example, there may have been a file in a project that you deleted or purged. Later, you attempt to go back to an earlier version of the project. VSS attempts to reconstruct the project exactly as it was in the past, but it cannot, because the file cannot be recovered.

    Click Yes, and VSS tries to restore the old file, but restoration is not guaranteed.

    Click No, and the attempt to restore an old file is halted.

    Click Yes All, and VSS does not notify you that other selected files may also be unrecoverable, but continues with the best possible reconstruction.
