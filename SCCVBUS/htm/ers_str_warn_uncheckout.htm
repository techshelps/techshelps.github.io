<File> has changed. Undo check out and lose changes?

See Also

This warning has the following cause and solution:

  • You are about to undo the check out of a file. If you do so, you discard all the changes made since you checked out the file.

    If you want to put your changes into VSS instead of losing them, use the Check-In command instead.

Note   If you do not want to receive this message every time you undo a file check out, you can disable it on the Warnings tab in the SourceSafe Options dialog box on the Tools menu.
