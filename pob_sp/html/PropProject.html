Diãlogo Propiedades de Proyecto: un Proyecto

Aquí encontrarã parãmetros generales de Proyecto.

  • Nombre de Proyecto

    Ingrese el nombre del Proyecto tal como aparece en el ãrbol de Proyectos.

  • Dirección (URL)

    Portable Offline Browser comienza a descargar un Proyecto empezando por esta Dirección de Internet.

Usted puede tener múltiples URLs por Proyecto.

Consejos:Si desea expandir el campo de URLs, presione F11en su teclado. Si desea ver una lista completa de comandos adicionales, presione F12. Ctrl-A selecciona todo el texto.

Actualmente, Portable Offline Browser es compatible con los siguientes tipos de URL:

  • http:// - para acceder a sitios web

  • https:// - para acceder a sitios web protegidos

  • mms:// - para acceder a paquetes de datos Microsoft Media Player

  • mmst:// - para acceder a paquetes de datos Microsoft Media Player TCP

  • pnm:// - para acceder a versiones anteriores de paquetes de datos de Real Player

  • rtsp:// - para acceder a paquetes de datos de Real Player

  • ftp:// - para acceder a sitios FTP

  • file:// - para acceder a archivos HTML guardados localmente en su disco duro

Formatos URL:

http://web_server_name/path/filename.ext

o

https://secured_server/path/filename.ext

mms://media_server_name/path/filename.ext

file://c:/path/filename.ext

o

file://c:\path\filename.ext

o

file://server/path/file.ext

Por favor, no olvide de incluir el prefijo http://, https://, ftp://, mms://, mmst://, pnm://, rtsp:// or file:// al ingresar la URL.

Usted puede especificar múltiples URLs en un Proyecto . Por favor ingrese una URL por línea.

Se pueden agregar comentarios al campo de las URLs si tipea el símbolo # al comienzo de la línea.

Portable Offline Browser soporta macros de URL. Los macro son útiles para programar descargas automãticas cuando la URL cambia cada vez.

Por ejemplo, un sitio web tiene una imagen que se actualiza diariamente. Su URL tiene un número de día dentro.

Así, http://www.weather.com/image5.png corresponde al 5º día del corriente mes, mientras que http://www.weather.com/image12.png corresponde al 12º día del corriente mes.

En tal caso, los siguientes macros de URL serãn útiles: http://www.weather.com/image{:day}.png

Cuando Portable Offline Browser comienza a descargar el Proyecto, sustituye los macros con sus valores. El macro deberã empezar con los símbolos {: y terminar con }

Se soportan los siguientes macros:

{:day} el número del día actual (1 - 31)

{:0day} día actual con un cero delante, cuando sea necesario (01 - 31)

{:shortday} día como abreviatura (Dom-Sab)

{:longday} nombre completo del nombre del día (Domingo- Sãbado)

{:month} el mes corriente como número, sin cero delante (1-12)

{:0month} el mes corriente como número, con cero delante (01-12)

{:shortmonth} el mes como abreviatura (Ene-Dic)

{:longmonth} nombre completo del mes (Enero- Diciembre)

{:year} el año corriente como número de dos dígitos (00-99)

{:longyear} el año como número de cuatro dígitos (0000-9999)

{:hour} la hora actual sin cero delante (0-23)

{:0hour} la hora actual con cero delante (00-23)

{:minuto} el minuto actual sin cero delante (0-59)

{:0minute} el minuto actual con cero delante (00-59)

{:5..12}este es un macro especial que cargarã URLs con números del 5 al 12. Usted también puede usar ceros delante del primer número.

{:0..100|25} el separador | define la etapa de enumeración. En este ejemplo se generarãn los números 0, 25, 50, 75, 100 .

{:a..z}para cargar URLs con letras de la a a la z. Aquí puede usar una letra de la A a la Z y de la a a la z. Mãs información sobre macros de URL disponible en el menú Ayuda | Contenido | Funciones Avanzadas | Cómo usar macros de URL.

  • Nivel

    El valor de Nivel define con cuanta profundidad Portable Offline Browser buscarã documentos HTML.

    • Nivel 0 significa que Portable Offline Browser cargarã una pãgina, remitido por una dirección (URL) con todas las imãgenes, vídeo, sonidos, secuencias de comandos de Java, clases de Java, etc., en esa pãgina.

    • Nivel 1significa que Portable Offline Browser cargarã la primera pãgina y todas las pãginas vinculadas a ella, con sus imãgenes, etc.

    • Nivel 2 - todo igual que en el Nivel 1 y cada pãgina vinculada a aquellas descargadas. Y así sucesivamente... Imãgenes, marcos, secuencias de comandos de Java, applets de Java, sonidos, video - todos ellos se descargarãn con la pãgina web. Con cualquier configuración de nivel obtendrã pãginas web completas. En otras palabras, el Nivel es el número de clics del ratón sobre los vínculos web necesarios para ir de la primera pãgina a la última deseada. El valor usual del Nivel va de 0 a 3. Configurar el Nivel en valores mãs altos puede hacer que se carguen demasiadas pãginas web. Si desea descargar un servidor web completo, desmarque la casilla de verificación de Nivel y no olvideapagar la descarga desde otros servidores, seleccionando Filtros URL | Servidor | Cargar archivos sólo desde el servidor inicial. De otra forma puede ocurrir que todo el contenido de internet se descargue a su disco duro.

  • Comprobación de modificación de Archivo

    Define cómo Portable Offline Browser actualizarã los sitios web descargados. Se permite la siguiente configuración:

    • Descargar sólo archivos modificados

      Al descargar el Proyecto, Portable Offline Browser verificarã cada archivo descargado anteriormente para determinar si fue modificado en el servidor web desde la última descarga. Si el archivo parece ser el mismo, no serã descargado por segunda vez. Nota: no todos los servidores web y archivos son compatibles con la comprobación de modificación de archivo. Por ejemplo, aplicaciones CGI-BIN y ASP, base de datos de secuencias de comandos de acceso, etc. Este método es mejor para actualizar sitios web descargados frecuentemente.

    • Verificación de tamaño de archivo

      Al descargar el Proyecto, Portable Offline Browser verificarã cada archivo descargado anteriormente para determinar si fue modificado en el servidor web desde la última descarga. Si el archivo parece ser el mismo, no serã descargado por segunda vez. Pero no todos los servidores web y archivos son compatibles con la comprobación de modificación de archivo. Por ejemplo, aplicaciones CGI-BIN y ASP, base de datos de secuencias de comandos de acceso, etc. En este quizãs desee verificar las modificaciones de archivo también por tamaño de archivo. Si el tamaño de archivo no ha cambiado desde la descarga previa, Portable Offline Browser considera que el archivo es el mismo. En pocos casos sucede que el archivo actualizado tiene el mismo tamaño de archivo. Por tanto, por favor, utilice esta función con cuidado.

    • Saltear archivos Media existentes

      Por lo general, las imãgenes en los sitios web muy raramente cambian. Si activa la casilla Saltear archivos Media existentes, Portable Offline Browser únicamente cargarã nuevas imãgenes, archivos de audio y de video, y no verificarã si los archivos de medios descargados previamente han cambiado. Esto puede reducir significativamente el tiempo de actualización del sitio web.

    • Descargar todos los archivos

      Portable Offline Browser no llevarã a cabo la verificación de modificación al descargar sitios web. Cada archivos descargado previamente serã sobrescrito. Este método puede usarse si ha sucedido algo a un sitio web descargado y usted necesita reemplazarlo completamente.

    • No descargar archivos existentes

      Portable Offline Browser no llevarã a cabo la verificación de modificación al descargar sitios web. En cambio, cada archivo descargado serã mantenido y ni siquiera se descargarãn copias actualizadas. Esta es la mejor forma de completar sitios Web descargados parcialmente, por ejemplo, si la descarga previa fue interrumpida por alguna razón.

    • Saltear archivos existentes en niveles mayores a

      Este método es el mejor para la descarga de sitios de noticias, etc. que se actualizan frecuentemente. Portable Offline Browser verificarã cada pãgina en los niveles especificados y sólo descargarã pãginas que no estén en su disco duro local. Por ejemplo, la mayoría de los sitios de noticias tienen una pãgina índice con vínculos a artículos de noticias y varias pãginas de artículos. Estos artículos no son actualizados, sólo se agregan o borran a menudo. En este caso, definir la configuración "Saltear archivos existentes en niveles mayores a 0" significa que la primera pãgina (en el Nivel 0) serã verificada y descargada si ha sido modificada, y luego Portable Offline Browser descargarã únicamente los artículos de noticias faltantes. Todas las pãginas cargadas previamente no serãn ni verificadas por modificaciones ni descargadas nuevamente.
