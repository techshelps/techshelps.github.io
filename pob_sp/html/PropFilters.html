Diãlogo Propiedades de Proyecto: Secciones de filtros URL

Los filtros URL le permiten controlar fãcilmente las descargas de Proyectos configurando los archivos que se deben cargar y los que se deben saltear.

Los Filtros URL se dividen en cuatro partes:

  • Protocolo - restringir cargado por protocolo (http://www.server.com/dir1/dir2/file.htm);

  • Servidor - restringir cargado por nombre de servidor (http://www.server.com/dir1/dir2/file.htm);

  • Directorio - restringir cargado por ruta http://www.server.com/dir1/dir2/file.htm);

  • Nombre de Archivo - restringir carga por nombre de archivo y nombres de vínculo (http://www.server.com/dir1/dir2/file.htmy texto entre etiquetas).

Por favor observe que si una URL no contiene nombre de archivo (como http://www.srv.com/path), debe terminar con el guión "/". De otra manera, "path" serã reconocido como nombre de archivo.

Las secciones de Filtro URL trabajan como grupo. Si un filtro especifica que no se debe descargar un archivo, no serã descargado incluso si otros filtros subsiguientes lo permiten.

Cada filtro permite especificar si cargar dentro del directorio/nombre de archivo/servidor inicio y excluir/incluir palabras clave para ajustar las descargas.

Cuando Portable Offline Browser decide si un vínculo se descargarã o no, primero activa la casilla "Cargar sólo dentro del ... de inicio". Si la casilla estã marcada y el vínculo no se corresponde con la parte inicial de la URL (por ej. el vínculo señala a un servidor externo), el vínculo no serã descargado. Si el link se corresponde con la parte inicial de la URL, entonces se compararã con las listas de palabras clave incluidas/excluidas.

Cargar sólo durante el inicio...

  • ...dominio - Sólo se permite la descarga del archivo si dominio es el mismo, como en la URL inicial (Proyecto | Dirección (URL)). Por ejemplo, si el servidor inicial el www.server.com, se permitirã que se cargue desde cgi.server.com, images.server.com, etc.

  • ...servidor - sólo se permite la descarga del archivo si nombre del servidor es el mismo, como en la URL inicial (Proyecto | Dirección (URL)). Por ejemplo, si el servidor inicial es www.server.com, sólo se permitirã que se cargue desde www.server.com. Se deshabilitarã la carga de cualquier otro servidor.

  • ...Cargar hasta X vínculos desde otros servidores - Las pãginas que estén fuera del servidor o dominio inicial se pueden descargar usando el límite de los otros servidores. Cargar hasta un vínculo significa que Portable Offline Browser cargarã todos los vínculos externos pero no irã mãs allã. Al configurarlo en dos, cargarã pãginas de vínculos externos y todas las pãginas vinculadas a ellas.

  • ...directorio - el directorio de archivo serã descargado si su ruta es igual a o comienza con la ruta en la URL inicial. Por favor observe que si una URL no contiene nombre de archivo (como http://www.srv.com/path), deberã terminar con el guión "/". De otra manera, "path" serã reconocido como nombre de archivo.

Si la casilla "Cargar sólo dentro del ...inicial" no estã marcada, el vínculo serã comparado únicamente con las listas de palabras clave incluidas/excluidas.

Existen dos tipos de palabras clave - habilitar y deshabilitar la carga del archivo si la palabra clave coincide. Hay dos listas por cada tipo de palabra clave.

Portable Offline Browser compara el segmento correspondiente de la URL de cada archivo a descargar con las palabras clave de cada tipo. Si alguna de las palabras clave en la listaIncluircoincide, entonces se permite la descarga del archivo. Si ninguna de las palabras clave coincide, entonces el archivo no se cargarã. Si alguna de las palabras clave en la listaExcluircoincide, entonces no se permite la descarga del archivo. Si ninguna de las palabras clave coincide, entonces el archivo se cargarã. La palabra clave puede incluir cualquier símbolo. Coincidir con una palabra clave significa que el segmento de URL correspondiente (nombre de servidor, ruta o nombre de archivo con extensión) contiene la palabra clave.

Ejemplos: nombre del servidorwww.zdnet.comcontiene la palabra clave zdnet. La ruta /~user015/images/ no contiene la palabra clave user705.

Los dos símbolos especiales "^" y "$" indican el comienzo y el fin del segmento de URL.

Por ejemplo, la palabra clave ^www.cnn significa que el segmento de URL deberã comenzar con www.cnn. La palabra clave .htm$ significa que el segmento de URL deberã terminar con .htm.

Para definir un conjunto de símbolos posibles en una posición, use los caracteres []. Así, la palabra claveg[eo]tcoincide con las palabras get y con got. El símbolo "-" implica un intervalo entre un símbolo y otro.

Por ejemplo, [a-z] significa todos los símbolos entre la a y la z (incluidos). [0-9]significa todos los símbolos numéricos.

Use un asterisco "*" para cualquier número de cualquier símbolos, tales como ^a*.gif$, que significa todos los archivos que empiezan con a y terminan en .gif.

Si desea excluir un directorio, como /ads/, no olvide colocar guiones "/" al principio y al final. De otra forma, otros directorios, como /leads/, también serãn excluidos de la descarga.

Usted puede especificar el nombre del servidor (que comienzan con http://, ftp://, https://, rtsp://, pnm://, mms:// or mmst://) en las palabras clave de Directorio.

También puede especificar URLs completas en las palabras clave de Nombre de Archivo.

Por ejemplo, se permite la siguiente palabra clave en los Filtros URL | Nombre de Archivo | Configuración Personalizada: http://www.srv.com/directory/img[0-9]/*.gifFiltros URL | Directorio | Configuración Personalizada pueden contener la siguiente palabra clave: http://www.zdnet.*/newfiles/

También puede especificar una palabra clave para buscar en nombres de vínculos (texto entre etiquetas): link:someword

Esta palabra clave sólo funcionarã en la sección Filtros URL | Nombre de Archivo.

El ícono de "ãrbol" al lado del campo palabra clave muestra un mapa de los sitios descargados para elegir un nombre de archivo, directorio o servidor como palabra clave. Nota: la comprobación de palabras clave no distingue mayúsculas.

Consejos: Para filtrar URLs que terminan con un guión (nombre de archivo sin especificar), use default.htm en la sección Filtros URL | Nombre de Archivo.

A excepción del Protocolo Filtro URL, las secciones de Configuración Personalizada de cada Filtro URL contienen una lista Incluir y Excluir, y usted puede ingresar cualquier número de palabras clave en esas listas. Si una palabra clave coincide con el segmento de URL correspondiente en una lista Incluir, entonces se descargarã el archivo. Si una palabra clave coincide con el segmento de URL correspondiente en una lista Excluir, entonces no se descargarã el archivo.

La sección configuración personalizada del Protocolo de filtro de URL permite la selección o deselección de cada protocolo directamente.

Algunas teclas útiles para administrar las listas de palabras clave de Filtros URL:

Ctrl-L carga palabras clave desde un archivo de texto a la lista seleccionada.

Ctrl-Mmueve todas las palabras clave en la lista actual a otra (desde Incluido a Excluido, por ejemplo).

Ctrl-S agrega el servidor de inicio, directorio o nombre de archivo a la lista de palabras clave seleccionada. Por ejemplo, cuando desea permitir la carga únicamente desde el directorio de inicio en Filtros URL | Directorios | Configuración personalizada.

Ctrl-C copia todas las palabras clave al portapapeles de Windows.

Ctrl-A marca todas las palabras clave en la lista.

Ctrl-N desmarca todas las palabras clave.

Presione la tecla F2 sobre una palabra clave seleccionada para editarla en allí mismo.

Nota:La configuración de Filtros URL no aplica a archivos cuya categoría Filtros de Archivos no tenga seleccionada "Cargar usando configuración de Filtros URL" en su casilla Ubicación .

También puede usar macros de URL en las palabras clave de Filtros URL. Por ejemplo:

filename{:0day}.htm

Consejo:Para filtrar URLs que terminan con un guión (nombre de archivo sin especificar), use default.htm en la sección Filtros URL | Nombre de Archivo.

Nota: Portable Offline Browser saltea una URL si alguna configuración de los Filtros URL o Filtros de Archivo no permite que sea cargado (incluso si todos los demãs parãmetros configurados permiten que se cargue esa URL).
