Exportar Proyectos

Usted puede Exportar Proyectos a otro directorio o archivo haciendo clic en el botón Exportar de la Cinta de Opciones - Pestaña Inicio. Esta función le permite copiar sitios descargados desde uno o varios Proyectos.

Al exportar sitios cargados, Portable Offline Browser cambiarã los vínculos en archivos HTML, Flash o archivos de secuencias de comandos Java de acuerdo con los nombres nuevos. Puede utilizar la configuración siguiente para ajustar el proceso Exportar:

La lista de la izquierda le permite copiar archivos exportados al directorio especificado, o comprimirlos a ZIP, archivos MHT, visor EXE , archivos de ayuda HTML comprimidos (.CHM) o cargarlos a un servidor FTP.

Los archivos MHT no usan compresión, pero mantienen todos los sitios exportados en un único archivo. Ademãs, se pueden explorar directamente usando MS Internet Explorer.

La ayuda HTML tiene los beneficios de los archivos MHT ademãs de que ocupa mucho menos espacio en disco debido a la compresión.

De todas formas, tanto los archivos MHT como de ayuda HTML sólo se pueden visualizar si tiene instalado MS Internet Explorer 5.0 o superior.

Si selecciona Caché MS Internet Explorer, Portable Offline Browser enviarã los archivos exportados al caché de Internet Explorer y estarãn accesibles en modo "Trabajar sin conexión".

No cree subdirectorios - coloque todos los archivos en un único directorio. Los archivos con el mismo nombre serãn renombrados para evitar conflictos de nombre de archivo.

Si marca la opción Crear CD de Autoarranque, Portable Offline Browser también crearã archivos autorun.inf y oe_open.exe. Los tres archivos, cuando se los ubica en el directorio raíz del CD, hacen que el CD sea autoejecutable -si coloca el disco en la unidad de CD-ROM, el archivo contents.htm se abrirã automãticamente en el explorador.

Exportar únicamente archivos cargados recientemente - copia sólo aquellos archivos que fueron descargados o actualizados durante la última sesión. Si algunos archivos descargados previamente no fueron actualizados, serãn salteados. Desmarque la casilla para exportar todos los archivos pertenecientes al Proyecto.

Mantener tal cual estã copia archivos con sus nombres de archivo originales, lo que estã bien si desea exportar un Proyecto a otro directorio de Windows 95/98/NT/2000/XP.

8+3 - compatible con MS-DOS esta opción es útil al exportar Proyectos a Windows 3.11, disquetes, unidades de red de Novell u otros sistemas de archivo antiguos. Portable Offline Browser cambiarã los nombres de archivo de forma de que no usen mãs de 8 símbolos para la parte de Nombre de Archivo y 3 símbolos para la extensión.

Símbolos Joliet - CD-R 64 excelente para publicar en discos de CD-ROM, ya que estos requieren que cada nombre de archivo no tenga mãs de 64 símbolos.

Uso de extensiones estãndar para archivos con tipos de archivo conocidos - muchos sitios Web contienen archivos estãndar con extensiones que no son estãndar. Por ejemplo, los archivos HTML normalmente usan extensiones .htm o .html. Sin embargo, existen varios casos en que secuencias de comandos CGI-BIN, pãginas ASP, sitios Lotus Domino y muchas otras devuelven contenidos en formato HTML bajo nombres de archivo no estãndar. Las extensiones pueden ser completamente diferentes, tales como .shtml, .nsf, o .asp. Muchos archivos pueden no contener extensión alguna.

Los navegadores Web normales determinan el tipo de contenido de un archivo usando su extensión. Así, un archivo con un tipo de extensión no estãndar serã usado como desconocido, y al explorar sin conexión sin ejecutar Portable Offline Browser, los navegadores pueden preguntar si guardar un archivo en el disco en lugar de mostrarlo.

Para superar este problema, Portable Offline Browser agregarã extensiones estãndar como .htm, .gif, .jpg, .js y otros a los archivos copiados dependiendo de su tipo verdadero.

Deshabilitar vínculos emergentes cambia todos los emergentes para que se abran en la misma ventana del explorador. Se recomienda verificar esta configuración al exportar a ayuda HTML.

Cuando se exporta un Proyecto, Portable Offline Browser crearã un archivo, llamado contents.htm en el directorio con los sitios exportados. Este archivo contiene vínculos a los Proyectos exportados.

Preservar vínculos en línea - si algún archivo HTML descargado tiene vínculos que apuntan en línea, serãn mantenido tal cual, y no serãn traducidos fuera de línea.

Agregar la URL original a archivos HTML -agregar la URL en línea a cada pãgina Web exportada como comentario al contenido de la pãgina. Ademãs, ayuda a solucionar problemas de compatibilidad al explorar archivos bajo Windows XP Service Pack 2 o posterior.

Exportar mãs rãpido mejora la velocidad de anãlisis durante la exportación. Sin embargo, puede que algunos vínculos no funcionen correctamente, especialmente cuando se usa Impedir que los directorios se sobrecarguen (en el diãlogo Opciones). También se saltea la resolución de vínculos y vínculos en archivos .ram.

Abrir archivo cuando se completa la exportación abre automãticamente el archivo exportado cuando se exporta a un directorio, archivo ZIP, MHT, EXE o CHM.

Carga FTP

Requiere que se especifique el servidor. También puede especificar el tipo de directorio al que cargar: ftp://ftp.site.com/path/. Si el puerto servidor no es el 21, especifíquelo de esta forma: ftp://ftp.site.com:8021/

Puede requerir también nombre de usuario y contraseña.

La carga FTP exportarã a un directorio en su disco duro. Utiliza otra configuración de exportación, como formato de nombres de archivo, extensiones estãndar, etc. Cuando se completa la exportación, inicia una utilidad de carga que envía los archivos en el directorio exportado al servidor. También puede querer borrar archivos exportados desde el directorio local cuando la carga se completa. Esto no afecta al servidor remoto.

También es posible exportar Proyectos automãticamente una vez que se descargaron.

Solución de problemas

Si tiene problemas al exportar, tales como que no todos los archivos se exportaron como esperaba, por favor, verifique en el Mapa de Proyecto si se enumeran todos los archivos o no. La exportación sólo funciona con los archivos listados en el Mapa de Proyecto.

Si el Mapa estã incompleto y los archivos de Proyecto descargados estãn en su disco, por favor seleccione el Proyecto y presione Ctrl + F5 en su teclado. Esto comenzarã a descargar el Proyecto en el "Modo no cargar archivos existentes". Portable Offline Browser revisarã todos los archivos descargados y los agregarã al Mapa de Proyecto. Si hay archivos faltantes, serãn descargados.
