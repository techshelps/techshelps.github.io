Se pueden configurar varios parãmetros de conexión a Internet:

    Portable Offline Browser puede descargar entre uno y 500 archivos en forma simultãnea, haciendo que la descarga sea mãs fãcil y rãpida. El número mãs común de conexiones estã entre 10 y 20. Pocas conexiones pueden enlentecer las descargas, mientras que muchas pueden hacerlas poco confiables debido a posible interrupción de conexión. Si cree que sus descargas no son confiables, pero no quiere disminuir el número de conexiones, intente aumentando el valor de Tiempo de interrupción. Puede cambiar este parãmetro en cualquier momento. Si aumenta el número de conexiones mientras descarga, los nuevos canales se agregarãn inmediatamente. Si reduce el número de conexiones, los canales extra no se eliminarãn enseguida, ya que Portable Offline Browser esperarã hasta que los archivos en descarga a través de ellos se completen antes de remover los canales. Advertencia: seleccionar mãs de 100 conexiones simultãneas puede causar inestabilidad del sistema, ya que Windows puede tener problemas con el manejo de tantos sockets TCP/IP abiertos.

  • Número de reintentos

    Las descargas de archivos pueden fallar por varios motivos: servidor no encontrado, la conexión no fue establecida, el archivo no se transmitió completamente, interrupción de conexión, etc. Para recuperarse de estas fallas, ajuste el Número de reintentos en el panel Opciones | Internet | Conexiones para indicar a Portable Offline Browser cuãntas veces debe intentar restablecer una conexión. Si no se define el número de reintentos, Portable Offline Browser reintentarã indefinidamente hasta tener éxito o hasta que se lo detenga manualmente.

  • Demora entre descargas

    Para reducir el trãfico creado por Portable Offline Browser, puede aumentar laDemora entre descargas. Una vez que se cargó un archivo, Portable Offline Browser esperarã el número especificado de segundos antes de intentar recuperar el siguiente. Usted también puede usar demoras al azar para hacer que Portable Offline Browser actúe como una exploración humana. Esto puede ayudar a superar a la protección anti navegadores sin conexión de algunos sitios web. Sólo siga el siguiente formato: 2-10 Lo anterior harã que Portable Offline Browser elija aleatoriamente un retraso en el rango entre 2 y 10 segundos antes de descargar cada archivo.

    Usted puede especificar retrasos no enteros, como 0,23 segundos. O 2,5-5,7.

  • Interrupción de conexión

    Este valor puede ayudar a ajustar sus descargas. Si Portable Offline Browser se encuentra esperando datos desde Internet mãs tiempo que el número de segundos especificado en la configuración del Tiempo de Interrupción de Conexión, la conexión se cortarã. Un número mãs alto de Tiempo de Interrupción de Conexión generalmente indica que algo estã mal con la ruta al servidor. Para remediar un número extremadamente grande de interrupciones, o baje el Número de conexiones (número de canales de descarga) o aumenta el valor del Tiempo de Interrupción de Conexión.

  • Soporte de Cookies

    Verifique este parãmetro para usar cookies de MS IE mientras descargas sitios web. Esto es útil para cargar algunos sitios protegidos que almacenan información del usuario en los archivos de cookie. En este caso, es mejor ingresar el sitio usando MS Internet Explorer y luego cargarlo con Portable Offline Browser. Algunos sitios usan cookies temporales, así que es mejor usar el navegador interno en Portable Offline Browser que entrar al sitio primero. Portable Offline Browser usarã tanto cookies permanentes como temporales del navegador interno.

  • Puerto Servidor HTTP Interno

    Portable Offline Browser incluye un servidor HTTP (web) interno para facilitar ver y compartir pãginas descargadas. Por defecto, el servidor HTTP interno usa el puerto 800. Puede modificar esta configuración si lo desea. Para acceder al servidor desde la computadora donde se estã ejecutando Portable Offline Browser, puede tipear: http://127.0.0.1:800/ en el campo URL de su navegador web. Los usuarios en otras computadoras deberãn conocer la dirección IP de la computadora para acceder a las pãginas descargadas.

  • Identificación de Agente

    A veces es necesario cambiar la forma en que Portable Offline Browser se muestra a otros servidores web. Esto se llama "Agente de usuario". Este es, por defecto, MS Internet Explorer. Portable Offline Browser intentarã identificar la versión de MS IE para usar exactamente la misma representación. Hay algunos valores de agente de usuario predefinidos en el cuadro combinado desplegable. La verdadera representación se muestra en el campo Usar esta identificación. Si necesita usar valores distintos a los predefinidos, elija Usar esta identificación e ingrese el tipo de agente de usuario deseado. El valor del Agente de Usuario no afecta a la mayoría de los sitios web. Sin embargo, puede tener problemas al descargar algunos sitios protegidos, o los documentos HTML pueden ser devueltos con codificación incorrecta hasta que se use el Agente de Usuario correcto (según el sitio web).

  • Sólo acceso local

    Cuando este parãmetro estã activo, Portable Offline Browser impedirã que otras computadoras accedan al servidor HTTP Interno (web). Esto puede usarse por seguridad.

    Al desactivar este parãmetro, podrã compartir con otros los sitios web que ha descargado.
