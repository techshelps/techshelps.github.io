¿Cómo especificar un remitente personalizado, cookie o petición POST?

Usted puede especificar una solicitud POST, cookie o remitente en el campo URL debajo de la URL de inicio (en una línea aparte).

Ejemplos:

Solicitud de POST:

http://www.metaproducts.com/

POST=parameter=value&param2=somevalue

Cookie:

http://www.metaproducts.com/

Cookie=some_value

Remitente:

http://www.metaproducts.com/

Remitente=http://www.othersite.com/

Si necesita especificar el mismo valor de Remitente en todas las URLs que Portable Offline Browser descargarã de un sitio, use el parãmetro GlobalReferer= .

También puede usar Macros de URL en estos parãmetros. Por ejemplo:

http://www.metaproducts.com/

POST=parameter={:day}

Si se especifica una macro de enumeración en la solicitud POST, Portable Offline Browser cargarã varias direcciones URL con cada una de las peticiones POST enumeradas. Por ejemplo:

http://www.metaproducts.com/

POST=parameter={:0..2}

El parãmetro Cookie= funciona de la misma forma

El anterior cargarã 3 http://www.metaproducts.com/ URLs pero no con tres peticiones POST diferentes. Para asegurarse de que los archivos descargadas no se sobrescriben unos a otros debido al nombre de archivo, por favor, use Copias de Archivo en el diãlogo Propiedades de Proyecto.

Cómo ejecutar un ejecutable externo

Otra función es la capacidad de ejecutar un archivo ejecutable externo una vez que finaliza la descarga del Proyecto:

http://www.metaproducts.com/

RunAfter=C:\Windows\scandisk.exe
