Microsoft Replication Manager

You can use Microsoft Replication Manager to:

  • Manage a large number of replicas.

  • Support laptop users who aren't always connected to a network.

  • Easily create replicas of more than one database.

  • Set schedules for synchronizing members of the replica set.

  • Troubleshoot errors.

Replication Manager provides a visual interface for designating Design Masters, creating replicas, viewing the connections between members of a replica set, and setting the properties for replicas.

Note   If you move or copy a global replica that is a hub replica for local or anonymous replicas by using any tool other than the Move Replica command in Replication Manager, you will not be able to synchronize the global replica to the local or anonymous replicas in the replica set. And the local or anonymous replicas will not be able to synchronize with the global replica that created them.
