Stale Code Warning Dialog Box

This dialog box appears when you have made code changes that Edit and Continue could not immediately apply. As a result, some code in the current stack frame is now out of date (stale).

Don't show me this dialog again

If you select this checkbox, Edit and Continue will apply code changes without asking permission in the future. You can turn this warning on again by going to the Options dialog box, opening the Debugging Folder, Edit and Continue category, and selecting Warn about stale code.
