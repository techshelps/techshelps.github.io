Debugging Multiple Programs

With Visual Studio.NET, you can debug multiple programs in a single solution. There are two ways to debug multiple programs in the Visual Studio debugger:

  • Start debugging the first project, and then start the additional project from Solution Explorer.
  • Start debugging the first project, and then attach to a program that is running outside of Visual Studio.

Attaching is discussed in another section.

To start a project in Solution Explorer

  1. In the Solution Explorer window, select the project you want to start debugging.
  2. Right-click the project name or icon.
  3. From the shortcut menu, choose Debug and click Step Into.

When you are debugging two or more or more projects in a solution, you can switch back and forth between them in either of two ways:

To switch between projects while debugging

  • In the Solution Explorer window, select the project you want to switch to.

    -or-

  • On the Debug Location toolbar, select the program you want to switch to from the Program list box.

Switching to a project makes it the chosen project for debugging purposes. Any command you select from the Debug menu (such as Break or Continue) will affect the chosen project.
