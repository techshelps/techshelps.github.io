Global and Non-Global Hosted Applications
 

From a life cycle management perspective, UII has two types of hosted applications: global hosted applications and session-based or non-global hosted applications. A global hosted application starts when the Integrated Agent Desktop starts or at the start of a workflow. A non-global hosted application is started either when the session starts or at the start of a workflow if the application is defined to be workflow-dependent. It is usually recycled at the end of the session.

Note:
By default, the application will be a session-based application.
