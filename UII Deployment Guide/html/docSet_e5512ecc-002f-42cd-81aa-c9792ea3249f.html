Dynamic Hosted Applications
 

UII also supports dynamic hosted applications. This allows an agent to start or close a hosted application. It also allows a developer to programmatically open or close a hosted application. A dynamic application can be global-dependent, session-dependent, or workflow-dependent.

If a dynamic application is part of a workflow, the workflow will start the application, and the application will stop when the workflow completes.

There are two states for hosted applications:

  • Active hosted application – An active hosted application is one that is enabled and ready for use. (See the following illustration.)


  • Inactive hosted application – An inactive hosted application is not enabled; therefore, it cannot be used.
