Managing a Workflow
 

This section explains the workflow that controls the Integrated Agent Desktop. Each workflow can be either forced or unforced. A forced workflow is one in which the steps must be executed in a predetermined sequence, and the step execution cannot end until the final step is completed. In an unforced workflow, the steps must follow a prescribed order, but the operation can be completed at any time. Either workflow type can be canceled at any time. Cancellation stops all transactions and no data is saved or stored. Workflows can be mapped to individual users or to teams.
