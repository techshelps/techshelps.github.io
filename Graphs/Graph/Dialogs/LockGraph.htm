Lock Graph

When a graph is open, select Lock Graph from the Tools menu or click the Lock graph button in the Graph Options dialog - Graph: Window options pane to access the Lock Graph dialog.

Prevent changes to. Use the options in this box to specify the aspects of the graph that should be locked. When users try to change a locked feature via the user interface, a message will be displayed, informing them that the graph is locked. When an automation operation is attempted on a locked graph, a dispatch execution will be generated indicating that the operation is locked.

Graph Options dialog. Select the Graph Options dialog check box to prevent changes to the settings in the Graph Options dialog (accessible by double-clicking in the background of the graph). When the Graph Options dialog is locked, users are unable to change any graph options that are accessed in the Graph Options dialog (e.g., titles/legends, scaling, plot layout, graph window, etc.). If users try to access these options, they will be prompted to first unlock the graph.

Titles/legends. Select the Titles/legends check box to prohibit the modification or addition of graph titles and legends.

Objects. Select the Objects check box to prevent users from changing existing graph objects or adding additional graph objects.

Appearance. Select the Appearance check box to keep users from making changes to the plot markers or plot colors.

Axes scales. Select the Axes scales check box to prevent all changes to axis scaling. When this check box is selected, users will not be able to change the axis scaling. Additionally, the Zoom in, Zoom out, and Zoom Pan features will be locked (e.g., the buttons will be disabled).

Graph data. Select the Graph data check box to prohibit changes to the graph data. When this check box is selected, users can still display the Graph Data Editor; however, they cannot modify the data in the editor. When graph data are locked, auto updates from spreadsheets will not be applied. Note that locking the graph data does not prohibit case states (i.e., the graph can still be brushed and any spreadsheet brushing will be displayed in the graph). To prevent changes to the graph as a result to spreadsheet case state changes, select the Brushing check box.

Brushing. Select the Brushing check box to prevent users from using the brushing features. When this check box is selected, the Brushing dialog cannot be displayed and changes to cases states of the connected spreadsheet are confined to the spreadsheet and not propagated to the graph.

Enter password. Enter a password to use when locking and unlocking the graph. Although a password is not required, it is strongly recommended. If a password is not entered and confirmed, then any user can unlock graph features by simply clearing the check boxes in the Prevent changes to box. Note that if locks have already been defined, you must enter the correct password before locks can be changed or modified.

Confirm password. Re-enter the specified password to confirm it in the system. Note that this option is only displayed when a password is created.

OK. Click OK to accept the changes you have made and return to the graph.

Cancel. Click Cancel to close the dialog without making any changes.
